# 信息学竞赛课程计划（基于实际课时约束）

## 课时约束

| 阶段 | 上课频率 | 上课时长 | 课后练习 |
|------|---------|---------|---------|
| 入门级（阶段1-2） | 1次/周 | 90分钟 | 5次/周，40分钟/次 |
| 提高级（阶段3-4） | 1次/周 | 90分钟 | 5次/周，90分钟/次 |

---

## 总体规划

```
第一年（入门级）                 第二年（提高级）                 第三年（冲刺+进阶）
│                               │                               │
│ Q1 Q2 Q3 Q4                   │ Q1 Q2 Q3 Q4                   │ Q1 Q2 ...
│                               │                               │
├─ C++语言基础 ┤                ├─ 数据结构深入 ┤               ├─ NOI 级选修 ┤
├─ 基础算法 ───┤                ├─ 图论 ────────┤               ├─ 省选/国赛备考┤
                   ─ 参赛 CSP-J              ─ 参赛 CSP-S
                                                    ─ 参赛 NOIP/省选
```

### 时间估算

- **入门级**：约 40 次课（约一学年，9-10个月，除去假期）
- **提高级**：约 50-60 次课（约 1.5 学年）
- **NOI 级**：根据目标选择性学习

## 可行性评估

### ✅ 可行的理由

1. **时间够用**：第一年40次课覆盖入门到CSP-J水平，节奏合理。每周5次练习能保证手感。
2. **入门期练习40分钟/次刚好**：注意力持续时间适合，不容易疲劳。
3. **提高期练习90分钟/次必要**：一道线段树/网络流题从理解到AC，30-60分钟很正常。
4. **第二年末刚好冲CSP-S**：如果从9月开始学，第二年10月正好参加CSP-S。

### ⚠️ 需要注意

1. **寒暑假要加量**：仅靠每周1次课不够。寒暑假建议 **每周2-3次课**，否则进度会慢。
2. **练习频次 > 练习时长**：入门期宁可每天30分钟也别周末3小时。编程手感需要每日维护。
3. **因人而异**：上面的进度适合 **中等偏上** 的学生。天赋好的可以快1.5倍，基础薄的可能要慢0.7倍。
4. **每个学生的"卡点"不同**：有人卡在递归，有人卡在DP，有人卡在图论。必须及时发现、针对辅导。
5. **不建议完全按这个进度走**：把它当路线图，实际进度根据学生掌握情况灵活调整。宁可慢一点把基础打牢。

### 📊 关键建议

- **入门级**：保证 600道题的底线。达不到这个量，CSP-J一等奖基本没戏。
- **提高级**：保证 500道 + 5次以上模拟赛。题量不是绝对的，但要确保每种经典算法都至少练过3-5道不同变体。
- **最重要的不是上课，是课后练习的执行力。** 这点怎么强调都不过分。

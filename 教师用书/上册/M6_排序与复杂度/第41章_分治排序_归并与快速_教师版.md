### 第 1 课时：分治思想 + 归并排序（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | "一堆 1000 张乱序卡片怎么排？一个人排太累——分成两堆各 500 张，两个人各排一堆，然后把两堆有序卡片合并。"引出分治思想：**分解→解决→合并** |
| 8-25 分 | 讲授 | 归并排序三步：①分——递归把数组分成两半直到只剩1个元素 ②治——单个元素自然有序 ③合——把两个有序数组合并成一个。板书画合并过程（两个指针分别指向两个有序数组头部，每次取较小的） |
| 25-38 分 | 讲授 | 时间复杂度推导：每层合并 O(n)，共 log n 层→O(n log n)。空间 O(n)（需要临时数组）。归并排序是**稳定**的（合并时相等元素取左边优先） |
| 38-45 分 | 小结 | "归并排序 = 分一半 → 各自排 → 合并。O(n log n)，稳定，但要额外 O(n) 空间。" |

### 第 2 课时：归并代码 + 逆序对（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-15 分 | 🎮 不插电活动 | **"两支有序队伍合并"**（详见活动区）。两组已排好序的学生合并成一组大有序队伍——每人指挥"两组头部比较，小的出列" |
| 15-30 分 | 讲授 | 板书 merge 函数完整代码。关键点：①临时数组 `tmp[]` ②三 while 循环（两组都有元素→取小的；左边剩了→全倒；右边剩了→全倒）③把 tmp 拷回原数组 |
| 30-42 分 | 讲授 | **逆序对**：在 i<j 时 a[i]>a[j] 的 (i,j) 对数。归并排序可以顺带计算：在合并过程中，当右半部分的元素被选中时，左半部分剩余元素都大于它→累加 `mid-i+1`。这是经典的分治应用 |
| 42-45 分 | 小结 | "逆序对计数 = 归并排序的一个附加产品。从 O(n²) 暴力到 O(n log n) 分治——又是 n 到 log n 的飞跃。" |

### 第 3 课时：快速排序（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | "归并排序需要额外空间 O(n)。能不能原地排序且 O(n log n)？" |
| 10-30 分 | 讲授 | 快排思想：选 pivot→把小于 pivot 的放左边，大于的放右边→递归排左右。板书划分过程：两个指针 i 和 j，i 找大于 pivot 的，j 找小于 pivot 的，交换。演示一次完整划分 |
| 30-40 分 | 讲授 | 快排复杂度：平均 O(n log n)，最坏 O(n²)（pivot 选到最大/最小）。为什么 STL sort 用了快排变体（Introsort）——混合快排+堆排防止最坏情况。快排**不稳定**（划分时元素可能跳跃） |
| 40-45 分 | 小结 | "快排 = 选 pivot → 分左右 → 递归。原地 O(1) 空间，平均 O(n log n)，不稳定。STL sort 底层就是它的优化版。" |

## 🔥 重点难点

### 教学重点

1. **归并排序的合并**：三 while 循环 + tmp 数组拷回——这是归并排序的核心。
2. **逆序对计数**：`ans += mid - i + 1`——右半元素被选中时累加左半剩余个数。这是分治思想最漂亮的应用之一。
3. **快排的划分**：pivot 选择 + 两个指针扫描——理解划分就理解了快排。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 归并排序忘了拷回 | tmp 算对了但原数组没变 | 合并函数末尾必须加 `for(i=l;i<=r;i++) a[i]=tmp[i];` |
| 逆序对 `mid-i+1` 不理解 | 不知道为什么是这个数 | 画图：左右各 [2,5,8] 和 [1,3,6]。当右 1 被选中时，左剩 [2,5,8]（3 个）都大于 1 |
| 快排 pivot 选 a[l] 遇到有序数组 | 退化 O(n²) | 改进：随机选 pivot 或三数取中。但竞赛中直接用 sort，手写快排通常是教学目的 |

## 💬 常见问答

**Q1："归并排序这么好（稳定、永远 O(n log n)），为什么 sort 不用它？"**

A：因为归并需要 O(n) 额外空间。sort 的实现是 Introsort（快排+堆排混合），原地 O(log n) 额外空间，平均 O(n log n)。

**Q2："竞赛中要手写归并排序吗？"**

A：很少直接考"写归并排序"。但**用归并排序求逆序对**是经典考题，因为 sort 本身不能计算逆序对。这是你要重点掌握的模式。

**Q3："快排和归并排序，哪个在实际中更快？"**

A：快排常数小（原地操作，缓存友好），通常比归并快。但归并适合外部排序（数据在磁盘上）和链表排序。

**Q4："为什么归并排序是稳定的？"**

A：合并时 `if(a[i] <= b[j])` 取左边的——相等元素优先取左边，保持了原顺序。如果写成 `<` 就会破坏稳定性。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 8 分）

1. 分治算法的三个步骤是 __________、__________、__________。
2. 归并排序的空间复杂度是 __________。
3. 快速排序在最坏情况下的时间复杂度是 __________。

### 二、读代码（6 分）

4. 以下 merge 中计算逆序对的代码，`ans += mid - i + 1` 应在什么条件下执行？
```cpp
while (i <= mid && j <= r) {
    if (a[i] <= a[j]) tmp[k++] = a[i++];
    else { tmp[k++] = a[j++]; /* ans += ??? */ }
}
```

### 三、写代码（16 分）

5. 实现归并排序的合并函数 `void merge(int a[], int l, int mid, int r)`。

---

### 答案

1. 分解，解决，合并
2. O(n)
3. O(n²)
4. 在 `a[i] > a[j]` 时（else 分支）累加——因为左半 a[i..mid] 都大于 a[j]
5. 略（三 while + 拷回原数组）

# 第 38 章  算法的速度——复杂度分析（教师用书）

---

## 🎯 教学目标

| 维度 | 目标 |
|------|------|
| **知识** | 解释时间复杂度的定义（程序运行时间随数据规模增长的趋势）；背出常见复杂度等级 O(1)/O(log n)/O(n)/O(n log n)/O(n²)/O(n³)/O(2ⁿ)；区分最好/最坏/平均情况 |
| **能力** | 通过数循环层数和每次缩小比例估算复杂度；一眼看出嵌套循环的复杂度；忽略常数项和低阶项提取大 O；用复杂度判断算法在给定数据范围下是否可行 |
| **素养** | 建立"先估复杂度再写代码"的条件反射——拿到题先算数据范围能承受多大复杂度，再选择算法 |

## 🗺️ 教学流程

### 第 1 课时：为什么需要复杂度 + 基本等级（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | 讲小明和小红的故事——同样判断重复数，双重循环 10 秒，排序后比较 0.1 秒。"同一个功能，速度差 100 倍！不看代码漂亮不漂亮，看它跑得多快。"引出核心问题：**能不能不写代码就知道算法快不快？**
| 10-25 分 | 讲授 | 时间复杂度定义：程序运行时间随数据规模 n 增长的趋势。三种基本模式：①O(n)——单层循环，n=10⁶ 秒出 ②O(n²)——双重循环，n=10⁵ 就危险 ③O(log n)——每次翻倍/减半，n=10⁹ 也秒出。每讲一种都板书对应代码和执行次数推导 |
| 25-35 分 | 演示 | IDE 现场跑对比：同样处理 n=50000，O(n) 循环瞬间完成，O(n²) 双重循环明显卡顿。让学生肉眼感受 "n 和 n² 不是差一点，是天壤之别" |
| 35-42 分 | 练习 | 口头快问：分析 `for(i=1;i<=n;i*=2)` 的复杂度（O(log n)）；分析 `for(i=1;i<=n;i++) for(j=1;j<=i;j++)` 的复杂度（1+2+...+n = O(n²)） |
| 42-45 分 | 小结 | "复杂度 = 算法的身份证。以后每学一个新算法，第一个问题就是——它的复杂度是多少？" |

### 第 2 课时：大 O 记号体系（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 回顾 | 快速复习三种基本复杂度，提问"`while(x<n) x*=2` 多少次？" |
| 8-22 分 | 🎮 不插电活动 | **"复杂度排队"**（详见活动区）。学生手持复杂度卡片，按增长速度从慢到快排队：O(1)→O(log n)→O(n)→O(n log n)→O(n²)→O(n³)→O(2ⁿ) |
| 22-35 分 | 讲授 | 大 O 记号规则：①只保留增长最快的项 ②忽略常数系数。举例：3n²+5n+100→O(n²)，n+log n→O(n)，1000000→O(1)。板书"复杂度阶层图"——纵轴是运行时间，横轴是 n，画出各等级的曲线，让学生看到 n 变大时曲线的"分道扬镳" |
| 35-42 分 | 练习 | 给 10 个表达式，让学生提取大 O（如 `2n³+n²`→O(n³)，`n+1000000`→O(n)，`5×2ⁿ+n¹⁰⁰`→O(2ⁿ)） |
| 42-45 分 | 小结 | 口诀："**最高次项定天下，常数系数全扔掉。对数比线快，线比平方快，指数是魔鬼。**" |

### 第 3 课时：嵌套循环 + 最好/最坏/平均（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 温故 | 复习大 O 记号规则，快速出几题让学生抢答 |
| 10-25 分 | 讲授 | 嵌套循环复杂度推导：①内外层独立→乘起来（O(n×m)）②内层依赖外层→求和（O(n²/2)=O(n²)）③循环变量每次翻倍→O(log n)。三种模式各给一道例题，板书画出循环轨迹 |
| 25-35 分 | 讲授 | 最好/最坏/平均情况：以"在数组里找目标值"为例——最好情况目标在第一位 O(1)，最坏在最后或不存在 O(n)，平均 O(n/2)=O(n)。强调：**竞赛中通常看最坏情况**，除非题目明确说数据随机 |
| 35-42 分 | 讨论 | 分组讨论：冒泡排序的最好/最坏/平均复杂度分别是什么？（最好 O(n) 已有序，最坏 O(n²) 完全逆序，平均 O(n²)） |

| 时间 | 环节 | 操作 |
|------|------|------|

| 10-25 分 | 讲授 | 空间复杂度：衡量程序占用内存随 n 增长的规律。`int a[n]`→O(n)，`int a[n][n]`→O(n²)，递归调用栈→O(递归深度)。CSP-J 通常内存限制 256MB = 约 6×10⁷ 个 int。大部分 O(n) 空间都安全，O(n²) 在 n>5000 时危险 |
| 25-38 分 | 独立练习 | 学生完成复杂度分析练习题（给代码段→判断时间/空间复杂度）。典型陷阱：①把 `for(i=1;i<=n;i+=i)` 误判为 O(n) ②忘了 STL sort 是 O(n log n) ③递归程序漏算调用栈的空间 |
| 38-45 分 | 总结 | "复杂度是一把尺子。从下一章开始，每学一种排序算法，我们都先用复杂度这把尺子量一量——它适合什么数据范围、什么场景。" |

## 🔥 重点难点

### 教学重点

1. **大 O 记号忽略常数和低阶项**：学生最难接受的是"3n 和 n 在复杂度上没区别"。用 n=10⁶ 时 3n=3×10⁶ 和 n²=10¹² 的对比来说服。
2. **对数复杂度的识别**：循环变量每次 ×2 或 while(n>1) n/=2 → O(log n)。区分 `i+=i`（对数）和 `i+=2`（线性）。
3. **数据范围→复杂度反推**：n≤30→O(2ⁿ)可，n≤10³→O(n²)可，n≤10⁵→O(n log n)可，n≤10⁷→O(n)可。这个速查表是竞赛第一反应。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 分不清 O(log n) 和 O(n/2) | 把 `i*=2` 判为 O(n/2) | 画表对比：n=10⁶ 时 O(log n)=20 次，O(n/2)=5×10⁵ 次——差两万倍。**对数 = 每次翻倍/减半，不是每次减固定值** |
| 内层依赖外层的循环不会分析 | `for(i=1;i<=n;i++) for(j=1;j<=i;j++)` 直接判为 O(n²) | 正确结果确实是 O(n²)，但需要推导：1+2+...+n = n(n+1)/2 = O(n²)。让学生手动加一遍小 n 的值，体会"等差数列求和" |
| 混淆"时间复杂度"和"实际运行时间" | "我的 O(n) 程序比他的 O(n) 程序慢" | 演示：O(n) 里常数不同（比如循环里有 10 行代码 vs 1 行代码），但 n 变大时两者的差距不变，而 O(n²) 会飞速拉开。**复杂度是"趋势"，不是"秒数"** |
| 递归复杂度不会分析 | 不知道递归深度和每层工作量 | 画递归树：汉诺塔 n 层→2ⁿ-1 次移动=O(2ⁿ)。阶乘→O(n)。斐波那契朴素递归→O(2ⁿ)。每画一棵树，复杂度一目了然 |

## 💬 常见问答

**Q1："为什么忽略常数？O(2n) 和 O(n) 明明差了 2 倍！"**

A：因为当 n=10⁶ 时，2n=2×10⁶，而 O(n²)=10¹²。2 倍的差距在百万和万亿面前完全微不足道。大 O 关注的是"n 翻倍时，运行时间翻多少倍"——O(n) 翻 2 倍，O(n²) 翻 4 倍，O(2ⁿ) 翻指数倍。至于常数 2 还是 3——随着 n 增长，这种固定倍数的差距越来越不重要。

**Q2："O(log n) 中的 log 是 log₂ 还是 log₁₀？"**

A：在计算机科学中默认是 log₂。因为很多算法"每次减半"——二分查找每次把范围 ÷2，循环次数就是 log₂ n。但好消息是：换底公式告诉我们 log₂ n 和 log₁₀ n 只差一个常数倍，而大 O 忽略常数——所以 O(log₂ n) = O(log₁₀ n) = O(log n)。**只写 log 就行，不用纠结底数。**

**Q3："空间复杂度为什么也要关心？程序不是内存很大吗？"**

A：竞赛内存通常 256MB≈6×10⁷ 个 int。`int a[10000][10000]` 需要 10⁸×4=400MB——直接 MLE！特别是二维数组和递归（调用栈也占内存），空间超限和超时一样是零分。

**Q4："什么时候复杂度分析看平均情况而不是最坏？"**

A：竞赛中几乎永远看最坏情况——因为出题人会特意构造最坏数据来卡你。唯一的例外是随机化算法（如随机快排），它的"最坏"几乎不可能出现，此时讨论"期望复杂度"更有意义。

**Q5："`for(i=1;i<=n;i++) for(j=1;j<=30;j++)` 复杂度是多少？"**

A：O(30n) = O(n)。因为内层只跑 30 次——这是常数，不是随 n 增长的。**只有随 n 变化的循环层才算复杂度**。这个知识对后续很多"看起来像两重循环但实际是 O(n)"的场景很重要。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 单层 for 循环从 1 到 n，时间复杂度是 __________。
2. `while (n > 1) { n /= 2; }` 的时间复杂度是 __________。
3. 大 O 记号中，3n² + 5n + 100 简化为 __________。
4. CSP-J 中 1 秒大约能执行 __________ 次基本操作。
5. 空间复杂度衡量的是 __________。

### 二、分析题（每题 5 分，共 10 分）

6. 分析以下代码的时间复杂度：
```cpp
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= i; j *= 2)
        cout << i + j;
```

7. n=10⁵ 时，以下哪些复杂度的算法可以通过 1 秒时限？（多选）
A. O(n)  B. O(n log n)  C. O(n²)  D. O(2ⁿ)

### 三、写代码 + 分析（10 分）

8. 写一个程序判断一个数 n 是否是质数。分别写 O(n) 版和 O(√n) 版，说明 n=10¹² 时哪个能跑。

---

### 答案

1. O(n)
2. O(log n)
3. O(n²)
4. 10⁷~10⁸
5. 程序占用内存随数据规模增长的趋势
6. 外层 O(n)，内层 O(log i)≤O(log n)。总复杂度 O(n log n)。
7. A 和 B。O(n²)=10¹⁰ 超时，O(2ⁿ) 指数爆炸。
8. O(n) 版：`for(i=2;i<n;i++) if(n%i==0)...`；O(√n) 版：`for(i=2;i*i<=n;i++) if(n%i==0)...`。n=10¹² 时 O(n) 需要 10¹² 次操作≈10000 秒；O(√n) 只需 10⁶ 次操作≈0.01 秒。

## 📦 补充题库

1. `for(i=1;i<=n;i++) for(j=n;j>=1;j/=2)` 的复杂度？（O(n log n)——外层 O(n)，内层 O(log n)）

## 🔥 重点难点

### 教学重点

1. **cmp 返回值规则**：`return a < b` 表示 a 排在 b 前面。这是最容易写反的地方——学生经常想"我要降序"然后写 `return a < b`。
2. **严格弱序**：cmp 必须满足严格弱序——`a < a` 永远为 false。写成 `<=` 会导致 sort 内部崩溃（违反反自反性）。
3. **结构体数组排序**：这是竞赛中最实用的技能——成绩排名、任务排序、区间调度全用到。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| cmp 返回值写反 | 想降序写了 `return a < b` | 口诀："**想让 a 在前就 return a<b（升序），想 a 在后就 return a>b（降序）**"。在 cmp 旁边注释"a 排 b 前面时返回 true" |
| cmp 用 `<=` 导致崩溃 | 运行时报错或死循环 | 强调：**cmp 里永远用 `<` 或 `>`，不能用 `<=`、`>=`、`==`**。这是 C++ 标准的规定 |
| vector 的 sort 写错 | `sort(v, v+n)` 而不是 `sort(v.begin(), v.end())` | 对比记忆：数组用地址 `a, a+n`，vector 用迭代器 `v.begin(), v.end()` |
| 多关键字 cmp 漏掉次级比较 | 两个学生成绩相同时顺序随机 | cmp 里必须处理"相等"情况：`if(a.x!=b.x) return ...; return a.y < b.y;`——先判不等，相等了再往下比 |

## 💬 常见问答

**Q1："sort 和 stable_sort 什么时候用哪个？"**

A：大多数场景用 sort（更快）。需要保证稳定性（相等元素保持原序）时才用 stable_sort。例如"按成绩排，但成绩相同的保持输入顺序"——如果你用 sort，必须 cmp 里增加"输入顺序"作为第三关键字；用 stable_sort 只需先按输入顺序标号再 stable_sort 按成绩排。

**Q2："sort 的时间复杂度是 O(n log n)，log 的底是 2 吗？"**

A：是的——sort 底层是快排变体（Introsort），划分约 log₂ n 层。但正如上一章所说，大 O 记号中 log 的底不重要。

**Q3："cmp 能引用外部变量吗？比如按与 target 的距离排序。"**

A：传统 cmp 函数不能捕获外部变量。解决方案：①把 target 设为全局变量 ②用 C++11 的 lambda：`sort(a,a+n, [&](int x,int y){ return abs(x-target)<abs(y-target); });`。lambda 的 `[&]` 可以引用外部变量。

**Q4："sort 能排 string 吗？"**

A：能！`string` 默认按字典序比较，`sort(str.begin(), str.end())` 可以把字符串内部字符排序。`sort(arr, arr+n)` 对 string 数组排序是按字典序排字符串。

**Q5："结构体排序除了写 cmp 函数，还有别的办法吗？"**

A：可以重载 `<` 运算符：`bool operator<(const Student &o) const { return score > o.score; }`。这样 `sort` 不加 cmp 也能按你定义的规则排。cmp 函数更灵活（不同的排序需求用不同的 cmp），运算符重载更简洁（结构体自带排序规则）。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 6 分）

1. `sort` 函数定义在 __________ 头文件中。
2. 默认情况下 `sort(a, a+n)` 按 __________ 序排列。
3. cmp 函数的规则是：a 应该排在 b __________ 时返回 true。

### 二、改错题（8 分）

4. 以下 cmp 函数想按成绩降序排序，有什么问题？
```cpp
bool cmp(Student a, Student b) {
    if (a.score >= b.score) return true;
    return false;
}
```

### 三、写代码（16 分）

5. （8 分）写一个 cmp，使数组按"奇偶分开，奇数在前偶数在后，各自内部从小到大"排序。
6. （8 分）定义结构体 `Book{string name; int price; int pages;}`，写 cmp 先按价格升序，价格相同按页数降序。

---

### 答案

1. `<algorithm>`
2. 升
3. 前面
4. ①`>=` 违规（严格弱序），应用 `>` ②缺少 return false（a.score < b.score 的情况没覆盖）
5. `if(a%2!=b%2) return a%2>b%2; return a<b;`
6. ```cpp
bool cmp(Book a, Book b) {
    if (a.price != b.price) return a.price < b.price;
    return a.pages > b.pages;
}
```

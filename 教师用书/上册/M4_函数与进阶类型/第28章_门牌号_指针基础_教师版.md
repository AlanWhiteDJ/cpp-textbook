## 🗺️ 教学流程

### 第 1 课时：内存地址概念 + 取地址 + 不插电活动（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 老师在黑板上画一排方格，标注"门牌号"：101、102、103... 每个方格里写一个数。"每个变量在内存里都有一个门牌号——这就是**地址**。你知道了门牌号，就能找到这个变量。" |
| 8-18 分 | 🎮 不插电活动 | **"快递寻址"**（详见活动区）。每张课桌贴一个地址标签（&变量名），桌仓里放一张写着变量值的卡片。学生当快递员，拿到地址后找到对应的课桌，执行操作（读值/改值） |
| 18-30 分 | 讲授 | **`&` 取地址运算符**：`int a = 10; cout << &a;`——输出 a 在内存中的地址（一串十六进制数如 `0x7ffd1234`）。每次运行地址可能不同（操作系统分配），但概念不变。"&a 就是问：你家门牌号是多少？" |
| 30-38 分 | 演示 | 老师用 IDE 连续声明几个变量（`int a, b, c;`），打印它们的地址。学生会发现地址是连续递减或递增的——证实了"变量在内存里是紧挨着放的" |
| 38-45 分 | 讨论 | "知道地址有什么用？"——让学生自由讨论 3 分钟。老师总结：①可以通过地址间接访问变量 ②函数可以通过地址修改外部变量 ③数组名本身就是地址 |

### 第 2 课时：指针声明 + 解引用（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | "`&a` 返回什么？"——a 的地址。"地址存在哪里？"——引出指针变量 |
| 5-18 分 | 讲授 | **指针变量**：专门用来存地址的变量。声明语法：`int *p;`——`p` 是一个指针，它指向一个 `int`。**`*` 的双重含义**：①声明时：`int *p` 表示"p 是指针类型" ②使用时：`*p` 表示"p 指向的那个变量的值"。板书：`int a = 10; int *p = &a; cout << *p;`——输出 10。`*p = 20; cout << a;`——输出 20（通过指针修改了 a） |
| 18-28 分 | 代码演示 | 老师现场演示：指针的基本四个操作——①声明指针 ②让指针指向某个变量 ③通过指针读值 ④通过指针写值。强调安全习惯：声明指针时如果暂时不知道指向谁，初始化为 `nullptr`（C++11）或 `NULL` |
| 28-38 分 | 对比辨析 | 板书表格：`int a = 10; int *p = &a;`——`a` 的值是什么？（10）`&a` 的值是什么？（a 的地址）`p` 的值是什么？（存了 &a，即 a 的地址）`*p` 的值是什么？（p 指向的变量的值，即 10 = a）`&p` 的值是什么？（指针变量 p 自己的地址——指针也是变量，也有自己的地址） |
| 38-45 分 | 练习 | 纸上做题：给你 `int x = 5, y = 8; int *p1 = &x, *p2 = &y;` 执行 `*p1 = *p2; *p2 = 3;` 后 x 和 y 分别是多少？（x=8, y=3）|

### 第 3 课时：指针与数组 + 指针运算（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 回顾+导入 | "数组名是什么？"——`int arr[5]; cout << arr;` 输出一个地址！数组名就是数组**首元素的地址**。`arr` 等价于 `&arr[0]` |
| 8-22 分 | 讲授 | **指针遍历数组**：`int *p = arr;` 让 p 指向 arr[0]。`*p` 就是 arr[0]。`p++` 让指针向后移动一个元素（指向 arr[1]）。代码演示：`for (int *p = arr; p < arr + 5; p++) { cout << *p << " "; }`——用指针遍历数组。`arr + i` 等价于 `&arr[i]`，`*(arr + i)` 等价于 `arr[i]` |
| 22-30 分 | 讲授 | **指针加减法规则**：指针 +1 不是地址值 +1，而是地址值 + `sizeof(类型)`。`int *p; p+1` 的地址比 p 大 4 字节（int 占 4 字节）。`double *p; p+1` 的地址比 p 大 8 字节。这保证了 `p+1` 刚好指向下一个元素 |
| 30-42 分 | 上机 | 学生上机实验：①用指针遍历打印数组 ②用指针实现数组逆序（两个指针，一头一尾，交换后向中间移动）③打印数组中每个元素的地址，验证它们确实是连续的 |
| 42-45 分 | 总结 | "指针和数组的关系极为紧密。但小心：指针可以乱指，数组名不能乱改——`arr++` 是错的，因为数组名是**常量指针**。" |

### 第 4 课时：指针与函数 + 综合练习（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 复习 | "我们已经学过两种让函数修改外部变量的方法——引用传递。指针是第三种方法。" |
| 8-22 分 | 讲授 | **指针作为函数参数**：`void swap(int *a, int *b) { int t = *a; *a = *b; *b = t; }`。调用：`swap(&x, &y);`——把 x 和 y 的地址传给函数。对比三种方式：①值传递（改不了）②引用传递（`int &a`）③指针传递（`int *a`）。指针方式最灵活但也最容易出错（忘写 `&` 或忘写 `*`） |
| 22-35 分 | 讲授 | **用指针让函数返回多个值**（引用方式的替代方案）：`void divide(int a, int b, int *quot, int *rem) { *quot = a / b; *rem = a % b; }`。对比与引用方式：指针需要传地址（`&q`），函数内用 `*` 访问 |
| 35-45 分 | 练习 | 学生完成：①用指针参数写 `void minMax(int a, int b, int *min, int *max)` ②写 `void swapIfGreater(int *a, int *b)`——如果 *a > *b 则交换 |

### 第 5 课时：陷阱 + 小结 + 小测（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 陷阱精讲 | **空指针**：`int *p = nullptr;`——指针不指向任何变量。对空指针解引用（`*p`）会崩溃。**野指针**：指针声明了但没初始化——`int *p; *p = 5;`，p 指向一个随机地址，修改它可能破坏内存。**悬垂指针**：指针指向的变量已经被销毁（如函数返回了局部变量的地址）。口诀："**指针三不：不空解、不野奔、不悬垂**" |
| 10-20 分 | 对比总结 | 板书：引用 vs 指针对照表。引用：声明用 `&`，必须初始化，不能改指向，用法同原变量。指针：声明用 `*`，可以不初始化（危险），可以改指向，用法要用 `*` 和 `&`。简化：**引用 = 别名（安全但不够灵活），指针 = 门牌号（灵活但危险）** |
| 20-35 分 | 小测 | 7 道题（填空+判断+代码分析+写代码），闭卷 15 分钟 |
| 35-45 分 | 互批+展望 | 同桌互批。"下一章我们学文件操作——这是让你的程序能'记住东西'的关键技术。" |

## 🔥 重点难点

### 教学重点

1. **`*` 和 `&` 的双重含义**：必须根据**上下文**区分。声明时 vs 使用时含义不同。这是指针学习的第一道坎。
2. **指针与数组的关系**：`arr[i]` = `*(arr + i)`，`&arr[i]` = `arr + i`。理解了这个，数组和指针就通了。
3. **指针安全三原则**：初始化、判空、不越界。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|------|
| 混淆声明 `*` 和运算 `*` | `int *p = &a;` 和 `*p = 5;` 里的 `*` 分不清 | 规则：`*` 前面有**类型名**（`int`、`double`等）→ 这是声明，"p 是指针"。`*` 前面没有类型名 → 这是解引用，"取 p 指向的值"。同类型的 `*` 绝对不要出现在同一个位置让学生混淆 |
| `&` 的三重含义 | 取地址 `&a`、引用声明 `int &r`、按位与 `a & b`——三个 `&` 让学生崩溃 | 明确告知：C++ 设计者复用了符号。**看位置分辨**：①前面有类型 → 引用声明 ②前面没有类型，后面是变量 → 取地址 ③在两个整数之间 → 按位与（M2 学过） |
| 指针运算的"步长" | 学生以为 `p+1` 就是地址值 +1 | 类比：指针不是按"米"走路，是按"房间大小"走路。int 的房间是 4 平方米，指针每次跨一个房间。double 的房间是 8 平方米。`p+1` 永远跳到下一个房间的门口，不管房间有多大 |

## 💬 常见问答

**Q1："指针真的有必要学吗？大家都说指针很难。"**

A：指针是 C/C++ 的灵魂。不学指针 = 学了一半的 C++。核心原因：①动态内存分配需要指针（`new`/`delete`）②高效的数组/字符串操作依赖指针 ③数据结构的实现（链表、树）全靠指针。而且，一旦你真正理解了指针，你会发现它并不"难"——它就跟"门牌号"一样自然。难的不是指针，是模糊的认知。

**Q2："什么时候用指针，什么时候用引用？"**

A：简单原则——①如果你只是想给变量起个别名，用引用（`int &r = a;`）。②如果你需要"不指向任何东西"（空状态），或者需要"改变指向"，用指针。③函数参数想要修改外部变量：优先考虑引用（语法更简洁），但如果这个参数可能"没有值"（可选），用指针（传 `nullptr` 表示"没有"）。

**Q3："`int *p` 和 `int* p` 有区别吗？"**

A：对编译器没区别，但阅读习惯有差异。`int* p` 强调"`int*` 是一种类型"——更符合 OOP 思维。`int *p` 强调"`*p` 是一个 int"。两者都没错，但一行声明多个变量时要小心：`int* p, q;`——p 是指针，q 是普通 int（不是指针）。所以建议每行只声明一个指针变量，或者用 `int *p, *q;`。

**Q4："指针变量的地址（`&p`）和它存的地址（`p`）有什么区别？"**

A：`p` 是一个指针变量，它自己也在内存中有地址（`&p`）。`p` 的**值**是另一个变量的地址。类比：你口袋里有一张写着朋友家门牌号的纸条——你的口袋有自己的位置（`&p`），纸条上写的是朋友家的门牌号（`p` 的值）。去找朋友：先看纸条上的地址（`p`），再按地址找过去（`*p`）。

## 📋 本章小测（15 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 声明一个指向 int 的指针变量 p：__________。
2. 获取变量 a 的地址：__________。
3. 通过指针 p 访问它所指向的变量的值：__________。
4. 如果 `int a = 10; int *p = &a;`，执行 `*p = 20;` 后，a 的值是 __________。
5. 让指针 p 指向数组 arr 的首元素：__________。

### 二、判断题（每题 3 分，共 9 分）

6. `int *p; *p = 5;` 是正确的用法。（ ）
7. 数组名 arr 是一个常量指针，不能修改它的值（如 `arr++` 是非法的）。（ ）
8. 未初始化的指针（野指针）解引用可能导致程序崩溃。（ ）

### 三、代码分析（每题 6 分，共 12 分）

9. 以下代码输出什么？
```cpp
int a = 10, b = 20;
int *p = &a;
*p = b;
cout << a << " " << b;
```

10. 以下代码输出什么？
```cpp
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr + 2;
cout << *p << " " << *(p - 1);
```

### 四、写代码（9 分）

11. 写一个函数 `void doubleValue(int *p)`，将 p 指向的变量的值翻倍。

---

### 答案

1. `int *p;`
2. `&a`
3. `*p`
4. 20
5. `p = arr;`（或 `p = &arr[0];`）
6. ×（p 未初始化，野指针）
7. ✅
8. ✅

## 🎮 不插电活动

### 活动 1：快递寻址（第 1 课时用，约 12 分钟）

**道具**：
- 便利贴若干（写地址标签："101 &score"、"102 &name"、"103 &age"）
- 小卡片若干（写变量的值："score = 95"、"name = 张三"、"age = 14"）
- 透明胶带

**场地布置**：
- 选 6 张课桌，每张课桌贴一个地址标签（如 `101 &score`）
- 在课桌抽屉里放入对应的值卡片（初始值）
- 剩余的便利贴作为"快递单"

**流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 介绍规则："每张课桌就是一个内存单元，桌上的标签是地址，抽屉里的卡片是变量的值。现在请 6 位快递员上台。" |
| 2-5 分 | 老师→快递员 | **任务一（读值）**：老师给每个快递员一张"快递单"，上面写着地址如 `&score`。快递员根据地址找到对应课桌，从抽屉里取出值卡片，大声读出并交给老师。老师问："这叫直接访问（`a`）还是间接访问（`*p`）？"——间接访问！ |
| 5-8 分 | 老师→快递员 | **任务二（改值）**：老师给新的快递员一张快递单，上面写着 `&age 改成 15`。快递员找到课桌，把抽屉里旧卡片拿出，放入新值卡片。老师问："如果程序中另一个函数通过 `age` 读取这个值，读到的是什么？"——15！"这就是通过指针修改变量。" |
| 8-10 分 | 老师 | **对比演示**：老师走到 `&score` 课桌前，直接打开抽屉看卡片——"这是直接访问（`score`）"。再拿起一张快递单走到课桌前——"这是间接访问（`*p`）。区别：间接访问需要先拿到地址。" |
| 10-12 分 | 全班讨论 | "如果快递员拿到的地址是错的（比如 `999`，没有课桌），会怎样？"——找不到！这就是**野指针**——指针指向了一个不存在的地址。 |

### 活动 2：指针接力——数组遍历（第 3 课时用，约 8 分钟）

**道具**：6 张课桌连续排成一行，贴地址标签 `1000 &arr[0]` 到 `1005 &arr[5]`，抽屉里放值卡片

**流程**：

| 分钟 | 操作 |
|------|------|
| 0-2 分 | 一个学生当"指针 p"，站在第一张课桌前（p = &arr[0]）。读出抽屉里的值（`*p`） |
| 2-4 分 | 老师喊"p++！"——p 向后移动一张课桌（站在 &arr[1] 前）。读出新值 |
| 4-6 分 | 继续 p++，依次读出 arr[2]、arr[3]……直到 arr[5] |
| 6-8 分 | 总结："`p++` 就是快递员向后走一步到下一家的门口。不是走 1 米，是走 1 个房间。" |

## 📦 补充题库

1. 以下代码有什么问题？
```cpp
int *p;
*p = 100;
```
（参考答案：p 是野指针——声明了但没初始化，指向随机地址。对随机地址写入 100 非常危险。应改为 `int a; int *p = &a; *p = 100;`。）

2. 以下代码输出什么？
```cpp
int a = 5, b = 10;
int *p1 = &a, *p2 = &b;
p1 = p2;
*p1 = 20;
cout << a << " " << b;
```
（参考答案：`5 20`。`p1 = p2` 让 p1 也指向 b，之后 `*p1 = 20` 修改的是 b。a 没被影响。）

3. 写一个函数 `void swap(int *a, int *b)` 用指针交换两个整数的值，并通过 `main` 调用验证。（参考答案：`int t = *a; *a = *b; *b = t;`，调用 `swap(&x, &y);`。）

4. 用指针实现：给定数组 `int arr[] = {1,2,3,4,5}`，用指针逆序打印。（参考答案：`int *p = arr + 4; while (p >= arr) { cout << *p << " "; p--; }`）

5. "指针的指针"是什么？`int **pp` 是什么意思？（参考答案：`pp` 是指向"指向 int 的指针"的指针。即 `pp` → `p` → `a`。二维数组名本质上就是指针的指针。这是进阶内容，初学可暂不深究。）

| 0-8 分 | 🎮 不插电活动 | **"教室规则"**（详见活动区）。老师宣布："1 班的班长只能管 1 班的事，不能跑到 2 班发号施令。同理，函数里的变量只能在自己的函数里使用。"指定不同学生扮演不同"班级"（函数），每个班级选出"班长"（局部变量），班长在自己的班级区域内活动 |

| 38-45 分 | 小结 | 口诀："**在哪出生（声明），在哪活动，出了门就消失。**"课后思考："有没有变量可以管全校？" |

### 第 2 课时：全局变量 + 作用域优先级（45 分钟）

|------|------|------|
| 0-8 分 | 回顾+导入 | 复习局部变量。老师问："如果需要一个所有函数都能用的变量，怎么办？"学生可能会说"用参数传来传去"——老师说："那如果有 10 个函数都要用呢？每次传 10 个参数？有没有更简单的办法？" |
| 8-20 分 | 讲授 | **全局变量**：在所有函数外面（通常在最前面）声明的变量。特点：①从声明处开始到文件结束，所有函数都能访问 ②程序启动时"出生"，程序结束时"死亡"。板书示例：`int globalScore = 0;` 放在所有函数上方，`addScore()` 和 `printScore()` 都能读写它 |

| 38-45 分 | 练习 | 口头判断：①全局 x=1, f() 里 x=2，输出？(2) ②全局 x=1, g() 里没有声明 x 但用了 x，输出？(1) ③如果非要在局部作用域里访问被"遮蔽"的全局变量怎么办？（引出 `::x`，简单提一下不做要求） |

## 🔥 重点难点

### 教学重点

1. **局部变量 vs 全局变量**：作用域差异是本章核心。局部有界、全局无界——但全局有风险。
2. **生命周期**：作用域 = "在哪能用"，生命周期 = "活多久"。两个概念相关但不完全相同（static 就是例外：作用域小但生命长）。
3. **同名变量的遮蔽规则**：就近原则。学生必须能判断"这个位置的 `x` 指的是哪个 `x`"。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 不理解"同名变量可以独立存在" | 学生震惊于"两个函数可以有同名变量 x" | 类比：两个班级都有"班长"这个职位，1 班的班长和 2 班的班长是两个人——名字一样，但不是同一个人 |
| 滥用全局变量 | 学生把所有变量都声明在全局，说"这样方便" | 反问："你家的钥匙放在门口地毯下面确实方便，但小偷也方便。全局变量就是门口钥匙——你的任何一个函数（甚至将来别人写的函数）都能改它。你的程序足够大时，出了 bug 你都不知道是谁改的。" |
| 混淆作用域和生命周期 | 学生认为"出了作用域变量就立刻被销毁" | 用 `static` 打破这个误解：`static` 局部变量出了作用域不能用，但它**没有死**——下次进作用域还能用，值还保留着 |

## 💬 常见问答

**Q1："既然全局变量这么危险，为什么 C++ 还允许它？"**

A：因为有些场景全局变量是合理的——比如游戏里的最高分记录、程序的配置信息（语言、主题、音量）。这些数据确实需要被多个函数访问。关键不是"能不能用"，而是"谨慎地用"：①尽量用 `const` 全局常量替代全局变量（只读不写最安全）②确实需要全局变量时，用清晰的命名（如 `g_score`）来标注它是全局的。

**Q2："`static` 局部变量和全局变量到底有什么区别？"**

A：**可访问范围不同**。全局变量——所有函数都能碰；`static` 局部变量——只有它所在的函数能碰。但它们活得一样长（程序开始到结束）。所以 `static` = 全局的生命 + 局部的权限。这是"最小权限原则"的体现——即使你需要一个活得久的数据，也不一定要让所有人都能碰它。

**Q3："函数里的参数算局部变量吗？"**

A：算。函数参数的作用域就是整个函数体，生命周期也是函数执行期间。它跟函数体内第一行声明的变量没有本质区别——只是参数的值在调用时由外部传入。

**Q4："不同的文件里可以有同名的全局变量吗？"**

A：这涉及到"链接属性"——多文件编程的内容，属于进阶话题。简单说：默认情况下，如果在两个 `.cpp` 文件里分别定义了 `int x;`，链接时会报重复定义错误。但可以用 `static` 或 `namespace` 解决。这个知识点一般在学到多文件项目时再深入。

| 分钟 | 角色 | 操作 |

## 📦 补充题库

1. 以下代码能否通过编译？如果不能，为什么？
```cpp
void f() {
    int x = 10;
}
void g() {
    cout << x;
}
```
（参考答案：不能。x 是 f 的局部变量，g 中不可见。）

2. 以下代码输出什么？
```cpp
int x = 5;
int main() {
    int x = 10;
    {
        int x = 20;
        cout << x << " ";
    }
    cout << x << " ";
    cout << ::x;
}
```
（参考答案：`20 10 5`。最内层块 x=20，出了块用外层局部 x=10，`::x` 访问全局 x=5。）

3. 写一段代码验证：在 `for` 循环的初始化部分声明的变量，循环结束后不能再使用。（参考答案：`for (int i=0; i<10; i++) { } cout << i;` 编译报错。）

4. 以下代码中，`a` 被修改了几次？每次谁改的？
```cpp
int a = 0;
void f() { a++; }
void g() { int a = 5; a++; }
int main() { f(); g(); f(); }
```
（参考答案：全局 a 被改了 2 次（两次 `f()` 调用）。`g()` 改的是自己局部的 a，不影响全局。）

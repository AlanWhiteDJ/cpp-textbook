## 🗺️ 教学流程

### 第 1 课时：为什么要用函数 + 函数定义格式（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 老师在黑板上写这样一段"伪代码"：`唱生日歌的代码` × 10 遍。问："如果我要在 10 个地方唱生日歌，我要抄 10 遍代码吗？" 引出函数概念——**给一段代码起名字，以后叫名字就行** |
| 8-20 分 | 讲授 | 函数定义格式四要素：①返回类型（`int`/`void`/`double` 等）②函数名（命名规则同变量）③形参列表（`(int a, int b)`）④函数体（`{ ... }`）。板书示例：`int add(int a, int b) { return a + b; }`。逐一拆解每个部分 |
| 20-30 分 | 类比 | **"函数 = 榨汁机"**。参数 = 放进去的水果（苹果、橙子），函数体 = 榨汁过程，`return` = 倒出来的果汁。`void` 函数 = 碎纸机——放进去东西，什么都不吐出来 |
| 30-38 分 | 演示 | 老师现场用 IDE 写一个 `void sayHello()` 和一个 `int add(int, int)`，编译运行。让学生观察：sayHello 没有 `return`，add 有 `return` |
| 38-45 分 | 小结 | 口诀：**"类型名 括号参 花括号里写实现"**。布置思考题："如果函数没有 `return` 会怎样？" |

### 第 2 课时：函数调用 + return 详解（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速提问："函数的四个组成部分是什么？" |
| 5-15 分 | 讲授 | **函数调用**：`add(3, 5)`——把 3 和 5 传给函数，拿到返回值 8。关键点：调用时实参的**顺序**必须和定义时的形参一一对应。`add(3, 5)` 和 `add(5, 3)` 在这个函数里结果一样（加法交换律），但在 `subtract(a, b)` 里结果完全不同 |
| 15-25 分 | 讲授 | **`return` 的两大作用**：①返回一个值给调用方；②结束函数执行。`return` 之后的代码永远不会被执行。板书示例：一个函数里有 `return` 之后再写 `cout`——那是"死代码" |
| 25-35 分 | 互动练习 | 老师出题："写一个函数 `max(a, b)`，返回较大的那个数。"学生在纸上写出函数定义。老师巡场，挑两个典型错误投影讲解 |
| 35-42 分 | 纠错 | 常见错误：①调用了函数但忘了接收返回值（`add(3,5);` 而不是 `int r = add(3,5);`）②在 `void` 函数里写 `return 0;` ③定义了函数但从来没调用过 |
| 42-45 分 | 总结 | "记住：定义函数 = 造榨汁机，调用函数 = 用榨汁机。光造不用等于白干。" |

## 🔥 重点难点

### 教学重点

1. **函数定义的四要素**：返回类型 + 函数名 + 形参列表 + 函数体。这是整个 C++ 函数体系的根基，必须让学生烂熟于心。
2. **形参与实参的区别**：形参是定义时的"占位符"，实参是调用时的"具体值"。这个区分决定了后续值传递/引用传递的理解基础。
3. **`return` 的双重作用**：很多学生只记住 `return` 是"返回结果"，不知道它同时会**终止函数**。这对后续理解分支语句中的 `return` 至关重要。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 函数调用忘写括号 | `add 3, 5;` 或 `add();` 只写一个括号 | 让学生想想：如果不写括号，编译器怎么知道 `add` 是函数还是变量？没有括号就不知道要执行它。口诀：**"函数不括号，等于没喊到"** |
| 混淆函数的"定义"和"调用" | 学生问"我定义了函数它为什么不运行" | 类比：你造了一台榨汁机（定义），但你不放水果按开关（调用），它永远不会动。定义 ≠ 执行 |
| 把 `return` 和 `cout` 混为一谈 | 以为输出到屏幕就是返回 | 严格区分：`cout` 是给**人**看的（显示在屏幕上）；`return` 是给**调用者**用的（把值传回给程序）。可以在屏幕上显示 42 的同时返回 0——两件事 |
| 试图在函数里定义另一个函数 | `int f() { int g() { ... } }` | 直接告诉学生：C++ 语法规定函数必须平级定义，不能嵌套。这是规则，记住就行。如果需要一个函数在内部调用另一个函数——用函数调用，别用嵌套定义 |

## 💬 常见问答

**Q1："什么时候该把一段代码写成一个函数？"**

A：两个标准——①这段代码会被用到**两次及以上**（避免重复）；②这段代码有**独立的功能意义**（即使只用一次，但逻辑完整独立，起个名字更好理解）。比如"检查密码是否正确"——即使程序里只检查一次，写成函数 `checkPassword()` 也比把代码直接塞进 `main` 清晰得多。

**Q2："`void` 函数有什么用？不返回值那写它干嘛？"**

A：很多函数不需要返回值——它们靠"副作用"工作。比如 `printMenu()` 的作用是在屏幕上打印菜单，你不需要它返回什么值，打印本身就是目的。再比如游戏里的 `playerJump()`——角色跳起来就行了，不需要返回一个数字给你。

**Q3："一个函数里能写好几个 `return` 吗？"**

A：能，但只有**第一个被执行到的** `return` 会生效。比如在 `if-else` 里：`if (x > 0) return 1; else return -1;`——两个 `return` 都有用，但每次调用只会命中其中一个。如果所有 `return` 都在同一条执行路径上，后面的 `return` 是死代码。

**Q4："为什么有的函数定义写在 `main` 上面，有的写在下面但在上面写一行声明？"**

A：C++ 编译器从上往下读代码。如果你在 `main` 里调用 `add`，但 `add` 的定义在 `main` 下面，编译器会说"我不认识 `add`"。解决方案：把完整的 `add` 定义挪到 `main` 上面，或者在 `main` 上面写一行**函数原型**（也叫函数声明）：`int add(int, int);`——这相当于提前给编译器打个招呼："后面会有一个叫 `add` 的函数，长这样"。

## 📋 本章小测（15 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 定义函数 `int square(int n)` 中，`int` 是函数的 __________，`square` 是 __________，`int n` 是 __________。
2. 如果函数不需要返回任何值，返回类型应写为 __________。
3. 调用函数时传入的具体值称为 __________。

### 二、改错题（每题 5 分，共 10 分）

4. 以下代码有两处错误，请找出并改正：
```cpp
void main() {
    int add(int a, int b) {
        return a + b;
    }
    cout << add(3, 5);
}
```

5. 以下代码有什么问题？
```cpp
int divide(int a, int b) {
    return a / b;
    cout << "计算完毕";
}
```

### 三、写代码（每题 10 分，共 20 分）

6. 写一个函数 `int absolute(int n)`，返回 n 的绝对值。
7. 写一个函数 `bool isLeapYear(int year)`，闰年返回 `true`，否则返回 `false`。（闰年规则：能被 4 整除但不能被 100 整除，或者能被 400 整除）

---

### 答案

1. 返回类型，函数名，形参
2. `void`
3. 实参
4. 两处错误：① `void main()` 应为 `int main()`（或 `int main`）②函数 `add` 不能嵌套定义在 `main` 内部，应提到 `main` 外面
5. `return` 之后的 `cout` 永远不会执行（死代码），应把 `cout` 移到 `return` 之前
6. ```cpp
int absolute(int n) {
    if (n < 0) return -n;
    else return n;
}
```
7. ```cpp
bool isLeapYear(int year) {
    if (year % 400 == 0) return true;
    if (year % 100 == 0) return false;
    if (year % 4 == 0) return true;
    return false;
}
```

## 🎮 不插电活动

### 活动 1：我是函数机器（第 3 课时用，约 20 分钟）

**道具**：
- 两个纸盒（鞋盒大小），分别贴上"📥 输入箱"和"📤 输出箱"标签
- 一叠空白便签纸或纸条（至少 30 张）
- 一支马克笔

**场地布置**：
- 讲台中央放一把椅子（函数机器工位）
- 椅子前面放"输入箱"，侧面放"输出箱"
- 讲台黑板上写下当轮的函数签名，例如：`int doubleMe(int x)`

**规则与流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-1 分 | 老师 | 讲解规则："现在这个同学就是一台'函数机器'。你们排队把数字放进输入箱，他取出数字、执行运算、把结果放进输出箱，你们从输出箱取走结果。" |
| 1-3 分 | 老师 | 指定一名学生坐上讲台，担任"函数机器"。给该生一张"任务卡"，上面写着函数规则（第一轮：`x × 2`） |
| 3-8 分 | 全体 | **第一轮（×2）**：全班排队，每人用便签写一个数字（如 3、7、15、42），放进输入箱。函数机器同学依次取出便签、心算结果、写在新便签上、放入输出箱。提交者取走输出箱里的结果 |
| 8-10 分 | 老师 | 第一轮复盘："同学们给你的数字是什么？"——实参。"黑板上写的是什么？"——形参。"你执行的操作是什么？"——函数体。"你放回输出箱的是什么？"——返回值 |
| 10-12 分 | 老师 | 更换任务卡（第二轮：`x × 3 + 1`），换一个学生当函数机器。全班再玩一轮 |
| 12-15 分 | 全体 | **第二轮（×3+1）**：重复排队-递纸条-计算-返回流程 |
| 15-18 分 | 老师 | 更换任务卡（第三轮：判断奇偶，奇数返回"奇"、偶数返回"偶"）。强调：这次返回类型变了——不是数字，是"字符串"（让学生体会返回类型的概念） |
| 18-20 分 | 老师 | 总结："函数 = 输入箱（形参）→ 处理（函数体）→ 输出箱（return）。你们今天排队的过程就是'函数调用'。每次调用可以给不同的数字（实参），函数执行的操作永远一样。" |

**教学要点**：
- 如果函数机器算错了，老师大声说："函数有 bug！"——让学生笑的同时记住 bug 概念
- 如果输入箱里同时有很多纸条，强调"函数一次只能处理一个调用，多个调用要排队"
- 第三轮故意让返回类型变化（数字→文字），引出"返回类型必须预先声明"的重要性

### 活动 2：函数抽签（第 4 或第 5 课时用，约 10 分钟）

**道具**：一个抽签筒（可用粉笔盒代替），内放写有函数签名的纸条

**规则**：
- 纸条上写函数签名如 `int max(int a, int b)`、`void greet()`、`double area(double r)` 等
- 学生抽签后，要在 2 分钟内口头说出：①这个函数的功能（合理即可）②一个调用示例 ③调用后的预期返回值
- 其他同学判断回答是否正确

**教学要点**：训练学生"读函数签名猜功能"的能力——这是阅读他人代码的基础技能。

## 📦 补充题库

1. 以下函数调用哪个是正确的？（函数定义为 `int calc(int x, int y)`）  
A. `calc(3);` B. `calc(3, 5, 7);` C. `int r = calc(3, 5);` D. `calc();`  
（答案：C。实参个数必须与形参匹配）

2. 函数 `int mystery(int n) { return n % 10; }` 的功能是什么？（参考答案：返回 n 的个位数字。）

3. 以下代码输出什么？
```cpp
int f(int x) { return x + 1; }
int main() {
    cout << f(f(f(0)));
    return 0;
}
```
（参考答案：3。f(0)=1, f(1)=2, f(2)=3）

4. 写一个函数 `int digitSum(int n)`，返回 n 的各位数字之和。例如 `digitSum(123)` 返回 `1+2+3=6`。（参考答案：使用循环，每次取 `n%10` 加到 sum，然后 `n/=10`。）

5. 小明写了一个函数 `int add(int a, int b) { a + b; }`，调用 `cout << add(3, 5);` 却输出了一串奇怪的数字，为什么？（参考答案：函数声明返回 `int`，但函数体里只写了 `a + b;` 没有 `return`。没有 `return` 语句的函数会返回一个不确定的值——即"垃圾值"。应改为 `return a + b;`。）

## 🗺️ 教学流程

### 第 1 课时：值传递原理（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | **"复印vs原件"演示**——老师拿出一张写了数字"A4 纸（变量），说："我把这张纸复印一份交给课代表，让他在复印件上把数字加 10。"课代表在复印件上修改。老师问："我手里原件的数字变了吗？"——没变。引出：**值传递 = 复印一份给函数** |
| 8-20 分 | 讲授 | 用板书画出内存示意图：`int a = 10;` 在内存中占一个格子。调用 `f(a)` 时，**另开一个格子**存 a 的值（10），这个新格子叫形参。函数操作的是新格子，原格子纹丝不动 |
| 20-30 分 | 代码演示 | 老师现场写代码：`void change(int x) { x = 999; }` 然后在 `main` 里 `int a = 10; change(a); cout << a;`——输出 10，不是 999。让学生亲眼看到：**函数改不了外面的变量** |
| 30-38 分 | 讨论 | "这个设计好还是不好？"——分组讨论 3 分钟后汇报。老师总结：好（保护数据不被误改，安全），但有时不好（当我们就想让函数改外部变量时，值传递做不到） |
| 38-45 分 | 预告 | "下一节我们学另一种传递方式——引用传递。它能把原件直接给函数。" |

### 第 2 课时：引用传递 + 不插电活动（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | "值传递时，形参改了，实参会变吗？"——全班齐答"不会！" |
| 5-20 分 | 🎮 不插电活动 | **"复印vs原件"升级版**（详见活动区）。老师准备一份"文件"（A4 纸打印，上面写变量名和值）。第一轮：值传递——老师复印文件，把复印件给学生，学生在复印件上修改，原件不变。第二轮：引用传递——老师直接把原件给学生（文件上贴便利贴写着"原件，请直接修改"），学生直接在原件上修改，交回后原件已变 |
| 20-30 分 | 讲授 | 引用传递语法：`void change(int &x) { x = 999; }`——多了一个 `&` 符号。**`&x` 的意思是"x 是外面那个变量的别名/外号"，不是新变量**。此时调用 `change(a)`，a 变成 999 |
| 30-38 分 | 对比 | 板书对照表：值传递 `(int x)`：实参→形参是**复制**，函数体内改 x 不影响实参；引用传递 `(int &x)`：实参→形参是**绑定**，函数体内改 x 直接影响实参 |
| 38-45 分 | 快速练习 | 口头判断：① `void f(int x)` 调用 `f(a)` 后 a 变不变？② `void g(int &x)` 调用 `g(a)` 后 a 变不变？③ `void h(int a)` 调用 `h(b)`，函数里改了 a，b 变不变？ |

| 时间 | 环节 | 操作 |
|------|------|------|

| 10-25 分 | 案例教学 | **案例 1**：`int square(int n)`——算平方。只需要读 n 的值，不需要改 n。用什么传递？→值传递。**案例 2**：`void swap(int &a, int &b)`——交换两个变量的值。必须修改外部变量。用什么传递？→引用传递。**案例 3**：`void printArray(int arr[], int n)`——打印数组。数组不需要被修改。但数组很大，复制一份太浪费。→用引用？（先埋个伏笔，数组默认就是传地址） |

## 🔥 重点难点

### 教学重点

1. **值传递 ≠ 引用传递**：值传递复制数据，函数操作副本；引用传递传递变量本身，函数操作原件。这是 C++ 最核心的机制之一。
2. **`&` 的上下文含义**：出现在类型声明里（`int &x`）表示引用；出现在变量前（`&a`）表示取地址。前者是"给变量起外号"，后者是"问你家门牌号"。务必一开始就严格区分。
3. **引用参数作为输出**：打破"函数只有一个返回值"的思维定式。引用参数让函数拥有多个"出口"。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 忘记写 `&` | 写了 `void swap(int a, int b)` 然后困惑"为什么交换不了" | 让学生先说出来："swap 需要改外面的 a 和 b，所以必须传引用。"然后检查代码里有没有 `&`——如果有这句话但没有 `&`，那就是 bug。形成"要改 → 加 &"的肌肉记忆 |
| 不清楚什么时候用引用 | 全部传引用，或者全部传值 | 对照精简口诀："我只读你的（值传递），我要改你（引用传递），你很大我不想复印（const 引用传递）" |
| 把 `int &x` 里的 `&` 和 `&a`（取地址）混淆 | 学生困惑"为什么引用也要写 & 号" | 明确告诉学生：C++ 的设计者偷懒复用了 `&` 符号，它有两个完全不同的含义。**看位置**：在类型后面（`int &`）= 引用声明；在变量前面（`&a`）= 取地址。就像"长"字有两个读音一样 |

## 💬 常见问答

**Q1："既然引用传递这么好用，为什么不全用引用，还要值传递？"**

A：因为**安全**。值传递给你一个副本，你在函数里随便改都不会影响外面的变量。这就像你借给别人一份复印件而不是原件——他爱怎么涂改都行，你的原件完好无损。对于只需要"读"数据的函数（比如 `max`、`isEven`），用值传递最安全。

**Q2："能不能给引用参数传一个字面量，比如 `swap(3, 5)`？"**

A：不能。引用需要一个"有家"的变量——它要绑定到内存中一个真实存在的地址。字面量 3 和 5 是临时值，没有固定地址，编译会报错。不过如果参数是 `const int &x`（常量引用），可以传字面量——编译器会先创建一个临时变量存这个值，再绑定引用。

**Q3："怎么判断一个函数参数该用值传递还是引用传递？"**

A：问自己三个问题：①这个参数需要被函数**修改**吗？→是：引用传递。②不需要修改，但它**体积很大**（比如结构体、大数组）？→是：const 引用传递。③不需要修改，体积也小（int、double、char）？→值传递。

**Q4："值传递和引用传递，哪个更快？"**

A：对于小类型（int、char、bool），值传递甚至可能更快（直接传值，不需要间接访问地址）。对于大类型（struct、string），引用传递快得多（只传地址，不复制全部数据）。但初学阶段不要执着于"快不快"，选**正确**的传递方式比选快的更重要。bug 的代价远大于性能的微小差距。

## 🎮 不插电活动

### 活动：复印 vs 原件（第 2 课时用，约 15 分钟）

**道具**：
- 一张 A4 纸，用马克笔写上变量名和初始值，如 `int score = 85`
- 便利贴若干
- 复印机或预先准备好的复印件（如果现场没有复印机，用两张外观相同的纸代替——一张标注"原件"，一张标注"复印件"即可）

**流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 展示原件："这是一张成绩单，score = 85。现在我需要一个函数 `addBonus` 来加分。同学们注意看两次操作的差别。" |
| 2-5 分 | 老师+学生 | **第一幕——值传递**：老师把原件复印一份，把复印件递给一名学生（扮演函数）。说："请在复印件上把分数改为 95。"学生照做。老师举起原件："我手里的原件变了吗？"全班："没有！"老师："这就是**值传递**——函数拿到的是复印件，随便改，原件不受影响。" |
| 5-8 分 | 老师 | 便利贴上写"& 引用——直接修改原件"，贴到原件上。再次举起原件："这次，我把原件直接给你——这就是**引用传递**。" |
| 8-11 分 | 老师+学生 | **第二幕——引用传递**：老师把贴了便利贴的原件直接递给另一个学生（重新准备好 score=85 的纸）。说："请你在上面把分数改为 95。"学生照做后交回。老师举起修改后的纸："这次原件变了吗？"全班："变了！" |
| 11-15 分 | 老师 | 总结对比，在黑板上画两个方框：左边写"值传递 = 复印 → 改复印件 → 原件不变"；右边写"引用传递 = 给原件 → 改原件 → 原件变了"。强调："`&` 符号就是那张便利贴——它告诉编译器'这不是复印件，是原件'。" |

**教学要点**：
- 两轮演示之间，确保原件恢复初始值，让学生看到"同样的初始值，不同的传递方式，不同的结果"
- 可以加一个"恶作剧环节"：值传递时，让扮演函数的学生故意在复印件上乱画，然后问："原件也被乱画了吗？"——没有！强化值传递的安全性
- 如果有学生问"那有没有既想保护原件又想省复印纸的办法"——这正是 `const &` 的伏笔，可以简单提一句"后面的章节会学到"

## 📦 补充题库

1. 以下代码输出什么？
```cpp
void mystery(int &a, int b) {
    a = a + b;
    b = a - b;
}
int main() {
    int x = 5, y = 3;
    mystery(x, y);
    cout << x << " " << y;
}
```
（参考答案：`8 3`。a 是引用，改了 x；b 是值传递，改不了 y。）

2. 判断以下场景该用值传递还是引用传递：①求一个数的绝对值 ②把两个数从小到大排好 ③打印一个数组的全部元素 ④把一个字符串反转（参考答案：①值传 ②引用传（要修改）③值传或 const 引用 ④引用传）

3. 小明写了 `void addOne(int &n) { n = n + 1; }`，然后在 main 里写 `addOne(5);`，编译报错。为什么？怎样修改让 `addOne(5);` 合法？（参考答案：非 const 引用不能绑定字面量 5。改为 `int x = 5; addOne(x);` 或把参数改为 `int n`（值传递）。）

4. 写一个函数 `void swapIfGreater(int &a, int &b)`，如果 a > b 则交换两个数，否则保持不变。调用后 a 一定不大于 b。（参考答案：同本章小测第 9 题。）

## 🗺️ 教学流程

### 第 1 课时：递归概念 + 阶乘 + 不插电活动（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | **"俄罗斯套娃"**——老师拿出一个俄罗斯套娃（如果没有实物，用 PPT 图片）。"打开最大的，里面有个小一点的；再打开，里面又有个更小的……直到最小的那个打不开了——这就是递归。" 引出：**递归 = 函数自己调用自己，每次处理更小的问题，直到小到可以直接解决** |
| 8-13 分 | 🎮 不插电活动 | **"俄罗斯套娃拆解"**（详见活动区）。邀请 5 个学生站成一排，由高到矮。最高的人说："我解决不了，我让下一个人解决比我小一号的问题。"依次传递，直到最矮的人可以直接解决。然后从最矮的开始，逐层向上汇报结果 |
| 13-25 分 | 讲授 | **阶乘递归**：`factorial(5) = 5 × 4 × 3 × 2 × 1`。递归定义：`factorial(n) = n × factorial(n-1)`，且 `factorial(1) = 1`。板书全过程：`factorial(4) = 4 × factorial(3) = 4 × 3 × factorial(2) = 4 × 3 × 2 × factorial(1) = 4 × 3 × 2 × 1 = 24`。强调：如果没有 `factorial(1)=1` 这个"停止条件"，函数会无限调用自己直到崩溃 |
| 25-35 分 | 代码演示 | 老师用 IDE 写 `int factorial(int n) { if (n <= 1) return 1; return n * factorial(n - 1); }`。编译运行，测试 `factorial(5)`。然后故意去掉 `if (n<=1)` 那行，运行——程序崩溃（栈溢出），让学生直观看到"没有停止条件的递归 = 灾难" |
| 35-45 分 | 练习 | 学生模仿写出递归求和函数 `int sum(int n)`，返回 1+2+...+n。提示：`sum(n) = n + sum(n-1)`，停止条件 `sum(1) = 1` |

### 第 2 课时：递归的两个条件 + 调用栈（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 回顾 | 默写 `factorial` 递归函数。提问："为什么需要 `if (n <= 1)`？"——引出递归的两个条件 |
| 8-22 分 | 讲授 | **递归两条件**：①**基线条件**（Base Case）——最简单的情况，不再递归，直接返回。如 `n <= 1`。②**递归条件**（Recursive Case）——把问题变小，然后递归调用。如 `n * factorial(n-1)`。口诀："**小事自己干，大事推给下一个**"。没有基线条件 = 死循环，没有递归条件 = 不算递归 |
| 22-35 分 | 讲授 | **调用栈**演示。老师在黑板上画栈结构：调用 `factorial(3)` → 压栈 `factorial(3)` → 压栈 `factorial(2)` → 压栈 `factorial(1)` → 返回 1 → 弹出 `factorial(1)` → 计算 2×1=2 返回 → 弹出 `factorial(2)` → 计算 3×2=6 返回 → 弹出 `factorial(3)` → 得到 6。强调：每层递归调用都会在调用栈上占一个位置，如果递归太深，栈会被撑爆（Stack Overflow） |
| 35-45 分 | 互动 | 老师出题 `factorial(5)`，让学生逐层写出调用过程（在纸上画调用栈）。收集 2-3 份投影点评 |

### 第 3 课时：经典递归问题（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | "递归不只是用来算阶乘的。今天我们来见识两个经典的递归问题——斐波那契和汉诺塔。" |
| 10-22 分 | 讲授 | **斐波那契数列**：1, 1, 2, 3, 5, 8, 13, ... 递归定义：`fib(1) = 1, fib(2) = 1, fib(n) = fib(n-1) + fib(n-2)`。代码：`int fib(int n) { if (n <= 2) return 1; return fib(n-1) + fib(n-2); }`。关键特点：每次调用会**分成两路**递归（二叉树结构），效率远低于迭代。以此引出"递归不一定是最优解" |
| 22-35 分 | 分组讨论 | **汉诺塔问题**（可选，视班级水平定）。64 个盘子从 A 柱移到 C 柱，每次只能移一个，大盘不能放小盘上面。递归解法思路：要把 n 个盘子从 A 移到 C → ①先把 n-1 个从 A 移到 B（借助 C）②把最大的从 A 移到 C ③再把 n-1 个从 B 移到 C（借助 A）。代码简要展示，不要求掌握 |
| 35-45 分 | 上机 | 学生上机写斐波那契递归函数，测试 `fib(10)`=55。再测试 `fib(40)`——观察程序明显变慢。老师解释：递归版斐波那契有大量重复计算（`fib(5)` 被计算了多次），这正是效率问题的根源 |

### 第 4 课时：递归 vs 迭代 + 递归陷阱（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 对比 | 板书对比：阶乘的递归版 vs 循环版。循环：`int r=1; for(int i=1;i<=n;i++) r*=i;`。递归：`return n*factorial(n-1)`。讨论：哪个更直观？哪个更快？——循环更高效（无函数调用开销），递归更贴近数学定义 |
| 10-25 分 | 陷阱精讲 | **陷阱 1**：忘记基线条件 → 无限递归 → 栈溢出。**陷阱 2**：基线条件永远达不到（如 `if (n == 0)` 但每次传 `n-2`，n 永远跳不过 0）→ 无限递归。**陷阱 3**：递归深度太大 → 栈溢出（如 `factorial(100000)`）。**陷阱 4**：重复计算——像斐波那契那样树状递归，复杂度爆炸 |
| 25-38 分 | 练习 | 学生完成：①递归求 a 的 b 次幂：`int power(int a, int b)`，`power(a, b) = a × power(a, b-1)`，`power(a, 0) = 1` ②递归求最大公约数（辗转相除法）：`int gcd(int a, int b)`，`gcd(a, b) = gcd(b, a%b)`，`gcd(a, 0) = a` |
| 38-45 分 | 讨论 | "什么时候该用递归？"——①问题本身具有递归结构（树的遍历、分治算法）②用递归表达比迭代清晰得多。什么时候不该用？——①能用简单循环解决的问题 ②深度不可控的问题 ③性能敏感的场合 |

## 🔥 重点难点

### 教学重点

1. **递归的两条件**：基线条件 + 递归条件。必须让学生养成"写递归先写基线条件"的习惯。
2. **调用栈的理解**：递归不是魔法——每次调用都压栈，返回时弹栈。这个模型是理解所有递归行为的钥匙。
3. **递归 vs 迭代的选择**：递归优雅但不总是高效。学生需要建立这个判断力。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|------|
| "想不通递归是怎么算出来的" | 学生试图在脑子里展开所有递归层，越想越乱 | 教"信任法"：你不需要追踪每一层。你只需要相信 `factorial(n-1)` 会正确返回 (n-1)!，然后你只需要处理"用这个结果再乘 n"。就像你不需要知道下属怎么做事的，你只需要知道他会交给你正确的结果 |
| 忘记写基线条件 | 代码只有 `return n * f(n-1);` 没有 if | 强迫习惯：先写 `if (n <= 1) return 1;`，再写 else 分支。形成"递归函数的第一个语句一定是 if（基线条件）"的肌肉记忆 |
| 递归调用写错 | `return factorial(n) * n;`——参数没变小 | 检查口诀："每次递归参数必须**变小或变简单**，朝着基线条件前进。"让同桌互相检查：参数是否在缩小？ |

## 💬 常见问答

**Q1："递归和非递归（循环）都能解决的问题，我该用哪个？"**

A：初学阶段优先用循环——循环不会栈溢出、效率更高、更容易调试。递归只在以下情况使用：①问题用递归表达明显更清晰（如树的遍历、分治算法）②循环写法极其复杂别扭。一个简化的判断标准：如果你能用 3 行循环解决问题，就别用递归。

**Q2："我的递归程序崩溃了，怎么知道是不是无限递归？"**

A：三个信号——①程序运行很久不出结果 ②程序直接崩溃退出无输出 ③报错信息里有 `stack overflow` 或 `segmentation fault`。排查方法：在递归函数第一行加 `cout << "进入 " << n << endl;`，看参数是否在缩小、是否趋向基线条件。如果参数不变（或变大），就是基线条件永远达不到。

**Q3："C++ 能支持多少层递归？"**

A：取决于栈的大小，没有固定数字。通常几千到几万层是安全的，几百万层必崩。可以通过 `ulimit -s`（Linux）或编译器选项调整栈大小。但实际编程中应该避免深层递归——如果数据规模可能很大，改用循环。

**Q4："斐波那契用递归那么慢，有没有既清晰又快的写法？"**

A：有——记忆化搜索（Memoization）。用数组保存已经算过的 `fib(n)`，每次递归前先查数组，算过了就直接取，没算过就递归计算并存入数组。这样把时间复杂度从 O(2^n) 降到 O(n)。这是动态规划的基础思想，以后会学到。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 递归函数必须包含两个条件：__________ 和 __________。
2. 递归函数每次调用自身时，会在 __________ 上增加一层。
3. 如果忘记写基线条件，会导致 __________。
4. 计算 `factorial(100000)` 会因为 __________ 而崩溃。

### 二、改错题（8 分）

5. 以下递归函数本意是计算 `1+2+...+n`，但运行会崩溃。找出问题并修改：
```cpp
int sum(int n) {
    return n + sum(n - 1);
}
```

### 三、写代码（每题 8 分，共 16 分）

6. 用递归写一个函数 `int countDigits(int n)`，返回正整数 n 的位数。（提示：`countDigits(1234) = 1 + countDigits(123)`，停止条件 n < 10 时返回 1）
7. 写一个递归函数 `void printReverse(int n)`，从 n 打印到 1。（提示：先打印 n，再递归调用 printReverse(n-1)，n=0 时停止）

### 四、代码分析（6 分）

8. `mystery(4)` 的返回值是多少？
```cpp
int mystery(int n) {
    if (n == 0) return 0;
    return n + mystery(n - 2);
}
```

---

### 答案

1. 基线条件，递归条件
2. 调用栈
3. 无限递归（或 "栈溢出"）
4. 栈溢出（递归深度过大）
5. 缺少基线条件。应改为：
```cpp
int sum(int n) {
    if (n <= 0) return 0;
    return n + sum(n - 1);
}
```
6. ```cpp
int countDigits(int n) {
    if (n < 10) return 1;
    return 1 + countDigits(n / 10);
}
```
7. ```cpp
void printReverse(int n) {
    if (n == 0) return;
    cout << n << " ";
    printReverse(n - 1);
}
```
8. `mystery(4) = 4 + mystery(2) = 4 + 2 + mystery(0) = 4 + 2 + 0 = 6`

## 🎮 不插电活动

### 活动 1：俄罗斯套娃拆解（第 1 课时用，约 8 分钟）

**道具**：
- 如果有俄罗斯套娃实物最好；没有则用 5 张大小不同的便利贴，从大到小标注 `n=5`、`n=4`、`n=3`、`n=2`、`n=1`
- 或者让 5 个身高不同的学生站成一排模拟

**流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-1 分 | 老师 | 邀请 5 个学生上台站成一排，由高到矮，编号 n=5 到 n=1。给每个学生发一张纸作为"答案板" |
| 1-3 分 | 老师 | 宣布问题："我们要计算 5 的阶乘！（5! = ?）" 告诉 n=5 的学生："你的任务是把你的编号（5）乘以 (n-1) 的阶乘。但 (n-1) 的阶乘你不知道，你问 n=4 的人。" |
| 3-6 分 | 学生 | **向下传递（压栈）**：n=5 → 问 n=4："4! 是多少？" n=4 → 问 n=3："3! 是多少？" ... n=2 → 问 n=1："1! 是多少？" n=1（最矮的）大声说："1! = 1，我知道！不用再问了！"——这就是**基线条件** |
| 6-8 分 | 学生 | **向上返回（弹栈）**：n=1 告诉 n=2："1! = 1"。n=2 在自己的纸上算 2×1=2，告诉 n=3。n=3 算 3×2=6，告诉 n=4。n=4 算 4×6=24，告诉 n=5。n=5 算 5×24=120，举起纸大声宣布："5! = 120！" |

**教学要点**：
- 强调 n=1 的角色——"他是英雄！没有他喊停，所有人会一直问下去直到世界末日。这就是基线条件的重要性。"
- 可以故意让中间某个学生"卡住"（不知道乘什么），然后让下一个学生帮他——体现递归的依赖关系
- 如果 n=1 的学生被问到后思考很久，老师催他："快回答！上面 4 个人都在等你！"——这就是调用栈阻塞

### 活动 2：递归vs循环——站队对决（第 4 课时用，约 10 分钟）

**分组**：全班分两队——"循环队"（左）和"递归队"（右）

**流程**：

| 分钟 | 操作 |
|------|------|
| 0-2 分 | 老师在黑板上写 5 个任务（求 1~n 和、求 n!、求斐波那契第 n 项、反转字符串输出、计算 a 的 b 次方），两队各自选择"用循环"还是"用递归"解决。每队派代表到黑板写伪代码 |
| 2-6 分 | 两队各自讨论和书写，老师巡场指导 |
| 6-10 分 | 两队展示代码，全班投票：哪个版本的代码更容易理解？老师总结："有些问题循环更简单，有些递归更自然。优秀的程序员两种都会，根据情况选择。" |

## 📦 补充题库

1. 以下递归函数有什么问题？
```cpp
int f(int n) {
    if (n == 1) return 1;
    return f(n) + f(n - 1);
}
```
（参考答案：`f(n)` 调用了 `f(n)`——参数没变小，会无限递归导致栈溢出。应改为 `f(n-1) + f(n-2)` 或其他缩小参数的调用。）

2. 写递归函数 `int sumOfDigits(int n)`，返回 n 的各位数字之和。如 `sumOfDigits(123) = 1+2+3 = 6`。（参考答案：`if (n < 10) return n; return n%10 + sumOfDigits(n/10);`）

3. 分析：递归函数 `int f(int n) { if (n <= 1) return 1; return f(n-1) + f(n-1); }` 计算 f(4) 的过程中，f(1) 被调用了多少次？（参考答案：8 次。这是指数级爆炸——f(2) 调用 2 次 f(1)，f(3) 调用 4 次，f(4) 调用 8 次。）

4. 用递归实现欧几里得算法（辗转相除法）求最大公约数。（参考答案：`int gcd(int a, int b) { if (b == 0) return a; return gcd(b, a % b); }`）

5. 小明说"既然递归能做的事循环都能做，那学递归有什么用？"请写出你的回答。（参考答案：递归的价值不在于"能不能做"，而在于"思维的清晰度"。很多问题——如树的遍历、分治算法、回溯——用递归表达极其简洁自然，用循环写会非常复杂。另外，递归是计算机科学的基础思想，理解了递归才能真正理解很多算法。）

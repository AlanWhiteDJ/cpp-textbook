## 🔥 重点难点

### 教学重点

1. **文件读写的基本流程**：打开→检查→读写→关闭。四步缺一不可，尤其是检查（`is_open()`）和关闭（`close()`）。
2. **`ofstream` 与 `ifstream` 的区别**：一个负责"写出去"，一个负责"读进来"。命名记忆：`o` = output（输出到文件），`i` = input（从文件输入）。
3. **打开模式的选择**：默认覆盖 vs `app` 追加——搞错就会丢数据。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|------|
| 忘记 `#include <fstream>` | 编译器报错 `ofstream was not declared` | 口诀："文件操作三件套——`fstream` 头文件、`ofstream` 写、`ifstream` 读"。头文件名字本身就是提示（file + stream） |
| 文件路径问题 | 代码写 `"data.txt"` 但程序找不到文件 | 解释相对路径和绝对路径的区别。`"data.txt"` 是在程序运行的"当前目录"下找文件。IDE 里运行时的当前目录通常是项目目录。建议演示时直接用全路径或先确认当前目录 |
| `>>` 读字符串遇到空格就停 | 文件中一行 `"张三 95"`，用 `fin >> name >> score` 读到 name 只有"张" | 用 `getline(fin, line)` 读一整行，再解析。或者如果字段固定，直接用 `fin >> name` 配合无空格的名字。两种方案对比讲清楚 |
| 打开模式混淆 | `ifstream fin("f.txt", ios::out);`——在 ifstream 上用写模式 | `ios::out` 只能用于 `ofstream` 或 `fstream`。`ifstream` 默认就是读模式，不需要也不能指定 `ios::out` |

## 💬 常见问答

**Q1："`fin >> x` 和 `cin >> x` 用法一模一样吗？"**

A：基本一样，因为 `cin` 和 `fin` 都是 `istream` 的子类。区别只是数据来源不同——`cin` 从键盘来，`fin` 从文件来。同样地，`cout` 和 `fout` 都是 `ostream` 的子类，用法一致，只是目的地不同。这就是 C++ 流（stream）设计的优雅之处。

**Q2："怎么判断文件读完了？"**

A：两种方法——①`while (fin >> var)`：每次尝试读取，如果成功就进入循环体。读到文件末尾或遇到无法解析的数据时，条件变为 false，循环结束。②`while (!fin.eof())`：检查是否到达文件末尾（EOF = End Of File）。推荐用第一种（它同时检查读取是否成功），第二种在最后一行有空行时容易多读一次。

**Q3："文件操作也占内存吗？什么时候必须 close？"**

A：文件操作使用了**缓冲区**——一小块内存用于暂存数据。只有缓冲区满了或显式调用 `close()` 时，数据才真正写入硬盘。如果不 `close()`：①程序正常退出时，`ofstream` 的析构函数会自动关闭文件（通常没问题）②但程序崩溃时，缓冲区里的数据可能丢失。所以**写完重要数据后立即 `close()`** 是最安全的做法，就像编辑完文档要保存一样。

**Q4："二进制文件和文本文件有什么区别？"**

A：文本文件里存的是"人能看懂"的字符（你用记事本打开能看到文字）。二进制文件里存的是内存中的原始字节（用记事本打开是乱码）。例如：存数字 123456 时，文本文件存字符 '1' '2' '3' '4' '5' '6'（6 字节），二进制文件存 int 的 4 字节原始数据（4 字节）。二进制文件更省空间但不可读。竞赛中 99% 用文本文件。

5. `ofstream fout("C:\test\data.txt");`——路径中 `\t` 被解释为制表符

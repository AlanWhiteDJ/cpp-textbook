## 🔥 重点难点

### 教学重点

1. **三步走的代码模板**：`sort → unique+erase → lower_bound`。这 4 行代码要让学生背下来——离散化的标准模板在竞赛中反复出现。
2. **"段统计"而非"点统计"**：离散化后统计覆盖区间时，统计的是相邻坐标之间的**段**，不是坐标点本身。这是最容易出错的概念转换。
3. **`lower_bound` 返回值**：返回迭代器，`- v.begin()` 得到从 0 开始的下标，`+1` 得到从 1 开始的编号。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 离散化后忘了去重 | 重复坐标导致数组多开 | 演示：`{5,5,5,10}` 不去重 → 离散化后变成 4 个值 `{5,5,5,10}`（第 1/2/3 个都是 5）——浪费空间且逻辑混乱。去重后 `{5,10}` 变成 2 个值 |
| `lower_bound` 没用对 | 传入参数错误或忘了 `-v.begin()` | 标准化写法板书三遍：`int id = lower_bound(v.begin(), v.end(), x) - v.begin() + 1;`——让学生记住这个固定句式 |
| 段统计时误把离散化下标当实际长度 | `ans += 1`（加的是 1 而不是实际长度） | 板书画图：坐标 1→编号 1，坐标 100→编号 2。如果有一段被覆盖了，它的实际长度是 99 而不是 1。必须用原始坐标之差：`v[i]-v[i-1]` |
| 差分数组下标越界 | 离散化后最大下标是 m，`diff[r+1]` 可能 = m+1 | 开 `diff[m+5]` 或在 `r+1 <= m` 时才操作。在火烧赤壁中用的是"段"差分，需要开 `m+2` 的大小 |
| 不知道什么时候需要离散化 | 所有题都离散化 | 判断标准：**数组下标范围是否远超数据数量？** 如果 n=10⁵ 但坐标范围是 10⁹——需要离散化。如果 n=10⁵ 坐标范围也是 10⁵——直接用原坐标 |

## 💬 常见问答

**Q1："离散化和哈希表有什么区别？都需要把大值映射到小值。"**

A：核心区别——离散化**保留顺序**（5→1, 10→2, 100→3，顺序不变），哈希表**不保留顺序**（5 可能映射到 7, 10 映射到 2）。所以需要前缀和/差分/二分查找的场景必须用离散化；只需要快速查找的场景可以用哈希表。

**Q2："`unique` 函数到底做了什么？"**

A：`unique(v.begin(), v.end())` 把有序数组中重复的元素"挤"到末尾，返回一个迭代器指向"不重复部分的末尾"。但它**不改变容器大小**——所以需要配合 `v.erase(迭代器, v.end())` 真正删掉重复部分。完整版：`v.erase(unique(v.begin(), v.end()), v.end());`

**Q3："火烧赤壁那题，为什么用差分而不是直接统计覆盖次数？"**

A：因为一个区间覆盖了连续多个段。“对 [l,r] 每个段覆盖次数 +1” = 区间修改——用差分 O(1) 做。最后前缀和还原出每个段的覆盖次数。这就是离散化 + 差分的经典组合：离散化把大坐标压成小下标，差分高效处理区间操作。

**Q4："离散化后能保留原始坐标吗？"**

A：能——原始坐标存在 `v` 数组里。`v[id-1]`（或 `v[id]`，取决于是否 +1）就是编号 `id` 对应的原始坐标。所以离散化并没有丢掉信息，它只是建立了一个双向映射。

**Q5："为什么要 `+1` 让编号从 1 开始？"**

A：因为后面的前缀和/差分通常用 1-based 下标（`pre[0]=0` 的需要）。如果不用前缀和，0-based 也没问题——完全取决于后续算法的习惯。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 离散化的三个步骤是：①__________ ②__________ ③__________。
2. `lower_bound` 的功能是在有序数组中查找第一个 __________ 给定值的元素的位置。
3. 在火烧赤壁问题中，离散化后统计的不是点而是 __________。

### 二、读代码题（5 分）

4. 以下离散化代码的输出是什么？
```cpp
vector<int> v = {100, 5, 100, 20, 5};
sort(v.begin(), v.end());
v.erase(unique(v.begin(), v.end()), v.end());
for (int x : {5, 20, 100})
    cout << lower_bound(v.begin(), v.end(), x) - v.begin() + 1 << " ";
```

### 三、写代码（15 分）

5. （8 分）输入 n 个坐标，输出每个坐标离散化后的编号（从 1 开始）。重复坐标应得到相同编号。

6. （7 分）简化版火烧赤壁：n 条线段 [l, r]（坐标范围 -10⁹~10⁹），求被至少一条线段覆盖的总长度。n ≤ 10⁵。

---

### 答案

1. 排序（sort），去重（unique + erase），二分查找（lower_bound）
2. 大于或等于（≥）
3. 相邻坐标之间的段
4. `1 2 3`（5→1, 20→2, 100→3）
5. ```cpp
vector<int> v, orig;
int n; cin >> n;
for (int i = 0; i < n; i++) { int x; cin >> x; v.push_back(x); orig.push_back(x); }
sort(v.begin(), v.end());
v.erase(unique(v.begin(), v.end()), v.end());
for (int x : orig)
    cout << lower_bound(v.begin(), v.end(), x) - v.begin() + 1 << " ";
```
6. 核心思路：收集所有端点→离散化→差分统计每段的覆盖次数→累加被覆盖段的实际长度。
```cpp
vector<int> xs;
for (int i = 0; i < n; i++) {
    xs.push_back(l[i]); xs.push_back(r[i]);
}
sort(xs.begin(), xs.end());
xs.erase(unique(xs.begin(), xs.end()), xs.end());
int m = xs.size();
vector<int> diff(m + 2, 0);
for (int i = 0; i < n; i++) {
    int L = lower_bound(xs.begin(), xs.end(), l[i]) - xs.begin() + 1;
    int R = lower_bound(xs.begin(), xs.end(), r[i]) - xs.begin() + 1;
    diff[L]++; diff[R]--;  // 注意：右端点不包含被覆盖的最后一段
}
int cur = 0, ans = 0;
for (int i = 1; i < m; i++) {
    cur += diff[i];
    if (cur > 0) ans += xs[i] - xs[i-1];
}
cout << ans;
```

## 📦 补充题库

1. 离散化的时间复杂度是多少？  
（参考答案：排序 O(n log n) + 去重 O(n) + 每次查询 O(log n)。总体 O(n log n)。）

2. 如果坐标值已经在 1~n 范围内且 n≤10⁵，还需要离散化吗？  
（参考答案：不需要——直接用原坐标当下标。离散化是"不得不压缩"时的工具，不是越多越好的习惯。）

3. 二维离散化：n 个点 (x, y)，坐标范围 10⁹。如何离散化？  
（参考答案：x 坐标和 y 坐标**分别**离散化——对 x 做一次三步走，对 y 做一次三步走。两个维度独立压缩。）

4. 换一种离散化方法：不用 `lower_bound`，用 `unordered_map` 存储映射关系。比较两种方法的优劣。  
（参考答案：map 方法 O(n log n) 建立映射 + O(1) 查询；二分方法 O(n log n) 建立 + O(log n) 查询。map 后续查询更快但常数大，二分方法常数小但每次要二分。实际竞赛中用二分法更多——因为一般需要二叉树结构的场景会直接离散化。）

5. 离散化 + 差分组合题：n 个矩形，每个矩形给定左下 (x1,y1) 和右上 (x2,y2)，坐标范围 10⁹。求被至少一个矩形覆盖的总面积。  
（参考答案：二维离散化 + 二维差分。x 坐标和 y 坐标分别离散化 → 开 m×m 的网格（m≤2n）→ 二维差分处理每个矩形覆盖 → 还原后统计被覆盖的"小格子"面积。每个小格子的实际面积 = (x[i]-x[i-1]) × (y[j]-y[j-1])。）

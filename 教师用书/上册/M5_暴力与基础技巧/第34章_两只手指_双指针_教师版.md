## 🗺️ 教学流程

### 第 1 课时：对撞指针——两数之和（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | 老师讲珠子故事：一条绳上串着从小到大排列的珠子，找两颗珠子之和等于 100。笨办法——左手拿一颗，右手摸遍所有（O(n²)）。聪明办法——左手最左，右手最右，和太小左手右移，和太大右手左移（O(n)）。引出核心问题：**为什么可以放心地跳过一个指针？** |
| 10-25 分 | 讲授 | 板书对撞指针模板：`int l=0, r=n-1; while(l<r) { if(sum==target) 找到; else if(sum<target) l++; else r--; }`。关键讲清楚"为什么可以放心跳过"：①如果 `a[l]+a[r] < target`，a[l] 和 r 左边任何数之和只会更小 → a[l] 没希望了 → `l++` ②同理 a[r] 没希望了 → `r--`。**前提：数组有序**——无序先 sort |
| 25-35 分 | 演示 | IDE 演示：①用两重循环 O(n²) 跑 n=100000 时明显卡顿 ②用对撞指针 O(n) 瞬间出结果。让学生直观感受"n² 到 n"的差距——不是"快一点"，而是"能跑和不能跑"的区别 |
| 35-42 分 | 练习 | 学生上机：输入一个升序数组和一个 target，用对撞指针判断是否存在两数之和等于 target。扩展：如果要求输出**所有**这样的数对呢？（找到一对后同时移动 `l++; r--;` 继续找） |
| 42-45 分 | 小结 | "对撞指针 = 双指针的第一形态。核心只有一个——**利用有序性排除不可能**。记住：指针移动不是随机的，每一步都是有根据的排除。" |

### 第 2 课时：快慢指针 + 滑动窗口入门（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | "上节课两个指针一左一右对撞。但如果两个指针在同一端，有办法吗？"——引出快慢指针和滑动窗口 |
| 8-22 分 | 🎮 不插电活动 | **"双人跑道"**（详见活动区）。操场场景：快的人跑两步，慢的人跑一步。当快的跑到终点时，慢的刚好在中间。让学生用身体体验"快慢指针"的几何直觉 |
| 22-32 分 | 讲授 | **快慢指针在数组上的应用**：找中点——`slow++` 一次，`fast+=2` 一次。当 fast 到末尾时 slow 在中位。板书代码和轨迹图（每一步标注 slow 和 fast 位置）。**滑动窗口**：维护一个可变区间 [l, r]，窗口根据条件伸缩。类比：蚯蚓蠕动——头往前探(r++)，尾巴跟上(l++)，身体长度就是窗口大小 |
| 32-42 分 | 练习 | 学生上机：①用快慢指针找数组的中点元素 ②滑动窗口入门——给定数组和 S，找最短的连续子数组使其和 ≥ S。提示：r 向右扩展累加，当 sum≥S 时记录长度并 l++ 缩窗，直到 sum<S 再继续扩展 r |
| 42-45 分 | 总结 | "滑动窗口的精髓：**右指针负责'吃进来'，左指针负责'吐出去'**。窗口里永远维护着你关心的那段数据。时间复杂度：每个元素最多被 l 和 r 各访问一次→O(n)。" |

### 第 3 课时：滑动窗口进阶——A-B 数对 + 综合练习（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-12 分 | 讲授 | **A-B 数对问题**（NOIP 2007 普及组变体）：给定 n 个正整数和 C，求 A-B=C 的数对个数。暴力：枚举 B 然后二分找 A → O(n log n)。双指针：排序后，用两个指针维护"第一个等于 B+C"和"最后一个等于 B+C"的位置，对于每个 B，`(r-l)` 就是 A 的个数。强调：两个指针都需要单调移动（只增不减），否则会退化为 O(n²) |
| 12-25 分 | 独立练习 | 学生完成 A-B 数对。提示分步：①排序 ②初始化 l=0, r=0 ③枚举 B = a[i]，l 跳到第一个 ≥ a[i]+C 的位置，r 跳到第一个 > a[i]+C 的位置 ④`ans += r-l`。老师巡场，检查 l 和 r 的 while 循环条件是否正确 |
| 25-38 分 | 综合练习 | ①逛画展（简化版）：找最短区间包含所有画家（每种编号至少出现一次）——典型滑动窗口 ②两数之和（无序版）：sort + 对撞指针。让学生体会"不同形态的双指针解决不同类型的问题" |
| 38-45 分 | 讲评 | 投影典型错误：①A-B 数对中 l 和 r 在每轮 B 迭代中未继承上一轮位置（重新从 i 开始）②逛画展中窗口收缩条件写错。总结三种形态的使用口诀 |

## 🔥 重点难点

### 教学重点

1. **对撞指针的"排除法"逻辑**：学生必须理解为什么可以放心跳过——不是因为"算法说可以"，而是因为**数学上不可能**（和太小→左边这个数配谁都太小）。
2. **滑动窗口的伸缩机制**：右扩累加→满足条件→左缩去掉多余→不满足了继续右扩。这个"呼吸节奏"是滑动窗口的灵魂。
3. **双指针都是 O(n)**：每个元素最多被两个指针各访问一次，总移动步数 ≤ 2n。让学生从"移动次数"的角度理解复杂度。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 忘了排序就用对撞指针 | 无序数组直接两数之和双指针，结果不对 | 给出一个反例数组 `{5,1,3,7,2}` target=8。双指针会漏掉 1+7。总结：**对撞指针 = sort + 双指针** |
| 滑动窗口的 while vs if | 用 `if(sum>=S)` 缩窗一次就停了 | 例子：`{1,1,1,100}` S=50，右扩到 100 时 sum=103。用 if 只缩一次（去掉 1），sum=102 仍然≥50，但窗口不是最短。必须用 while 一直缩到不满足 |
| A-B 数对中 l 和 r 每轮重置 | 每轮 `l=i; r=i;` 导致 l 和 r 反复从头跳 | 板书演示：如果每轮重置，l 已经走过的路还要重走一遍——O(n²)。**双指针的优势就在于"走过的路不回头"**——l 和 r 只增不减 |
| 快慢指针条件写错 | `while(fast < n)` 越界访问 `a[fast+1]` | 板书画出数组尾部：当 fast=n-1 时 `fast+1` 越界。改为 `while(fast+1 < n && fast+2 < n)` 或 `while(fast < n-1)` |
| 窗口收缩后 sum 忘了更新 | `l++` 了但 sum 没减 `a[l]` | 强调：**指针移动和状态更新必须绑定**。`l++` 前必须先 `sum -= a[l]`。可以写成 `sum -= a[l++];` 把两步合一 |

## 💬 常见问答

**Q1："双指针和暴力枚举有什么本质区别？"**

A：暴力枚举是"我检查所有组合，保留对的"——每个数对都被检查。双指针是"我根据当前信息排除一批，只检查有希望的"——a[l] 和 a[r] 的和太小，说明 a[l] 跟所有人都太小，a[l] 被永久跳过。**暴力是"全都看"；双指针是"边看边排除"**。

**Q2："对撞指针必须要求数组有序吗？"**

A：对撞指针本身要求的不是"有序"，而是"单调性"——即 `a[l]` 增加时，要配的 `a[r]` 如何变化是确定的。有序数组恰好满足这个性质。如果数组是"先增后减"的，对撞指针就失效了。实际上对撞指针 = 利用单调性的排除法。

**Q3："滑动窗口和之前学的队列（M8）有关系吗？"**

A：有！滑动窗口可以用队列实现——右扩=入队，左缩=出队。本质上滑动窗口维护的就是一个"滑动着的双端队列"。后面学单调队列时，你会看到滑动窗口的升级版——窗口内不仅维护元素，还维护"最值"。

**Q4："A-B 数对为什么需要两个指针 l 和 r？"**

A：因为同一个 B 可能对应多个 A——比如 `{5, 5, 5, 7, 7, 7}`，C=2，B=5 时 A=7 出现了 3 次。l 指向第一个 7，r 指向第一个 >7 的位置（即 7 的下一个）。`r-l` = 3 就是 A 的个数。如果只用单指针，你需要额外计数——双指针把"计数"变成了"位置差"。

**Q5："三种双指针形态能不能总结一张速查表？"**

A：

| 形态 | 起始 | 移动方向 | 条件 | 典型问题 |
|------|------|----------|------|----------|
| 对撞 | 两端 | 相向 | 有序 | 两数之和 |
| 快慢 | 同端 | 同向（倍速） | — | 找中点、判环 |
| 滑动窗口 | 同端 | 同向（伸缩） | 单调性 | 最短子数组、A-B 数对 |

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 8 分）

1. 双指针的三种形态是 __________、__________、__________。
2. 对撞指针要求数组必须 __________。

### 二、读代码题（每题 5 分，共 10 分）

3. 以下对撞指针代码有什么 bug？
```cpp
int l = 0, r = n - 1;
while (l <= r) {
    int sum = a[l] + a[r];
    if (sum == target) { cout << l << " " << r; break; }
    else if (sum < target) l++;
    else r--;
}
```

4. 以下滑动窗口代码（求最短子数组和≥S）有什么问题？
```cpp
int l = 0, sum = 0, ans = n + 1;
for (int r = 0; r < n; r++) {
    sum += a[r];
    if (sum >= S) {
        ans = min(ans, r - l + 1);
        sum -= a[l++];
    }
}
```

### 三、写代码（12 分）

5. 给定一个升序数组，判断是否存在两个不同位置的元素之和等于 target。如果存在输出两个元素值，否则输出 "No"。要求 O(n) 时间。

---

### 答案

1. 对撞指针，快慢指针，滑动窗口
2. 有序（或具有单调性）
3. `l <= r` 允许 l==r 时进入循环——只有一个元素，是同一个数被用了两次。题目要求两个**不同位置**的数，所以条件是 `while (l < r)`。
4. `if` 应改为 `while`。当窗口内有多个元素时，缩一次可能不够——如 `{1,1,1,100}` S=50，sum=103≥50，缩一次后 sum=102 仍然≥50，需要继续缩到最短。
5. ```cpp
int l = 0, r = n - 1;
while (l < r) {
    int sum = a[l] + a[r];
    if (sum == target) { cout << a[l] << " " << a[r]; return 0; }
    else if (sum < target) l++;
    else r--;
}
cout << "No";
```

## 🎮 不插电活动

### 活动 1：双人跑道——快慢指针（第 2 课时用，约 12 分钟）

**道具**：
- 在地上用胶带标出一条 10 米长的"跑道"，每米一个标记
- 两个学生，分别贴"slow""fast"标签

**规则与流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 讲解规则：slow 每次走 1 格，fast 每次走 2 格。两人同时从起点出发。"预备——走！" |
| 2-4 分 | slow/fast | 第一轮：跑道 10 格。slow 走 1→fast 走 2，slow 走 2→fast 走 4……当 fast 到达终点（第 10 格）时，slow 刚好在第 5 格。全班喊出"slow 在中间！" |
| 4-6 分 | 老师 | 问："如果跑道是 9 格呢？"——fast 到达第 9 格时 slow 在第 4 或 5 格。"如果 fast 能走但跑道前面还有一个更远的位置呢？"——引出代码里 `while(fast+1 < n && fast+2 < n)` 的边界判断 |
| 6-10 分 | 全体 | 变化玩法：**判环模拟**。学生手拉手围成一圈（环形链表的概念前奏）。fast 和 slow 从同一点出发，fast 跑得比 slow 快一倍——他最终会从后面追上 slow（相遇=有环！）。全班观察"套圈"现象 |
| 10-12 分 | 老师 | 总结："快慢指针的直觉就是——快的跑两步，慢的跑一步。快的到终点时慢的在中间。快的追上慢的说明有环。记住了这个画面，代码写起来就容易了。" |

**教学要点**：
- 可以在跑道上用数字标记数组下标，让学生直观看到"slow=2, fast=4"就是 `a[2]` 和 `a[4]`
- 判环模拟是链表章节的前奏，这里种下直觉种子即可

### 活动 2：伸缩毛毛虫——滑动窗口（第 4 课时用，约 8 分钟）

**道具**：
- 5 个学生站成一排，每人举一个数字牌
- 两个学生分别当"左手"和"右手"

**规则**：
- 左手=窗口左边界 l，右手=窗口右边界 r
- 目标：找最短的连续段使数字之和 ≥ 10
- 右手一个一个往右走（扩窗），每走一步全班加出当前和
- 当和 ≥ 10 时停住，左手开始往右走（缩窗），每走一步全班重算和。直到和 < 10 停下
- 记录"最短的那段"的长度

**教学要点**：让学生身体感受"右扩左缩"的节奏——左右手的移动不是同时的，而是交替的。

## 📦 补充题库

1. 以下对撞指针的做法，什么时候不适用？  
A. 有序数组找两数之和 B. 无序数组找两数之和 C. 有序数组找差值 D. 有序数组找最接近的两数  
（答案：B。无序时对撞指针的"排除逻辑"失效——无法保证跳过的元素没希望。）

2. 用双指针求"三数之和"：给定有序数组，找 a[i]+a[j]+a[k]=0。提示：固定 i，问题变成"在 i+1 到 n-1 中找两数之和 = -a[i]"——对撞指针。复杂度 O(n²)。

3. 快慢指针找中点的代码，如果数组长度是偶数，返回的是偏左还是偏右的中点？  
（参考答案：取决于 while 条件。`while(fast+1 < n && fast+2 < n)` 返回偏左的中点。）

4. 滑动窗口："不含重复字符的最长子串"——这是经典变体。窗口内需要维护"字符是否出现过"（用 `bool[256]` 或 `set`），当右扩遇到重复字符时左缩到重复字符的下一个位置。

5. 对撞指针 + 贪心："盛水最多的容器"——给定 n 个高度，找两个竖线使它们和 x 轴围成的容器装水最多。对撞指针：`l=0, r=n-1`，每次移动较矮的那根。（这是双指针的一个经典变体——不要求和，而是求面积。）

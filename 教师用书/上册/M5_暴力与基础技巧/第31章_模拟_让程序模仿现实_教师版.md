### 第 1 课时：模拟思想 + 读题四步法 + 石头剪刀布（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 老师讲阿杰的故事：第一场竞赛遇到一道模拟题——初始生命 100，每回合扣血回血，问第几回合降到 0。阿杰绞尽脑汁列方程，旁边的队友用 `for` 循环 3 分钟写完。引出发问：**为什么有人觉得难、有人觉得简单？差别只是——会不会"翻译"** |
| 8-22 分 | 讲授 | **模拟的定义**：把题目描述的操作规则，按顺序翻译成程序语句。类比：菜谱和厨师——题目=菜谱，你=翻译者（中文→C++），计算机=厨师（按步骤执行）。只要菜谱没歧义，菜就做得出来。**模拟题考的不是智商，是细心和耐心** |
| 22-32 分 | 讲授 | **读模拟题四步法**：①圈关键词（在草稿纸上圈出"谁和谁比""怎么判输赢""多少轮"）②定义变量（列出所有需要的变量和类型）③写步骤（用伪代码或注释搭骨架）④验证样例（用题目样例手工走一遍）。强调：**先纸上谈兵，再动手敲代码**——这个习惯能省掉一半调试时间 |
| 32-42 分 | 演示 | 老师现场用四步法分析"石头剪刀布"：①圈关键词→"两人出拳""赢的+1 分""比 5 轮"②定义变量→`int scoreA, scoreB; char a, b;`③写步骤→`// 循环5轮 {输入→判断→加分}`④验证→手工走一轮样例。然后 IDE 敲完整代码跑给学生看 |
| 42-45 分 | 小结 | "模拟题的'翻译感'——题目的每一个分句，对应代码里的一个 `if` 或一句 `cout`。翻译对了，分就到手了。"布置思考题："如果先赢 3 局就算胜，`for` 要改成什么？" |

| 时间 | 环节 | 操作 |

| 20-30 分 | 讲授 | **掷骰子**：`rand()%6+1` 生成 1~6，10 轮比较大小。强调 `srand(time(0))` 的作用——没有它每次运行结果一样。**时间模拟**：进位逻辑——`分钟加到 60 → 进位给小时 → 分钟%60 → 小时%24`。套路：`大单位 += 小单位/进制; 小单位 %= 进制;`。板书时间进位"公式模板" |

## 🔥 重点难点

### 教学重点

1. **读题四步法**：这是模拟题的核心元技能。学生最大的敌人不是代码能力，而是"题读不懂"或"读了就忘"。四步法强制他们在敲代码前完成理解。
2. **时间进位模板**：`大单位 += 小单位/进制; 小单位 %= 进制;`——这是模拟题中最高频的代码模式，必须形成肌肉记忆。
3. **状态数组的更新模式**：`light[pos] = !light[pos]` 这种"取反"操作和 `pos += person` 这种"步进"遍历，是后续图论遍历、筛法的基础。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 题目规则的顺序翻译错 | 题目说"先扣血再回血"，代码写成先回血再扣血 | 让学生用荧光笔在题目上标序号①②③，然后代码注释也标对应序号。**翻译的第一个原则是顺序**——顺序一变，结果可能完全不同 |
| 状态变量忘了初始化 | `int score;` 没赋初值就 `score++` | 统一规则：**所有计数器、累加器、状态变量必须 `= 0` 或 `= false` 初始化**。板书"初始化三兄弟"——计数器、累加器、布尔状态 |
| 循环是 `<` 还是 `<=` 搞不清 | 开关灯那道题，题目说"第 k 个人处理第 k 的倍数"，写成 `pos < n` | 教学生一个技巧：用最小数据（n=3, k=2）手算一遍，看第 3 盏灯应该被处理吗？如果应该，那就是 `<=` |
| 用 `cin` 读带格式的时间吞分隔符失败 | `cin >> hour >> minute` 读 "08:30" 失败 | 板书："`cin` 只认空格和回车，不认冒号"。解决方法：`char c; cin >> hour >> c >> minute;`——用一个"吃符号"的 `char` 变量吞掉冒号 |
| 面对长题干就发懵 | "这道题题目好长，我不会" | 先让他在题干里用红笔画关键词，画完再问"现在你觉得这道题在说什么？"——通常画完关键词他就懂了。**长题干≠难题，很多时候长题干是为了"没有歧义"** |

## 💬 常见问答

**Q1："模拟和枚举到底怎么区分？感觉都是用循环啊。"**

A：区别在于**思路起点**：
- 枚举 = "我要列出所有可能的答案，然后筛出符合条件的那几个"（比如所有三位水仙花数）
- 模拟 = "我要按照题目规定的流程，一步步往下走，走到结束"（比如 10 回合的生命值变化）
枚举的循环在"生成可能性空间"里转，模拟的循环在"时间线"上走。不过确实，有些题两种方法都能做——用哪种顺手就用哪种。

**Q2："开关灯那道题，为什么内层循环是 `pos += person` 而不是 `pos++`？"**

A：因为题目说"第 k 个人操作第 k 的倍数位置"——第 1 个人操作 1,2,3,4...（全部），第 2 个人操作 2,4,6,8...（步长 2），第 3 个人操作 3,6,9...（步长 3）。步长恰好就是"第几个人"。`pos += person` 就是"每次跳 person 步"。如果用 `pos++`，就变成每个人都操作所有灯了——不符合题意。

**Q3："`rand()` 生成的随机数为啥每次运行都一样？"**

A：因为 `rand()` 是伪随机——它内部有一个"种子"，根据一个固定公式生成下一个数。种子不变，序列就不变。`srand(time(0))` 把当前时间（每秒都在变）设为种子，这样每次运行种子不同，序列就不同了。竞赛中一般不依赖随机数（结果不确定），但做模拟题时偶尔会用到。

**Q4："模拟题的调试有什么特别的技巧？"**

A：最关键的一招——**在每一步后打印所有状态变量**。比如开关灯那道题，在每轮循环后加 `cout << "第 " << person << " 个人操作后："; for (int i=1;i<=n;i++) cout << light[i]; cout << endl;`。这样你就能看到每一步发生了什么变化。调试完把打印语句注释掉（别删，下次出错还能用）。

**Q5："存钱罐那题，`total += day` 和 `total += 1` 有什么区别？"**

A：`total += day` 是"存当天日期对应的金额"——1 号存 1 元，15 号存 15 元。`total += 1` 是"每天存 1 元"。一字之差，逻辑完全不同。这恰恰说明模拟题的关键：**题目怎么说，代码怎么翻**。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 读模拟题的四个步骤是：①__________ ②__________ ③__________ ④__________。
2. 时间计算中，将分钟进位到小时的公式模板是：`hour += __________; minute __________;`

### 二、改错题（每题 5 分，共 10 分）

3. 以下代码想模拟"初始生命 100，第 i 回合扣 damage[i] 再回 recover[i]"，哪里有错？
```cpp
int hp = 100;
for (int i = 1; i <= 10; i++) {
    hp += recover[i];
    hp -= damage[i];
    if (hp <= 0) { cout << i; break; }
}
```

4. 以下代码想读入时间 "14:30"，哪里有错？
```cpp
int h, m;
cin >> h >> m;
cout << h << "时" << m << "分";
```

### 三、写代码（每题 10 分，共 20 分）

5. 写程序模拟：初始分数 0。掷一个骰子（1~6），如果掷出 6 则加 10 分并再掷一次；掷出 1 则扣 5 分（最低 0 分）；其他情况加对应分数。共掷 10 次（额外掷的不算在 10 次里）。输出最终分数。

6. 写程序模拟：走廊有 20 盏灯（初始全关）。第 1 个人把所有灯打开；第 2 个人把每第 2 盏灯取反；第 3 个人把每第 3 盏灯取反……一直到第 5 个人。输出最后哪些灯亮着。

---

### 答案

1. 圈关键词，定义变量，写步骤，验证样例
2. minute / 60，%= 60
3. 顺序错了！题目说"先扣血再回血"，代码是先回血再扣血。应改为先 `hp -= damage[i]` 再 `hp += recover[i]`。
4. `cin >> h >> m` 不知道怎么处理冒号 `:`。应改为 `char c; cin >> h >> c >> m;`
5. ```cpp
int score = 0, throws = 0;
srand(time(0));
while (throws < 10) {
    int dice = rand() % 6 + 1;
    throws++;
    if (dice == 6) { score += 10; throws--; }  // 额外掷，不计次数
    else if (dice == 1) { score = max(0, score - 5); }
    else { score += dice; }
}
cout << score;
```
（注意：throws-- 让额外掷的不算在 10 次里，但会导致无限循环风险——如果一直掷 6 程序跑不停。更好的写法是嵌套 while 处理连续掷 6，但此处不做严格要求）
6. ```cpp
bool light[21] = {false};
for (int p = 1; p <= 5; p++)
    for (int i = p; i <= 20; i += p)
        light[i] = !light[i];
for (int i = 1; i <= 20; i++)
    if (light[i]) cout << i << " ";
```

## 🎮 不插电活动

### 活动 1：人体计算机——开关灯模拟（第 2 课时用，约 15 分钟）

**道具**：
- 20 张白纸和 20 张黄纸（两面不同色，代表"开/关"）
- 5 个胸牌，分别写"第1人""第2人"……"第5人"

**场地布置**：
- 20 个学生站成一排，每人举一张白纸（代表初始"关"状态）
- 5 个"操作者"站在排前，面对全班

**规则与流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 讲解规则："你们 20 个人是走廊里的 20 盏灯。白纸=关，黄纸=开。现在有 5 个人轮流来'翻转'你们——被翻到的同学要把纸翻面（白变黄、黄变白）。" |
| 2-4 分 | 第1人 | "第1人"从队伍最左边走到最右边，**每个人都翻**——全部从白变黄（全部打开） |
| 4-6 分 | 第2人 | "第2人"从队伍第 2 个开始，隔一个翻一个——位置 2、4、6、8……翻面 |
| 6-8 分 | 第3人 | "第3人"从队伍第 3 个开始，隔两个翻一个——位置 3、6、9……翻面 |
| 8-10 分 | 第4人 | "第4人"翻位置 4、8、12、16、20 |
| 10-12 分 | 第5人 | "第5人"翻位置 5、10、15、20 |
| 12-15 分 | 老师 | 全班观察最后状态："哪些灯亮着（黄纸朝外）？"——位置 1、4、9、16 亮着（恰好是完全平方数！）。引导学生发现规律：**被翻转奇数次的灯最终亮着 → 只有完全平方数有奇数个因子 → 被翻了奇数次**。这是一个美丽的数学结论从模拟中自然浮现 |

**教学要点**：
- 让学生用身体感受"第 k 个人翻第 k 的倍数"是怎么回事
- 最后揭示"完全平方数亮着"的结论时不要剧透——让学生先观察，再自己发现
- 如果时间不够可以只用 3 个人和 9 盏灯（1,4,9 也会亮）

### 活动 2：模拟翻译官（第 4 课时用，约 10 分钟）

**道具**：无（纯口头）

**规则**：
- 老师说一句中文规则，学生举手翻译成 C++ 伪代码
- 规则 1："如果分数大于 100，就输出'赢了'"→ `if (score > 100) cout << "赢了";`
- 规则 2："每轮开始前，体力减 5，然后检查体力是否小于等于 0"→ `hp -= 5; if (hp <= 0) ...`
- 规则 3："A 出石头、B 出剪刀，或者 A 出剪刀、B 出布，或者 A 出布、B 出石头，A 赢"→ `if ((a=='S'&&b=='J') || (a=='J'&&b=='B') || (a=='B'&&b=='S'))`
- 规则 4："从第 3 分钟开始，每 5 分钟触发一次事件"→ `for (int t=3; t<=T; t+=5)`
- 快问快答训练"中文→C++"的神经反射

## 📦 补充题库

1. 模拟题中，以下哪种情况最容易出错？  
A. 有多层循环 B. 规则顺序与代码顺序不一致 C. 变量太多 D. 数据太大  
（答案：B。规则顺序翻译错误是最隐蔽的 bug——程序能跑但结果不对。）

2. 以下哪项是模拟题的正确调试方式？  
A. 改一行代码就跑一遍，直到对了为止 B. 在每一步后打印所有变量观察状态变化 C. 直接把代码发给同学帮忙看 D. 换一种算法重新写  
（答案：B。打印中间状态是调试模拟题最直接有效的方法。）

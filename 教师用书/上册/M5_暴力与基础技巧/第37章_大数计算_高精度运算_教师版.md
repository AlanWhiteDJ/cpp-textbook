### 第 1 课时：高精度存储 + 加法（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | 板书：`int` 最大约 ±2×10⁹，`long long` 最大约 ±9×10¹⁸。"如果要算 100!（100 的阶乘），或者两个 50 位整数相乘呢？"——引出核心问题：**数据类型的上限不是算法的上限。用数组模拟竖式计算，可以处理任意位数。** 类比：小学生列竖式——CPU 不认识 100 位的整数，但认识 100 个 0~9 的数字 |
| 10-25 分 | 讲授 | **存储方式**：字符串输入（给人看的）→ 整型数组倒序存储（给 CPU 算的）。`a[0]` 存个位，`a[1]` 存十位……这样进位时往高处加更方便。板书存储代码：`string s; cin >> s; int len = s.size(); for (int i=0; i<len; i++) a[i] = s[len-1-i] - '0';` |
| 25-40 分 | 讲授 + 演示 | **高精度加法**：逐位相加 + 进位。板书完整代码框架：`for (int i=0; i<max(lenA,lenB); i++) { c[i] += a[i] + b[i]; c[i+1] += c[i] / 10; c[i] %= 10; }`。关键细节：①进位可能产生新的最高位——`len = max(lenA, lenB); if (c[len]) len++;` ②输出时要**倒序输出**（因为 c[0] 存的是个位）。IDE 演示：计算 123456789 + 987654321 |
| 40-45 分 | 小结 | 口诀："**字符入，数组存，个位在 0 好进位。逐位加，模 10 留，/10 是进位。倒着输出别忘记！**"布置思考：如果两个数一个很大一个很小，怎么判断大小？ |

| 时间 | 环节 | 操作 |
|------|------|------|

| 22-32 分 | 讲授 | **高精度减法**：`a - b`，保证 a ≥ b。逐位相减，不够借位：`if (a[i] < b[i]) { a[i] += 10; a[i+1]--; } c[i] = a[i] - b[i];`。处理前导零：`while (len > 1 && c[len-1] == 0) len--;`。如果需要处理负数：先比大小，如果 a<b 则标记负号并交换 |
| 32-42 分 | 讲授 + 演示 | **高精度乘法**：分两个难度。①高精×单精：`for (int i=0; i<len; i++) { c[i] += a[i] * b; c[i+1] += c[i] / 10; c[i] %= 10; }`——每一位乘以单精数 b，统一进位。②高精×高精：双层循环 `c[i+j] += a[i] * b[j]`——模拟竖式乘法中的"错位相加"。关键是理解为什么索引是 `i+j`：因为 a 的第 i 位（10ⁱ）× b 的第 j 位（10ʲ）= 结果贡献到 10ⁱ⁺ʲ 位 |
| 42-45 分 | 小结 | "高精×高精的核心是**双层循环 + 错位相加**。索引 `i+j` 来自指数运算：10ⁱ × 10ʲ = 10ⁱ⁺ʲ。" |

## 🔥 重点难点

### 教学重点

1. **倒序存储**：`a[0]` 存个位。这是整个高精度运算的基石——进位往高索引加，比往低索引加自然得多。
2. **进位处理**：`c[i+1] += c[i] / 10; c[i] %= 10;` 这两行是加法/乘法的核心。学生必须能用嘴说出这两行的含义。
3. **高精×高精的 `c[i+j]`**：这是本章最抽象的概念点。用竖式乘法画图——把 a 的每一位展开为 a[i]×10ⁱ——能帮助学生理解为什么是 i+j。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 倒序存储和正序输出混淆 | 输入后忘了倒序存，或输出时没有倒序输出 | 固定模式：`输入→倒序存入 a[]→计算→倒序输出从 len-1 到 0`。板书写成三阶段流程图，让学生每次写高精度都先画这个图 |
| 加法/乘法最后一位进位忘了 | `123+987` 结果是 `110` 而不是 `1110`（丢了最后的进位） | 在循环后加 `if (carry) a[len++] = carry;` 或 `if (c[len]) len++;`。口诀：**"最高位上的进位，是新数字的第一位"** |
| 高精减法被减数 < 减数时出错 | `23-45` 得到奇怪的负值或大数 | 先比大小：写 `compare(a, b)` 函数，如果 a<b 则标记负号并 swap。减法只处理大减小的情况 |
| 高精×高精的数组大小不够 | 两个 100 位数相乘，数组只开了 200 | 法则：**lenC = lenA + lenB**（两个数的乘积最多 lenA+lenB 位）。数组至少开到这个大小 |
| 前导零忘了处理 | `1000-999=001` 输出了 "001" | 统一在输出前用 `while (len>1 && c[len-1]==0) len--;` 去掉前导零。但至少保留 1 位（结果为 0 时输出 "0" 而非空） |

## 💬 常见问答

**Q1："为什么倒序存储？正序（高位在 0）不行吗？"**

A：正序也行，但进位/借位操作需要往低索引（数组头部）插入元素，极其低效（需要整体后移）。倒序存储让进位往**数组尾部**扩展——`a[len++] = carry`——O(1) 操作。这是"让操作方向 = 数组增长方向"的工程智慧。

**Q2："高精度除了用数组，还能用别的吗？"**

A：能。①`string`——方便输入输出，但每次修改需要转换，效率较低 ②`vector<int>`——与数组类似但自带长度管理 ③压位技巧——每个数组元素存 4 位或 9 位（如存 0~9999 而非 0~9），大幅减少循环次数。竞赛中常用"万进制"（每位存 4 个十进制位）。

**Q3："高精×高精的复杂度是多少？"**

A：朴素方法（本章教的）O(n²)，其中 n 是位数。有更快的算法如 Karatsuba O(n^log₂3) ≈ O(n^1.585) 和 FFT O(n log n)——但这些远超入门级范围，大学算法课才学。NOIP 普及组级别的高精度乘法，O(n²) 足够了。

**Q4："高精除法为什么从高位到低位？和加减乘相反？"**

A：因为除法是从最高位开始"分糖果"的过程——先把最大的那部分拆分掉，余数留给下一位继续分。这和手工长除法一模一样：你先看最高位能不能除，能就先除，余下的和次高位合起来继续除。加减乘是从低位往高位处理，除法是从高位往低位——**除法是唯一"反着走"的高精度运算**。

**Q5："高精度运算在竞赛中常见吗？"**

A：入门级（CSP-J/NOIP 普及组）偶尔出现，通常作为一道题的辅助工具（比如求 2¹⁰⁰ + 3⁵⁰）。提高级更多会用 Python（自带大整数）或 Java（BigInteger）——但 C++ 选手必须自己写。掌握高精度，本质上是在说："我不依赖语言特性，我理解运算本身。"

## 📋 本章小测（10 分钟，闭卷）

### 一、填空题（每空 2 分，共 8 分）

1. 高精度运算中，数组通常采用 __________ 存的方式，即 a[0] 存 __________ 位。
2. 高精度加法的进位处理是 `c[i+1] += __________; c[i] __________;`
3. 两个长度分别为 m 和 n 的高精度数相乘，结果最多 __________ 位。
4. 高精度除法从 __________ 位往 __________ 位做（与加减乘相反）。

### 二、改错题（6 分）

5. 以下高精度加法的代码有什么 bug？
```cpp
for (int i = 0; i < len; i++) {
    c[i] = a[i] + b[i];
    c[i+1] = c[i] / 10;
    c[i] %= 10;
}
```

### 三、写代码（6 分）

6. 写一个函数计算 n!（n ≤ 50）。要求使用高精度乘法（高精 × 单精）。

---

### 答案

1. 倒序，个
2. `c[i] / 10`，`%= 10`
3. m + n
4. 高，低
5. `c[i] = a[i] + b[i]` 应改为 `c[i] += a[i] + b[i]`。因为 `c[i]` 可能已经从前面的进位中获得了值，直接用 `=` 会覆盖掉之前的进位。同时 `c[i+1]` 也应 `+=` 而非 `=`。
6. ```cpp
int a[1005] = {1}, len = 1;  // a = 1
for (int k = 2; k <= n; k++) {
    int carry = 0;
    for (int i = 0; i < len; i++) {
        a[i] = a[i] * k + carry;
        carry = a[i] / 10;
        a[i] %= 10;
    }
    while (carry) { a[len++] = carry % 10; carry /= 10; }
}
for (int i = len - 1; i >= 0; i--) cout << a[i];
```

## 🎮 不插电活动

### 活动 1：竖式接力——高精度乘法（第 2 课时用，约 14 分钟）

**道具**：
- A3 大纸（或白板），画好竖式乘法的格子
- 两组学生，每组 3 人
- 计时器

**规则与流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 在黑板上写两个 6 位数的乘法题目：`123456 × 789012`。两组各自准备，第一个同学负责个位×被乘数，第二个负责十位×被乘数，第三个负责百位×被乘数 + 把所有部分积加起来 |
| 2-8 分 | 两组 | 比赛开始！第一组和第二组同时开始，各自在 A3 纸上列竖式。全班喊倒计时 |
| 8-10 分 | 老师 | 两组展示结果，全班确认答案。引导观察：①部分积要"错一位"写（因为十位乘的结果其实是×10，所以在个位下面空一格）②"错位"恰好对应代码里的 `c[i+j]` |
| 10-12 分 | 老师 | 映射到代码："你们刚才的过程——被乘数 123456 的每一位（6→5→4→3→2→1）依次乘以 789012 的每一位（2→1→0→9→8→7），然后错位相加——就是代码里的双层循环 `c[i+j] += a[i] * b[j]`" |
| 12-14 分 | 老师 | 总结："手工列竖式时你自动做了'错位'——写第二行时往左移一格。代码里的 `i+j` 就是这'移一格'的数学表达。" |

**教学要点**：
- 竖式计算容易写错位——故意让一组"忘掉错位"直接对齐加，和正确结果对比，体会错位的重要性
- 把纸上的"错位"和代码的 `i+j` 直接连线——视觉辅助理解抽象索引

### 活动 2：进位链——加法可视化（第 1 课时用，约 8 分钟）

**道具**：10 个学生坐成一排，每人手里一个数字牌（0~9）

**规则**：
- 模拟 `999 + 1`：初始最后一个人举"0"（个位）
- 老师喊"加 1！"——个位变成 10，"进位！"——个位变 0，十位 +1
- 十位变成 10，"进位！"——十位变 0，百位 +1
- ……一路进位到最前面！全班看"多米诺骨牌"式的进位链
- 总结：这就是代码里 `c[i+1] += c[i]/10; c[i] %= 10;` 的物理呈现

## 📦 补充题库

1. 高精度加法中，如果两个数位数不同，循环条件应该是什么？  
（参考答案：`i < max(lenA, lenB)`，短的数用 0 补齐——因为短数的高位在数组中本来就是 0。）

2. 用高精度减法计算 1000! - 999!（提示：不要真的算两个阶乘——1000! = 1000 × 999!，所以结果是 999 × 999!。但作为练习可以做。）

3. 把高精度加法改成"压 4 位"（万进制）：每个数组元素存 0~9999，进制从 10 变成 10000。输出时需要 `printf("%04d", a[i])` 补前导零。  
（参考答案：只需把 `/10` 改成 `/10000`，`%10` 改成 `%10000`。注意最后一个元素不需要补 0。）

4. 写一个函数比较两个高精度数的大小。  
（参考答案：先比长度——长者大；长度相同从高位往低位逐位比。`compare(a, lenA, b, lenB)`——返回 -1（a<b）、0（a=b）、1（a>b）。）

5. 综合题：用高精度计算斐波那契数列的第 500 项。提示：`F[1]=1, F[2]=1, F[n]=F[n-1]+F[n-2]`，每次加法都是高精度。  
（参考答案：用二维数组 `int f[505][200]`，每一位存一个十进制位。循环 `for (int n=3; n<=500; n++)`，每次调用高精度加法。）

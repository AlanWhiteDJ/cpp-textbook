## 🗺️ 教学流程

### 第 1 课时：排列枚举——`next_permutation` + 三连击（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 老师提问："1~9 的九位数，全排列有多少种？"学生答 9! = 362880。"如果用九重循环枚举呢？"——9⁹ ≈ 3.87 亿。引出核心问题：**排列天然保证不重复，远比多重循环高效——但怎么生成排列？** |
| 8-20 分 | 讲授 | 介绍 `next_permutation`：①头文件 `<algorithm>` ②参数：起始和结束地址 ③返回值：有下一排列返回 true，到底返回 false ④必须用 `do-while` 而非 `while`（否则会漏掉初始排列）。板书 `{1,2,3}` 的 6 个排列输出，让学生观察**字典序**的顺序 |
| 20-30 分 | 讲授 | **三连击问题**：1~9 分成三组三位数，使 y=2x, z=3x。板书分析：把 1~9 放入数组，`do-while` 遍历全排列，每个排列取前 3 位→x，中 3 位→y，后 3 位→z，检查倍数关系。强调：9! = 362880 次对计算机来说眨眼间完成——**不必找数学规律，靠遍历拿下** |
| 30-40 分 | 演示 | 老师 IDE 演示：①先展示没有 `sort` 的错误版（只输出部分排列）②加上 `sort` 的正确版（输出全部 6 个排列）③跑三连击（输出两组答案）。强调：`next_permutation` 的神奇之处——三个参数就解决了九重循环的噩梦 |
| 40-45 分 | 小结 | 口诀："**排列枚举不用愁，`next_perm` 解你忧。sort 升序起好头，do-while 循环走。**"布置思考："如果要求 1~9 数字不重复，但只取 3 个排列呢？" |

### 第 2 课时：子集枚举——二进制掩码 + 位运算（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 老师假设场景："有 5 本书，每本书可以带或不带进考场。你一共有多少种带书的方案？"学生答 2⁵ = 32。"怎么用代码列出全部 32 种方案？"——引出子集枚举 |
| 8-20 分 | 🎮 不插电活动 | **"二进制手势"**（详见活动区）。5 个学生举左手（0）或右手（1），用身体表示 5 位二进制数，从 00000 数到 11111。老师喊一个 0~31 的数，学生用身体摆出对应二进制 |
| 20-32 分 | 讲授 | **二进制枚举原理**：n 个元素 → 2ⁿ 个子集。用一个整数 `mask` 从 0 到 `(1<<n)-1`，第 i 位为 1 表示选第 i 个元素。板书位运算速查表：`mask & (1<<i)` 判第 i 位、`mask | (1<<i)` 设 1、`mask & ~(1<<i)` 设 0、`mask ^ (1<<i)` 翻转。让学生在本子上用 4 位二进制手写出 0~15 的子集对应关系 |
| 32-42 分 | 练习 | 学生上机：①输出 1~5 的所有子集（用二进制枚举）②改进：输入 n 个整数，输出所有和为 0 的子集。老师巡场，检查 `mask & (1<<i)` 的括号（位运算优先级低于 `==`！） |
| 42-45 分 | 总结 | "今天你掌握了一把瑞士军刀——二进制枚举。它是后续状压 DP 的基础。记住：**n 个开关 → 2ⁿ 种状态 → 一个整数全表示**。" |

### 第 3 课时：多重循环优化 + 剪枝枚举（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 讲授 | **多重循环优化**：a+b+c=100 且 a≤b≤c。板书推导：a 最大 33（≤100/3），b 从 a 到 (100-a)/2，c=100-a-b 直接算。三重→两重，次数从 10⁶ 降到 ~5000。强调：**"有序约束"（a≤b≤c）是缩小范围的神器**——有了它，每个变量的上限都变紧 |
| 10-20 分 | 讲授 | **剪枝思想**：在枚举过程中提前剔除不可能的分支。类比：图书馆找红书→看到全蓝书架跳过整排。核心句式：`if (!可能) continue;`。**回文日期**：与其枚举每天判回文，不如枚举年份构造回文日期。1000 年只需 1000 次。板书剪枝前后的复杂度对比：枚举天→O(365×N) vs 枚举年→O(N) |
| 20-38 分 | 独立练习 | 学生完成两道题：①a+b+c=30 且 a≤b≤c，列出所有(a,b,c)组合（用缩范围+公式替代）②回文日期：输入起始和结束日期（如 20200101 20201231），统计回文日期个数。提示：拆成年月日验证合法性，注意闰年判断 |
| 38-45 分 | 讲评 | 投影 2-3 份代码。重点讲：①回文日期构造函数——`rev = rev*10 + tmp%10` 循环 4 次 ②闰年判断条件 `(y%4==0 && y%100!=0) || y%400==0` ③2 月天数特判。强调：剪枝的本质是**改变枚举的对象**——不是优化同一对象的枚举方式，而是换一个更少的对象来枚举 |

### 第 4 课时：小结 + 小测（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|

## 🔥 重点难点

### 教学重点

1. **`next_permutation` 的 do-while 循环**：这是最容易写错的地方——用 `while` 会漏掉初始排列。让学生记住"先做再判"的 do-while 语义。
2. **二进制枚举的位运算**：`mask & (1<<i)` 是后续所有位运算应用的基础。要求学生在纸上画 4 位的表格，手写出 mask 0~15 对应的子集。
3. **剪枝的本质**：不是优化"检查方式"，而是改变"枚举对象"。这是算法思维的关键飞跃——从"怎么更快地检查"到"检查什么东西更少"。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| `next_permutation` 初始数组没排序 | 只输出部分排列 | 刻意演示"不排序版"的 bug——输出 `{3,1,2}` 的下一个排列而非全部。让学生看到 bug 后自己加 `sort`。口诀：**"排列要全套，sort 不可少"** |
| 位运算优先级搞错 | `mask & 1 << i` 不加括号 | 法则：**位运算一律加括号**。写成 `(mask & (1 << i))`。宁可多写括号，不靠优先级猜 |
| 子集枚举 `1 << n` 在 n 大时溢出 | n=31 时 `1<<31` 是负数 | 讲明 `int` 最多 31 位（去掉符号位）。当 n ≥ 31 时用 `1LL << n`。同时在题目中 n ≤ 20，避免此问题 |
| 剪枝条件写反 | `if (条件) continue;` 把合法情况跳过了 | 让学生朗读自己的 if 条件——"如果这个情况是合法的就跳过"——一听就不对。正确：`if (!条件) continue;`——"如果不合法就跳过" |
| 回文日期中月份和日期边界 | 13 月 32 日也通过了 | 板书月份天数表，强调 2 月 + 闰年特判。让学生用边界值测试：`20201301`（13 月不存在）、`20200230`（2 月无 30 日） |

## 💬 常见问答

**Q1："`next_permutation` 能用 `string` 吗？"**

A：能！`string s = "123"; do { ... } while (next_permutation(s.begin(), s.end()));`——`string` 也有 `begin()` 和 `end()`，用法一模一样。这对处理字符排列（如全排列字母）特别有用。

**Q2："二进制枚举为什么从 0 开始？0 代表什么？"**

A：0 的二进制是 `000...0`，表示"一个都不选"——即空集。空集也是合法的子集。2ⁿ 个子集包含空集 + 全部 2ⁿ-1 个非空子集。

**Q3："什么时候用排列枚举，什么时候用子集枚举？"**

A：看题目关键词——"排列""全排列""不同顺序算不同"→排列枚举；"子集""组合""选或不选"→子集枚举。排列关心**顺序**（{1,2} 和 {2,1} 是两种），子集不关心顺序（{1,2} 就是一种）。

**Q4："回文日期那题，为什么不直接枚举所有日期？1000 年才 365000 天啊。"**

A：确实，1000 年 ≈ 365000 天，C++ 毫秒级跑完。这道题用剪枝是为了教学——让学生体会"改变枚举对象"的思想。但当日期跨度是 10⁹ 级别时（公元 1 年到公元 10⁹ 年），枚举每天就真的炸了。好习惯从简单的题开始养。

**Q5："位运算符 `&` `|` `^` `~` 什么时候会用到？"**

A：除了枚举子集，位运算在竞赛中用途极广：①判断奇偶（`n & 1`）②乘除 2 的幂（`n << 1` = n×2, `n >> 1` = n/2）③状态压缩 DP（用整数的二进制位表示集合）④哈希、加密等。位运算是 C++ 竞赛选手的"肌肉记忆"——越快掌握越好。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. `next_permutation` 要求初始数组必须是 __________ 序，这样才能遍历所有排列。
2. 二进制枚举 n 个元素的子集，循环范围是 `mask` 从 0 到 __________。
3. 判断 `mask` 的第 i 位是否为 1 的表达式是 __________。
4. 剪枝的核心思想是在枚举过程中 __________ 不可能的分支。

### 二、读代码题（每题 5 分，共 10 分）

5. 以下代码有什么 bug？
```cpp
int a[] = {3, 1, 2};
do { cout << a[0] << a[1] << a[2] << endl; }
while (next_permutation(a, a + 3));
```

6. 以下代码想枚举数组 `arr[5]` 的所有子集，运行结果为什么有漏？
```cpp
for (int mask = 1; mask < (1 << 5); mask++) {
    for (int i = 0; i < 5; i++)
        if (mask & 1 << i) cout << arr[i] << " ";
    cout << endl;
}
```

### 三、写代码（每题 10 分，共 20 分）

7. 用 `next_permutation` 输出 1~4 的所有全排列。

8. 输入 n 个整数，输出所有子集（包括空集）。要求使用二进制枚举。

---

### 答案

1. 升序
2. `(1 << n) - 1`
3. `mask & (1 << i)`（或 `(mask >> i) & 1`）
4. 提前排除
5. 数组未排序，只会输出从 `{3,1,2}` 开始的后半排列。应在 `do-while` 前加 `sort(a, a+3)`。
6. 两处问题：① `mask` 从 1 开始漏了空集（`mask=0`）② `mask & 1 << i` 缺少括号——`<<` 优先级高于 `&`，应改为 `(mask & (1 << i))`。
7. ```cpp
int a[] = {1, 2, 3, 4};
do {
    for (int i = 0; i < 4; i++) cout << a[i] << " ";
    cout << endl;
} while (next_permutation(a, a + 4));
```
8. ```cpp
int arr[100], n;
cin >> n;
for (int i = 0; i < n; i++) cin >> arr[i];
for (int mask = 0; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++)
        if (mask & (1 << i)) cout << arr[i] << " ";
    cout << endl;
}
```

## 🎮 不插电活动

### 活动 1：二进制手势——子集枚举（第 2 课时用，约 12 分钟）

**道具**：
- 5 张椅子排成一排
- 可用左手（0）右手（1）或举纸牌

**规则与流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 请 5 个学生坐在椅子上，面向全班。规则：举左手 = 0（不选这个元素），举右手 = 1（选这个元素）。5 个人就是一个 5 位的二进制数 |
| 2-4 分 | 全体 | 从 00000（全左手）开始，老师喊"加 1！"——最右边的同学换手。类似二进制计数器：00000→00001→00010→00011→00100……全班一起大声数 0、1、2、3…… |
| 4-8 分 | 全体 | 加速！老师快速喊数字（3、7、15、21、31），5 个学生必须快速用身体摆出对应二进制。全班判断对不对 |
| 8-10 分 | 老师 | 随机指定 3 个同学摆出一个 3 位的二进制数（如 101），问全班："这是数字几？""第 0 位是几？""这个子集包含了哪些元素？" |
| 10-12 分 | 老师 | 总结："你们身体摆出的每一个手势组合，对应 `for (int mask = 0; mask < 32; mask++)` 中的一次循环。32 种手势 = 32 个子集。这就是计算机看到的世界——**一切都是 0 和 1**。" |

**教学要点**：
- 注意"右手=1"这个设定让手势直观（右手通常更有力），提前说清楚避免混淆
- 从 00000 数到 11111 的过程让学生亲身体验"遍历所有子集"的规模（2⁵=32 还行，2¹⁰=1024 就数不过来了——体会指数增长）

### 活动 2：排列快排——字典序接力（第 4 课时用，约 10 分钟）

**道具**：
- 3 个学生，胸前分别贴"A""B""C"
- 白板写 6 种排列的参照表：ABC, ACB, BAC, BCA, CAB, CBA

**规则**：
- 老师喊当前排列（如"ABC"），3 个学生站好。老师喊"下一排列！"——学生要快速换位成下一个字典序排列（ACB）
- 全班按顺序喊出 6 个排列，3 个学生不停变换队形
- 加速！老师随机跳到一个排列（如"CAB"），让学生摆出"下一排列"（CBA）
- 最后问："为什么 CBA 没有下一排列了？"——因为它是最大字典序

## 📦 补充题库

1. 以下关于 `next_permutation` 的说法，正确的是？（多选）  
A. 头文件是 `<algorithm>` ✅  
B. 初始数组需要升序 ✅  
C. 返回 `false` 时数组变回最小排列 ✅  
D. 可以用在 `string` 上 ✅  

2. n=4 时，二进制枚举共遍历多少次？空集对应 mask 等于多少？  
（参考答案：16 次（0~15），空集对应 mask=0。）

3. 用二进制枚举找出数组 `{2, -3, 5, -1, 4}` 中所有和为 0 的子集。  
（参考答案：mask 遍历 0~31，累加每位对应的元素，判 sum==0。答案：{-3, -1, 4} 即 mask=01011=11。）

4. 回文日期问题：如果日期范围是从 10000101 到 99991231，用"枚举年构造回文"的方法需要多少次检查？如果用"枚举每一天"呢？  
（参考答案：枚举年=9999-1000+1≈9000 次；枚举天≈9000×365≈328 万次。差距约 365 倍。）

5. 有一个 0/1 背包问题：n 个物品，重量 w[i]，价值 v[i]，背包容量 W。用二进制枚举找出最大价值。当 n=20 时，枚举次数是多少？n=30 呢？  
（参考答案：2²⁰≈10⁶ 次（安全），2³⁰≈10⁹ 次（危险）。这正好解释了为什么二进制枚举只能处理 n≤20 的题目——n=30 时 2³⁰≈10 亿，一秒跑不完。）

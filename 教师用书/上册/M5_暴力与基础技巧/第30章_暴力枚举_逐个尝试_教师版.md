## 🗺️ 教学流程

### 第 1 课时：枚举思想 + 次数爆炸（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 老师讲密码锁故事："小明忘了三位密码锁，从 000 试到 247 开了。人试要 10 分钟，电脑试 1000 次连眨眼的功夫都不用。"引出核心问题：**计算机比人快在哪里？** |
| 8-22 分 | 讲授 | 枚举定义：列出所有可能，逐一检查。枚举三要素：①循环生成候选 ②检查条件 ③符合就输出。板书"找 1~100 中的偶数"示例，拆解每一步。强调：枚举不需要灵光一现的巧思，只需要踏实列出来——**这是算法世界里最诚实的方法** |
| 22-32 分 | 讲授 | **次数爆炸**：枚举虽好，不能无脑用。板书估算表：1 万次→瞬间，100 万次→~0.01 秒，1 亿次→~1 秒，10 亿次→超时。经验法则：单重循环（10⁶）随便枚举；双重循环（10⁶×10³=10⁹）小心；三重循环（10³×10³×10³=10⁹）几乎必超时 |
| 32-40 分 | 互动 | 老师出题："八位密码锁（00000000~99999999），电脑一秒试一亿次，多久试完？"让学生口算：10⁸/10⁸=1 秒。"如果是十位密码呢？"→10¹⁰/10⁸=100 秒。体会"指数增长"的可怕 |
| 40-45 分 | 小结 | 口诀："**枚举诚实不取巧，次数多了就烦恼。单重放心跑，三重要检讨。**"布置思考题："百钱百鸡三重循环 103 万次能跑吗？能不能更少？" |

### 第 2 课时：百钱百鸡 + 枚举优化三法（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | "上节课说三重循环 10⁹ 必超时。那百钱百鸡三重循环 103 万次呢？"——能跑，但浪费。引出优化 |
| 5-20 分 | 🎮 不插电活动 | **"你是编译器"**（详见活动区）。三个学生分别扮演 gong、mu、xiao 循环变量，在老师指挥下演示三重循环→两重循环的优化过程 |
| 20-30 分 | 讲授 | 板书百钱百鸡从三重到两重的优化全过程：①公鸡最多 100/5=20 只（缩范围）②母鸡最多 100/3≈33 只（缩范围）③小鸡=100-公鸡-母鸡（减变量，用等式推导省掉一层循环）。次数：101³≈103 万→21×34=714 次，缩小 1400 倍！ |
| 30-40 分 | 讲授 | 枚举优化三法总结：①**缩范围**——根据约束缩小循环起止（公鸡 0~20 而非 0~100）②**减变量**——能用等式推导的变量不枚举（小鸡自动算出）③**早退出**——找到答案立刻 break（搜索类问题）。口诀："**范围缩到最小，变量能少就少，找到赶紧跑**" |
| 40-45 分 | 演示 | 老师现场 IDE 敲百钱百鸡两版代码，跑给学生看——两版在感觉上都快如闪电（都在毫秒级）。强调：百万级枚举对计算机完全不是负担。让学生破除"循环多就是差代码"的错误观念 |

### 第 3 课时：勾股数 + 水仙花数 + 方程整数解（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 讲授 | **勾股数**：a²+b²=c²。从三重（a、b、c 各 1~100，100³=100 万）优化到两重（c=√(a²+b²)自动算出，~1 万次）。重点讲浮点误差处理：`int c = (int)(sqrt(c2) + 0.5); if (c*c == c2)`——加 0.5 再取整防舍入，再用整数平方回验 |
| 10-20 分 | 练习 | 学生在电脑上写勾股数程序。要求：①先用三重循环跑一次（体会 100 万次）②再改成两重循环（体会差别）。老师巡场，检查浮点误差处理是否到位 |
| 20-30 分 | 讲授 | **水仙花数**和**方程整数解**：这两个是"能一行枚举就不拆多层"的典型案例。水仙花数直接 `for (n=100; n<=999; n++)` 拆位判断——为什么不用三重循环分别枚举百十个位？因为三位数天然连续，一行搞定。方程 `3x+5y=100`：x 枚举 0~33，y=(100-3x)/5，用 `%5==0` 判整除 |
| 30-42 分 | 独立练习 | 学生在电脑上完成：①水仙花数（所有三位数）②方程 `3x+5y=100` 全部自然数解 ③思考：如果把 100 换成 10000，三重循环分别枚举 x、y、z 解方程 `3x+5y+7z=10000` 要多少次？怎么优化？ |
| 42-45 分 | 总结 | "今天的四道题，优化后次数分别是：714（百鸡）、1 万（勾股数）、900（水仙花）、34（方程）。没有一道超过 10⁶——现代计算机毫秒级跑完。这就是枚举的魅力：**思路简单到不用技巧，但靠速度硬算出来。**" |

### 第 4 课时：小结 + 小测（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 思维导图 | 师生共同构建本章知识树：枚举定义（三要素）→ 次数估算（10⁶/10⁸/10⁹ 分界线）→ 优化三法（缩范围/减变量/早退出）→ 经典题（百鸡/勾股/水仙花/方程） |
| 10-28 分 | 小测 | 6 道题（填空+读代码+写代码），闭卷 18 分钟 |
| 28-35 分 | 互批 | 同桌交换批改，老师公布答案并逐题讲解 |
| 35-45 分 | 展望 | "枚举是把所有可能列出来。但有一类题目——比如'石头剪刀布，赢 3 局才算胜'——它的过程是一步步走的，每一步影响下一步。这就不是枚举能简单解决的了。下一章，我们学**模拟**——把现实规则翻译成代码，让程序一步步执行。模拟题是竞赛的'送分大户'！" |

## 🔥 重点难点

### 教学重点

1. **枚举三要素**：循环生成→条件检查→记录输出。这是整个算法思维的起点，必须让学生形成肌肉记忆。
2. **次数估算能力**：学生必须学会在看到循环嵌套时本能地估算总次数。10⁶（安全）、10⁸（谨慎）、10⁹（危险）的分界线要刻在脑子里。
3. **减变量优化**：百钱百鸡从三重降到两重——"能用等式推出的变量不枚举"——这是后续所有算法优化的思想种子（动态规划的核心也是这个思想）。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 分不清"枚举"和"模拟" | 学生把所有循环都叫枚举 | 给两个对比题：①"找出 1~100 所有素数"（枚举——生成所有数逐个判断）②"模拟掷骰子 10 轮比赛"（模拟——按照规则一步步执行）。枚举 = 列出所有可能性再筛选；模拟 = 按时间线一步步走 |
| 不知道枚举范围怎么定 | `for (int gong=0; gong<=100; gong++)` 而不是 `<=20` | 反问："100 文全买公鸡能买几只？"让学生自己算 100/5=20。口诀：**"先算最大买几只，上限就是那个数"** |
| 小鸡数用整数除法静默出错 | 写了 `xiao/3` 但不先检查 `xiao%3==0` | 板书画图：33/3=11（整除），34/3=11（C++整数除法截断，也等于 11！）。让学生体会：不检查整除性会错误地把 34 只小鸡当成 11 文钱算 |
| 浮点 `sqrt` 直接 `==` 比较 | `if (sqrt(c2) == c)` 有时对有时错 | 演示：`sqrt(25)` 可能是 `4.999999999999999` 而不是 `5.0`。解释浮点误差来源，教给学生"加 0.5 取整 + 整数回验"的套路 |
| 看到复杂题就想"枚举肯定不行" | 拿到题先自我否定，不敢动手 | 教给学生倒推法：①算最大枚举次数 ②如果次数 ≤ 10⁷，直接枚举 ③如果太大，再想优化。**数字比直觉可靠** |

## 💬 常见问答

**Q1："枚举不就是暴力吗？暴力听起来很 low，有没有更聪明的方法？"**

A：暴力不 low——它是所有算法的基础。很多高级算法（动态规划、贪心）都是从暴力出发，发现重复计算后才优化的。而且，竞赛中很多题的数据范围就是设计给你暴力的——**枚举写对就是 100 分，高级算法写错就是 0 分**。先保证"能拿的分绝不丢"，再追求"更优解"。

**Q2："为什么 10⁸ 次约等于 1 秒？这个数字怎么来的？"**

A：现代 CPU 主频约 2~4 GHz，即每秒 20~40 亿个时钟周期。C++ 的一条简单语句（比如一次循环迭代里的比较+加法+赋值）大概消耗几个到几十个时钟周期。所以每秒大约能执行 10⁸~10⁹ 次循环迭代。这是经验估算，不是精确值——不同的操作（比如循环里有 `sqrt` 就会慢很多）会浮动。但"10⁸ ≈ 1 秒"是一个好记的锚点。

**Q3："百钱百鸡那题，公鸡为什么最多 20 只？题目没说要'恰好花完 100 文'啊。"**

A：题目说了"100 文钱买 100 只鸡"——恰好 100 文，恰好 100 只。公鸡 5 文一只，全买公鸡也只能买 20 只（20×5=100 文），再多钱就不够了。这就是"最优边界"——用题目里的约束条件把循环范围压到最小。

**Q4："小鸡必须是 3 的倍数这个条件，为什么不在循环里直接保证，而要放到 if 里判断？"**

A：好问题！确实可以在循环里 `for (xiao=0; xiao<=100; xiao+=3)` 直接步进 3。但教学中先让学生写 `if (xiao%3==0)`，因为：①这是"逐个检查"的枚举思维最直观的体现 ②等学生写完后再点出 `xiao+=3` 的优化，能让他们体会"从能工作到能高效工作"的进化过程。

**Q5："勾股数那题，100 万次明明能跑，为什么要优化到 1 万次？"**

A：练手。100 万次确实能跑，但养成优化的习惯比这道题本身更重要。当你面对的不再是 100 以内的勾股数，而是 10000 以内的——三重循环就是 10¹² 次，必超时。现在就学会"能推导的变量不枚举"，将来才不会在更大的数据面前手足无措。

## 📋 本章小测（18 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. 枚举算法的三个步骤是：__________ → __________ → __________。
2. 一般经验：枚举次数在 __________ 以下可以放心跑；超过 __________ 就要小心超时。
3. 枚举优化的三种方法是 __________、__________、__________。

### 二、读代码题（每题 5 分，共 10 分）

4. 以下代码的枚举次数大约是多少？是否会超时？
```cpp
for (int i = 0; i < 1000; i++)
    for (int j = 0; j < 1000; j++)
        if (i * j == 2024) cout << i << " " << j << endl;
```

5. 下面这段百钱百鸡优化版代码有什么 bug？
```cpp
for (int gong = 0; gong <= 20; gong++)
    for (int mu = 0; mu <= 33; mu++) {
        int xiao = 100 - gong - mu;
        if (5*gong + 3*mu + xiao/3 == 100)
            cout << gong << " " << mu << " " << xiao << endl;
    }
```

### 三、写代码（每题 10 分，共 20 分）

6. 写程序找出 1~1000 之间的所有"完数"（完数 = 所有真因子之和等于自身的数，如 6=1+2+3）。提示：对于每个数 n，枚举 1~n/2 检查因子。

7. 写程序解决"马驮瓦"问题：100 匹马驮 100 块瓦，大马驮 3 块，中马驮 2 块，两匹小马合驮 1 块。列出所有解。要求：小马数必须为偶数，且利用总数约束减少一层循环。

---

### 答案

1. 循环生成候选，检查是否符合条件，符合就记录/输出
2. 10⁷（或 10⁶~10⁷），10⁸（或 10⁹）
3. 缩范围，减变量，早退出
4. 1000×1000=10⁶ 次，不会超时（毫秒级跑完）
5. 没有检查 `xiao % 3 == 0`。当 xiao 不是 3 的倍数时，`xiao/3` 会截断小数，导致金额算错。（还应检查 `xiao >= 0`）
6. ```cpp
for (int n = 2; n <= 1000; n++) {
    int sum = 0;
    for (int i = 1; i <= n/2; i++)
        if (n % i == 0) sum += i;
    if (sum == n) cout << n << endl;
}
```
7. ```cpp
for (int big = 0; big <= 33; big++)         // 大马最多 100/3≈33
    for (int mid = 0; mid <= 50; mid++) {   // 中马最多 100/2=50
        int small = 100 - big - mid;         // 小马自动算出
        if (small >= 0 && small % 2 == 0 &&
            3*big + 2*mid + small/2 == 100)
            cout << big << " " << mid << " " << small << endl;
    }
```

## 🎮 不插电活动

### 活动 1：你是编译器——枚举优化（第 2 课时用，约 15 分钟）

**道具**：
- 三张 A4 纸，分别写上"gong 0→100""mu 0→100""xiao 0→100"
- 一条长绳或胶带在地上标出"循环轨道"
- 白板或黑板

**场地布置**：
- 三位同学站成一排，每人举一张纸，代表三重循环变量
- 地上用胶带标出三个同心圆，内圈="内层循环"，外圈="外层循环"

**规则与流程**：

| 分钟 | 角色 | 操作 |
|------|------|------|
| 0-2 分 | 老师 | 布置场景：三个同学代表 gong、mu、xiao，依次排开。老师喊"gong=0!"→gong 同学举纸喊"到我！"，"mu=0!"→mu 同学举纸，"xiao=0!"→xiao 同学举纸。三人一起喊"条件检查——gong+mu+xiao==100？5*公鸡+3*母鸡+小鸡/3==100？"——这就是三重循环的每一步 |
| 2-5 分 | 全体 | **第一轮（三重循环版）**：gong 同学从 0 数到 5（缩短版），每数一下，mu 从 0 数到 5，每数一下，xiao 从 0 数到 5。全班一起数步数：6×6×6=216 步。体会嵌套的"乘法效应" |
| 5-10 分 | 老师 | "现在优化！"①老师问："公鸡最多几只？"全班喊"20！"——gong 同学的纸改成"0→20"②"母鸡最多几只？"全班喊"33！"——mu 同学的纸改成"0→33"③"小鸡数能不能算出来？"——xiao 同学把纸翻过来，背面写"=100-gong-mu"，退到一边不跑了（减掉一层循环！） |
| 10-13 分 | 全体 | **第二轮（优化版）**：gong 从 0 到 20，mu 从 0 到 33，xiao 站旁边直接报数。全班数步数：21×34=714 步。对比第一轮：714 vs 216（缩短版）——如果跑完 100×100×100=100 万步呢？差距更夸张 |
| 13-15 分 | 老师 | 总结："今天这个游戏的关键只有一个——**能算出来的就不要枚举！** xiao 同学被从循环里'开除'了，但他仍然在工作——他只是换了一种方式。这就是减变量优化的本质。" |

**教学要点**：
- 让学生用身体体验"嵌套循环 = 乘法"——三层嵌套是三层人在同时数数，数完要三层都回到原点才能开始下一轮
- xiao 同学"被开除"时要给掌声——这不是惩罚，而是光荣的"被优化"
- 强调：减掉的变量仍然参与计算（通过等式），只是不再参与循环

### 活动 2：枚举抢答赛（第 4 课时用，约 10 分钟）

**道具**：无（纯口头）

**规则**：
- 老师说一个枚举场景，学生举手抢答"需要多少次？"
- 场景 1："找 1~1000 中所有能被 7 整除的数"（1000 次）
- 场景 2："三位数的所有回文数"（900 次，100~999）
- 场景 3："两个骰子，所有可能的点数组合"（36 次，6×6）
- 场景 4："1~9 九位数全排列"（362880≈3.6×10⁵ 次）
- 场景 5："10 位密码锁，枚举所有密码"（10¹⁰ 次——超时！）
- 每道题答完后老师追问："会超时吗？"训练估算本能

**教学要点**：快速建立"范围→次数→是否超时"的神经反射弧。这是竞赛中最实用的元技能之一。

4. 写一个程序，找出所有满足 a²+b²=c² 且 a+b+c ≤ 100 的勾股数三元组（a < b < c）。要求使用"减变量"优化。（参考答案：枚举 a 和 b，c=√(a²+b²)，检查 c 是否为整数且 a+b+c ≤ 100）

5. 小明说："枚举 1~10000 的所有完全平方数，需要枚举 10000 次。"小红说："只需要 100 次。"谁对？为什么？  
（参考答案：小红对。完全平方数 = i²，i 从 1 到 √10000=100，只需枚举 100 次。这是一个用"生成式思维"替代"检查式思维"的典型案例——与其检查每个数是不是平方数，不如直接生成平方数。）

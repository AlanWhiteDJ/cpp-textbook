## 🔥 重点难点

### 教学重点

1. **`mid+1`/`mid-1`**：这是二分不死循环的关键。`l=mid` 和 `r=mid` 各在什么条件下用——需要刻意练习。
2. **lower_bound vs upper_bound**：前者是"第一个可以插入 x 且不破坏有序性的位置（如果 x 存在就是第一个 x）"，后者是"第一个 >x 的位置"。

### 常见卡点 + 突破

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| while(l<=r) vs while(l<r) 混淆 | 有时死循环，有时漏元素 | 规则：l<=r 配 mid+1/mid-1（最安全）；l<r 配 mid 和 mid+1 的特定组合 |
| mid 溢出 | n 很大时 (l+r) 超过 int 上限 | 统一用 `mid = l + (r-l)/2` |

## 💬 常见问答

**Q1："二分查找到底用 while(l<=r) 还是 while(l<r)？"**

A：推荐 `while(l<=r)` + `l=mid+1; r=mid-1`——最简单不易出错的组合。等熟练掌握后再学其他变体。

**Q2："如果数组里有重复元素，lower_bound 返回的是第几个？"**

A：第一个等于 x 的位置（如果 x 存在）。例如 `{1,2,2,2,3}` 中 `lower_bound(2)` 返回下标 1（第二个元素）。

**Q3："二分能不能用在非数组的场景？"**

A：当然！二分答案（下一章）就把二分用在函数值域上。任何有"单调性"的东西都可以二分。

**Q4："如果 mid 用 `(l+r)/2` 溢出怎么办？"**

A：当 l 和 r 都接近 int 上限（约 2×10⁹）时，l+r 超过 4×10⁹ 溢出为负数。根治方案：`mid = l + (r-l)/2`。这个写法永远不会溢出，且和 `(l+r)/2` 结果等价。

**Q5："为什么首推 while(l<=r) 而不是 while(l<r)？"**

A：`while(l<=r)` + `mid±1` 的组合保证每次循环都缩小搜索区间，永远不会死循环。`while(l<r)` 需要根据场景小心搭配 mid 的取整方向（向上/向下取整），搭配错了就死循环。l<=r 版虽然多一次迭代，但换来了绝对的确定性——适合初学者。

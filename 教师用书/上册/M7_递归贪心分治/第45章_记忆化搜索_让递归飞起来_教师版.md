## 🗺️ 教学流程

### 第 1 课时：重复计算 + 斐波那契记忆化（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | "数学作业重算 vs 翻草稿纸"故事。板书斐波那契递归树——F(5) 调 F(4)+F(3)，F(4) 又调 F(3)+F(2)……F(3) 被算了 2 次，F(2) 3 次，F(1) 5 次！ |
| 10-28 分 | 讲授 | 记忆化三步：①开 `memo[n+1]` 数组，初始化为 -1（表示未计算）②函数开头 `if(memo[n]!=-1) return memo[n]` ③计算结束后 `memo[n]=result; return result`。板书完整斐波那契记忆化代码，复杂度 O(2ⁿ)→O(n) |
| 28-38 分 | 演示 | IDE 对比：n=45 的朴素递归（等很久）vs 记忆化（瞬间）。让学生亲眼看到指数到线性的跨越 |
| 38-45 分 | 小结 | "记忆化 = 给递归装一个备忘录。算过的绝不重算。**时间换空间？不——是用很少的空间换巨大的时间。**" |

### 第 3 课时：网格路径 + 记忆化 vs DP（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-20 分 | 讲授 | 网格路径：从左上到右下，只能向右/下走，求路径数。递推：`dp[i][j]=dp[i-1][j]+dp[i][j-1]`。记忆化版：`memo[i][j] = dfs(i-1,j) + dfs(i,j-1)`。**记忆化是自顶向下（从目标往回找），DP 是自底向上（从起点往前推）** |

## 🔥 重点难点

### 教学重点

1. **memo 初始化和查询**：`-1` 表示未计算（或其他特殊值）。这是记忆化能工作的前提。
2. **记忆化代码改写**：在原有递归上只需加 3 行代码——开数组、开头查、结尾存。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| memo 初值选错 | 用 0 表示未计算，但合法结果可能也是 0 | 用 -1 或其他不可能的值。口诀："**取一个计算结果不可能等于的值作为'未计算'标记**" |
| 多维 memo 不知道维度 | 问"memo 该开几维？" | 看递归函数的参数有几个"状态"——递归有几个参数随调用变化，memo 就开几维 |

## 💬 常见问答

**Q1："记忆化和 DP 到底什么关系？"**

A：记忆化 = 自顶向下的 DP（从原问题出发，递归到边界）。传统 DP = 自底向上（从边界出发，递推到原问题）。复杂度相同，记忆化更容易写（不需要设计递推顺序），DP 常数更小。

**Q2："什么时候用记忆化，什么时候用 DP？"**

A：①状态转移不规律（如博弈搜索）→记忆化 ②状态转移有明确的拓扑序（如网格）→DP ③初学→先写记忆化再改 DP。

**Q3："记忆化能解决所有递归问题吗？"**

A：不能。只有存在**重叠子问题**的递归才有效——即同一个状态被反复计算。汉诺塔每次的状态都是新的，记忆化对它无效。

**Q4："memo 数组需要多大？怎么确定维度？"**

A：维度 = 递归函数中会变化的参数个数。比如 `fib(n)` 只有 n 一个参数→一维 memo[n+1]。`dfs(i,j)` 有两个→二维 memo[n][m]。大小 = 每个参数的最大取值 + 1。

**Q5："记忆化搜索和 DP 表格有什么本质关系？"**

A：它们是同一个递推关系的两种计算方向。记忆化用递归从大到小（自顶向下），DP 用循环从小往大（自底向上）。memo 数组就是 DP 的表格——只是填表顺序不同。理解了记忆化，DP 就只剩"把递归转成循环"的语法差异了。

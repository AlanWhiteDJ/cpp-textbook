## 🗺️ 教学流程

### 第 1 课时：string 是什么 + 声明与输入输出（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | **问题驱动**：板书"Hello World"和"张三"。提问："这些文字怎么在 C++ 里存储？用 int？用 char？"引出 string。强调——编程不仅要处理数字，还要处理文字 |
| 8-18 分 | 讲授 | string 类型介绍。`#include <string>` 头文件（实际用 `iostream` 也包含但养成好习惯）。声明：`string s;` / `string name = "张三";`。强调：字符串用**双引号**，单引号是字符 |
| 18-28 分 | 不插电 | **"传话游戏"**（详见活动1）——让学生直观感受字符串传递与"丢失信息" |
| 28-35 分 | 讲授+演示 | string 的输入输出：`cin>>s; cout<<s;`。关键发现：`cin` 读字符串**遇到空格就停**！演示：输入"Hello World"，只输出"Hello" |
| 35-42 分 | 互动 | 让学生猜测："我想读入'Zhang San'（中间有空格）该怎么办？"——先不急着给答案，让他们思考。有同学可能会猜"用两个 string"，追问"那要存的姓名数量不确定呢？" |
| 42-45 分 | 总结 | "string = 装文字的容器。它的输入方式和 int 不完全一样——遇到空格就停。下一节课教你怎么读一整行。" |

### 第 2 课时：字符串拼接与长度 + 索引访问（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速复习：string 声明、`cin` 遇空格停止 |
| 5-15 分 | 讲授 | **字符串拼接**。"用 + 号就能把两个字符串粘起来"。板书：`string a="Hello", b=" World"; string c=a+b; // c = "Hello World"`。强调：`+` 对 string 是拼接，对 int 是加法——同一个符号不同含义，这叫"运算符重载"（点到为止） |
| 15-25 分 | 讲授 | **字符串长度**。`s.length()` 或 `s.size()`。板书示例：`string s="abc"; cout<<s.length(); // 3`。强调：length 返回的是字符个数，不是字节数（英文字母 1 个，中文字一般占 2-3 个字节但不在这章讨论） |
| 25-35 分 | 讲授+上机 | **索引访问**——"字符串像字符数组一样，可以用下标访问每个字符"。`s[0]` 是第一个字符，`s[s.length()-1]` 是最后一个。学生上机：输入一个字符串，输出它的第一个和最后一个字符 |
| 35-42 分 | 讨论 | 提问："`s[s.length()]` 是什么？"——越界！和数组一样的规则：最大下标是 `length()-1` |
| 42-45 分 | 总结 | string 三个基本操作：`+` 拼接、`.length()` 长度、`[i]` 索引访问 |

### 第 3 课时：`cin` vs `getline` + 字符串遍历（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | 演示：输入"Hello World"并输出。`cin>>s` 只得到"Hello"。"有没有办法读到完整的'Hello World'？" |
| 5-15 分 | 讲授 | **`getline`——读一整行**。`getline(cin, s);` 会读入一整行（包括空格），直到遇到换行符。板书对比表：| | cin>>s | getline(cin,s) ||---|---|---|| 遇空格 | 停止 | 继续读 || 遇换行 | 停止 | 停止 || 用途 | 读单个词 | 读整行 |
| 15-20 分 | 讲授 | **重要陷阱**：`cin>>n` 后直接 `getline` 会出 bug！因为 `cin>>n` 后在输入缓冲区里还剩一个换行符没被吃掉，`getline` 一上来就读到换行符，得到一个空字符串。解决：`cin>>n; getchar();` 或 `cin.ignore();` |
| 20-28 分 | 上机 | 练习 1：读入一个整数 n，再读入一行字符串，输出 n 和字符串。让学生亲身体验"getline 读不到东西"的 bug，再教 `getchar()` 解法 |
| 28-38 分 | 讲授 | **字符串遍历**。"字符串也能用 for 循环遍历！"板书：`for(int i=0; i<s.length(); i++) cout<<s[i]<<endl;`——竖着每个字符一行。变体：倒序遍历、每隔一个字符输出 |
| 38-43 分 | 上机 | 练习：输入一行英文，统计其中大写字母个数（提示：`s[i]>='A' && s[i]<='Z'`） |
| 43-45 分 | 总结 | "cin vs getline，二选一要看场景。记住 `cin>>n` 后面接 `getline` 要先吃掉换行符！" |

### 第 4 课时：字符串比较 + 基础操作（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 热身 | 口头判断：`"abc"+"def"` 的结果是什么？`"123"+"456"` 是 579 吗？（不是！是 "123456"，字符串拼接不是数字加法） |
| 5-15 分 | 讲授 | **字符串比较**。"string 可以直接用 `==`、`!=`、`>`、`<` 比较！"`if(s1==s2)` 判断两个字符串是否完全相同。比较规则：按字典序（ASCII 码）逐个字符比较，和查字典的顺序一样 |
| 15-25 分 | 上机 | 练习：读入两个字符串，按字典序输出较小的那个。提升：读入 3 个字符串，输出字典序最小的 |
| 25-35 分 | 讲授 | **空字符串**。`string s = "";` 或 `string s;`（默认就是空串）。`s.empty()` 判断是否为空。常用模式："在不知道输入多长时，用空串不断拼接" |
| 35-45 分 | 综合练习 | 读入 n 个英文单词（n≤20），把它们全部拼成一句话（单词之间加空格），输出结果 |

## 🔥 重点难点

### 教学重点

1. **string 基本操作**：`+` 拼接、`.length()` 长度、`[i]` 索引访问——这是后续所有字符串操作的基础。
2. **`cin` vs `getline`**：区分两种输入方式的适用场景，尤其是混用时的陷阱。
3. **字符串遍历模式**：和数组一样用 for 循环 + 下标访问每个字符。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| `cin>>n` 后 `getline` 读不到 | 程序运行结果"少了一行" | 实物演示：在黑板上画一个"输入缓冲区"。`cin>>n` 读取了数字但留下了换行符。`getline` 上来就"撞到"换行符，以为这行结束了。解决 = 在中间插一个"吃掉换行符"的动作 |
| 用 `+` 拼接数字和字符串 | `string s = "age:" + 18;` 报错 | 说明 `+` 号两侧必须至少有一个是 string。正确写法：`"age:" + to_string(18)`。用类比："+ 是个服务员，见到 string 顾客就知道要拼接，见到两个数字顾客就做加法" |
| 混淆 `length()` 和下标最大值 | 访问 `s[s.length()]` | 数和编号的区别：length 是"有几个"，下标是"第几个（从 0 数）"。有 5 个字符，下标是 0~4。口诀："长度减一等于最后下标" |

## 💬 常见学生问题

**Q1："string 和字符数组 `char s[100]` 有什么区别？"**

A：string 是 C++ 提供的"智能字符串"，自动管理内存，可以用 `+`、`==`、`.length()` 等方便的操作。`char s[100]` 是 C 语言的"原始字符串"，功能简陋，容易出错，但速度极快。信奥入门阶段推荐用 string，等学到高级算法需要极致性能时，可能会涉及字符数组。对初学者：**能用 string 就用 string**。

**Q2："`"123" + "456"` 是 579 吗？"**

A：不是！字符串里的 "123" 是文字，不是数字。"123"+"456" = "123456"（把两个文字粘在一起）。要想做数字加法，先把字符串转成 int：`stoi("123") + stoi("456")` 才是 579。

**Q3："中文 string 可以用 `.length()` 吗？"**

A：可以，但结果可能不是你以为的"字数"。`string s="你好";` 在大多数环境下 `s.length()` 是 4 或 6（一个中文占多个字节），而不是 2（两个汉字）。处理中文的"字数"需要额外知识，入门阶段先用英文练习，中文暂时特殊对待。

**Q4："`getline` 读入的空格也算长度吗？"**

A：算。`getline` 读进来的所有字符（包括空格、标点）都在 string 里，都是 length 的一部分。只有换行符（你按的回车键）不会被存进去——它只是告诉 getline"这行结束了"。

## 📋 本章小测

1. **填空题**：声明一个字符串变量 name 并初始化为 "Alice" 的代码是：`___________`。

2. **选择题**：以下代码输出什么？
```cpp
string s = "Hello World";
cout << s[4];
```
A. `o` B. ` ` (空格) C. `H` D. `W`

3. **判断题**：`cin >> s` 可以读入包含空格的字符串。（ ）

4. **填空题**：获取字符串 s 的长度的代码是：`________`。

5. **编程题**：读入一行英文，将其中的小写字母全部转成大写字母输出。（提示：`s[i] = s[i] - 32;` 或者 `s[i] = s[i] - 'a' + 'A';`）

6. **改错题**：以下代码有什么问题？
```cpp
int n;
string s;
cin >> n;
getline(cin, s);
cout << n << " " << s;
```

7. **简答题**：简述 `cin>>s` 和 `getline(cin,s)` 的区别，各适用于什么场景？

---

### 参考答案

1. `string name = "Alice";`
2. A（s[0]='H', s[1]='e', s[2]='l', s[3]='l', s[4]='o'）
3. ×（`cin>>s` 遇到空格、Tab、换行会停止读入，只能读一个词）
4. `s.length()` 或 `s.size()`
5. 思路：`getline(cin,s); for(i=0;i<s.length();i++) if(s[i]>='a'&&s[i]<='z') s[i]=s[i]-32; cout<<s;`
6. `cin>>n` 后输入缓冲区留有换行符，`getline` 直接读到换行符得到空字符串。应在 `cin>>n` 后加 `getchar();` 或 `cin.ignore();`
7. `cin>>s`：遇空白字符（空格/Tab/换行）停止，适合读单个单词或数字。`getline(cin,s)`：遇换行符才停止，适合读包含空格的一整行（如姓名、句子）

## 🎮 不插电活动

### 活动 1：传话游戏——字符串传输（第 1 课时用，12 分钟）

**道具**：纸条和笔（每组一张）

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师宣布规则："我们要玩一个传话游戏。每组第一排的同学会拿到一句话，只能用耳语传到最后。传的过程中不能重复、不能写字。这就是'字符串的传递'。" |
| 1-2 分 | 5-6 人一组。老师给每组第一排同学发一张写着同一句话的纸条（如"今天下午三点在操场集合跑步"），只给第一排看 10 秒后收回 |
| 2-3 分 | 第一排同学耳语传给第二排，依次传到最后一排。最后一排同学把听到的写在纸上 |
| 3-5 分 | 各组最后一排同时亮出纸条。对比原句——通常会出现"字符串丢失/损坏"（信息走样） |
| 5-7 分 | 老师引导讨论："为什么信息会走样？"——引出**精确存储和传递的重要性**。"程序里的字符串不会自动丢失，除非你写错了代码——但 `cin>>s` 遇空格就停，相当于传话传到一半突然被打断" |
| 7-9 分 | 第二轮游戏：老师"升级"——用书面传递（把纸条从第一排传到最后一排，每人看 5 秒）。对比两轮结果："书面传递 = 精确无误 → 这就是变量赋值。耳语 = 容易出错 → 这就是没用好工具" |
| 9-11 分 | 再升级——老师："现在我在第一排的纸条上写两句话，中间有空格——'Hello World'。请一位同学用 `cin` 读（只看第一个词）传给下一位。然后再用 `getline` 读（看整句话）。" |
| 11-12 分 | 总结："string 像一个纸条——可以存一个词，也可以存一句话。关键是你用 `cin`（只读到空格）还是 `getline`（读到换行）。工具选对了，信息才不会丢。" |

### 活动 2：字符串拼图——拼接实战（第 2 课时用，5 分钟，可作热身）

**道具**：两张 A4 纸，分别写上 "Hello" 和 " World"

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 请两位同学分别举着"Hello"和" World"站在讲台两侧 |
| 1-2 分 | 老师："C++ 里用 + 号把两个字符串粘在一起。来，你们两个——'加'在一起！"两位同学走到一起，把纸拼成 "Hello World" |
| 2-3 分 | 老师："`"Hello".length()` 是多少？"让学生数——5。"那拼接之后的 `c.length()` 呢？"——11（5+1空格+5=11，" " 也是一个字符） |
| 3-4 分 | 老师："`c[0]` 是什么？"→H。"`c[5]` 是什么？"→空格。让学生理解空格也是一个下标位置 |
| 4-5 分 | 总结："字符串拼接就是把小纸条拼成大纸条。每个字符（包括空格）都有下标，都可以用 `[i]` 访问" |

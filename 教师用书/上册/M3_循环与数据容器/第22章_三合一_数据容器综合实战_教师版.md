## 🗺️ 教学流程

### 第 1 课时：知识回顾 + 三容器对比（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "M3 模块到现在，我们学了三种'容器'来装数据：一维数组、二维数组、string。今天我们不学新东西——我们把旧知识揉在一起，看看它们能组合出什么威力。" |
| 5-15 分 | 思维导图 | 师生共同在黑板上画出本模块知识树：一维数组（存取/遍历/统计/变换/查找）、二维数组（行列遍历/求和/转置/对角线）、string（输入/拼接/长度/索引/五大函数）、ASCII（编码/判断/转换/凯撒密码） |
| 15-25 分 | 讲授 | **三容器对比表**。板书一个 4 列的表格——| 容器 | 存什么 | 怎么访问 | 最强操作 |。逐行填写：一维数组=同类型批量数字，a[i]索引，打擂法/排序；二维数组=表格数据，a[i][j]行列，矩阵变换；string=文字，s[i]索引+s.find()，查找替换 |
| 25-35 分 | 不插电 | **"编程挑战赛——数据流图"**（详见活动1）。分组为综合题设计数据流方案 |
| 35-42 分 | 讨论 | 每组代表用 1 分钟汇报自己的数据流方案。全班投票最佳方案。老师点评各组优劣 |
| 42-45 分 | 总结 | "没有最好的容器，只有最适合的容器。选择容器之前，先想三个问题：存什么类型？要查找吗？要修改吗？" |

### 第 2 课时：混合综合题训练（一）（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-3 分 | 热身 | 描述问题："读入 n 个英文单词，按字典序排序输出。"——提问学生用什么容器、什么思路 |
| 3-15 分 | 讲授+演示 | **字符串数组**。`string words[100];`——这就是"一维数组 + string"。遍历时 `cin>>words[i]`，输出时 `cout<<words[i]`。讲解字典序比较：string 可以直接用 `>` `<` 比较——C++ 真方便。演示冒泡排序字符串数组（用交换法，此处不深入排序算法） |
| 15-25 分 | 上机 | 练习 1：读入 n（n≤50）个英文单词（纯小写），按字典序从小到大排序输出 |
| 25-40 分 | 讲授+上机 | **统计型综合题**。题目："读入 n（n≤100）个同学的姓名（英文）和 3 科成绩。输出每个同学的总分、平均分，以及全班每科的平均分。"引导学生分析：姓名用 `string names[100]`，成绩用 `int scores[100][3]`——这是一维 string 数组 + 二维 int 数组的配合 |
| 40-45 分 | 讲评 | 抽查代码，强调"两个容器配合使用时下标必须对齐"——`names[i]` 和 `scores[i][j]` 中的 i 是同一个学生 |

### 第 3 课时：混合综合题训练（二）（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速复习上节课的"姓名+成绩"模式 |
| 5-15 分 | 分组讨论 | 给出题目："读入一段英文（不超过 500 个字符），统计每个单词出现的次数，按字典序输出单词及其出现次数。"4 人一组讨论方案 5 分钟。提示：需要"存单词"+"统计频次" |
| 15-20 分 | 各组汇报 | 各组讲思路。老师引导出方案：`string words[500]` 存单词，`int cnt[500]` 统计频次。读入→如果在 words 中找到→cnt++；如果没找到→加到 words 末尾，cnt=1 |
| 20-35 分 | 独立编程 | 学生上机实现上述方案。老师巡视，帮助遇到困难的学生 |
| 35-42 分 | 讲评 | 挑一份正确代码投影讲解，重点讲解"查找单词是否已存在"的线性查找逻辑（又回到了 Ch17 的线性查找——知识是相通的） |
| 42-45 分 | 总结 | "这个题用了一维数组（存单词）+ 线性查找（找是否出现过）+ 计数统计（cnt）。三道题的技能都派上用场了——这就是综合。" |

### 第 4 课时：项目实战——成绩排名系统（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 发布项目 | 展示完整需求："成绩排名系统。输入：n 个同学的姓名（英文，无空格）和语文、数学、英语三科成绩。输出：① 按总分从高到低排名（并列时按姓名字典序）；② 每科最高分和获得者姓名；③ 每科及格率（≥60 分为及格，保留一位小数）。" |
| 5-10 分 | 需求分析 | 老师带学生分解需求：需求①=排序（总分排序+字典序作为第二关键字）；需求②=打擂法找最高分（每科遍历一次，记录值和姓名下标）；需求③=计数统计÷总人数×100 |
| 10-35 分 | 独立编程 | 学生独立实现。由于工作量较大，老师明确告知"完成前两项即算及格，三项全完成为优秀"。鼓励学生分步骤：先存数据→再算总分→再排名→最后统计 |
| 35-42 分 | 代码互评 | 同桌交换看代码。评价标准：① 能否编译通过？② 测试数据能否正确输出？③ 代码有无注释？④ 变量命名是否清晰？ |
| 42-45 分 | 颁奖 | 表扬完成全部三项的学生，展示优秀代码片段。强调："这个系统再完善一下，就是学校教务系统里'成绩分析'模块的雏形" |

| 8-20 分 | 小测 | 闭卷 12 分钟，侧重"选择合适容器"的判断能力 |

## 🔥 重点难点

### 教学重点

1. **容器选择决策**：面对一个问题，能快速判断应该用一维数组、二维数组、string 还是组合使用。
2. **数据流设计**：先想清楚"输入什么→存在哪→怎么处理→输出什么"再动手写代码。
3. **下标对齐意识**：多个数组配合时，同一个下标必须指向同一条数据。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 不知道该用哪个容器 | 面对问题呆住，无从下手 | 教三个筛选问题：① 数据是同类型的多个数字吗？→数组 ② 数据是表格吗？→二维数组 ③ 数据是文字/名字/句子吗？→string。答完这仨基本就有方向了 |
| 多个容器下标对不齐 | 输出时张冠李戴（张三的成绩配给李四） | 演示"同步操作"原则——所有的读入、处理、输出都放在同一个 for 循环里，i 从头走到尾，保证每一步的 i 指向同一个人 |
| 综合题代码太长，写到一半放弃 | 思路混乱，不知道写到哪了 | 教"分步攻破"法：先把代码框架写好（#include、main、return 0），然后按 1-2-3-4 每个步骤写 3-5 行，写完一段就测试一段，别等全写完再跑 |

## 💬 常见学生问题

**Q1："我们同学了四种数据类型——int、double、char、string。到底什么时候用哪个？"**

A：记口诀——"**算数用 int，小数用 double，一个字母用 char，一串文字用 string**。"选错类型的代价：该用 double 用了 int → 结果不准确；该用 string 用了 char → 只能存一个字符。

**Q2："数组 + string 的组合（`string arr[100]`）和二维字符数组（`char arr[100][50]`）有什么区别？"**

A：`string arr[100]` 更好用——每个元素是一个完整的 string，可以自动管理长度，可以用 `+` 拼接，用 `==` 比较，用 `.length()` 获取长度。`char arr[100][50]` 是 C 语言的旧风格，功能简陋。信奥入门阶段，凡是文字类型统统用 string。

**Q3："综合题这么长，考试时做得完吗？"**

A：考试的综合题通常是"分解给分"——你做完第一步就有第一步的分。即使没全做完，只要核心思路对、代码结构清晰，就能拿到一大半的分数。所以关键是：**先搭框架，再做细节；先保证正确，再追求完整**。

**Q4："M3 学完了，我感觉自己还是不会写大程序。正常吗？"**

A：非常正常！M3 是在给你"工具箱"——你学会了锤子、螺丝刀、扳手。但要真正造出东西来，还需要大量的练习。编程就是这样：**学 30%，练 70%**。后面 M4 函数会让你写程序的能力再上一个台阶——函数能把长代码拆成小块，到时候你会发现"原来大程序就是很多个小块拼起来的"。

## 📋 本章小测

1. **填空题**：如果要存储全班 40 个人 3 次考试的成绩，最合适的数据容器是 _____________。

2. **选择题**：以下哪个问题最适合用 string 来解决？  
A. 计算 100 个数的平均值 B. 判断一个句子中是否有 "C++" C. 统计一个班的身高中位数 D. 输出九九乘法表

3. **判断题**：一个程序只能使用一种数据容器。（ ）

4. **简答题**：描述"成绩排名系统"中，如何利用一维 string 数组和二维 int 数组配合完成数据存储。为什么下标要对齐？

5. **填空题**：`string names[50]; int scores[50][3];` 中，要输出第 k 个学生的姓名和数学成绩（第 2 科），代码是：  
`cout << ________ << " " << ________;`

6. **编程题**：读入 n 个英文名字（n≤30），找出字典序最大和最小的名字并输出。

7. **设计题**：某班级要选班长，每个同学投票给一个候选人（共 5 个候选人，编号 0~4）。请设计数据存储方案并描述处理流程。（只需方案，不需完整代码）

---

### 参考答案

1. 二维 int 数组（`int score[40][3];`）或 int 数组+string 数组组合
2. B（判断是否包含 "C++" 是字符串查找问题）
3. ×（可以在同一程序中使用多种容器，如同时用数组存成绩、string 存名字）
4. names[i] 存第 i 个学生的姓名，scores[i][j] 存第 i 个学生的第 j 科成绩。同一个 i 指向同一个人，如果下标不对齐就会出现"张三的成绩配给了李四"
5. `names[k]`，`scores[k][1]`（第 2 科下标是 1）
6. 思路：`string maxName = names[0], minName = names[0];` 遍历比较，`if(names[i] > maxName) maxName = names[i];`（字典序比较），`if(names[i] < minName) minName = names[i];`
7. 方案：`int votes[5] = {};` 用一维数组存 5 个候选人的票数。处理流程：遍历 n 张投票→每张票的编号作为下标 `votes[编号]++`→统计完成后，用打擂法找 votes 数组中的最大值及其下标→输出得票最多的候选人编号和票数

## 🎮 不插电活动

### 活动 1：编程挑战赛——数据流图设计（第 1 课时用，15 分钟）

**道具**：黑板/大白纸、彩色粉笔/马克笔（每组不同颜色）

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师宣布规则："每组抽一道综合题。不能用电脑——在黑板上画出解决方案的数据流图。用方框表示数据容器，用箭头表示数据流动，用圆圈表示处理步骤。10 分钟后各组汇报。" |
| 1-2 分 | 6 组抽题（每组一道），题目示例：① 成绩排名系统（姓名+3 科成绩）；② 班级通讯录（姓名+电话+QQ）；③ 英文单词词频统计；④ 简易加密聊天（输入明文+偏移量→输出密文）；⑤ 考试分数段统计（0-59/60-79/80-89/90-100 各段人数）；⑥ 座位表调整（按身高重新排座位） |
| 2-4 分 | 各组在分配给自己的黑板区域或大白纸上画图。老师巡视，提醒关键点："先想用什么容器装数据！" |
| 4-10 分 | 各组继续完善方案，老师提示："数据流图要有起点（输入）和终点（输出）。中间每一步处理要写清楚在干什么"。学有余力的组可以标注伪代码 |
| 10-12 分 | 每组派代表用 1 分钟汇报。汇报模板："我们的问题是……，我们选择用……来存数据。处理分 X 步：第一步……，第二步……，最后输出……" |
| 12-14 分 | 全班投票：最佳方案（1 票）、最有创意方案（1 票）。老师点评各组——重点分析容器选型是否合理、流程是否完整 |
| 14-15 分 | 总结："画数据流图 = 在脑子里把程序跑了一遍。高手写代码之前都先画图。这个习惯从今天开始培养。" |

### 活动 2：代码接力赛（第 4 课时热身或穿插使用，8 分钟）

**道具**：黑板、粉笔

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师出题："用最快的速度在黑板上写出代码框架——统计 10 个同学的姓名和成绩，输出平均分最高的人。" |
| 1-2 分 | 全班分成 4 组。每组派一人跑到黑板前，只能写一行代码，然后回来把粉笔交给下一个组员。下一个组员接着写下一行 |
| 2-6 分 | 接力进行，直到代码基本完成。同组同学可以在下面提示（但不能冲上去帮忙写） |
| 6-7 分 | 老师逐组检查代码，指出问题（变量未声明、缺少头文件、逻辑错误等） |
| 7-8 分 | 宣布获胜组。总结："团队写代码 = 每个人的代码都能被队友看懂。所以注释重要、命名规范重要、代码整洁重要——不然队友根本接不上你的代码。" |

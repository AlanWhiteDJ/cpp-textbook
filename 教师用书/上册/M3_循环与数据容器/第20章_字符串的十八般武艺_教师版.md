## 🗺️ 教学流程

### 第 1 课时：find——大海捞针（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "上节课我们学会了把字符串存起来、拼起来、数长度。但这些只是'搬运工'的活。今天开始学真正的'手艺'——在字符串里找东西、剪东西、塞东西。" |
| 5-15 分 | 讲授 | **`find` 函数**。`s.find("abc")`——在 s 里找 "abc"，返回第一次出现的位置（下标）。如果没找到，返回 `string::npos`。板书：`int pos = s.find("world"); if(pos != string::npos) cout << "在位置 " << pos; else cout << "没找到";` |
| 15-22 分 | 演示 | 演示 find 的几个用法：①找子串；②找单个字符 `s.find('a')`；③从指定位置开始找 `s.find("abc", 5)`——从下标 5 开始往后找 |
| 22-30 分 | 不插电 | **"文字手术台"——find 篇**（详见活动1步骤1-3）。用纸条和笔模拟 find 操作 |
| 30-38 分 | 上机 | 练习 1：读入一个长字符串和一个短字符串，判断短字符串是否在长字符串中出现过。练习 2：如果出现过，输出它出现了几次（提示：用循环 + find 的第二个参数） |
| 38-42 分 | 讲评 | 重点讲解统计出现次数时的循环写法：`while((pos = s.find(t, pos)) != string::npos) { cnt++; pos += t.length(); }` |
| 42-45 分 | 总结 | "find 是'找'，npos 是'没找到'。两个搭档构成了所有查找操作的底层" |

### 第 2 课时：substr——抽丝剥茧（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速复习 find 的返回值 |
| 5-18 分 | 讲授 | **`substr` 函数**。"substr = substring = 子串。从大字符串里'切'一小段出来。"两个参数：`s.substr(起始位置, 长度)`。板书：`string s = "Hello World"; string sub = s.substr(6, 5); // "World"`。演示省略第二个参数→从起始位置切到结尾 |
| 18-28 分 | 上机 | 练习：读入一个字符串，输出它的第 2 个到第 5 个字符（即 s[1] 到 s[4]）——答案：`s.substr(1, 4)` |
| 28-38 分 | 讲授+上机 | **find + substr 组合技**。"先找到位置，再切出来"。例：读入一个 Email 地址（如 "zhangsan@school.com"），提取用户名（@之前的部分）和域名（@之后的部分）。引导：先用 `find('@')` 找到位置，再分别用 substr |
| 38-43 分 | 讨论 | 提问："如果 substr 的起始位置 + 长度超过了字符串长度会怎样？"——不会崩溃，C++ 会自动截断到末尾。这是安全设计，但也要注意逻辑上是否正确 |
| 43-45 分 | 总结 | "find 是眼睛，substr 是剪刀。先看到，再剪下来" |

## 🔥 重点难点

### 教学重点

1. **五大函数的参数含义**：find 返回位置、substr 要位置+长度、erase 要位置+长度、insert 要位置+内容、replace 要位置+长度+新内容。
2. **find + npos 判断模式**：这是字符串处理中最常用的"找到/没找到"判断。
3. **函数组合使用**：`find` 找到位置 + `substr/erase/insert/replace` 执行操作。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 混淆各函数的参数顺序 | erase 位置和长度谁先谁后记不清 | 统一口诀："**位置永远是第一个参数**"——find、substr、erase、insert、replace 的第一个参数都是位置 |
| string::npos 不理解 | 写成 `-1` 或 `0` | 说明 npos 是一个特殊的"大数"，代表"没找到"。不需要知道具体值，永远写成 `string::npos` 就好。类比："npos 就是'查无此人'" |
| replace 和 erase+insert 的选择 | 不知道该用哪个 | replace 更简洁但参数多。初学建议先用 erase+insert 分两步写，逻辑更清晰。熟练后再用 replace 一步搞定 |

## 💬 常见学生问题

**Q1："`find` 找不到时返回 -1 吗？为什么一定要写 `string::npos`？"**

A：`string::npos` 实际是一个很大的无符号数（不是 -1），直接和 -1 比较可能在类型转换时出问题。为了代码的可移植性（换一个编译器也能正常跑），养成写 `string::npos` 的习惯。

**Q2："substr 切出来的字符串，修改它会影响原来的字符串吗？"**

A：不会。substr 返回一个**全新的字符串**（副本），修改副本不影响原件。就像你把一个文件复制了一份，改复印件不会影响原件。如果想让改动同步，需要用引用——那是后面章节的内容。

**Q3："`erase` 之后原来的下标会不会乱掉？"**

A：会的！删除之后后面的字符会向前移动，原来的位置关系就被打乱了。所以在循环里做 erase 操作要特别小心——通常应该**从后往前**删除，或者用 while 循环每次重新 find。这是字符串处理最容易出 bug 的地方。

**Q4："这五个函数一定要背下来吗？忘了怎么办？"**

A：不需要死记硬背！忘了随时查——Google 搜索"c++ string erase"或者查看 C++ Reference 网站。职业程序员也是边查边写的，关键是理解每个函数能干什么。考试时如果不让查，建议重点记住 find 和 substr 的参数（最常用）。

## 📋 本章小测

1. **填空题**：`string s = "Hello World"; int pos = s.find("World");` 执行后 pos 的值是 ____。

2. **选择题**：以下哪个函数调用会返回 "World"？  
A. `s.substr(6, 5)` B. `s.substr(5, 6)` C. `s.substr(0, 5)` D. `s.substr(6, 4)`

3. **填空题**：判断字符串 s 中是否包含子串 "abc" 的代码：`if(s.find("abc") != ______)`

4. **编程题**：读入一个字符串，删掉其中所有的数字字符（'0'~'9'），输出处理后的字符串。

5. **代码阅读**：以下代码的输出是什么？
```cpp
string s = "abcdefg";
s.erase(2, 3);
s.insert(2, "XYZ");
cout << s;
```

6. **改错题**：以下代码想统计 "ab" 在 s 中出现的次数，有什么问题？
```cpp
int cnt = 0, pos = 0;
while(pos != string::npos) {
    pos = s.find("ab");
    cnt++;
}
```

7. **简答题**：描述 replace 函数和"先 erase 再 insert"的关系。

---

### 参考答案

1. 6（从下标 6 开始是 'W'）
2. A（s.substr(6,5)：从下标 6 开始取 5 个字符 = "World"）
3. `string::npos`
4. 思路：遍历字符串，如果是非数字字符就拼到新字符串中；或用 erase 从后往前删除数字字符
5. `abXYZfg`（先 erase(2,3) 把 "cde" 删掉变成 "abfg"，再 insert(2,"XYZ") 在位置 2 插入变成 "abXYZfg"）
6. ① `pos` 应在循环内更新但用了 `find("ab")` 没有指定起始位置，每次都是从头找，导致死循环；② `cnt++` 在没找到时也会执行。正确写法：`while((pos=s.find("ab",pos))!=string::npos){ cnt++; pos+=2; }`
7. `s.replace(pos, len, newStr)` 等价于 `s.erase(pos, len); s.insert(pos, newStr);`。replace 是这两个操作的组合，一步完成删除+插入

| 13-15 分 | 总结："剪刀=erase/substr，胶水=insert，覆盖贴=replace。C++ 的字符串操作跟你刚才做的完全一样——只是程序不需要胶水和剪刀，几行代码就能搞定。" |

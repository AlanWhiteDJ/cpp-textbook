## 🗺️ 教学流程

### 第 1 课时：为什么需要数组 + 数组概念（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | **问题驱动**："班上有 40 个同学，每人一个数学成绩。如果只用变量，你要定义多少变量？"学生脱口而出"40 个"。追问："那全校 2000 人呢？"让学生感受到变量的局限 |
| 8-18 分 | 讲授 | 引出数组概念。**核心类比**：一排柜子。每个柜子编号（下标），里面放东西（元素）。柜子从 0 号开始编号。板书图示：`a[0]│a[1]│a[2]│a[3]│a[4]`，下标写在下方 |
| 18-30 分 | 不插电 | **"储物柜游戏"**（详见活动1）。让抽象概念变成身体感受 |
| 30-38 分 | 讲授 | 数组三要素：①同一类型（柜子只能放试卷，不能混放水瓶）；②连续存放（柜子排成一排）；③下标从 0 开始（第 1 个柜子编号是 0）。配合板书反复强调第 3 条 |
| 38-42 分 | 互动 | 快速问答："数组第 5 个元素的下标是？"（4）。"`a[10]`是第几个元素？"（第 11 个）。反复操练到全班都能立刻反应 |
| 42-45 分 | 总结 | 口诀："**一排柜子一个名，编号从零开始数**" |

### 第 2 课时：数组声明与访问 + 遍历（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速问答复习下标从 0 开始 |
| 5-15 分 | 讲授 | 数组声明：`int a[5];`——"请计算机给我留一排 5 个格子的 int 柜子"。强调：中括号里的数字是大小，声明后不能改 |
| 15-25 分 | 演示+上机 | 老师演示 `cin >> a[0]` 读入第一个元素，`cout << a[3]` 输出第四个元素。学生跟着敲 |
| 25-35 分 | 讲授 | **数组 + for 循环 = 最佳搭档**。板书：`for(int i=0; i<5; i++) cin>>a[i];` 对比"一个一个写 5 行 cin"的笨办法，突出循环的优雅 |
| 35-42 分 | 上机 | 学生练习：定义长度 5 的 int 数组，用循环读入 5 个数，再用循环倒序输出 |
| 42-45 分 | 讲评 | 抽查学生代码，纠正常见错误：`i<=5`（越界）、`i`从 1 开始（跳过第一个元素） |

### 第 3 课时：数组初始化 + 越界问题（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | 提问："定义一个数组 `int a[5]` 之后，里面的值是什么？"引发好奇 |
| 5-15 分 | 讲授 | 初始化三种方式：①全部置 0 → `int a[5] = {};` ②指定值 → `int a[5] = {1,2,3,4,5};` ③部分指定 → `int a[5] = {1,2};` 后面自动补 0。逐条演示 |
| 15-25 分 | 上机 | 学生验证：定义 `int a[5];` 不初始化，用循环输出看看里面是什么（垃圾值），再定义 `int a[5] = {};` 对比 |
| 25-35 分 | 讲授 | **越界问题——最危险的 bug**。演示：`int a[5]; a[5] = 100;` 访问了不存在的第 6 个格子。比喻："你只有 5 个柜子，硬要往第 6 个柜子里塞东西——塞进了别人的柜子" |
| 35-42 分 | 纠错 | 给出典型越界代码让学生找错（如循环写 `i<=n`、下标写 `a[n]` 访问第 n 个元素），分组讨论后汇报 |
| 42-45 分 | 总结 | "数组越界 = 翻墙进别人家。C++ 不会拦你，但后果很严重" |

## 🔥 重点难点

### 教学重点

1. **下标从 0 开始**：这是学生最容易出错的点，需要持续强化。
2. **数组 + for 循环的遍历模式**：`for(int i=0; i<n; i++)` 是后续所有操作的基础模板。
3. **数组越界的危险**：让学生形成"访问前检查下标范围"的习惯。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 搞混下标和元素值 | 问"数组第 k 个元素"时回答 `a[k]` | 用储物柜实物演示：柜子编号（下标）≠ 柜子里装的东西（元素值）。指着 3 号柜子问"这个柜子里面放了什么？"——"东西"和"3 号"是不同的概念 |
| 循环条件写成 `i<=n` | 运行结果不对或崩溃 | 让学生数：n=5 时，i 从 0 到 4 一共 5 个数；i<=5 会多出一个 i=5，访问 a[5] 越界。口诀："n 个元素，i 从 0 到 n-1" |
| 不知道数组大小怎么定 | 随意写一个很大的数 | 讲解"数组大小必须在编译时确定，运行时不能变"的规则（先不涉及 vector）。建议：题目给的数据范围 + 5 作为安全余量 |

## 💬 常见学生问题

**Q1："为什么下标从 0 开始，不能从 1 开始吗？"**

A：这是历史原因——C 语言的设计者用"偏移量"来计算地址。数组名指向第一个元素的地址，`a[0]` 就是偏移 0 个位置，`a[1]` 偏移 1 个位置……这样计算最快。虽然一开始别扭，但全世界的 C/C++ 程序员都这么用，习惯就好。等你学到指针那一章，就明白"从 0 开始"有多巧妙了。

**Q2："`int a[100]` 和 `int a[50]` 哪个好？用大一点有没有坏处？"**

A：定义的大小就是向计算机申请的内存空间。`int a[100]` 占 400 字节（4×100），`int a[50]` 占 200 字节。多申请不会让程序变快，只是多占了一点内存。在竞赛里，一般按题目说的数据范围上限 + 10 来定义。但不要写 `int a[999999999]`——可能会超过内存限制导致编译失败。

**Q3："数组只能存数字吗？能不能存字母？"**

A：可以存任何单一类型！`char a[10]` 存字符，`double a[10]` 存小数，`string a[10]` 存字符串。但一个数组只能存一种类型的数据——规则是"一个柜子放一种东西"。我们后面会学到各种类型的数组。

**Q4："我怎么知道该用多大的数组？"**

A：看题目给的数据范围。比如题目说"n ≤ 1000"，你就定义 `int a[1005]`（多 5 个保险）。如果题目没说范围，问老师或根据常识估一个。初学者最容易犯的错误是数组开太小导致越界，开太大倒不太要紧。

## 📋 本章小测

1. **填空题**：`int a[8];` 定义了一个可以存放 ____ 个整数的数组，最后一个元素的下标是 ____。

2. **判断题**：`int a[5] = {1,2,3,4,5,6};` 是正确的定义。（ ）

3. **选择题**：以下哪个是访问数组 a 的第 3 个元素？  
A. `a[2]`　B. `a[3]`　C. `a[4]`　D. `a[0]`

4. **改错题**：以下代码有什么问题？
```cpp
int a[10];
for(int i=0; i<=10; i++)
    cin >> a[i];
```

5. **编程题**：读入 5 个整数存入数组，输出它们的和。

6. **判断题**：数组越界时，编译器一定会报错。（ ）

7. **填空题**：`int a[5] = {1,2};` 执行后，`a[3]` 的值是 ____。

8. **简答题**：数组和普通变量（如 `int x;`）相比有什么优势？至少说出两点。

---

### 参考答案

1. 8 个，7
2. ×（初始化数量超过了数组大小）
3. A（下标从 0 开始，第 3 个元素下标是 2）
4. 循环条件 `i<=10` 会导致访问 `a[10]`，但数组下标只有 0~9，造成越界。应改为 `i<10`
5. 略（思路：定义数组 → for 循环读入并累加 → 输出 sum）
6. ×（编译器不检查越界，运行时可能崩溃或产生奇怪结果）
7. 0（部分初始化时，未指定的元素自动设为 0）
8. ①数组可以存多个数据，变量只能存一个；②数组可以和循环配合实现批量处理；③数组元素通过下标访问，方便按位置操作

## 🎮 不插电活动

### 活动 1：储物柜游戏（第 1 课时用，12 分钟）

**道具**：粉笔（在地上画格子）、编号卡片（0~9）

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师在教室空地用粉笔画一排 10 个格子，每个格子里写编号 0~9 |
| 1-2 分 | 选 10 名同学，每人发一张编号卡，站进对应编号的格子 |
| 2-4 分 | 老师发指令："请 3 号同学举手！"——学生确认下标和位置的关系。再问："5 号格子里是谁？"——区分下标（格子编号）和元素值（格子里是谁） |
| 4-6 分 | 升级："请下标为 i 的同学向前一步"（不喊具体数字，让学生理解下标是变量）。老师变换 i 的值：`i=0`→0 号出来，`i=7`→7 号出来 |
| 6-8 分 | 模拟遍历："现在我要按顺序看每个格子里的人"——老师从 0 号格子走到 9 号格子，边走过喊"a[0]是张三，a[1]是李四……" |
| 8-10 分 | 模拟赋值："张三，请你站到 2 号格子里去"——让学生体会 `a[2] = 张三` 就是把张三"放进"2 号柜子 |
| 10-12 分 | 总结：回到座位，老师总结"格子编号=下标，格子里的人=元素值，从 0 到 9 一共 10 个" |

**教学要点**：活动过程中反复问"第 k 个格子的下标是多少？"让全班齐声回答，形成条件反射。

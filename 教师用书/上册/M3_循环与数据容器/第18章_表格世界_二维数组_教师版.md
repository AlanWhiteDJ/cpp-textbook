## 🗺️ 教学流程

### 第 1 课时：为什么需要二维数组 + 基本概念（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | **问题驱动**："我要存全班 40 个人的 5 科成绩。用一维数组？`int s1[40],s2[40],s3[40],s4[40],s5[40];`——5 个数组，太麻烦了。有没有更好的办法？"引出二维数组 |
| 8-20 分 | 讲授 | 二维数组概念。**核心类比**：Excel 表格。`int score[40][5];` = 40 行 × 5 列的表格。第一个下标是行号（第几个人），第二个下标是列号（第几科成绩）。板书画一个 3 行×4 列的格子表并标注下标 |
| 20-30 分 | 不插电 | **"座位表寻宝"**（详见活动1）——用教室座位让学生直观感受行列下标 |
| 30-38 分 | 讲授 | 二维数组内存模型——按行存储。虽然看起来是表格，内存里其实是一长条：第 0 行全放完 → 接第 1 行 → 接第 2 行。这个知识点对理解指针和地址有帮助 |
| 38-42 分 | 互动 | 快速问答："`a[2][3]` 是第几行第几列？"（第 2 行第 3 列，下标从 0 算起其实是第 3 行第 4 列）。"5 行 6 列的数组，最后一个元素是？"（`a[4][5]`） |
| 42-45 分 | 总结 | "**行在前，列在后。行列都从 0 数。**" |

### 第 2 课时：遍历与输入输出（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速问答：二维数组下标规则 |
| 5-15 分 | 讲授 | **双层循环遍历**——"外层走行、内层走列"。板书标准模板：`for(i=0;i<n;i++) for(j=0;j<m;j++) cin>>a[i][j];`。强调：i 管行、j 管列，外 i 内 j |
| 15-25 分 | 演示+上机 | 老师演示读入一个 3×4 矩阵并"原样打印"（行列式输出，每行结束后换行）。学生跟着敲 |
| 25-32 分 | 讲授 | **按行输出 vs 按列输出**。两种遍历在循环顺序上的区别：按行→外 i 内 j；按列→外 j 内 i。演示"把成绩表竖着打印出来" |
| 32-40 分 | 上机 | 练习：读入一个 n×m 矩阵（n,m≤10），先按行输出，再按列输出 |
| 40-45 分 | 讲评 | 纠正常见错误：行列下标写反、输出格式忘记换行 |

### 第 3 课时：行列求和 + 矩阵转置（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "我们班 5 行×4 列的成绩表，怎么算每个同学的总分？怎么算每科的平均分？" |
| 5-18 分 | 讲授+演示 | **行求和**："每行求和 = 固定行号 i，遍历所有列 j"。板书画图：一个虚线框横跨一整行的数据，框内全部相加。**列求和**同理——固定列号 j，遍历所有行 i |
| 18-28 分 | 上机 | 读入 3×4 成绩表，输出每个学生的总分（行求和）、每科的平均分（列求和÷3） |
| 28-38 分 | 讲授 | **矩阵转置**——"行变列、列变行"。`b[j][i] = a[i][j]`。板书对比图示：左边是原矩阵 3×4，右边是转置后 4×3 |
| 38-43 分 | 上机 | 读入一个 n×m 矩阵，输出它的转置矩阵 |
| 43-45 分 | 总结 | 行列操作的通用心法："操作行→固定外循环的 i；操作列→固定内循环的 j" |

### 第 4 课时：综合应用（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 热身 | 快速判断："`a[0][3]`和`a[3][0]`是同一个位置吗？"（不是！行和列不同） |
| 5-15 分 | 讲授 | **对角线元素**。主对角线：`i==j` 的位置。副对角线：`i+j == n-1` 的位置。板书用不同颜色标出两种对角线的格子 |
| 15-25 分 | 上机 | 练习：读入 n×n 方阵，输出主对角线元素之和 |
| 25-35 分 | 拔高 | 挑战题："杨辉三角"——用二维数组生成前 10 行杨辉三角。引导思路：`a[i][0]=1; a[i][i]=1; a[i][j]=a[i-1][j-1]+a[i-1][j]` |
| 35-45 分 | 讲评 | 逐行讲解杨辉三角的规律和代码对应关系，重点理解 `a[i][j]` 依赖上一行的 `a[i-1][j-1]` 和 `a[i-1][j]` |

## 🔥 重点难点

### 教学重点

1. **双层循环的遍历模板**：`for(i=0;i<n;i++) for(j=0;j<m;j++)` 是二维数组的骨架。
2. **行列下标不混淆**：反复强调第一个下标是行、第二个是列。
3. **转置的循环写法**：理解为什么转置需要一个新的数组来存结果。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 行列下标搞混 | `a[列][行]` 乱写 | 口诀："**行(line)在前，列(column)在后**"。每次写二维数组下标前默念。实在记不住就想"先有行，列才挂在行下面" |
| 按列遍历时循环不知道怎么写 | 想也不想就写外 i 内 j | 动手画：要读一列数据，必须先定住列号 j，然后从上到下（i=0,1,2…）。所以外循环是 j，内循环才是 i |
| 转置想原地完成 | 代码写 `a[i][j]=a[j][i]` | 画 2×3 矩阵演示：如果原地交换会把数据覆盖掉。除非是方阵，否则必须另用一个数组 |

## 💬 常见学生问题

**Q1："二维数组和一维数组有什么关系？"**

A：二维数组就是"一维数组的数组"。`int a[3][4]` 可以看作 3 个长度为 4 的一维数组摞在一起。更具体地说，`a[0]` 本身就是一个一维数组（第 0 行）。等你以后学指针和 vector，这个理解会非常有帮助。

**Q2："三维数组存在吗？有什么用？"**

A：存在！`int a[10][20][30]` 就是三维数组。三维数组可以类比"魔方"——每个小块需要三个坐标确定位置。实际应用比如：CT 扫描的三维图像数据、多个表格按时间排列（时间×行×列）。不过信奥普及组阶段很少用三维，二维已经能解决绝大多数表格问题。

**Q3："遍历时，可不可以先列后行？"**

A：完全可以！看你要干什么。如果你要按列处理数据（比如计算每列的平均分），就应该固定列号 j，在里面遍历行号 i。循环的先后顺序取决于你的需求，没有"必须外行内列"的绝对规则——只是大部分情况按行操作更方便。

**Q4："`int a[100][100]` 会不会太大？"**

A：`100×100×4 字节 = 40000 字节 ≈ 40KB`，对现代计算机来说很小。一般信奥题目里二维数组不超过 `1000×1000`（约 4MB），都在安全范围内。但如果在函数内部定义极大的二维数组（如 `int a[10000][10000]`），可能爆栈，需要定义在全局。

## 📋 本章小测

1. **填空题**：`int a[5][6];` 一共可以存放 ____ 个整数，最后一个元素的下标是 `a[____][____]`。

2. **判断题**：二维数组在内存中是按列连续存储的。（ ）

3. **选择题**：以下哪个代码正确遍历了一个 n 行 m 列的二维数组？  
A. `for(j=0;j<m;j++) for(i=0;i<n;i++)`  
B. `for(i=0;i<n;i++) for(j=0;j<m;j++)`  
C. 以上两种都正确，取决于需求

4. **填空题**：输出 4×4 方阵主对角线元素的循环条件可以写成 `if(____)`。

5. **编程题**：读入一个 3×3 的矩阵，输出每一行的和。

6. **改错题**：以下代码想把矩阵转置输出，有什么问题？
```cpp
int a[3][4];
// ... 读入数据 ...
for(int i=0; i<3; i++)
    for(int j=0; j<4; j++)
        cout << a[j][i] << " ";
```

7. **简答题**：比较按行遍历和按列遍历在代码写法上的区别。

---

### 参考答案

1. 30 个（5×6），`a[4][5]`
2. ×（按行连续存储——第 0 行全放完才放第 1 行）
3. C（两种都可以，取决于要按行处理还是按列处理）
4. `i == j`
5. 略（思路：`for(i=0;i<3;i++){ sum=0; for(j=0;j<3;j++) sum+=a[i][j]; cout<<sum<<endl; }`）
6. ① `cout << a[j][i]` 时 j 的范围是 0~3，i 的范围是 0~2。当 j=3 时访问 `a[3][i]` 越界（原矩阵只有 3 行）。② 没有换行，全输出在一行里。解决：循环应改为外 j（列）内 i（行），且 j<4, i<3
7. 按行遍历：外循环 i（行号），内循环 j（列号）→ `for(i=0;i<n;i++) for(j=0;j<m;j++)`；按列遍历：外循环 j（列号），内循环 i（行号）→ `for(j=0;j<m;j++) for(i=0;i<n;i++)`

## 🎮 不插电活动

### 活动 1：座位表寻宝（第 1 课时用，10 分钟）

**道具**：粉笔、小奖品（糖果/贴纸）

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师宣布规则："教室座位就是我们的'二维数组'。第 0 排、第 1 排……从前往后；第 0 列、第 1 列……从左往右。不按平时叫名字，只喊行列号！" |
| 1-3 分 | 老师给每排每列重新编号——在过道上用粉笔标出行号 0,1,2,…，在最前面标出列号 0,1,2,… |
| 3-5 分 | 热身："请第 2 行第 3 列的同学站起来！"——找到的同学站起来。"请第 0 行第 0 列的同学！"——第一排第一个。反复几轮让学生适应 |
| 5-7 分 | 升级——"寻宝"："我在 `seat[3][4]` 的位置放了一张贴纸，谁能找到？"——学生用行列坐标定位，找到的同学获得奖品 |
| 7-9 分 | 再升级——"遍历全班"："现在我要'输出全班同学的名字'——从第 0 行开始，每行从左到右，请每位同学报出你的名字！"——模拟 `for(i=0;i<rows;i++) for(j=0;j<cols;j++) cout<<name[i][j]` |
| 9-10 分 | 总结："你们刚才体验的就是二维数组的访问和遍历。行号在前，列号在后。以后写代码时就想今天的座位！" |

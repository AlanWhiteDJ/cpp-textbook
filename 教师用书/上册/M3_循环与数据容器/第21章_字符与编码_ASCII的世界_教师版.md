## 🗺️ 教学流程

### 第 1 课时：什么是编码 + char 与整数（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | **问题驱动**："计算机只认识 0 和 1。那字母 'A' 是怎么存在计算机里的？"——每个字符背后有一个数字编号！这就是编码。就像每个人有身份证号，每个字符也有自己的编号 |
| 8-18 分 | 讲授 | **ASCII 码概念**。ASCII = American Standard Code for Information Interchange（美国信息交换标准代码）。用 0~127 共 128 个数字表示所有英文字母、数字、标点符号和控制字符。板书展示部分 ASCII 表（只列 A-Z、a-z、0-9） |
| 18-28 分 | 讲授 | **char 与 int 的转换**。演示：`char c = 'A'; cout << (int)c; // 65`。说明 `char` 其实就是一个 1 字节的小整数（范围 -128~127 或 0~255）。`'A'+1` = 66 = `'B'` |
| 28-38 分 | 不插电 | **"间谍密码本"**（详见活动1）——学生用 ASCII 表解码秘密信息 |
| 38-43 分 | 演示+上机 | 学生上机验证：输出 `'A'` 到 `'Z'` 的 ASCII 码值。`for(char c='A'; c<='Z'; c++) cout << c << " = " << (int)c << endl;` |
| 43-45 分 | 总结 | 三个关键数字：**'0'=48, 'A'=65, 'a'=97**。可以记不住全部，但这三个必须记住 |

### 第 2 课时：常用 ASCII + 字符判断（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速问答："'A' 的 ASCII 码是多少？'a' 呢？大小写字母之间差多少？"（65, 97, 32） |
| 5-15 分 | 讲授 | **常用字符判断模板**。板书：判断大写字母 `c>='A' && c<='Z'`；判断小写字母 `c>='a' && c<='z'`；判断数字 `c>='0' && c<='9'`；判断空格 `c==' '`。强调：这些判断之所以成立，是因为 ASCII 码把字母、数字连续排列 |
| 15-25 分 | 上机 | 练习 1：读入一个字符，判断它是大写字母、小写字母、数字还是其他。练习 2：读入一行字符串，统计其中字母、数字、空格的个数 |
| 25-35 分 | 讲授 | **ASCII 的巧妙应用**。`'5'` vs `5`：字符 '5' 的 ASCII 值是 53，不是数字 5。`char c='5'; int x = c - '0';`→ x = 5。口诀："**字符数字转真数字，减去 '0' 就搞定**" |
| 35-42 分 | 上机 | 练习：读入一个纯数字字符串（如 "12345"），把它转成整数（不直接用 stoi，用 ASCII 手动计算） |
| 42-45 分 | 总结 | 两个关键：判断字符类型用区间比较；字符数字转数值用减 '0' |

### 第 3 课时：大小写转换 + 字符统计（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "ASCII 表里，大写 A=65，小写 a=97，差 32。这意味着什么？"——引出大小写转换 |
| 5-18 分 | 讲授+演示 | **大小写转换**。`c = c - 'a' + 'A';`——把小写转大写；`c = c - 'A' + 'a';`——大写转小写。更推荐：`c = c - 32;`（小写→大写）和 `c = c + 32;`（大写→小写），但要先判断是否在正确范围内。演示 `toupper()` / `tolower()` 函数（`#include <cctype>`） |
| 18-28 分 | 上机 | 练习 1：读入一行英文，全部转成大写输出。练习 2：读入一行英文，首字母大写（其余小写）输出 |
| 28-38 分 | 讲授+上机 | **字符频率统计**。"统计一段英文中每个字母出现了多少次"。引导：用长度为 26 的数组 `int cnt[26]={};`，遇到字母 `c` 时 `cnt[c-'a']++`（或 `cnt[c-'A']++`）。演示这段代码 |
| 38-43 分 | 讨论 | 提问："如果英文里有大小写混合呢？"——统一转成大写（或小写）再统计。提问："最常用字母是什么？"——引出英文中最常用的字母是 'e'。让学生用代码验证 |
| 43-45 分 | 总结 | "大小写转换 = ±32。字符统计 = 数组下标 = 字符 - 'a'/'A'" |

### 第 4 课时：综合应用 + 密码学趣味题（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 热身 | 口头题："把 'H' 加密成它后面的第 3 个字母"→ 'K'（凯撒密码雏形） |
| 5-15 分 | 讲授 | **凯撒密码**。最简单的加密方法：每个字母向后移动固定位数。明文 "HELLO" → 偏移 3 → 密文 "KHOOR"。解密就是把密文向前移动。板书核心代码：`c = (c - 'A' + shift) % 26 + 'A';` |
| 15-30 分 | 分组挑战 | **"密码攻防战"**（详见活动2）：一半组加密，一半组解密。用凯撒密码给出一段英文，让另一组猜偏移量并破解 |
| 30-42 分 | 上机 | 综合编程：① 实现凯撒加密程序（读入明文和偏移量，输出密文）；② 实现暴力破解程序——已知密文但不知道偏移量，尝试 1~25 每种偏移，找出有意义的明文（最简单的判断：包含常见单词如 "the"、"and"） |
| 42-45 分 | 总结 | "ASCII 码不是枯燥的编码表——它是密码学、文本处理、数据压缩的基石。你今天已经入门了密码学！" |

## 🔥 重点难点

### 教学重点

1. **三个关键 ASCII 值**：'0'=48, 'A'=65, 'a'=97。以这三个为锚点推算其他字符。
2. **字符判断模板**：`c>='A' && c<='Z'`、`c>='a' && c<='z'`、`c>='0' && c<='9'`。
3. **字符数组索引技巧**：`cnt[c-'a']++`——用字符的偏移量作为数组下标。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 混淆字符 '5' 和数字 5 | 直接 `int x = '5';` 得到 53 | 画图：字符是"穿衣服的数字"，'5' 穿了一件"ASCII 码=53"的衣服。要拿到里面的数字，得"脱衣服"——减 '0' |
| 大小写转换只 ±32 不判断范围 | 把 '@' 也 "转大写" 得到奇怪结果 | 强调："±32 之前必须用 `if` 确认当前字符确实是大写/小写字母。乱转其他字符会出 bug" |
| 凯撒密码对 Z 的处理 | Z+3 变成 ']' 而非 'C' | 引入模运算 `%26`：首尾循环。比喻："26 个字母围成一个圈，Z 后面是 A，就像 12 月后面是 1 月" |

## 💬 常见学生问题

**Q1："ASCII 码只有 128 个，中文怎么办？"**

A：中文用 Unicode（统一码），可以表示全世界几乎所有文字。UTF-8 是 Unicode 的一种编码方式，一个中文字占 3 个字节。C++ 中的 char 是 1 字节，不够装中文，所以处理中文要用 wchar_t 或专门的库。初学阶段我们只处理英文字符。

**Q2："`'A' + 32` 一定等于 `'a'` 吗？在所有电脑上都一样？"**

A：在 ASCII 编码的系统中一定一样。现代几乎所有计算机都兼容 ASCII。考试和竞赛的评测机也使用 ASCII 兼容编码，所以放心用 `+32` / `-32`。

**Q3："为什么要记 ASCII 码的值？不能直接用字符吗？"**

A：大部分时候可以直接用字符（`if(c=='A')`）。但有一些场景必须要知道数值——比如字符排序底层比的就是 ASCII 值；大小写转换的 `±32` 也依赖知道它们差 32。记住三个锚点值就够了，其余可以推出来。

**Q4："凯撒密码也太好破解了吧？有没有更安全的？"**

A：当然有！凯撒密码是最古老的加密方式，跟"纸糊的门"差不多。现代加密（如 AES、RSA）涉及复杂的数学运算，你学到的数学越多，能理解的加密方式就越深。高中信息学竞赛里也可能出现简单的密码题——都是基于 ASCII 码的变换。

## 📋 本章小测

1. **填空题**：大写字母 'A' 的 ASCII 码是 ____，小写字母 'a' 的 ASCII 码是 ____，两者相差 ____。

2. **选择题**：以下哪个表达式可以判断字符 c 是数字？  
A. `c >= 0 && c <= 9` B. `c >= '0' && c <= '9'` C. `c >= 48 && c <= 57` D. B 和 C 都对

3. **填空题**：将小写字母 ch 转换为大写字母的表达式是 `ch = ch - ____`（需先确保 ch 是小写字母）。

4. **代码阅读**：以下代码输出什么？
```cpp
char c = 'C';
cout << (char)(c + 2);
```

5. **编程题**：读入一个字符，如果是大写字母就转成小写输出，如果是小写字母就转成大写输出，其他字符原样输出。

6. **判断题**：`'9' - '0'` 的结果是 9。（ ）

7. **编程题**：读入一行字符串，统计其中元音字母（a, e, i, o, u）出现的总次数（忽略大小写）。

---

### 参考答案

1. 65, 97, 32
2. D（B 和 C 等价，因为 '0'=48, '9'=57，C 是 B 的数值形式）
3. 32（`ch = ch - 32;` 或 `ch = ch - 'a' + 'A';`）
4. 'E'（'C'+2 = 67+2 = 69 = 'E'）
5. 思路：`if(c>='A'&&c<='Z') c+=32; else if(c>='a'&&c<='z') c-=32; cout<<c;`
6. ✅（'9' 的 ASCII 码 57 减去 '0' 的 48 = 9）
7. 思路：先统一转成小写/大写，再遍历判断每个字符是否为元音，维护计数器

## 🎮 不插电活动

### 活动 1：间谍密码本——ASCII 解码（第 1 课时用，10 分钟）

**道具**：大张 ASCII 表（贴在黑板上，至少包含 A-Z、a-z 部分）、数字密码纸条

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师事先准备：在黑板上贴一张放大的 ASCII 码对照表（或投影展示），包含大写 A-Z(65-90) 和小写 a-z(97-122) |
| 1-2 分 | 老师讲情景："你是一名间谍，截获了一组神秘数字。敌人用 ASCII 码加密了情报。你的任务——用墙上的密码本，把数字翻译成字母！" |
| 2-3 分 | 把全班分成 4-6 组，每组发一张纸条，上面写一串数字（如 `72 101 108 108 111` → "Hello"） |
| 3-5 分 | 各组对照墙上的 ASCII 表，将数字逐个翻译成字母，写在纸条背面。最先翻译出正确单词的组举手 |
| 5-7 分 | 各组亮出答案。老师揭晓正确答案（可准备 4 组不同难度的密码——最简单 2 组是短单词，中等 1 组是一句话，挑战 1 组混了标点符号） |
| 7-9 分 | 反向挑战："现在你是加密方！我说一个单词，你们把每个字母转成 ASCII 码数字喊出来。"——例如 "CAT" → "67 65 84" |
| 9-10 分 | 总结："每个字母背后都有一个数字编号。计算机里没有'字母'—只有数字。你看到的每一个字母，屏幕背后都是一个 0~127 之间的数。这就是 ASCII 码。" |

### 活动 2：密码攻防战——凯撒密码（第 4 课时用，15 分钟）

**道具**：纸条、笔

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 把全班分为 A 组（加密方）和 B 组（解密方），4-5 人一小组。A 组和 B 组一一配对 |
| 1-3 分 | A 组任务：选一个偏移量 k（1~25），写一段 5-8 个单词的英文短句，用凯撒密码加密（每个字母向后移 k 位），把密文交给配对的 B 组。注意：偏移量 k 保密！ |
| 3-5 分 | B 组拿到密文后，不知道偏移量，需要破解。老师提示："你们可以尝试每个可能的偏移量，看哪个能出有意义的英文单词" |
| 5-8 分 | B 组尝试破译。策略：先在纸上画一个 26 字母的轮盘或写两行字母对照表，然后逐一试 k=1,2,3… |
| 8-10 分 | B 组宣布破解结果（明文 + 猜出的偏移量）。A 组验证是否正确 |
| 10-12 分 | 角色互换！B 组加密，A 组破解 |
| 12-14 分 | 全班讨论："什么样的偏移量最难猜？"（1 和 25 太简单；13 左右相对难一些但暴力破解 26 次总能破）。"如果敌人不知道你用凯撒密码呢？"——引入密码学基础概念 |
| 14-15 分 | 总结："凯撒密码就是 ASCII 码的加减法。你今天的代码——`c = (c - 'A' + k) % 26 + 'A'` 就是凯撒密码的数学本质。暴力破解 = for 循环 k 从 1 到 25。" |

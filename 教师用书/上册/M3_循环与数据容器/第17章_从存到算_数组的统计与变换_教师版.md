## 🗺️ 教学流程

### 第 1 课时：数组统计（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "上次我们学会了把数字存进数组。存进去之后呢？就像你把东西放进柜子不是为了让它吃灰——你得用它！今天学怎么'用'数组" |
| 5-15 分 | 讲授 | **求和+均值**。板书标准模板：`sum=0; for(i=0;i<n;i++) sum+=a[i]; avg=sum/n;`。强调 sum 必须初始化为 0，这是血泪教训 |
| 15-25 分 | 讲授 | **打擂法求极值**。比喻："选班长——先指定第一个人当临时班长，后面每个人和他比，更强的就当新班长"。板书：`max=a[0]; for(i=1;i<n;i++) if(a[i]>max) max=a[i];` |
| 25-35 分 | 不插电 | **"唱票统计"**（详见活动1）。全班投票统计，用数组模拟统计过程 |
| 35-42 分 | 上机 | 学生练习：读入 8 个同学的身高（整数 cm），输出最高、最矮、平均身高 |
| 42-45 分 | 总结 | 三种统计的代码模板，强调区别：求和的累加器初始化为 0；求极值的记录器初始化为第一个元素 |

### 第 2 课时：线性查找 + 计数统计（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速复习上节课三种统计模板 |
| 5-15 分 | 讲授 | **线性查找**——"在一个抽屉柜里找你要的东西，从第一个抽屉翻到最后一个"。板书：`for(i=0;i<n;i++) if(a[i]==x) { cout<<i; break; }`。讨论：找到了要不要继续找？（不用，break 跳出） |
| 15-25 分 | 上机 | 练习题：读入 10 个数和一个目标值 x，输出 x 第一次出现的下标。如果没找到，输出 -1 |
| 25-35 分 | 讲授 | **计数统计**。"班上有多少人数学及格？"——遍历数组，遇到 ≥60 的给计数器 +1。板书模板：`cnt=0; for(i=0;i<n;i++) if(a[i]>=60) cnt++;` |
| 35-42 分 | 上机 | 综合题：读入 20 个整数，输出正数个数、负数个数、零的个数 |
| 42-45 分 | 总结 | 查找和计数的共同点：都要遍历数组，都用 if 做条件判断。不同点：查找找到就停，计数要全部看完 |

### 第 3 课时：逆序反转 + 标记思想（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | 提问："如何把一个数组的元素顺序倒过来？比如 [1,2,3,4,5] 变成 [5,4,3,2,1]？"让学生先自己思考 2 分钟 |
| 5-15 分 | 讲授 | **逆序输出 vs 逆序存储**。先讲简单的——逆序输出（从 n-1 遍历到 0）。再讲逆序存储：首尾交换法。板书：`for(i=0;i<n/2;i++) swap(a[i], a[n-1-i]);` |
| 15-20 分 | 演示 | 老师在黑板画 5 个格子，用箭头演示第 0 个和第 4 个交换、第 1 个和第 3 个交换、中间第 2 个不动。让学生看到"n/2 次就够了" |
| 20-25 分 | 讲授 | **标记思想（flag）**。引入 bool 变量作为"开关"。例子："判断数组里有没有负数"——`bool flag=false;`遍历中遇到负数就 `flag=true;` |
| 25-35 分 | 上机 | 练习 1：逆序存储一个数组。练习 2：判断数组中所有元素是否都是正数（全是输出 YES，否则 NO） |
| 35-42 分 | 拔高 | 挑战题："判断数组是否为回文数组"（如 [1,3,2,3,1]）。让学生综合运用首尾比较 + flag 标记 |
| 42-45 分 | 总结 | 逆序的核心：交换、只做 n/2 次。标记的核心：用一个 bool 变量"记住状态" |

## 🔥 重点难点

### 教学重点

1. **打擂法模板**：`max = a[0]` 然后从 i=1 开始比较，这是整个信奥最常用的模式之一。
2. **线性查找的 break 时机**：找到第一个就停 vs 找到全部继续。
3. **逆序交换的次数**："n/2 次"而非 n 次——想明白为什么只需要一半。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 打擂法从 i=1 开始而不是 i=0 | 多比较一次不影响结果但逻辑上不必要 | 问学生："你已经是临时班长，还需要和你自己比谁更厉害吗？" |
| 逆序交换写成 `i<n` 而不是 `i<n/2` | 交换了 n 次，等于交换两次回到原位 | 黑板演示：5 个格子，交换 5 次会发生什么——第 0 个和第 4 个交换→换回来 |
| 线性查找没找到时的处理 | 忘记输出 -1 或"No" | 统一模式：先设 `ans=-1`，找到时更新，遍历结束看 ans 是否还是 -1 |

## 💬 常见学生问题

**Q1："求最大值时，`max` 初始化为 0 可以吗？"**

A：如果数组里全是负数，初始化为 0 就错了——因为 0 比所有负数都大，结果会错误输出 0。所以**打擂法必须用数组第一个元素初始化最大值/最小值**，这是铁律。

**Q2："线性查找和之前学的二分查找有什么区别？"**

A：线性查找是一个一个翻，不需要数组有序，适用于任何情况。二分查找（后面会学）要求数组必须先排序，但速度极快。就像在大字典里查一个英文单词——你可以从 A 字母那页开始一页页翻（线性），也可以直接翻开中间、判断在左边还是右边（二分）。

**Q3："`break` 跳出循环后，如果还想知道找到了几个怎么办？"**

A：那就不该用 break——用计数。`for` 循环体里遇到相等就 `cnt++`，不 break，一次遍历全部统计完。break 适用场景是"只要找到第一个"。

**Q4："逆序存储和直接逆序输出，什么时候用哪个？"**

A：如果后面还要用这个数组，就必须真·逆序存储；如果只是最后一次输出数据，逆序输出就够了（省事、代码少）。考试时认真读题——"将数组中的元素倒序排列"就是要改数组，"倒序输出数组元素"只需要改输出顺序。

## 📋 本章小测

1. **填空题**：求数组 a（长度 n）的最大值，`max` 应该初始化为 ______。

2. **选择题**：以下哪段代码正确实现了数组逆序存储？  
A. `for(i=0;i<n;i++) swap(a[i],a[n-1-i]);`  
B. `for(i=0;i<n/2;i++) swap(a[i],a[n-1-i]);`  
C. `for(i=0;i<n/2;i++) swap(a[i],a[n-i]);`  
D. `for(i=0;i<=n/2;i++) swap(a[i],a[n-1-i]);`

3. **判断题**：线性查找要求数组必须事先排好序。（ ）

4. **填空题**：统计数组 a 中等于 x 的元素出现的次数，核心代码填空：  
`int cnt = ____ ; for(int i=0;i<n;i++) if(________) cnt++;`

5. **编程题**：读入 n 个整数（n≤50），找出最小值及其下标（如果有多个最小值，输出第一个）。

6. **改错题**：以下代码想求数组平均值，有什么问题？
```cpp
int a[5] = {80, 90, 75, 85, 95};
int sum, avg;
for(int i=0; i<5; i++) sum += a[i];
avg = sum / 5;
```

7. **简答题**：flag 标记变量一般用在什么场景？举一个具体例子。

---

### 参考答案

1. `a[0]`（数组第一个元素）
2. B（i<n/2，对应 a[i] 和 a[n-1-i] 交换。A 交换了两次回到原位；C 的 n-i 越界；D 的 i<=n/2 多交换了一次）
3. ×（线性查找不要求有序；只有二分查找才要求有序）
4. `0`，`a[i] == x`
5. 思路：`minVal = a[0]; minPos = 0;` 遍历时如果 `a[i] < minVal`（严格小于以保证取第一个），更新两个变量
6. ①sum 未初始化为 0（局部变量值是垃圾值）；② avg = sum/5 是整数除法，得不到精确的平均值，应该用 `double avg = sum / 5.0;`
7. flag 用于"是否发生过某件事"的判断。例：判断数组中是否出现过负数——`bool flag=false;` 遍历时碰到负数就 `flag=true;`，遍历结束看 flag 的值

## 🎮 不插电活动

### 活动 1：唱票统计（第 1 课时用，10 分钟）

**道具**：粉笔、选票用小纸条、黑板

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|
| 0-1 分 | 老师在黑板上画一排 4 个格子，编号 0~3，分别代表 4 首候选班歌 |
| 1-2 分 | 给全班每人发一张小纸条，写上自己最喜欢的候选歌曲编号（0~3） |
| 2-3 分 | 老师收齐纸条，说明："这些纸条就是数组的元素，每个纸条上有一个数字" |
| 3-5 分 | 老师逐一念纸条："张三投了 1 号歌"——在黑板上第 1 个格子里画一笔"正"字（模拟 `cnt[1]++`）。全班看着格子里的笔画越来越多 |
| 5-7 分 | 全部念完后，老师："现在每个格子的笔画数 = `cnt[0]` 到 `cnt[3]` 的值。怎么找出得票最多的歌？"——引出打擂法 |
| 7-9 分 | 请 4 名同学上讲台，各自代表一个格子，比赛"谁的笔画最多"——模拟打擂比较过程，一步步走完 i=0 到 3 |
| 9-10 分 | 总结：纸条数据（数组元素）→ 遍历（念纸条）→ 计数（画正字）→ 查找最值（打擂），这就是用数组做统计分析的全流程 |

### 活动 2：翻转人墙——逆序反转（第 3 课时用，5 分钟，可作热身）

**道具**：无

**操作步骤**：

| 时间 | 谁做什么 |
|------|----------|

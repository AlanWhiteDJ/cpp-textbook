## 🗺️ 教学流程

### 第 1 课时：整数类型（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | 讲小刚的 85 亿分变成 523 万的故事。问学生"你觉得问题出在哪？" |
| 5-20 分 | 讲授 | 数据类型 = 杯子。int = 中杯（约 21 亿），long long = 大杯（约 9×10¹⁸）。用实物大小比喻 |
| 20-30 分 | 板书 | 在白板上写出 int 的范围：-2,147,483,648 ~ 2,147,483,647。让学生感受"21 亿"有多大（中国人口约 14 亿，所以 int 能装下中国人口但装不下全球人口×2） |
| 30-40 分 | 动手 | 让学生声明 int 和 long long 变量并输出 sizeof |
| 40-45 分 | 口诀 | "普通数字用 int，超过 20 亿换 long long" |

### 第 2 课时：溢出与浮点数（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 回顾 | 复习 int 和 long long |
| 10-20 分 | 演示 | **溢出实验**：让学生把 `int a = 2147483647; a = a + 1; cout << a;`。观察结果变成 -2147483648。全班惊呼时解释"就像汽车里程表，到了 999999 再往前就翻成 000000" |
| 20-30 分 | 讲授 | 浮点数。为什么叫"浮点"——小数点可以浮来浮去（0.00314 = 3.14×10⁻³）。强调：float 精度太低，竞赛中一律用 double |
| 30-40 分 | 演示 | 整数除法 vs 浮点除法。`5/2` vs `5.0/2` |
| 40-45 分 | 总结 | float vs double：就像直尺 vs 游标卡尺，都能量长度但精度差很远 |

### 第 3 课时：类型选择实战（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-30 分 | 练习 | 动手试试 1-3 + 场景判断题。给出一系列场景，让学生选类型（如"全校学生人数"→ int，"全球人口"→ long long，"圆周率"→ double） |
| 30-45 分 | 讨论 | 为什么 int 的范围是约 21 亿？引入 2³² 的概念（不深入，只种种子） |

### 第 4-5 课时：练习 + 小测（90 分钟）

同前两章模式：纠错 → 练习 → 互批 → 小测。

## 🔥 重点难点

### 重点

1. **int vs long long 的选择**：竞赛中最常见的优化就是"换 long long"。口诀："乘法之前先想范围"
2. **溢出不报错**：这是最危险的地方。必须通过动手实验让学生**亲眼看到**溢出产生垃圾值

### 突破策略

| 卡点 | 突破 |
|------|------|
| 不理解"为什么 int 有范围" | 用杯子容量的比喻。8 个 bit 就是 8 个小格子，每个格子可以放 0 或 1，总共 256 种组合。int 有 32 个格子=约 42 亿种组合=一半负数一半正数≈21 亿 |
| 不理解浮点精度 | 让学生用 float 和 double 各存 π=3.14159265358979，输出看后几位 |
| sizeof 概念 | 板书画出不同类型的内存占用条形图 |

## 💬 常见问答

**Q1："int 不够用的时候，我能不能所有变量都声明成 long long？"**

A：可以，而且很多竞赛选手就是这么干的。long long 占 8 字节，int 占 4 字节，对现代电脑来说这点差距可以忽略。但养成"全用 long long"的习惯也有坏处——哪天不能用的时候（比如大数组），你不知道怎么选。所以建议：知道什么时候该用什么。

**Q2："溢出为什么有时变成负数有时不是？"**

A：好问题！这和计算机用补码表示负数有关。这个知识点在模块 5 会详细讲。现阶段记住"溢出 = 不可预测的垃圾值"就行了。

## 🎲 课堂活动

### 活动：溢出接力赛

**道具**：白板

**规则**：
- 分两组，老师出一个算式（如 100000 × 100000）
- 每组抢答：这个结果 int 能装下吗？如果不能，应该用什么类型？
- 每次答对记 1 分，答错扣 1 分
- 最后一道题是 1×2×3×...×20，让两组各自估计结果多大

**目的**：训练类型选择的直觉。

## ✅ 习题详解

### 基础题

**1. 选择题**
（1）B（4 字节）  
（2）B（-1 亿在 int 范围内）  
（3）C（圆面积涉及 π，需要小数）

**2. 填空题**
（1）8  
（2）溢出  
（3）8

### 提高题

**3. 分析题**

```
a = 2（5/2 整数除法）
b = 2.0（5/2 整数除法得 2，赋值给 double 变成 2.0）
c = 2.5（5.0 触发浮点除法）
输出：2 2 2.5
```

**4. 编程题**

20! ≈ 2.43×10¹⁸，在 long long 范围内（约 9.22×10¹⁸）。用 int 会在中途溢出。

### 挑战题

**5. 探究题**

int 最小值 = -2,147,483,648，最大值 = 2,147,483,647。负数多一个是因为补码表示法中，0 占了一个正数位置，所以正数比负数少一个。

## 📦 补充题库

1. `long long` 的最大值是多少？写出具体数字。（9,223,372,036,854,775,807）

2. 如果 `int a = 5 / 2;` 和 `double b = 5.0 / 2;`，`a + b` 是多少？（2 + 2.5 = 4.5，int 和 double 混合运算时，int 自动提升为 double）

3. 什么时候会用到 `unsigned int`？（当确定数值不会为负，且需要更大的正数范围时。如数组下标、计数。但竞赛中不常用。）

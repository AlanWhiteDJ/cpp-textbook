# 第 2 章  你的第一个程序（教师用书）

---

## 🎯 教学目标

| 维度 | 目标 |
|------|------|
| **知识** | 说出 Hello World 程序每行的作用；区分编译和运行 |
| **能力** | 独立在 IDE 中写出、编译、运行 C++ 程序；使用注释 |
| **素养** | 养成"亲手敲代码、不复制粘贴"的习惯 |

---

## ⏱️ 课时分配（5 课时）

| 课时 | 内容 | 形式 |
|------|------|------|
| 第 1 课时 | IDE 安装与环境配置 | 动手实操 |
| 第 2 课时 | Hello World 逐行讲解 + 编译运行 | 讲授 + 实操 |
| 第 3 课时 | 编译 vs 运行 + 注释 + 常见错误排查 | 讲授 + 故意犯错 |
| 第 4 课时 | 动手试试 + 练习 | 独立练习 |
| 第 5 课时 | 小测 + 小结 | 测验 + 纠错 |

## 🗺️ 教学流程

### 第 1 课时：环境配置（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 讲授 | 什么是 IDE，推荐 Dev-C++。展示安装步骤（投影/屏幕共享） |
| 10-35 分 | 实操 | 学生自行安装 IDE。老师逐排检查，确保每个人都装好 |
| 35-45 分 | 演示 | 新建 C++ 源文件、保存为 `.cpp`、认识界面（代码区、编译按钮、输出区） |

> ⚠️ 这节课最乱的可能是电脑问题。提前准备一个 U 盘装着安装包，防止网络不通。预留时间给各种意外（系统不兼容、权限不够等）。

### 第 2 课时：Hello World（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 导入 | "今天我们要让电脑跟我们打招呼。准备好了吗？" |
| 5-25 分 | 逐行敲码 | 老师在投影上敲一行，学生跟着敲一行。**每行敲完都要停一下，简单解释这行在干什么**。重点是让学生**跟着敲**，不是听 |
| 25-35 分 | 逐行讲解 | 六行代码各是什么。用"厨房做饭"类比——include = 拿锅，using namespace = 打开调料柜，main = 灶台，cout = 盛菜，return 0 = 关火 |
| 35-45 分 | 编译运行 | 每个人点击编译运行。看到 "Hello, World!" 的瞬间，全班鼓掌！——仪式感很重要 |

### 第 3 课时：深入理解 + 错误教育（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 复习 | 回顾 Hello World 每行代码 |
| 10-20 分 | 讲授 | 编译 vs 运行的区别。用"翻译 vs 表演"类比：编译 = 把剧本翻译成演员能看懂的语言，运行 = 演员上台表演 |
| 20-25 分 | 讲授 | 注释：单行 `//` 和多行 `/* */` |
| 25-38 分 | 活动 | **"故意犯错"**——老师让学生逐个犯典型错误：少分号、少引号、Cout 大写、中文标点。每次犯错后让学生读错误提示，试着理解。这个环节能**极大降低未来的求助量** |
| 38-45 分 | 总结 | "错误不可怕，可怕的是不看错误提示" |

### 第 4 课时：练习（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 热身 | 重新敲一遍 Hello World |
| 5-35 分 | 独立练习 | 做动手试试 1-3 + 练习题 1-4。老师在机房里走动答疑 |
| 35-45 分 | 展示 | 请两三名学生展示他们的挑战题（ASCII 图案），全班投票谁画得最好看 |

### 第 5 课时：小结 + 小测（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 思维导图 | 黑板上画 Hello World 结构图 |
| 10-25 分 | 小测 | 纸笔测试：填空 + 改错 + 默写 Hello World 程序 |
| 25-35 分 | 互批 | 同桌互批 |
| 35-45 分 | 展望 | "下章——数字的秘密！" |

## 🔥 重点难点

### 重点

1. **Hello World 每行的作用**：不要求深入理解，但每行大概在干什么得知道
2. **编译 vs 运行**：两步走，不能混
3. **分号的重要性**：C++ 的句号，不能丢

### 卡点 + 突破

| 卡点 | 突破方法 |
|------|----------|
| 分号总忘 | "每行指令最后都打个句号。你写作文每句最后打句号吗？对，C++ 也一样。" |
| 中英文标点分不清 | 让学生把输入法状态栏钉在屏幕显眼位置。敲代码时强行切换到英文。前 5 次课可以不写中文注释 |
| 错误提示看不懂 | 教他们看最后一行。编译错误提示的最后一行通常指出具体位置。不用全懂，先找行号 |
| 觉得"我只是照着敲，没理解" | 纠正心态："这就是对的。你学说话的时候也是先模仿，后理解。先下水，再学泳姿。" |

## 💬 常见问答

**Q1："为什么一定要从 Hello World 开始？"**

A：编程界有一个迷信：你写的第一个程序如果不是 Hello World，后面的程序都会出 bug。（开玩笑的。）真正的原因是：它足够简单，只需要输出，能一步验证你的环境是否正确安装、编译器是否正常工作。

**Q2："`using namespace std` 是什么意思？不写行吗？"**

A：试试不写——所有 `cout` 前面都得加 `std::`。这行代码的作用是把标准工具箱里的工具直接拿出来用。现阶段把它当固定开场白，你学到后面会理解得更深。

**Q3："我敲代码特别慢，是不是不适合学编程？"**

A：每个程序员都是从"一指禅"起步的。慢是正常的。快的秘诀不是练打字，是**多敲代码**。你在 IDE 里敲 100 个小时代码后，手会自动记住那些符号的位置。别急。

## 🎲 课堂活动

### 活动：故意犯错大赛（第 3 课时用）

**规则**：
- 老师给 Hello World 代码（完整正确的版本）
- 老师依次说"删掉分号"、"把 cout 改成 Cout"、"用中文分号"
- 学生每次犯错后编译，观察错误提示
- 每轮第一个找到并说出错误在哪里的人记 1 分
- 总分最高的学生获得"Bug 侦探"称号

**目的**：让学生不惧怕错误提示，养成自己排查的习惯。

## ✅ 习题详解

### 基础题

**1. 填空题**

（1）编译  
（2）单行、多行  
（3）输出内容到屏幕  
（4）main

**2. 代码改错**

三处错误：
1. `include` 前缺 `#`
2. `cout << "I can code!"` 后面缺分号 `;`
3. 没有 `return 0;`

### 提高题

**3. 简答题（参考答案）**

如果删掉 `#include <iostream>`，编译器会报错说 `cout` 未定义。因为 `cout` 是 iostream 库里的工具，不 include 就用不了。就像你没从柜子里拿锅就想炒菜——当然炒不了。

**4. 编程题（参考代码）**

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "我叫小明" << endl;
    cout << "今年 12 岁" << endl;
    cout << "我在学 C++ 编程" << endl;
    return 0;
}
```

### 挑战题

**5. 参考答案**：三角形示例

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "   *" << endl;
    cout << "  ***" << endl;
    cout << " *****" << endl;
    cout << "*******" << endl;
    cout << "   *" << endl;
    cout << "   *" << endl;
    return 0;
}
```

评分类似创意、图案复杂度、代码整洁度。

## 📦 补充题库

1. 如果我把 `return 0;` 改成 `return 1;`，程序还能跑吗？试试看。（能跑，但 1 通常表示程序异常退出。对操作系统来说，返回值 0 是一切正常的信号。）

2. `cout << "Hello" << " " << "World" << endl;` 输出什么？（`Hello World`，中间有空格。证明 `<<` 可以串联多个内容。）

3. 如何在一行里输出 `Hello` 和 `World`，但它们之间换行？（`cout << "Hello\nWorld" << endl;`，其中 `\n` 在字符串内部起到换行效果。）

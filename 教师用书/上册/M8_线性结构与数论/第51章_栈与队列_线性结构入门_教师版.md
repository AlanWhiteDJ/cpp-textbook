## 🗺️ 教学流程

### 第 1 课时：stack——叠盘子（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | 食堂叠盘子故事——最后放上去的最先被拿走（LIFO）。浏览器后退按钮——回到"上一个页面"=最近访问的。编程中的撤销操作（Ctrl+Z）也是栈。日常生活还有很多 LIFO 场景：穿脱衣服（先穿的后脱）、摞书 |
| 10-25 分 | 讲授 | stack 五种操作：`push(x)` 压栈、`pop()` 弹栈（⚠️ 无返回值！）、`top()` 查看栈顶、`empty()` 判空、`size()` 大小。板书画图：用一摞书演示栈顶指针的上下移动——每 push 一本，栈顶上升；每 pop 一本，栈顶下降。关键口诀：**"push 放顶上，pop 取顶上，top 只看不拿"** |
| 25-38 分 | 讲授+演示 | 括号匹配——stack 的经典应用。算法：遍历字符串，遇 `(` 压栈，遇 `)` 弹栈。三种错误：①弹栈时栈空→右括号多了 ②遍历完栈非空→左括号多了 ③类型不匹配（`([)]`）。IDE 逐字符演示 `"(()())"` 和 `"(()"` 每一步的栈状态——用 `cout` 打印栈的大小变化 |
| 38-45 分 | 练习 | 学生口头推演 `"(()(()))"` 的栈变化。快速问答：弹栈时发现栈空说明什么？（右括号多余）。遍历完栈里还有元素说明什么？（左括号多余）。布置思考题："如果括号有三种 `()` `[]` `{}`，怎么改代码？" |

### 第 2 课时：queue——排队（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-15 分 | 🎮 不插电活动 | **"食堂打饭"**。10 个学生排成一队，老师喊"push 张三"→张三排到队尾。喊"pop"→第一个人出队。喊"front"→第一个人举手（不离开）。用粉笔在地上标 front 和 rear 两个标记位，观察它们随 push/pop 的移动方向——都往一个方向走！ |
| 15-30 分 | 讲授 | queue 五种操作：`push(x)` 入队、`pop()` 出队、`front()` 队首、`back()` 队尾、`empty()`/`size()`。数组模拟循环队列：`q[(rear++)%MAX]=x` 入队，`x=q[(front++)%MAX]` 出队。板书示意图：数组画成环形——rear 追 front，追上说明满了。满/空判断技巧：**牺牲一个位置**——满=(rear+1)%MAX==front，空=front==rear |
| 30-42 分 | 练习 | 学生上机三道递进题：①用 STL stack 判断括号匹配 ②用 STL queue 模拟叫号系统（1号请到3号窗口……）③用普通数组 + 两个下标变量自己实现 stack（不依赖 STL），然后再实现 queue |
| 42-45 分 | 小结 | "stack = LIFO（叠盘子），queue = FIFO（排队）。选容器的核心问题是：**谁先出来？** 最近加的先出→stack；最早加的先出→queue。选对了，代码减半。" |

### 第 3 课时：deque + 综合 + 小测（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-15 分 | 讲授 | deque（double-ended queue）双端队列：前后都能 O(1) 插入/删除。`push_front/push_back/pop_front/pop_back`。对比 vector：vector 只有尾部 O(1)，头部插入 O(n)。应用场景：滑动窗口最值（未来 M12 单调队列的前奏）——窗口右扩时 push_back，左缩时 pop_front |
| 15-30 分 | 综合练习 | 三道递进题：①基本括号匹配（只有 `()`）②扩展——三种括号 `()` `[]` `{}`，遇右括号时检查栈顶是否是对应的左括号（用 `if-else` 或 `switch` 判断对应关系）③经典题——用两个 stack 模拟 queue（stack1 压入队；出队时若 stack2 为空，把 stack1 全部弹出压入 stack2，再从 stack2 弹出） |
| 30-45 分 | 小测+展望 | 5 道题闭卷 15 分钟。互批+讲评 10 分钟。"stack 和 queue 是线性结构的两种经典范式。但有的时候我们想要一个既能随机访问、又能动态扩容的容器——下一章 vector！" |

## 🔥 重点难点

### 教学重点

1. **pop() 无返回值**：这是 STL 的统一设计——pop 只删除元素，top/front 负责取值。两个步骤分开，避免"取值失败时元素已被删除"的问题。
2. **括号匹配的多类型判断**：不能只看数量相等，还要类型对应。解法：遇右括号时检查栈顶是否匹配——如遇 `)` 时栈顶必须是 `(`。`([)]` 不合法正是因为它违反了类型对应的规则。
3. **循环队列的牺牲位**：为了区分空和满，约定牺牲一个数组位置。这虽然"浪费"了一个位置，但免去了额外维护计数变量的复杂性——是工程上的经典权衡。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| `s.pop()` 当返回值用 | `int x = s.pop()` 编译报错 | 固定写法：`int x = s.top(); s.pop();`。告诉学生这是 STL 的设计哲学——分离"查看"和"删除"两个操作，更安全 |
| 循环队列满/空判断 | front==rear 时分不清 | 画环形图：牺牲一个位置后，满=(rear+1)%MAX==front，空=front==rear。类比停车场——留一个空位方便调度 |
| 括号类型不匹配被漏掉 | `([)]` 被判为合法 | 演示：遇 `)` 时栈顶是 `[`→不匹配→非法。强调"匹配"不仅是数量对，还要位置嵌套正确 |

## 💬 常见问答

**Q1："什么时候用 stack，什么时候用 vector？"**

A：需要"后进先出"或"撤销/回溯"→stack；需要随机访问或动态扩容→vector。stack 只暴露栈顶接口，保护数据不被随意访问——这是"封装"的好处。

**Q2："括号匹配如果是 `([{)]}` 能通过吗？"**

A：不能——虽然数量对，但类型不匹配（`{` 后期望 `}` 却遇到了 `)`）。解法：遇到右括号时检查栈顶是否是对应的左括号。

**Q3："数组模拟 queue 为什么要循环？"**

A：不循环的话，front 一直往前走，前面出队的空间浪费了。循环队列把数组首尾相连，空间复用。

**Q4："什么时候用 stack，什么时候用递归？"**

A：递归本质上就是系统帮你维护了一个隐式的栈（调用栈）。当你需要"后进先出"的行为但不想写显式栈时→递归；当你需要自己控制栈的大小和操作时→显式 stack。事实上，任何递归都可以用 stack + 循环改写——只是代码会变复杂。

**Q5："deque 和 vector 有什么区别？什么时候用 deque？"**

A：deque 支持两头 O(1) 插入/删除，vector 只支持尾部 O(1)。但 vector 内存连续（缓存友好），deque 是分段存储（访问稍慢）。需要头尾都操作→deque；只需要尾部操作+随机访问→vector。滑动窗口最值问题常用 deque 实现单调队列。

## 📋 本章小测（15 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. stack 的规则是____（LIFO/FIFO），queue 的规则是____。
2. `stack.pop()` 的返回值是____（有/无）。
3. 查看 queue 队首用____，队尾用____。
4. 用 stack 判断 `"(()"` 是否合法时，最后栈____（空/非空）。
5. deque 可以从____端和____端插入/删除。

### 二、写代码（10 分）

6. 写一个函数 `bool isValid(string s)` 判断括号字符串（只含 `()`）是否合法。用 stack 实现。

---

### 答案

1. LIFO, FIFO
2. 无（void）
3. front(), back()
4. 非空（多了一个左括号）
5. 前，后
6. ```cpp
bool isValid(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(') st.push(c);
        else {
            if (st.empty()) return false;
            st.pop();
        }
    }
    return st.empty();
}
```

## 🎮 不插电活动

### 活动：食堂打饭（第 2 课时用，约 15 分钟）

**道具**：10 个学生；粉笔在地上标出 front 和 rear 位置

**规则**：学生排成一队。老师喊"push 张三"→张三排到队尾（rear 往后移一格）。喊"pop"→第一个人出队（front 往后移一格）。喊"front"→第一个人举手但不离开。观察 front 和 rear 都只往一个方向移动——出队的位置被"废弃"了。引出循环队列的必要性——如果把队列首尾连起来，废弃的位置就能复用。

**教学要点**：第二遍用"循环"的方式——当 rear 走到尽头时回到 0 号位置。让学生用身体感受"取模"的含义。

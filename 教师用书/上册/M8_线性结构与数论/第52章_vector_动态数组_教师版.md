## 🔥 重点难点

### 教学重点

1. **vector 的下标访问和数组完全一样**：`v[i]` 直接用，不需要特殊语法。这是 vector 最大的易用性优势。
2. **中间插入/删除的代价**：O(n)——因为需要移动后续所有元素。画图演示插入位置 2——下标 2~n-1 的所有元素都要往后移一格。
3. **capacity vs size**：`v.size()` 是当前元素数，`v.capacity()` 是已分配的内存能容纳的元素数。当 size 超过 capacity 时触发扩容（通常是翻倍），扩容涉及重新分配内存+拷贝全部元素——这是一次性的大开销。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| `v.size()` 返回无符号数 | `for(int i=0;i<v.size()-1;i++)` 当 size=0 时溢出为超大值 | 警告：`v.size()-1` 在空 vector 时是一个巨大的正数（unsigned 下溢）。正确写法：`for(int i=0;i+1<v.size();i++)` 或先 `int n=v.size()` |
| 范围 for 修改元素无效 | `for(int x:v) x=5;` 改不了原元素 | 用引用：`for(int &x:v) x=5;`——`&` 让 x 成为原元素的别名 |
| insert 在头部很慢 | 不知道 insert 是 O(n) | 演示：在 vector 头部 insert 10⁵ 次 vs 在尾部 push_back 10⁵ 次——前者 O(n²)，后者 O(n) |

## 💬 常见问答

**Q1："vector 和数组什么时候该用哪个？"**

A：99% 用 vector（安全、自动管理大小、有 size()）。数组只在两种情况下用：①大小在编译时确定且永远不会变 ②对性能有极致要求且已知固定大小。竞赛中推荐默认用 vector。

**Q2："vector 每次 push_back 都会扩容吗？"**

A：不是——vector 有 capacity（容量）和 size（大小）两个概念。只有当 size 超过 capacity 时才重新分配内存，通常扩容到原来的 2 倍。所以 push_back 的**均摊**复杂度是 O(1)，偶尔的扩容开销被分摊到大量 O(1) 操作中。

**Q3："vector 能和 sort 一起用吗？"**

A：能！`sort(v.begin(), v.end())` 或 `sort(v.begin(), v.end(), greater<int>())`。完全和数组一样方便。

**Q4："vector 的下标访问和数组一样快吗？"**

A：几乎一样。vector 的 `operator[]` 就是指针偏移——和数组无区别。但 vector 比数组多了一次间接跳转（堆 vs 栈），这个差异在 99% 的场景中可忽略。如果需要极致性能且大小固定不变，才考虑用数组。

**Q5："什么时候用 v.reserve(n) ？"**

A：当你提前知道 vector 最终大约会存多少元素时，用 `v.reserve(n)` 预分配内存。这样可以避免多次 push_back 触发扩容（扩容涉及重新分配+拷贝全部元素）。如读入 10⁵ 个数据前：`v.reserve(100000)`。这不改变 size，只改变 capacity。

## 📋 本章小测（15 分钟，闭卷）

### 一、填空题（每空 2 分，共 10 分）

1. vector 的头文件是____。
2. 在 vector 末尾添加元素用____，删除末尾用____。
3. vector 中间插入的时间复杂度是____。
4. 范围 for 遍历时，要修改元素必须用____。
5. `v.empty()` 在容器为空时返回____。

### 二、写代码（10 分）

6. 读入 n 个整数存到 vector 中，先升序输出，再降序输出。要求用 sort。

---

### 答案

1. `<vector>`
2. push_back, pop_back
3. O(n)
4. 引用 `&`（`for(int &x:v)`）
5. true
6. ```cpp
vector<int> v; int n, x;
cin >> n;
for (int i = 0; i < n; i++) { cin >> x; v.push_back(x); }
sort(v.begin(), v.end());
for (int x : v) cout << x << " "; cout << endl;
sort(v.begin(), v.end(), greater<int>());
for (int x : v) cout << x << " ";
```

## 🎮 不插电活动

### 活动：弹性购物车（第 2 课时用，约 12 分钟）

**道具**：6 个学生手拉手围成可变大小的圈（模拟 vector 的底层数组）；几个"物品"标签

**规则**：每次加物品（学生加入圈）——加到末尾时，圈尾两个学生松手，新学生插入拉手（O(1)）。加到中间时——指定位置的学生松手，新学生插入，但后面所有学生必须往后挪一位（O(n)）。两轮对比让学生亲身体会"末尾 O(1) vs 中间 O(n)"。

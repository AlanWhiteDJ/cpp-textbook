## 🗺️ 教学流程

### 第 1 课时：while 循环入门（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-10 分 | 导入 | 三连问引发思考：①"从 1 加到 100，用 for 很简单——但如果是"一直加直到总和超过 1000 为止"呢？你预先不知道加多少次。"②"输入密码——你不知道用户第几次才能输对。"③"游戏进行——你不知道玩家什么时候选择退出。"——引出认知：有些循环**次数不确定**，for 循环的三个要素在这里不够用。我们需要 while |
| 10-25 分 | 讲授 | while 语法：`while (条件) { 循环体; }`。核心口诀："当条件为真，就执行循环体；执行完再判断条件；还是真就再来一次；直到条件为假才退出。"——把 while 的执行过程分解成"判断→执行→判断→执行→…→判断→退出"的链条。板书对比：for 适合"知道次数"的循环，while 适合"知道条件"的循环 |
| 25-35 分 | 互动推演 | 经典例题：从 1 开始累加，直到总和超过 100。师生共同推演：`int sum=0, i=1; while(sum<=100) { sum+=i; i++; }`——每次循环后在黑板表格中记录 sum 和 i 的值。学生会看到：sum 依次为 1, 3, 6, 10……直到 105（此时 i=14）退出 |
| 35-42 分 | 练习 | "输入密码直到正确"——`string pwd; while(pwd != "8888") { cin >> pwd; }`。追问两个细节：① pwd 初始值是什么？如果没有初始值会怎样？②如果用户第一次就输入正确，循环体执行了吗？ |
| 42-45 分 | 总结 | "while 就像一个门卫——你走到门前，他先检查条件，符合就放你进去干活；干完活出来，他又检查一遍，直到你不再符合条件为止。" |

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-5 分 | 回顾 | 快速复习 while 执行流程，抽查学生口头推演 |
| 5-25 分 | 讲授+演示 | 两个经典 while 实战案例：①**数位分离**：输入一个正整数，把它的每一位数字分离出来。`while(n>0) { cout << n%10; n/=10; }`——这是倒序输出。如果要求正序输出，需要额外技巧。用 1234 一步一步在黑板推演：n%10 拿到个位→输出→n/10 去掉个位→重复直到 n=0。②**多组输入直到 EOF**：`while(cin>>x) { ... }`——Ctrl+Z 结束输入。这是竞赛中常见的输入模式 |

### 第 3 课时：do-while 循环（45 分钟）

| 时间 | 环节 | 操作 |
|------|------|------|
| 0-8 分 | 导入 | 回顾上节课的"密码验证"："如果密码正确，while 的循环体一次都不执行。"这合理吗？但也有些场景：**无论如何至少做一次**。比如：先显示菜单，再问用户选择；先出一道题，再问是否继续——引出 do-while |
| 8-20 分 | 讲授 | do-while 语法：`do { 循环体; } while (条件);`。核心区别：①**先执行后判断**——循环体至少执行一次；②while 后面的分号不能少。板书画出两种循环的流程图：while（菱形在矩形上方）vs do-while（菱形在矩形下方）。让学生视觉上记住"先判断 vs 后判断" |
| 20-32 分 | 对比实验 | 同一个问题用 while 和 do-while 分别写，让学生对比。例：输入正整数 n（保证 ≥1），但如果不是正整数则重复输入。while 写法需要先读一次（循环前）再判断；do-while 更自然：直接 do { cin >> n; } while (n <= 0);——这正是 do-while 的主场 |
| 32-40 分 | 讨论 | "while 和 do-while 能否互相替代？"引导学生发现：while 可以模拟 do-while（先执行一次再判断），但代码会变冗余；do-while 不能直接模拟 while（因为它至少执行一次）。结论：两种循环有交集但不能完全等价 |
| 40-45 分 | 总结 | "while 是'进门先刷卡'，do-while 是'进来再说，出门再刷卡'。" |

### 第 4 课时：死循环 + 循环选择 + 综合练习（45 分钟）

| 0-15 分 | 讲授+演示 | **死循环**专题。演示三个经典死循环：① `while(true)`——永远为真；② `int i=1; while(i>0) i++;`——虽然条件会变化，但 i 永远大于 0，到溢出前都不会停；③ `int i=10; while(i != 5) i+=2;`——i 是 10,12,14……永远不会等于 5。总结死循环三病因：条件永远为真、变量方向错误、退出条件永远达不到。教调试方法：在循环体里加 `cout << i << endl;` 观察变量变化 |

## 🔥 重点难点

### 教学重点

1. **while 与 for 的本质统一**：`for (A; B; C) { D; }` 几乎等价于 `A; while (B) { D; C; }`。让学生理解它们只是同一思想的两种表达，消除"又要学新东西"的抵触心理。
2. **do-while 的"至少一次"语义**：这是它唯一的不可替代场景。用输入验证和菜单显示两个典型例子反复强化。
3. **死循环的预防**：这是写循环最需要养成的习惯。教学生"写循环前先在注释里写退出条件"。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| while 条件写反 | `while(n > 0)` 想表达"n 小于 0 时继续"却写成 n>0 | 让学生养成一个习惯：写完条件后用中文念一遍。`while (n > 0)` 意思是"当 n 大于 0 时循环"，如果你想要的是"当 n 不是正数时循环"，应该写 `while (n <= 0)` |
| 忘记更新循环变量 | `while (i < 10) { cout << i; }`——i 永远是 1 | 这条错误有个名字叫"无限循环罪犯第一条"。强制习惯：写完 while 循环体后立刻检查——循环变量在循环体里变了吗？ |
| do-while 忘记分号 | `do { ... } while (条件)`——少分号编译报错 | 口诀："do-while 三件套：do、花括号、while 后面跟分号。"把这个口诀写在黑板上，让学生念三遍 |

## 💬 常见问答

**Q1："既然 for 可以改写成 while，为什么不只用一种？"**

A：因为"简洁就是力量"。`for (int i=0; i<10; i++)` 把三要素浓缩在一行，读代码的人秒懂这是"从 0 到 9 跑 10 次"。如果用 while 写同样逻辑，三要素会分散在三行——分散就容易遗漏。选对表达方式，就是帮读你代码的人减少认知负担。

**Q2："while 和 do-while 在实际项目中谁更常用？"**

A：while 远多于 do-while。因为"先判断后执行"符合大多数业务逻辑。do-while 只在特定场景出现（菜单、输入验证、游戏主循环）。但你在考试和竞赛中两种都要会——出题人喜欢考它们的区别。

**Q3："while(true) 是不是永远不能写？"**

A：在某些场景下是标准写法。比如游戏主循环——游戏一直在运行，直到玩家点击退出按钮，此时用 `break` 跳出。但作为初学者，优先建议你用明确的退出条件。等你经验丰富了，再学 `while(true) + break` 的组合。先学走稳，再学跑酷。

**Q4："循环变量在 while 里面声明不行吗？"**

A：可以在循环体里声明，但每次循环都会重新创建和销毁。更致命的是：循环条件在循环体外判断，循环体内声明的变量在条件里无法访问。所以 while 的循环控制变量必须在 while 之外声明——这和 for 是不同的。

## 📋 本章小测（15 分钟，共 6 题）

**1. 填空题**（2 分）
while 循环的特点是"先________后________"；do-while 循环的特点是"先________后________"。

**2. 选择题**（2 分）
以下哪种场景最适合使用 do-while？
A. 打印 1 到 100　　B. 输入密码直到正确　　C. 至少显示一次菜单再让用户选择　　D. 求 1 到 n 的和

**3. 判断输出题**（3 分）
写出以下代码的输出：
```cpp
int x = 1, s = 0;
while (s < 10) {
    s += x;
    x++;
}
cout << x;
```

**4. 改错题**（2 分）
以下代码有什么问题？
```cpp
int n = 10;
while (n > 0)
    cout << n << " ";
```

**5. 编程题**（4 分）
输入一个正整数，用 while 循环将它反转输出。（如输入 1234，输出 4321）

**6. 分析题**（2 分）
while 和 do-while 能否完全互相替代？如果能，请说明如何替代；如果不能，举出反例。

---

**参考答案：**

1. 先**判断**后**执行**；先**执行**后**判断**。
2. **C**。do-while 确保菜单至少显示一次。A 更适合 for，B 不一定至少执行一次（密码可能第一次就正确），D 也适合 for。
3. **6**。推演过程：x=1, s=0→s=1, x=2; s=1<10→s=3, x=3; s=3→s=6, x=4; s=6→s=10, x=5; s=10<10? 假，退出。此时 x=5，输出 x，即 **5**。（注：x 最后被 ++ 到 5 但没进循环体，最终输出 5）
4. 问题：循环体内没有改变 n 的值——`n` 永远是 10，条件永远为真，死循环。应加 `n--;` 或 `n -= 1;`。
5. 参考答案：
```cpp
int n, rev = 0;
cin >> n;
while (n > 0) {
    rev = rev * 10 + n % 10;
    n /= 10;
}
cout << rev;
```
评分：循环条件正确 1 分，数位分离正确 1 分，反转算法正确 2 分。
6. 不能完全互相替代。while 可以模拟 do-while：先执行一次循环体，再用 while 判断条件。但 do-while 不能模拟"循环体执行 0 次"的 while——因为 do-while 至少执行一次。反例：`while (false) { ... }` 循环体不执行，但 `do { ... } while (false);` 循环体会执行一次。

## 🎮 不插电活动

### 活动：密码守卫（第 5 课时用）

**道具**：黑板、粉笔、一张预先写好"密码"的卡片（密码是一个 3 位数，如 527）

**角色设置**：
- 1 名学生当"守卫"（背对黑板，不能看到密码）
- 1 名学生当"系统"（在黑板上写下密码 527，只有自己和全班其他同学能看到）
- 其余学生当"提示者"

**游戏流程**：

**第一轮——while 模式（先判断后执行）**：
1. "守卫"站在讲台边，面对全班，背对黑板
2. "系统"在黑板上写下密码（如 527）
3. 全班同学不能直接告诉守卫密码，只能通过举手来回答守卫的问题
4. 守卫开始猜测：
   - 守卫："密码是 500 吗？"
   - 全班回应："**太小了！**"（大家齐声喊）
   - 守卫："密码是 800 吗？"
   - 全班："**太大了！**"
   - 守卫继续调整猜测……直到猜中 527
5. 每次猜测对应 while 循环的一次迭代——猜不中就继续（条件为真），猜中则退出（条件为假）
6. 老师在黑板记录猜测次数

**第二轮——二分查找优化**：
- 告诉守卫："你每次猜中间值，500→750→625→562→531→515→523→527，只需要 7 次左右就能猜中 512 个可能的数"
- 对应代码：while 循环 + 二分策略，体现"好的算法能大幅减少循环次数"

**复盘环节**：
- 在黑板上写出对应的 while 代码：
```
密码 = 527;
猜测 = 0;
while (猜测 != 密码) {
    获取新的猜测();
    if (猜测 > 密码) 喊"太大了";
    else if (猜测 < 密码) 喊"太小了";
}
喊"猜对了！";
```
- 提问引导：
  ①"循环的退出条件是什么？"（猜测 == 密码）
  ②"为什么这里用 while 而不是 for？"（不知道要猜多少次）
  ③"如果守卫一开口就猜 527，循环体执行了几次？"（0 次——这就是 while 的"可能一次都不执行"）
  ④"二分查找时，从 1-1000 最多需要猜多少次？"（约 10 次，因为 2^10 = 1024 > 1000）

**变体玩法——do-while 模式**：
- 改成：守卫必须先猜一次，然后全班给出反馈，再继续猜
- 对应 do-while：`do { 猜测(); 反馈(); } while (猜测 != 密码);`

**教学要点**：
- 整个游戏不需要电脑，但完美展示了 while 循环的核心：不确定次数、有明确退出条件、每次循环后向目标靠近
- 二分查找的引入让学生初步感受"算法效率"——这是信息学竞赛的核心概念之一
- 通过对比 brute-force（一个一个猜）和二分策略，埋下"算法优化"的种子

## 📦 补充题库

1. 用 while 实现辗转相除法求最大公约数。（参考答案：`while(b != 0) { int temp = a % b; a = b; b = temp; } cout << a;`——欧几里得算法的 while 实现，比 for 更自然。）

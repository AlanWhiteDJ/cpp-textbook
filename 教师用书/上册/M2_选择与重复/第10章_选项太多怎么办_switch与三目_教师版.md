## 🔥 重点难点

### 教学重点

1. **switch 的适用边界**：学生最容易犯的错误是把 switch 当成 if-else 的"另一种写法"。必须反复强调 switch 只适合等值多分支。建立判断力——"这个是范围问题，不能用 switch。"
2. **case 穿透机制**：这是 switch 最不直观的特性。解释清楚 case 只是"入口标签"这一点后，学生才能真正理解为什么没有 break 会一直往下走。
3. **三目运算符的可读性边界**：三目好用但容易被滥用。教学生在"一行内能看懂"的前提下才用三目。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 忘记写 break | 程序输出超出预期，但学生找不到 bug | 让学生逐行加注释标注"走到这里会干什么"，用纸笔推演 switch 执行流程——一旦亲手 trace 过穿透，就不会再忘 |
| switch 里写范围条件 | case x > 5: 编译报错 | 画出 if-else vs switch 的"地盘图"：数轴上等点是 switch 的地盘，区间是 if-else 的地盘。两者不可互相侵略 |
| 三目运算嵌套 | 写出 `a>b?c>d?e:f:g` 这种谁也看不懂的代码 | 把这种代码投影到屏幕上，让全班猜它的作用——没有人猜对。结论：你写出来别人看不懂的代码就是糟糕的代码 |

## 💬 常见问答

**Q1："switch 和 else-if 本质上有区别吗？"**

A：语法上有区别，底层上编译器可能会为 switch 生成更高效的跳转表（尤其是 case 值连续的时候）。但对你现在来说，记住这个选择原则就够了：判断等值用 switch（更清晰），判断范围用 else-if（更灵活）。

**Q2："case 穿透听起来像个 bug，为什么 C++ 要这样设计？"**

A：C++ 继承了 C 语言的这个特性。它让你能把多个 case 合并在一起，减少重复代码。比如 `case 1: case 3: case 5: cout << "奇数"; break;` 比写三段相同的代码优雅得多。设计者把"穿透"作为默认行为，用 break 主动停止——这是一种"默认直行，主动刹车"的设计哲学。

**Q3："三目运算符是不是就是为了少写几行代码？没必要吧？"**

A：不只是少写代码的问题。在需要"根据条件选择两个值之一"的场景下，三目表达的是"求值"而非"执行语句"。比如在 cout 里：`cout << (score >= 60 ? "及格" : "不及格");` 用 if-else 做不到这么简洁。选对工具，代码更清晰。

**Q4："default 可以不写吗？"**

A：语法上可以不写，但工程上必须写。就算你"确定"输入一定在 case 范围内，也要写一个 default 输出 "未知输入"。这是防御性编程的基本素养——你今天确定的输入范围，明天就可能被改代码的人打破。

## 📋 本章小测（15 分钟，共 6 题）

**1. 选择题**（2 分）
以下哪类问题最适合用 switch 解决？
A. 判断分数是否及格　B. 根据星期几数字输出中文名　C. 比较三个数的大小　D. 判断闰年

**2. 填空题**（2 分）
写出下面代码的完整输出：
```cpp
int x = 2;
switch(x) {
    case 1: cout << "A";
    case 2: cout << "B";
    case 3: cout << "C"; break;
    default: cout << "D";
}
```

**3. 判断题**（2 分）
switch 的判断表达式可以是 `float` 类型。（　　）

**4. 填空题**（2 分）
用三目运算符改写：若 a > b 则 max = a，否则 max = b。
`max = ________;`

**5. 编程题**（4 分）
写一个 switch 程序：输入 1-12 的月份数字，输出该月所属季节（12/1/2 冬季，3/4/5 春季，6/7/8 夏季，9/10/11 秋季）。利用 case 穿透减少重复代码。

**6. 改错题**（3 分）
下面的代码有什么问题？
```cpp
switch(score) {
    case score >= 90: cout << "A"; break;
    case score >= 80: cout << "B"; break;
}
```

---

**参考答案：**

1. **B**。星期数字是等值判断（1-7），天然适合 switch。其他选项都是范围判断。
2. **BC**。x=2，从 case 2 入口进入，输出 B，因为没有 break 继续穿透到 case 3 输出 C，然后遇到 break 停止。default 不会执行（已经有 case 匹配）。
3. **×**。switch 只支持整数类型（int、char、枚举等），不支持浮点数。
4. `max = (a > b) ? a : b;`
5. 参考答案：
```cpp
int month; cin >> month;
switch(month) {
    case 12: case 1: case 2: cout << "冬季"; break;
    case 3: case 4: case 5: cout << "春季"; break;
    case 6: case 7: case 8: cout << "夏季"; break;
    case 9: case 10: case 11: cout << "秋季"; break;
    default: cout << "请输入1-12"; break;
}
```
评分：case 穿透正确 2 分，default 处理 1 分，语法正确 1 分。
6. 问题：case 后面必须是**常量表达式**，不能是 `score >= 90` 这样的条件表达式。这是范围判断，应该用 if-else 而不是 switch。另外 `switch(score)` 中的 score 如果是浮点数也会有问题。

## 🎮 不插电活动

### 活动：自动售货机角色扮演（第 5 课时用）

**道具**：
- 一个纸箱（当售货机机身）
- 6 张卡片，每张正面写商品编号（01-06），背面写商品名称和价格（如 01-可乐 3元、02-雪碧 3元、03-矿泉水 2元、04-果汁 4元、05-牛奶 5元、06-巧克力 6元）
- 若干"模拟纸币"（用裁好的纸片代替）
- 粉笔在黑板上写"菜单"

**角色分配**：
- 1 名学生扮演"售货机"（站在纸箱后面）
- 1 名学生扮演"顾客"
- 1 名学生扮演"程序员"（负责在黑板上写 switch 代码）

**游戏流程**：

**第一轮（简单模式）**：
1. 顾客走到售货机前，报出一个编号（如 "03"）
2. 售货机从纸箱中找到对应卡片，报出：货品名称 + 价格，递出卡片
3. 顾客递上对应的"纸币"
4. "程序员"在黑板上书写对应的 case 分支：`case 3: cout << "矿泉水 2元"; break;`
5. 重复 3-4 次，直到黑板上形成一个完整的 switch 结构

**第二轮（错误输入处理）**：
1. 顾客故意报一个不存在的编号（如 "09"）
2. 售货机应该回答"没有这个商品，请重新选择"——这就是 default 的作用
3. "程序员"补充 `default: cout << "无此商品"; break;`

**第三轮（穿透合并）**：
1. 设置"打折规则"：编号 01-03 的都是"特价饮料 2元"
2. 售货机必须修改行为：01/02/03 三个编号输出相同结果
3. "程序员"用 case 穿透重写：`case 1: case 2: case 3: cout << "特价饮料 2元"; break;`

**教学要点**：
- 游戏结束后，让"售货机"分享感受：当顾客报出编号时，你是怎样决策的？引出认知——switch 就是"查表匹配"，跟你翻菜单卡一模一样。
- 对比：如果售货机卖的不是 6 种而是 100 种商品，用 if-else 写会疯掉，用 switch 就清爽得多。
- 强调 case 后面的数字就是卡片编号——必须精确匹配，不能是"比 3 大的"这种模糊描述。
- 总结板书：switch 就是自动售货机的灵魂——按下几号键，输出什么东西，没有对应按钮就报错。

## ✅ 习题详解

### 基础题

**1. 语法填空补充完整 switch 结构**
`switch(day) { case 1: ... break; case 2: ... break; default: ... }`

**2. 判断输出**
（1）输出 "二三"——case 2 入口，穿透 case 3（输出"三"），遇到 break 停止
（2）输出 "未知"——5 不匹配任何 case，走 default

### 提高题

**3. 三目运算符改写**
（1）`result = (n % 2 == 0) ? "偶数" : "奇数";`
（2）`min = (a < b) ? a : b;`
（3）`cout << ((score >= 60) ? "及格" : "不及格");`

**4. 月份天数 switch 实现**
评分要点：31 天月份穿透合并正确，30 天穿透合并正确，2 月单独处理，default 处理非法月份。

### 挑战题

**5. 三目嵌套求最大数**
`int max = (a > b) ? ((a > c) ? a : c) : ((b > c) ? b : c);`
点评：虽然正确，但更推荐分步写或用 if-else——可读性比简洁更重要。

## 📦 补充题库

1. 解释为什么 switch 不能判断 string 类型。（参考答案：C++ 的 switch 底层用跳转表实现，需要 case 值在编译时确定且能转化为整数偏移。string 是复杂对象，无法直接映射为整数跳转表。如果要判断字符串，必须用 if-else + ==。）

2. 用 switch 实现猜拳游戏：用户输入 0（石头）/1（剪刀）/2（布），电脑随机出拳，判断输赢。（参考答案思路：`(player - computer + 3) % 3` 的结果 0 平局、1 赢、2 输，然后用 switch 判断这个差值。）

3. case 标签后面如果不写任何语句直接写下一个 case，这种行为叫什么？有什么实际用途？（参考答案：叫 case 穿透/fall-through。用于多个 case 共享同一段代码块，减少重复。）

4. 小丽写的代码：`switch(x) { case 1: int y = 5; break; case 2: y = 10; break; }` 第二句报错说 y 未定义，为什么？（参考答案：case 标签内部声明的变量作用域是整个 switch 块，但初始化只发生在该 case 执行路径上。编译器无法保证 y 一定被初始化。解决方法：在 case 内用 `{}` 包裹变量声明，或把变量声明放到 switch 之前。）

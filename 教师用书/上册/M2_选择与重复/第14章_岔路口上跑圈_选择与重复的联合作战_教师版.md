## 🔥 重点难点

### 教学重点

1. **选择与循环的层次关系**：在混用两种结构时，明确"谁在外面谁在里面"是正确性的关键。外层决定"做什么的顺序"，内层决定"每个步骤的细节"。
2. **经典算法的"模板化"思维**：质数判断、完数、水仙花数不是让学生背代码，而是提取其中的通用模式——外层遍历+内层筛选+条件输出。掌握了这个模板，以后遇到"回文数""自守数""亲密数"都能独立解决。
3. **程序的防御性思维**：除法判 0、输入验证、边界处理——这些都是"选择+循环"的综合体现，也是从"会写代码"到"会写可靠的代码"的质变。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| 混用时搞不清执行顺序 | 分支在循环内 vs 循环在分支内，学生分不清谁先谁后 | 强制画流程图——先画最外层的结构框，再往里面填。流程图比代码更容易看出层次关系 |
| 综合题无从下手 | 看"猜数字"完整代码觉得头大 | 分步教学：第一步只写随机+while 框架 → 第二步加反馈 → 第三步加次数统计。让学生看到程序是"长"出来的，不是一下子写完的 |
| 变量生命周期混乱 | 在循环内声明变量、在分支内修改，出了作用域找不到 | 画"变量生命周期图"——一个时间轴标注每个变量什么时候诞生、什么时候消亡。让学生直观感受作用域 |

## 💬 常见问答

**Q1："学完 M2，我能写出什么级别的程序？"**

A：能写出任何"计算+判断"类的程序。比如计算器、成绩管理系统、猜数字游戏、质数查找器、大量数据的统计分析……信息学竞赛的入门题你已经可以做了。你不是在学"理论"，你是在积累真正的编程能力。

**Q2："质数/完数/水仙花数，这些数学题跟编程有什么关系？"**

A：编程 ≠ 数学，但数学题是训练编程思维最好的素材。为什么？因为数学题有明确的输入、明确的规则、明确的输出——这恰好是一个完整的 IPO 模型。用数学题练习编程，你可以 100% 专注于逻辑设计，而不会被"这个需求是什么意思"分心。

**Q3："为什么有些循环用 for，有些用 while？我已经有点乱了。"**

A：看"循环变量"。如果你能明确写出"从哪开始、到哪结束、每次怎么变"这三个要素，就用 for。如果你只知道"什么时候停"，就用 while。如果你确定"至少做一次"，用 do-while。这不是语法规则，是代码可读性的选择——让你三个月后回头看自己的代码还能秒懂。

**Q4："学了这么多，我觉得最难的还是找 bug，怎么办？"**

A：找 bug 是编程的"另一半时间"——专业程序员 50% 的时间在调试。三个锦囊：①先假设最简单的可能原因（别一上来就想"是不是编译器坏了"）；②用 cout 在关键位置打印变量的值（"用数据说话"）；③如果 5 分钟还没找到，站起来走一圈再回来看（"换双眼睛"）。debug 不是智商问题，是耐心问题。

## 📋 本章小测（15 分钟，共 6 题）

**1. 填空题**（2 分）
程序的三种基本控制结构是：________、________、________。

**2. 选择题**（2 分）
以下哪种组合最常用于"统计一组数据中满足条件的个数"？
A. 循环内嵌循环　　B. 分支内嵌分支　　C. 循环内嵌分支　　D. 分支内嵌循环

**3. 判断输出题**（3 分）
写出以下代码的输出：
```cpp
int cnt = 0;
for (int i=1; i<=20; i++) {
    if (i % 2 == 0 && i % 3 == 0) cnt++;
}
cout << cnt;
```

**4. 编程题**（4 分）
输入 n，用"循环+分支"判断 n 是否为质数，并输出结果。

**5. 分析题**（3 分）
以下"猜数字"代码存在什么问题？至少说出两个：
```cpp
int target = rand() % 100 + 1, guess;
while (guess != target) {
    cout << "猜: ";
    cin >> guess;
    cout << (guess > target ? "大了" : "小了") << endl;
}
```

**6. 流程图题**（3 分）
画出"输入 5 个数字，找出其中最大值"的流程图。（提示：需要循环+分支）

---

**参考答案：**

1. **顺序、选择（分支）、循环（重复）**
2. **C**。遍历数据是循环，判断是否满足条件是分支——"循环内嵌分支"是最典型的统计模式。
3. **3**。1-20 中同时被 2 和 3 整除的数（即被 6 整除）：6、12、18，共 3 个。
4. 参考答案：
```cpp
int n;
cin >> n;
bool isPrime = true;
if (n < 2) isPrime = false;
for (int i=2; i*i <= n; i++) {
    if (n % i == 0) {
        isPrime = false;
        break;
    }
}
cout << (isPrime ? "质数" : "不是质数");
```
评分：循环范围正确 1 分，整除判断正确 1 分，break 优化 1 分，输出正确 1 分。
5. 问题：
① guess 未初始化——while 条件 `guess != target` 在第一次循环前就检查 guess，但 guess 是未定义的（垃圾值），可能导致循环不该进却进了或该进却不进；
② 反馈逻辑完全反了——`guess > target` 时应该输出"大了"，但这里用三目输出的是"大了→大了""小了→小了"，逻辑是正确但描述有误导。真正的问题是：猜的数字等于 target 时，三目仍会输出"小了"（因为 `guess > target` 为假），而 while 条件 `guess != target` 为假退出——用户看不到"猜对了"的提示！
③ 缺少随机种子 `srand(time(0))`——每次运行随机数列相同，游戏失去随机性。
（评分：说出 2 个合理问题各 1.5 分。）
6. 评分：有循环框 1 分，有分支判断（菱形）1 分，流程方向正确 1 分。

## ✅ 习题详解

### 基础题

**1. 统计 1-100 中 3 和 5 的公倍数个数**
```cpp
int cnt = 0;
for (int i=1; i<=100; i++) {
    if (i % 3 == 0 && i % 5 == 0) cnt++;
}
cout << cnt;  // 结果: 6 (15,30,45,60,75,90)
```

**2. 判断闰年（循环输入多组年份，输入 0 退出）**
```cpp
int year;
while (cin >> year && year != 0) {
    if ((year % 4 == 0 && year % 100 != 0) || year % 400 == 0)
        cout << "闰年" << endl;
    else
        cout << "平年" << endl;
}
```

## 📦 补充题库

1. 输入一个整数 n，用"循环+分支"输出它的所有因子。若因子之和等于 n，输出 "完数"；否则输出"不是完数"。（参考答案：外层循环找因子并累加，循环后用 if 比较和与 n。）

2. 角谷猜想：输入正整数 n，如果 n 为偶数则 n/=2，奇数则 n=3*n+1，最终 n 会变成 1。输出变化过程中出现的最大数。（参考答案：用 while(n!=1) 循环，每次记录 max 值。）

3. 求 100-999 中满足"百位的立方 + 十位的立方 + 个位的立方 = 这个数本身"的所有数（水仙花数）的个数和总和。（参考答案：4 个水仙花数——153、370、371、407。总和 1301。）

4. 设计一个程序：输入 10 个人的年龄，统计未成年人（<18）、青年（18-45）、中年（46-65）、老年（>65）各多少人。（参考答案：循环+多分支 else-if。这是"数据分类统计"的标准模板。）

5. 小明写了一个"找完数"的程序但运行很慢。有哪些方法可以优化？（参考答案：①内层循环只需要找到 n/2 而不是 n；②内层循环可以只到 sqrt(n)，每找到一个因子 i，同时得到 n/i；③用数组存储已找到的完数避免重复计算。此题引出"算法优化"概念，为后续学习铺垫。）

## 🏁 M2 模块教学总结

M2「选择与重复」是 C++ 入门阶段最核心的模块。通过 6 章教学，学生应达到以下里程碑：

1. **能读代码**：看到任何包含 if/switch/for/while 的代码，能用手推演的方式说出执行结果
2. **能写逻辑**：能将"如果……就……否则""重复……直到……"等自然语言转化为正确的代码
3. **能选工具**：分得清什么时候用 if/switch/三目，什么时候用 for/while/do-while
4. **能排错**：见到死循环知道加 cout 观察变量，见到奇怪的输出知道 trace 条件分支

M2 结束后，学生已经掌握了程序设计的三大基本结构，能够独立完成信息学竞赛入门难度的题目。M3「数据的海洋」将引入数组和字符串，让程序处理数据的能力从"单个变量"升级到"批量数据"——届时，M2 学到的循环将成为遍历数组的核心武器。

**教学寄语**：选结构是骨骼，循环是肌肉，数据是血液。M2 打完骨骼和肌肉的基础，M3 注入血液——整个程序就活起来了。

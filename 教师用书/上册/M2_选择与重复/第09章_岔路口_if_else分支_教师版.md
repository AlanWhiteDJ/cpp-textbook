## 🔥 重点难点

### 教学重点

1. **if-else-if 的条件顺序**：else-if 是不等价的"阶梯判断"，条件顺序直接影响结果。强调前面的条件会"过滤"掉后面的情况。
2. **== 与 = 的区分**：这可能是学生编程生涯中最常见也最隐蔽的 bug。教他们一个检查口诀："判断真假用双等，赋值数值用单等。"
3. **短路求值**：不仅是语言特性，更是安全编程的基础。让学生理解 `a != 0 && b / a > 5` 为什么不报错。

### 常见卡点 + 突破策略

| 卡点 | 表现 | 突破方法 |
|------|------|----------|
| if 后面多写分号 | `if (x > 0); cout << "正";` 导致不管条件真假都输出 | IDE 自动缩进演示——多分号后 cout 与 if 不对齐，视觉上立刻发现问题 |
| else-if 条件冗余 | 写 `score >= 80 && score < 90` 而不是 `score >= 80` | 板书阶梯图：画一个数轴，标注每个区间。学生一看就明白已经过滤过 ≥90 的情况 |
| 混淆 && 和 \|\| | 判断"60 到 80 之间"写成 `score > 60 \|\| score < 80` | 口诀：与（&&）是"并且"——两个都得满足；或（\|\|）是"或者"——满足一个就行。判断区间用 && |

## 💬 常见问答

**Q1："为什么判断相等要用两个等号？"**

A：因为一个等号已经被"赋值"占用了。C++ 的设计者希望你去区分这两种操作——赋值是把一个值"放入"变量，判断相等是问"这两个一样吗"。两个等号就像是在说"等于等于"，双重强调你在比较而不是赋值。

**Q2："什么时候用 if，什么时候用 else-if？"**

A：如果两个条件是互斥的（只会有一个发生），用 else-if——效率更高，也更能表达"多选一"的逻辑。如果两个条件完全独立（可能同时发生），必须用两个独立的 if。比如：`if (身高 > 180)` 和 `if (体重 > 80)` 是独立的两件事，不能用 else-if。

**Q3："‘短路求值’是 bug 还是 feature？"**

A：是 feature（特性）。短路求值是故意设计的，用来写更安全的代码。比如 `if (ptr != nullptr && ptr->value > 0)`——如果指针为空，后半句不会执行，避免了访问空指针的崩溃。

**Q4："我写 if 的时候到底要不要加大括号？"**

A：永远加。即使只有一行代码也加。理由：①以后你可能在这行下面加第二行，不加括号就掉坑里了；②加了括号意图更清晰；③很多大公司的代码规范强制要求加括号。养成习惯，受益终身。

## 📋 本章小测（15 分钟，共 6 题）

**1. 填空题**（每题 2 分）
写出判断"变量 a 不等于 10"的 C++ 表达式：________

**2. 选择题**（2 分）
以下代码输出什么？
```cpp
int x = 3;
if (x = 5)  cout << "A";
else        cout << "B";
```
A. A　　B. B　　C. 编译错误　　D. 运行错误

**3. 判断题**（2 分）
在 else-if 链中，如果多个条件同时满足，只会执行第一个满足条件的分支。（　　）

**4. 填空题**（每题 2 分）
逻辑表达式 `false && (5 / 0 == 0)` 的值是________，这体现了________特性。

**5. 编程题**（4 分）
输入一个年份，判断是否为闰年。（写出完整代码）

**6. 流程图题**（3 分）
画出"输入年龄，判断能否考驾照（≥18 岁可以，否则不行）"的流程图。

---

**参考答案：**

1. `a != 10`
2. **A**。因为 `x = 5` 是赋值表达式，值为 5（非 0 即真），所以 if 条件为真，输出 A。这是典型的 == 误写为 = 的坑。
3. ✅ 正确。else-if 是一种"互斥链"——前面的分支匹配后，后面的条件根本不会被判断。
4. **false**，**短路求值**。因为 && 左边为 false，右边不会执行，避免了除以 0 的错误。
5. 参考答案：
```cpp
int year;
cin >> year;
if ((year % 4 == 0 && year % 100 != 0) || year % 400 == 0)
    cout << "闰年";
else
    cout << "平年";
```
评分：逻辑正确 2 分，语法无误 2 分。
6. 评分：有菱形判断框 1 分，条件标注正确 1 分，两条分支都有输出框 1 分。

## 🎮 不插电活动

### 活动：红绿灯游戏（第 5 课时用）

**道具**：三张 A4 纸（分别写"红灯""黄灯""绿灯"）、两根粉笔在地上画出起点线和过道区

**规则**：
1. 全班同学站成一排在"马路起点线"后面
2. 老师随机举起一张纸（红灯/黄灯/绿灯），学生根据看到的颜色做动作
3. 规则映射：
   - **绿灯**：向前迈一步（if 条件为真→执行动作）
   - **红灯**：原地不动（if 条件为假→不执行）
   - **黄灯**：原地踏步但不过线（else-if 扩展分支）
4. 如果有学生在红灯时前进，自动淘汰到一旁做"监督员"
5. 老师可以玩心理战术：快速切换牌子、假装要举又收回去（锻炼学生对条件的快速判断能力）

**进阶玩法（引入 else-if）**：
- 老师手中牌子升级为四种颜色对应四种动作：
  - 红→停、绿→走一步、黄→拍手+停、蓝→转身向后
- 用流程图事前板书对应关系：`if (红) 停; else if (绿) 走; else if (黄) 拍; else if (蓝) 转;`
- 游戏结束后回到座位，让学生自己把刚才的"规则表"翻译成 C++ 的 if-else-if 结构

**教学要点**：
- 游戏结束后提问："如果老师同时举两张牌子，该怎么办？"——引出条件互斥性的讨论（好的程序设计应该让条件互斥）
- 再问："如果举了红牌，你们还会去看黄牌绿牌吗？"——引出 else-if 的短路特性：前面的条件匹配后，后面的不再判断
- 总结板书：`if (条件) { 动作 }` 就是红绿灯规则——条件满足就执行，不满足就跳过。
- 让学生体会到：他们的大脑一直在做 if-else 判断，只是以前没有意识到。编程就是把这种"下意识判断"显式地写出来。

## ✅ 习题详解

### 基础题

**1. 关系运算符填空**
（1）`>` （2）`!=` （3）`==` （4）`>=` （5）`<=`

**2. 判断输出**
（1）输出 "B"——因为 `a == 10` 为假
（2）输出 "偶数"——`6 % 2 == 0` 为真
（3）输出 "A"——`(a = 5)` 赋值表达式值为 5，非 0 为真

### 提高题

**3. 逻辑表达式求值**
（1）`true`（2）`true`（第一个条件满足，第二个表达式不执行）（3）`false`（4）`false`（!true 为 false，&& 短路）

**4. else-if 重写**
参考答案：先判断 a 最大、再判断 b 最大、else c 最大。关键：不要写成三个独立 if，要用 else-if 体现互斥性。

### 挑战题

**5. 出租车计费（多种方案）**
基础方案（if-else）：三种情况，if 距离 ≤ 3 / else if ≤ 10 / else
优化方案：用 else-if 链，注意分段计费——超过 10 公里的部分是 3 元/公里，不是全程 3 元。

**评分标准**：正确分三种情况 2 分；else-if 结构正确 1 分；边界值处理正确（等于号）1 分；优化方案加 1 分。

## 📦 补充题库

1. 写出判断"字符 ch 是大写字母"的条件表达式。（参考答案：`ch >= 'A' && ch <= 'Z'`。因为字符在 C++ 中本质是整数，可以直接比较。）

2. 为什么 `if (100 <= score <= 90)` 在数学上不成立，在 C++ 中却能通过编译？（参考答案：C++ 从左向右结合。先算 `100 <= score` 得到 0 或 1，再拿这个 0 或 1 与 90 比较——永远为真！所以这样写是致命的逻辑错误。）

3. 用逻辑运算符简化以下代码（不用 if）：根据布尔变量 `isWeekend` 和 `isSunny` 决定是否去公园。（参考答案：`bool goPark = isWeekend && isSunny;`——有时候不需要 if，直接赋值布尔表达式更简洁。）

4. 三个数 a、b、c，用 else-if 找出中间值（不是最大也不是最小）。（参考答案思路：`if ((a > b && a < c) || (a < b && a > c)) mid = a;`，此类推。）

5. 小明写了 `if (x > 0) if (y > 0) cout << "都是正的"; else cout << "x不是正的";`，他的 else 和谁配对？（参考答案：else 和最近的 if 配对，即 `if (y > 0)`。所以这个 else 永远不可能输出"x不是正的"。要解决这个问题，两个 if 都要加大括号。）

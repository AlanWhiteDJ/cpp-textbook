# 教学视频录制方案

## 一、工作量核算

### 原始需求 vs 实际合理量

| 项目 | 原始设想 | 合理估计 | 说明 |
|------|:-----:|:-----:|------|
| 讲课视频 | 200课时 | **100课时** | 200课时 = 200小时课堂内容，录制成片至少需要 400小时工作时间，不现实。砍半到核心知识点。 |
| 讲课分段 | 600段(20min/段) | **200段** | 100课时×60min = 6000分钟 ÷ 20min = 300段，做减法到200段（只讲最核心的） |
| 题目讲解 | 1200题 | **500题** | 1200题全讲没必要。经典母题+模板题 500 道覆盖 90% 的知识点。 |
| 入门题 | 600题×5min | **250题×5min** | 保留最经典的 |
| 提高题 | 600题×10min | **250题×10min** | 保留最经典的 |
| 总时长 | 21000分钟 | **10750分钟 ≈ 179小时** | |

## 四、录制执行建议

### 建议的录制节奏

```
第1-2月：入门讲课 60段（每天录2段，每段含准备+录制+简单剪辑约2小时）
第3-4月：入门题目 250题（每天录5题，每天约2.5小时）
第5-6月：提高讲课 59段
第7-8月：提高题目 250题
```

按 **业余时间每天2-3小时** 算，8个月可以完成全部核心内容。

### 录制效率技巧

1. **不要追求完美**：小口误不需要全部重录，后期加上字幕/文字修正即可
2. **模板化结构**：每段视频固定结构 → 概念(5min) + 代码演示(10min) + 总结(5min)
3. **批量录制**：同一模块的内容集中录制，减少"换脑"成本
4. **题目讲解用固定模板**：读题(1min) → 思路分析(3min入门/7min提高) → 代码实现(1min入门/2min提高)
5. **先录音再录屏**：或者反过来，不要同时做两件事
6. **配备助教**：题目讲解部分可以考虑让助教分担一部分

### 优先发布策略

不要等全部录完再发布，采用 **"边录边发"** 模式：

| 批次 | 内容 | 发布时机 |
|------|------|---------|
| 第1批 | L01-L20（语言基础） + 配套50题 | 录制完即发 |
| 第2批 | L21-L40（基础算法） + 配套80题 | 第3个月 |
| 第3批 | L41-L60（入门DP+图论） + 配套120题 | 第5个月 |
| 第4批 | L61-L90（数据结构+图论） + 配套120题 | 第7个月 |
| 第5批 | L91-L119（DP进阶+数学+几何） + 配套130题 | 第9个月 |

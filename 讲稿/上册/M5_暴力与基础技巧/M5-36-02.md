# 第36章 第2课时：lower_bound 与 unique——离散化的两个核心工具
**时长**：15-20分钟 | **对应教材**：第36章 36.3

## 🎯 本课时目标
- 深入理解 `unique` 的工作原理和返回值
- 掌握 `lower_bound` 的语义：第一个 ≥ x 的位置
- 能独立写出完整的离散化函数

## 📝 讲稿正文

上节课我们学了离散化三步走，这节课深入理解其中两个关键函数——`unique` 和 `lower_bound`。

先说 `unique`。它干了什么？**把相邻重复的元素"挤"到末尾**，返回指向"第一个重复垃圾"的迭代器。注意——它只移除**相邻**重复，这就是为什么必须先 `sort`！

举个例子：数组 `{3, 5, 5, 100, 200}` 经过 `unique` 变成 `{3, 5, 100, 200, ?}`，返回指向位置 4（即 `?` 的位置）。不重复元素个数 = 返回位置 - 数组起始 = 4。

代码中：`m = unique(b + 1, b + n + 1) - (b + 1);`

`(b + 1)` 是数组起始（从下标 1 开始），`unique(...)` 返回去重后的"新末尾"，两者相减得到不重复元素个数。⚠️ `unique` 不会真正删除元素——它只是把不重复的挪到前面，后面的"垃圾"还在。所以要用返回值截断。

再说 `lower_bound`。它返回**第一个 ≥ x 的位置**的迭代器。对于离散化，我们要找的是 x 在去重数组 `b` 中的下标。

```cpp
int b[] = {0, 3, 5, 100, 200};  // b[0] 不用，有效下标 1~4
// lower_bound(b+1, b+5, 5)    → 指向 b[2]（值=5），减 b 得 2
// lower_bound(b+1, b+5, 7)    → 指向 b[3]（值=100，第一个≥7），减 b 得 3
// lower_bound(b+1, b+5, 300)  → 指向 b[5]（end，没找到），减 b 得 5
```

由于离散化时 x 一定来自原始数据，一定在 b 的某个位置，`lower_bound` 总能精准命中。返回的迭代器减去数组首地址 `b`，就得到 1-based 下标。

⚠️ 常见错误：忘记 `-b`！

```cpp
// ❌ 错误
int id = lower_bound(b+1, b+m+1, x);  // 返回的是迭代器（地址），不是下标！

// ✅ 正确
int id = lower_bound(b+1, b+m+1, x) - b;  // 减去 b 得到整数下标
```

现在把两个函数串起来，写一个完整的离散化+查询程序：

```cpp
int n, a[100005], b[100005], m;

void init() {
    for (int i = 1; i <= n; i++) b[i] = a[i];
    sort(b + 1, b + n + 1);
    m = unique(b + 1, b + n + 1) - (b + 1);
}

int getID(int x) {
    return lower_bound(b + 1, b + m + 1, x) - b;
}
```

这套代码要练到可以**默写**的程度——因为它是离散化、坐标压缩、甚至很多数据结构题的基础代码。

## 🖥️ 板书/屏幕要点
- **unique**：相邻去重 → 返回新末尾 → `m = unique(b+1,b+n+1)-(b+1)`
- **lower_bound**：第一个 ≥ x 的位置 → `lower_bound(b+1,b+m+1,x) - b`
- **必须 sort 再 unique**：unique 只去相邻重复
- **常见错误**：忘记 `-b` 导致 id 是地址不是下标

## ✏️ 随堂练习
手工模拟：数组 `{8, 3, 8, 2, 5, 3}` 经过 sort 和 unique 后 b 变成什么？m 是多少？2 和 8 离散化后下标各是多少？

## 📌 小结
`unique` 去相邻重复，必须先 sort。`lower_bound` 返回第一个 ≥ x 的位置，要减 b 才得到整数下标。这两个函数是离散化的核心，要能默写。

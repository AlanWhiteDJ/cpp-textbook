# 第30章 第2课时：缩小范围——枚举优化的核心心法
**时长**：15-20分钟 | **对应教材**：第30章 30.3

## 🎯 本课时目标
- 理解枚举优化的核心思路：缩小枚举范围、减少变量
- 掌握"能用推导的变量不枚举"原则
- 独立写出百钱百鸡从三重到两重的优化过程

## 📝 讲稿正文

上节课我们说了，枚举最大的敌人是次数爆炸。那怎么打败它？两个字——**缩小**。

缩小什么？缩小**范围**，缩小**变量个数**。

我们来看一道经典题目——百钱买百鸡。题目是这样的：公鸡一只 5 文钱，母鸡一只 3 文钱，小鸡三只 1 文钱。现在有 100 文钱，要买正好 100 只鸡，问公鸡、母鸡、小鸡各几只？

（停顿，让学生想一下怎么枚举）

最直接的想法是什么？三重循环！

```cpp
for (int gong = 0; gong <= 100; gong++) {
    for (int mu = 0; mu <= 100; mu++) {
        for (int xiao = 0; xiao <= 100; xiao++) {
            if (gong + mu + xiao == 100 &&
                5 * gong + 3 * mu + xiao / 3 == 100 &&
                xiao % 3 == 0) {
                cout << "公鸡 " << gong << " 母鸡 " << mu << " 小鸡 " << xiao << endl;
            }
        }
    }
}
```

这个程序跑多少次循环？101×101×101 ≈ 103 万次。能跑，但太浪费了。

你们觉得哪里可以优化？（让学生发言）对，公鸡最贵，5 文一只，最多买 100÷5=20 只。母鸡呢？最多 100÷3≈33 只。更重要的是——公鸡和母鸡数量确定了，小鸡不就自动出来了？100 - 公鸡 - 母鸡 = 小鸡！根本不需要第三重循环！

看优化后的代码：

```cpp
for (int gong = 0; gong <= 20; gong++) {
    for (int mu = 0; mu <= 33; mu++) {
        int xiao = 100 - gong - mu;
        if (xiao >= 0 && xiao % 3 == 0 &&
            5 * gong + 3 * mu + xiao / 3 == 100) {
            cout << "公鸡 " << gong << " 母鸡 " << mu << " 小鸡 " << xiao << endl;
        }
    }
}
```

优化后几次？21×34 = 714 次！从 103 万降到 714——缩小了 **1400 倍**！

这里有一条黄金法则，大家记在心里：**能用已知条件直接推导出来的变量，就不要枚举它。**

这就是枚举优化的第一心法——**减少变量**。变量之间有等式约束时，循环数 = 变量数 - 约束数。

还有第二心法——**缩小范围**。根据题目的约束条件，把循环的起止范围缩到最小。比如公鸡不会超过 20 只，你能想到的所有限制条件，都要用到循环边界上。

两个心法配合使用：先缩范围，再减变量。这样哪怕原来需要一亿次的问题，优化后可能只要几千次。

## 🖥️ 板书/屏幕要点
- **优化心法一：缩小范围** → 公鸡≤20, 母鸡≤33
- **优化心法二：减少变量** → 小鸡=100-公鸡-母鸡, 不需枚举
- **黄金法则**：能用已知条件推导的变量就不枚举
- **效率对比**：103 万次 → 714 次（1400 倍！）

## ✏️ 随堂练习
把百钱百鸡的三重循环版和优化版都敲进 IDE，运行对比。然后在优化版基础上修改，找出所有公鸡比母鸡多的解。

## 📌 小结
枚举优化的核心是"缩小"——缩小范围和减少变量。百钱百鸡从三重循环的三变量变为两重循环的两变量，次数从 103 万降到 714。记住黄金法则：能推导出来的就不要枚举。

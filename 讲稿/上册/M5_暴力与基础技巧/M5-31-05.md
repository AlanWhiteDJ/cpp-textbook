## 🎯 本课时目标
- 掌握用数组记录状态的模拟方法

## 📝 讲稿正文

前面几节课的模拟题，状态都很少——几个分数、几个变量。但有一类模拟涉及**大量对象的状态变化**，这时候就需要**数组**来帮忙了。

来看经典的**开关灯问题**：走廊有 n 盏灯，初始全部关闭。第 1 个人把所有灯打开；第 2 个人把每第 2 盏灯取反（开变关、关变开）；第 3 个人把每第 3 盏灯取反……到第 k 个人。问最后哪些灯亮着？

这题的模拟思路非常清晰——每个人都按规则操作他负责的灯。外层循环模拟人，内层循环模拟这个人影响的灯的位置。

```cpp
int n, k;
cin >> n >> k;
bool light[101] = {false};  // false=关, true=开

for (int person = 1; person <= k; person++) {
    for (int pos = person; pos <= n; pos += person) {
        light[pos] = !light[pos];  // 状态取反
    }
}

for (int i = 1; i <= n; i++) {
    if (light[i]) cout << i << " ";
}
```

注意内层循环 `pos += person`——这保证了第 person 个人只操作 person 的倍数位置。代码和题目描述一一对应：第 k 个人处理第 k 的倍数位置。

`!light[pos]` 是取反——true 变 false，false 变 true。这个写法比 `if (light[pos]) light[pos]=false; else light[pos]=true;` 简洁多了。

好，这个模块已经学了五种模拟类型。在结束之前，我要总结模拟题的**调试三板斧**：

第一，**打印中间状态**——在关键步骤加 `cout`，看每一步的变量值对不对。

第二，**用小数据测试**——先用 n=3、k=2 这种手动能验的数据跑，确认逻辑正确再放大。

第三，**对样例人工演算**——拿笔在纸上按你的代码逻辑走一遍样例，看是否和题目给的输出一致。

再提醒几个容易犯的错：状态变量必须初始化（`int score = 0;`）、题目规则的顺序不能随便改（先扣血再回血和先回血再扣血完全不同）、循环边界要看清是 `<` 还是 `<=`。

模拟告一段落。记住一句话：**模拟题考的不是你的智商，是你的细心和耐心。**

## 🖥️ 板书/屏幕要点
- **开关灯**：外层人、内层位置、`pos+=person`、`light[pos] = !light[pos]`
- **调试三板斧**：打印中间状态、小数据测试、人工演算样例
- **常见错误**：未初始化、顺序搞反、边界 `<` vs `<=`

## ✏️ 随堂练习
修改开关灯程序：初始所有灯是**打开**的，第 1 个人把所有灯关掉，后面规则不变。求最后亮着的灯。

## 📌 小结
状态模拟的核心是用数组批量管理状态变化。调试三板斧：打印中间状态、小数据测试、人工演算。模拟题不难，难在细心。下个模块我们学另一种"暴力"——排列和子集的枚举。

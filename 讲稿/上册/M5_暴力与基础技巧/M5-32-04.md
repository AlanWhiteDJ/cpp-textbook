## 📝 讲稿正文

前面我们学了排列和子集——它们的枚举量是"可控"的。但有时候，题目给的范围很大，全枚举就超时了。这时候需要——**剪枝**。

剪枝是什么意思？想象你在图书馆找一本红色的书。如果走到历史区，看到书架上全是蓝色封面的——你可以直接跳过这排，不用每一本都抽出来看封面。这就是剪枝：**在搜索过程中提前淘汰不可能的分支。**

来看一个经典的剪枝题——**回文日期**。给定起止日期（如 20200101 到 20201231），求这个时间段内有多少个日期是回文日期（正读反读都一样，如 20200202）。

直接思路：枚举每一天，判断是否回文。一年最多 366 天，OK。但如果跨度 1000 年呢？枚举每天就慢了。

剪枝思路：**只枚举回文日期！** 一个 8 位回文日期的前 4 位（年份）决定后 4 位。比如前 4 位是 2020，回文就是 20200202。所以枚举年份 → 构造回文 → 判断是否合法且在区间内。

```cpp
int days[] = {0, 31, 29, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};

bool isValid(int date) {
    int y = date/10000, m = (date/100)%100, d = date%100;
    if (m<1 || m>12 || d<1) return false;
    bool leap = (y%4==0 && y%100!=0) || (y%400==0);
    int maxd = days[m];
    if (m==2 && !leap) maxd = 28;
    return d <= maxd;
}

int solve(int start, int end) {
    int cnt = 0;
    for (int year = start/10000; year <= end/10000; year++) {
        int rev = 0, tmp = year;
        for (int k = 0; k < 4; k++) {
            rev = rev*10 + tmp%10;
            tmp /= 10;
        }
        int date = year*10000 + rev;
        if (date>=start && date<=end && isValid(date)) cnt++;
    }
    return cnt;
}
```

原本枚举每一天——跨度 1000 年要枚举约 365000 天。剪枝后只枚举年份——1000 次。快了 365 倍！而且年份还可以进一步缩小：找到起止年份即可。

这个思路的核心是**构造代替枚举**——用数学关系直接"造出"候选，而不是盲目遍历所有可能。这和百钱百鸡中用 `100-gong-mu` 算出小鸡是同一个哲学。

多重循环枚举的剪枝技巧还有：缩小每层循环的范围、用公式替代最后一层。这些我们前面都见过——今天给你一个统一的名字：**剪枝**。

## 🖥️ 板书/屏幕要点
- **剪枝**：提前排除不可能分支，不走到最后才发现白干
- **回文日期优化**：枚举年份→构造回文→判断合法性
- **复杂度**：365000 天 → 1000 年（365 倍提升）
- **构造代替枚举**：用数学关系直接生成候选

## ✏️ 随堂练习
修改回文日期程序，统计 10000101 到 99991231 之间所有回文日期，并统计其中有几个是 ABABBABA 型的。

## 📌 小结
剪枝是枚举的加速器——在不影响正确性的前提下，跳过不可能的情况。回文日期通过"构造代替枚举"，从每天枚举变为每年枚举。剪枝和缩小范围的本质相同：减少无效检查。

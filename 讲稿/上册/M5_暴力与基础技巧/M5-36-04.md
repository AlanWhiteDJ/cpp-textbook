# 第36章 第4课时：火烧赤壁与覆盖计数——离散化变体应用
**时长**：15-20分钟 | **对应教材**：第36章 36.4（变体）

## 🎯 本课时目标
- 掌握离散化+差分解决"被覆盖 k 次的总长度"问题
- 理解"被覆盖最多的点"类问题的离散化处理
- 建立离散化应用的通用模式

## 📝 讲稿正文

上节课我们学会了火烧赤壁——求所有线段覆盖总长度（覆盖 ≥1 次就算）。但如果题目是"求刚好被 k 条线段覆盖的总长度"呢？改动非常小！

还是同样的流程——收集端点、离散化、差分、还原——只需在统计时把条件从 `cur > 0` 改成 `cur == k`：

```cpp
int cur = 0, ans = 0;
for (int i = 1; i < m; i++) {
    cur += diff[i];
    if (cur == k) {
        ans += pos[i+1] - pos[i];
    }
}
```

就这么简单！从"覆盖 1 次"到"覆盖 k 次"，只改了一个判断条件。

如果题目是"求被覆盖最多次的点，它被覆盖了几次"呢？更简单——在还原差分的过程中，维护 `cur` 的最大值：

```cpp
int cur = 0, maxCover = 0;
for (int i = 1; i < m; i++) {
    cur += diff[i];
    maxCover = max(maxCover, cur);
    // 注意：这里不需要求长度，只需要 cur 的最大值
}
cout << maxCover << endl;
```

有没有发现一个规律？离散化+差分的模式下，**差分负责"操作"，还原循环负责"统计"**。统计时可以自由定义——覆盖总长度、刚好 k 次覆盖、最大覆盖次数……骨架不变，只改统计逻辑。

这引出一个重要的编程思想：**把"计算"和"统计"分开**。先无脑算差分，然后在还原的循环里做你想要的任何统计。就像做菜——先把食材切好（差分），然后你想炒想炖想凉拌都行（统计）。

离散化不是只能和差分组合。它还能和前缀和、DP、线段树等配合。但万变不离其宗——思想永远是"把稀疏的大坐标压缩到紧凑的小下标"。

还有一个注意事项：如果题目中的坐标是闭区间 [l, r] 而不是左闭右开 [l, r)，离散化后怎么处理？（停顿）可以在 r 上加 1——用 r+1 参与离散化，然后按照 [l, r+1) 的左闭右开方式来做。或者直接用 `diff[L]++; diff[R+1]--;` 的写法。两种方式等价，选你习惯的。

## 🖥️ 板书/屏幕要点
- **刚好覆盖 k 次**：`if (cur == k) ans += pos[i+1]-pos[i];`
- **最大覆盖次数**：`maxCover = max(maxCover, cur);`
- **通用模式**：差分负责操作 → 还原循环负责统计
- **闭区间处理**：将 r+1 纳入离散化，或直接用 `diff[R+1]--`

## ✏️ 随堂练习
4 条线段：[1,4], [2,5], [3,6], [2,5]。求刚好被 3 条线段覆盖的总长度。

## 📌 小结
离散化+差分的骨架是固定的：离散化→差分→还原统计。改统计条件就能适应不同需求。"把计算和统计分开"是写好这类题的关键。闭区间可通过加 1 转为半开区间。

# 第30章 第1课时：枚举——计算机最可靠的武器
**时长**：15-20分钟 | **对应教材**：第30章 30.1-30.2

## 🎯 本课时目标
- 理解枚举的核心思想："逐个尝试所有可能"
- 掌握枚举的三步模板：生成候选 → 检查条件 → 输出结果
- 建立枚举次数估算意识，知道什么时候枚举会超时

## 📝 讲稿正文

同学们，今天我们要正式进入算法思维的学习。你们之前写过一个"水仙花数"的程序——用 `for` 从 100 试到 999，把每个数的百位、十位、个位的三次方加起来，看等不等于自己。这种方法叫什么？（停顿）对，叫**枚举**。

枚举可能是所有算法里最"笨"的一个，但它也是**最可靠**的一个。

我给大家讲个故事。小明忘了自行车密码锁的三位数密码，他的密码锁有 1000 种组合。你们觉得他怎么办？坐在那儿想？还是……一个一个试？（让学生回答）对，他一个一个试，从 000 试到 247，"咔嗒"开了。他才花了十分钟。

但小明的爸爸是程序员，他说："你这十分钟才试了 248 次。写成程序，一秒钟能试一亿次——所有 1000 种可能瞬间试完。"

这就是枚举的力量——在人的世界里显得笨拙，在计算机的世界里却是最可靠的武器。因为计算机不会累、不会漏、不会烦。

那什么是枚举？三个字：**逐个试**。用编程语言来说，就是一个循环生成所有候选值，用 `if` 检查是否满足条件，满足了就输出。我们看一个最简单的例子——找 1 到 100 中的所有偶数：

```cpp
for (int i = 1; i <= 100; i++) {   // 循环生成候选
    if (i % 2 == 0) {              // 检查条件
        cout << i << " ";          // 符合就输出
    }
}
```

是不是清晰得像说明书？这就是枚举的魅力——不需要灵光一现的巧思，只需要踏踏实实地列出来。

但枚举也有天敌。什么天敌？（停顿）**次数爆炸**。3 位密码锁才 1000 种组合，8 位呢？1 亿种。虽然电脑也能试完，但在竞赛里，每道题只有 1 秒的时间。

大家记一个经验法则：1 万次、10 万次，瞬间；100 万次，约 0.01 秒；1000 万次，约 0.1 秒；1 亿次，约 1 秒——要看情况；10 亿次以上，基本就超时了。

那三重循环呢？比如每层循环 1000 次，1000×1000×1000 = 10 亿——几乎必超时。所以我们学枚举，关键不是"会不会枚举"，而是**"能不能缩小枚举范围"**。这个我们下节课细讲。

## 🖥️ 板书/屏幕要点
- **枚举三步模板**：`for 生成候选 → if 检查条件 → 输出结果`
- **次数估算表**：10⁶ ✅ | 10⁸ ⚠️ | 10⁹+ ❌
- **三重循环警告**：10³×10³×10³ = 10⁹，必超时

## ✏️ 随堂练习
写程序输出 1~1000 中所有能被 7 整除或被 11 整除的数。

## 📌 小结
枚举就是"把所有可能列出来逐个试"。三步走：循环生成、条件检查、结果输出。枚举在计算机上非常快——百万次毫秒级。但要注意次数爆炸，三重循环要特别小心。下节课我们学怎么"缩小范围"来优化枚举。

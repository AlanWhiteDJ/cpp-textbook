## 📝 讲稿正文

最后一种模拟类型——**二维模拟**。最经典的就是扫雷。

扫雷的核心逻辑：给定 n×m 的雷区（`*` 是雷，`?` 是非雷），输出每个非雷格周围 8 个方向上的雷数。

最"笨"的写法是写 8 个 if：

```cpp
if (mp[i-1][j-1] == '*') cnt++;
if (mp[i-1][j] == '*') cnt++;
if (mp[i-1][j+1] == '*') cnt++;
// ... 还有 5 个
```

这样又长又容易漏。聪明的做法是用**方向数组**：

```cpp
int dx[8] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[8] = {-1, 0, 1, -1, 1, -1, 0, 1};

int cnt = 0;
for (int k = 0; k < 8; k++) {
    int nx = i + dx[k], ny = j + dy[k];
    if (nx >= 1 && nx <= n && ny >= 1 && ny <= m && mp[nx][ny] == '*') {
        cnt++;
    }
}
```

8 行重复代码变成 4 行循环。而且方向数组还有一个好处——如果要改成 4 方向（上下左右），只需要改 `dx` 和 `dy`：

```cpp
int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};
```

这个技巧在网格类问题中超级常用——走迷宫、扫雷、生命游戏、填色……都会用到。

好，模拟进阶的五节课到此结束。我们来总结：
- **大模拟**用四步法：读题→规则→数据→实现
- **环形模拟**用取模：`(pos±s%n+n)%n`
- **规则型模拟**用"条件+重置"模式
- **格式型模拟**列"特殊情况检查表"
- **二维模拟**用方向数组

模拟题是竞赛的"送分大户"。它不要求你有多聪明，但要求你**细心、耐心、有方法**。只要按四步法走，绝大多数模拟题都能拿下。

## 🖥️ 板书/屏幕要点
- **方向数组**：`dx[8]={-1,-1,-1,0,0,1,1,1}`, `dy[8]={-1,0,1,-1,1,-1,0,1}`
- **8 邻格**：`for(k=0;k<8;k++) { nx=i+dx[k]; ny=j+dy[k]; }`
- **模拟进阶总结**：大模拟/环形/规则型/格式型/二维
- **一句箴言**：模拟题不难，难在细心

## 📌 小结
二维模拟用方向数组替代重复 if，简洁不易错。模拟进阶五大类型覆盖了竞赛模拟题的绝大部分。记住核心：细心+四步法=送分题。下章开始学算法优化的重头戏——双指针。

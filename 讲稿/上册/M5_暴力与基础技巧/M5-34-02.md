## 📝 讲稿正文

上节课我们学会了对撞指针解决"两数之和"。如果题目变成"三数之和"呢？——在有序数组中找三个数，使得 a[i] + a[j] + a[k] == target。

直接思路：枚举第一个数 a[i]，剩下的问题不就是"在 i+1 到 n-1 中找两数之和等于 target - a[i]"吗？两数之和——对撞指针！

```cpp
// 在有序数组 a[l..r] 中找两数之和为 target
// 返回所有不重复的组合
void twoSum(int a[], int l, int r, int target, int first) {
    while (l < r) {
        int sum = a[l] + a[r];
        if (sum == target) {
            cout << first << " " << a[l] << " " << a[r] << endl;
            // 去重：跳过相同的值
            while (l < r && a[l] == a[l+1]) l++;
            while (l < r && a[r] == a[r-1]) r--;
            l++; r--;
        } else if (sum < target) l++;
        else r--;
    }
}

// 三数之和
void threeSum(int a[], int n, int target) {
    sort(a, a + n);
    for (int i = 0; i < n - 2; i++) {
        if (i > 0 && a[i] == a[i-1]) continue;  // 跳过重复的第一个数
        twoSum(a, i+1, n-1, target - a[i], a[i]);
    }
}
```

注意去重处理！如果没有去重，数组 `{1,1,2,3,4}` 中找三数和为 6，可能会输出多组重复的 `1,2,3`。

去重分两层：
1. 外层：`if (i>0 && a[i]==a[i-1]) continue;`——跳过重复的第一个数
2. 内层：找到一组解后，`while (l<r && a[l]==a[l+1]) l++;`——跳过重复的左值和右值

这个"固定一个，对撞另外两个"的套路不仅适用于三数之和，还适用于任何"n 数之和"问题——固定 n-2 个，最后两个对撞指针。

复杂度分析：外层循环 O(n)，内层对撞 O(n)，总 O(n²)。虽然比两数之和的 O(n) 慢，但比三重循环暴力的 O(n³) 快了一个数量级。

这里还引出一个重要技巧——**降维**。把三维问题降为二维，把二维降为一维。这是算法设计中最常用的策略之一。双指针本身也是一种降维——把两次遍历降为一次遍历。

## 🖥️ 板书/屏幕要点
- **三数之和 = 固定第一个 + 两数之和（对撞）**
- **去重两层**：外层 `a[i]==a[i-1]` continue；内层跳过重复 l/r
- **复杂度**：O(n²)，比暴力 O(n³) 快一个数量级
- **降维思维**：三维→二维，二维→一维

## ✏️ 随堂练习
写三数之和程序，输入数组和 target=0，输出所有不重复的三元组。测试 `{-1,0,1,2,-1,-4}` target=0，输出 `{-1,-1,2}` 和 `{-1,0,1}`。

## 📌 小结
三数之和通过"固定一个+对撞两个"实现 O(n²)。去重是关键——外层跳过重复首数，内层跳过重复左右值。这套"降维"思路可以推广到 n 数之和。

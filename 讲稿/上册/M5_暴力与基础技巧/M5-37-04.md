# 第37章 第4课时：高精度除法——从高位到低位的特殊方向
**时长**：15-20分钟 | **对应教材**：第37章 37.5

## 🎯 本课时目标
- 理解高精度除法为何方向与加减乘不同（从高位到低位）
- 掌握高精 ÷ 单精的逐步试商过程
- 学会余数的维护和商的前导零清理

## 📝 讲稿正文

除法是最"另类"的高精度运算——加减乘都是从低位往高位走，但除法是从**高位往低位**走。为什么？因为手工竖式除法就是这样的——从最高位开始，一位一位往下"拉"。

这节课学**高精 ÷ 单精**（大数除以普通 int）。高精 ÷ 高精比较复杂（需要多次减法模拟试商），竞赛里不常见，我们重点掌握单精除法。

```cpp
int divSingle(int a[], int la, int b, int q[], int &remainder) {
    long long r = 0;  // 当前余数
    for (int i = la; i >= 1; i--) {  // 从高位往低位！
        r = r * 10 + a[i];           // 把当前位"拉"下来
        q[i] = r / b;                // 试商
        r %= b;                      // 新的余数
    }
    remainder = r;

    // 确定商的长度（去除前导零）
    int lq = la;
    while (lq > 1 && q[lq] == 0) lq--;
    return lq;
}
```

我们来手工走一遍 `1234 ÷ 5`：

- a[] = {0, 4, 3, 2, 1}（倒序，la=4）
- i=4：r = 0×10 + 1 = 1；q[4] = 1/5 = 0；r = 1%5 = 1
- i=3：r = 1×10 + 2 = 12；q[3] = 12/5 = 2；r = 12%5 = 2
- i=2：r = 2×10 + 3 = 23；q[2] = 23/5 = 4；r = 23%5 = 3
- i=1：r = 3×10 + 4 = 34；q[1] = 34/5 = 6；r = 34%5 = 4
- 结果：商 = 246（q[3]q[2]q[1]，q[4]=0 被前导零清理掉），余数 = 4

注意到了吗？**除法的循环方向是 `for (i = la; i >= 1; i--)`**——从最高位往最低位走。而加减乘都是 `for (i = 1; i <= lc; i++)`——从低位往高位走。这是除法最特殊的地方。

为什么除法要倒着走？因为手工除法就是这样的——你不可能从个位开始除。想想看，如果从个位开始，1÷5 得 0 余 1，然后呢？你没法继续了。必须从高位往下"拉"数字。

`r = r * 10 + a[i]` 这一行模拟的就是"把下一位数字拉下来"的动作。r 是上一步的余数，乘以 10（进一位），加上当前位的数字，构成新的被除数。

商的长度可能比被除数短——因为可能有前导零。比如 1234÷5=246，q[4]=0，需要清理。`while (lq>1 && q[lq]==0) lq--;` 确保余数至少保留一位。

高精 ÷ 高精的原理是用减法不断试商——每次从被除数中减去除数的倍数。如果你感兴趣，可以课后查阅资料，但竞赛中极少要求手写高精÷高精。

## 🖥️ 板书/屏幕要点
- **除法方向**：`for (i = la; i >= 1; i--)` 从高位往低位
- **核心步骤**：`r = r*10 + a[i]; q[i] = r/b; r %= b;`
- **余数维护**：r 保存上一步余数，乘 10 拉下一位
- **去前导零**：`while(lq>1 && q[lq]==0) lq--;`

## ✏️ 随堂练习
实现高精 ÷ 单精。测试 123456789 ÷ 123，输出商和余数。

## 📌 小结
高精度除法从高位往低位处理——方向与加减乘完全相反。维护余数 r，每步 `r=r*10+a[i]` 模拟"拉下一位"。商要去除前导零。高精÷高精不常用，了解原理即可。

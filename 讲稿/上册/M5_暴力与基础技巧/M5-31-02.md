# 第31章 第2课时：石头剪刀布——回合制游戏模拟
**时长**：15-20分钟 | **对应教材**：第31章 31.3

## 🎯 本课时目标
- 实现石头剪刀布的回合制模拟
- 掌握多条件判断的 `if-else` 组织方式
- 学会在模拟过程中打印中间状态辅助调试

## 📝 讲稿正文

上节课学了模拟的四步法。这节课我们来真正实现一个经典模拟——石头剪刀布。两人比赛 5 轮，赢的得 1 分，最后输出比分。

先用四步法分析：
- 关键词："两人""5 轮""S 石头/J 剪刀/B 布""赢+1 分"
- 变量：`scoreA`, `scoreB`（双方得分），`char a, b`（双方出拳），`int i`（轮次）
- 步骤：循环 5 轮，每轮输入双方出拳，判断输赢，更新比分
- 验证：手工走一遍样例

好，动手写代码：

```cpp
int scoreA = 0, scoreB = 0;
char a, b;

for (int i = 1; i <= 5; i++) {
    cout << "第 " << i << " 轮，A 出拳（S/J/B）：";
    cin >> a;
    cout << "B 出拳：";
    cin >> b;

    if (a == b) {
        cout << "平局！" << endl;
    } else if ((a == 'S' && b == 'J') ||
               (a == 'J' && b == 'B') ||
               (a == 'B' && b == 'S')) {
        cout << "A 胜！" << endl;
        scoreA++;
    } else {
        cout << "B 胜！" << endl;
        scoreB++;
    }
    cout << "当前比分 A:" << scoreA << " B:" << scoreB << endl;
}
```

注意这个 `if-else if-else` 的结构。判断 A 胜的条件是什么？（让学生回答）S 胜 J、J 胜 B、B 胜 S。三组条件用 `||` 连在一起。这个写法比分别写三个 `if` 更简洁。

大家注意最后一行——每轮结束我都打印了当前比分。为什么？（停顿）这就是模拟题的**调试技巧**。你可以在每个关键步骤后面加一行 `cout`，看看程序的中间状态是不是跟你预期的一样。如果中间状态就错了，后面的肯定不对。

调试完成后，把调试用的 `cout` 注释掉就行。

我们来玩个变化：把"比 5 轮"改成"先赢 3 局者胜"。怎么改？（让学生思考）把 `for` 改成 `while`，条件是双方都不到 3 分：

```cpp
while (scoreA < 3 && scoreB < 3) { ... }
```

看到了吗？模拟程序的结构非常灵活——改规则就是改循环条件和判断逻辑。这是因为模拟的本质就是把规则"翻译"成代码。

## 💬 互动提问
1. "如果规则改成'三局两胜'（赢得两局就结束），怎么改代码？"
2. "这三组 `||` 条件有没有可能漏掉某种情况？你怎么验证？"
3. "打印中间状态为什么能帮助调试？你以前用过这个方法吗？"

## 🖥️ 板书/屏幕要点
- **判断 A 胜**：`(a=='S'&&b=='J') || (a=='J'&&b=='B') || (a=='B'&&b=='S')`
- **变量初始化**：`int scoreA = 0, scoreB = 0;`（必须！）
- **调试技巧**：每轮后 `cout` 当前状态
- **改规则 = 改循环条件**：`for i<=5` → `while score<3`

## ✏️ 随堂练习
把石头剪刀布改成"先赢 3 局者胜"，注释掉调试输出，只输出最终比分和胜者。

## 📌 小结
石头剪刀布是典型的回合制模拟。核心是 `if-else` 组织多条件判断。模拟题的调试靠"打印中间状态"。模拟程序改规则非常容易——改循环条件和判断逻辑就行。

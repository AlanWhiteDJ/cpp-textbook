## 📝 讲稿正文

上节课学了加法——那减法呢？减法的核心是**借位**：当前位不够减时，向高位"借 1 当 10"。

但减法比加法多一个前提——**先判断大小，确保大减小**。如果 a < b，就要交换并输出负号。

先写比较函数：

```cpp
int cmp(int a[], int la, int b[], int lb) {
    if (la != lb) return la - lb;   // 长度不同，长的更大
    for (int i = la; i >= 1; i--)   // 长度相同，从高位比到低位
        if (a[i] != b[i])
            return a[i] - b[i];
    return 0;  // 相等
}
```

比较的逻辑：先比长度——长度不同时长的更大。长度相同时，从最高位往下逐位比，第一次出现不同的位置决定大小。

现在写减法（保证 a ≥ b）：

```cpp
int sub(int a[], int la, int b[], int lb, int c[]) {
    int lc = la;
    int borrow = 0;
    for (int i = 1; i <= lc; i++) {
        int ai = a[i] - borrow;
        int bi = (i <= lb) ? b[i] : 0;
        if (ai < bi) {
            ai += 10;      // 向高位借 1 当 10
            borrow = 1;
        } else {
            borrow = 0;
        }
        c[i] = ai - bi;
    }
    // 去除前导零
    while (lc > 1 && c[lc] == 0) lc--;
    return lc;
}
```

注意借位的逻辑：如果 `ai < bi`，就向高位借 1——ai 加 10，borrow 设为 1（下一位要减掉这次借走的 1）。否则 borrow 恢复为 0。

还有一个重要操作——**去除前导零**。比如 100 - 99 = 1，但数组里是 `c[1]=1, c[2]=0, c[3]=0`，lc=3。如果直接按 lc=3 输出会变成 "001"。所以要用 `while (lc>1 && c[lc]==0) lc--;` 清理高位零。

`lc > 1` 这个条件不能丢——如果结果本身就是 0（如 123 - 123），至少保留一位 "0"。

完整的减法调用：

```cpp
if (cmp(a, la, b, lb) >= 0) {  // a ≥ b
    int lc = sub(a, la, b, lb, c);
    print(c, lc);
} else {                         // a < b
    cout << "-";
    int lc = sub(b, lb, a, la, c);  // 交换：大减小
    print(c, lc);
}
```

减法中最容易犯的错：忘了比较大小（直接减，结果变成补数），忘了清理前导零（输出 001 而不是 1），borrow 忘了重置（上一次的借位影响下一位）。

## 🖥️ 板书/屏幕要点
- **比较大小**：先比长度 → 长度相同从高位逐位比
- **借位逻辑**：`if (ai<bi) { ai+=10; borrow=1; } else borrow=0;`
- **去前导零**：`while (lc>1 && c[lc]==0) lc--;`
- **负号处理**：比较 → a<b 则交换并输出 "-"

## ✏️ 随堂练习
实现高精度减法。测试 1000 - 1（注意前导零处理）和 123 - 456（注意负号和交换）。

## 📌 小结
高精度减法核心是借位。减法前必须先比较大小，确保大减小；a < b 时交换并加负号。结果要去除前导零。

## 📝 讲稿正文

这节课我们来总结双指针，然后做一道聪明的对撞指针题——**盛水最多的容器**。

题目：给定 n 个非负整数表示挡板高度，每个挡板间距为 1。找两个挡板，使它们与 x 轴围成的容器能装最多的水。

如果暴力枚举所有挡板对——O(n²)。双指针怎么做？

思路：左指针 l=0，右指针 r=n-1。容量 = `min(height[l], height[r]) * (r-l)`。每次移动**较矮的那根**——因为移动高的不会增加容量（宽度减少，高度受限于矮的那根）。移动矮的才有可能找到更高挡板来抵消宽度损失。

```cpp
int maxArea(int height[], int n) {
    int l = 0, r = n - 1, ans = 0;
    while (l < r) {
        int h = min(height[l], height[r]);
        ans = max(ans, h * (r - l));
        if (height[l] < height[r]) l++;
        else r--;
    }
    return ans;
}
```

妙不妙？同样是 O(n)，但这里的"移动哪边"不是简单的 sum vs target，而是"谁矮谁走"——因为矮的那个是瓶颈，它不走容量永远不会增加。

好，现在总结双指针的三种形态：
- **对撞指针**（l=0, r=n-1）：有序数组，相向而行。典型题：两数之和、三数之和、盛水容器。
- **快慢指针**（slow/fast）：同向不同速。典型场景：找中点、判环。
- **滑动窗口**（[l,r]）：同向维护区间。典型题：最长无重复子串、最短覆盖区间、A-B 数对。

三者的共同核心：**两个变量在数据上游走，每次移动排除一批不可能，O(n) 解决 O(n²) 问题。**

我给你们一个"什么时候用双指针"的快速判断：
- 数组**有序** + 找两数关系 → 对撞指针
- 子数组/子串 + 连续区间 → 滑动窗口
- 需要找中点/判循环 → 快慢指针

双指针是算法优化的第一课。从此以后，你遇到 O(n²) 的问题就会本能地问一句："能不能用双指针？"

## 🖥️ 板书/屏幕要点
- **盛水容器**：`min(h[l],h[r])*(r-l)` → 谁矮谁移
- **三形态总结**：对撞（相向）| 快慢（同向不同速）| 滑动窗口（可变区间）
- **复杂度跃迁**：O(n²) → O(n)
- **判断法则**：有序+两数→对撞；子数组+连续→滑动窗口

## 📌 小结
双指针三种形态覆盖了绝大多数"数组元素间关系"和"子数组"问题。核心是每次移动排除一批不可能，从不回头。O(n²)→O(n) 的优化从此成为你的常规武器。

## 📝 讲稿正文

这节课我们来做一道综合性枚举题——**火柴棒等式**。题目：给你 n 根火柴棒，拼出 A+B=C 的等式。每个数字 0~9 需要的火柴棒数不同，"+"和"="各需 2 根。求能拼出的等式个数。

分析思路：
- 先建数组 `cost[10]`，记录每个数字需要多少根火柴
- 写函数 `need(x)`，计算整数 x 需要的火柴棒总数
- 枚举 A 和 B，计算总火柴数：`need(A) + need(B) + need(A+B) + 4`（+4 是运算符）
- 当 `total == n` 时找到一个等式

核心代码：

```cpp
int cost[10] = {6, 2, 5, 5, 4, 5, 6, 3, 7, 6};  // 0~9 各需火柴数

int need(int x) {
    if (x == 0) return cost[0];  // 0 特殊处理
    int sum = 0;
    while (x) {
        sum += cost[x % 10];
        x /= 10;
    }
    return sum;
}
```

那 A 和 B 枚举到多少？这是关键！n≤24，除去运算符 4 根，剩下 20 根。三个数字共享这 20 根，每个数字至少 2 根（数字 1 只要 2 根）。所以每个数字最多到 1111 左右。A 和 B 各自枚举到 1000 就足够。

但这里有一个陷阱——A 和 B 的枚举量是 1000×1000 = 100 万次。你能不能再缩小？（让学生思考）可以用 `need(A)` 先判断 A 是否已经超过总火柴数限制，如果超过了就不需要再枚举 B。这就是典型的**循环内剪枝**。

今天这堂课把排列枚举、子集枚举、剪枝优化都串起来了。我们来总结枚举进阶的核心：

排列枚举用 `next_permutation`——处理"不同顺序"的场景，n 个元素的全排列。
子集枚举用二进制 mask——处理"选或不选"的场景，2ⁿ 种子集。
剪枝优化用"提前淘汰"——减少无效检查，是枚举的加速器。

三者结合，让"暴力算法"在 n 不太大时变得高效实用。记住——暴力出奇迹的前提是，你暴得聪明。

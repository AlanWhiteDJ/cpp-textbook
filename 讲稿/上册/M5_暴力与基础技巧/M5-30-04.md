## 📝 讲稿正文

前面几节课我们都在讲怎么缩小范围、减少循环。这节课我们来看几道"换个马甲"的枚举题，你会发现——思路完全一样。

先看一道方程题：求 3x + 5y = 100 的所有自然数解。

x 的范围？最大 100/3 ≈ 33。y 呢？不用循环——直接算：y = (100 - 3x) / 5。但 y 必须是整数，所以要判断整除。

```cpp
for (int x = 0; x <= 33; x++) {
    int rest = 100 - 3 * x;
    if (rest % 5 == 0) {
        int y = rest / 5;
        cout << "x = " << x << ", y = " << y << endl;
    }
}
```

一次循环 34 次，瞬间完成。注意——这里的关键是用 `%` 判断整除，而不是用浮点数除法再判断。

再看一道更贴近生活的题——**换硬币**。用 1 元、2 元、5 元的硬币凑出 20 元，有多少种凑法？

（停顿，让学生思考怎么枚举）

思路一模一样：两个循环枚举 1 元和 2 元硬币的数量，5 元硬币的数量自动算出来。

```cpp
int count = 0;
for (int a = 0; a <= 20; a++) {        // 1 元最多 20 个
    for (int b = 0; b <= 10; b++) {    // 2 元最多 10 个
        int rest = 20 - a - 2 * b;
        if (rest >= 0 && rest % 5 == 0) {
            int c = rest / 5;
            cout << "1元:" << a << " 2元:" << b << " 5元:" << c << endl;
            count++;
        }
    }
}
cout << "共 " << count << " 种" << endl;
```

有没有发现规律？不管是百钱百鸡、勾股数、方程整数解还是换硬币——套路完全一样：**枚举前几个变量，最后一个用计算代替循环**。

这就是枚举思维的魅力——不是靠灵感，而是靠一套固定的方法论。拿到一道题，先问自己三个问题：
1. 候选值范围是什么？
2. 几个变量？能用约束条件减掉几个？
3. 是否需要判断整除或其他约束？

这三个问题想清楚，代码基本就写完了。

## 🖥️ 板书/屏幕要点
- **方程整数解**：枚举 x → 算 y = (100-3x)/5 → 判整除 rest%5==0
- **换硬币**：枚举 1 元、2 元 → 算 5 元 → 判整除
- **通用套路**：枚举 n-1 个变量 → 最后一个用公式算出 → 判约束

## ✏️ 随堂练习
写程序解决"马驮瓦"问题：100 匹马驮 100 块瓦。大马每匹驮 3 块，中马每匹驮 2 块，两匹小马合驮 1 块。问大、中、小马各几匹？

## 📌 小结
方程整数解和换硬币问题都是枚举的变体。核心套路不变：枚举前几个变量、最后一个用公式算、判整除。这套方法可以解决绝大多数"凑数"类问题。

# 第33章 第3课时：规则型模拟——乒乓球计分与 Lambda 表达式
**时长**：15-20分钟 | **对应教材**：第33章 33.3

## 🎯 本课时目标
- 掌握规则型模拟的"条件+重置"模式
- 理解乒乓球计分中的"领先两分"规则实现
- 初步接触 Lambda 表达式，减少重复代码

## 📝 讲稿正文

这节课我们来模拟一场乒乓球比赛。题目是这样的（NOIP 2003 普及组）：输入一个由 'W'（华华赢）和 'L'（对手赢）组成的字符串，以 'E' 结束。分别输出 11 分制和 21 分制下的完整比分。

规则核心：一方得分 ≥ 11（或 21）**且**领先对手 ≥ 2 分时，本局结束，比分清零开始下一局。

这道题需要处理两个关键：
1. 什么时候一局结束？`(w>=limit || l>=limit) && abs(w-l) >= 2`
2. 两种赛制（11 分和 21 分）怎么避免代码重复？

对第二点，我们可以用 **Lambda 表达式**——它就像一个"可以传参数的函数"，同一个逻辑处理两种赛制：

```cpp
string all;

auto solve = [&](int limit) {
    int w = 0, l = 0;
    for (char c : all) {
        if (c == 'E') break;
        if (c == 'W') w++; else l++;
        if ((w >= limit || l >= limit) && abs(w - l) >= 2) {
            cout << w << ":" << l << endl;
            w = l = 0;  // 新一局
        }
    }
    cout << w << ":" << l << endl;  // 最后一局（可能未完成）
};

solve(11);
cout << endl;
solve(21);
```

这个 `auto solve = [&](int limit) { ... }` 是什么意思？（停顿）`[&]` 表示 Lambda 可以访问外部变量（如 `all`），`(int limit)` 是参数。调用 `solve(11)` 执行 11 分制，`solve(21)` 执行 21 分制。一个函数处理两种赛制，零重复代码。

注意比赛结束后的"最后一局"处理——可能打到一半就收到 'E' 了（输入结束），这时 `w` 和 `l` 还没有满足结束条件。所以循环结束后要再输出一次当前比分。

规则型模拟的关键是：**把所有规则列出来，一条一条对应 if 语句**。比如乒乓球的规则就两条：① W 加华华分、L 加对手分 ② 满足条件时本局结束、清零。你看代码是不是正好对应这两个 if？

这就是模拟的美——规则和代码一一映射，很难出错。

## 🖥️ 板书/屏幕要点
- **结束条件**：`(w>=limit || l>=limit) && abs(w-l)>=2`
- **Lambda**：`auto solve = [&](int limit) { ... };` 减少重复
- **最后一局**：循环结束后无论是否满足结束条件都输出当前比分
- **规则→if 映射**：一个规则对应一个 if 判断

## ✏️ 随堂练习
扩展乒乓球程序，增加 11 分制下的统计：总共进行了几局完整比赛？

## 📌 小结
规则型模拟的套路是"条件判断 + 状态重置"。Lambda 表达式减少重复代码。乒乓球的"≥11 且领先 ≥2"是经典条件模式。写完代码逐条对照规则清单验证。

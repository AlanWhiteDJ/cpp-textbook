## 📝 讲稿正文

上节课学了 `next_permutation`，这节课用它来解决一道经典题——**三连击**。题目是：将 1~9 这九个数字分成三个三位数，使得第二个数是第一个数的 2 倍，第三个数是第一个数的 3 倍。求所有分法。

如果用九重循环？9⁹ 次——3.87 亿，太慢。用 `next_permutation` 呢？9! = 362880 次，计算机瞬间完事。

思路很直接：生成 1~9 的全排列，前三位拼成第一个数，中间三位拼成第二个数，后三位拼成第三个。然后检查倍数关系。

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int a[] = {1, 2, 3, 4, 5, 6, 7, 8, 9};
    do {
        int x = a[0] * 100 + a[1] * 10 + a[2];
        int y = a[3] * 100 + a[4] * 10 + a[5];
        int z = a[6] * 100 + a[7] * 10 + a[8];
        if (y == 2 * x && z == 3 * x) {
            cout << x << " " << y << " " << z << endl;
        }
    } while (next_permutation(a, a + 9));
    return 0;
}
```

代码简洁得不像是"暴力"——就一个 `do-while`，里面几行计算+判断。这就是`next_permutation` 的威力：它帮你管理了"生成全排列"这件复杂的事，你只需要关心"拿到一个排列后怎么检查"。

输出结果有四组（按 x 排序）：192 384 576，219 438 657，273 546 819，327 654 981。你们可以验算一下——每组都满足第二个数是第一个的 2 倍、第三个是第一个的 3 倍。

这里有个小优化——其实 x 的范围可以缩小。因为三个三位数要用完 9 个数字，所以 x 至少是 123，最大是 329（因为 329×3=987）。但这道题 n 很小，300 多万次检查对计算机来说就是一瞬间的事，不优化也没关系。

这引出一个经验：**当数据规模很小时（比如 n≤10），怎么方便怎么来，不需要过度优化。** 写代码的时间也是成本。

排列枚举的解题模式可以总结为三步：**生成排列 → 拆分数据 → 检查条件**。套用这个模式，很多"排列+约束"的问题都能解决。

## 🖥️ 板书/屏幕要点
- **三连击解题模式**：生成 1~9 全排列 → 拆成三个三位数 → 检查 y==2x && z==3x
- **答案**：192/384/576，219/438/657，273/546/819，327/654/981
- **小规模原则**：n 小就别优化，写代码的时间也是成本

## ✏️ 随堂练习
修改三连击程序，改为找"第一个三位数个位是 2"的所有解——即在原答案基础上加过滤条件。

## 📌 小结
三连击是排列枚举的经典应用。`next_permutation` 让全排列生成变得简单，你只需关心拿到排列后怎么检查。排列枚举模式：生成→拆分→检查。小规模问题不要过度优化。

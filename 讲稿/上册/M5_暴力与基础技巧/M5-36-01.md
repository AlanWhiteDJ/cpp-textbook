# 第36章 第1课时：离散化——当坐标大到数组装不下
**时长**：15-20分钟 | **对应教材**：第36章 36.1-36.2

## 🎯 本课时目标
- 理解离散化的场景：坐标范围巨大但实际数据点稀少
- 掌握离散化三步走：排序→去重→二分查找
- 理解离散化保留的是"相对顺序"而非"绝对数值"

## 📝 讲稿正文

上节课我们学了前缀和与差分——好用。但有一个尴尬的问题：如果坐标范围是 -10⁹ 到 10⁹，但只有 10⁵ 个点有数据，你还能开 2×10⁹ 大小的数组吗？显然不行——内存直接爆炸。

这个问题怎么解决？**离散化**。

离散化是什么？一句话——**保留相对顺序，压缩坐标范围**。把"稀疏的大坐标"映射成"紧凑的小下标"。

我举个例子。假设你是城市规划师，要在 10⁹ 米长的地图上标记 100 栋楼。如果每米一个数组元素——10⁹ 个格子，几百 MB 内存，而且 99.99999% 都是空的。

但你想想：你关心的只是**楼与楼的相对位置**。楼 A 在东经 5000 米，楼 B 在东经 3×10⁸ 米——你只需要知道"B 在 A 的右边"，它们之间具体隔着多少空地跟你没关系。

所以，把 100 栋楼按坐标排序后映射成 1 到 100：原来的坐标 {5000, 300000000, 700000000} 变成 {1, 2, 3}。相对顺序没变，坐标范围却从 10⁹ 压缩到了 100。

离散化的标准操作是三步走：

**Step 1：排序**。把原始数据复制一份，排序。

**Step 2：去重**。用 `unique` 去掉重复值，得到"所有出现过的不重复坐标"。

**Step 3：二分查找**。对每个原始值，用 `lower_bound` 找到它在去重数组中的下标——这个下标就是离散化后的值。

代码模板：

```cpp
int a[100005];  // 原始数据
int b[100005];  // 去重后的有序副本
int m;          // 不重复元素个数

void discrete() {
    for (int i = 1; i <= n; i++) b[i] = a[i];
    sort(b + 1, b + n + 1);                           // Step 1: 排序
    m = unique(b + 1, b + n + 1) - (b + 1);           // Step 2: 去重
}

int getID(int x) {
    return lower_bound(b + 1, b + m + 1, x) - b;      // Step 3: 二分查下标
}
```

类比：离散化就像**考试排名**。你考了 685 分——这个具体分数不重要，重要的是"你是第 3 名"。排名保留了相对顺序（第 1 > 第 2 > 第 3），但把 0~750 的分数范围压缩到了 1~N。

记住，去重数组 `b` 不要丢弃——后面可能还需要从离散化下标反查原始坐标。

## 🖥️ 板书/屏幕要点
- **离散化三步走**：① `sort` ② `unique` 去重 ③ `lower_bound` 查下标
- **核心公式**：`m = unique(b+1, b+n+1) - (b+1)`
- **映射函数**：`lower_bound(b+1, b+m+1, x) - b` → 1-based 下标
- **本质**：保留相对顺序，压缩坐标范围

## ✏️ 随堂练习
输入 `{100, 5, 3, 5, 200000000}`，写出排序后、去重后、每个原始值离散化后的下标。

## 📌 小结
离散化解决"坐标太大、数据稀疏"的矛盾。三步走：排序→去重→二分查下标。保留去重数组 b 用于反向映射。本质是用"排名"替代"绝对数值"。

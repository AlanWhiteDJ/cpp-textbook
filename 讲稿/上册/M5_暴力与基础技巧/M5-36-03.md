# 第36章 第3课时：离散化 + 差分——大范围区间覆盖问题
**时长**：15-20分钟 | **对应教材**：第36章 36.4

## 🎯 本课时目标
- 将离散化与差分结合，解决大坐标范围的区间覆盖问题
- 理解离散化后统计长度时要"映射回原始坐标"
- 掌握差分在离散化后的"半开区间"处理方式

## 📝 讲稿正文

前面我们学了两个武器：差分做区间修改 O(1)，离散化压缩坐标范围。如果把它们结合起来呢？——就能解决坐标范围 10⁹ 但线段数只有 10⁵ 的区间覆盖问题！

这就是**火烧赤壁**——数轴上有 n 条线段，求所有线段覆盖的总长度（重复覆盖只算一次）。

坐标可能到 10⁹，但只有 2n 个端点（每个线段两个端点）。离散化后坐标缩到 2n 级别，再用差分统计覆盖。

关键步骤：

**第一步**：收集所有线段端点，离散化。

```cpp
struct Segment { int l, r; } seg[100005];
int pos[200005];   // 所有端点（大小 2n）
int cnt = 0;

for (int i = 0; i < n; i++) {
    cin >> seg[i].l >> seg[i].r;
    pos[++cnt] = seg[i].l;
    pos[++cnt] = seg[i].r;
}
sort(pos + 1, pos + cnt + 1);
int m = unique(pos + 1, pos + cnt + 1) - (pos + 1);
```

**第二步**：在离散化后的坐标上做差分。

```cpp
int diff[200005] = {0};
for (int i = 0; i < n; i++) {
    int L = lower_bound(pos+1, pos+m+1, seg[i].l) - pos;
    int R = lower_bound(pos+1, pos+m+1, seg[i].r) - pos;
    diff[L]++;
    diff[R]--;  // 注意是 R 不是 R+1！因为区间是 [L, R)
}
```

为什么是 `diff[R]--` 而不是 `diff[R+1]--`？（停顿让学生想）因为离散化后，线段覆盖的是离散点之间的"段"。点 L 到点 R 之间的段是 `[pos[L], pos[R])`，这是一个左闭右开区间。所以在 R 位置就停止覆盖。

**第三步**：还原差分，统计真实长度。

```cpp
int ans = 0, cur = 0;
for (int i = 1; i < m; i++) {
    cur += diff[i];
    if (cur > 0) {
        ans += pos[i+1] - pos[i];  // 不是 +1！是原始坐标的真实长度
    }
}
```

⚠️ **最容易犯的错**：把 `pos[i+1] - pos[i]` 写成 `1`！离散化后相邻下标之间可能隔着很远的原始距离。比如 pos[1]=3, pos[2]=1000000，中间实际被覆盖了 999997 个单位——你如果只加 1 就全错了。

这就是离散化的精髓：**操作在压缩后的坐标上进行，但计算最终答案时要映射回原始坐标的真实长度。**

## 💬 互动提问
1. "为什么差分用 `diff[R]--` 而不是 `diff[R+1]--`？离散化前和离散化后有什么不同？"
2. "统计长度时为什么用 `pos[i+1] - pos[i]`？这里的 i 从 1 到 m-1，为什么不是到 m？"
3. "如果有线段完全落在两个离散坐标之间——这种情况可能发生吗？"

## 🖥️ 板书/屏幕要点
- **离散化+差分流程**：收集端点→离散化→差分→还原→映射回原始长度
- **差分**：`diff[L]++; diff[R]--;`（区间 [L, R)）
- **统计长度**：`ans += pos[i+1] - pos[i];`（不是加 1！）
- **复杂度**：O(n log n)（排序），其余 O(n)

## ✏️ 随堂练习
3 条线段：[1,5], [3,8], [10,12]。手工走一遍离散化+差分流程，验证覆盖总长度 = 10。

## 📌 小结
离散化+差分是处理大范围区间覆盖的标准套路。操作在压缩坐标上，答案映射回原始长度。最重要的坑：统计长度时用 `pos[i+1]-pos[i]`，不是 1。

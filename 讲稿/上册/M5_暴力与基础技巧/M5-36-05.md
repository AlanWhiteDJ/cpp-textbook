## 📝 讲稿正文

这节课是离散化的最后一课——我们学一下双向映射，然后做综合练习。

有时候你需要在离散化后的下标和原始坐标之间来回转换。比如"电影放映"问题：n 场电影，每场有开始时间 l、结束时间 r、愉悦值 w，同一时间只能看一场。求最大总愉悦值。

这题要用 DP——但时间可能是 10⁹ 级别。离散化！把所有 l 和 r 收集起来排序去重，时间点变成 1~2n。DP 在离散化后的时间点上进行。

关键在于——你需要保留去重数组 `pos[]` 来做反向映射：

| 方向 | 方法 |
|------|------|
| 原始坐标 → 离散下标 | `lower_bound(pos+1, pos+m+1, x) - pos` |
| 离散下标 → 原始坐标 | `pos[idx]` |

看到了吗？`pos[idx]` 直接查回去！这就是为什么去重数组不能丢弃。

除了差分和 DP，离散化还常用于：
- **前缀和**：坐标很大但数据点少的区间求和
- **扫描线**：处理矩形面积并（结合线段树）
- **搜索**：把大棋盘上的稀疏棋子坐标压缩

离散化的思想在竞赛中无处不在——任何"坐标太大开不下数组"的场景，第一时间就应该想到离散化。

好，我们来总结离散化模块：

核心三步：`sort` → `unique` → `lower_bound`。目的是把稀疏的大坐标映射到紧凑的小下标。

关键注意点：
1. `unique` 前必须 `sort`
2. `lower_bound` 返回值要减去数组首地址
3. 统计长度时用 `pos[i+1] - pos[i]`，不是 1
4. 保留去重数组 `pos` 用于反向映射

给你一个判断条件：**当坐标范围 > 10⁷ 而数据量 ≤ 2×10⁵ 时，考虑离散化**。这是一条经验法则，可以帮你快速做出判断。

下个模块我们将学习高精度运算——当数字大到 `long long` 也装不下时，怎么用字符串模拟手工计算。

## 🖥️ 板书/屏幕要点
- **双向映射**：`pos[idx]` 反查原始坐标
- **应用场景**：差分、前缀和、DP、扫描线、搜索
- **经验法则**：坐标 > 10⁷ 且数据 ≤ 2×10⁵ → 离散化
- **四注意**：sort 先、减 b、用 pos[i+1]-pos[i]、保留 pos

## ✏️ 随堂练习
给出 n=3 条线段 `{[1,100], [50,200], [150,300]}`，用离散化+差分求至少被 2 条线段覆盖的总长度。

## 📌 小结
离散化是"坐标压缩"的万能工具。三步走 + 双向映射 = 解决大范围坐标问题。保留去重数组用于反查。当坐标范围远超数据量时，离散化是你的第一选择。

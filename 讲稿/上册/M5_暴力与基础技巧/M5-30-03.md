# 第30章 第3课时：勾股数——用平方关系减少枚举
**时长**：15-20分钟 | **对应教材**：第30章 30.4-30.5

## 🎯 本课时目标
- 掌握勾股数问题的枚举优化技巧
- 理解"用开方反推第三变量"的思路
- 复习水仙花数，体会"单重枚举"的优雅

## 📝 讲稿正文

上节课我们用百钱百鸡演示了"能推导的变量不枚举"。这节课再看一个更妙的例子——勾股数。

勾股数是什么？就是满足 a² + b² = c² 的三个整数，比如 3、4、5。题目要求找出 100 以内所有的勾股数，且 a < b < c。

如果直接用三重循环：

```cpp
for (int a = 1; a <= 100; a++)
    for (int b = 1; b <= 100; b++)
        for (int c = 1; c <= 100; c++)
            if (a*a + b*b == c*c && a < b && b < c)
                cout << a << " " << b << " " << c << endl;
```

100³ = 100 万次。还行，但能更好吗？

注意！c 是由 a 和 b 决定的——c = √(a² + b²)！我们根本不用枚举 c，直接算出来就行：

```cpp
for (int a = 1; a <= 100; a++) {
    for (int b = a + 1; b <= 100; b++) {   // b > a
        int c2 = a * a + b * b;
        int c = (int)(sqrt(c2) + 0.5);     // 开方取整
        if (c <= 100 && c * c == c2) {     // 验证是否完全平方数
            cout << a << " " << b << " " << c << endl;
        }
    }
}
```

从三重降到两重！次数约 100×100 = 1 万次，比 100 万快 100 倍。

这里有个小细节要注意——`sqrt(c2) + 0.5` 是干嘛的？（停顿让学生想）因为 `sqrt` 返回浮点数，比如 sqrt(25) 可能返回 4.9999999，转成 int 就变 4 了。加 0.5 是为了四舍五入，防止浮点精度问题。然后再用 `c * c == c2` 验证——双重保险。

你们注意到没有，我们还顺便缩小了 b 的范围：`b = a + 1`，因为题目要求 a < b。每多一个约束条件，我们就少枚举一些无意义的组合。

现在回头看看水仙花数。为什么水仙花数不需要三重循环分别枚举百位、十位、个位？（停顿）因为三位数天然就是连续的 100~999，一行 `for (n=100; n<=999; n++)` 就够了。然后通过 n/100、n/10%10、n%10 拆出各位。**能一行枚举的，不拆成多层**——这也是枚举优化的一个重要原则。

## 🖥️ 板书/屏幕要点
- **勾股数优化**：三重→两重，c = √(a²+b²)
- **浮点安全**：`(int)(sqrt(c2) + 0.5)` + `c*c == c2` 双重验证
- **水仙花数**：单重循环 100~999，拆位求立方和

## ✏️ 随堂练习
修改勾股数程序，只输出 a+b+c=1000 的那一组勾股数（提示：限定 a, b 范围更小）。

## 📌 小结
勾股数用开方反推 c，三重变两重，从 100 万降到 1 万。水仙花数单重循环搞定。优化的本质始终是：能不枚举的就不枚举，能推导的就算出来。

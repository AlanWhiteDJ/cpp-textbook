## 📝 讲稿正文

很多模拟题涉及**环形结构**——n 个人围成一圈、n 盏灯围成一圈……在数组里，下标从 n-1 再往前走一步应该回到 0。怎么办？用**取模**！

来看一道经典题——玩具谜题（NOIP 2016 提高组）。n 个小人围成一圈，每个小人朝内或朝外。给定若干指令"向左/向右数 s 个人"，问最后指向谁。

这道题的难点是：**朝向和左右方向会产生四种组合**。朝内的人"向左"是顺时针，"向右"是逆时针；朝外的人反过来。如果写四重 if-else 会非常臃肿。

聪明的做法是用**异或**。令 `dir[pos]` = 0 表示朝内、1 表示朝外，`a` = 0 表示向左、1 表示向右。则 `dir[pos] ^ a` = 0 表示顺时针，1 表示逆时针。一行搞定四个分支！

```cpp
int n, m;
int dir[100005];      // 0=朝内，1=朝外
string name[100005];

int main() {
    cin >> n >> m;
    for (int i = 0; i < n; i++) cin >> dir[i] >> name[i];

    int pos = 0;
    for (int i = 0; i < m; i++) {
        int a, s;
        cin >> a >> s;
        if ((dir[pos] ^ a) == 0)  // 顺时针
            pos = (pos + s) % n;
        else                       // 逆时针
            pos = (pos - s % n + n) % n;
    }
    cout << name[pos] << endl;
}
```

来看环形移动的两个核心公式：
- **顺时针走 s 步**：`pos = (pos + s) % n`
- **逆时针走 s 步**：`pos = (pos - s % n + n) % n`

为什么逆时针要多加一个 `+ n`？（停顿让学生想）因为 C++ 中负数的 `%` 结果可能是负数。比如 `-3 % 5 = -3`，但我们需要的是 2。所以写成 `(-3 + 5) % 5 = 2`。`+ n` 就是为了保证结果非负。

这个公式要像九九乘法表一样背下来——竞赛里只要碰到环形结构，这两个公式一定会用到。

还有一个细节：`pos - s % n`，为什么是 `s % n` 而不是直接 `s`？因为 s 可能大于 n——绕了好几圈，取模后实际等效于走了 `s % n` 步。

环形模拟的套路就是：**取模处理下标循环 + 异或或 if 处理方向判断**。掌握了这个，围圈报数、圆圈游戏这些问题都能搞定。

## 🖥️ 板书/屏幕要点
- **顺时针**：`pos = (pos + s) % n`
- **逆时针**：`pos = (pos - s % n + n) % n`
- **方向合并**：`dir[pos] ^ a` → 0=顺时针, 1=逆时针
- **`+n` 防负**：C++ 负数取模为负，加 n 保证非负

## ✏️ 随堂练习
n=7 个人围成一圈，编号 0~6。从 0 开始，走以下指令：(左,3), (右,2), (左,5)。输出最终位置。

## 📌 小结
环形模拟的核心是取模。顺时针加、逆时针减，再加 n 防负。异或可以优雅地合并方向判断。这两个公式要烂熟于心，因为环形问题在竞赛中无处不在。

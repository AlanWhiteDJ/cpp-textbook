# 第31章 第4课时：时间模拟——进位套路与字符处理
**时长**：15-20分钟 | **对应教材**：第31章 31.5

## 🎯 本课时目标
- 掌握时间模拟中"进位"的标准套路
- 学会用 `cin` 读入带分隔符的格式化输入
- 理解 `printf` 格式化输出（补零）

## 📝 讲稿正文

这节课我们来学一个竞赛里超级常见的问题类型——时间模拟。比如"从 08:30 开始，经过 n 分钟后是几点几分？"

这个问题看着简单，但有一个关键套路——**进位**。分钟加到 60 要进位给小时，小时到 24 要归零。这和你们小学学的"满十进一"是一个道理。

首先看怎么读入 "08:30" 这种格式。如果你直接用 `cin >> hour >> minute`，hour 读到 8，但 minute 读到的是冒号":"——就出错了！

怎么办？用一个 `char` 变量把冒号"吞掉"：

```cpp
int hour, minute, pass;
char colon;
cin >> hour >> colon >> minute;  // colon 吞掉 ':'
cin >> pass;                     // 经过的分钟数
```

小时和分钟读进去了，冒号被 `colon` 吃了。这个技巧在处理"hh:mm:ss"这种格式时非常实用。

接下来是进位的核心套路：

```cpp
minute += pass;               // 总分钟数
hour += minute / 60;          // 溢出的小时
minute %= 60;                 // 剩余的分钟
hour %= 24;                   // 24 小时制循环

printf("%02d:%02d\n", hour, minute);
```

注意这个 `printf("%02d", hour)`——`%02d` 表示"输出整数、至少两位、不够补零"。这样 8:05 会显示成 "08:05" 而不是 "8:5"。

这个进位套路可以**通吃所有进制转换**！秒→分钟（60 进制）、分钟→小时（60 进制）、小时→天（24 进制）、天→月（看月份），套路都是：

```
大单位 += 小单位 / 进制
小单位 %= 进制
```

你可以把这个套路用在任何"单位转换"的场景——钱的分→角→元（10 进制），时间的秒→分→时（60 进制），还有英尺→英里、盎司→磅……全是同一套！

来，我们一起口头走一遍：从 23:45 开始，经过 30 分钟。45+30=75 分钟，75/60=1 小时，75%60=15 分钟，23+1=24 小时，24%24=0。所以是 00:15。对不对？

## 🖥️ 板书/屏幕要点
- **吞分隔符**：`char c; cin >> h >> c >> m;`
- **进位套路**：`大单位 += 小单位/进制; 小单位 %= 进制;`
- **补零输出**：`printf("%02d:%02d", h, m);`
- **通吃所有进制**：60 进制（时间）、24 进制（天）、10 进制（金额）

## ✏️ 随堂练习
写程序输入起始时间 "HH:MM" 和经过分钟数 n，输出结束时间（24 小时制）。测试：23:45 + 30 → 00:15；08:30 + 200 → 11:50。

## 📌 小结
时间模拟的核心是进位套路：小单位溢出进大单位。用 `char` 变量吞掉输入中的分隔符。`printf("%02d")` 实现补零输出。这个进位套路通吃所有单位转换问题。

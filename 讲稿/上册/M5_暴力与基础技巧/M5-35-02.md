# 第35章 第2课时：差分数组——区间修改的 O(1) 神器
**时长**：15-20分钟 | **对应教材**：第35章 35.2

## 🎯 本课时目标
- 理解差分数组的定义：前缀和的逆运算
- 掌握区间修改的两步操作：`d[l]+=v, d[r+1]-=v`
- 学会差分构建、操作、还原的完整流程

## 📝 讲稿正文

上节课学了前缀和——区间查询 O(1)。但如果有另一种需求：**多次区间修改**（比如把 [l, r] 每个元素都加 val），然后再查询，怎么办？

暴力做法：每次修改用循环把 [l, r] 的每个元素加 val。m 次修改 = O(mn)，超时。

这时候你需要前缀和的"逆运算"——**差分**。

什么是差分？对于数组 a，它的差分数组 d 定义为：
- d[1] = a[1]
- d[i] = a[i] - a[i-1]（i > 1）

反过来——a 是 d 的前缀和！a[i] = d[1] + d[2] + ... + d[i]。

差分的神奇之处在于：**要对区间 [l, r] 每个元素都加 val，只需要改两个位置！**

```
d[l] += val;      // 从 l 开始，增量生效
d[r+1] -= val;    // 从 r+1 开始，增量取消
```

为什么？因为 a[i] 是 d[1..i] 的和。d[l] 加了 val 后，所有 a[i≥l] 都加了 val；d[r+1] 减了 val 后，所有 a[i≥r+1] 就不受影响了。所以最终只有 a[l..r] 加了 val。

来看完整代码：

```cpp
int d[100005] = {0};

// 构建差分
d[1] = a[1];
for (int i = 2; i <= n; i++)
    d[i] = a[i] - a[i - 1];

// 区间 [l, r] 全部 +val
void add(int l, int r, int val) {
    d[l] += val;
    d[r + 1] -= val;
}

// 还原：对 d 做前缀和
for (int i = 1; i <= n; i++)
    a[i] = a[i - 1] + d[i];  // 这里的 a 已经是修改后的值了
```

注意数组大小——`d[r+1]` 可能访问到 `d[n+1]`，所以差分数组至少要开到 `n+2`。

我打个比方：差分就像**记账**。a[i] 是你第 i 天的存款余额，d[i] 是"第 i 天存了多少钱"。当你在 3 月 1 日~10 日每天多存 100 元——记两笔账：d[1]+=100（从 1 号开始每天+100），d[11]-=100（11 号开始取消）。重新"轧账"一次（前缀和），就得到了更新后的全部余额。

差分复杂度：每次区间修改 O(1)，最后还原 O(n)。m 次修改 + 1 次还原 = O(m+n)，远优于暴力 O(mn)。

## 🖥️ 板书/屏幕要点
- **差分定义**：d[1]=a[1], d[i]=a[i]-a[i-1]
- **区间加 v**：`d[l]+=v; d[r+1]-=v;`
- **还原**：对 d 做前缀和 → 得到修改后的 a
- **数组大小**：开到 n+2，防 d[n+1] 越界

## ✏️ 随堂练习
初始数组全为 0，n=10。进行 3 次操作：[1,5]+3, [3,8]+2, [6,10]+1。用差分还原最终数组。

## 📌 小结
差分是前缀和的逆运算。区间修改只需改两个位置——d[l] 加、d[r+1] 减。操作完做一次前缀和还原。m 次修改 + 1 次还原 = O(m+n)。

# 第34章 第1课时：双指针的思想——为什么两根手指比一根快
**时长**：15-20分钟 | **对应教材**：第34章 34.1-34.2

## 🎯 本课时目标
- 理解双指针的核心思想：两个指针协同移动，避免重复检查
- 掌握对撞指针的基本形态和"每次排除一批"的哲学
- 写出有序数组两数之和的双指针解法

## 📝 讲稿正文

同学们，前面我们学了枚举和模拟——很多问题用暴力法 O(n²) 能解决。但竞赛题有时 n=10⁵，O(n²) 就是 100 亿次——铁定超时。有没有办法把 O(n²) 降到 O(n)？

有！**双指针**。

我讲个形象的故事。有一条长绳上串着很多珠子，数字从小到大排列。你的任务是找到两颗珠子，数字之和等于 100。

最笨的办法：左手拿一颗，右手把剩下的全摸一遍。然后左手换一颗，右手再摸一遍……总共摸 n² 次。

聪明的方法：左手放最左（最小数），右手放最右（最大数）。计算两数之和：
- 和等于 100 → 找到了！
- 和小于 100 → 左手往右移（因为左边太小了，需要更大的数）
- 和大于 100 → 右手往左移（因为右边太大了，需要更小的数）

每步排除一批不可能的组合，两根手指各走一遍，总共最多 O(n) 步。

来看代码：

```cpp
bool twoSum(int a[], int n, int target) {
    int l = 0, r = n - 1;
    while (l < r) {
        int sum = a[l] + a[r];
        if (sum == target) {
            cout << a[l] << " " << a[r] << endl;
            return true;
        } else if (sum < target) {
            l++;  // 太小了，左指针右移
        } else {
            r--;  // 太大了，右指针左移
        }
    }
    return false;
}
```

为什么可以放心地 `l++` 或 `r--`？关键在"有序"二字。如果 `a[l] + a[r] < target`，那么 a[l] 和任何在 r 左边的数之和只会**更小**。a[l] 已经没有任何希望了——果断放弃它。

同样，如果和太大了，a[r] 和任何在 l 右边的数之和只会更大——a[r] 也可以放弃了。

这就是对撞指针的哲学：**每次移动都排除一批不可能，从不后悔、从不回头**。左右指针各最多走 n 步，总复杂度 O(n)。

⚠️ 但有一个前提：**数组必须有序**。如果无序怎么办？先用 `sort` 排序——总复杂度 O(n log n)，依然远优于 O(n²)。

双指针不是一种数据结构，而是一种**解题套路**：用两个变量标记数组位置，根据条件协调移动，一次遍历解决二次遍历的问题。

## 🖥️ 板书/屏幕要点
- **对撞指针**：l=0, r=n-1 → 相向而行
- **移动规则**：sum<target → l++（需要更大）；sum>target → r--（需要更小）
- **复杂度**：O(n²) → O(n)，前提是有序
- **哲学**：每次移动排除一批不可能

## ✏️ 随堂练习
写对撞指针解决"两数之和"。测试：数组 `{1,3,5,7,9,11}`，target=10，应输出 1 和 9 或 3 和 7。

## 📌 小结
对撞指针用两根手指从两端向中间走，凭"有序"做决策——小了左移、大了右移。每步排除一批，O(n) 解决 O(n²) 问题。记住：每次移动都不可逆、不后悔。

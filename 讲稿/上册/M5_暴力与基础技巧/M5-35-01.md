# 第35章 第1课时：前缀和——预处理一次，查询一步到位
**时长**：15-20分钟 | **对应教材**：第35章 35.1

## 🎯 本课时目标
- 理解前缀和的核心思想：用空间换时间
- 掌握一维前缀和的构建和区间查询公式
- 建立"先算后用"的思维模式

## 📝 讲稿正文

同学们，双指针帮我们把 O(n²) 降到 O(n)。但如果问题是"多次查询区间和"呢？比如给你一个数组，要回答 10 万次"区间 [l, r] 的元素和是多少"。每次用循环累加？每次 O(n)×10 万次 = 太慢了。

聪明人怎么做？提前算好一个"累计表"——前 i 个元素的和是 `pre[i]`。那么区间 [l, r] 的和就是 `pre[r] - pre[l-1]`，**一次减法搞定**！

这就叫**前缀和**。先花 O(n) 时间预处理，之后每次查询 O(1)。

我打个比方——前缀和就像汽车里程表。你从第 0 公里出发，开到第 5 公里时里程表显示 5km，开到第 20 公里时显示 20km。第 5 公里到第 20 公里的路程是多少？20 - 5 = 15km。这就是 `pre[20] - pre[4]`——注意这里减的是 `pre[4]`，因为第 5 公里本身要算进去。

来写代码：

```cpp
int a[100005], pre[100005];
int n;

// 构建前缀和
pre[0] = 0;                          // 空前缀和！
for (int i = 1; i <= n; i++) {
    pre[i] = pre[i - 1] + a[i];
}

// 查询区间 [l, r] 的和
int query(int l, int r) {
    return pre[r] - pre[l - 1];
}
```

三个要点：
1. `pre[0] = 0`——这步不能省略。没有它，`query(1, r)` 就会变成 `pre[r] - pre[0]` 但 `pre[0]` 没定义。
2. 下标从 **1 开始**——这是为了和 `pre[0]` 对齐，方便公式。
3. 区间 [l, r] 的公式是 `pre[r] - pre[l-1]`，不是 `pre[r] - pre[l]`——前者包含了 a[l]，后者丢了 a[l]。

前缀和的精髓是四个字：**先算后用**。就像超市购物——收银员不会把每位顾客的商品都从第一件开始累加，而是直接读取扫描枪里的"累计金额"。因为累计金额就是实时更新的前缀和。

复杂度对比：暴力 q 次查询 = O(nq)，前缀和 = O(n+q)。q=10⁵, n=10⁵ 时，暴力要 10¹⁰ 次，前缀和只要 2×10⁵ 次——快了 5 万倍。

## 💬 互动提问
1. "为什么 `pre[0]` 要设为 0？如果忘写了，查询 `query(1, r)` 会发生什么？"
2. "如果数组下标从 0 开始（不是 1），前缀和公式要改成什么？"
3. "前缀和的'空间换时间'体现在哪里？多用了多少空间？"

## 🖥️ 板书/屏幕要点
- **构建**：`pre[0]=0; for(i=1;i<=n;i++) pre[i]=pre[i-1]+a[i];`
- **查询**：`sum(l,r) = pre[r] - pre[l-1]`
- **下标从 1 开始**，配合 `pre[0]=0`
- **复杂度**：预处理 O(n)，查询 O(1)，总 O(n+q)

## ✏️ 随堂练习
输入数组 `{3,1,4,1,5,9}`，构建前缀和，然后查询区间 [2,4] 和 [3,6] 的和。

## 📌 小结
前缀和用空间换时间——预处理 O(n)，每次查询 O(1)。区间查询公式 `pre[r]-pre[l-1]` 要像九九乘法表一样记住。`pre[0]=0` 是必须的。

## 📝 讲稿正文

这节课我们来个"总复习+实战"。前面四节课学了这么多枚举技巧，今天把它们串起来，然后做两道有挑战的题。

先回顾枚举的**三步模板**：`for 生成候选 → if 检查条件 → 符合就输出`。无论题目怎么变，这三步永远不变。

再回顾**三大优化方法**：
- 第一，**缩范围**——根据题目约束，把循环起止范围算到最紧。公鸡最多 20 只，a ≤ 33……
- 第二，**减变量**——能用等式推导的变量不枚举。百钱百鸡的小鸡、勾股数的 c、换硬币的 5 元硬币……
- 第三，**早退出**——找到答案立刻 break，不浪费后续循环。

现在来第一道实战题——**回文平方数**。找出 1~10000 中所有满足"自己是回文数，且自己的平方也是回文数"的数。比如 1²=1，11²=121，都是回文。

思路分两步：先写一个判断回文的函数，再枚举每个数。

```cpp
bool isPal(int n) {
    int rev = 0, tmp = n;
    while (tmp) {
        rev = rev * 10 + tmp % 10;
        tmp /= 10;
    }
    return rev == n;
}

int main() {
    for (int i = 1; i <= 10000; i++) {
        if (isPal(i) && isPal(i * i)) {
            cout << i << " → " << i * i << endl;
        }
    }
}
```

这里"翻转数字判回文"是个很常用的小技巧，不用转字符串，纯整数运算就搞定了。

再来一道——**四平方和定理**。输入 n，输出四个不超过 √n 的整数 a,b,c,d 使 a²+b²+c²+d²=n。

提示是"枚举前三个，d 自动得出"。你们试试？枚举 a、b、c，算 d² = n - a² - b² - c²，然后判断 d 是不是整数且在范围内。

最后提醒大家几个**常见错误**：枚举范围漏边界（`<`写成`<=`）、能推导的量还在循环、浮点数用`==`比较、小鸡忘了判`%3==0`……这些坑大家都踩一遍才能记住。

## 🖥️ 板书/屏幕要点
- **枚举模板**：for生成 → if检查 → 输出
- **优化三法**：缩范围、减变量、早退出
- **回文技巧**：翻转数字：`rev = rev*10 + tmp%10`
- **常见错误**：边界、浮点、整除检查、推导变量

## 📌 小结
枚举是算法世界的第一课。三步模板 + 三大优化 = 解决大多数基础枚举题。关键是建立"拿到题先想枚举"的习惯，同时注意估算枚举次数，避免超时。从下节课开始，我们学另一种思路——模拟。

# 第31章 第1课时：模拟——把题目的规则翻译成代码
**时长**：15-20分钟 | **对应教材**：第31章 31.1-31.2

## 🎯 本课时目标
- 理解模拟与枚举的区别：模拟是按规则一步步演算，枚举是列出所有可能
- 掌握读模拟题的四步法：圈关键词→定义变量→写步骤→验证样例
- 建立"题目怎么写，代码就怎么翻"的模拟直觉

## 📝 讲稿正文

上几节课我们学了枚举——把所有可能情况列出来逐个试。但有一类题目不要求你"找所有解"，而是要求你**按规则一步步演算**。就像下棋、打牌——每一步的决策都依赖之前的状态。

来看一个真实的竞赛故事。阿杰第一次参加信息学竞赛，看到一道题：小明初始生命值 100，第 1 回合受到 10 点伤害、恢复 5 点，第 2 回合受到 15 点伤害、恢复 8 点……共 10 回合。问第几回合生命值降到 0 以下？

阿杰的第一反应："列方程！"算了半天，越来越乱。旁边的队友 3 分钟写完交卷了。阿杰凑过去看——对方没列任何方程，就写了一个循环：

```cpp
int hp = 100;
for (int i = 1; i <= 10; i++) {
    hp -= damage[i];
    hp += recover[i];
    if (hp <= 0) { cout << i; break; }
}
```

"就这样？"阿杰震惊了。队友说："题目怎么写，代码就怎么翻——这就是模拟啊。"

这个故事告诉我们什么？（停顿）不是每道题都需要巧妙的数学公式。很多时候，**把题目的规则一字一句翻译成代码，就是标准答案**。

模拟就像做菜。菜谱是题目，你是翻译者（把中文翻成 C++），计算机是厨师。只要菜谱没有歧义，菜就做得出来——不需要厨师自己有创意。

那怎么读模拟题？很多同学一看到题干长就懵了。我教你们一个四步法：

第一步，**圈关键词**。在草稿纸上圈出"谁和谁比""怎么判输赢""循环几次"。

第二步，**定义变量**。列出所有需要的变量和类型——计分器、计数器、状态标记……

第三步，**写步骤**。用注释或伪代码搭骨架。比如 `// 循环5轮 {输入→判断→加分}`。

第四步，**验证样例**。用题目给的样例，拿笔在纸上按你的逻辑走一遍。如果纸上算出来的都不对，代码更不可能对。

记住一句话：**先纸上谈兵，再动手敲代码。**这个习惯能省掉你一半的调试时间。

## 🖥️ 板书/屏幕要点
- **模拟 = 题目规则 → C++语句 → 计算机逐步执行**
- **四步法**：① 圈关键词 ② 定义变量 ③ 写步骤（注释骨架） ④ 验证样例
- **核心心态**：题目怎么写，代码就怎么翻

## ✏️ 随堂练习
给你一道模拟题：每天存钱，第 1 天存 1 元，第 2 天存 2 元……第 30 天存 30 元。用四步法（写纸上）分析，然后写代码输出 30 天共存了多少钱。

## 📌 小结
模拟是把题目规则一步步翻译成代码。跟枚举不同——枚举是"列出所有可能性"，模拟是"按规则一步步走"。读模拟题用四步法：圈关键词→定义变量→写步骤→验证样例。记住：先纸上谈兵，再敲代码。

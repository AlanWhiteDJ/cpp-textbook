# 第11章 第3课时：累加模式——从 1 加到 100 的传奇
**时长**：15-20分钟 | **对应教材**：第11章 · 11.5节

## 🎯 本课时目标
- 掌握 for 循环的累加模式：累加器初始化为 0，每轮累加
- 理解 `sum += i` 这种复合赋值运算符的含义
- 能用累加模式解决求和类问题

## 📝 讲稿正文

同学们好！今天讲 for 循环最重要的应用模式之一——**累加**。

我先讲一个真实的数学故事。高斯上小学的时候，老师想偷懒，出了一道题："全班算 1+2+3+……+100，算完才能下课。"老师以为这帮小屁孩要算半天。结果高斯几秒钟就举手了：5050。

老师震惊了。高斯的诀窍是：1+100=101，2+99=101……一共 50 对，101×50=5050。

（在黑板上写高斯的方法）

你们没有高斯的数学天赋？没关系——你们有计算机。来看 for 循环怎么算：

```cpp
int sum = 0;                     // 累加器，从 0 开始！
for (int i = 1; i <= 100; i++) {
    sum = sum + i;               // 每次把 i 加进 sum
}
cout << sum << endl;             // 输出 5050
```

（在黑板上画一个模拟运行的表格）

来，我们一起在脑子里模拟这段代码的"内心活动"：

```
初始：sum = 0
i=1：sum = 0 + 1 = 1
i=2：sum = 1 + 2 = 3
i=3：sum = 3 + 3 = 6
i=4：sum = 6 + 4 = 10
...
i=100：sum = 4950 + 100 = 5050
```

核心是这一行：`sum = sum + i`。初看是不是有点奇怪——sum 等于 sum 加 i？等号两边都有 sum？

不奇怪。你要按"先算右边，再赋值给左边"来理解。等号右边是"旧的 sum 加上 i"，等号左边是"新的 sum"。每次循环，新的 sum 都比旧的 sum 多了当前的 i。就像你的存钱罐——每次放一枚硬币进去，存钱罐里的钱 = 原来有的钱 + 新放进去的钱。

C++ 里有一个专门的简写：`sum += i`，等价于 `sum = sum + i`。这个叫**复合赋值运算符**。类似的还有：`-=` `*=` `/=` `%=`。

```cpp
sum += i;    // sum = sum + i;
cnt -= 1;    // cnt = cnt - 1;
fact *= i;   // fact = fact * i;
```

复合赋值不仅少打字，更重要的是它表达了一个明确的意思：**"在原有基础上增加"**——代码的语义更清晰。

好，累加模式的本质是：

```
累加器 = 0;                      // 从 0 开始
for (...) {
    累加器 += 这一轮要加的值;     // 每轮累加
}
```

光会加 1 到 100 不算本事，我们来几个变体：

```cpp
// 求 1~n 中所有偶数的和
int sum = 0;
for (int i = 2; i <= n; i += 2) {
    sum += i;
}

// 求 1² + 2² + 3² + ... + n²
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i * i;
}

// 求 1 + 1/2 + 1/3 + ... + 1/n（浮点版）
double sum = 0.0;
for (int i = 1; i <= n; i++) {
    sum += 1.0 / i;    // 注意 1.0 不是 1！整数除法会截断
}
```

细心的同学发现最后一个例子里写的是 `1.0 / i` 而不是 `1 / i`。为什么？因为 `1 / i` 是整数除法——当 i > 1 时，1/i 永远是 0！写成 `1.0 / i` 让 1.0 成为 double 类型，触发浮点除法，得到正确的小数结果。

好，现在动手练习。三道题任选：
1. 输入 n，求 1+2+...+n 的和
2. 输入 n，求 1~n 中所有奇数的和
3. 输入 n，求 1²+2²+3²+...+n²

（给学生练习时间，选一道最简单的即可，快的做第二道）

小结：累加模式 = 累加器从 0 开始 + 每轮 `sum += 当前值`。这是循环里最常用的模式——记住 `int sum = 0;` 这个初始化，少写就等于白算。

## 🖥️ 板书/屏幕要点
- 累加模式：`int sum = 0; for(...) sum += i;`
- `sum = sum + i`：右边旧和+i → 赋给左边新和
- 复合赋值：`+=` `-=` `*=` `/=` `%=`
- ⚠️ 整数除法截断：想得小数，分子分母至少一个是浮点类型

## ✏️ 随堂练习
输入一个正整数 n，计算 1+2+...+n 的和并输出。试试 n=1000（正确结果 500500）。再试试如果 sum 忘记初始化为 0 会怎样。

## 📌 小结
- 累加器必须初始化为 0
- `sum += i` 等价于 `sum = sum + i`
- 累加模式可推广到平方和、倒数和等各种求和
- 浮点求和注意用 `1.0 / i` 避免整数截断

# 第9章 第1课时：岔路口——认识 if 语句
**时长**：15-20分钟 | **对应教材**：第9章 · 9.1节

## 🎯 本课时目标
- 理解什么是程序的分支，为什么程序需要做判断
- 掌握 `if` 语句的基本语法和花括号的作用
- 能独立写出"条件成立则执行"的单分支程序

## 📝 讲稿正文

同学们好！欢迎来到 M2 模块——「选择与重复」。从今天开始，我们写的程序不再是一条路走到黑了。

（停顿，看向学生）

大家回忆一下，每天放学回家经过红绿灯的时候，你的大脑在做什么？"绿灯亮了，我就走；红灯亮了，我就停。"对不对？这就是一个判断过程。你的大脑收到了一个信号——灯的颜色——然后根据这个信号决定了"走"还是"停"。

那如果让你把这个逻辑写成程序呢？你会怎么写？你先别急着想代码，先用大白话：

"如果现在是绿灯，就显示'可以通行'；否则，就显示'请等待'。"

看到了吗？「如果……就……」这个句式，就是今天要学的 `if` 语句的核心。

好，我们来看 C++ 里最基础的岔路口——if 语句怎么写。它的格式非常简单：

```cpp
if (条件) {
    条件成立时执行的代码
}
```

翻译成人话就是：**如果括号里的条件成立，就做花括号里的事；否则，直接跳过这一段。**

来看一个具体的例子——判断一个数是不是正数。请打开你们的编程环境，跟着我一起敲：

```cpp
int n;
cin >> n;
if (n > 0) {
    cout << "这是一个正数" << endl;
}
cout << "判断结束" << endl;
```

（停顿，等学生敲完）

来，我们一起跑一遍。输入 `5`，回车——屏幕上输出"这是一个正数"，然后输出"判断结束"。对不对？

现在再输入 `-3`，回车——屏幕上只输出"判断结束"，上面的那行"这是一个正数"没出来。为什么？

因为 `-3 > 0` 这个条件是**假**的，所以 `if` 花括号里的代码被跳过了。程序直接跳过那段，继续往下走，执行了最后那行 `cout << "判断结束"`。

这个例子虽然简单，但它揭示了一个非常重要的概念：**程序不再是一条直线了**。它有了选择的能力。就像你走到一个岔路口，你要么走左边那条路，要么不走——if 语句就是告诉程序："看这个条件，决定走不走这条路。"

（在黑板上画一条竖线，然后在中间画一个菱形岔开）

有同学可能会问：如果花括号里只有一条语句，花括号能不能省略？答案是——语法上可以。但我强烈不建议你们省。为什么呢？因为省略花括号是无数 bug 的来源。你们刚开始学，养成"永远加花括号"的习惯，以后能少踩好多坑。

好，我们再看一个稍微复杂一点的例子。假设我们要判断一个年份是不是闰年——闰年的规则很简单：能被 400 整除就是闰年。

```cpp
int year;
cin >> year;
if (year % 400 == 0) {
    cout << year << "是闰年" << endl;
}
```

注意这里的 `year % 400`，`%` 是取余运算，M1 学过的。`year % 400 == 0` 的意思就是"year 除以 400 的余数等于 0"，也就是"year 能被 400 整除"。

记住：**等于号在 C++ 里是两个等号 `==`**，一个等号 `=` 是赋值。这个区别是初学者最容易犯的错误，你们一定要刻在脑子里。我们在后面的课时还会专门讲这个问题。

好了，现在请你们自己动手写一个小程序：输入一个整数，如果它是偶数，就输出"偶数"。

（给学生 2-3 分钟练习）

写出来的同学举手我看看。好，大部分同学都写出来了。你是怎么写的？`if (n % 2 == 0) { cout << "偶数" << endl; }`，非常棒！

来，我们小结一下这节课学到的：`if` 语句是程序里最基础的判断结构。它的格式是 `if (条件) { 代码 }`。条件成立就执行花括号里的代码，不成立就跳过。这是程序从"傻子"变成"聪明人"的第一步。

下节课我们要学 if-else——二选一的岔路口。两个方向都能走，但只能走一条。好，下课！

## 🖥️ 板书/屏幕要点
- `if (条件) { 代码; }` —— 单分支结构
- 条件成立 → 执行花括号内代码；条件不成立 → 跳过
- 程序流程图：直线 → 菱形（判断）→ 分叉
- 铁律：`==` 是比较，`=` 是赋值

## ✏️ 随堂练习
写一个程序：输入一个整数，如果它是负数，输出"这是负数"。然后用正数和负数分别测试你的程序。

## 📌 小结
- `if` 是最基础的分支结构，让程序有了"判断"的能力
- `if (条件) { 代码 }`，条件成立执行、不成立跳过
- 永远加花括号，这是好习惯的起点
- `==` 是比较相等，别和赋值 `=` 搞混

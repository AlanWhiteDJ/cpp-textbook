## 📝 讲稿正文

同学们好！今天是第11章的最后一节课。我们要学最后一个循环模式——**找最值**，然后画 for 循环的流程图，最后做全章总结。

先看一个问题：输入 n 个整数，找出其中最大的那个。你怎么做？

思路很简单——就像你在班里找最高的同学。你先记住第一个人的身高，然后一个一个看过去，看到更高的就替换掉记忆中的那个。写成代码：

```cpp
int n;
cin >> n;                        // 先读入数字的个数
int maxVal;
cin >> maxVal;                   // 读入第一个数，设为"当前最大"

for (int i = 1; i < n; i++) {    // 还剩 n-1 个数
    int num;
    cin >> num;
    if (num > maxVal) {
        maxVal = num;            // 发现更大的，更新！
    }
}
cout << "最大值：" << maxVal << endl;
```

注意几个细节：

第一，我们先读入第一个数单独当作"当前最大"，循环只跑 n-1 次。这样做的好处是 maxVal 总是有一个有效的初始值。如果你把 maxVal 初始化为 0，那当所有数都是负数时，0 会被错误地当作"最大"——因为它根本不在输入里。

第二，`if (num > maxVal)` 里的 `>` 是"严格大于"。如果你写 `>=`，那当出现相同最大值时，会用最后出现的那个——一般不影响结果，但 `>` 更符合直觉。

找最小值同理——把 `>` 改成 `<`，把 `maxVal` 改成 `minVal` 就行。

（板书找最值的模板）

找最值的思维是"竞争思维"——先设一个擂主，然后每个挑战者来打擂，打赢了就换人。

如果同时要找最大和最小呢？那就设两个变量，同时维护：

```cpp
int maxVal, minVal;
cin >> maxVal;
minVal = maxVal;             // 第一个数既是当前最大也是当前最小
for (int i = 1; i < n; i++) {
    int num; cin >> num;
    if (num > maxVal) maxVal = num;
    if (num < minVal) minVal = num;
}
```

好，接下来画 for 循环的流程图。上章我们画了 if-else 的流程图，现在给 for 循环画一个：

```
         ┌──────┐
         │ 开始 │
         └──┬───┘
            ▼
     ┌──────────────┐
     │ int i = 1    │  ← 初始化（只做一次）
     └──────┬───────┘
            ▼
        ╔═══════╗
   ┌───→║i<=5?  ║────否───→ 退出循环
   │    ╚═══╤═══╝
   │        │是
   │        ▼
   │  ┌───────────┐
   │  │ 循环体     │  ← 做你想做的事
   │  └─────┬─────┘
   │        ▼
   │  ┌───────────┐
   │  │ i++        │  ← 迭代
   │  └─────┬─────┘
   │        │
   └────────┘
```

记住这个流程：**初始化 → 判断 → 执行 → 迭代 → 判断 → 执行 → 迭代 → …… → 条件不满足 → 退出。**

判断发生在每次循环的开头——这意味着条件一开始就不成立的话，循环体一次都不执行。

现在，我们来做一个全章总结（请学生跟着一起在纸上写）。

第11章的核心是**重复**。围绕"重复"，我们学了：

1. **for 三要素**：初始化（从哪开始）、条件（什么时候停）、迭代（每步怎么变）
2. **循环变量**：在 for 里声明则作用域有限，在 for 外声明则可以延续
3. **多变量 for**：`for (int i=0, j=n; i<j; i++, j--)` 两端向中间
4. **四种经典模式**：
   - 累加：`int sum = 0; sum += i;`
   - 累乘：`long long fact = 1; fact *= i;`
   - 计数：`int cnt = 0; if(...) cnt++;`
   - 找最值：先设第一个，遇到更大/更小的就替换
5. **for 流程图**：初始化→判断→循环体→迭代→判断→……

## 🖥️ 板书/屏幕要点
- 找最值：先读第一个 → 循环 n-1 次 → 遇到更大/更小就更新
- for 流程图：初始化(1次) → 判断(每轮开头) → 循环体 → 迭代 → 判断...
- 全章知识树：三要素 → 循环变量 → 多变量 → 四种模式

## ✏️ 随堂练习
输入 n 和 n 个整数，找出最大值和最小值，输出它们的差。例如输入 `5 3 9 2 7 1`，最大值 9，最小值 1，差 8。

## 📌 小结
- 找最值：先设第一个为当前最值，循环中遇到更优就替换
- for 流程图核心：判断在每轮开头
- 第11章四种模式：累加、累乘、计数、最值
- 所有模式本质相同：变量初始值 → 循环中更新

## 📝 讲稿正文

同学们好！今天我们来挑战一道信息学竞赛的经典题——**哥德巴赫猜想验证**。

先讲背景。哥德巴赫猜想是数学史上最著名的未解问题之一：**任何一个大于 2 的偶数，都可以写成两个质数之和。** 比如 10 = 3 + 7 = 5 + 5，20 = 3 + 17 = 7 + 13。这个猜想至今没有被完全证明，但已经被计算机验证到了极大范围。

今天我们就来写一个程序，对于输入的偶数 n，找出两个质数使它们的和等于 n。

走三步选结构：

① **需要分岔吗？** 是！需要判断某个数是不是质数 → if-else

② **需要重复吗？** 是！需要在 2~n-2 范围内搜索 → for

③ **需要嵌套吗？** 是！外层枚举第一个质数、内层判断两个数是否都是质数

这道题本质是"在范围内搜索满足条件的数对"。和第 8 课时的二维搜索本质一样。

来，写代码。思路：枚举第一个数 i 从 2 到 n-2，如果 i 是质数且 n-i 也是质数，就找到了一组解。

```cpp
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    int n;
    cout << "请输入一个大于 2 的偶数：";
    cin >> n;

    if (n <= 2 || n % 2 != 0) {
        cout << "输入必须是大于 2 的偶数！" << endl;
        return 0;
    }

    bool found = false;
    for (int i = 2; i <= n / 2; i++) {    // 只需搜到 n/2，避免重复
        // 判断 i 是不是质数
        bool iIsPrime = true;
        if (i <= 1) iIsPrime = false;
        for (int j = 2; j * j <= i; j++) {
            if (i % j == 0) { iIsPrime = false; break; }
        }

        if (!iIsPrime) continue;     // i 不是质数，跳过

        int k = n - i;               // 另一个数
        // 判断 k 是不是质数
        bool kIsPrime = true;
        if (k <= 1) kIsPrime = false;
        for (int j = 2; j * j <= k; j++) {
            if (k % j == 0) { kIsPrime = false; break; }
        }

        if (kIsPrime) {
            cout << n << " = " << i << " + " << k << endl;
            found = true;
            break;                    // 找到一组就停
        }
    }

    if (!found) {
        cout << "未找到解（这在已验证范围内不可能发生）" << endl;
    }
    return 0;
}
```

（带着学生分析代码，指出关键部分）

来，走一遍 n=20 的执行过程：

- i=2：2 是质数 → k=18，18 不是质数 → 跳过
- i=3：3 是质数 → k=17，17 是质数！→ 输出 "20 = 3 + 17"，break

找到一组就停了。如果你想找所有解，去掉 break 和 found 标记就行。

注意这段代码有一个可以优化的地方——质数判断重复了。我们在判断 i 和 k 时各跑了一个质数判断循环。如果 i 从小到大枚举，k 从大到小——当 i 超过 n/2 时，就出现了重复。所以外层的 `i <= n/2` 避免了重复搜索。

代码中 `continue` 用得很妙：`if (!iIsPrime) continue;` —— i 不是质数就直接跳过，不用再去判断 k。这省了一半的计算量。

现在做一个扩展思考：如果题目要求找出"质数差最大的一组解"（即最小的 i 和最大的 k），那应该怎么找？

答案很简单：**从小到大搜，第一组找到的解就是质数差最大的。** 因为 i 最小 → k 最大 → 差最大。所以上面的代码已经天然满足这个需求了。

现在动手练习：修改程序，找出输入偶数 n 的**所有**哥德巴赫分解方案（而不是只找第一组）。

小结：哥德巴赫猜想验证 = 外层枚举第一个质数 + 内层质数判断。融合了 for 循环、质数判断、break、continue——是对 M2 能力的一次全面检验。

## 🖥️ 板书/屏幕要点
- 哥德巴赫猜想：大于 2 的偶数 = 两个质数之和
- 结构：for 枚举 i + 判断 i 和 n-i 是否质数
- 技巧：外层到 n/2，避免重复
- continue 跳过非质数 i，省计算量
- 找所有解：去掉 break 就行

## ✏️ 随堂练习

## 📌 小结
- 哥德巴赫猜想验证 = 枚举 + 双重质数判断
- 融合：for 循环 + 质数判断 + if + continue + break
- 搜索到 n/2 避免重复
- 是"搜索满足条件的数对"类问题的典型解法

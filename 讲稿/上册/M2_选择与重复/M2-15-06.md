## 📝 讲稿正文

同学们好！今天我们来攻克信息学竞赛里最经典的题目之一——**质数判断**。

先复习一下定义：质数（也叫素数）是大于 1 的整数，且只能被 1 和它本身整除。比如 2、3、5、7、11 是质数；4、6、8、9 不是质数。特别要注意：**1 不是质数！** 这是最常见的坑。

走三步选结构：

① **需要分岔吗？** 是！判断是否能整除 → if

② **需要重复吗？** 是！从 2 开始逐个试除 → 知道范围，用 for

③ **需要提前终止吗？** 是！找到一个因子就确定不是质数 → break

先写基础版——从 2 试到 n-1：

```cpp
int n;
cin >> n;

bool isPrime = true;               // 标记：假设是质数
if (n <= 1) isPrime = false;       // 1 和更小的不是质数

for (int i = 2; i < n; i++) {
    if (n % i == 0) {              // 找到了能整除的因子
        isPrime = false;           // 不是质数
        break;                     // 不用再试了
    }
}

if (isPrime)
    cout << n << " 是质数" << endl;
else
    cout << n << " 不是质数" << endl;
```

来，手动验证 n=17：i 从 2 试到 16，17 除以 2 到 16 都不整除 → isPrime 一直是 true → 输出"是质数"。正确！

n=15：i=3 时 15%3=0 → isPrime=false，break → 输出"不是质数"。正确！

这个版本是对的，但是**太慢了**。如果 n=1000000，你要试将近 100 万次！能不能少试一些？

（在黑板上写出：n=36 的因子对）

关键观察：如果 n 有一个大于 √n 的因子，必然对应一个小于 √n 的因子。比如 36 的因子有：1×36、2×18、3×12、4×9、6×6——大于 6 的因子 9、12、18、36 都有对应的小于 6 的搭档。

所以，**只需试到 √n！** 如果 2 到 √n 之间都没有能整除 n 的，那 √n 到 n-1 之间也不可能有。

在 C++ 里，我们可以避免用 `sqrt()` 函数（浮点数有精度问题），用 `i * i <= n` 来代替：

```cpp
bool isPrime = true;
if (n <= 1) isPrime = false;

for (int i = 2; i * i <= n; i++) {   // 等价于 i ≤ √n
    if (n % i == 0) {
        isPrime = false;
        break;
    }
}
```

`i * i <= n` 和 `i <= sqrt(n)` 完全等价——前者还是整数运算，没有浮点误差，是竞赛推荐写法。

来感受一下优化的威力。判断 1000000 是不是质数：
- 基础版：试 2 到 999999 → 约 100 万次
- 优化版：试 2 到 1000 → 约 1000 次

**快了 1000 倍！** 这就是算法优化的魅力。

（在黑板上写出 1,000,000 vs 1,000 的对比）

这个程序融合了第 11 章的 for 循环、第 14 章的 break、和 bool 标记变量——是 M2 知识的集中体现。

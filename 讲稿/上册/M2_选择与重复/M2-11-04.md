# 第11章 第4课时：累乘与计数——阶乘与统计模式
**时长**：15-20分钟 | **对应教材**：第11章 · 11.6节、11.7节

## 🎯 本课时目标
- 掌握累乘模式：累乘器从 1 开始，每轮累乘，计算阶乘
- 掌握计数模式：计数器从 0 开始，满足条件加 1
- 区分累加（初值0）、累乘（初值1）、计数（初值0）的初始化差异

## 📝 讲稿正文

同学们好！上节课我们学了累加模式。今天学两个新模式——累乘和计数。三个模式放在一起对比，你会发现它们的套路是一样的，只是初始值和操作不同。

先看累乘。累乘就是"把数不断乘上去"。最常见的例子是**阶乘**：n! = 1 × 2 × 3 × ... × n。

比如 5! = 1×2×3×4×5 = 120。来，用 for 循环实现：

```cpp
int n;
cin >> n;
long long fact = 1;              // 累乘器，从 1 开始！
for (int i = 1; i <= n; i++) {
    fact *= i;                   // 等价于 fact = fact * i;
}
cout << n << "! = " << fact << endl;
```

（在黑板上模拟运行 n=5 的情况）

```
初始：fact = 1
i=1：fact = 1 × 1 = 1
i=2：fact = 1 × 2 = 2
i=3：fact = 2 × 3 = 6
i=4：fact = 6 × 4 = 24
i=5：fact = 24 × 5 = 120
```

注意两个关键点。

第一，**累乘器从 1 开始，不是 0！** 为什么？0 乘以任何数都是 0——如果你把 fact 初始化为 0，不管循环跑多少轮，结果永远是 0。这跟累加刚好相反——累加从 0 开始加不受影响，累乘从 0 开始就全毁了。

口诀：**累加初始 0，累乘初始 1。**

第二，我用的是 `long long` 而不是 `int`。为什么？因为阶乘增长极其快——8! = 40320 已经接近 int 的边界（约 21000），13! = 6227020800 已经超出 int 范围。用 `long long` 能算到大约 20!。如果你的 n 更大，阶乘会溢出——这是数据类型的天花板，后面学到高精度算法才能突破。

好，接下来看第二个模式——**计数**。

场景：输入 10 个整数，统计其中有多少个正数、多少个负数。这个"统计个数"的需求用计数模式：

```cpp
int posCount = 0, negCount = 0;   // 两个计数器，都从 0 开始

for (int i = 1; i <= 10; i++) {
    int num;
    cin >> num;
    if (num > 0) {
        posCount++;               // 正数计数器 +1
    } else if (num < 0) {
        negCount++;               // 负数计数器 +1
    }
}
cout << "正数：" << posCount << " 个" << endl;
cout << "负数：" << negCount << " 个" << endl;
```

计数器模式的套路：**先建一个初始为 0 的变量，在循环里每次满足条件就加 1。** 就像一个"统计员"拿着计数器站在旁边——每看到一个符合条件的，就按一下计数器。循环结束，计数器上的数字就是总数。

计数模式里 `posCount++` 等价于 `posCount = posCount + 1`，也等价于 `posCount += 1`。`++` 是 M1 学过的自增运算符，在计数场景下特别自然。

来，我们对比一下今天学的三种模式：

| 模式 | 初始值 | 每轮操作 | 例子 |
|------|--------|----------|------|
| 累加 | 0 | `sum += x` | 求总和 |
| 累乘 | 1 | `fact *= x` | 求阶乘 |
| 计数 | 0 | `count++` | 统计个数 |

三个模式的本质一模一样：**一个变量，初始化为特定值，在循环里不断更新。** 区别只在于"怎么更新"和"初始值是多少"。

记住两句话：**累加初始 0，累乘初始 1，计数也初始 0。** 累乘是唯一初始值不是 0 的模式——因为它要的是乘法单位元 1，就像累加要的是加法单位元 0。

好，现在动手练习：写一个程序，输入 n 和 n 个整数，统计其中有多少个偶数。

进阶题：计算这 n 个整数的平均值（总和除以个数）。注意！平均值很可能是小数，要用 double 类型，并且做除法时注意类型转换。

（给学生练习时间）

小结：累乘从 1 开始用 `*=`，计数从 0 开始用 `++`。三种模式——累加、累乘、计数——是循环里最基础也最实用的套路，后面会反复遇到。

## 🖥️ 板书/屏幕要点
- 累乘模式：`long long fact = 1; for(...) fact *= i;`
- ⚠️ 累乘初始 1，不是 0！
- 计数模式：`int cnt = 0; if(...) cnt++;`
- 三种模式对比：累加(sum=0, +=) | 累乘(fact=1, *=) | 计数(cnt=0, ++)

## ✏️ 随堂练习
输入 n 和 n 个整数，统计其中偶数的个数。进阶：用这 n 个数的总和除以 n 求平均值（用 double）。

## 📌 小结
- 累乘：fact 初始化为 1，每轮 `fact *= i`
- 计数：cnt 初始化为 0，满足条件 `cnt++`
- 阶乘增长极快，用 long long 防止溢出
- 口诀：累加初始 0，累乘初始 1，计数初始 0

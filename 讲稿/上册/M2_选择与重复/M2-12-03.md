# 第12章 第3课时：先做再说——do-while 循环
**时长**：15-20分钟 | **对应教材**：第12章 · 12.4节

## 🎯 本课时目标
- 理解 do-while 和 while 的核心区别：判断时机不同
- 掌握 do-while 的语法（包括末尾的分号）
- 能根据场景在 while 和 do-while 之间做选择

## 📝 讲稿正文

同学们好！前两节课我们学了 while 循环——"先判断，再执行"。但生活中有一些场景，需要**先执行，再判断要不要继续**。

（在黑板上写：while = 先问再走，do-while = 先走再问）

想象一个场景：你去餐厅，服务员递给你菜单。你是先看菜单再决定要不要点菜，还是先看了菜单再决定？当然是先拿到菜单。如果菜单都不给你看，你怎么知道要不要吃？

再比如游戏的菜单界面——你打开游戏，主菜单至少会显示一次。你不可能还没看到菜单就退出游戏。你一定是先看到菜单，然后选择"开始游戏"或"退出"。

这种"**至少做一次，然后看要不要继续**"的场景，用 do-while：

```cpp
do {
    // 循环体：先执行
} while (条件);   // ← 注意这里有分号！
```

（在黑板上把 while 和 do-while 并排写）

| | while | do-while |
|------|-------|----------|
| 判断时机 | 先判断，再执行 | 先执行，再判断 |
| 最少执行次数 | 0 次 | 1 次 |
| 适合场景 | 可能一次都不需要做 | 至少要做一次 |

写一个菜单程序来体会：

```cpp
int choice;
do {
    cout << "======== 菜单 ========" << endl;
    cout << "1. 开始游戏" << endl;
    cout << "2. 查看成绩" << endl;
    cout << "3. 退出" << endl;
    cout << "请选择：";
    cin >> choice;

    switch (choice) {
        case 1: cout << "游戏开始！" << endl; break;
        case 2: cout << "你的历史最高分：999" << endl; break;
        case 3: cout << "再见！" << endl; break;
        default: cout << "无效选项，请重选" << endl;
    }
} while (choice != 3);
```

（带着学生跑一遍这个程序）

这个程序的运行逻辑是：先显示菜单 → 用户选择 → 如果不是 3（退出），就回到菜单重新显示。用户至少会看到一次菜单——这就是 do-while 的"至少一次"保证。

注意一个非常重要的语法细节：**`do { ... } while (条件);` 末尾有一个分号！**（用红笔圈出）

而普通 while 是这样的：

```cpp
while (条件) {   // ← 这里没有分号！
    ...
}
```

很多初学者在 do-while 末尾忘了写分号，编译报错后一脸懵。记住口诀：**"do-while 尾巴带分号。"** 这个分号是 C++ 语法规定的——do-while 是一个完整的语句，语句必须以分号结束。

现在来做一个小实验。我们用 while 和 do-while 分别跑一个条件一开始就不成立的循环：

```cpp
// while 版本：条件一开始就不成立
int x = 10;
while (x < 5) {
    cout << "while 执行了" << endl;
}
// 输出：（什么都没有——0 次）

// do-while 版本：同样条件
int y = 10;
do {
    cout << "do-while 执行了" << endl;
} while (y < 5);
// 输出：do-while 执行了（至少 1 次！）
```

看！while 版本什么都没输出——因为 10<5 一开始就是假。但 do-while 版本输出了一行——因为循环体先跑了一次，跑完才判断 10<5 是假，然后退出。

这就是两者的本质区别：**while 是"先想再做"，do-while 是"先做再想"。**

选哪个？看你的场景——如果你需要"至少做一次"，用 do-while。如果"可能一次都不需要做"，用 while。菜单程序至少显示一次 → do-while。密码验证可能一次就对 → while。

现在动手练习：用 do-while 写一个"正数验证器"——反复让用户输入一个数，如果 ≤0 就提示重输，直到输入正数为止。注意和上节课用 while 做的版本对比，你觉得哪个更自然？

## 🖥️ 板书/屏幕要点
- `do { ... } while (条件);` ← 注意分号！
- while：先判断后执行 → 最少 0 次
- do-while：先执行后判断 → 最少 1 次
- 选择：需要至少一次 → do-while；可能零次 → while

## ✏️ 随堂练习
用 do-while 写一个"猜数字"程序的简化版：预设答案为 42，让用户反复猜，每次提示"大了"或"小了"，直到猜对。用户至少猜一次。对比用 while 写这个程序的差异。

## 📌 小结
- do-while 先执行循环体，再判断条件
- 保证循环体至少执行一次
- 末尾分号不能忘
- 适合菜单、至少猜一次的猜数字等场景

## 📝 讲稿正文

同学们好！今天是第12章最后一节课。我们学了三种循环：for、while、do-while。现在问题来了——**面对一道新题，我该用哪个循环？**

（在黑板上画三列，分别写 for、while、do-while）

选择指南很简单：

| 场景 | 推荐 | 原因 |
|------|------|------|
| 知道确切次数（1到100） | for | 三要素紧凑，不会漏 |
| 遍历固定范围 | for | 天生一对 |
| 输入直到满足条件 | while | 次数不定 |
| 游戏主循环 | while | `while(true)` + break |
| 至少执行一次 | do-while | 先做再判断 |
| 菜单程序 | do-while | 菜单至少显示一次 |

一句话口诀：**"有数用 for，没数用 while，必须一次用 do-while。"**

（在黑板上大字写出这个口诀）

来，我们实战选一下。听题：

**题1：求 1 到 n 的阶乘。** 知道循环次数（n 次），用 for。

**题2：输入密码直到正确。** 次数不定，用 while。

**题3：显示菜单让用户选，选"退出"才结束。** 菜单至少显示一次，用 do-while。

**题4：统计输入的一串数字的和，以 0 结束。** 次数不定，用 while（哨兵模式）。

**题5：输出 100 到 999 中所有的水仙花数。** 遍历固定范围（100~999），用 for。

选循环没有绝对的"正确"答案——for 能做的事 while 也能做，while 能做的事 for 也能做。但**选最自然的那个让代码更好读**。

（让学生在纸上画一个选择流程图）

好，现在做全章知识树回顾。

第12章的核心——**三种循环的适用边界**。

while：先判断后执行，最少 0 次。适合"不知道次数"的场景。经典模式：密码验证、哨兵输入、逐位处理。需要自己管理初始化和迭代。

do-while：先执行后判断，最少 1 次。适合"至少做一次"的场景。经典场景：菜单程序。末尾有分号。

for vs while vs do-while：有数用 for，没数用 while，必须一次用 do-while。

死循环：三种成因（忘更新、方向错、变量被还原）。三检查点：初始化了吗？更新了吗？能退出吗？

（在黑板上画知识树）

现在来做综合练习：用**辗转相除法**求两个正整数的最大公约数（GCD）。

算法步骤：设 a 和 b 是输入的两个数。计算 a % b 得到余数 r。如果 r = 0，那么 b 就是最大公约数。否则，让 a = b，b = r，回到第一步。

```cpp
int a, b;
cin >> a >> b;

while (b != 0) {       // 当余数不为 0 时继续
    int r = a % b;     // 求余数
    a = b;             // 被除数变除数
    b = r;             // 除数变余数
}
cout << "最大公约数：" << a << endl;
```

注意：这个循环次数是不确定的——取决于余数什么时候变 0。所以用 while 而不是 for。这就是"选对循环"的实战例子。

（给学生练习时间，让他们输入 12 和 18，应该得到 6）

小结：三种循环各有各的用武之地。for 知道次数、while 知道条件、do-while 至少一次。选对的循环让代码更自然、更容易理解。

下堂课我们进入第13章——嵌套循环。循环里面套循环，一层不够来两层。九九乘法表、星号图案，都是嵌套循环的经典应用。

## 🖥️ 板书/屏幕要点
- 选择口诀：有数用 for，没数用 while，必须一次用 do-while
- 全章知识树：while（先判后做） → do-while（先做后判） → 死循环 → 条件设计
- 辗转相除法：`while (b != 0) { r = a % b; a = b; b = r; }`

## ✏️ 随堂练习
用辗转相除法求两个正整数 a 和 b 的最大公约数。输入 12 18 → 输出 6。输入 1071 462 → 输出 21。先判断该用哪种循环，再写代码。

## 📌 小结
- for：知道次数；while：知道条件；do-while：至少一次
- 辗转相除法是 while 的经典应用
- 死循环三检查：初始化、更新、方向
- 三种循环可以相互转换，但选最自然的那种

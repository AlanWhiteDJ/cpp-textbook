## 📝 讲稿正文

同学们好！今天我们来学第三种经典数学编程题——**完数判断**。

先看定义：一个正整数，如果它等于它所有**真因子**（除了自己以外的因子）之和，就叫"完数"（也叫完全数）。

比如 6 的真因子有 1、2、3，加起来 1+2+3=6 → 所以 6 是完数。

再比如 28 的真因子有 1、2、4、7、14，1+2+4+7+14=28 → 28 也是完数。

走三步选结构：

① **需要分岔吗？** 是！判断 sum 是否等于 n → if

② **需要重复吗？** 是！枚举所有可能的因子，还要枚举所有要检查的数 → for

③ **需要嵌套吗？** 是！外层枚举数，内层找因子 → 嵌套 for

这道题和第 11 章学的"累加模式"完美对接——找因子并累加。

先写单个数的判断：

```cpp
int n;
cin >> n;
int sum = 0;                      // 累加器：因子之和

for (int i = 1; i < n; i++) {     // 注意：i < n，不包括 n 自己！
    if (n % i == 0) {             // i 是 n 的因子
        sum += i;                 // 累加
    }
}

if (sum == n)
    cout << n << " 是完数" << endl;
else
    cout << n << " 不是完数" << endl;
```

（在黑板上走一遍 n=6 的执行过程）

i=1：6%1=0 → sum=1
i=2：6%2=0 → sum=3
i=3：6%3=0 → sum=6
i=4：6%4=2≠0 → 跳过
i=5：6%5=1≠0 → 跳过
循环结束，sum=6=n → 是完数！

注意两个细节。第一，`i < n`——真因子不包括 n 自己，所以试到 n-1 就够了。第二，`sum` 初始化为 0——这是累加模式的标准做法。

现在扩展：找出 1~10000 内所有的完数。只需在外面再套一层 for：

```cpp
for (int n = 1; n <= 10000; n++) {
    int sum = 0;
    for (int i = 1; i < n; i++) {
        if (n % i == 0) sum += i;
    }
    if (sum == n) {
        cout << n << " 是完数，真因子：";
        // 再跑一遍，打印因子（可选）
        for (int i = 1; i < n; i++) {
            if (n % i == 0) cout << i << " ";
        }
        cout << endl;
    }
}
```

运行结果会找到三个完数：6、28、496。

（在黑板上写出结果）

第 4 个完数是 8128，第 5 个是 33550336——已经超出 int 范围了。完数在自然数中非常稀有，目前人类只找到了 51 个。

这个解题模式"**枚举数 + 找因子 + 累加 + 比较**"和上节课的"枚举+拆位+判定"是不一样的两套模板，但本质相同——**在一个范围内搜素满足特定条件的数。**

区别在于"条件怎么判断"：
- 水仙花数 → 拆位后公式计算
- 质数 → 试除找因子（找得到一个就不是）
- 完数 → 找所有因子累加后比较

三道题，三种判断逻辑，但外层框架一模一样：**for 枚举 + 内层判定 + if 输出。**

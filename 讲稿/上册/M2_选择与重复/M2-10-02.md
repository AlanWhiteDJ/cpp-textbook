# 第10章 第2课时：穿透的秘密——break 与巧用穿透
**时长**：15-20分钟 | **对应教材**：第10章 · 10.2节、10.3节

## 🎯 本课时目标
- 理解 switch-case 的"穿透"机制：case 只是入口不是出口
- 掌握 break 在 switch 中的作用——阻止穿透
- 学会利用穿透特性合并相同处理的 case

## 📝 讲稿正文

同学们好！上节课我们学了 switch-case 的基本写法，每个 case 末尾都写了 `break`。今天这节课，我们来回答一个关键问题：**不写 break 会怎样？**

先来看一段有问题的代码：

```cpp
int n = 2;
switch (n) {
    case 1: cout << "A";
    case 2: cout << "B";
    case 3: cout << "C";
}
```

大家猜猜，输入 2，输出什么？

（等学生回答）

A 说输出 "B"——常见误会。B 说输出 "BC"——对了！C 说输出 "B C"——也对，只是有没有空格的区别。

为什么会输出 "BC" 呢？我们预期的难道不是只输出 "B" 吗？

这里就引出了 switch-case 最核心的一个设计——**case 标签只是"入口"，不是"出口"。**

程序从 `switch (n)` 进来，看到 `n` 等于 2，于是它跳到 `case 2:` 这个标签所在的位置——就像一个跳水运动员从 2 号跳台跳下去。然后呢？然后它就开始**一路向下执行**，把后面的所有 case 里的代码全部跑一遍！`case 2` 里输出 B，然后继续往下走，`case 3` 里输出 C……直到遇到 `break` 或者 switch 的花括号结束。

这个现象叫做**穿透**——不加 break，程序就像瀑布一样从当前 case 一路流到底。

（在黑板上画一个瀑布图：从 case 2 处一个箭头往下流，经过 case 3、default…）

正确的版本是每个 case 末尾加 break：

```cpp
int n = 2;
switch (n) {
    case 1:
        cout << "A" << endl;
        break;           // ← 在这里停下，跳出 switch
    case 2:
        cout << "B" << endl;
        break;           // ← 在这里停下
    case 3:
        cout << "C" << endl;
        break;           // ← 在这里停下
}
// 输出：B
```

`break` 的作用就是一句话：**"别往下掉了，跳出 switch！"**

好，穿透是 bug 的温床——但是，如果刻意利用它，穿透可以变得非常优雅。

来看一个经典例子——判断月份有多少天。一年 12 个月，大月 31 天，小月 30 天，2 月特殊。如果不用穿透，你得给 7 个大月每个都写一遍 `cout << "31天"`。但如果利用穿透……

```cpp
int month;
cin >> month;

switch (month) {
    case 1:
    case 3:
    case 5:
    case 7:
    case 8:
    case 10:
    case 12:
        cout << "31天" << endl;
        break;
    case 4:
    case 6:
    case 9:
    case 11:
        cout << "30天" << endl;
        break;
    case 2:
        cout << "28天或29天" << endl;
        break;
    default:
        cout << "无效月份" << endl;
}
```

看！1、3、5、7、8、10、12 七个 case 标签排成一列，共享下面那行 `cout << "31天"`。因为 case 1 没有 break，程序就从 case 1 一路"穿透"下去，经过 3、5、7……直到遇到 `cout << "31天"` 执行，然后 break 跳出。

这比 `if (month == 1 || month == 3 || month == 5 || ...)` 那条长得吓人的条件清爽多了——每一个 case 标签整整齐齐地排列，读代码的人一眼就能看清哪些月份归为一组。

另一个经典用法——成绩评级：

```cpp
int score;
cin >> score;

switch (score / 10) {   // 整数除法：95→9，82→8，67→6
    case 10:            // 100 分
    case 9:
        cout << "A" << endl;
        break;
    case 8:
        cout << "B" << endl;
        break;
    case 7:
        cout << "C" << endl;
        break;
    case 6:
        cout << "D" << endl;
        break;
    default:
        cout << "E" << endl;
}
```

`score / 10` 是整数除法，能把任何 0~100 的分数映射到 0~10 这 11 个值。95 分变成 9，82 分变成 8，67 分变成 6……然后 switch 一条龙搞定。

这是 switch-case 最漂亮的应用——**先用算术把连续范围转成离散值，再用 switch 匹配**。

好，动手练习：用 switch-case 的穿透特性，写一个"季节判断"程序。输入月份，输出季节。3~5 春，6~8 夏，9~11 秋，12/1/2 冬。

记住今天的核心：case 是入口不是出口，break 是出口。不写 break 就穿透——穿透是 bug 的起源，也是优雅代码的来源，关键看你知不知道自己为什么要穿透。

## 🖥️ 板书/屏幕要点
- 穿透：case 只标记入口，不标记出口
- break = 出口："别往下掉了，跳出 switch"
- 巧用穿透：多个 case 标签 → 共享同一段代码
- 成绩评级技巧：`score / 10` → 离散化

## ✏️ 随堂练习
用 switch-case 的穿透特性写一个"季节判断"程序。输入 1~12 的月份，输出对应的季节。3~5 春天，6~8 夏天，9~11 秋天，12/1/2 冬天。

## 📌 小结
- 穿透 = case 只是入口，不加 break 一路掉到底
- break 阻止穿透，每个 case 末尾加 break 是好习惯
- 刻意穿透可以优雅地合并相同处理
- `score / 10` 是把范围转成离散值的经典技巧

# 第10章 第1课时：多选一的利器——switch-case 入门

## 📝 讲稿正文

同学们好！上章我们学了 if-else 分支，包括 else if 的多分支。但是大家有没有发现一个问题——当选项特别多的时候，else if 写起来像一堵墙？

（指着黑板上的代码）

比如自动售货机有五种饮料：1-可乐、2-雪碧、3-芬达、4-矿泉水、5-果汁。用 else if 写是什么样？

```cpp
if (choice == 1) {
    cout << "可乐" << endl;
} else if (choice == 2) {
    cout << "雪碧" << endl;
} else if (choice == 3) {
    cout << "芬达" << endl;
} else if (choice == 4) {
    cout << "矿泉水" << endl;
} else if (choice == 5) {
    cout << "果汁" << endl;
}
```

五行 else if，每个都要重新写一遍 `choice ==`。如果售货机有 50 种饮料呢？你要写 50 行 else if，光 `choice ==` 就打了 50 遍——手不酸吗？而且逻辑上，"判断这个数等于 1，等于 2，等于 3……"其实都是在做同一件事——**看这个数是多少，跳到对应入口**。

有没有一种写法，不用反复写 `choice ==`，直接告诉程序"看这个值跳过去"？

有！这就是今天要学的 `switch-case`。

来，switch-case 的语法长这样：

```cpp
switch (表达式) {
    case 值1:
        // 表达式等于值1 时执行
        break;
    case 值2:
        // 表达式等于值2 时执行
        break;
    default:
        // 所有 case 都不匹配时执行
}
```

用 switch-case 重写自动售货机：

```cpp
int choice;
cin >> choice;

switch (choice) {
    case 1:
        cout << "你选了：可乐" << endl;
        break;
    case 2:
        cout << "你选了：雪碧" << endl;
        break;
    case 3:
        cout << "你选了：芬达" << endl;
        break;
    case 4:
        cout << "你选了：矿泉水" << endl;
        break;
    case 5:
        cout << "你选了：果汁" << endl;
        break;
    default:
        cout << "没有这个编号" << endl;
}
```

（对比两个版本的代码）

看清楚区别了吗？switch 版本里，你只写了一次 `choice`——就是最上面 `switch (choice)`。后面的每个 `case` 后面直接跟值，不用再写 `choice ==`。程序自己会拿 choice 的值去匹配每一个 case。

对比一下两种写法：

| | else if | switch-case |
|------|------|------|
| 匹配方式 | 每个条件都要写完整表达式 | case 只写值 |
| 可读性 | 选项多了像一堵墙 | 结构清晰，一眼看到头 |
| 灵活度 | 可以写任意条件 | 只能精确匹配值 |

现在我问一个问题：为什么叫"switch"？switch 的英文原意是"开关"或"切换"。就像铁轨上的道岔——列车来了，看道岔扳到哪个方向，就开去哪条轨道。switch-case 就是程序的"道岔"。

好，现在注意几个细节：

第一，`switch` 后面括号里只能放**整数类型**（int, long long, char 等）或枚举类型。不能放 double（浮点数比相等不可靠），也不能放 string（C++ 不支持）。

第二，`case` 后面必须跟**常量**——不能是变量，不能是范围。你不能写 `case a:`，也不能写 `case 60..100:`。

第三，每个 `case` 末尾的 `break` 非常重要——下一节课我们专门讲它。

现在我们做一个练习：写一个程序，输入 1~7 的数字，输出对应的星期几（1→星期一，2→星期二……7→星期日）。用 switch-case 实现。

（给学生练习时间）

小结：switch-case 是处理"精确匹配多个值"场景的最佳选择。它比 else if 更清晰，尤其当选项很多的时候。但它的限制也很明确——只能用整数/字符类型，case 必须是常量。范围判断还是得用 if-else。

## 🖥️ 板书/屏幕要点
- `switch (表达式) { case 值: ... break; default: ... }`
- switch = 道岔，根据值跳到对应入口
- 只能整型/字符型，case 必须是常量
- vs else if：选项多时 switch 更清晰，但失去灵活性

## ✏️ 随堂练习
用 switch-case 写一个"星期转换器"：输入 1~7，输出对应星期几的中文名称。如果输入不在 1~7 范围，用 default 提示"输入错误"。

## 📌 小结
- switch-case 是"多选一"的跳转表
- 比 else if 更适合"精确匹配多个值"的场景
- 有限制：只能整型/字符型，case 后只能是常量
- 每个 case 末尾的 break 不能少（下节课详讲）

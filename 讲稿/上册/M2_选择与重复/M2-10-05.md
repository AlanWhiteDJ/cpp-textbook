# 第10章 第5课时：怎么选？——switch vs if-else 与本章总复习
**时长**：15-20分钟 | **对应教材**：第10章 · 10.6节 + 全章复习

## 🎯 本课时目标
- 掌握 switch-case、if-else、三目运算符的选择指南
- 通过综合练习巩固第10章全部知识
- 能根据具体场景做出正确的结构选择

## 📝 讲稿正文

同学们好！今天是第10章的最后一节课。我们学了三种"做选择"的工具：switch-case、if-else、还有三目运算符。现在问题来了——**面对一道新题，我该用哪个？**

（在黑板上画三列）

这不是一个非黑即白的问题，而要看具体的场景。我给你们一个选择指南：

| 场景 | 推荐 | 原因 |
|------|------|------|
| 判断范围（>  <） | if-else | switch 不支持范围 |
| 精确匹配几个值 | switch-case | 结构更清晰 |
| 条件涉及逻辑运算 | if-else | switch 里不能写 && |
| 选项很多（5+） | switch-case | 代码更整洁 |
| 简单二选一赋值或输出 | 三目 ? : | 一行搞定 |
| 涉及字符串比较 | if-else | switch 不支持 string |
| 浮点数判断 | if-else | switch 不支持 double |

来，我们一个个看。第一个场景：输入一个分数，判断等级 A/B/C/D/E。这是"判断范围"——≥90→A，≥80→B……显然用 if-else if 最自然。虽然可以用 `switch (score/10)` 的变通方法，但那个技巧是花活，if-else 才是正道。

第二个场景：输入 1~7 输出星期几。这是"精确匹配几个值"——每个数字对应一个确定的输出。用 switch-case 更清晰。

第三个场景：判断闰年。条件涉及 `&&` 和 `||` 的逻辑组合——switch 无能为力，必须用 if-else。

选工具的核心理念只有一句话：**选让代码更清晰好读的那个，不要炫技。**

好，现在做一个全章知识树回顾。请拿出一张纸，跟我写。

第10章的核心是——**当选项很多时的处理方案**。

我们有三种武器：switch-case、if-else、三目运算符。它们的适用边界各不相同。

switch-case：精确匹配、整型/字符型、case 常量、需要 break 防穿透、用 default 兜底。巧用穿透可以合并相同处理。

三目运算符 `? :`：简单二选一、一行搞定、禁止嵌套。

if-else：范围判断、逻辑组合、最灵活、最通用。

现在请大家完成一个综合练习。这道题需要你综合运用本章知识，并且还要结合第9章的闰年判断——

**题目：日期合法性判断。** 输入年、月、日，判断这个日期是否合法。需要考虑：
- 大月（31天）：1、3、5、7、8、10、12
- 小月（30天）：4、6、9、11
- 2月：闰年29天，平年28天
- 月份不在1~12、日期不在合理范围 → 不合法

思路提示：
1. 先用 if 判断月份是否在 1~12 之间
2. 用 switch-case（穿透）确定该月的最大天数
3. 2月的最大天数用逻辑运算符判断闰年来决定
4. 最后判断输入的日期是否 ≤ 最大天数

这道题融合了 if-else、switch-case、逻辑运算符——是对第9章和第10章的综合检验。

（给学生 8~10 分钟练习，然后请同学上来分享代码）

好。第10章到这里就结束了。记住：没有谁比谁好，只有谁更适合这个场景。switch-case 清晰但受限，if-else 灵活但啰嗦，三目简洁但只能二选一——你手上有三把刀，知道哪把适合切什么，才是真本事。

## 🖥️ 板书/屏幕要点
- 选择指南表：范围→if-else、精确多值→switch、简单二选一→三目
- switch 三要素：case + break + default
- 穿透慎用（要么有意识用，要么加 break）
- 三目禁忌：不嵌套、cout 里加括号

## ✏️ 随堂练习
综合题：输入年、月、日，判断日期是否合法。用 switch-case 处理月份天数，用逻辑运算符判断闰年，用 if 判断最终合法性。

## 📌 小结
- 选工具 = 选最清晰的，不是最短的
- switch-case：精确匹配多值
- if-else：范围、逻辑组合、字符串、浮点
- 三目 `? :`：简单二选一
- 穿透和 default 是 switch 的两大精髓

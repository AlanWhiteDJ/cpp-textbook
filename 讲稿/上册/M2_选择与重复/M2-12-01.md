## 📝 讲稿正文

同学们好！欢迎来到第12章——while 和 do-while 循环。

上章我们学 for 循环的时候，有一个隐含的前提：你知道要重复多少次。"跑 3 圈""加 100 个数""统计 10 个输入"——次数都是明确的。

但真实世界不是这样的。比如你去吃自助餐，你的策略是"只要还能吃，就继续拿"——你不会在进门的时候就想"我要吃 8 盘"。再比如你登录电脑，系统让你输密码——你不知道会输错几次，只知道"输对了就能进"。

（在黑板上写：for = 知道次数，while = 只知道条件）

这些"不知道重复多少次，只知道什么时候该停"的场景，就是 while 循环的用武之地。如果说 for 是"跑 5 圈"，while 就是"跑到腿软为止"。

来看 while 的语法——比 for 更简单：

```cpp
while (条件) {
    // 条件成立就执行，执行完再回来检查条件
}
```

对比一下 for 和 while 写同一个功能——输出 1 到 5：

```cpp
// for 版本：三要素浓缩在括号里
for (int i = 1; i <= 5; i++) {
    cout << i << " ";
}

// while 版本：三要素拆开了
int i = 1;               // 初始化（写到外面）
while (i <= 5) {         // 只有条件在括号里
    cout << i << " ";
    i++;                 // 迭代（写在循环体里）
}
```

看出区别了吗？for 把初始化、条件、迭代三个东西整整齐齐锁在括号里——你想忘都忘不掉。while 只把条件放在括号里，初始化和迭代你得自己管。

这就是 while 的特点——**简洁但危险**。简洁是因为括号里只有一个条件，看着清爽；危险是因为初始化和迭代需要你自己管理——忘了任何一步都是灾难。

（在黑板上用红笔写：**while 需要你自己管理初始化和迭代！**）

来，我们看一个 while 的经典场景——密码验证：

```cpp
string password;
string correct = "cpp123";

cout << "请输入密码：";
cin >> password;

while (password != correct) {
    cout << "密码错误，请重试：";
    cin >> password;
}

cout << "登录成功！欢迎回来。" << endl;
```

运行效果可能是：

```
请输入密码：abc
密码错误，请重试：12345
密码错误，请重试：cpp123
登录成功！欢迎回来。
```

这里 while 循环跑了几次？不知道——取决于你输错了几次。你可能一次就对了（循环 0 次），可能输错 3 次（循环 3 次），可能输错 10 次（循环 10 次）。**while 不在乎次数，它只盯着条件 `password != correct`。**

注意代码结构：我们**先在循环外面读了一次密码**。这很重要！如果你把 `cin >> password` 只放在循环里面，那第一次循环时 password 可能是个垃圾值——你得先拿到用户的第一次输入，才能判断对不对。

这个模式叫"**先读再判**"——循环前读第一次，循环末尾读下一次。后面还会反复遇到。

好，现在动手练习：写一个程序，反复让用户输入一个整数，直到输入的数是正数为止。每次输入负数或零时提示"请重新输入"。

（给学生练习时间）

小结：while 循环适合"不知道次数、只知道停止条件"的场景。它的语法更简单，但需要你自己管理初始化和迭代。记住——while 括号外要初始化，循环体里要更新。

## 🖥️ 板书/屏幕要点
- `while (条件) { ... }` —— 先判断，后执行
- for vs while：
  - for：三要素全在括号里
  - while：只括号里有条件，初始化和迭代自己管
- 经典模式：循环前读第一次，循环末尾读下一次
- while 优势：次数不定的场景（密码、输入直到满足条件）

## ✏️ 随堂练习
写一个程序，反复让用户输入一个整数，直到输入正数为止。每次输入 ≤0 时提示"请输入正数"，输入正数后输出"你输入了 X，谢谢！"并结束。

## 📌 小结
- while 适合"不知道重复几次"的场景
- 语法简单：括号里只有条件
- 需要自己管理初始化和迭代——for 替你管，while 靠自觉
- "先读再判"模式：循环外读第一次，循环内读下一次

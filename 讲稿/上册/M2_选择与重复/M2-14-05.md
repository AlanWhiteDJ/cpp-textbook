## 📝 讲稿正文

同学们好！今天是第14章最后一课。前四节课我们学了 break、continue、嵌套中的限制、标记变量跳出多层循环。今天我们要完成两件事：第一，看 break/continue 在 while 中的用法；第二，搞清楚什么时候该用它们，什么时候不该用。

先看 while 中的 break。一个经典场景——密码登录，最多试 3 次：

```cpp
int attempts = 0;
string password;
while (attempts < 3) {
    cout << "请输入密码：";
    cin >> password;
    if (password == "cpp123") {
        cout << "登录成功！" << endl;
        break;            // 对了就立刻退出
    }
    attempts++;
    cout << "密码错误，还剩 " << 3 - attempts << " 次机会" << endl;
}
if (attempts == 3) {
    cout << "3 次机会用完，账户已锁定。" << endl;
}
```

注意这里的 break——循环条件 `attempts < 3` 控制的是"最多 3 次"，但密码对了就应该立刻退出，不等次数用完。break 和循环条件各司其职——循环条件是"常规退出"，break 是"提前退出"。

while 中的 continue 用法也一样——跳过本轮，判断条件后继续：

```cpp
int sum = 0, n;
while (cin >> n) {
    if (n < 0) continue;   // 负数跳过
    sum += n;              // 只累加非负数
}
```

好，现在来回答一个关键问题：**什么时候该用 break/continue，什么时候不该用？**

（在黑板上列出原则）

✅ **该用 break 的场景：**
- 找到第一个就够了，提前退出省时间
- 循环条件本身无法表达"提前终止"（如密码对了就退出）
- 在 `while(true)` 中作为唯一的退出方式

✅ **该用 continue 的场景：**
- 过滤不需要的数据，简化主体逻辑
- 跳过无效输入

❌ **不该用的场景：**
- 能用循环条件自然表达的退出——别用 break。比如你知道循环 100 次，第 35 次要跳，那为什么不把条件写成能自然结束的？
- continue 放在循环体最后一行——跳了个寂寞
- 滥用导致逻辑混乱——如果你自己都要想半天才能理解 break 在那里干什么，那说明该重写了

核心理念：**break 和 continue 是"额外控制"，不是"常规控制"。能用循环条件表达的就用条件，条件说不清楚的再用 break/continue。**

（在黑板上写这个核心理念）

现在来做全章总结。

第14章只有两个关键词：break（刹车）和 continue（跳过）。

| | break | continue |
|------|-------|----------|
| 作用 | 跳出当前循环 | 跳过本轮，下轮继续 |
| 比喻 | 脚崴了，立刻停 | 踩线了，这圈不算 |
| 嵌套中 | 只跳一层 | 只影响当前层 |
| 适用范围 | for/while/do-while/switch | for/while/do-while |

跳出多层循环用标记变量：`bool found = false;` 内层插旗，外层检查。

现在做综合练习——**猜数字·限次版**：

程序预设一个秘密数字（1~100），玩家最多猜 7 次。每次猜完提示"大了"或"小了"，猜对用 break 退出。7 次猜不对显示"你输了，答案是 X"。用 for 循环 + break 实现。

这道题融合了循环和 break，是对第14章和第11章的综合应用。

（给学生 8~10 分钟练习）

小结：break 和 continue 是循环的精细控制工具。记得：break 只跳一层，continue 跳过本轮。能用条件表达的就用条件，条件说不清的再用这两个工具。

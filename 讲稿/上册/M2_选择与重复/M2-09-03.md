# 第9章 第3课时：多岔路口——else if 链条
**时长**：15-20分钟 | **对应教材**：第9章 · 9.4节、9.5节

## 🎯 本课时目标
- 掌握 `else if` 多分支结构的语法
- 理解 else if 链条的"从上到下、命中即停"的执行逻辑
- 能正确排列 else if 顺序，避免"条件被吞"的 bug

## 📝 讲稿正文

同学们好！前面两节课我们学了 `if` 的单岔路和 `if-else` 的双岔路。但真实世界可不是只能二选一。

（出示一张成绩单图片或板书）

比如，考试成绩评级：
- 90 分及以上 → A
- 80 分到 89 分 → B
- 70 分到 79 分 → C
- 60 分到 69 分 → D
- 60 分以下 → E

五个等级！这就不是简单的"成立就做，不成立就不做"了。我们需要"多岔路口"——程序能像过收费站一样，根据不同情况走不同的通道。

C++ 里用 `else if` 来实现多分支：

```cpp
if (条件1) {
    // 情况1
} else if (条件2) {
    // 情况2
} else if (条件3) {
    // 情况3
} else {
    // 以上都不满足时
}
```

把刚才的成绩评级翻译成代码：

```cpp
int score;
cin >> score;

if (score >= 90) {
    cout << "A" << endl;
} else if (score >= 80) {
    cout << "B" << endl;
} else if (score >= 70) {
    cout << "C" << endl;
} else if (score >= 60) {
    cout << "D" << endl;
} else {
    cout << "E" << endl;
}
```

来，我们一起在脑子里跑一遍这段代码。假设输入 `85`：

第一步，程序问"score >= 90？"——85 不小于 90，跳过。
第二步，程序问"score >= 80？"——85 大于等于 80，成立！执行 `cout << "B"`，然后**直接跳到整个 if-else if 链条的末尾**——后面的 `>= 70`、`>= 60` 全部不看了。

这就是 else if 链条最重要的规则：**从上往下依次检查，第一个成立的就执行，执行完就跳出——后面的条件全被跳过。**

（在黑板上用粉笔画出从上到下的箭头，在第一个命中处画一个星号并标"停！"）

这个规则有一个非常重要的推论：**条件的排列顺序极其关键！**

来，我犯一个经典的错误给你们看。如果我把最宽松的条件放最前面：

```cpp
if (score >= 60) {       // ❌ 最宽松的放最前面！
    cout << "D" << endl;
} else if (score >= 70) {
    cout << "C" << endl;
} else if (score >= 80) {
    cout << "B" << endl;
} else if (score >= 90) {
    cout << "A" << endl;
} else {
    cout << "E" << endl;
}
```

输入 `95` 分，猜猜输出什么？

（等学生回答）

对，输出 "D"！因为 95 ≥ 60 第一个就成立了，程序高高兴兴打印了 D，然后就走人了——后面的 `>= 90` 永远没机会被执行。95 分的学霸拿了个 D，这程序你敢用吗？

所以，**else if 链条的铁律是：要么从大到小排，要么从小到大排，把最严格的条件放最前面。** 你心里得先排好序。

好，现在讲另一个问题：if 里面还能不能再套 if？答案是——可以。这就是嵌套 if。

比如判断年份是不是闰年的完整规则：能被 400 整除，或者能被 4 整除但不能被 100 整除。如果不用逻辑运算符（下节课才学），只能用嵌套来写：

```cpp
if (year % 400 == 0) {
    cout << "闰年" << endl;
} else {
    if (year % 4 == 0) {
        if (year % 100 != 0) {
            cout << "闰年" << endl;
        } else {
            cout << "平年" << endl;
        }
    } else {
        cout << "平年" << endl;
    }
}
```

（指着这个嵌套结构）

三层套娃！读起来像不像俄罗斯套娃？一层套一层，头晕不晕？

这就是嵌套 if 的问题——写起来还行，读起来灾难。下一节课我们要学的逻辑运算符，能把这种四层嵌套浓缩成一行。但在学逻辑运算符之前，嵌套 if 你得先会，因为它是理解"条件组合"的基础。

现在请大家做一个练习：输入一个整数，判断它是正数、负数还是零。用 if-else if-else 实现。

（给学生练习时间）

小结一下：else if 是多分支结构，从上到下检查，命中即停。这决定了条件顺序非常重要——宽松条件放前面会"吞掉"后面的严格条件。嵌套 if 是 if 里面套 if，能处理更复杂的逻辑，但代码会变得难读。

## 💬 互动提问
1. "else if 链条中，如果输入满足多个条件，会执行几个分支？为什么？"
2. "如果把 `score >= 60` 写在 `score >= 90` 前面，为什么 95 分的人会拿到 D？"
3. "什么时候该用 else if，什么时候该用嵌套 if？"

## 🖥️ 板书/屏幕要点
- `if (...) { } else if (...) { } else { }` —— 多选一
- 执行规则：从上到下，命中即停，后面全不执行
- ⚠️ 铁律：条件要么从大到小，要么从小到大
- 嵌套 if = if 里面还有 if，慎用，能简化就简化

## ✏️ 随堂练习
写一个"简易计算器"：输入一个运算符（`+` `-` `*` `/`中的一个字符），再输入两个数，用 if-else if 判断运算符并输出结果。如果是除法且除数为 0，输出"除数不能为 0"。

## 📌 小结
- else if 实现多分支，从上到下只会执行一个分支
- 条件顺序决定程序正确性——宽松条件绝不放前面
- 嵌套 if 能处理复杂逻辑，但可读性差，需要逻辑运算符来救场

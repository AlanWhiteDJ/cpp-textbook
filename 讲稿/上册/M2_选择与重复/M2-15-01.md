# 第15章 第1课时：看到题目先别写——三步选结构思维
**时长**：15-20分钟 | **对应教材**：第15章 · 15.1节

## 🎯 本课时目标
- 建立"三步选结构"的思维框架
- 面对新题能快速判断需要分支还是循环（还是两者结合）
- 养成先画图再写代码的习惯

## 📝 讲稿正文

同学们好！欢迎来到第15章——M2 模块的综合实战。从第9章到第14章，我们学了六种武器：if-else、switch-case、三目运算符、for 循环、while/do-while、嵌套循环、break/continue。每一样你都会用了——但问题是，**看到一道新题，你怎么知道该用哪一个？**

这就是今天要建立的核心能力——**三步选结构思维**。

（在黑板上画出三步流程）

### 第一步：需要分岔路吗？

拿到题目后，先问自己：**程序的执行路线是不是要根据某个条件走不同的方向？**

比如："判断一个数是正数还是负数"——显然，要分两条路。→ 用 `if-else`。

比如："输入 1~7，输出星期几"——7 条不同的路。→ 用 `switch-case`。

如果答案是否定的——程序不用根据任何条件分岔——那就跳过这一步。

### 第二步：需要重复做某件事吗？

**有没有一件事需要做很多遍？**

比如："计算 1 到 100 的和"——加 100 次，明显要重复。→ 用循环。

那你再问：知道重复多少次吗？

- 知道 → `for` 循环（如 1 到 100）
- 不知道 → `while` 循环（如密码输对为止）
- 至少做一次 → `do-while`（如菜单程序）

### 第三步：需要嵌套吗？

**分岔路里面还要重复？重复里面还要分岔路？**

比如猜数字游戏——需要反复猜（循环），但每次猜完要根据大小给提示（分支）。这是**循环套分支**。

比如九九乘法表——每行要打印多列（循环），而且有多行（循环）。这是**循环套循环**。

比如菜单计算器——显示菜单是循环，选了某操作后要根据运算符分岔（分支），而且可能还要判断除零（又一个分支）。这是**循环套分支套分支**。

（在黑板上列出三步流程图）

来，我们实战练几个题目。

**题1**：输入一个整数，判断它是奇数还是偶数。
- ① 需要分岔吗？是 → if-else
- ② 需要重复吗？否
- ③ 需要嵌套吗？否
- → 一行 if-else 搞定

**题2**：输入一串整数，以 -1 结束，统计其中正数的个数。
- ① 需要分岔吗？是（判断是不是正数）
- ② 需要重复吗？是（读入直到 -1），次数不定 → while
- ③ 需要嵌套吗？是（循环里套 if）
- → while + if 嵌套

**题3**：打印 n 行等腰三角形。
- ① 需要分岔吗？否
- ② 需要重复吗？是（打 n 行），知道次数 → for
- ③ 需要嵌套吗？是（每行里又要打空格又要打星号）
- → 嵌套 for

（带着学生过这三题，每题不超过 30 秒分析）

现在强调一个习惯：**先画流程图，哪怕在纸上随便画几个框也可以。** 图画出来了，代码的骨架就有了。别一上来就敲键盘——那会让你被细节淹没，忘了全局结构。

画流程图不需要多漂亮——几个框、几个箭头，把"先做什么、再做什么、什么条件分岔、哪里循环回来"理清楚就够了。考场上你只有一张草稿纸，画个简单的流程图能帮你少错一半的逻辑错误。

（在黑板上示范画一个简单的流程图：猜数字的逻辑——输入 → 判断（菱形框）→ "大了/小了" → 回到输入）

现在动手练习：分析下面三题每道该用什么结构，然后写代码验证：
1. 求 1 到 n 的阶乘
2. 输入密码直到正确
3. 打印 n 行的倒三角星号

（给学生练习时间）

小结：三步选结构——分岔吗？重复吗？嵌套吗？面对任何新题，先走这三步，再动手写代码。这个习惯会让你从"碰运气写代码"变成"有章法地写代码"。

## 🖥️ 板书/屏幕要点
- 三步选结构：
  ① 需要分岔？→ if/switch
  ② 需要重复？→ for/while/do-while
  ③ 需要嵌套？→ 画图再写
- 先画流程图再写代码
- 核心：理解题目 → 选结构 → 写代码

1. 求 1 到 n 的阶乘（n 由用户输入）

3. 打印 n 行的倒三角星号

每道题先说出"分岔？重复？嵌套？"，再写代码。

# 第14章 第4课时：插旗子——标记变量跳出多层循环
**时长**：15-20分钟 | **对应教材**：第14章 · 14.4节

## 🎯 本课时目标
- 掌握用 bool 标记变量跳出多层循环的技巧
- 理解"内层插旗、外层检查、看到旗子就停"的模式
- 能在二维搜索等场景中正确使用标记变量

## 📝 讲稿正文

同学们好！上节课我们学了一个重要限制：break 只跳一层。那你肯定要问了——"如果我真的需要在内层发现目标后把外层也停了，怎么办？"

（在黑板上画一个嵌套循环，内层 location 标"发现目标！"，外层标"怎么知道？"）

答案是用一个**标记变量**——也叫"标志位"或"旗子"。思路很简单：

1. 在外面设一个 `bool` 变量，初始值为 `false`（还没发现目标）
2. 内层发现目标后，把这个变量改成 `true`（插旗子！），然后 break 跳出内层
3. 外层每轮结束时检查这个变量——如果旗子立起来了（`true`），外层也 break

来，看代码。还是那个问题——在 5×5 格子里找第一个 i×j > 10 的全局坐标：

```cpp
bool found = false;           // 旗子：初始没找到

for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= 5; j++) {
        if (i * j > 10) {
            cout << "找到了：" << i << "×" << j << "=" << i*j << endl;
            found = true;    // 插旗子！
            break;           // 跳出内层
        }
    }
    if (found) break;        // 看到旗子？外层也停！
}
// 输出：找到了：3×4=12
```

完美！只输出了一个结果——全局第一个满足条件的坐标。

（在黑板上逐步走一遍执行流程）

来，走一遍：
- i=1：内层 j 从 1 到 5 全部不满足 i×j>10，found 还是 false，外层继续
- i=2：同样，全部不满足，found 还是 false
- i=3：j=1,2,3 都不满足。j=4 时：3×4=12>10！found=true，break 跳出内层
- 外层检查：found 是 true！break 跳出外层
- 循环全部结束

这个过程就像你在一个大的停车场找自己的车。你一排一排地看（外层），每排一辆一辆地看（内层）。找到车后，你不会继续看这一排剩下的车位，也不会继续看后面的排——你插个旗子表示"找到了"，然后整件事结束。

这就是**标记变量法**——跳出多层循环最清晰的方法。

标记变量的几个关键点：
1. 用 `bool` 类型，初始化为 `false`（没找到）
2. 在内层满足条件时设为 `true`（找到了），然后 break
3. 外层每轮检查 `if (found) break;`（看到旗子也停）
4. 如果是三层嵌套，每层外层都要检查

（板书这个模式的标准代码框架）

除了 `bool`，有时用 `int` 做标记变量也行——比如标记"退出的原因"：

```cpp
int exitCode = 0;  // 0=正常，1=找到目标，2=出错
```

但对初学者来说，`bool found` 是最清晰的选择。

现在动手练习：写一个程序，在二维范围 i 从 1 到 20、j 从 1 到 20 中，找第一个满足 `i*j` 能被 13 整除且 `i+j` 能被 7 整除的 (i,j) 对。找到就报告并退出所有循环，用标记变量实现。

（给学生练习时间）

小结：标记变量是跳出多层循环的标准方法。内层插旗，外层检查。这个思路不复杂，但需要你记住"内层 break 之后外层要检查"——忘了检查等于白插旗。

## 🖥️ 板书/屏幕要点
- 标记变量法：`bool found = false;`
- 内层：`found = true; break;`（插旗）
- 外层：`if (found) break;`（检查旗子）
- 三层嵌套：每层外层都要检查
- 适用于"二维/多维搜索"场景

## ✏️ 随堂练习
在 i(1~20)、j(1~20) 范围内，找第一对满足 `i*j` 能被 13 整除且 `i+j` 能被 7 整除的 (i,j)。用标记变量找到即退出所有循环。

## 📌 小结
- break 只跳一层 → 跳出多层需要标记变量
- 模式：外设 bool=false → 内层找到设 true + break → 外层检查 break
- 多层嵌套时每层都要检查旗子
- 这是跳出多层循环的标准做法

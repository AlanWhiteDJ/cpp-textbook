# 第14章 第2课时：踩线了这圈不算——continue 语句
**时长**：15-20分钟 | **对应教材**：第14章 · 14.2节

## 🎯 本课时目标
- 理解 continue 的作用：跳过本轮剩余代码，进入下一轮
- 掌握 continue 和 break 的本质区别
- 能在合适的场景下使用 continue 简化代码

## 📝 讲稿正文

同学们好！上节课我们学了 break——循环的紧急刹车，立刻跳出。今天学它的兄弟——continue。

（在黑板上并列写：break vs continue）

先讲一个运动会的故事。400 米比赛，你跑到第 2 圈时踩了内圈的线——裁判吹哨："踩线！这圈成绩不算，下一圈重新跑！"

你不是退赛（那不是 break），也不是重跑这一圈——你是**跳过这圈，直接进入下一圈**。这就是 continue 的精神。

C++ 里 `continue` 的作用：**立刻结束本轮循环，跳过 continue 之后的所有代码，直接进入下一轮的条件判断（for 的话还会先执行迭代部分）。**

来看代码：

```cpp
// 输出 1~10 中的所有奇数，跳过偶数
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;    // 偶数 → 跳过，不输出
    }
    cout << i << " ";
}
// 输出：1 3 5 7 9
```

来，一步步走一遍：

- i=1：1%2=1，不满足 if，跳过 continue → 输出 1
- i=2：2%2=0，满足 if → 执行 continue → **跳过下面的 cout** → 直接执行 i++ → i=3
- i=3：3%2=1，不满足 if → 输出 3
- ……
- i=10：10%2=0 → continue → i++ → i=11 → 条件不成立 → 退出

注意：continue 跳过后，for 循环的**迭代部分（i++）照常执行**。continue 只是跳过循环体里 continue 之后的代码，不影响循环本身的迭代机制。

（在黑板上用箭头标注 continue 跳到了哪里）

好，现在对比一下 break 和 continue：

| | break | continue |
|------|-------|----------|
| 比喻 | 脚崴了，不跑了 | 踩线了，这圈不算 |
| 效果 | 跳出整个循环 | 跳过本轮，下轮继续 |
| 之后 | 执行循环后面的代码 | 回到循环开头判断条件 |

（在黑板上画两个流程图对比）

continue 的典型使用场景是**过滤**——在一堆数据中跳过不需要的，只处理你关心的。

例1：输入一串数字，只累加正数，跳过负数和零：

```cpp
int sum = 0, num;
while (cin >> num) {
    if (num <= 0) {
        continue;     // 负数或零 → 跳过，不加
    }
    sum += num;
}
```

如果没有 continue，你得这么写：

```cpp
while (cin >> num) {
    if (num > 0) {    // 把想处理的部分包在 if 里
        sum += num;
    }
}
```

两种写法都可以。continue 版本的好处是把"不想要的"提前过滤掉，主体逻辑（累加）保持在最外层，看起来更清晰——这在"不想要的条件很复杂"时特别有用。

但要注意：**continue 不要放在循环体的最后一行**。如果 continue 之后什么代码都没有，那 continue 相当于什么都没跳——写了等于没写，还让人困惑。

```cpp
for (int i = 1; i <= 10; i++) {
    cout << i << " ";
    continue;   // ← 后面没代码了，写了等于没写
}
```

现在动手练习：输出 1~30 中所有**不是** 3 的倍数的数。用 continue 跳过 3 的倍数。

（给学生练习时间）

小结：continue 跳过本轮剩余代码，下一轮继续。它的精神是"这轮不算，下轮继续"。适合用来过滤不需要的数据。与 break 的区别：break 是退出，continue 是跳过。

## 🖥️ 板书/屏幕要点
- `continue`：跳过本轮剩余代码 → 下一轮继续
- break vs continue：退出循环 vs 跳过本轮
- for 中 continue 后迭代部分（i++）照常执行
- 典型场景：过滤不需要的数据
- ⚠️ continue 别放最后一行

## ✏️ 随堂练习
输出 1~30 中所有不是 3 的倍数的数。用 continue 实现——遇到 3 的倍数就跳过。

## 📌 小结
- continue = 跳过本轮，下轮继续
- break = 退出循环，continue = 跳过本轮
- 适合用来过滤数据
- 不要放在循环体末尾（后面没代码可跳）

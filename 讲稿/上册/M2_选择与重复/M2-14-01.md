# 第14章 第1课时：脚崴了立刻停——break 语句
**时长**：15-20分钟 | **对应教材**：第14章 · 14.1节

## 🎯 本课时目标
- 理解 break 的作用：立即跳出当前循环
- 掌握 break 在 for 循环中的典型用法
- 区分"正常退出循环"和"break 提前退出"

## 📝 讲稿正文

同学们好！前面三章我们学了各种循环——for、while、do-while、嵌套循环。但有一个问题我们一直没解决：**如果循环跑到一半，我不想跑了，怎么办？**

（在黑板上写一个大大的 "break"）

假设你在操场上跑步，老师让你跑 5 圈。跑到第 3 圈半，你突然脚崴了。你怎么办？你肯定是**立刻停下来找医生**，而不是"虽然脚崴了但循环条件说 i≤5 所以我必须跑完"——对吧？

但如果没有 break，程序就确实是那样傻——条件没变假，它就一直跑。**break 就是给循环装的紧急刹车**——不等跑完，立刻停。

来看一个例子：在 1~100 中找第一个能同时被 7 和 5 整除的数。

```cpp
for (int i = 1; i <= 100; i++) {
    if (i % 7 == 0 && i % 5 == 0) {
        cout << "找到了：" << i << endl;  // 输出 35
        break;  // ← 找到就停！不用再找 36~100
    }
}
```

循环本来要跑 100 轮，但 i=35 时就找到目标了——此时执行 break，**循环立刻终止**。后面的 65 轮（i=36 到 100）完全跳过。程序直接跳到循环后面的代码。

如果没有 break，程序会怎样？它会继续把 36~100 跑一遍——虽然你也知道 36~100 里可能还有第二个满足条件的数（比如 70），但题目说的是"找到第一个就够了"。有了 break，找到了就收工，不浪费力气。

（在黑板上画流程图：循环进入 → 判断条件 → 如满足则 break → 跳出循环）

break 的语义很单纯——**立即跳出当前所在的循环，继续执行循环后面的语句。** 不管循环还剩多少轮，不管条件是否还成立。

再看一个实用例子：判断一个数是不是质数。质数的定义是"大于 1，且只能被 1 和它本身整除"。我们要判断 n 是不是质数，需要从 2 试到 n-1，看看有没有能整除 n 的。

```cpp
int n;
cin >> n;
bool isPrime = true;

for (int i = 2; i < n; i++) {
    if (n % i == 0) {
        isPrime = false;     // 找到因子 → 不是质数
        break;               // 不用再试更大的 i 了
    }
}

if (isPrime)
    cout << n << " 是质数" << endl;
else
    cout << n << " 不是质数" << endl;
```

这里 break 的作用非常关键。如果我们已经找到了一个因子 i 能整除 n，那 n 已经确定不是质数了——继续试更大的 i 纯粹是浪费时间。break 一到，立刻停。

这就是 break 的核心使用场景：**"找到第一个就够了"**——找到就停，省时间。

现在动手练习：在 1~10000 中找第一个能同时被 3、5、7 整除的数（答案：105）。用 break 在找到后立刻退出。

（给学生练习时间）

小结：break 是循环的紧急刹车——立刻跳出当前循环，不等剩余轮次。典型场景是"找到第一个就够"，能大幅提高程序效率。

## 🖥️ 板书/屏幕要点
- `break`：立刻跳出当前循环，继续执行循环后面的代码
- 典型场景：找到第一个满足条件的 → break 提前退出
- 质数判断：找到因子 break，省时间
- 注意：break 只能跳出**一层**循环

## ✏️ 随堂练习
在 1~10000 中找第一个能同时被 3、5、7 整除的数（即 3×5×7=105 及其倍数中找到最小的），用 break 在找到后立即停止并输出结果。

## 📌 小结
- break = 循环的紧急刹车
- 找到目标后立刻退出，不浪费 CPU 时间
- 典型场景：第一个满足条件、质数判断、搜索问题
- break 只跳出当前这一层循环

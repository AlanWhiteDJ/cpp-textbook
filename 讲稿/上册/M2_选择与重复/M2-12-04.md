# 第12章 第4课时：转不出去了——死循环与条件设计

## 🎯 本课时目标
- 理解死循环的成因和预防方法

## 📝 讲稿正文

同学们好！今天这节课，我们来直面每个程序员都害怕的东西——**死循环**。

（在黑板上画一个圆圈，里面写"出不去了"）

先看一个经典的翻车现场：

```cpp
int i = 1;
while (i <= 5) {
    cout << i << " ";    // 屏幕疯狂刷 1 1 1 1 1……
}
```

程序卡住了！风扇狂转，屏幕疯狂输出，你按什么键都没反应——只能强制关闭窗口。为什么？因为 `i` 始终是 1，`i <= 5` 永远成立，循环永远退不出来。

这就是**死循环**——循环条件永远为真，循环体里没有任何语句能让条件最终变假。

（等学生笑完，继续）

死循环的成因一般有三种：

**第一种：忘了更新循环变量。** 这是最常见的——while 不像 for 那样强制你把迭代写在括号里，你得自己记得在循环体里更新。上例中少了 `i++`。

**第二种：更新方向不对。** 看这个例子：

```cpp
int x = 1;
while (x != 10) {
    x += 2;        // x 是 1, 3, 5, 7, 9, 11, 13...
                   // 永远跳不到 10！
}
```

x 每次加 2，奇数永远跳不到偶数 10。条件用 `!=` 但更新路径绕开了目标值——这也是死循环。解决方案：要么用 `<=` 而不是 `!=`，要么加一个"防止跑过头"的检查。

**第三种：条件中的变量在循环里被改了导致条件永远真。** 比如：

```cpp
int n = 10;
while (n > 0) {
    n = n / 2 * 2;   // n 被"还原"了，永远 > 0？
    cout << n << " ";
}
```

这类比较隐蔽，但本质都是：**条件中依赖的变量没有沿着"让条件变假"的方向演变。**

（在黑板上写：**预防死循环的三个检查点**）

好，怎么预防死循环？每次写完一个 while 循环，问自己三个问题：

1. ✅ **循环变量初始化了吗？** — 循环前给它一个合适的初始值
2. ✅ **循环体内有更新循环变量的语句吗？** — 没有的话，条件永远不会变
3. ✅ **更新方向会让条件最终变为 false 吗？** — 用具体数字验证：假设初始值，模拟几轮，看条件最终会不会变假

这三个问题就像出门前的"手机、钥匙、钱包"检查——每写完一个 while，在脑子里过一遍这三个问题，死循环的概率能降 90%。

（在黑板上写：初始化了吗？更新了吗？能走到退出吗？）

死循环不全是坏事。有时候你就需要一个"永远在跑"的循环——比如游戏的主循环、服务器的监听循环。这种场景下，C++ 标准写法是：

```cpp
while (true) {
    // 游戏主逻辑
    // 当玩家选择退出时，用 break 跳出
    if (quit) break;
}
```

但即使用 `while (true)`，循环体里也必须有 `break`——否则就是真的死循环。下一章我们会专门学 break。

最后，讲一下循环条件设计的好习惯。用"哨兵"模式让退出条件明确：

```cpp
// ❌ 不好——用户不知道输入什么能退出
int x;
while (x != 42) {
    cin >> x;
    cout << x * 2 << endl;
}

// ✅ 好——明确告诉用户输入 -1 退出
int x;
cout << "请输入数字（-1 退出）：";
while (cin >> x && x != -1) {
    cout << x * 2 << endl;
}
```

小结：死循环是每个程序员的必经之坑，但也是最能帮你理解循环本质的体验。写完 while 后，用三个检查点过一遍——初始化、更新、方向——你就能把死循环的概率控制到最低。

## 🖥️ 板书/屏幕要点
- 死循环 = 条件永远成立 + 没有出口
- 三种成因：忘更新、方向错、变量被还原
- 三检查点：初始化了吗？更新了吗？能退出吗？
- `while (true)` 合法但要配合 break

## ✏️ 随堂练习

```cpp
int n;
cin >> n;
while (n != 0) {
    if (n % 2 == 0) n = n / 2;
    else n = n * 3 + 1;
    cout << n << " ";
}
```

## 📌 小结
- 死循环：条件永远成立，永远退不出
- 三检查点：初始化、更新、退出方向
- 有意识的 `while(true)` + break 是合法模式
- 用哨兵值明确退出条件

# 第14章 第3课时：只跳一层——嵌套循环中的 break 陷阱
**时长**：15-20分钟 | **对应教材**：第14章 · 14.3节

## 🎯 本课时目标
- 理解 break 在嵌套循环中只跳出当前所在的那一层
- 通过实验验证"内层 break 管不到外层"
- 理解这个限制背后的设计逻辑

## 📝 讲稿正文

同学们好！前面两节课我们学了 break 和 continue，它们都很"安分"——在单层循环里用得清清楚楚。但今天我们要面对一个重要的陷阱：**在嵌套循环里写 break，它只跳出它所在的那一层！**

（在黑板上写：⚠️ break 只跳出最近的一层循环）

很多同学会想当然地认为："我在最内层写 break，那所有循环都应该停下来。"——这是最常见的误解。我们直接上代码验证：

```cpp
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= 5; j++) {
        if (j == 3) {
            break;  // ← 这个 break 跳到哪里？
        }
        cout << "(" << i << "," << j << ") ";
    }
    cout << endl;
}
```

来，猜猜输出是什么？

（等学生思考）

正确答案是：

```
(1,1) (1,2)
(2,1) (2,2)
(3,1) (3,2)
(4,1) (4,2)
(5,1) (5,2)
```

看到了吗？每一行内层循环只跑到 j=2（到 3 就 break 了），但**外层 i 从 1 到 5 全部执行了一遍**！

（指着输出逐行讲解）

为什么会这样？因为 `break` 只作用于它**直接所在的**那个循环。在这个例子里，break 在 `for (int j = ...)` 的内层循环里，所以它只跳出内层的 j 循环。外层 i 循环完全不受影响——它继续按自己的节奏走。

这就像大楼里的消防门——每一层有每一层的门。你在 3 楼推开消防门，只能离开 3 楼，不等于你离开了整栋大楼。每层循环有每层循环的 break——它们各自为政，互不相干。

（在黑板上画"楼层的门"示意图）

这个规则在 C++ 里是铁律——break 永远是"就近跳出"。你需要理解它、接受它，然后学会应对它。

来看一个更实际的例子：在二维格子里找第一个满足某个条件的坐标。

假设我们有一个 5×5 的网格，要找第一个 i×j > 10 的位置：

```cpp
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= 5; j++) {
        if (i * j > 10) {
            cout << "第一个 >10 的：(i=" << i << ", j=" << j << ")" << endl;
            break;  // 只跳出内层 j！
        }
    }
}
```

输出会是什么？它会输出：

```
第一个 >10 的：(i=3, j=4)
第一个 >10 的：(i=4, j=3)
第一个 >10 的：(i=5, j=3)
```

它找到了**每一行**的第一个——因为每次内层 break 后，外层继续下一行。这不是我们想要的"全局第一个"。

如果我们只想找全局第一个（(3,4)），找到了就应该全停。那怎么办？——这就是下一节课的内容：用标记变量跳出多层循环。

好，现在动手做一个实验。把上面的代码跑一遍，确认你的理解。然后在内层 break 改成 continue，观察输出有什么不同，进一步巩固 continue 和 break 在嵌套中的行为。

（给学生练习和观察时间）

小结：break 在嵌套循环中只管它自己那一层。内层的 break 停不了外层。理解这个限制是正确使用 break 的前提。下节课我们学怎么突破这个限制。

## 🖥️ 板书/屏幕要点
- ⚠️ break 只跳出**直接所在的**那一层循环
- 内层 break → 只结束内层，外层照常运行
- 类比：大楼每层的消防门 → 只能离开那一层
- 如需跳出多层 → 需要额外手段（下节课讲）

## ✏️ 随堂练习
运行以下代码，观察输出并解释为什么：
```cpp
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 4; j++) {
        if (j == 2) break;
        cout << i << "," << j << "  ";
    }
    cout << endl;
}
```
然后把 break 改成 continue，再观察。

## 📌 小结
- break 只跳出一层——它所在的那一层
- 嵌套循环中内层 break 不影响外层
- 这是 C++ 的铁律，不是 bug
- continue 在嵌套中也只影响自己那一层

## 📝 讲稿正文

同学们好！今天是第13章的最后一课，我们要学嵌套循环的"毕业作品"——九九乘法表，然后做全章总结。

九九乘法表是每个学编程的人必写的第一个"有实用价值"的程序。它的逻辑很简单：对每一对 (i, j)，输出 `i × j = 乘积`。

（先在黑板上写出标准乘法表的下半三角）

观察标准乘法表：第一行只有 1×1=1，第二行有 1×2=2 和 2×2=4……第 i 行有 i 列。这和我们之前画的数字三角形一模一样！

```cpp
for (int i = 1; i <= 9; i++) {
    for (int j = 1; j <= i; j++) {        // j <= i：下半三角
        cout << j << "×" << i << "=" << i * j << "\t";
    }
    cout << endl;
}
```

注意两个细节：

第一，输出顺序是 `j × i` 而不是 `i × j`。虽然乘法交换律告诉我们结果一样，但标准乘法表是小乘大——j 是小因子，i 是大因子。写成 `j × i` 更符合阅读习惯。

第二，`\t` 是制表符（Tab 键）。它让各列自动对齐，看起来整整齐齐。如果不加 `\t`，1×1=1 和 1×2=2 会紧挨在一起，列不对齐。试试把 `\t` 换成空格，看看混乱的效果——这能让你直观理解 `\t` 的价值。

（运行代码，展示输出效果）

如果你想输出**完整的矩形乘法表**（9×9 全部格子都显示），只需要把内层上限从 `i` 改成 `9`：

```cpp
for (int i = 1; i <= 9; i++) {
    for (int j = 1; j <= 9; j++) {    // j <= 9：完整矩形！
        cout << i << "×" << j << "=" << i * j << "\t";
    }
    cout << endl;
}
```

三角版和矩形版——区别仅仅是内层上限。三角版 `j <= i`，矩形版 `j <= 9`。这再次验证了我们两节课前的结论：**改变内层循环的上限，就能改变输出的形状。**

好，现在做全章总结。

第13章的核心——**循环的嵌套**。我们从一个时钟的比喻出发，建立了两个关键理解：外层管"行"，内层管"列"；外层走一步，内层跑一轮。

在这个基础上，我们画了各种图形：

| 图形 | 内层上限 | 特殊要求 |
|------|----------|----------|
| 矩形 | 固定值 | — |
| 正三角 | `j <= i` | — |
| 倒三角 | `j <= n+1-i` | — |
| 等腰三角 | `2i-1` | 前面加空格 |
| 菱形 | 正+倒 | 分两段 |
| 空心 | if 判断边框 | else 打空格 |
| 乘法表 | `j <= i` 或 `j <= 9` | 格式化输出 |

（在黑板上写出这个对照表）

这些图形练习看起来像玩，但它们在训练你三种核心能力：
1. **对内层/外层变量的控制力**——知道什么时候固定、什么时候动态
2. **对循环条件的精确把握**——`j <= i` 和 `j <= n+1-i` 之间的微妙差别
3. **分解复杂问题的能力**——菱形 = 上半 + 下半，空心 = 边框判断

这三种能力在未来处理数组、矩阵、二维数据时会反复用到。所以不要觉得"画星星"是玩具——它是在练基本功。

现在做最终练习：输入一个奇数 n，打印由星号组成的空心菱形。例如 n=7：

```
   *
  * *
 *   *
*     *
 *   *
  * *
   *
```

这道题融合了菱形和空心的知识，是对第13章的综合检验。

## 📌 小结
- 九九乘法表是嵌套循环的经典应用
- 内层上限决定输出形状（矩形 vs 三角形）
- `\t` 制表符用于多列对齐
- 第13章核心：外层管行、内层管列、上限变形状

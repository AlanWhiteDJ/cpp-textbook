## 📝 讲稿正文

上节课我们写了计数排序的最简版：统计 → 还原。但那个版本有一个问题——**不稳定**。它在还原的时候，所有值相同的元素被无差别地吐出来，原来的顺序全丢了。

今天我们来写"稳定版"的计数排序。它的核心多了一步：**前缀和**。

什么是前缀和？看一个例子。假设值域 0~5，计数数组 `cnt = [0, 2, 1, 3, 0, 1]`。意思是：0 出现了 0 次，1 出现了 2 次，2 出现了 1 次，3 出现了 3 次，4 出现了 0 次，5 出现了 1 次。

现在把 cnt 变成前缀和：`cnt[i] = cnt[i] + cnt[i-1]`。

- cnt[0] = 0
- cnt[1] = 0 + 2 = 2（值 ≤1 的有 2 个元素）
- cnt[2] = 2 + 1 = 3（值 ≤2 的有 3 个元素）
- cnt[3] = 3 + 3 = 6（值 ≤3 的有 6 个元素）
- cnt[4] = 6 + 0 = 6
- cnt[5] = 6 + 1 = 7（值 ≤5 的有 7 个元素）

前缀和之后，`cnt[v]` 的含义变了：**值等于 v 的元素，最后一个应该放在结果数组的位置 cnt[v] − 1**。

然后从原数组的**最后面**往前扫描。碰到一个值 v，它就应该放在 `output[cnt[v] − 1]`，然后把 `cnt[v]` 减一（给下一个同值元素腾位置）。从后往前扫描，保证了稳定性——先碰到的（在原数组中靠后的）放在靠后的位置。

来看完整代码：

```cpp
void countingSortStable(int a[], int n, int output[]) {
    memset(cnt, 0, sizeof(cnt));
    
    // 1. 统计
    for (int i = 0; i < n; i++)
        cnt[a[i]]++;
    
    // 2. 前缀和
    for (int i = 1; i <= MAXV; i++)
        cnt[i] += cnt[i - 1];
    
    // 3. 从后往前放置（关键！保证稳定）
    for (int i = n - 1; i >= 0; i--) {
        output[cnt[a[i]] - 1] = a[i];
        cnt[a[i]]--;
    }
}
```

注意第三步的循环方向：`for (int i = n - 1; i >= 0; i--)`，**从后往前**。如果你写成从前往后，稳定性就没了——因为同值的元素中，后面的会先被放置。从后往前保证：原数组里先出现的同值元素，最终在结果里也在前面。

复杂度呢？统计 O(n)，前缀和 O(MAXV)，放置 O(n)。总复杂度还是 O(n + MAXV)，空间 O(MAXV + n)。比上节课多了 output 数组的空间。

那我问大家一个问题：稳定版为什么比最简版多用了 output 数组？因为最简版可以原地覆盖（直接把 cnt[v] 个 v 塞回 a 里），但稳定版需要知道每个值具体放哪，所以必须用一个输出数组。

好，稳定计数排序学完了。它不仅是计数排序的"完整版"，更是下节课基数排序的**核心组件**。基数排序的每一轮都需要一个稳定的排序算法，而稳定计数排序恰好完美胜任——因为每一位的值域只有 0~9，MAXV=9，O(MAXV) 几乎可以忽略。

## 🖥️ 板书/屏幕要点
- **前缀和作用**：cnt[v] 变成"≤v 的元素总数"，确定最后位置
- **稳定版三步**：统计 → 前缀和 → 从后往前放置
- **稳定性关键**：`for (i = n-1; i >= 0; i--)` 从后往前
- **复杂度**：O(n + MAXV)，空间 O(n + MAXV)
- **与基数排序关系**：基数排序每一轮用稳定计数排序处理一位

## 📌 小结
稳定版计数排序通过前缀和精确定位每个元素的位置，解决了最简版"不稳定"的问题。它的"从后往前放置"技巧是理解基数排序的关键。下节课我们正式学习基数排序——把大数值拆成"位"来排。

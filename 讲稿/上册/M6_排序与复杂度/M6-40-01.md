# 第40章 第1课时：sort 函数——一行搞定排序

**时长**：15-20分钟 | **对应教材**：第40章

## 🎯 本课时目标
- 掌握 `sort()` 函数的基本用法：头文件、参数、默认升序
- 理解"左闭右开"区间约定 `[begin, end)`
- 能够用 `sort()` 对数组和一部分元素排序

## 📝 讲稿正文

同学们，前面我们手写了冒泡、选择、插入——每次都要十几行代码，还只能 O(n²)。今天我要给你们介绍一位"高手"——C++ STL 里的 **`sort()`** 函数。

先讲个真实的故事。小林在模拟赛里碰到一道题："给 n 个人的姓名和成绩排序"。小林心想，我写个冒泡吧，加上比较规则……于是双重循环写出来了。结果 n=100000，程序跑了 15 秒，超时。

老师走过来看了一眼，把那 20 行代码删掉，换成了一行：`sort(a, a + n, cmp);`。提交，0.1 秒通过。

小林震惊了。老师说："竞赛考的不是你写排序，而是你用排序解决问题的能力。`sort()` 才是你该用的工具。把精力花在设计比较规则上，而不是排序本身。"

好，我们来认识这个神器。`sort()` 藏在 `<algorithm>` 头文件里，用法极简：

```cpp
#include <algorithm>
using namespace std;

sort(起始地址, 结束地址);       // 默认升序
```

注意——**结束地址**不是最后一个元素的地址，而是**最后一个元素之后的位置**。这是 STL 的统一约定："左闭右开"区间 `[begin, end)`。

看一个完整例子：

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int a[] = {3, 1, 4, 1, 5, 9, 2, 6};
    int n = 8;

    sort(a, a + n);            // 升序排序

    for (int i = 0; i < n; i++)
        cout << a[i] << " ";
    // 输出：1 1 2 3 4 5 6 9
    return 0;
}
```

这里 `a` 是数组名，表示数组首地址。`a + n` 就是首地址往后偏移 n 个位置——恰好是最后一个元素的下一个位置。

那如果我只想排数组的一部分呢？比如只排前 5 个元素？

```cpp
sort(a, a + 5);   // 只排 a[0] 到 a[4] 这 5 个元素
```

如果要对 vector 排序呢？vector 不能用 `a + n` 这种指针写法，因为 vector 不是数组。要用迭代器：

```cpp
vector<int> v = {3, 1, 4, 1, 5};
sort(v.begin(), v.end());   // vector 用 begin() 和 end()
```

这个 `sort()` 的速度有多快？它的底层是**内省排序**——混合了快速排序、堆排序和插入排序的智能算法，平均 O(n log n)。对比一下：O(n²) 的冒泡在 n=100000 时要执行 100 亿次操作；O(n log n) 的 sort 只需大约 170 万次级别。差了近 6000 倍！

在竞赛中，只要题目没有明确要求手写某种排序，**`sort()` 就是你的默认选择**。不要重复造轮子——造出来的轮子往往还没标准库的好。

## 🖥️ 板书/屏幕要点
- **头文件**：`#include <algorithm>`
- **基本用法**：`sort(begin, end)` → 默认升序，O(n log n)
- **区间约定**：左闭右开 `[begin, end)`
- **vector**：`sort(v.begin(), v.end())`
- **底层算法**：内省排序（快排 + 堆排 + 插入），比你手写的强多了

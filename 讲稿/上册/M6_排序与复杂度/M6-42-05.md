## 📝 讲稿正文

前面我们学了两种"不比较"的排序：计数排序和基数排序。今天先介绍第三种——**桶排序**，然后做一次完整总结。

桶排序可以理解为计数排序的"泛化版"。计数排序每个"值"一个桶，桶排序是把值域**划分成若干个区间**，每个区间一个桶。元素先扔进对应的桶里，然后每个桶内部再单独排序（随便用什么方法，比如 sort），最后把桶按顺序倒出来。

举个直观例子：你要给全校 1000 个人的身高排序。身高的范围大约是 140cm 到 200cm。你可以先设 6 个桶：140-150、150-160……190-200。每个人扔进对应的桶。每个桶里再单独排，因为桶里人不多，随便用什么方法都很快。最后按桶的顺序输出。

桶排序的核心代码框架：

```cpp
void bucketSort(int a[], int n) {
    // 1. 确定桶的数量和每个桶的范围
    // 2. 把每个元素放进对应的桶
    // 3. 每个桶内部排序（用 sort）
    // 4. 按桶顺序合并
}
```

桶排序在**数据均匀分布**时效果最好——每个桶的元素数量差不多，平均能达到 O(n)。但如果数据分布极端（比如所有数据都挤在一个桶里），就退化成了桶内排序算法的复杂度。

竞赛中桶排序用得不多——因为 sort 已经 O(n log n) 了，加上分桶的额外开销，往往不如直接 sort。但是"分桶"这个思想在别的算法里经常出现，比如分块、莫队算法（M8 会接触）。

好，现在我们来对比三种非比较排序：

|  | 计数排序 | 基数排序 | 桶排序 |
|------|---------|---------|-------|
| 时间复杂度 | O(n+MAXV) | O(d×n) | 平均 O(n) |
| 空间复杂度 | O(MAXV) | O(n) | O(n+k) |
| 适用条件 | 值域小 | 整数，位数少 | 数据均匀分布 |
| 稳定性 | ✅ 可稳定 | ✅ 稳定 | 取决于内部排序 |
| 能否排浮点数 | ❌ | ❌ | ✅ |
| 能否排字符串 | ❌ | ✅（定长） | ✅ |

计数排序：简单直接，但值域一炸就完蛋。

基数排序：解决了值域大的问题——通过拆位把值域压到 10。但只能排整数（或可以逐位处理的数据）。对 int 就是 O(10n)，常数比 sort 大一点但渐近更优。有时对海量数据有奇效。

桶排序：最灵活——可以排浮点数、可以排字符串。但性能依赖于数据分布和桶的设计。

那么竞赛中怎么选？我的建议是：

- **一般情况下**：直接用 `sort()`，不要想太多。
- **值域很小（如 0~1000）且 n 很大（如 10⁶）**：考虑计数排序，可能比 sort 快。
- **值域很大但位数少，且 n 极大**：考虑基数排序。
- **桶排序**：基本用不到，了解思想即可。

记住一句话：**没有万能排序，每种排序都有它的"舒适区"。** 把 sort 当默认工具，特殊场景才动用计数和基数。

最后，给你一个检查清单——面对一道需要排序的题：
1. n 有多大？值域多大？→ 决定基本的复杂度要求
2. 需要稳定吗？→ 决定用 sort 还是 stable_sort
3. 在排序过程中需要做额外的事吗？（如求逆序对）→ 可能需要手写归并

## 🖥️ 板书/屏幕要点
- **桶排序**：值域分区间→元素入桶→桶内排序→合并
- **三种非比较排序对比**：计数 vs 基数 vs 桶
- **选择策略**：默认 sort / 值域小且 n 大 → 计数 / 位数少且 n 极大 → 基数
- **桶排序**：思想重要（分桶），但竞赛中直接用的情况少

## 📝 讲稿正文

前几节课我们一直在学"怎么看算法的快慢"，从今天开始，我们来**亲手写**排序算法。

第一个登场的叫**冒泡排序**。这个名字非常形象，你想象一杯汽水——气泡咕噜噜地从底下往上冒，大的气泡冒得快。冒泡排序也一样：每一轮比较相邻的两个元素，如果前面比后面大就交换，这样一轮下来，最大的数就像气泡一样"冒"到最后面去了。

我们来看一个具体的例子。假设数组是 `[5, 3, 8, 1]`，我要从小到大排。

第一轮：先看 5 和 3，5 比 3 大，交换→变成 `[3, 5, 8, 1]`。再看 5 和 8，5 比 8 小，不动→ `[3, 5, 8, 1]`。再看 8 和 1，8 比 1 大，交换→ `[3, 5, 1, 8]`。一轮结束，8 就稳稳地到了最后面。

第二轮：范围缩小到前三个。3 和 5，不动。5 和 1，交换→ `[3, 1, 5, 8]`。5 到了倒数第二。

第三轮：范围缩小到前两个。3 和 1，交换→ `[1, 3, 5, 8]`。排好了！

n 个元素，最多需要 n−1 轮。每轮比较的范围逐渐缩小——因为后面的已经排好了嘛。

好，我们来看代码怎么写：

```cpp
void bubbleSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {        // 第 i 轮
        for (int j = 0; j < n - 1 - i; j++) { // 比较相邻
            if (a[j] > a[j + 1]) {
                swap(a[j], a[j + 1]);
            }
        }
    }
}
```

注意内层循环的边界：`j < n - 1 - i`。为什么减 i？因为每轮结束，最大的 i 个元素已经排好在数组末尾了，不需要再检查。

现在我再给大家一个重要的优化。想想看——如果某一轮从头走到尾，一次交换都没有发生，说明什么？说明数组已经排好了！后面几轮全白跑。所以我们可以加一个标记：

```cpp
void bubbleSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;                 // ✅ 标记
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                swap(a[j], a[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;                  // ✅ 没有交换,提前退出
    }
}
```

这个优化的威力有多大？最好的情况——数组本来就有序——加了这个标记后，第一轮跑完发现没交换，直接结束。时间复杂度从 O(n²) 降到 O(n)！当然，最坏情况（数组完全倒序）还是 O(n²)。

冒泡排序的空间复杂度是 O(1)——只用了几个变量，没有开额外的数组。而且它是**稳定**的：值相等的元素不会交换，相对顺序保持不变。稳定性这件事我们后面会专门讲。

## 🖥️ 板书/屏幕要点
- **冒泡排序**：相邻比较，大的后移，一轮一个最大"冒"到最后
- **轮数**：最多 n−1 轮，每轮比较范围缩小
- **优化**：`swapped` 标记，没有交换就提前结束 → 最好 O(n)
- **复杂度**：最坏 O(n²)，最好 O(n)，空间 O(1)，稳定

## 📌 小结
冒泡排序是排序算法的"入门第一课"。核心就八个字：**相邻比较，大的后移**。记住加 swapped 优化，它能让你在数组基本有序时跑得飞快。下节课我们学第二种 O(n²) 排序——选择排序，它的"选最小放前面"和冒泡的"冒最大到最后"思路恰好相反。

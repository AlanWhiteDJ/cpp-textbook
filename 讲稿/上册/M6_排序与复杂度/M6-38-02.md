# 第38章 第2课时：大O记号——算法的"速度等级"

**时长**：15-20分钟 | **对应教材**：第38章

## 🎯 本课时目标
- 理解大 O 记号的含义：只关注增长最快的项，忽略常数
- 记住常见复杂度等级：O(1)、O(log n)、O(n)、O(n²)、O(2ⁿ)
- 能够识别双重循环的 O(n²) 和对数循环的 O(log n)

## 📝 讲稿正文

上节课我们认识了 O(n)，知道单层循环差不多执行 n 次。但光知道 O(n) 可不够——现实中算法的复杂度五花八门。今天我们来认识一门"官方语言"：**大 O 记号**。

大 O 记号是什么呢？你可以把它理解成**算法的"速度等级"**。就像车速——你是走路、骑自行车，还是开汽车，这是"等级"的差别。具体是 58 公里每小时还是 62 公里每小时，这不是大 O 关心的。大 O 只看等级，不看细节。

这个"不看细节"就引出了大 O 记号的第一个重要规则：**忽略常数**。

比如这段代码：

```cpp
for (int i = 1; i <= n; i++) doSomething();     // 执行 n 次
for (int i = 1; i <= n; i++) doSomethingElse(); // 又执行 n 次
```

两个 for 循环，总共执行了 2n 次。按大 O 记号的规则，常数 2 被忽略，所以复杂度还是 O(n)，不是 O(2n)。为什么可以忽略？因为不管你是跑 2n 次还是 3n 次，数据翻倍的时候，执行次数都是翻倍——增长趋势没变。

好，现在来看另一种情况。如果有一个双重循环嵌套呢？

```cpp
int count = 0;
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= n; j++) {
        count++;
    }
}
```

外层走 n 次，内层每次也走 n 次，总共执行了多少次？n × n = n² 次。这个复杂度就是 **O(n²)**——平方级。

O(n²) 有多可怕？我给大家算一算：如果 n=1000，n² 就是一百万次，还行；n=10000，n² 就是 1 亿次，开始慢了；n=100000，n² 是 100 亿次，基本就超时了。相比之下，O(n) 在 n=100000 时只要十万次——这就是等级的差别！

接下来看一个有意思的：

```cpp
int x = 1;
while (x < n) {
    x = x * 2;
}
```

x 的变化轨迹是：1 → 2 → 4 → 8 → 16 → ... 每次翻倍。翻多少次会超过 n 呢？大约 log₂n 次。比如 n=1024，x 翻 10 次就到 1024 了。这个循环的复杂度是 **O(log n)**——对数级。这是非常快的增长，接近常数。

现在我把常见的复杂度从快到慢排个队，大家记一下：

- **O(1)**：常数级——跟 n 无关，直接出答案。比如用公式 `n*(n+1)/2` 求 1 到 n 的和
- **O(log n)**：对数级——二分查找、每次范围减半
- **O(n)**：线性级——一轮 for 循环
- **O(n log n)**：线性对数级——高效的排序算法，比如快排和归并排序
- **O(n²)**：平方级——双重 for 循环
- **O(2ⁿ)**：指数级——暴力递归，n 稍微大一点就爆炸

还有一个规则：**高等级"吞掉"低等级**。比如一段代码前面有个 O(n²) 的双重循环，后面有个 O(n) 的单层循环，总复杂度是 O(n²)，因为 n² 比 n 增长快得多，n 那部分在 n 很大时根本不值一提。

好，今天我们认识了大 O 这个"速度等级"体系。记住口诀：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)。

## 🖥️ 板书/屏幕要点
- **大 O 记号规则**：只保留增长最快的项，忽略常数和系数
- **常见复杂度**：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
- **O(n²)**：双重循环，n=100000 时 100 亿次，基本超时
- **O(log n)**：每次翻倍/减半的循环，非常快
- **高等级吞掉低等级**：O(n² + n) = O(n²)

## 📌 小结
今天正式认识了"大 O 记号"——算法速度的等级标尺。记住它的核心规则：忽略常数、只看最高等级。几种常见等级从快到慢排好序，这个顺序在竞赛中非常非常重要。下次遇到一道题，你先看看 n 有多大，然后就知道必须用哪个等级的算法了。

# 第38章 第3课时：一眼看出循环的复杂度

**时长**：15-20分钟 | **对应教材**：第38章

## 🎯 本课时目标
- 掌握四条"看复杂度法则"，能够快速判断代码的复杂度
- 特别关注：嵌套循环的乘积、循环变量翻倍的对数特征
- 理解为什么常数项和低阶项在大 O 中被吞掉

## 📝 讲稿正文

前两节课我们学习了时间复杂度的概念和大 O 记号。今天我们来点"实战训练"——给你一段代码，你能不能一眼看出它是 O(几) 的？

我教大家四条法则，请记牢。

**法则一：单层循环，从 1 到 n → O(n)**

这个最简单。`for (int i = 1; i <= n; i++)`，里面的语句执行 n 次，就是 O(n)。注意，步长是多少不重要——`i+=2` 是 O(n)，`i+=5` 也是 O(n)，因为次数是 n/2 或 n/5，常数被大 O 忽略。

**法则二：嵌套循环 → 各层循环次数的乘积**

看这段代码：

```cpp
for (int i = 1; i <= n; i++)        // n 次
    for (int j = 1; j <= n; j++)    // n 次
        doSomething();              // 总共 n² 次 → O(n²)
```

外层走 n 次，内层每次走 n 次，乘起来就是 n²。再复杂一点：

```cpp
for (int i = 1; i <= n; i++)        // n 次
    for (int j = 1; j <= m; j++)    // m 次
        doSomething();              // 总共 n×m 次 → O(nm)
```

这时候有两个变量，复杂度就是 O(nm)，不能简化。

**法则三：循环变量每次翻倍 → O(log n)**

这个比较容易被忽视。看这段：

```cpp
for (int i = 1; i <= n; i *= 2) {
    doSomething();
}
```

i 的变化：1, 2, 4, 8, 16, 32, ... 一直翻倍到超过 n。翻多少次？log₂n 次。所以复杂度是 **O(log n)**。注意——不管是 `i*=2` 还是 `i*=3` 还是 `i*=10`，都是 O(log n)，因为底数的差异只是一个常数因子，被大 O 吞掉了。

还有一种写法：

```cpp
int i = n;
while (i > 1) {
    i = i / 2;
    doSomething();
}
```

i 每次减半：n → n/2 → n/4 → ... → 1，循环次数也是大约 log₂n。同样是 O(log n)。

**法则四：只看最大项，忽略低阶项和常数**

看这个例子：

```cpp
for (int i = 1; i <= n; i++)        // n 次 → O(n)
    for (int j = 1; j <= n; j++)    // n² 次 → O(n²)
        doSomething();
for (int i = 1; i <= n; i++)        // n 次 → O(n)
    doSomethingElse();
```

总执行次数是 n² + n 次。但大 O 只看最高项——O(n²)。那个 n 在 n² 面前根本不够看。

现在我们来一个"混合型"的，考考大家：

```cpp
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= n; j *= 2)
        doSomething();
```

外层是 O(n)，内层 j 每次翻倍是 O(log n)，乘起来就是 **O(n log n)**。这就是高效排序算法的复杂度等级，后面学快排和归并的时候你会反复听到它。

还有一个经典陷阱题：

```cpp
int s = 0;
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= i; j++)
        s++;
```

外层 i 从 1 到 n。内层 j 从 1 到 i。执行次数 = 1+2+3+...+n = n(n+1)/2 ≈ n²/2。所以是 O(n²)，不是 O(n³)！这里虽然也是嵌套循环，但内层的范围在变，要仔细算。

## 🖥️ 板书/屏幕要点
- **法则一**：单层循环 1→n → O(n)
- **法则二**：嵌套循环 → 各层次数乘积
- **法则三**：循环变量翻倍/减半 → O(log n)
- **法则四**：只保留最高项，忽略低阶项和常数
- **经典陷阱**：`for(j=1;j<=i;j++)` 是 O(n²)，不是 O(n)

## ✏️ 随堂练习
分析以下代码的复杂度：
```cpp
// (a)
for (int i = 1; i <= n; i += 2)
    for (int j = 1; j <= m; j++)
        doSomething();

// (b)
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= n; j *= 2)
        doSomething();

// (c)
int i = n;
while (i > 1) {
    i = i / 2;
    doSomething();
}
```

## 📌 小结
今天学会了"一眼看复杂度"的四条法则。最关键是：看到嵌套就乘起来，看到翻倍就是 log n，看到复杂循环就算总次数。这几条法则在后面的排序学习和竞赛中会反复用到，必须练熟。下节课我们讨论空间复杂度和最好/最坏/平均情况。

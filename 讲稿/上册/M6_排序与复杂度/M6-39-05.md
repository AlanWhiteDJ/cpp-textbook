# 第39章 第5课时：基础排序综合练习

**时长**：15-20分钟 | **对应教材**：第39章

## 🎯 本课时目标
- 综合运用三种 O(n²) 排序解决实际问题
- 掌握按自定义规则排序的方法（如按个位数排序）
- 了解双向冒泡排序（鸡尾酒排序）作为拓展

## 📝 讲稿正文

前三节课我们把冒泡、选择、插入都学了一遍，今天来一次综合练习，考验大家到底有没有真正掌握。

先来看第一道题：输入 n 个数，请**分别**用冒泡、选择、插入三种方法排序，每排好一种就输出结果。这题看起来很基本，但它要求你快速无误地写出三个完整函数。写完之后对比三个输出——结果必须一模一样，这就验证了你的实现是否正确。

写代码的时候检查这几件事：冒泡排序内层循环的边界是不是 `n-1-i`？选择排序的 `minIdx` 是不是在扫描后更新了？插入排序的 while 条件是不是 `j >= 0` 写在前面？如果有任何一条没做好，结果可能就不对了。

好，第二道题稍微有趣一点：**按个位数排序**。输入 n 个数，用插入排序的方法，按它们的**个位数从小到大**排。如果个位数相同，数值小的在前。

这个题的要点是——你不能先把数据做某种变换再排序，而要**直接修改比较逻辑**。在插入排序中，比较发生在 `while (j >= 0 && a[j] > key)` 这里。现在 `>` 不够用了，因为比较的不是数值本身，而是个位数。

怎么办呢？我们把比较抽象成一个自定义的逻辑。可以写一个小函数来判断"a 是不是应该排在 b 前面"：先比较 `a%10` 和 `b%10`，如果个位数不同，个位数小的在前面；如果个位数相同，再比数值本身，数值小的在前面。

在插入排序里，把 `a[j] > key` 替换为：个位数更大，或者个位数相同但数值更大。这样就可以用自定义规则排序了。

这其实是一个重要的意识——**排序的核心是"比较规则"**。一旦你理解了如何自定义比较，不管题目要求按什么排序，你都能应对。这个思想在下一章学 `sort()` 和 `cmp` 时会更强大。

第三道题是一个拓展——**鸡尾酒排序**，也叫双向冒泡排序。普通的冒泡只从左边走到右边，把最大值冒到最后。但你想过没有——能不能来回冒？先从左到右把最大值冒到右边，再从右到左把最小值冒到左边，如此来回。这样一轮可以同时排好最大和最小两个元素。

鸡尾酒排序的好处是：对于某些特殊的数组（比如 `[2,3,4,5,1]`，只有 1 在前面捣乱），普通冒泡需要 4 轮才能把 1 挪到前面，单向冒泡要跑很多轮。双向冒泡一轮就能把它推过去。

不过鸡尾酒排序最坏情况还是 O(n²)，和普通冒泡一样。它只是一个有意思的变种，让大家看到——算法的改进往往来自于对过程的细心观察。

今天的练习清单：写三种排序函数 → 按自定义规则排序 → 了解鸡尾酒排序。希望大家通过这些练习，把 O(n²) 排序的手写能力练扎实。从下节课开始，我们就不再手写 O(n²) 排序了——因为 C++ 给了我们一把"瑞士军刀"：`sort()` 函数，一行代码，O(n log n)，还能自定义规则！

## 🖥️ 板书/屏幕要点
- **综合练习目标**：手写三种 O(n²) 排序，确保无误
- **自定义排序规则**：修改比较逻辑，不修改数据本身
- **鸡尾酒排序**：双向冒泡，来回"冒"，一轮排好两端
- **即将过渡**：从手写 O(n²) → 使用 `sort()`（下一章）

## ✏️ 随堂练习
1. 写三个完整函数 `bubbleSort()`, `selectionSort()`, `insertionSort()`，对数组 `{29, 10, 14, 37, 13}` 分别排序并输出，确认三个结果一致。
2. 用插入排序实现"按个位数升序，个位数相同按数值升序"，测试 `{31, 45, 22, 18, 14}`。（个位数：1, 5, 2, 8, 4 → 排序后应为 31, 22, 14, 45, 18）

## 📌 小结
第 39 章「基础排序」到此结束。冒泡、选择、插入——三种 O(n²) 排序虽然速度不快，但它们是理解排序思想的基石。你已经能用手写出这三种排序的完整代码，也能按自定义规则修改比较逻辑。从下章开始，我们走进"C++ 标准武器库"——`sort()` 函数和自定义 cmp。

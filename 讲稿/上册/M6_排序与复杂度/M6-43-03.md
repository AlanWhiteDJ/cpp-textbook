## 📝 讲稿正文

第 41 章我们简单提过逆序对，今天来彻底拿下它。逆序对是排序最经典的应用之一，很多竞赛题换个马甲就是在考逆序对。

先复习定义：在一个序列中，如果 **i < j 但是 a[i] > a[j]**，那么 (a[i], a[j]) 构成一个**逆序对**。简单说：**前面的数比后面的数大，这就是一个逆序对**。

为什么叫"逆序对"？因为排好序的序列是"前面小后面大"——这叫顺序。反过来，前面大后面小违背了这个规则，所以叫"逆序"。

来看个例子：序列 `[3, 1, 4, 2]`。我们一个一个排查：

- 看 3：后面有 1（3>1✅逆序），4（3<4❌），2（3>2✅逆序）→ 2 个
- 看 1：后面有 4（1<4❌），2（1<2❌）→ 0 个
- 看 4：后面有 2（4>2✅逆序）→ 1 个
- 看 2：后面没了 → 0 个

总共 = 2 + 0 + 1 + 0 = 3 个逆序对。它们是 (3,1)、(3,2)、(4,2)。

逆序对有一个非常重要的物理意义：**逆序对的数量 = 用相邻交换使序列升序的最小交换次数**。换句话说，冒泡排序中的交换次数就等于逆序对的数量。你可以验证——对 `[3,1,4,2]` 做冒泡排序，看交换了几次，是不是 3 次？

但注意：**只有相邻交换才等于逆序对的数量**。如果允许任意交换（比如选择排序的交换），那最少的交换次数就不能用逆序对来算了，需要用"环"的方法。

那怎么计算逆序对呢？最直觉的方法就是双重循环暴力：

```cpp
long long countInversions(int a[], int n) {
    long long inv = 0;
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (a[i] > a[j])
                inv++;
        }
    }
    return inv;
}
```

非常直观：对每个位置 i，检查它后面所有比它小的元素。复杂度 O(n²)。

这个暴力方法在 n ≤ 10000 时还可以，但在 n = 100000 时，n² = 10¹⁰，绝对超时。而且我要提醒一个**非常容易被忽略的坑**：

**逆序对的数量可能非常大！大到超出 int 的范围！**

最坏情况——序列是严格降序排列的，比如 `[n, n-1, ..., 2, 1]`。逆序对的数量 = (n−1) + (n−2) + ... + 2 + 1 = n(n−1)/2 ≈ n²/2。

如果 n = 100000，n²/2 = 5 × 10⁹，这已经超过了 int 的最大值（约 2.1 × 10⁹）。如果用 `int` 存结果，**数据会溢出变成负数**，答案完全错。

所以求逆序对时，**结果必须用 `long long` 存储！** 这是竞赛中无数人踩过的坑——算法是对的，结果因为溢出拿了零分。

那 n 大的时候怎么办？必须用 O(n log n) 的方法。目前有两种主流方法：
- **归并排序法**：在 merge 的过程中顺便计数，O(n log n)。下节课细讲。
- **树状数组法**：M8 模块会详细展开，也是 O(n log n)。

两种方法竞赛都会考，先掌握归并法即可搞定大多数情况。

## 🖥️ 板书/屏幕要点
- **逆序对**：i < j 且 a[i] > a[j]
- **物理意义**：相邻交换排序的最小次数 = 逆序对数
- **暴力求法**：双重循环 O(n²)
- **🛑 重要**：结果用 **long long**，n=100000 时 n²/2 > int 上限！
- **高效求法**：归并排序 O(n log n) / 树状数组 O(n log n)

## 📌 小结
逆序对是排序的经典延伸。暴力 O(n²) 简单但只适合小数据。记住逆序对可能很多——必须用 long long 存。下节课我们用归并排序实现 O(n log n) 的高效求法。

# 第42章 第1课时：计数排序——用"桶"来排序

- 能够手写简单版计数排序

## 📝 讲稿正文

前面我们学过的排序——冒泡、选择、插入、归并、快排——全都是靠**比较**来决定元素顺序的。这些"比较排序"有一个理论下限：最快只能到 O(n log n)。

但你有没有想过：能不能不比较，直接"算"出每个元素该放哪？能！今天我们要学一种完全不同思路的排序——**计数排序**。

计数排序的思路特别朴素。想象一个场景：你要给全班 50 个人按考试成绩（0 到 100 分）排序。你可以这样——在黑板上摆 101 个盒子，标着 0 分、1 分、2 分……100 分。你挨个念成绩，念到 85 分就把那个人的名字丢进 85 号盒子里。全部念完后，从 100 号盒子开始一个个往外掏——分数从高到低自然排好了。

这个过程中你**没有比较任何两个人的分数**，你只是把每个人归类到对应的盒子里。这就是计数排序的思想。

来看代码：

```cpp
const int MAXV = 100000;  // 最大可能值
int cnt[MAXV + 1];

void countingSort(int a[], int n) {
    // 1. 清空计数数组
    memset(cnt, 0, sizeof(cnt));

    // 2. 统计每个值出现了多少次
    for (int i = 0; i < n; i++)
        cnt[a[i]]++;

    // 3. 根据计数还原有序序列
    int idx = 0;
    for (int v = 0; v <= MAXV; v++) {
        while (cnt[v] > 0) {
            a[idx++] = v;
            cnt[v]--;
        }
    }
}
```

第一步：`memset` 把计数数组全部清零。第二步：遍历原数组，`cnt[a[i]]++` 统计每个值的出现次数。比如 a 里有 3 个 `5`，那 `cnt[5]` 就等于 3。第三步：从最小值到最大值，把每个值按出现次数"吐出"来——v 出现了 cnt[v] 次，就把 cnt[v] 个 v 依次放进数组。

这个算法的时间复杂度是多少？统计那一步，遍历了 n 个元素，是 O(n)。还原那一步，遍历了值域 0 到 MAXV，是 O(MAXV)。总复杂度 = **O(n + MAXV)**。

如果 MAXV 和 n 差不多大（比如都是 10⁵ 级别），那复杂度接近 O(n)，比 O(n log n) 还快！但如果 MAXV 是 10⁹——值域范围一亿——你开不出 `cnt[1000000001]` 这么大的数组，而且遍历值域的时间也受不了。

所以计数排序有一个铁的条件：**值域必须足够小**。具体来说，MAXV 一般不超过 10⁷（一千万），大概就是你能开出来的数组上限。

另外注意，计数排序"只能排整数"——或者说，只能排能映射到整数下标的东西。浮点数没法直接当下标用。

还有一个问题：上面的代码只适用于**非负整数**。如果数据里有负数怎么办？比如数据范围在 `[-100, 100]`。很简单——偏移！把每个数减去最小值，映射到非负范围。先找最小值 minVal，然后 `cnt[a[i] - minVal]++`，还原时 `v + minVal`。

好，今天理解了计数排序的原理和最简单的实现。下节课我们要写"稳定版"的计数排序，它还是基数排序的基础。

## 🖥️ 板书/屏幕要点
- **计数排序**：不比较，不交换，直接统计每个值出现次数
- **三步**：`memset` 清零 → `cnt[a[i]]++` 统计 → 按 cnt 还原
- **复杂度**：O(n + MAXV)
- **适用条件**：值域 MAXV 必须小（一般 ≤ 10⁷），数据必须是整数（或可映射为整数）
- **处理负数**：偏移法，减去 minVal 映射到非负

2. 处理含负数的数据：先找最小值，用偏移法排序 `{-3, 5, -1, 2, -3, 0, 4}`。

## 📌 小结
计数排序打开了"非比较排序"的大门。它的思路极其简洁：统计出现次数，然后按值还原。代价是空间和值域绑定——值域一大就不可用。下节课我们写稳定版并用前缀和优化位置分配。

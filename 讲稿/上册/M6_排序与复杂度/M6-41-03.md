## 📝 讲稿正文

上节课我们学了归并排序——先分后合，左右都排好了再合并。今天来学另一种分治排序：**快速排序**。它和归并有个有趣的对比：归并是"先分，排好再合"，快排是"先分好（大的和小的分开），然后什么都不用合，左右各自递归完就排好了"。

想象一个场景：你有一堆杂乱的试卷，随手抽一张当"标杆"——比如学号 25 的那份。然后把学号小于 25 的放左边，大于 25 的放右边。虽然左右内部还是乱的，但有一点确定：左边的所有试卷都在右边之前。然后左右两堆各抽一张标杆，继续分，直到每堆只剩 1 张。

这个"标杆"在快排里叫**基准值（pivot）**。分左右的过程叫**分区（partition）**。

我们来看一个具体例子。数组 `[38, 27, 43, 3, 9, 82, 10]`，选最后一个元素 10 当基准。

partition 怎么做？我们需要两个指针：`i` 标记"小于区"的右边界，`j` 用来扫描。

初始时 i = −1（小于区为空）。j 从 0 扫描到 n−2（基准的前一个）。

- j=0：a[0]=38 > 10，不管，i 还是 −1。
- j=1：27 > 10，不管。
- j=2：43 > 10，不管。
- j=3：3 ≤ 10！i 变成 0，交换 a[0] 和 a[3] → `[3, 27, 43, 38, 9, 82, 10]`
- j=4：9 ≤ 10！i 变成 1，交换 a[1] 和 a[4] → `[3, 9, 43, 38, 27, 82, 10]`
- j=5：82 > 10，不管。
- j=6：到头了。最后交换 a[i+1] 和 a[6]（基准位置）→ `[3, 9, 10, 38, 27, 82, 43]`

现在 10 在最终的位置上了——左边的 3,9 都 ≤ 10，右边的 38,27,82,43 都 ≥ 10。分别对左右两部分递归做同样的事就行了。

注意，快排不需要"合"这一步！因为分区操作保证了：基准左边的所有元素 ≤ 基准 ≤ 基准右边的所有元素。左右各自递归排好序以后，整个数组自然就有序了。

现在我们看 partition 的代码：

```cpp
int partition(int l, int r) {
    int pivot = a[r];                // 选最右边元素为基准
    int i = l - 1;                   // i 指向"小于区"的末尾
    for (int j = l; j < r; j++) {
        if (a[j] <= pivot) {
            i++;
            swap(a[i], a[j]);        // 把小的换到小于区
        }
    }
    swap(a[i + 1], a[r]);            // 基准放到正确位置
    return i + 1;                    // 返回基准的最终位置
}
```

那快排的复杂度呢？最理想的情况：每次基准恰好把数组分成两个相等的部分，那深度是 log n，每层 partition 开销 O(n)，总共 O(n log n)。

但最坏的情况就惨了——如果每次选的基准恰好是最大或最小值呢？比如数组本来就是 `[1,2,3,4,5]`，你选最后一个 5 当基准，分出来左边 4 个，右边 0 个——每次只减少一个元素。那深度就是 n，每次 partition 扫一遍，总复杂度退化到 O(n²)！

这就是快排的死穴。下节课我们学怎么用随机化来避免这种最坏情况。

## 🖥️ 板书/屏幕要点
- **快排核心**：选基准 → 小的放左、大的放右 → 递归
- **partition 过程**：i 标记小于区末尾，j 扫描，遇到小的就 `swap(a[++i], a[j])`
- **不需要合**：分区保证了全局有序性
- **复杂度**：平均 O(n log n)，最坏 O(n²)（每次基准是极值）
- **与归并对比**：归并"先分后合"，快排"先分好再递归"

## 📌 小结
快速排序的核心是 partition——选基准、分左右。这个操作的精妙之处在于它不需要"合并"这一步。但选基准的方式直接决定了速度。下节课我们来实现完整的快速排序，并用随机化来避免最坏情况。

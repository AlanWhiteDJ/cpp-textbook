## 📝 讲稿正文

前面四节课我们把 `sort` 和 `cmp` 玩得差不多了。今天来补充一个重要话题：**稳定性**，然后总结排序题的套路。

你有没有想过——`sort()` 是稳定的吗？答案你可能已经猜到了：**不，`sort()` 是不稳定的**。它底层用的是快速排序的变体，相等元素的相对顺序可能被改变。

那如果我真的需要稳定性怎么办？比如我先把学生按学号排好了，现在按成绩排——我想让同分的人保持学号的顺序。如果直接用 `sort()`，同分的人学号顺序可能乱掉。

解决办法：**`stable_sort()`**。用法和 `sort()` 一模一样：

```cpp
stable_sort(a, a + n);           // 稳定升序
stable_sort(a, a + n, cmp);      // 稳定的自定义排序
```

`stable_sort()` 保证：排序后相等的元素，原来的相对顺序不会变。代价是比 `sort()` 略微多占一点内存（它底层一般用归并排序），但对于竞赛规模的数据完全不叫事。

那问题来了——既然 `stable_sort` 几乎没缺点，为什么不总是用它？嗯，你完全可以总是用它！很多选手的习惯是：不确定要不要稳定，就用 `stable_sort()`。多一点点开销，换来确定的正确性。

不过，更常用的做法是：**把你的所有规则都写进 cmp**——这样就不需要依赖排序的稳定性了。比如先按成绩排再按学号排，你直接在 cmp 里写"成绩不同按成绩，成绩相同按学号"，一次 sort 搞定。

好，接下来我要给大家总结一下——在实际竞赛中，排序题有哪些常见套路。学了可以直接套。

**模式一：直接排序。** "把 n 个数从小到大输出"。最简单的 sort，一行搞定。但别小看它——它经常是更复杂题目的第一步。

**模式二：多关键字排序。** 排行榜题，刚才练过了。结构体 + cmp，升序降序看仔细。

**模式三：排序后处理。** 排序本身只花 1 行，但排序之后你可以做的事情就多了：去重（检查相邻元素）、求中位数（取 `a[n/2]`）、求众数（统计相邻相同元素）。

**模式四：贪心算法前置。** 很多贪心策略（下一模块会学）需要先把数据排好序。比如活动安排问题——按结束时间排序后才能贪心。sort 在这些题里是"开胃菜"。

**模式五：二分查找前置。** `lower_bound` / `upper_bound` 这些二分查找函数（后面会学）**要求数据必须先排序**。查找之前 sort 一下很常见。

最后，我简单说一下 sort 底层到底用了什么算法——不要求手写，但了解原理能让你更自信。sort 底层用的是**内省排序**——一个混合算法，从快速排序开始（选基准分左右），如果发现递归太深会切成堆排序（保证 O(n log n) 最坏情况），数据量小时切换成插入排序（小数据时插入几乎 O(n)）。所以你用 sort 拿到的，是全世界顶尖工程师调出来的最优算法，比自己手写的强太多。

记住竞赛原则：**除非题目明确要求手写某个排序，否则一律用 sort。**

## 🖥️ 板书/屏幕要点
- **`stable_sort()`**：用法同 sort，保证稳定（相等元素相对顺序不变）
- **最佳实践**：把所有规则写进 cmp，不依赖稳定性
- **五种排序题模式**：直接排序 / 多关键字 / 排序后处理 / 贪心前置 / 二分前置
- **sort 底层**：内省排序 = 快排 + 堆排 + 插入，混合调优
- **竞赛准则**：sort 优先，不重复造轮子

## ✏️ 随堂练习

2. 设计一个场景：先用 `sort()` 排序，然后用 `stable_sort()` 排序同一组数据，观察结果哪里不同。

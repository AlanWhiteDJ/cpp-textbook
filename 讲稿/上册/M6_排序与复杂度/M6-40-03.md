## 📝 讲稿正文

上节课我们用 `greater` 做了降序，又初步接触了 cmp 函数。今天我们来彻底掌握 cmp——它是 sort 的灵魂，也是竞赛中最常用的技巧之一。

先复习一下 cmp 的写法口诀：**`return a 应该排在 b 前面;`**。cmp 返回 `true` 就表示 a 应该排 b 前面，就这么简单。

来看第一个进阶例子——**偶数在前、奇数在后，同类内部升序**：

```cpp
bool cmp(int a, int b) {
    if (a % 2 != b % 2)          // 如果奇偶不同
        return a % 2 < b % 2;    // 偶数(0)排在奇数(1)前面
    return a < b;                 // 奇偶相同，按数值升序
}
```

大家注意这个**分段判断**的模式——先判断最高优先级的条件，如果不相等，直接返回；如果相等，再看下一个条件。这个模式非常清晰，不容易出错。

再看一个例子：**按字符串长度升序排列，长度相同时按字典序**。

```cpp
bool cmp(const string &a, const string &b) {
    if (a.length() != b.length())
        return a.length() < b.length();   // 短的在前
    return a < b;                          // 长度相同,字典序
}

string arr[] = {"cat", "dog", "apple", "banana", "car"};
sort(arr, arr + 5, cmp);
// 结果：{"car", "cat", "dog", "apple", "banana"}
```

好，现在我要强调一个**极其重要**的铁律，很多同学在这一条上栽了跟头，程序直接崩溃。

**cmp 函数里绝对不能出现 `<=` 或 `>=`！**

为什么？因为 C++ 标准要求比较函数必须满足"严格弱序"——简单说就是：如果 `a == b`，那么 `cmp(a, b)` 和 `cmp(b, a)` 都必须返回 `false`。如果你写了 `<=`，当 a 等于 b 时，`cmp(a, b)` 返回 true，同时 `cmp(b, a)` 也返回 true——这就造成逻辑冲突，sort 内部可能会越界访问，程序直接 RE（运行时错误）。

所以记住：❌ `return a.score >= b.score;`  
✅ `return a.score > b.score;`

只允许用 `<` 和 `>`，严禁用 `<=` 和 `>=`。这个坑踩一次就记住了，但代价太大——可能比赛时莫名其妙丢分。

回到 cmp，再看一个实用例子：**对结构体 Person 排序——按年龄升序，年龄相同按身高降序**。

```cpp
struct Person {
    string name;
    int age, height;
};

bool cmp(const Person &a, const Person &b) {
    if (a.age != b.age)
        return a.age < b.age;       // 年龄小的在前
    return a.height > b.height;     // 年龄相同,身高高的在前
}
```

你看，模式完全一样：if-return 逐级判断。第一关键字是年龄，第二关键字是身高。想加第三个关键字？再加一个 if-return 就行。

最后总结一下 cmp 的编写三步法：  
**第一步**：确定排序的"关键字"有哪些，排好优先级  
**第二步**：对每个关键字，写一个 `if (a.key != b.key) return ...;`  
**第三步**：确保只用 `<` 和 `>`，不用 `<=` 和 `>=`

## 🖥️ 板书/屏幕要点
- **cmp 黄金口诀**：`return a 应该排在 b 前面;`
- **分级优先级模式**：
  ```
  if (a.key1 != b.key1) return a.key1 < b.key1;
  if (a.key2 != b.key2) return a.key2 < b.key2;
  return a.key3 < b.key3;
  ```
- **铁律**：cmp 中**禁止** `<=` 和 `>=`，只能用 `<` 和 `>`
- **原因**：违反严格弱序 → sort 内部行为不确定 → 可能崩溃

## 📌 小结
cmp 是 sort 的"灵魂"。分级优先级模式清晰且安全，记住只用 `<` 和 `>`。下节课我们深化结构体排序——多关键字排行榜是竞赛中的高频题型。

# 第38章 第1课时：为什么要分析算法速度

**时长**：15-20分钟 | **对应教材**：第38章

## 🎯 本课时目标
- 理解算法效率的重要性：同样的功能，不同写法速度差异可能巨大
- 初步了解时间复杂度的概念——用循环执行次数衡量程序快慢
- 能够估算单层循环的复杂度 O(n)

## 📝 讲稿正文

同学们，前面五个模块我们学了变量、分支、循环、数组、函数和结构体。现在你已经能写出能跑的程序了。但是今天我要问大家一个问题——你有没有想过，**你写的程序到底有多快**？

我来给大家讲一个小故事。小明和小红各写了一个程序，功能一模一样：判断 n 个数里有没有重复的。小明的写法简单粗暴——双重循环，每个数都和它后面的所有数比较一遍。小红呢，先排序，再检查相邻的数是不是一样。

输入 10 个数的时候，这两个程序都是"秒出结果"，看不出差别。但是！当输入十万个数的时候，奇妙的事情发生了——小明的程序跑了十几秒还在转圈，小红的程序 0.1 秒就出来了。

**同样的功能，为什么速度差了上百倍？** 这就是我们今天要研究的问题——算法的效率。

在信息学竞赛里，一道题的正确解法往往不止一种。但是，只有复杂度合理的算法才能拿满分。有些同学会觉得奇怪："我的程序明明是对的呀，为什么只拿了 30 分？"——答案往往是：**你的算法太慢了**。

好，那我们怎么衡量一个算法的快慢呢？你可能会想：拿个秒表掐一下呗。但我要告诉你，不能只看秒数。为什么？因为同样的代码，在评测机上的服务器跑，可能只要 0.1 秒；在你的老电脑上跑，可能要 2 秒。环境不一样，秒数没法比。

所以计算机科学家发明了一个办法：不看具体秒数，而是**数一数程序执行了多少次基本操作**。比如，一个 for 循环从 1 到 n，里面的语句就执行了 n 次。如果数据量 n 翻一倍，执行次数也大约翻一倍。这个"执行次数随数据规模增长的规律"，就是**时间复杂度**。

来，我们看一段最简单的代码：

```cpp
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;
}
```

这个 for 循环，i 从 1 变到 n，里面的 `sum += i` 执行了多少次？n 次，对吧。我们说这段代码的时间复杂度是 **O(n)**——读作"大 O n"。意思就是：执行次数差不多是 n 这个数量级。

再看如果 n=100，执行大约 100 次；n=10000，大约 10000 次。你看，执行次数和 n 是成正比的，这就是 O(n) 的含义。

为什么那个双重循环的程序会慢一百倍呢？我们下节课就会揭晓——因为它每一轮都要和后面所有的数比较，总比较次数大约是 n²。当 n=100000 时，n² 是 100 亿，而 n log n 只有大约 170 万——这差距可不是一星半点。

好，今天我们先记住第一个复杂度：**O(n)**，也就是线性复杂度。它的特点是：数据量翻一倍，执行时间也大约翻一倍。当你看到一个单层 for 循环从头走到尾，基本就是 O(n) 了。

## 💬 互动提问

3. "下面这段代码执行了多少次？它的复杂度是多少？" —— 展示 `for (int i = 1; i <= n; i++) cout << i;`

## 🖥️ 板书/屏幕要点
- **算法效率**：同样的功能，不同写法速度差异巨大
- **时间复杂度**：程序执行次数随数据规模 n 增长的规律
- **O(n)**：单层循环，执行次数 ≈ n 次
- **经验**：n=10 和 n=100000 时，O(n) 和 O(n²) 的差距可达百倍

## ✏️ 随堂练习
1. 写一个 for 循环，计算 1 到 n 的累加和，并回答：如果 n=1000，`sum+=i` 执行了多少次？
2. 想一想：如果代码里有两个并列的单层 for 循环（从 1 到 n），复杂度是 O(2n) 还是 O(n)？先猜一猜，下节课揭晓。

## 📌 小结
今天我们迈出了算法分析的第一步。核心记住一句话：时间复杂度不看秒数，看**执行次数随数据规模的增长趋势**。单层循环就是 O(n)，数据翻倍时间也翻倍。下节课我们正式认识"大 O 记号"这个衡量算法的标准语言。

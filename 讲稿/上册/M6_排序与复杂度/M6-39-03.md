## 📝 讲稿正文

今天我们来学第三种 O(n²) 排序——**插入排序**。这个排序的思路特别生活化，你一定做过类似的事情：打扑克牌的时候，你手里已经抓了几张牌，排好了顺序。现在又摸到一张新牌，你会怎么做？你会把它**插到**手里已经排好的正确位置。插入排序就是这个逻辑。

假设数组 `[5, 3, 8, 1]`。我们从第二张牌开始处理（第一张牌天然有序）。摸到 3：和前面的 5 比，3 比 5 小，把 5 往后挪，3 放在 5 原来的位置→ `[3, 5, 8, 1]`。摸到 8：和前面的 5 比，8 比 5 大，不用动→ `[3, 5, 8, 1]`。摸到 1：先和 8 比，1 比 8 小，8 往后挪；再和 5 比，1 比 5 小，5 往后挪；再和 3 比，1 比 3 小，3 往后挪；前面没了，1 放在最前面→ `[1, 3, 5, 8]`。排好了！

来看代码实现：

```cpp
void insertionSort(int a[], int n) {
    for (int i = 1; i < n; i++) {             // 摸第 i 张牌
        int key = a[i];                        // 这张牌的值
        int j = i - 1;
        while (j >= 0 && a[j] > key) {          // 比 key 大的往后挪
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = key;                        // 插入到正确位置
    }
}
```

这段代码里有一个**极其重要**的细节，我必须重点强调。大家看 while 的条件：`while (j >= 0 && a[j] > key)`。这里 `j >= 0` 必须写在 `a[j] > key` 的**前面**！为什么？

因为 C++ 的 `&&` 是**短路求值**——如果左边的条件为 false，右边的条件根本不会执行。当 `j` 变成 `-1` 的时候，`j >= 0` 为 false，`a[-1]` 就不会被访问了，避免了数组越界。如果你写反了——`while (a[j] > key && j >= 0)`——先访问 `a[-1]`，程序就崩了！

这个坑很多同学都会踩，一定要记住：**先检查下标是否有效，再访问数组**。

好，来看复杂度。最坏情况：数组是降序排列的，每摸一张牌都要一路挪到头。第 i 个元素最多挪 i 次，总移动次数 = 1+2+3+...+(n-1) ≈ n²/2，O(n²)。

最好情况：数组本来就是升序的。每摸一张牌，while 条件 `a[j] > key` 一开始就不成立，一步都不挪，只比较了 1 次。总共比较 n−1 次，O(n)！所以插入排序在数据基本有序时非常快。

空间还是 O(1)，用了几个变量而已。

插入排序是**稳定**的。因为挪动的时候，只有 `a[j] > key` 才挪，相等的元素不会被挪到前面，相对顺序不变。

总结一下插入排序的特长：它是三种 O(n²) 排序中在"数据基本有序"时表现最好的。还有一个好处——它可以"在线"排序：数据是一个一个来的，来一个排一个，不需要等到全部到齐。

## 🖥️ 板书/屏幕要点
- **插入排序**：摸一张牌，插到手中已排好的正确位置
- **关键变量**：`key` = 当前摸的牌，`j` = 已排序部分的最后一个位置
- **短路求值陷阱**：`j >= 0` 必须在 `a[j] > key` 前面！
- **复杂度**：最好 O(n)（基本有序），最坏 O(n²)（倒序），空间 O(1)，✅ 稳定

## 📌 小结
插入排序是三种 O(n²) 排序中最"聪明"的一个——遇到基本有序的数据接近 O(n)，并且可以处理"在线"数据。记住那个短路求值的坑，这是手写排序最容易犯的错。下节课我们把三种排序放到一起对比，再学一个重要的新概念——稳定性。

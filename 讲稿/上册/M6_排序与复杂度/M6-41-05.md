## 📝 讲稿正文

前面四节课我们把两种 O(n log n) 分治排序都学完了。今天先来做个大对比，然后引入排序的一个重要应用——逆序对。

看这张对比表：

|  | 归并排序 | 快速排序 |
|------|---------|---------|
| 时间复杂度 | 总是 O(n log n) ✅ | 平均 O(n log n)，最坏 O(n²) |
| 空间复杂度 | O(n) ❌（需临时数组） | O(log n) ✅（递归栈） |
| 稳定性 | ✅ 稳定 | ❌ 不稳定 |
| 分治顺序 | 先递归，再合并 | 先分区，再递归 |
| 是否需要合 | 需要 merge | 不需要！ |
| 缓存友好 | 一般 | ✅ 好（原地操作） |

归并排序最大的优点是**稳定且永远 O(n log n)**——不挑数据，性能保证。但它需要 O(n) 的额外空间，这是它的代价。

快速排序最大的优点是**原地排序**——不需要临时数组，缓存友好。在随机化后平均表现极佳。但它不稳定，最坏可能退化到 O(n²)。

竞赛中怎么选？**一般情况下用 `sort()`**——它内部是内省排序，综合了快排和堆排的优势，已经做了所有优化。归并排序更大的价值不在排序本身，而在**合并过程中可以顺便做些事**——比如求逆序对。

好，说到逆序对，这是一个非常重要的概念。**逆序对**的定义是：在一个序列中，如果 i < j 但是 a[i] > a[j]，那么 (a[i], a[j]) 就是一个逆序对。

说得通俗点：**前面的数比后面的数大，这两个数就叫一个逆序对**。

比如序列 `[3, 1, 4, 2]` 中有多少逆序对？我们来数一数：
- (3, 1)：3 在 1 前面，但 3 > 1 → 逆序对！
- (3, 2)：3 在 2 前面，3 > 2 → 逆序对！
- (4, 2)：4 在 2 前面，4 > 2 → 逆序对！
- (1, 2)：1 在 2 前面，1 < 2 → 不是逆序对
- 总共 3 个逆序对。

逆序对有什么用？一个很重要的意义：序列中逆序对的个数 = **用相邻交换使序列有序的最小交换次数**。换句话说，冒泡排序的交换次数就等于逆序对的数量。很多竞赛题"求最少交换次数"其实就是求逆序对。

怎么求逆序对？最简单的方法——双重循环暴力 O(n²)，n ≤ 10000 时可行。但 n=100000 时 n²=10¹⁰，铁定超时。必须用 O(n log n) 的方法。

**归并排序可以顺便求逆序对**！原理是这样的：在 merge 合并阶段，如果左半当前元素 a[i] 大于右半当前元素 a[j]，由于左右两半各自已经有序，左半中从 a[i] 到 a[mid] 的**所有元素**都大于 a[j]。这就一下子贡献了 `mid - i + 1` 个逆序对。

下下章我们会完整实现归并求逆序对的代码。今天你只需要知道：逆序对是什么，以及归并排序为什么能算它。

## 🖥️ 板书/屏幕要点
- **归并 vs 快排**：时间/空间/稳定性/分治顺序对比
- **竞赛指南**：一般用 sort()，求逆序对时用归并
- **逆序对定义**：i < j 且 a[i] > a[j]
- **物理意义**：逆序对个数 = 相邻交换排序的最小交换次数
- **求法**：暴力 O(n²) / 归并 O(n log n) / 树状数组（M8）

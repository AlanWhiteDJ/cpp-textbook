# 第40章 第2课时：降序排序——greater 与自定义比较入门

**时长**：15-20分钟 | **对应教材**：第40章

## 🎯 本课时目标
- 掌握用 `greater<int>()` 实现降序排序
- 了解 `greater` 适用于不同数据类型
- 初步接触自定义比较函数 cmp 的概念

## 📝 讲稿正文

上节课我们学会了 `sort(a, a+n)` 默认升序排列。但现实中很多题目要求降序——比如排行榜从高到低排。今天我们就来解决这个问题。

C++ 的 STL 给我们准备了一个现成的"降序工具"：**`greater<int>()`**。它是一个函数对象，含义很简单——"a > b 才算正确"。直接把 `greater<int>()` 作为 sort 的第三个参数就行了：

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int a[] = {3, 1, 4, 1, 5, 9, 2, 6};
    int n = 8;

    sort(a, a + n, greater<int>());    // 降序！

    for (int i = 0; i < n; i++)
        cout << a[i] << " ";
    // 输出：9 8 7 6 5 4 3 2 1
    return 0;
}
```

你看，就加了一个参数 `greater<int>()`，排序方向就反过来了。如果你觉得 `greater<int>()` 有点难记，一个助记法：`greater` 就是"更大的"，所以更大的排在前面。

那如果我排的是 `double` 呢？用 `greater<double>()`。排 `string` 呢？`greater<string>()`。数据类型写在尖括号里就行。

其实 sort 的第三个参数叫什么？叫"比较函数"——它告诉 sort：两个元素 a 和 b，什么情况下 a 应该排在 b 前面。默认的升序等价于 `less<int>()`——"a < b 才算正确"。降序就是 `greater<int>()`——"a > b 才算正确"。

但 `greater` 只能做简单的升序降序翻转。如果题目说"按绝对值升序排列"呢？`greater` 就无能为力了——因为它不知道什么叫"绝对值"。这时候我们需要自己写比较函数：**cmp**。

先看一个最简单的 cmp：

```cpp
bool cmp(int a, int b) {
    return a > b;   // a 大于 b 时，a 排在 b 前面 → 降序
}

sort(a, a + n, cmp);   // 效果和 greater<int>() 一样
```

cmp 函数的原型是：接收两个同类型的参数，返回 `bool`。返回 `true` 表示"第一个参数应该排在第二个参数前面"。理解这个逻辑，你就掌握了自定义排序的钥匙。

我们来一个稍微复杂点的例子——**按绝对值升序**：

```cpp
bool cmpAbs(int a, int b) {
    return abs(a) < abs(b);    // 绝对值小的排前面
}

int a[] = {-5, 3, -2, 4, -1};
sort(a, a + 5, cmpAbs);
// 结果：{-1, -2, 3, 4, -5}
// 因为 abs: 1<2<3<4<5
```

你看，cmp 函数里面爱怎么写就怎么写——只要返回 `true` 表示"a 应该排 b 前面"就行。这个自由度非常大，是竞赛中最常用的技巧之一。

有一个黄金记忆口诀，请大家记住：**`return a 应该排在 b 前面`**。每次写 cmp 函数心里默念这一句。

## 🖥️ 板书/屏幕要点
- **降序**：`sort(a, a+n, greater<int>())`
- **其他类型**：`greater<double>()`、`greater<string>()`
- **cmp 函数原型**：`bool cmp(const T &a, const T &b)`
- **黄金口诀**：`return a 应该排在 b 前面;`
- **实例**：按绝对值升序 → `return abs(a) < abs(b);`

## 📌 小结
`greater<int>()` 是降序的快捷方式，但 cmp 才是真正释放 sort 威力的关键。记住口诀"return a 应该排在 b 前面"，下节课我们深入 cmp 的高级玩法——多条件判断和结构体排序。

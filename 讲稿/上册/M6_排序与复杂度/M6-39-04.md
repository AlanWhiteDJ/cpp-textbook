# 第39章 第4课时：三种排序对比与稳定性

**时长**：15-20分钟 | **对应教材**：第39章

## 🎯 本课时目标
- 对比冒泡、选择、插入三种排序的核心操作和性能
- 理解稳定性的概念，能判断每种排序是否稳定
- 学会根据数据特征选择合适的 O(n²) 排序

## 📝 讲稿正文

前面三节课我们学了三种 O(n²) 排序：冒泡、选择和插入。今天我们来一次大对比，把它们的"性格"弄清楚，再学习一个非常重要的新概念——**稳定性**。

先看这张对比表：

|  | 冒泡 | 选择 | 插入 |
|------|------|------|------|
| 核心操作 | 相邻交换 | 选最小→放前面 | 后移→插入 |
| 最好时间 | O(n) | O(n²) | O(n) |
| 最坏时间 | O(n²) | O(n²) | O(n²) |
| 交换次数 | 最多 n²/2 | 最多 n-1 | 最多 n²/2 |
| 稳定性 | ✅ 稳定 | ❌ 不稳定 | ✅ 稳定 |

大家注意看——三个都是 O(n²)，但性格完全不一样。冒泡和插入有"最好情况"的红利：数据本来就排好了，它们只需 O(n) 扫一遍就完事。选择排序没有这个好处——不管数据什么样，老老实实扫描完每一轮。

那选谁呢？我给大家一个实用指南：

**数据基本有序**？→ 选插入排序，它在这种场景下接近 O(n)，是三种里最强的。

**数据量很小（n ≤ 20）**？→ 三个都可以，插入通常最快。其实 STL 的 sort 在数据量小的时候内部也会切换成插入排序！

**想交换次数最少**？→ 选择排序，每轮最多一次交换，总共最多 n−1 次。

**初学理解**？→ 冒泡排序最直观，适合教学和面试。

好，现在来讲**稳定性**。什么是稳定性？看一个例子：

假设我有三个元素：`(3, A)`、`(1, B)`、`(3, C)`。括号里的第一个数是排序依据的值，第二个字母是"附加信息"。我要按数值从小到大排。

如果排序是稳定的，结果应该是 `(1, B), (3, A), (3, C)`——两个 3 的相对顺序没变，A 还是排在 C 前面。

如果不稳定，可能会变成 `(1, B), (3, C), (3, A)`——附加信息被打乱了。

稳定性什么时候重要？当你**分多次排序**的时候。比如你先把学生按姓名排好，再按分数排——如果第二次排序是不稳定的，同分的人姓名顺序可能就乱了。当然，如果一次 cmp 把所有规则都写进去，就不需要依赖稳定性。

冒泡排序为什么稳定？因为只有当 `a[j] > a[j+1]` 时才交换（不是 `>=`），相等的元素不会发生交换。

插入排序为什么稳定？挪动时条件是 `a[j] > key`，相等时不挪，相对顺序不变。

选择排序为什么不稳定？来看反例 `[5₁, 3, 5₂, 2]`。第一轮找最小 2，和 5₁ 交换→ `[2, 3, 5₂, 5₁]`。两个 5 的相对顺序反了！因为选择排序的交换可能跳过中间的元素，一下子把前面的元素扔到最后。

总结一下：三种 O(n²) 排序在你的"排序工具箱"里属于初级工具。大部分场景你不需要用它们——因为我们下一章就要学 `sort()`，一行代码搞定，还更快。但理解它们的原理仍然非常重要：它们是排序思想的起点，也是面试和考试中的常客。而且，当题目明确说"请用冒泡排序实现"时，你必须会写。

## 🖥️ 板书/屏幕要点
- **三种 O(n²) 排序对比**：核心操作、最好/最坏、交换次数、稳定性
- **选谁？** 基本有序→插入；交换最少→选择；最直观→冒泡
- **稳定性**：相等元素相对顺序不变 → 稳定
- **稳不稳定？** 冒泡✅ 选择❌ 插入✅
- **牢记反例**：`[5₁, 3, 5₂, 2]` → 选择排序让两个 5 顺序颠倒

## 📌 小结
今天把三种 O(n²) 排序放在一起比了比。冒泡最直观，选择交换最少，插入在基本有序时最快。稳定性这个概念务必修清晰——它决定了相等元素的命运。下节课我们做一次综合练习，把三种排序的手写能力彻底巩固。

## 📝 讲稿正文

上节课我们说暴力求逆序对 O(n²) 不够快。今天用归并排序来实现 O(n log n) 的高效解法。

为什么归并排序能求逆序对？关键在 **merge 合并阶段**。我们回顾一下 merge 的过程：左右两个子数组都已经各自有序了，我们用两个指针 i（左半）和 j（右半）合并。

当 `a[i] <= a[j]` 时，正常把 a[i] 放入 tmp。但当 `a[i] > a[j]` 时——注意，此时**左半中从 a[i] 到 a[mid] 的所有元素都大于 a[j]**（因为左半是升序的，a[i] 后面的更大）。这就意味着：a[j] 和左半中从 i 到 mid 的每一个元素都构成逆序对。一下子贡献了 `mid - i + 1` 个！

这个洞察是整个算法的精髓。我们来看具体代码：

```cpp
long long mergeCount(int l, int mid, int r) {
    long long inv = 0;                    // 局部逆序对计数
    int i = l, j = mid + 1, k = l;
    
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) {
            tmp[k++] = a[i++];            // 正常归并
        } else {
            tmp[k++] = a[j++];
            inv += (mid - i + 1);         // ✅ 核心！左半从 i 到 mid 都 > a[j]
        }
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r)   tmp[k++] = a[j++];
    for (int p = l; p <= r; p++)
        a[p] = tmp[p];
    
    return inv;
}
```

和普通 merge 几乎一样，只是加了 `long long inv` 和那一行 `inv += (mid - i + 1)`。

然后递归函数把各层的逆序对数加起来：

```cpp
long long countInversions(int l, int r) {
    if (l >= r) return 0;
    int mid = l + (r - l) / 2;
    long long inv = 0;
    inv += countInversions(l, mid);       // 左半内部的逆序对
    inv += countInversions(mid + 1, r);   // 右半内部的逆序对
    inv += mergeCount(l, mid, r);         // 跨越左右的逆序对
    return inv;
}
```

三层逆序对：左半内部的、右半内部的、跨越左右的。所有可能的逆序对都覆盖到了——因为任意一对元素 (a[x], a[y]) 要么同在左半，要么同在右半，要么分跨两边。

我们来走一个例子验证：`[3, 1, 4, 2]`。

递归过程：
```
拆分：         [3, 1, 4, 2]
              /            \
        [3, 1]           [4, 2]
        /    \           /    \
      [3]   [1]       [4]   [2]
       \    /          \    /
    merge: 3>1 → inv+=1  merge: 4>2 → inv+=1
    [1,3] inv=1          [2,4] inv=1
           \              /
          merge: i=0(1), j=0(2): 1<=2 ✓
                  i=1(3), j=0(2): 3>2 → inv += (1-1+1) = 1
                  j=1(4): 3<=4 ✓
          最终 inv = 1+1+1 = 3 ✓
```

完美！答案 3，和之前暴力计算的一致。

复杂度分析：递归深 log n 层，每层合并 O(n)，总 O(n log n)。空间还是 O(n)（tmp 数组）+ O(log n)（递归栈）。结果必须是 `long long`，这个再强调一遍。

现在你有了两种求逆序对的方法：暴力 O(n²) 用于 n≤10⁴ 时验证，归并 O(n log n) 用于 n≤10⁵ 甚至更大。这就是 M6 模块最有"含金量"的技能之一。

## 🖥️ 板书/屏幕要点
- **核心公式**：`inv += mid - i + 1;`
- **原理**：左半有序，a[i] > a[j] 意味着 a[i..mid] 都 > a[j]
- **三层来源**：左半内部 + 右半内部 + 跨越左右
- **代码改动**：普通 merge + 一行 inv 计数 + 返回 long long
- **复杂度**：O(n log n)，结果用 long long！

归并排序求逆序对是 M6 的"高光时刻"——在 merge 过程中一行 `inv += mid - i + 1` 就能把 O(n²) 变成 O(n log n)。记住公式，记住 long long，这招在竞赛中非常好用。下节课做综合实战来收尾整个模块。

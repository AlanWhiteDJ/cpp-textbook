# 第43章 第2课时：排序选择决策指南

**时长**：15-20分钟 | **对应教材**：第43章

## 🎯 本课时目标
- 掌握面对一道题时如何快速判断用哪种排序
- 记住竞赛黄金法则：sort 优先
- 能够根据数据规模和特征做出正确选择

## 📝 讲稿正文

上节课我们把八种排序放在一起对比了。今天来解决一个实际问题：**拿到一道题，我怎么知道该用哪种排序？**

我给大家一个三步决策法。

**第一步：看题目有没有指定算法。**

这是最简单的——题目说"请使用冒泡排序实现"，那你就必须手写冒泡，别偷懒用 sort。题目说"请输出排序结果，不限算法"，恭喜你，直接用 sort。题目说"请用归并排序求逆序对"，那就手写归并。

**第二步：看数据规模 n。**

这是最关键的判断依据。回想第 38 章给的经验表：
- n ≤ 1000 → O(n²) 可行
- n ≤ 100000 → 需要 O(n log n)
- n ≤ 10000000 → 必须 O(n)

所以，大于 10000 的数据，基本就别想 O(n²) 了。sort() 是最安全的选择。

**第三步：看数据特征。**

数据有什么特殊之处？值域小吗？所有数在 0~1000 之间？→ 可以考虑计数排序。数据基本有序吗？→ sort 就够了，但插入排序可能更合适（不过大多数情况 sort 也不会差）。全是 int 而且位数很少？→ 基数排序可能比 sort 快（但好处有限）。

我来举几个例题，大家跟着我一起判断。

**例题 1**：n=100000，值域 [0, 10⁹]，输出排序结果。

判断：n 很大，O(n²) 不行。值域 10⁹，计数排序开不出数组。所以直接 **sort()**。一行搞定。

**例题 2**：n=1000000，值域 [0, 1000]，输出排序结果。

判断：n 百万级，sort 的 O(n log n) 没问题。但值域只有 1000——计数排序只要 O(n + 1000) ≈ O(n)，更快！所以选**计数排序**。

**例题 3**：n=500，值域 [0, 10⁹]，要求稳定排序。

判断：n 很小，O(n²) 完全可行。要求稳定 → 冒泡或插入都可以。直接用 **stable_sort** 也行，更省心。

**例题 4**：n=100000，求逆序对数量。

判断：必须 O(n log n)。求逆序对 = 在排序过程中做额外工作 → **归并排序**。

好，现在我要宣布竞赛中的**黄金法则**，请大家记牢：

> **能用 sort() 就用 sort()。**
> **除非题目强迫你手写，否则不要自讨苦吃。**

为什么？因为 sort() 是全世界顶尖 C++ 工程师调出来的最优解——内省排序混合了快排、堆排和插入排序，最坏 O(n log n)，自动避免了快排的退化问题。你自己手写的快排，大概率没它好。

只有两种情况你需要用别的排序：
1. **题目指定了算法**（比如"请用冒泡排序"）
2. **sort 不够好**（需要稳定时用 stable_sort，需要求逆序对时用归并，值域极小且 n 极大时用计数）

其他一切情况——sort 即正义。

## 🖥️ 板书/屏幕要点
- **三步决策法**：看题目要求 → 看数据规模 → 看数据特征
- **经验对照**：n≤10³→O(n²) / n≤10⁵→O(n log n) / n≤10⁷→O(n)
- **竞赛黄金法则**：**sort() 优先！**
- **例外情况**：题目指定算法 / 需要稳定性 / 求逆序对 / 值域极小且 n 极大

## ✏️ 随堂练习

1. n=200，值域 [0, 1000000]，输出排序后的数组。
2. n=5000000，所有数据在 [0, 500] 之间，输出排序结果。
3. n=100000，要求求序列中逆序对的数量。
4. n=20，要求用尽可能少的比较次数排序。

## 📌 小结
面对排序题，三步决策法帮你快速锁定方案。第一步看题目要求，第二步看 n 规模，第三步看数据特征。绝大多数情况答案都是 sort。记住黄金法则，别在不必要的地方浪费时间。下节课我们深入学习逆序对。

## 📝 讲稿正文

前面三节课我们一直在聊时间——程序跑多快。但还有一件事同样重要：程序用了多少内存。这就是今天的第一部分：**空间复杂度**。

什么是空间复杂度？就是算法在运行过程中**额外占用的内存量**，同样用大 O 记号表示。

最简单的例子——只用了几个变量，不随数据量 n 增长：

```cpp
int sum = 0;
for (int i = 1; i <= n; i++) sum += i;
```

这段代码只用了 `sum` 和 `i` 两个变量，不管 n 是一百还是一百万，变量永远就这两个。所以我们说空间复杂度是 **O(1)**——常数空间。

但如果我开了一个数组，长度等于 n 呢？

```cpp
int a[n];
for (int i = 0; i < n; i++) a[i] = i;
```

这个数组的长度随 n 增长。n 是多大，数组就多大。空间复杂度是 **O(n)**。

那么问题来了：竞赛中内存有限制，一般是 256MB。你能开多大的数组？我们来算一算。一个 `int` 占 4 个字节。如果开 `int a[1000005]`——也就是一百万零五个 int——大约占 4MB，完全没问题。但如果开 `int a[100000005]`——一亿个 int——那就是 400MB，肯定超限了！

一个简单的估算公式：**数组占用的内存 ≈ 元素数量 × 每个元素的大小**。int 是 4 字节，long long 是 8 字节，double 是 8 字节，char 是 1 字节。开数组之前心里要有个数。

好，接下来是第二个话题：**最好、最坏和平均情况**。同一个算法，面对不同的输入，表现可能差很多。我们以一个最简单的"查找"为例：

```cpp
for (int i = 0; i < n; i++) {
    if (a[i] == target) break;
}
```

线性查找，从头找到尾。如果运气好，`target` 就在第一个位置——只比较了 1 次就找到了，这是**最好情况**，O(1)。如果运气差，`target` 在最后一个位置，或者根本不在数组里——比较了 n 次，这是**最坏情况**，O(n)。平均来说呢，大约要检查 n/2 个元素，常数被忽略，所以**平均情况**还是 O(n)。

在竞赛和算法分析中，我们通常**关注最坏情况**。为什么？因为评测数据不会总让你碰到最好的情况，出题人往往会精心构造最坏的数据来考验你的算法。你不能赌运气，要保证即使最坏情况也能在时限内跑完。

但是有些算法，最好和最坏的差距特别大，需要特别注意。比如我们后面要学的快速排序——平均是 O(n log n)，但如果每次选的基准值恰好是最小或最大的，就会退化到 O(n²)。这就必须用随机化等手段来避免最坏情况。

最后，我给大家一个竞赛中非常有用的经验表。看到题目给的数据范围 n，你大概就能猜到该用什么复杂度的算法：

- n ≤ 30：甚至 O(2ⁿ) 都能过
- n ≤ 1000：O(n²) 可行
- n ≤ 100000：需要 O(n log n)
- n ≤ 10000000：必须 O(n)

这个表不是绝对的，但在考场上能帮你快速排除不靠谱的方案。

## 🖥️ 板书/屏幕要点
- **空间复杂度**：额外占用的内存量
- **int 数组估算**：~1M 个 int ≈ 4MB，~1亿个 ≈ 400MB（超限！）
- **最好/最坏/平均**：通常关注最坏情况
- **竞赛经验**：n 的大小 → 可用复杂度等级

| n 规模 | 可用复杂度 |
|--------|-----------|
| n ≤ 30 | O(2ⁿ) |
| n ≤ 1000 | O(n²) |
| n ≤ 10⁵ | O(n log n) |
| n ≤ 10⁷ | O(n) |

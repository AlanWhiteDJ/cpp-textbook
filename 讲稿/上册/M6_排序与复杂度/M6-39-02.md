# 第39章 第2课时：选择排序——每次都挑出最小的

**时长**：15-20分钟 | **对应教材**：第39章

## 🎯 本课时目标
- 理解选择排序的核心思想：每轮选出最小值放到前面
- 能够手写选择排序的完整代码
- 知道选择排序的交换次数是 O(n²) 排序中最少的

## 📝 讲稿正文

上节课我们学了冒泡排序——相邻比较，大的往后冒。今天来学第二种：**选择排序**。

选择排序的思路更直接：每一轮，我在还没排好的部分里找到最小的那个数，把它放到最前面。一轮一轮下去，前面越来越有序，后面越来越短。

还是用 `[5, 3, 8, 1]` 来演示。

第一轮：我在位置 0 到 3 之间找最小的数。1 最小，在位置 3。把它和位置 0 的 5 交换→ `[1, 3, 8, 5]`。现在位置 0 是已经排好的了。

第二轮：我在位置 1 到 3 之间找最小的。3 最小，它已经在位置 1 了，不用动→ `[1, 3, 8, 5]`。

第三轮：在位置 2 到 3 之间找最小。5 比 8 小，在位置 3。和位置 2 的 8 交换→ `[1, 3, 5, 8]`。排好了！

大家注意，选择排序每轮最多只交换一次——找到最小值之后执行一次 swap。而冒泡排序可能在每一轮里交换很多次。所以选择排序的**交换次数是最少的**，这是它的独特优势。

来看代码：

```cpp
void selectionSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;                      // 假设当前位置最小
        for (int j = i + 1; j < n; j++) {    // 在后面找真正的最小
            if (a[j] < a[minIdx])
                minIdx = j;
        }
        swap(a[i], a[minIdx]);               // 把最小值放到前面
    }
}
```

这里要注意一个细节——`minIdx` 初始化为 `i`，然后从 `i+1` 开始往后扫描。如果后面有更小的，就更新 `minIdx`。扫描完以后，如果 `minIdx` 就是 `i`（说明它本来就是最小的），`swap(a[i], a[i])` 自己换自己，不影响结果。

选择排序的时间复杂度有什么特点呢？不管数组是正序还是倒序还是乱序——**每一轮都要完整扫描剩余部分找最小值**。所以最好、最坏、平均，全都是 O(n²)。没有冒泡那个"早期退出"的福气。

那它的空间复杂度呢？只用了 `i`、`j`、`minIdx` 几个变量，O(1)。

但是，有一个重要的提醒：选择排序是**不稳定**的！这点很多同学会搞错。你可能会想："它就是找到最小的放前面啊，怎么会打乱顺序？"

来看这个反例：`[5₁, 3, 5₂, 2]`。第一轮找最小，2 最小，和 5₁ 交换→ `[2, 3, 5₂, 5₁]`。发现没有？原本 5₁ 在 5₂ 前面，现在 5₂ 跑到 5₁ 前面去了！两个相等的 5 的相对顺序被改变了。这就是"不稳定"的含义，后面我们会更仔细地讲。

选择排序适合什么场景呢？当你对"交换次数"敏感的时候——比如每次交换的代价很高，但比较的代价很低。不过大多数情况下，它和冒泡一样，只适合小数据（n ≤ 10000 左右）。

## 🖥️ 板书/屏幕要点
- **选择排序**：每轮选未排序部分的最小值，放到已排序部分末尾
- **核心变量**：`minIdx` 记录最小值位置
- **复杂度**：无论什么情况都是 O(n²)，空间 O(1)
- **交换次数**：最多 n−1 次，O(n²) 排序中最少
- **稳定性**：❌ 不稳定

## ✏️ 随堂练习

2. 写代码实现选择排序，并在每轮打印"第 x 轮：最小值 = y，交换位置 i 和 minIdx"。

## 📌 小结
选择排序的思想非常直观：每轮挑最小，放到最前面。它的交换次数最少，算是 O(n²) 排序里在这方面表现最好的。但记住它是不稳定的，而且没有早期退出的优化空间——无论什么数据，每轮都要完整扫描。下节课我们学第三种基础排序：插入排序，它的思路更像打扑克牌。

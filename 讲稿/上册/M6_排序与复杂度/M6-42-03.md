# 第42章 第3课时：基数排序——按位排序的智慧

**时长**：15-20分钟 | **对应教材**：第42章

## 🎯 本课时目标
- 理解基数排序的核心思想：按位排序，低位到高位，每轮用稳定排序
- 理解为什么稳定是基数排序的关键
- 能够手动模拟基数排序的过程

## 📝 讲稿正文

前面两节课我们学了计数排序——O(n+MAXV)，但MAXV太大就不行。那怎么办呢？基数排序给出了一个巧妙的答案：**把大数拆成"位"来排**。

什么叫按位排？你想想，如果要比较两个三位数，比如 345 和 278，你其实不用直接比 345 和 278 谁大——你只需要看百位：3 > 2，所以 345 大。如果百位一样，再看十位；十位一样，再看个位。

这给了我们一个思路：按照**最低位到最高位**的顺序，逐位排序。每一轮只关心当前位的数字（0~9），用稳定的计数排序处理。

来看一个例子：`[170, 45, 75, 90, 802, 24, 2, 66]`。

第一轮，按**个位**排序：
```
170→个位0  45→个位5  75→个位5  90→个位0
802→个位2  24→个位4   2→个位2  66→个位6

按个位排完后：170, 90, 802, 2, 24, 45, 75, 66
（注意：170和90个位都是0，170在原数组中在前，排完后还在90前面——稳定！）
```

第二轮，在保持个位有序的基础上，按**十位**排序：
```
170→十位7  90→十位9  802→十位0   2→十位0(即 002)
24→十位2   45→十位4   75→十位7  66→十位6

按十位排完后：802, 2, 24, 45, 66, 170, 75, 90
```

注意这里！802 和 2 十位都是 0，但之前个位排序时 802 的个位 2，2 的个位 2——呃，不对，让我重新理一下。第一轮结束顺序是 170(0), 90(0), 802(2), 2(2), 24(4), 45(5), 75(5), 66(6)。第二轮排十位时，802(十位0)和2(十位0)的相对顺序由第一轮的稳定性保证——802 排在 2 前面，因为第一轮后 802 就在 2 前面。

继续第三轮，按**百位**排序（前面不足三位的补 0）：
```
802→百位8  2→百位0  24→百位0  45→百位0
66→百位0  170→百位1  75→百位0  90→百位0

按百位排完后：2, 24, 45, 66, 75, 90, 170, 802
```

完美有序！三轮下来就排好了。注意每一轮用的都是稳定的计数排序——这是关键。如果某一轮用了不稳定的排序，会把之前低位排好的顺序打乱，最终结果可能全错。

现在我解释为什么一定要**从低位到高位**排。因为高位排好后，如果再从高往低排，高位的顺序会被低位排序破坏。从低往高，最后一轮是最高位，低位的顺序作为"第二关键字"被保留下来。

基数排序的复杂度是 O(d × n)，d 是最大数的位数。int 类型最多约 10 位（2¹⁰ ≈ 10³，所以 10 位十进制数约 10⁹），d ≤ 10。所以对于 int，基数排序几乎是 **O(n)**！

但要注意——每一轮虽然只处理 10 个桶，但需要 O(n) 的扫描和 O(n) 的 output 数组，所以常数比 sort 大一些。对于随机 int 数据，基数排序不一定比 sort 快；但当数据量很大且值的位数不多时，基数排序可能更优。

## 🖥️ 板书/屏幕要点
- **基数排序思想**：从低位到高位，逐位排序，每轮用稳定计数排序
- **稳定性是核心**：低位排好的顺序在后续轮次中保留
- **桶数**：每位 0~9，固定 10 个桶
- **复杂度**：O(d × n)，d = 位数，int 时 d ≤ 10
- **与计数排序关系**：计数排序是"单轮"，基数排序是"多轮计数"

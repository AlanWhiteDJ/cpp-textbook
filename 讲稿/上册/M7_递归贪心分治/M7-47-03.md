# 第47章 第3课时：跳石头——最小值最大化

## 📝 讲稿正文

今天我们来攻克一道二分答案的经典题——**跳石头**。这道题在竞赛中反复出现，因为它是"二分 + 贪心"组合的典范。

问题描述：一条直线上有 n 块石头，距离起点分别为 d₁, d₂, ..., dₙ。总长度是 L（终点也在 L 处）。你要去掉恰好 m 块石头，使得**相邻保留石头之间的最小距离尽可能大**。求这个最大化的最小距离。

读题要注意那个关键词："最小距离尽可能大"——"最小值最大化"，典型的二分答案信号。

分析一下：我们二分的是"最小距离"mid。对于给定的 mid，check(mid) 要回答：能否通过去掉不超过 m 块石头，使得任意相邻保留石头的间距都 ≥ mid？

怎么判断？贪心！从起点出发，依次检查每块石头。如果当前石头和上一块保留石头的距离 < mid，这块石头必须搬走（因为它破坏了最小间距要求）。如果 ≥ mid，就保留它。

```cpp
int d[50005];  // 石头位置
int L, n, m;

bool check(int mid) {
    int removed = 0;
    int last = 0;  // 上一块保留石头的位置（起点为0）
    for (int i = 1; i <= n; i++) {
        if (d[i] - last < mid) {
            removed++;           // 距离不够，搬走这块
        } else {
            last = d[i];         // 距离够，保留它
        }
    }
    // 别忘了检查终点
    if (L - last < mid) removed++;
    return removed <= m;         // 搬走的石头数不超过 m
}
```

这个贪心为什么对？因为每块石头的决策只取决于"上一块保留石头"——如果当前石头导致间距不够，不搬走它，后面的石头间距只会更难满足。所以"遇到不够就搬"是局部最优，且不会影响后面做出更优决策。

check 写好之后，二分求最大值：

```cpp
int l = 1, r = L, ans = 0;
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (check(mid)) {
        ans = mid;
        l = mid + 1;      // 可行，试试更大间距
    } else {
        r = mid - 1;
    }
}
cout << ans;
```

我们来数值模拟一下：L=25, n=5, m=2，石头位置 {2, 11, 14, 17, 21}。

假设 mid=4。从起点 0 出发：石头 2，间距 2 < 4，搬！last=0。石头 11，间距 11 ≥ 4，保留，last=11。石头 14，间距 3 < 4，搬！last=11。石头 17，间距 6 ≥ 4，保留，last=17。石头 21，间距 4 ≥ 4，保留，last=21。终点 25，间距 4 ≥ 4，保留。搬了 2 块，正好等于 m，可行！

mid=5 呢？搬走 2，11，14——搬了 3 块，超了。不行。

所以答案在 4 和 5 之间——最大是 4。

这道题的精华是"二分 + 贪心"的配合：二分负责在答案空间里搜索，贪心负责判断一个候选答案是否可行。这两个算法结合，能解决一大类问题——"在满足约束的条件下优化某个指标"。

## 💬 互动提问
1. "跳石头中，为什么贪心策略是'间距不够就搬'？有没有可能不搬当前石头反而更好？"
2. "如果题目改成'保留恰好 m 块石头'而不是'去掉 m 块'，代码怎么改？"
3. "check 函数中，终点也要检查间距，能不能把终点也当成一块石头来统一处理？"

## 🖥️ 板书/屏幕要点
- "最小值最大化" → 二分答案 + 贪心 check
- 跳石头 check：贪心扫描 → 间距不够就搬 → 统计搬走数量 → 比较 ≤ m
- 框架：二分 mid（最小间距） + 贪心验证

## 📌 小结
"最小值最大化"是二分答案最经典的题型。跳石头展示了"二分 + 贪心"的标准配合——二分负责缩小范围，贪心负责判定可行性。这个模式在木材加工、数列分段等问题中还会反复出现。下节课我们学木材加工和浮点二分。

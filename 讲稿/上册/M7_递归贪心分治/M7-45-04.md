## 📝 讲稿正文

今天我们做几道综合练习，把记忆化搜索的功夫练到家。

第一题：**爬楼梯**。每次可以爬 1 级或 2 级台阶，爬到第 n 级有多少种不同的方法？

这和斐波那契一模一样！爬到第 n 级的方案数 = 爬到第 n-1 级的方案数（再爬 1 级）+ 爬到第 n-2 级的方案数（再爬 2 级）。边界：n=1 有 1 种，n=2 有 2 种。

```cpp
long long memo[55];
memset(memo, -1, sizeof(memo));

long long climb(int n) {
    if (n == 1) return 1;
    if (n == 2) return 2;
    if (memo[n] != -1) return memo[n];
    return memo[n] = climb(n-1) + climb(n-2);
}
```

你看，和斐波那契唯一的区别就是边界——斐波那契的 f(1)=f(2)=1，爬楼梯的 f(1)=1、f(2)=2。很多问题换了个"故事"但本质是相通的，要学会识别。

第二题：**最小路径和**。一个 m×n 的网格，每个格子有一个非负整数。从左上走到右下，求路径经过的数字之和的最小值。

和网格路径计数一样的状态定义——`dfs(x,y)` 表示从 (x,y) 到终点的最小路径和。区别在于，这次不是"加法计数"而是"取最小值"：

```cpp
int dfs(int x, int y) {
    if (x == m && y == n) return a[x][y];
    if (x > m || y > n) return  INF;       // 越界返回无穷大
    if (memo[x][y] != -1) return memo[x][y];
    return memo[x][y] = a[x][y] + min(dfs(x+1,y), dfs(x,y+1));
}
```

注意越界返回了 INF（一个很大的数），这样在 min 比较时越界的路径自然被淘汰。这个技巧在很多记忆化题目里都会用到。

第三题，我们简单提一下**背包问题的记忆化版**。有 N 件物品，每件有重量 w[i] 和价值 v[i]，背包容量 C，问最多装多少价值？

状态定义：`dfs(i, cap)` 表示"考虑前 i 件物品、背包还剩 cap 容量"时的最大价值。对于第 i 件物品，要么不选（容量不变，考虑下一件），要么选（容量减少，价值增加，考虑下一件）：

```cpp
int dfs(int i, int cap) {
    if (i > n) return 0;                       // 没物品了
    if (memo[i][cap] != -1) return memo[i][cap];
    int res = dfs(i+1, cap);                    // 不选第 i 件
    if (cap >= w[i])
        res = max(res, dfs(i+1, cap-w[i]) + v[i]); // 选第 i 件
    return memo[i][cap] = res;
}
```

背包问题以后会专门讲 DP 解法，这里先让大家感受一下记忆化建模的方式。

好，第 45 章到这里全部结束。我们来做个总结。

记忆化搜索 = 递归 + 备忘录。核心就一句话：算过的存起来，下次直接用。它和动态规划本质相同——都是空间换时间。区别在于记忆化是"从大到小递归"，DP 是"从小到大循环"。记忆化更适合初学者，因为它贴近人脑"遇到难题拆成小问题"的思维习惯。DP 则在栈空间和效率上更有优势。

你们现在的学习路径应该是：先用记忆化写 → 再尝试改成 DP → 最终能直接用 DP 思考。这是一个循序渐进的过程。

## 📌 小结
记忆化搜索是一座桥——从递归走向动态规划。通过爬楼梯、最小路径和、背包等例子，大家应该能感受到：只要定义好状态、找到递推关系、加上 memo 数组，记忆化就能解决大量问题。下章我们转向二分查找——一个不依赖递归却同样强大的算法思想。

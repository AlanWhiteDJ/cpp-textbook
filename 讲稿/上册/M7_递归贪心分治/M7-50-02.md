## 📝 讲稿正文

上节课我们学了快速幂的原理。这节课把代码细节讲透，再看看它的变种。

先说几个坑。第一个坑：result 初始化为 0。有人写 `int result = 0;`，然后 `result = result * a`——0 乘任何数还是 0，答案永远出不来。必须初始化为 1，因为 1 是乘法的单位元。

第二个坑：忘了对 a 取模。a 本身可能很大（比如 a=10^9），直接 `a = a * a` 可能溢出 long long。标准的防护写法是在循环一开始就 `a %= p`：

```cpp
long long qpow(long long a, long long b, long long p) {
    long long res = 1;
    a %= p;          // ⚠️ 这一行不能少
    while (b) {
        if (b & 1) res = res * a % p;
        a = a * a % p;
        b >>= 1;
    }
    return res;
}
```

第三个坑：取模顺序。`res * a % p` 等价于 `((res % p) * (a % p)) % p`。因为每次循环都取模了，res 和 a 都已经小于 p，所以 res * a 最多是 (p-1)*(p-1)，在 `long long` 范围内是安全的（只要 p ≤ 10^9）。但如果 p 可以到 10^18 量级，就需要用 `__int128` 或"龟速乘"来处理溢出。

快速幂还有另一种写法——**递归版**，用分治思想：

```cpp
long long qpow(long long a, long long b, long long p) {
    if (b == 0) return 1;
    long long half = qpow(a, b / 2, p);   // 算一半
    half = half * half % p;                // 平方
    if (b % 2 == 1) half = half * a % p;   // 奇数多乘一个
    return half;
}
```

递归版的思想：a^b = (a^(b/2))²（b 偶）或 a × (a^(b/2))²（b 奇）。边界 b=0 时返回 1。这也是 O(log b)，但递归有函数调用开销，实际比赛中 while 循环版更常用。

现在我们聊一个快速幂的"升级版"——**矩阵快速幂**。

为什么需要矩阵快速幂？因为有些递推关系可以写成矩阵乘法，然后用快速幂一次性算出很远的结果。

比如斐波那契数列：F(n) = F(n-1) + F(n-2)。可以写成矩阵形式：

```
[ F(n+1) ]   [ 1 1 ] [ F(n)   ]
[ F(n)   ] = [ 1 0 ] [ F(n-1) ]
```

令 M = [[1,1],[1,0]]，那么：

```
[ F(n)   ]   [ 1 1 ]^(n-1) [ F(1) ]
[ F(n-1) ] = [ 1 0 ]        [ F(0) ]
```

算 M^(n-1) 用快速幂！只是这次"乘法"不是数字相乘，而是矩阵相乘。矩阵乘法是 O(2³)=O(1) 的（对于 2×2 矩阵），所以总复杂度还是 O(log n)。

有了矩阵快速幂，你可以秒求 F(10^18) mod p——纯递归和记忆化都做不到，因为数组开不到 10^18。

矩阵快速幂的代码结构和你写的快速幂一模一样，只是把 `res * a` 换成矩阵乘法的函数调用。这种"代数结构可替换"的思想，是快速幂最美的地方。

## 🖥️ 板书/屏幕要点
- 常见错误：result 初始化为 0、忘了对 a 取模、取模溢出
- 递归版快速幂：a^b = (a^(b/2))² × (b 奇时多乘 a)
- 矩阵快速幂：斐波那契的第 n 项 = M^(n-1) 的某个元素
- 核心一致：二进制拆分 + 翻倍平方，只是"乘"的对象不同

## 📌 小结
快速幂的细节决定成败——result 初始化为 1、a 先取模、每步取模防溢出。递归版和循环版本质相同。矩阵快速幂把"数字乘法"替换为"矩阵乘法"，但骨架不变。下节课我们学倍增查找和 LCA 预告。

# 第45章 第3课时：数字三角形与记忆化→DP的桥梁

- 初步建立 DP 的直觉

## 📝 讲稿正文

同学们，今天这节课非常重要——它不仅是记忆化的进阶应用，更是你们从"递归思维"过渡到"动态规划思维"的关键桥梁。

我们先看数字三角形问题。给你一个像金字塔一样的数字三角形：

```
     7
    3 8
   8 1 0
  2 7 4 4
```

从顶部出发，每次只能向左下或右下走一格，一直走到最底层。要求经过的数字之和最大。比如 7→3→8→2 = 20，7→8→1→7 = 23——你要找出所有路径中最大的那个和。

用记忆化怎么做？定义 `dfs(i, j)` 表示从第 i 行第 j 列出发，到达底部的最大路径和。

如果 i 是最后一行，那只有这个数字本身，直接返回。否则，你可以向左下走到 (i+1, j)，或向右下走到 (i+1, j+1)——取两者的最大值，加上当前数字。

```cpp
int a[105][105], memo[105][105];
int n;

int dfs(int i, int j) {
    if (i == n) return a[i][j];
    if (memo[i][j] != -1) return memo[i][j];
    return memo[i][j] = a[i][j] + max(dfs(i+1, j), dfs(i+1, j+1));
}
```

这个代码非常自然——从大问题出发，一路走到小问题。这就是**自顶向下**的思路。你想想，人脑也是这样想的：站在第 1 行，我往左走还是往右走？不知道，都要试试，看哪边最终能拿到更大的和。

但现在我想换一个角度。既然我已经知道最后一行的答案（就是它自己），我能不能从最底层出发，一行一行往上推？

能！这就是**自底向上**的动态规划：

```cpp
int dp[105][105];
for (int j = 1; j <= n; j++) dp[n][j] = a[n][j];  // 最后一行已知
for (int i = n-1; i >= 1; i--)
    for (int j = 1; j <= i; j++)
        dp[i][j] = a[i][j] + max(dp[i+1][j], dp[i+1][j+1]);
cout << dp[1][1] << endl;
```

我们来对比一下。记忆化是递归的，从 (1,1) 开始问"我往下走最大值是多少"，一路问下去，边走边记。DP 是循环的，从最底层开始，先算小问题，然后逐层往上填表。

两者本质完全相同——都是"用空间换时间，避免重复计算"。区别只在于方向：记忆化自顶向下（递归），DP 自底向上（循环）。

对于初学者来说，记忆化往往更直观。因为递归接近人脑的思考方式——"我不知道这个答案，但我可以把它拆成更小的问题"。而 DP 需要你先想清楚"填表的顺序"，这对思维的要求更高。

但 DP 也有优势：它没有递归调用的栈溢出风险。当问题规模特别大、递归深度特别深时，DP 更安全。

你们现在可以这样学：拿到一个新问题，先用记忆化写——思路最直接；写出来之后，再试着改成 DP——把递归调用的方向反一下，变成循环填表。这个转换练多了，DP 就自然而然掌握了。

## 🖥️ 板书/屏幕要点
- 数字三角形记忆化：dfs(i,j) = a[i][j] + max(dfs(i+1,j), dfs(i+1,j+1))
- 自顶向下（记忆化）vs 自底向上（DP）
- 共同本质：空间换时间
- DP 优势：无递归栈溢出风险

## 📌 小结
数字三角形是记忆化→DP 的经典桥梁。记忆化天然、直观；DP 安全、高效。两者本质相同，方向相反。多练习从记忆化到 DP 的转换，DP 就不再神秘了。下节课是本章最后一节，我们做综合练习。

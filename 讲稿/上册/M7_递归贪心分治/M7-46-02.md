# 第46章 第2课时：左闭右开与二分边界三bug

**时长**：15-20分钟 | **对应教材**：第46章 46.3

## 🎯 本课时目标
- 理解左闭右闭和左闭右开两种区间的区别
- 掌握左闭右开的二分写法
- 学会避免二分最常见的三个 bug

## 📝 讲稿正文

上节课我们学了左闭右闭的二分写法。但你知道吗，二分有"两大流派"——左闭右闭和左闭右开。C++ 标准库用的就是左闭右开。今天我们把两种都搞清楚。

先回想左闭右闭：搜索区间是 [l, r]，初始化 `l=0, r=n-1`，循环条件 `while(l <= r)`，更新 `l = mid + 1`，`r = mid - 1`。

左闭右开的搜索区间是 [l, r)——注意，r 是开区间，不包含 r。初始化时 r 等于 n（不是 n-1），因为 a[n] 是不存在的，刚好对应"不包含"。

循环条件变成 `while (l < r)`——当 l 等于 r 时区间为空（因为 r 不包含在内）。

最关键的区别在 r 的更新：因为 r 是开区间，排除 mid 时应该 `r = mid`（mid 不包含在新区间里）。而 l 的更新不变，还是 `l = mid + 1`。

```cpp
int binarySearch(int a[], int n, int x) {
    int l = 0, r = n;              // [l, r)，r = n
    while (l < r) {                // 区间非空
        int mid = l + (r - l) / 2;
        if (a[mid] == x)
            return mid;
        else if (a[mid] < x)
            l = mid + 1;
        else
            r = mid;               // 右开，所以 r = mid
    }
    return -1;
}
```

这两种写法哪个好？没有绝对的好坏。左闭右闭更直观（因为大部分人习惯了闭区间），左闭右开在 STL 和一些场景中更自然。我的建议是：**初学者固定用一种，推荐左闭右闭**。等你熟练了，两种都能轻松切换。

现在我们来说二分最容易犯的三个 bug。

**Bug 1：循环条件写错。** 左闭右闭用 `while(l < r)`——区间里只剩一个元素时，l 等于 r，循环直接跳过，那个元素永远不会被检查。这种 bug 特别隐蔽，因为大多数情况程序看起来运行正常，只有在目标恰好是区间的最后一个元素时才出问题。

**Bug 2：mid 溢出。** 直接写 `(l + r) / 2`，当 l 和 r 都接近 21 亿时，l+r 超过 42 亿，int 装不下就溢出了。防御性写法 `l + (r - l) / 2` 永远不会溢出。养成这个习惯，一辈子受益。

**Bug 3：l/r 更新写错导致死循环。** 有些人写 `l = mid` 或 `r = mid`（在左闭右闭时）——如果 l=2, r=3, mid=2，然后 l=mid=2，区间没缩小，死循环！正确的更新必须排除 mid：`l = mid + 1`，`r = mid - 1`。

这三个 bug 非常经典，据说 90% 的程序员都曾在二分上栽过跟头。甚至有人说："二分查找是思想最简单的算法，但在面试中能一次写对的不到 10%。"所以大家一定要重视这些细节。

## 🖥️ 板书/屏幕要点
- 左闭右闭 vs 左闭右开：初始化、循环条件、r 的更新不同
- STL 使用左闭右开
- Bug 三件套：① 循环条件 ② mid 溢出 ③ l/r 更新死循环

## 📌 小结
二分两派各有优劣，初学者固定一种即可。三个经典 bug 要刻在脑子里——循环条件、mid 溢出、死循环。把这些基础打牢，后面的 lower_bound 和二分答案才不会翻车。下节课我们学 lower_bound 和 upper_bound。

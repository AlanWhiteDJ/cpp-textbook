## 📝 讲稿正文

同学们，前面我们学了在"有序数组"里二分查找"具体的值"。但二分的威力远不止于此——如果把"值"扩展成"问题的答案"，二分就能用来"猜答案"。

什么叫猜答案？给你们讲个生活中的场景。你在洗澡，水太冷了，往热水那边拧一点；太烫了，往冷水那边拧一点。拧来拧去，终于调到一个刚刚好的温度。你其实就在做二分——在一个连续的范围内，不断调整，找一个"刚刚好"的平衡点。

编程里也一样。很多问题不需要你直接算出答案，而是让你在"答案的可能范围"里二分搜索——每次猜一个答案，验证一下行不行，然后缩小范围。这就是**二分答案**。

二分答案有一个硬性条件：**答案必须具有单调性**。什么意思？如果某个值可行，那么比它"更好"（更松或更紧）的一侧的值也全部可行或者全部不可行。这样才有"砍一半"的基础。

来看一个经典问题——**砍树**。一排 n 棵树，高度分别是 h₁ 到 hₙ。你有一个电锯，可以设定锯片高度 H。对于每棵树，高于 H 的部分都会被砍下来。你需要获得至少 M 米的总木材，同时希望 H 尽可能大（H 越大砍得越少，越环保）。求最大的 H。

H 的取值范围是 0 到最高的树的高度。如果 H=0，整棵树全砍，肯定够；如果 H=最高树高，什么都砍不到，肯定不够。而 H 越小，砍下的木材越多——这就是**单调性**。

所以我们二分 H！check(H) 判断：如果锯片设为 H 米，砍下的总木材是否 ≥ M？

```cpp
int n;
long long m;       // 注意：M 可能很大
int a[1000005];

bool check(int H) {
    long long sum = 0;
    for (int i = 0; i < n; i++) {
        if (a[i] > H) sum += a[i] - H;   // 高于H的部分砍下来
    }
    return sum >= m;
}
```

check 写好之后，二分框架和上一章一样。我们要求"最大的 H"，所以可行时往大里试：

```cpp
int l = 0, r = maxH, ans = 0;
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (check(mid)) {
        ans = mid;        // 可行，记录下来
        l = mid + 1;      // 试试能不能更高
    } else {
        r = mid - 1;      // 不行，降低要求
    }
}
cout << ans;
```

注意第 2 行：`long long m`——累加的总木材可能非常大（n 可以到 10^5，树高可以到 10^9），用 int 会溢出。二分答案里 check 函数经常要处理大数，务必注意类型。

砍树问题是二分答案的"Hello World"。它的精髓在于：**把"求最大值"变成"猜值+验证"**。你不需要直接解出 H，只需要会"判断一个给定的 H 行不行"——这比直接求解简单得多。

二分答案在竞赛中极其高频，特别是看到"最大值最小"或"最小值最大"这种表述时，百分之九十九就是它。

## 🖥️ 板书/屏幕要点
- 二分答案 = 猜答案 + 验证可行性
- 前提：答案具有单调性
- 砍树：check(H) = Σ max(0, aᵢ - H) ≥ M
- 求最大值：可行 → l = mid+1；不可行 → r = mid-1

## ✏️ 随堂练习
1. 实现砍树问题。测试：n=4, m=7, 高度 {20, 15, 10, 17}，H 应该是 15
2. 手工算一下 H=14,15,16,17 时各砍下多少，验证程序的二分过程

## 📌 小结
二分答案把"求解"变成"验证"，check 函数是灵魂。单调性是前提——没有单调性，二分答案就失效了。下节课我们系统学习 check 函数的设计和二分方向的选择。

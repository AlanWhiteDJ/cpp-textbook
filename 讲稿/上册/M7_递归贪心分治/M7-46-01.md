## 📝 讲稿正文

同学们，从今天开始我们进入二分查找。二分查找的思想非常朴素——每次把问题砍掉一半。

你们肯定玩过"猜数字"游戏：我心里想一个 1 到 1000 之间的数，你每次猜一个数，我告诉你"大了"还是"小了"或者"猜中了"。你怎么用最少的次数猜中？

笨办法是从 1 开始一个一个猜——最坏情况要猜 1000 次。聪明办法是：第一次猜 500。如果我说"大了"，答案就在 1 到 499；如果"小了"，答案就在 501 到 1000。不管哪种情况，你都一刀砍掉了一半！

然后你在剩下的范围里继续对半猜。500 → 250 → 125 → 63 → 32 → 16 → 8 → 4 → 2 → 1。最多 10 次就能猜中。为什么？因为 2^10 = 1024 > 1000。

这就是二分查找的威力——在 10 亿个数据里找一个值，也只需要约 30 次比较。复杂度 O(log n)。

二分查找有一个硬性前提：**数据必须有序**。为什么？因为二分依赖"根据中间值判断目标在哪一边"。数据是乱的，"中间值比目标大"说明不了任何问题——目标可能在左边也可能在右边。

好，我们来看标准代码。在升序数组 a[0..n-1] 中找值 x：

```cpp
int binarySearch(int a[], int n, int x) {
    int l = 0, r = n - 1;          // 搜索区间 [l, r]，左闭右闭
    while (l <= r) {               // 区间非空时继续
        int mid = l + (r - l) / 2; // 中间位置
        if (a[mid] == x)
            return mid;            // 找到了！
        else if (a[mid] < x)
            l = mid + 1;           // x 在右半部分
        else
            r = mid - 1;           // x 在左半部分
    }
    return -1;                     // 没找到
}
```

有三点要强调。第一，循环条件是 `l <= r` 而不是 `l < r`。为什么？因为区间是闭的，当 l 等于 r 时区间内还有一个元素需要检查。如果你写成 `l < r`，剩一个元素时直接跳过了。

第二，mid 的写法是 `l + (r - l) / 2`，而不是 `(l + r) / 2`。两者数学上等价，但后者在 l 和 r 都很大时可能溢出。这种防御性编程的习惯要从一开始就养成。

第三，更新 l 和 r 时，分别是 `mid + 1` 和 `mid - 1`，而不是 `mid`。因为 a[mid] 已经检查过了，不是我们要找的，所以区间要排除 mid。如果你写成 `l = mid` 或 `r = mid`，可能导致死循环。

我们跟踪一个例子：数组 {2, 5, 8, 12, 16}，找 10。第一轮 l=0, r=4, mid=2, a[2]=8 < 10，所以 l=3。第二轮 l=3, r=4, mid=3, a[3]=12 > 10，所以 r=2。此时 l=3 > r=2，循环退出，返回 -1。正确！

## 🖥️ 板书/屏幕要点
- 二分查找 = 每次砍一半，O(log n)
- 前提：数据有序
- 标准框架（左闭右闭）：l=0, r=n-1, while(l<=r), mid=l+(r-l)/2
- 更新：l=mid+1, r=mid-1

## 📌 小结
二分查找是基础但至关重要的算法，O(log n) 的效率令人惊叹。关键是记住三个最容易出错的地方：循环条件、mid 溢出、l/r 更新。下节课我们介绍左闭右开的写法，以及两者之间的差异。

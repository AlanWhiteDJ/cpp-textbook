## 📝 讲稿正文

今天我们继续二分答案的实战，先看**木材加工**问题。

有 n 根原木，长度分别为 L₁ 到 Lₙ。要切成等长的 k 段小木料（每根原木可以切成多段，但不能拼接）。求小木料的最大长度。

又是"最大值"问题！而且有单调性：小木料长度越小，能切出的段数越多。二分小木料长度 mid，check(mid) 判断能不能切出至少 k 段：

```cpp
int L[100005];
int n, k;

bool check(int mid) {
    long long cnt = 0;
    for (int i = 0; i < n; i++) {
        cnt += L[i] / mid;      // 第 i 根原木能切出几段
    }
    return cnt >= k;
}
```

够简单吧？对于每根原木，能切出的段数就是 `L[i] / mid`（整除）。累加起来 ≥ k 就可行。

二分框架和砍树一样——求最大值，可行时往大试：

```cpp
int l = 1, r = maxL, ans = 0;
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (check(mid)) {
        ans = mid;
        l = mid + 1;
    } else {
        r = mid - 1;
    }
}
```

好了，现在我们进入新内容——**浮点二分**。

有时候答案不是整数。比如"电缆切割"问题中，原木长度可能是 12.34 米这样的浮点数，答案也需要精确到小数点后两位。这时候二分就不能用整数了。

浮点二分最大的陷阱在**终止条件**。很多人习惯写 `while (r - l > 1e-8)`——当区间宽度小于某个精度时停止。但这有风险：如果精度设得太小（比如 1e-15），浮点精度不够，可能永远达不到，导致死循环。

**最佳实践是固定迭代次数**：

```cpp
double l = 0, r = 1e9;
for (int i = 0; i < 100; i++) {    // 迭代 100 次
    double mid = (l + r) / 2;
    if (check(mid))
        l = mid;       // 浮点二分：不需要 ±1
    else
        r = mid;
}
cout << fixed << setprecision(6) << l;
```

迭代 100 次后，区间宽度缩小为原来的 1/2^100 ≈ 10^(-30)，精度远超需要。而且不会死循环。

注意浮点二分的一个细节：更新 l 和 r 时直接写 `l = mid`，不需要 ±1。因为浮点数是连续的，mid 本身就有无穷精度。

来做一道经典题：**不用 sqrt 函数，求 sqrt(2) 的近似值**。二分搜索区间 [0, 2]，check(mid) 判断 mid² ≤ 2：

```cpp
double l = 0, r = 2;
for (int i = 0; i < 60; i++) {
    double mid = (l + r) / 2;
    if (mid * mid <= 2)
        l = mid;
    else
        r = mid;
}
cout << fixed << setprecision(10) << l;  // 输出 1.4142135624
```

60 次迭代精度已经到 10^(-18) 量级了，输出 10 位小数绰绰有余。

浮点二分在"电缆切割"类问题中特别好用：把输入 ×100 转成整数（厘米），用整数二分处理，输出时 ÷100 转回来。这招可以避免浮点精度问题，推荐优先使用。

## 🖥️ 板书/屏幕要点
- 木材加工：check(mid) = Σ floor(Lᵢ / mid) ≥ k
- 浮点二分：固定迭代 60~100 次，`l = mid` / `r = mid`（不用 ±1）
- 技巧：浮点 → ×100 转整数 → 整数二分 → ÷100 输出（避免精度问题）

## 📌 小结
木材加工展示了二分答案在"切分问题"中的应用，check 往往只需一行。浮点二分的核心是固定迭代次数——安全、精确、不会死循环。下节课我们学数列分段和本章总结。

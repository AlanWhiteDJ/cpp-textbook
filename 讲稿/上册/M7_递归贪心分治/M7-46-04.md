## 📝 讲稿正文

上节课我们手写了 lower_bound 和 upper_bound，理解了它们的原理。但在日常写代码时，我们不需要每次都自己写——C++ STL 已经帮你写好了，都在 `<algorithm>` 头文件里。

三个最常用的 STL 二分函数：

**binary_search**：判断一个值是否在有序序列中。返回 bool。

```cpp
#include <algorithm>
#include <vector>
vector<int> a = {1, 2, 2, 2, 3, 5};
bool found = binary_search(a.begin(), a.end(), 3);   // true
bool found2 = binary_search(a.begin(), a.end(), 4);  // false
```

**lower_bound**：返回第一个 ≥ x 的元素的迭代器。

**upper_bound**：返回第一个 > x 的元素的迭代器。

```cpp
auto it1 = lower_bound(a.begin(), a.end(), 2);  // 指向第一个 2
auto it2 = upper_bound(a.begin(), a.end(), 2);  // 指向 3

// 转成下标
int idx = it1 - a.begin();     // idx = 1

// 统计出现次数
int cnt = it2 - it1;           // 3 - 1 = 2，2 出现了 3 次? 等等...
```

等等，我们再算一下。数组是 {1, 2, 2, 2, 3, 5}。it1 指向下标 1（第一个 2），it2 指向下标 4（值 3）。it2 - it1 = 4 - 1 = 3。2 确实出现了 3 次。对了！

这里有个容易搞混的点：STL 的迭代器是左闭右开的。`a.begin()` 指向第一个元素，`a.end()` 指向最后一个元素的后面。所以传参时传 `a.begin()` 和 `a.end()` 就覆盖了整个数组。

如果你用的是普通数组而不是 vector，也可以用指针：

```cpp
int a[] = {1, 2, 2, 2, 3, 5};
int n = 6;
bool found = binary_search(a, a + n, 2);       // true
auto it = lower_bound(a, a + n, 2);
int idx = it - a;                               // idx = 1
```

对于数组，`a` 是指向首元素的指针，`a + n` 是指向尾后位置的指针——这和迭代器的左闭右开完全对应。

还有一个重要细节：**这些函数都要求序列有序**。如果你的数组是乱序的，STL 不会报错，但结果不可预测——可能返回错误结果，行为是未定义的。

实战中一个常见的坑是：二分查找返回的迭代器可能是 `a.end()`（没找到）。如果你不检查就直接解引用或做下标运算，会出问题：

```cpp
auto it = lower_bound(a.begin(), a.end(), 100);  // 100 大于所有元素
if (it != a.end()) {
    // 安全：it 指向有效元素
    cout << *it << endl;
} else {
    cout << "not found" << endl;
}
```

今天的重点是会用 STL。手写二分能帮你理解原理，而用 STL 能让代码更简洁、更不容易出错。两者都要掌握。

## 🖥️ 板书/屏幕要点
- `binary_search(begin, end, x)` → bool
- `lower_bound(begin, end, x)` → 迭代器（第一个 ≥ x）
- `upper_bound(begin, end, x)` → 迭代器（第一个 > x）
- 迭代器 - begin() = 下标
- 必须检查返回值是否等于 end()

## 📌 小结
STL 二分函数让二分查找变成一行代码的事。关键掌握三个函数（binary_search / lower_bound / upper_bound）、迭代器与下标的转换、未找到时的处理。下节课我们做一道综合挑战题——搜索旋转排序数组。

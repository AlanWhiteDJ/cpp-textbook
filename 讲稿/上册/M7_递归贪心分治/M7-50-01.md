## 📝 讲稿正文

同学们，M7 模块的最后一站——**倍增与快速幂**。

先来感受一组数字。一张纸厚度 0.1 毫米。对折 1 次：0.2 毫米。对折 10 次：10 厘米。对折 20 次：100 米。对折 30 次：**107 公里**——比大气层还厚，超过太空边界！

这就是翻倍的力量。一开始看着不起眼，越到后面越恐怖。编程里，这个思想变成了**快速幂**和**倍增**。

快速幂要解决什么问题？计算 a^b mod p。a 可能很大，b 可能高达 10^9。你不可能把 a 乘 b 次——10^9 次循环，一秒跑不完。

但我们可以用"二进制拆分"的神奇方法。比如 a^13。13 的二进制是 1101。这个二进制告诉我们：

```
13 = 8 + 4 + 1 = 2³ + 2² + 2⁰
```

所以 a^13 = a⁸ × a⁴ × a¹！只需要乘 3 次，而不是 12 次。

怎么得到 a¹, a², a⁴, a⁸？很简单，每次平方：a¹ → a²（平方）→ a⁴（再平方）→ a⁸（再平方）。每平方一次，指数翻倍一次。这就是"倍增"——每次翻倍，恰好对应二进制位权重的递增。

算法流程：用一个变量 result 存最终结果（初始 1），用 base 存当前的 a^(2^k)（初始 a）。逐位检查 b 的二进制：

- 如果 b 的最低位是 1，`result = result × base`
- base 平方：`base = base × base`
- b 右移一位：`b >>= 1`

```cpp
long long qpow(long long a, long long b, long long p) {
    long long result = 1;
    a %= p;                // 先取模，防溢出
    while (b > 0) {
        if (b & 1)         // b 的当前最低位是 1
            result = (result * a) % p;
        a = (a * a) % p;   // a 平方（翻倍）
        b >>= 1;           // b 右移一位
    }
    return result;
}
```

我们跟踪 2^13 mod 1000：

| 轮次 | b (二进制) | b&1 | result | a |
|------|-----------|-----|--------|---|
| 初始 | 1101 | — | 1 | 2 |
| 1 | 110**1** | 1 | 1×2=2 | 2²=4 |
| 2 | 11**0** | 0 | 2 | 4²=16 |
| 3 | 1**1** | 1 | 2×16=32 | 16²=256 |
| 4 | **1** | 1 | 32×256=8192≡192 | — |

结果 192。验证：2^13 = 8192, mod 1000 = 192。✅

四轮循环，每轮常数操作——复杂度 O(log b)。b=10^9 时只需约 30 次循环！

几个细节注意。第一，`result` 初始化为 1（乘法的单位元），不能是 0。第二，每一步都取模，防止 `result * a` 溢出 long long。第三，`b & 1` 判断最低位是否为 1，等价于 `b % 2 == 1`，但位运算更快。

还有，为什么 `b >>= 1` 放在最后？因为当前轮检查的是最低位，检查完后 b 右移，下一轮的最低位就是原本次低位——这恰好是"从低位到高位逐位检查"。

快速幂是竞赛中最常用的工具之一，后面学组合数学、数论时几乎每题都要用到。一定要练到闭着眼都能写出来。

## 🖥️ 板书/屏幕要点
- 翻倍的力量：2^30 ≈ 10 亿倍
- 二进制拆分：b = Σ 2^k → a^b = Π a^(2^k)
- 快速幂：result=1; while(b>0) { if(b&1) result=result*a%p; a=a*a%p; b>>=1; }
- 复杂度：O(log b)

## 📌 小结
快速幂的秘诀就两个：二进制拆分和翻倍平方。每次平方让底数的"权重"翻倍，b 的二进制位决定这一份要不要乘进去。O(log b) 的复杂度让天文数字级别的指数也能秒出结果。下节课我们练代码细节和变种应用。

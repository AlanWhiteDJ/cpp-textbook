## 📝 讲稿正文

前面几节课，我们的回溯程序会把所有可能的分支都走一遍。但有些分支，你一眼就能看出来"不用走了，肯定没戏"。如果能提前砍掉这些分支，程序会快很多——这就是**剪枝**。

打个比方：你在超市里找一瓶酱油。你直奔调味品区，不会去生鲜区和零食区找——因为你"剪"掉了那些不可能的区域。回溯里的剪枝也是这个道理。

先说一个最简单的剪枝例子。组合枚举中，我们要从 1~n 中选 k 个数。假设 n=10，k=8，当前已经选了 2 个数，还有 6 个要选。但剩下还没考虑的数字只剩 5 个了——即使全部选上也不够 8 个。这条路走到头一定是死胡同，不如现在就回头：

```cpp
void dfs(int x) {
    if (chosen.size() == k) { /* 输出 */ return; }
    if (x > n) return;
    // 剪枝：剩下的数字全选了也凑不够 k 个
    if (chosen.size() + (n - x + 1) < k) return;
    
    chosen.push_back(x); dfs(x + 1); chosen.pop_back();
    dfs(x + 1);
}
```

第 5 行就是剪枝。`n - x + 1` 是从 x 到 n 还有多少个数字。如果当前的个数加这个都不够 k，后面全是白费功夫。这一刀下去，大量分支不用走了。

接下来，我们介绍一个经典的回溯剪枝问题——**八皇后**。在 8×8 的棋盘上放 8 个皇后，要求任何两个皇后不能在同一行、同一列、同一对角线。问有多少种放法。

分析一下：每行必须放一个皇后（因为 8 行 8 个皇后），所以问题转化为——确定第 i 行的皇后放在第几列。

这是典型的回溯：第 1 行有 8 种选择，选完标记这一列和对角线不能用；然后第 2 行从剩余的列里选……这比全排列多了一个"对角线"的限制。

对角线的判断是个小技巧：对于坐标 (row, col)，从左上到右下的对角线满足 `row - col` 为常数；从右上到左下的对角线满足 `row + col` 为常数。我们用两个 bool 数组分别标记：

```cpp
int col[9];            // col[i] = 第 i 行皇后放的列号
bool usedCol[9];       // 列是否被占用
bool usedDiag1[20];    // 对角线1：row-col+8（+8防止负下标）
bool usedDiag2[20];    // 对角线2：row+col
int ans = 0;

void dfs(int row) {    // 正在放第 row 行
    if (row > 8) { ans++; return; }
    for (int c = 1; c <= 8; c++) {
        if (!usedCol[c] && !usedDiag1[row - c + 8] && !usedDiag2[row + c]) {
            col[row] = c;
            usedCol[c] = usedDiag1[row - c + 8] = usedDiag2[row + c] = true;
            dfs(row + 1);
            usedCol[c] = usedDiag1[row - c + 8] = usedDiag2[row + c] = false;
        }
    }
}
```

答案是多少？92 种。纯回溯不剪枝也能跑出来，但这个框架是你们以后遇到 n 皇后问题的基础。

好，到这里第 44 章的内容全部讲完。我们回顾一下：递归三要素 → 汉诺塔 → 回溯框架 → 全排列 → 组合枚举 → 剪枝。这根链条的核心是"试—记—撤"三步曲。记住：回溯就是遍历选择树，剪枝就是提前判断一个分支值不值得走下去。

## 🖥️ 板书/屏幕要点
- 剪枝 = 提前砍掉不可能的分支，不改变正确性
- 组合剪枝：`chosen.size() + (n - x + 1) < k` 则返回
- 八皇后：逐行放 → 列标记 + 对角线标记（row-col 常数 + row+col 常数）
- 第44章总结：回溯 = 递归 + 枚举 + 试记撤 + 剪枝

## 📌 小结
剪枝是回溯优化的核心手段——在搜索树中提前判断一个分支是否能通向答案，不能就砍掉。八皇后是回溯剪枝的经典题，对角线判重的小技巧值得记住。第 44 章到此结束，下章我们学记忆化搜索——用空间换时间，让递归飞起来！

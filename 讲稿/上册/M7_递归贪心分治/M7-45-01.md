# 第45章 第1课时：重复计算的代价与记忆化思想

**时长**：15-20分钟 | **对应教材**：第45章 45.1-45.2

## 🎯 本课时目标
- 感受纯递归的重复计算有多可怕
- 理解记忆化的核心思想：算过就存，下次直接用
- 用记忆化改写斐波那契，从指数级到线性级

## 📝 讲稿正文

同学们，上章我们学了回溯，感受到递归遍历所有可能性的能力。但递归有个致命弱点——它太"健忘"了。算过的结果转眼就忘，下次碰到同样的问题，还会傻乎乎地重算一遍。

我给你们看一个触目惊心的例子。还记得斐波那契数列的递归版吗？

```cpp
int fib(int n) {
    if (n <= 2) return 1;
    return fib(n-1) + fib(n-2);
}
```

我们来数一数，算 fib(6) 的时候，fib(3) 被算了几次？

fib(6) 调 fib(5) 和 fib(4)。fib(5) 调 fib(4) 和 fib(3)。fib(4) 又调 fib(3) 和 fib(2)……你顺着调用树画下去，fib(3) 被调用了 3 次，fib(2) 被调用了 5 次！fib(1) 更是被调用了 8 次！

这只是 n=6 啊！n=40 的时候，fib(40) 需要超过 3 亿次函数调用，你的程序直接卡死。但答案才 1.6 亿左右，本来 O(n) 就能算出来的东西，硬是被重复计算拖成了指数级。

问题出在哪？出在"算完就忘"。计算 fib(5) 时顺带算了 fib(3) 的结果，但等计算 fib(4) 时又要 fib(3)，却没有记录下来，只好重新算。

解决方案极其简单，就一句话：**算过的东西，记在小本本上，下次直接抄！**

这就是记忆化搜索。我们只需要加一个数组：

```cpp
long long memo[100];  // 记忆化数组

long long fib(int n) {
    if (n <= 2) return 1;
    if (memo[n] != 0)           // 算过吗？
        return memo[n];         // 算过了，直接拿！
    return memo[n] = fib(n-1) + fib(n-2);  // 没算过，算了存起来
}
```

注意第 5 行和第 6 行。`memo[n] != 0` 表示"这个值已经被算过了"——因为全局数组默认初始化为 0，而斐波那契数列的值都是正整数，0 永远不会是合法结果，正好当"未计算"的标记。

第 7 行 `return memo[n] = ...` 是一个紧凑写法：先把计算结果赋值给 memo[n]，然后返回它。

这个改动有多神奇？fib(50) 从之前的"算到天荒地老"变成了"瞬间出结果"。复杂度从 O(2^n) 直接降到 O(n)。每个 fib(k) 只被计算一次，算完就存起来，后面再要用到它时直接查表。

这种"用空间换时间"的思想，是全计算机科学最核心的优化策略之一。后面你们学的动态规划，本质也是这个——只不过 DP 是自底向上预先填表，而记忆化是自顶向下边算边记。

## 🖥️ 板书/屏幕要点
- 纯递归斐波那契：重复计算爆炸，O(2^n)
- 记忆化核心：`if (memo[n] 已算) return memo[n];`
- 三步：查表 → 计算 → 存储
- "未计算"标记必须选合法结果不会出现的值（斐波那契用 0，一般用 -1 更安全）

## ✏️ 随堂练习
1. 用 `clock()` 计时对比纯递归和记忆化计算 fib(45) 的运行时间
2. 把记忆化数组的初值从 0 改成 -1（用 `memset(memo, -1, sizeof(memo))`），改写判断条件

## 📌 小结
记忆化搜索一句话：算过就存，存了就用。它让斐波那契从 O(2^n) 变成 O(n)，效果翻天覆地。关键是选对"未计算"标记。下节课我们把记忆化的框架抽象出来，用在更复杂的问题上。

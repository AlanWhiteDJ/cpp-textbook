## 📝 讲稿正文

今天的课是二分查找的收官之战。我们来挑战一道经典的变种题：**搜索旋转排序数组**。

问题是这样：一个原本升序的数组在某个位置被"旋转"了。比如 {0, 1, 2, 4, 5, 6, 7} 在位置 3 旋转后变成 {4, 5, 6, 7, 0, 1, 2}。要求在这个数组里查找目标值，返回下标，找不到返回 -1。复杂度必须 O(log n)。

这个数组不是全局有序的，普通二分不能直接用。但我们仔细观察——虽然整个数组不是有序的，但**每次从中间切开后，至少有一半是有序的**。

比如 {4, 5, 6, 7, 0, 1, 2}，mid=3 对应 7。左半 {4,5,6,7} 是有序的，右半 {0,1,2} 也是有序的！关键是判断目标落在哪边。

思路分三步：

第一步，判断 mid 落在左有序段还是右有序段。如果 `a[l] <= a[mid]`，左半有序；否则右半有序。

第二步，判断目标是否在有序的那一半里。如果在，就在有序段里二分；如果不在，去另一半找。

```cpp
int search(int a[], int n, int target) {
    int l = 0, r = n - 1;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (a[mid] == target) return mid;
        
        if (a[l] <= a[mid]) {                    // 左半有序
            if (a[l] <= target && target < a[mid])
                r = mid - 1;                     // 目标在左半
            else
                l = mid + 1;                     // 目标在右半
        } else {                                 // 右半有序
            if (a[mid] < target && target <= a[r])
                l = mid + 1;                     // 目标在右半
            else
                r = mid - 1;                     // 目标在左半
        }
    }
    return -1;
}
```

这个代码比普通二分复杂一些，但骨架是一样的——while 循环、mid 计算、区间缩小。多出来的只是"判断哪边有序"和"目标在哪边"的逻辑。

追踪一下：在 {4,5,6,7,0,1,2} 中找 0。第一轮 l=0, r=6, mid=3, a[3]=7。a[0]=4 <= 7，左半有序。0 不在 [4, 7) 内，所以去右半：l=4。第二轮 l=4, r=6, mid=5, a[5]=1。a[4]=0 <= 1，左半有序。0 在 [0, 1) 内，去左半：r=4。第三轮 l=4, r=4，找到！

这道题的启示是：二分不一定要全局有序——**只要每次能把搜索范围砍掉一部分**就行。旋转数组的关键洞察是"至少一半有序"，这个有序的部分让我们能判断目标在不在里面。

好，第 46 章到此结束。回顾一下：我们从猜数字游戏出发，学了二分查找的原理、两种区间写法、lower_bound/upper_bound、STL 函数，最后挑战了旋转数组。核心始终是那句话：**在有序（或部分有序）的序列上，每次砍一半，O(log n)**。二分查找是你们今后写代码最频繁使用的工具之一，一定要练到闭着眼睛都能写对。

## 🖥️ 板书/屏幕要点
- 旋转数组：原本有序，某点旋转 → 至少一半有序
- 策略：判断哪半有序 → 目标是否在有序半 → 缩小范围
- 第46章总结：二分前提（有序）→ 标准框架 → lower/upper_bound → STL → 变种

## 📌 小结
二分查找看似简单，实则细节满满。从基本的有序数组二分到旋转数组变种，本质都是"每次砍掉不可能的区间"。打好二分基础，下章学二分答案时你会发现：原来"猜答案"和"猜数字"是同一回事！

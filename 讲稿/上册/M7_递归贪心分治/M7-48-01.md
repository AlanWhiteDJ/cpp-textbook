## 📝 讲稿正文

同学们，从今天开始我们进入一种非常"聪明"的算法范式——贪心算法。

什么叫贪心？很简单：**每走一步，都选当下看起来最好的那个选项。** 不回头、不后悔、不考虑"长远"。就像你去吃自助餐，肚子容量有限，你一盘接一盘地拿最贵的海鲜——每步都是当时最好的选择。

贪心的框架极其简单：只要还有选择，就选当前最优的那个，然后更新候选集合。

但贪心有个要命的陷阱：**局部最优不一定等于全局最优**。还是吃自助餐的例子——你拿了龙虾，确实单价最高，但它太占肚子了。也许你少吃一个龙虾，多吃几份牛排，总体更划算。这就是背包问题中贪心失效的典型场景。

所以贪心不像 DP 或回溯那样"稳妥"——回溯的答案一定对（因为它枚举了所有可能），DP 的答案也一定对（只要状态和转移正确）。但贪心呢？你得证明它正确，否则写出来的代码跑出的结果可能是错的。

来看一个"贪心有效"的经典例子：**找零钱**。

面值 {1, 5, 10, 20, 50, 100}，凑出金额 M，要求硬币数最少。贪心策略：每次选面值最大且不超过剩余金额的硬币。

```cpp
int coins[] = {100, 50, 20, 10, 5, 1};
int m, cnt = 0;
cin >> m;
for (int c : coins) {
    cnt += m / c;      // 尽量用大面值
    m %= c;            // 剩下的金额
}
cout << cnt;
```

比如 m=87。先看 100：太大，跳过。50：能用 1 个，剩 37。20：能用 1 个，剩 17。10：能用 1 个，剩 7。5：能用 1 个，剩 2。1：用 2 个。总共 1+1+1+1+2 = 6 枚硬币。

这个贪心对吗？对人民币面值是对的。但面值换一下呢？{1, 3, 4}，凑 6。贪心：先拿 4，剩 2，拿两个 1，总共 3 枚。但最优是 3+3，2 枚。贪心失效了！

这就是贪心的真面目：它对某些问题有效，但不是万能钥匙。人民币面值是精心设计的——每种大面值都是小面值的倍数（至少优于此性质），所以贪心成立。换成 {1, 3, 4} 就没有这个性质了。

那贪心什么时候一定对？需要满足两个条件：**贪心选择性质**（局部最优能推出全局最优）和**最优子结构**（问题的最优解包含子问题的最优解）。这两个性质通常需要数学证明——反证法、交换法、归纳法是三种常用手段。

后面几节课我们会看到更多贪心有效的例子——最优装载、活动选择、排队接水。你会发现，贪心在"排序后依次选择"这类场景中特别有效。

## 🖥️ 板书/屏幕要点
- 贪心 = 每步选局部最优，永不回头
- 框架：while(有选择) { 选当前最优; 更新候选集; }
- 找零钱：大面值优先，人民币面值下正确，{1,3,4} 下错误
- 贪心正确性需要证明（反证法/交换法/归纳法）

## 📌 小结
贪心简单、快速，但也容易被骗。它像一把快刀——用对了削铁如泥，用错了伤到自己。关键在于判断问题是否具有贪心选择性质。下节课我们学最优装载和排队接水——贪心完全健康的两个例子。

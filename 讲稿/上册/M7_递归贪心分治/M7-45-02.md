## 📝 讲稿正文

上节课我们用记忆化优化了斐波那契。今天把套路抽象出来，让它变成一个能用在各种问题上的标准框架。

看这个框架：

```
返回值 dfs(状态参数...) {
    if (边界条件) return 边界值;
    if (memo[状态] 已被计算) return memo[状态];

    计算结果 = 由 dfs(更小的状态) 推导;
    return memo[状态] = 计算结果;
}
```

三步走：第一步查表——如果这个状态已经算过了，直接返回；第二步计算——和普通递归一样算；第三步存储——把结果存进备忘录，再返回。

有没有觉得这个框架特别像你们做数学题的习惯？碰到一道题，先翻翻错题本看有没有做过类似的——这就是"查表"。没有的话自己算一遍，然后把答案记在错题本上——这就是"计算+存储"。

好，我们用网格路径问题来练手。问题描述：一个 m×n 的网格，从左上角 (1,1) 出发，走到右下角 (m,n)。每次只能向右或向下走一步。有多少条不同的路径？

先想想纯递归怎么写。从 (x,y) 出发，要么向右走到 (x+1,y)，要么向下走到 (x,y+1)。所以路径数等于向右走和向下走的路径数之和。边界：到达 (m,n) 算一条；超出边界返回 0。

```cpp
int paths(int x, int y, int m, int n) {
    if (x == m && y == n) return 1;
    if (x > m || y > n) return 0;
    return paths(x+1, y, m, n) + paths(x, y+1, m, n);
}
```

这个纯递归有重复计算吗？太多了！到达 (2,2) 这个格子，可以从 (1,1)→(2,1)→(2,2) 过来，也可以从 (1,1)→(1,2)→(2,2) 过来——(2,2) 被算了两次。格子越多，重复越严重。

加上记忆化：

```cpp
long long memo[25][25];
memset(memo, -1, sizeof(memo));

long long paths(int x, int y, int m, int n) {
    if (x == m && y == n) return 1;
    if (x > m || y > n) return 0;
    if (memo[x][y] != -1) return memo[x][y];    // 查表
    return memo[x][y] = paths(x+1,y,m,n) + paths(x,y+1,m,n);
}
```

注意这里的 memo 是二维的——因为状态由两个参数 (x, y) 共同决定。记忆化数组的维度必须覆盖所有状态变量。每个格子只被算一次，复杂度从指数降到 O(m×n)。

其实这个问题有一个数学公式——答案就是 C(m+n-2, m-1)，总共要走 m+n-2 步，选其中的 m-1 步向下走，剩下的向右走。但记忆化搜索的思路能推广到更复杂的问题——比如每个格子有不同的"过路费"，那就不能用组合数直接算了，但记忆化照样能解。

从一维到二维记忆化，核心不变：状态 → 查表 → 计算 → 存储。只是状态的维度从 1 变成了 2。

## 🖥️ 板书/屏幕要点
- 记忆化标准框架：查表 → 计算 → 存储
- 网格路径：状态 (x,y)，memo[x][y] 存从 (x,y) 到终点的路径数
- 递推：paths(x,y) = paths(x+1,y) + paths(x,y+1)
- 数组维度 = 状态变量数量

## 📌 小结
记忆化框架三步走，适用于任何有重叠子问题的递归。状态从一维扩展到多维时，memo 数组也跟着扩展维度。网格路径是非常好的练手题——它让你直观感受到，每个子问题只算一次带来的巨大性能提升。下节课我们学数字三角形，并比较记忆化和动态规划。

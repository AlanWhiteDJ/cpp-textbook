## 📝 讲稿正文

同学们，今天咱们正式进入 M7 模块——递归、贪心、分治。这些可是算法竞赛里最核心的思维武器。首先从递归讲起。

你们在第 26 章已经写过阶乘和斐波那契了，"自己调用自己"这个基本功过关了吧？那我来考考你们：写递归最重要的是什么？不是代码怎么写，而是你脑子里有没有想清楚**三件事**。

第一件事：**边界条件**。什么时候不用再递归了？阶乘的边界是 n 等于 1 的时候，斐波那契的边界是 n 等于 1 或 2。没有边界条件，递归就会死循环——就像你下楼梯，如果楼梯没有尽头，你就一直往下走，走到地心去了。

第二件事：**递推关系**。大问题怎么拆成小问题？阶乘：n! 等于 n 乘 (n-1)!；斐波那契：F(n) 等于 F(n-1) 加 F(n-2)。你必须找到这个"大变小"的公式。

第三件事：**递归调用**。调用自己的时候，参数必须向边界靠近。如果你写阶乘时传的是 `factorial(n)` 而不是 `factorial(n-1)`，那就原地踏步了。

好，现在咱们用这三要素来攻克一个经典问题——**汉诺塔**。

有同学听过这个故事吧？三根柱子，n 个大小不同的盘子，初始都在 A 柱。规则是每次只能移一个，大盘子不能压小盘子上。目标：把所有盘子移到 C 柱。

乍一看好复杂，但用递归的角度想，其实就三步。

假设我们想把 n 个盘子从 A 移到 C（B 是辅助柱）。最底下那个最大的盘子怎么动的？它一定是最后动的——因为大盘不能压小盘。那在动它之前，上面的 n-1 个小盘子必须先从 A 移到 B。好，第一步：把上面 n-1 个盘子从 A 移到 B（借助 C）。

这时候 A 柱只剩最大的那个盘子了，直接把它移到 C——第二步完成。

现在 n-1 个盘子都在 B 柱上，把它们移到 C 柱上就行——第三步：把 n-1 个盘子从 B 移到 C（借助 A）。

你看出来了没有？第一步和第三步，恰好是规模更小的汉诺塔问题！这就是递归的递推关系。边界呢？n 等于 1 的时候，直接移过去就行。

咱们看代码：

```cpp
void hanoi(int n, char from, char to, char aux) {
    if (n == 1) {
        cout << from << " -> " << to << endl;
        return;
    }
    hanoi(n - 1, from, aux, to);        // ① n-1个：from→aux，借助to
    cout << from << " -> " << to << endl; // ② 最底下那个：from→to
    hanoi(n - 1, aux, to, from);        // ③ n-1个：aux→to，借助from
}
```

n 个盘子需要多少步？递推公式：T(n) = 2 × T(n-1) + 1，T(1) = 1。展开就是 2^n - 1。64 个盘子要 2^64 - 1 步，每秒一步的话要 5800 亿年——比宇宙年龄还长。所以我们不用担心世界毁灭。

这个代码一共不到 10 行，但背后的递归思维极其深刻。能把这么复杂的问题用这么短的代码解决，这就是递归的魅力。

## 🖥️ 板书/屏幕要点
- **递归三要素**：边界条件 | 递推关系 | 递归调用（参数收敛）
- **汉诺塔三步**：n-1 from→aux → 最大盘 from→to → n-1 aux→to
- 代码框架（10行以内）
- 移动次数：T(n) = 2^n - 1

## 📌 小结
今天我们重温了递归三要素，并且用它优雅地解决了汉诺塔问题。关键是那个"三步拆解"的思维——把 n 个盘子的问题变成两个 n-1 个盘子的问题。下节课，我们进入递归更强大的用法——**回溯**，也就是"试一条路、不行就退"的探索模式。

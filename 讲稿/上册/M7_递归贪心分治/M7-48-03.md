# 第48章 第3课时：活动选择与贪心证明

## 📝 讲稿正文

今天这节课，我们学一个贪心里最优雅的例子——**活动选择**，也叫区间调度。

场景：你有一个会议室，今天有 n 个活动要申请使用。每个活动有开始时间 sᵢ 和结束时间 fᵢ。同一时间只能有一个活动使用会议室。你的目标是安排尽可能多的活动。

直觉上，你会怎么选？有人说"选最短的活动"——但一个很短的活动可能横在中间，把两个长活动都挡住了。有人说"选最早开始的"——但一个很早开始、很晚结束的活动可能霸占一整天。

正确答案是：**每次选最早结束且与已选不冲突的活动。**

为什么？因为越早结束，给后面留下的时间窗口就越大，后面能排的活动就更多。你仔细品品这个逻辑——你选的不是"哪个活动最好"，而是"哪个活动给后面留的空间最大"。这是一种"利他"的贪心。

```cpp
struct Act { int s, f; };
Act a[1005];

bool cmp(Act x, Act y) { return x.f < y.f; }

sort(a, a + n, cmp);       // 按结束时间从早到晚
int cnt = 1;
int lastEnd = a[0].f;      // 第一个选最早结束的
for (int i = 1; i < n; i++) {
    if (a[i].s >= lastEnd) {  // 不冲突
        cnt++;
        lastEnd = a[i].f;
    }
}
cout << cnt;
```

我们跟踪一下。假设有 5 个活动：(1,4), (3,5), (0,6), (5,7), (3,8)。按结束时间排序后：(1,4), (3,5), (0,6), (5,7), (3,8)。第一个选 (1,4)，lastEnd=4。第二个 (3,5)：开始 3 < 4，冲突，跳过。第三个 (0,6)：冲突，跳过。第四个 (5,7)：开始 5 ≥ 4，选！lastEnd=7。第五个 (3,8)：冲突。一共选了 2 个。

最优是 2 个吗？手工验证：选 (1,4) 和 (5,7) 确实是 2 个。或者选 (0,6) 后面什么都不能选了，只有 1 个。所以贪心正确。

这个策略为什么正确？用**反证法**。

假设最优解选的第一个活动和贪心选的（最早结束的那个）不同。贪心选的是结束时间最早的活动 A。最优解的第一个是 B，且 B 的结束时间 ≥ A 的结束时间。那把 B 换成 A——A 比 B 结束得更早或同时，不会和后面的活动冲突，而活动数量不变。所以存在一个以 A 开头的、活动数量不差于原最优解的新方案。重复这个过程，最优解终将变成贪心解。

你看，反证法的核心就是"假设不一样 → 替换成贪心选的不变差 → 它们可以一样"。

活动选择是"按结束时间排序"的经典案例。很多新手会本能地按开始时间排序，或者按时长排序——这两种都是错的。按开始排可能选一个超长的活动堵住后面；按时长排可能选一个"恰到好处"挡住两个大活动的短活动。只有按结束时间排，贪心才成立。

这道题的变种非常多——最少需要几个教室、每个活动有收益求最大总收益——但万变不离其宗，核心都是"按结束时间排序 + 贪心选择"或它的变形。

## 🖥️ 板书/屏幕要点
- 活动选择：按**结束时间**排序（不是开始时间、不是时长）
- 贪心策略：每次选最早结束且不冲突的
- 证明（反证法）：假设最优解第一个不同 → 换成贪心选的不变差
- 复杂度：O(n log n)（排序）

## 📌 小结
活动选择是贪心里最美的例子——"选最早结束"看似不贪心，实则是最聪明的大局观。反证法证明简洁有力。下节课我们总结贪心的陷阱和反例，完成本章学习。

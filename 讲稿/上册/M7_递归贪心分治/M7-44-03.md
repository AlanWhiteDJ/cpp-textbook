## 📝 讲稿正文

上节课我们搭好了全排列的框架。这节课我们钻深一点，把它的执行过程吃透。

我们拿 n=3 来跟踪。一开始 `dfs(1)` 被调用，pos=1，正在填第一个位置。进入 for 循环：i=1，used[1] 是 false，所以选 1。a[1]=1，used[1]=true，然后调用 `dfs(2)`。

现在到了 `dfs(2)`，pos=2。for 循环 i=1：used[1] 是 true，跳过。i=2：used[2] 是 false，选 2。a[2]=2，used[2]=true，调用 `dfs(3)`。

`dfs(3)`，pos=3。i=1 和 i=2 都被用了，跳过。i=3：选 3，调用 `dfs(4)`。

`dfs(4)` 里 pos=4 > n=3，边界触发！输出 "1 2 3"，然后 return。

关键来了：`dfs(4)` return 之后，程序回到哪里？回到 `dfs(3)` 的 for 循环里，紧接着 `dfs(pos+1)` 那句之后——也就是 `used[3] = false`。数字 3 被释放了。

然后 for 循环继续——咦，i 已经到 3 了，循环结束。`dfs(3)` 也 return 了。

回到 `dfs(2)`，执行 `used[2] = false`，释放 2。然后 for 循环走下一个：i=3。used[3] 是 false（刚被释放），选 3！a[2]=3，调用 `dfs(3)`……这条路走下去，输出 "1 3 2"。

同学们，这个过程就像树的深度优先遍历——先走到最深的叶子，然后一层层退回来，换一个分支再走到底。你们回去一定要自己用纸笔画一遍这个调用栈。

好，框架吃透之后，我们来看一个变形：**字符串全排列**。输入一个不含重复字符的字符串，比如 "abc"，输出它的所有排列。

思路几乎一样，只是把 int 数组换成 string。used 数组不再标记数字，而是标记字符的下标：

```cpp
string s;
string cur;       // 当前排列
bool used[10];

void dfs(int pos) {
    if (pos == s.length()) {
        cout << cur << endl;
        return;
    }
    for (int i = 0; i < s.length(); i++) {
        if (!used[i]) {
            cur.push_back(s[i]);       // 记
            used[i] = true;            // 记
            dfs(pos + 1);
            used[i] = false;           // 撤
            cur.pop_back();            // 撤
        }
    }
}
```

注意这里有两个"撤"：一个撤销 used 标记，一个把字符从当前排列里弹出去。这两个必须成对出现，缺一个都会导致状态污染。

这时候有同学可能要问：如果字符串里有重复字符怎么办？比如 "aab"。那就需要额外的去重处理——可以先用 sort 排序，然后在枚举的时候判断：如果当前字符和前一个字符相同，且前一个字符在这一层没有被使用，就跳过。这个技巧叫"同层去重"，是回溯里比较高级的内容，我们以后碰到了再细讲。

## 🖥️ 板书/屏幕要点
- n=3 全排列的完整调用树（三级递进、逐层回溯）
- 字符串全排列：`used[i]` 标记下标，`push_back` + `pop_back` 成对
- 回溯的两个层面：标记数组的恢复 + 结果数组的恢复

## ✏️ 随堂练习

2. 实现字符串全排列，输入 "abc"，验证输出 6 种排列
3. 挑战：输入 "aab"，尝试实现去重全排列，只输出 3 种结果

## 📌 小结
全排列是回溯最经典的入门题，理解了它的执行过程，就理解了回溯的本质——深度优先搜索选择树，走到叶子就输出，退回来换分支。变形到字符串也非常自然，核心的"试—记—撤"框架不变。下节课我们学另一种枚举模式——组合枚举。

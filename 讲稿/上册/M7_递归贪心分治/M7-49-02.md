## 📝 讲稿正文

归并排序是分治的教科书级案例。虽然你们在 M6 学过，今天我们换个角度——用"分治三步骤"的框架重新审视它。

分：把数组 a[l..r] 从中间切成 a[l..mid] 和 a[mid+1..r]。
治：递归排序左右两半。
合：把两个有序子数组合并成一个有序数组。

代码大家应该都熟悉：

```cpp
void mergeSort(int a[], int l, int r) {
    if (l >= r) return;              // 边界：1个或0个元素天然有序
    int mid = l + (r - l) / 2;
    mergeSort(a, l, mid);            // 治左
    mergeSort(a, mid + 1, r);        // 治右
    // 合：合并两个有序段
    int i = l, j = mid + 1, k = 0;
    int tmp[100005];
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else              tmp[k++] = a[j++];
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r)   tmp[k++] = a[j++];
    for (i = 0; i < k; i++) a[l + i] = tmp[i];
}
```

注意"合"这一步的两个 while——合并完之后，左右两边可能有剩余，必须全部收尾。

现在我们用**递归树**来理解归并排序的复杂度。假设 n=8。

```
          [0..7]          ← 第0层，合并代价 O(n)
         /      \
      [0..3]    [4..7]    ← 第1层，两个合并各 O(n/2)，总 O(n)
      /    \    /    \
   [0..1] [2..3] [4..5] [6..7] ← 第2层，四个合并各 O(n/4)，总 O(n)
   /  \   /  \   /  \   /  \
 [0] [1] [2] [3] [4] [5] [6] [7] ← 第3层，边界
```

树共 log₂ 8 = 3 层（不含叶子层）。每一层的合并总代价都是 O(n)——因为每层处理的数据总量就是 n 个元素。所以总复杂度 = 层数 × 每层代价 = O(n log n)。

这就是分治复杂度的典型推导方式：画出递归树，数层数（log n），算每层合并代价，相乘。后面我们会用"主定理"来统一处理，但递归树是最直观的。

归并排序还有一个被低估的优点：**它是稳定的**。稳定排序意味着相同值的元素，排序后相对顺序不变。快速排序和堆排序都不稳定，但归并排序稳定——因为在合并时，当左右两个元素相等时，优先取左边的。

另外，归并排序天然适合外部排序——当数据大到内存装不下时，可以分段读到内存，各自排序后写回磁盘，最后合并。这就是"外排序"的思想。

归并排序还能同时做别的事——比如统计**逆序对**。在合并的时候，当右半边的元素 a[j] 被拿进 tmp 时，左半边还剩的元素（从 i 到 mid）都大于 a[j]——这些就是逆序对。这就是著名的"归并求逆序对"，O(n log n)。

## 🖥️ 板书/屏幕要点
- 分：mid = (l+r)/2 → 治：递归左右 → 合：双指针合并
- 递归树：log n 层，每层 O(n)，总 O(n log n)
- 稳定性：相等时优先左元素
- 逆序对：右元素入 tmp 时，左剩余元素都构成逆序对

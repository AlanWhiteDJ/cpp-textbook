## 📝 讲稿正文

今天我们先聊聊分治的一个经典应用——**最近点对**，然后学主定理，最后做本章总结。

问题：平面上有 n 个点，找距离最近的一对。暴力是 O(n²)，分治能做到 O(n log n)。

分治思路很巧：先把所有点按 x 坐标排序。然后用一条竖线 x = midX 把点分成左右两半。递归求左半的最近距离 dL 和右半的最近距离 dR。令 d = min(dL, dR)。但还要考虑"跨线"的点对——一个在左边，一个在右边，它们可能比 d 更近。

关键洞察：只有落在 [midX-d, midX+d] 这个"带状区域"里的点，才有可能产生比 d 更近的点对。带状区域外的点距离中线超过 d，和另一侧任何点的距离至少是 d。

把带状区域里的点按 y 坐标排序，然后对每个点，只需检查它上面最多 7 个点。为什么是 7？因为在一个 d×2d 的矩形里，最多只能放进互相距离 ≥ d 的 8 个点（鸽巢原理的变形）。

这个算法的"合"步骤非常精妙——通过几何性质把 O(n²) 的跨线检查压到了 O(n)。你们现在不要求手写完整代码，但理解这个"切 → 递归两边 → 带域合并"的思想，对培养分治思维大有裨益。

接下来我们学一个实用工具：**主定理**。它帮你快速判断分治算法的时间复杂度。

分治的复杂度通常写成递推式：T(n) = a·T(n/b) + O(n^d)。含义是：把规模 n 的问题切成 a 个子问题，每个子问题规模是原来的 n/b，合并代价是 O(n^d)。

主定理说：

| 条件 | 复杂度 |
|------|--------|
| d > log_b a | O(n^d) |
| d = log_b a | O(n^d · log n) |
| d < log_b a | O(n^{log_b a}) |

比如归并排序：T(n)=2T(n/2)+O(n)，a=2, b=2, d=1。log₂2=1，d=1=log₂2，所以 O(n log n)。

最大子段和分治：同上，也是 O(n log n)。

二分查找（减治）：T(n)=T(n/2)+O(1)，a=1, b=2, d=0。log₂1=0，d=0=log₂1，所以 O(log n)。

不需要死记公式，知道直觉就够了：**log_b a 决定了递归树的分叉速度，d 决定了合并代价。哪个大，哪个主导复杂度。**

好，第 49 章结束。总结一下：

分治 = 分 → 治 → 合。归并排序是教科书级案例，递归树分析直观。最大子段和展示了"跨越中点"的合并技巧。最近点对是分治在几何中的华丽应用。主定理帮你快速分析分治复杂度。

分治和前面学的二分不一样——二分是减治（只走一条分支），分治是多分支 + 合并。分治和回溯也不一样——回溯是枚举所有可能性，分治是分解为同类子问题。

分治、贪心、二分——这三种算法范式构成了竞赛中最核心的思维工具箱。后面 M8 我们进入数据结构的世界，把这些算法思维用到具体的数据结构上去。

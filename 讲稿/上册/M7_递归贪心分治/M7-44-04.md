## 📝 讲稿正文

前面我们学的是全排列——排列讲究顺序，123 和 132 是不同的排列。但有时候我们只关心"选了哪些"，不关心"顺序"。比如从 5 个同学里选 3 个去参加比赛，选了小明、小红、小刚——你关心的是这三个人，至于名单上谁写前面谁写后面，不重要。这就是**组合**。

组合枚举有两种经典写法，今天我们都学。

**写法一：选与不选。** 对每个数字，你有两个选择：选它，或者不选它。这棵选择树是一个二叉树——每个节点分两个叉。

还是从 1 到 n，我们要选 k 个数。对于数字 x，选它就把它放进 `chosen` 数组，然后递归考虑 x+1；不选它就什么也不做，直接递归考虑 x+1。边界有两个：一是已经选够了 k 个，输出；二是数字已经超过 n 了，直接返回。

```cpp
int n, k;
vector<int> chosen;

void dfs(int x) {                      // 正在考虑数字 x
    if (chosen.size() == k) {          // 边界1：选够了
        for (int v : chosen) cout << v << " ";
        cout << endl;
        return;
    }
    if (x > n) return;                 // 边界2：数字用完了
    
    // 分支1：选 x
    chosen.push_back(x);
    dfs(x + 1);
    chosen.pop_back();                 // 撤！
    
    // 分支2：不选 x
    dfs(x + 1);
}
```

注意第 15 行的 `pop_back()`——这也是"撤"。选了之后，返回时要弹出来。不选的分支不需要撤，因为没做任何修改。

这种写法的优点是思路简洁，"选"和"不选"对应代码里两个 dfs 调用。缺点是当 n 很大 k 很小时，会探索很多"不选"的分支，效率稍低。

**写法二：按顺序选。** 为了避免重复，我们可以规定：选的数字必须升序排列。这样 (1,2) 是一种，但永远不会出现 (2,1)——因为 2 后面只能选比 2 大的。

```cpp
void dfs(int start, int cnt) {         // 从 start 开始选，已选 cnt 个
    if (cnt == k) {
        for (int v : chosen) cout << v << " ";
        cout << endl;
        return;
    }
    for (int i = start; i <= n; i++) {
        chosen.push_back(i);
        dfs(i + 1, cnt + 1);           // 下一层从 i+1 开始
        chosen.pop_back();             // 撤！
    }
}
```

这里的关键是 `dfs(i + 1, cnt + 1)`——`i+1` 保证了选的数字严格递增，天然去重。`start` 参数记录了"上一个选的是谁"，下一个必须从它后面开始。

同学们比较一下这两种写法。写法一是"对每个数决策"，写法二是"按位置选"。在组合问题中，写法二更常用，因为它天然去重而且效率更高。

不过写法一有它自己的适用场景。比如后面学 0-1 背包的时候，"选与不选"的决策方式就是最直接的建模。

排列和组合的区别，大家一定要搞清：排列关心顺序，n 个元素全排列有 n! 种；组合不关心顺序，n 选 k 有 C(n,k) = n!/(k!(n-k)!) 种。排列数通常比组合数大得多——5 选 3 的排列有 60 种，组合只有 10 种。

2. "从 1~5 中选 3 个数，用写法一的话，会不会生成 (1,2,3) 和 (2,1,3) 这样的重复？为什么？"
3. "如果要输出的是可重复组合（同一个数可以选多次），代码应该怎么改？"

## 🖥️ 板书/屏幕要点
- 排列 vs 组合：排列关心顺序，组合不关心
- 写法一（选与不选）：二叉树分叉，`chosen.push_back(x)` + dfs + `pop_back()`
- 写法二（按顺序选）：for 循环从 start 开始，`dfs(i+1, cnt+1)` 保证升序
- 组合数：C(n,k) = n!/(k!(n-k)!)

## ✏️ 随堂练习
1. 用写法一实现从 1~5 中选 3 个数的组合，验证输出 10 个
2. 用写法二实现同样的功能，对比输出是否一致
3. 尝试修改写法二，输出"可重复组合"（每个数可选多次），如 3 选 2 输出 (1,1),(1,2),(2,2)

## 📌 小结
组合和排列是回溯的两大基本类型。排列用 used 数组标记，组合用 start 参数保证递增。两种写法各有千秋——"选与不选"直观，"按顺序选"高效。下节课我们学最后一个回溯技巧——剪枝。

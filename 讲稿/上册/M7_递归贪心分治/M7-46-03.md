# 第46章 第3课时：lower_bound 与 upper_bound

## 📝 讲稿正文

今天我们学两个非常重要的概念：lower_bound 和 upper_bound。它们是二分查找的进阶用法，在竞赛中极其常用。

先搞清楚定义。对于升序数组 a，给定值 x：

- **lower_bound(x)**：返回第一个 **大于等于** x 的位置
- **upper_bound(x)**：返回第一个 **大于** x 的位置

"大于等于"和"大于"——就差一个等号，但区别很大。

举个具体例子。数组 {1, 2, 2, 2, 3, 5}，查 x=2：

lower_bound(2) 返回下标 1——那是第一个 2。upper_bound(2) 返回下标 4——那是第一个大于 2 的元素，也就是 3。

发现没有？**元素 2 的出现次数 = upper - lower = 4 - 1 = 3**。这个公式是竞赛中的常用技巧——不用遍历，直接用两次二分就算出某个值出现了几次。

还有一个细节：如果 x 比所有元素都大，lower_bound 和 upper_bound 都返回 n（数组长度），表示"找不到"。

好，我们来看怎么手写 lower_bound。关键的区别是：不像普通二分查找那样命中就返回，而是**命中后继续往左找**——因为你要找的是"第一个"大于等于 x 的位置。

```cpp
int lowerBound(int a[], int n, int x) {
    int l = 0, r = n - 1, ans = n;  // 默认 ans=n 表示找不到
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (a[mid] >= x) {
            ans = mid;               // 记录当前位置
            r = mid - 1;             // 往左看看有没有更靠前的
        } else {
            l = mid + 1;             // a[mid] < x，往右找
        }
    }
    return ans;
}
```

注意第 2 行：ans 初始化为 n。这是巧妙的——如果所有元素都小于 x，ans 保持 n，恰好表示"不存在"。

第 5 到 8 行是核心逻辑：当 `a[mid] >= x` 时，mid 是一个候选答案，记录下来，然后继续往左看还有没有更靠前的。当 `a[mid] < x` 时，mid 和它左边的都不满足条件，往右走。

如果你理解了 lower_bound，upper_bound 只需要改一个字符——把 `>=` 改成 `>`：

```cpp
if (a[mid] > x) {
    ans = mid;
    r = mid - 1;
} else {
    l = mid + 1;
}
```

这时候有同学可能要问：这些不是有现成的 STL 函数吗，为什么要手写？两个原因。第一，面试和考试中经常要求手写，你必须理解原理。第二，手写让你完全掌控行为——比如在二分答案中，你经常需要定制"查找条件"，那时候你就不是在找一个具体的值，而是在找"第一个满足条件的位置"，本质就是 lower_bound 的变形。

学懂了 lower_bound，二分查找就通关了一半。因为二分查找的本质就是在一个"单调序列"上找"临界点"——这正是 lower_bound 做的事。

## 🖥️ 板书/屏幕要点
- lower_bound：第一个 ≥ x 的位置
- upper_bound：第一个 > x 的位置
- 出现次数 = upper - lower
- 手写关键：命中后不直接返回，而是"记录 + 继续找"
- ans 初始化为 n（兜底值）

## ✏️ 随堂练习
1. 手写 lower_bound，在 {1, 3, 3, 3, 5, 7} 中查 3、4、8，验证返回的下标
2. 修改 lower_bound 为 upper_bound，只改一个字符，测试效果

## 📌 小结
lower_bound 和 upper_bound 是二分查找的"完全体"——它们不是简单的"等值查找"，而是"边界查找"。理解了它们，二分的思维就从"找一个值"升级为"找一个位置"。下节课我们学 STL 里的二分函数，让你的代码更简洁。

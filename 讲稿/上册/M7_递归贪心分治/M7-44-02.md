# 第44章 第2课时：回溯框架与全排列入门

**时长**：15-20分钟 | **对应教材**：第44章 44.3-44.4

## 🎯 本课时目标
- 理解"回溯"的核心思想：试一条路 → 不行就退 → 换下一条
- 掌握回溯"试—记—撤"三步曲
- 能用回溯框架生成 1~n 的全排列

## 📝 讲稿正文

上节课我们用递归解决了汉诺塔，那是"一条路走到黑"的单线递归——每次只有一个递归调用。但从今天开始，我们要进入递归更强大的用法：**回溯**。

什么叫回溯？想象你走迷宫。走到一个岔路口，有三条路。你选第一条走下去，走不通，退回来；再选第二条，走不通，退回来；选第三条，终于走通了。这个"试 → 退 → 换"的过程，就叫回溯。

在程序里，回溯就是**深度优先地遍历一棵"选择树"**。每个节点是一个"选择点"，每条从根到叶子的路径就是一个完整的方案。

今天我们用最经典的回溯问题来入门：**全排列**。问题很简单：输入 n，输出 1 到 n 的所有排列。比如 n=3，你要输出 6 种排列。

我们来画一下这棵"选择树"。填第一个位置时，你有 3 种选择：选 1、选 2、选 3。假设选了 1，填第二个位置时只剩 2 个选择：选 2 或选 3。再选了 2，第三个位置只能选 3——得到排列 123。然后退回去，第二个位置改选 3，第三个位置选 2——得到 132。

你看这个过程，有一个特点：**选了要记住，退回来要忘掉**。这就是回溯的"试—记—撤"三步曲。

**试**：判断当前这个选择能不能做。在全排列里，"试"就是检查这个数字有没有被用过。

**记**：做出选择，更新状态。把数字放进当前排列，标记它为"已使用"。

**撤**：递归返回后，撤销刚才的选择。把标记恢复为"未使用"。这一步特别关键——如果不撤，回到上一层时状态是脏的，另一条分支就会出错。

我们来看代码：

```cpp
int n;
int a[20];         // 存放当前排列
bool used[20];     // used[i]：数字 i 是否已被使用

void dfs(int pos) {
    if (pos > n) {                     // 边界：n个位置都填完了
        for (int i = 1; i <= n; i++)
            cout << a[i] << " ";
        cout << endl;
        return;
    }
    for (int i = 1; i <= n; i++) {     // "试"：枚举所有可能
        if (!used[i]) {                // 这个数字还没用过？
            a[pos] = i;                // "记"：放在当前位置
            used[i] = true;            // "记"：标记已使用
            dfs(pos + 1);              // 递归填下一个位置
            used[i] = false;           // "撤"：回溯！恢复状态
        }
    }
}
```

大家注意看第 16 行：`used[i] = false;`。这行如果删掉会发生什么？第一遍跑完之后，所有数字都被标记为"已使用"了，第二条分支什么数字都选不了，答案就只剩一组排列了。所以"撤"绝对不能省！

为什么叫"回溯"？因为递归返回的时候，程序会回到上一层的 for 循环中，继续尝试下一个 i——这就是"退回来，换一条路"。

回溯代码的结构其实很固定：一个 for 循环枚举选择 → 判断能不能选 → 选了，递归下一层 → 递归回来，撤销选择。以后你们写回溯题，基本上就是在这个框架里填具体的判断逻辑。

## 🖥️ 板书/屏幕要点
- 回溯 = 深度优先遍历选择树
- **试—记—撤** 三步曲：`if (!used[i])` → `a[pos]=i; used[i]=true;` → `used[i]=false;`
- 全排列代码框架：dfs(pos) + for枚举 + used标记
- n 个元素的全排列数 = n!

2. 故意删掉 `used[i] = false`，观察输出发生了什么变化，加深理解

## 📌 小结
回溯就是"试 → 记 → 撤"，用深度优先的方式遍历所有可能的选择路径。全排列是回溯的"Hello World"——用 used 数组标记哪些数字已选，填完所有位置就输出。记住：撤的那一步绝对不能省！下节课我们在全排列的基础上做一些变形和深入练习。

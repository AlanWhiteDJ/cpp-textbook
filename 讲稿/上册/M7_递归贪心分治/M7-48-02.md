## 📝 讲稿正文

上节课我们说了，贪心不是什么时候都对。但今天这两个例子，贪心是完全正确的，而且证明思路非常优美。

先看**最优装载**。有 n 个物品，重量分别是 w₁ 到 wₙ。一艘船载重为 C，物品不可拆分。问最多能装几件？

直觉告诉你：先装轻的，越轻装得越多。这就是贪心策略——按重量从小到大排序，一个个往上装，装到装不下为止。

```cpp
int w[1005];
sort(w, w + n);     // 从小到大排序
int cnt = 0, sum = 0;
for (int i = 0; i < n; i++) {
    if (sum + w[i] <= C) {
        sum += w[i];
        cnt++;
    } else break;
}
cout << cnt;
```

这个贪心为什么对？我们用**交换法**来证明。

假设存在一个最优解，它选了 k 件物品。如果它没有选当前最轻的那个物品 w_min，而是选了一个比 w_min 重的 w_heavy——那我们把 w_heavy 换成 w_min，总重量不增加（甚至减少），件数不变。所以这个新方案也是最优解。重复这个替换过程，最优解终将变成"先选最轻的"——贪心结果就是最优解。

交换法的套路：假设最优解和贪心解不同 → 找到第一个不同的位置 → 交换，证明不改变最优性 → 重复直到一致。

再看**排队接水**。n 个人排队，第 i 个人需要 tᵢ 时间接水。所有人的等待时间总和 = 每个人在开始接水前等待的时间之和。求最小的总等待时间。

直觉：接水快的人排前面，这样后面的人不用等太久。

```cpp
int t[1005];
sort(t, t + n);         // 按接水时间从小到大
long long wait = 0, total = 0;
for (int i = 0; i < n; i++) {
    total += wait;      // 当前这个人等了 wait 这么久
    wait += t[i];       // 下一个人要多等 t[i] 时间
}
cout << total;
```

为什么对？还是交换法。假设最优方案中有相邻的两个人 A 和 B，且 A 比 B 慢（t_A > t_B）。交换他们：A 后面的人等待时间不变（总共还是等了 t_A + t_B），但他们之间的等待时间变了。原先 A 在 B 前面，B 要等 t_A；交换后 B 在 A 前面，A 要等 t_B。由于 t_A > t_B，交换后总等待减少了。所以任何不是"快排前面"的方案都可以通过交换优化——最优方案必定是"快排前面"。

这道题还有一个经典版本叫"接水时间 × 重要性"，贪心策略就变成按 tᵢ/wᵢ 排序——这是"性价比"贪心在另一道题里的影子。不过今天我们先把基础打好。

最优装载和排队接水都展示了一个模式：**排序 + 贪心选择**。它们的特点是：存在一个明确的"越怎样越该选"的优先级，排序后按优先级依次选择即可。以后你遇到贪心题，第一个念头就应该是：我应该按什么排序？

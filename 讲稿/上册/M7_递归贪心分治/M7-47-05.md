## 📝 讲稿正文

今天是二分答案的最后一课。我们先解决**数列分段**问题，然后做全章总结。

问题：给定 n 个正整数，要分成连续的 m 段（每段非空），使得**每段和的最大值尽可能小**。求这个最小值。

关键词："最大值尽可能小"——"最大值最小化"，二分答案的经典信号。

我们二分"每段和的最大值"mid。check(mid) 判断：能否把这 n 个数分成不超过 m 段，使得每段之和 ≤ mid？

怎么判断？又是贪心！从头开始累加，当前段的和尽量塞满但不超过 mid。如果加上下一个数就超了，就在这个位置分段，从下一个数开始新的一段。最后看分成的段数是否 ≤ m：

```cpp
int a[100005];
int n, m;

bool check(int mid) {
    int segs = 1;         // 段数，至少 1 段
    int sum = 0;
    for (int i = 0; i < n; i++) {
        if (a[i] > mid) return false;  // 单个元素就超了，绝对不可行
        if (sum + a[i] > mid) {
            segs++;        // 新开一段
            sum = a[i];
        } else {
            sum += a[i];   // 继续往当前段加
        }
    }
    return segs <= m;
}
```

第 8 行是一个重要的防御：如果某个元素 a[i] 本身就大于 mid，那不管怎么分段，这一段都会超过 mid——直接返回 false。这也说明了 l 的下界应该是数组中的最大值（否则 check 永远失败）。

二分求最小值：

```cpp
int l = *max_element(a, a + n);  // l = 数组最大值
int r = accumulate(a, a + n, 0LL);  // r = 所有元素之和
int ans = r;
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (check(mid)) {
        ans = mid;
        r = mid - 1;     // 可行，试试更小的最大值
    } else {
        l = mid + 1;
    }
}
cout << ans;
```

数列分段把二分答案的两种方向都展示了：砍树是"求最大"，数列分段是"求最小"。放在一起对比，二分方向的区别就一目了然了。

好，第 47 章到这里全部结束。咱们来总结一下。

二分答案的核心在于：当答案具有单调性时，把"求答案"变成"猜答案+验证"。check 函数是灵魂——它决定了你的二分是否正确。

两类经典问题必须会：最大值最大化（砍树、木材加工）和最小值最小化（跳石头、数列分段）。区别只在可行时往哪边缩。

浮点二分用固定迭代次数，安全可靠。

最后送你一句话：看到"最大值最小"或"最小值最大"，立刻条件反射想到二分答案。这个技巧在竞赛里能帮你解决至少 20% 的中等难度题目。

## 📌 小结
二分答案的本质是"猜答案"——用 O(log 范围) 次 check 定位到答案的临界点。check 是核心竞争力，需要理解题意、保证单调性、注意数据范围。第 47 章结束，下章进入贪心算法——另一种充满智慧也充满陷阱的算法范式。

## 📝 讲稿正文

今天是 M7 模块的最后一课。我们先学倍增的另一个应用——**倍增查找**，然后展望 LCA，最后做全模块总结。

快速幂里，我们预处理了 a^(2^k)，查询时按 b 的二进制把对应份乘起来。同样的思想可以用在"跳步"问题上。

假设有一个很长的数组，你要多次查询"从位置 pos 开始往后走 steps 步，到达的位置"。steps 每次都不同且可能很大。笨办法是一次次加。

倍增做法：预处理一个二维数组 `go[i][k]`，含义是"从位置 i 跳 2^k 步后到达的位置"。然后查询时把 steps 二进制拆分，逐段跳。

```cpp
// 预处理
for (int i = 1; i <= n; i++) go[i][0] = i + 1;      // 跳1步
for (int k = 1; k <= 20; k++)
    for (int i = 1; i <= n; i++)
        go[i][k] = go[ go[i][k-1] ][k-1];  // 跳2^k = 2个2^(k-1)

// 查询：从 pos 跳 steps 步
int query(int pos, int steps) {
    for (int k = 0; steps > 0; k++, steps >>= 1)
        if (steps & 1) pos = go[pos][k];
    return pos;
}
```

注意预处理的递推公式：`go[i][k] = go[ go[i][k-1] ][k-1]`。"跳 2^k 步" = "先跳 2^(k-1) 步，到了某个位置，从那里再跳 2^(k-1) 步"。这和快速幂里 `a = a * a` 是同一个逻辑——翻倍！

这个预处理 O(n log V)，每次查询 O(log steps)。非常高效。

这个思想最经典的应用是**LCA（最近公共祖先）**。在树上，`up[u][k]` 表示"节点 u 往上跳 2^k 步到达的祖先节点"。预处理和查询的递推公式几乎一样：

```cpp
up[u][0] = parent[u];                               // 往上1步=父节点
up[u][k] = up[ up[u][k-1] ][k-1];                   // 往上2^k = 两个2^(k-1)
```

查询 LCA(u, v) 时：先用倍增把深的节点跳到和浅的同一深度，然后两个节点一起用倍增往上跳，直到它们的父节点相同。

你们现在不需要完全理解 LCA 的实现——那是 M9 或更后的内容。但"倍增预处理 + 二进制拆分查询"这个模式，你应该能识别了。快速幂、跳格子、LCA——本质都是这个模式。

好，让我们回到整个 M7 模块，做一个大总结。

这 7 章 30 节课，你们学了算法竞赛中最核心的几种思维范式：

**递归与回溯**（第44章）：递归三要素 → 汉诺塔 → 回溯"试—记—撤" → 全排列/组合 → 剪枝。回溯是枚举所有可能性的通用框架。

**记忆化搜索**（第45章）：算过就存 → memo 数组 → 标准框架 → 网格路径/数字三角形 → 记忆化到 DP 的桥梁。这是空间换时间的经典范例。

**二分查找**（第46章）：猜数字策略 → 左闭右闭/开 → lower_bound/upper_bound → STL → 旋转数组。每次砍一半，O(log n) 的威力。

**二分答案**（第47章）：猜答案 + check 验证 → 单调性 → 最大值最小/最小值最大 → 砍树/跳石头/数列分段。二分思维的"升维"应用。

**贪心算法**（第48章）：每步选局部最优 → 找零钱/最优装载/活动选择/排队接水 → 证明与反例。简单但需警惕。

**分治算法**（第49章）：分→治→合 → 归并排序/最大子段和 → 递归树/主定理。化整为零的智慧。

**倍增与快速幂**（第50章）：翻倍的力量 → 二进制拆分 → 快速幂/倍增查找 → LCA 预告。

这七种思维，加上 M6 的排序和优先队列，构建了你们算法竞赛的基础武器库。M8 开始，我们进入数据结构的世界——栈、队列、链表、vector……武器更锋利，战斗更精彩！

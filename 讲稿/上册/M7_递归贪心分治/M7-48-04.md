## 📝 讲稿正文

今天这节课特别重要——我要告诉你们贪心在哪里会"翻车"。知道什么不能贪心，比知道什么能贪心更重要。

**反例一：硬币找零——面值决定一切。**

面值 {1, 3, 4}，凑 6。贪心：4 + 1 + 1 = 3 枚。最优：3 + 3 = 2 枚。贪心贪了眼前最大的，却错过了整体最优。原因在于面值之间没有倍数关系——大面值不一定能用小面值完美替换。

**反例二：0-1 背包——价值密度骗了你。**

背包容量 10。物品 A：重量 6，价值 7；物品 B：重量 5，价值 5；物品 C：重量 5，价值 5。贪心按"性价比"：A 的性价比最高（7/6 ≈ 1.17），先拿 A。剩 4 容量，B 和 C 都拿不了。总价值 7。但最优是拿 B 和 C，总价值 10。贪心被"局部最优"骗了——选了高密度但"堵住"了更好的组合。

**反例三：最大子段和——不能只捡正数。**

数组 {3, -1, 4, -7, 10}。贪心说：从第一个正数开始累加，遇到负数就停。3 + (-1) + 4 = 6，停了。但继续加：6 + (-7) + 10 = 9 更大！贪心被中间的负数吓退了。

这三个反例教会我们：**用贪心之前，一定要先找反例。** 找到反例 = 贪心不适用 = 换 DP 或回溯。

但还有一类问题，贪心虽然对，策略却不是"显然的"。比如**合并果子**。

有 n 堆果子，每堆有 aᵢ 个。每次合并任意两堆，代价为两堆的果子数之和。问合并成一堆的最小总代价。

贪心策略：**每次选最小的两堆合并。** 为什么？因为小的堆如果"留到后面"会导致它们被反复累加多次，总代价更高。每步合并最小的两堆，让它们尽快消失。

```cpp
priority_queue<int, vector<int>, greater<int>> pq; // 小根堆
for (int i = 0; i < n; i++) { int x; cin >> x; pq.push(x); }
int ans = 0;
while (pq.size() > 1) {
    int a = pq.top(); pq.pop();
    int b = pq.top(); pq.pop();
    ans += a + b;
    pq.push(a + b);      // 合并后放回去
}
cout << ans;
```

这里用到了优先队列（堆），M6 学过的。优先队列能在 O(log n) 取出最小值——比每次排序高效得多。

好，第 48 章结束。总结一下：

贪心的特点是"每步选局部最优、永不回头"。它快（通常 O(n log n)），但不一定对。使用贪心要三步走：一，想清楚贪心策略是什么；二，脑测找反例；三，能证明再用。

经典贪心问题记牢：找零钱（面值决定）、最优装载（轻的优先）、活动选择（早结束优先）、排队接水（快的优先）。证明方法掌握交换法和反证法就够了。

贪心 vs DP：贪心是做决策时只选一个、不回头；DP 是考虑所有可能、用小问题推大问题。能用贪心解决的问题一定也能用 DP，但 DP 更慢。所以贪心是"DP 的特例"——当 DP 的最优选择恰好符合贪心策略时，DP 退化成了贪心。

## 🖥️ 板书/屏幕要点
- 三大反例：硬币（面值不标准）、0-1 背包（价值密度）、最大子段和（中间负数）
- 合并果子：最小两堆合并 + 优先队列
- 第48章总结：贪心 = 局部最优；贪心 ⊆ DP；使用前必找反例

## 📌 小结
贪心是把双刃剑——用对了快如闪电，用错了满盘皆输。核心能力不是"会用贪心"，而是"判断一个题能不能贪心"。记住三个反例、掌握两种证明方法、建立"先找反例"的习惯。下章我们进入分治算法——大问题切小、各自解决再拼起来。

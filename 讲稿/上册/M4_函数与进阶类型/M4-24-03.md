# 第24章 第3课时：const 引用——高效又安全的"只读模式"

**时长**：15-20分钟 | **对应教材**：第24章 24.3-24.4节

## 🎯 本课时目标
- 理解 `const &` 的三重好处：不复制、只读保护、能接受字面量
- 掌握三种传参方式的适用场景（值传递、引用传递、const引用）
- 能根据实际需求做出正确的传参方式选择

## 📝 讲稿正文

前两节课我们讲了值传递和引用传递两个极端：一个复制副本（安全但慢），一个直接操作原变量（快但危险）。有没有一种方式，又快又安全？

有，叫 **const 引用**。

先看场景。假设你写了个函数，功能是打印一个学生的信息。你只需要"看一眼"这个数据，绝不该修改它。

如果用值传递：`void show(string name, int age)`——每次调用都要把字符串 `name` 完整复制一遍。如果名字很短，"张三"两个字符，复制也无所谓。但如果字符串有几百个字符？每次复制成本就高了。

如果用引用传递：`void show(string &name, int &age)`——不复制，效率高。但万一函数里不小心写了 `name = "";`，调用者的数据就被抹掉了！

**const 引用就是为此而生的**——用 `&` 避免复制，用 `const` 锁住修改权限（板书）：

```cpp
void show(const string &name, const int &age) {
    cout << "姓名：" << name << "，年龄：" << age << endl;
    // name = "hacker";   // ❌ 编译错误！const 不允许修改
}
```

`const` 就像给数据上了一把锁——你能看、能用，就是不能改。试图修改，编译器直接报错，错误拦截在编译阶段而不是运行时。

**const 引用还有第三个好处：可以接受字面量。**

普通引用 `int &x` 不能绑定字面量——因为 42 这个数字没有独立的内存地址。但 `const int &x` 可以！编译器会临时创建一个匿名变量存 42，然后让 x 引用它：

```cpp
void print(const int &x) { cout << x << endl; }

int main() {
    print(42);       // ✅ const引用可以接字面量
    int a = 10;
    print(a);        // ✅ 也可以接变量
    return 0;
}
```

这在实际开发中非常方便——你写了一个函数，它既能接收变量，也能接收临时计算的结果、字面量，不用调用者自己先声明变量再传。

好了，现在咱们来做一个完整的**决策指南**。一个参数到底该用什么方式传？

**场景一**：参数是 `int`、`char`、`double` 这种小类型，且不需要修改原变量 → **值传递**。比如 `int square(int x) { return x*x; }`，复制一个 int 的成本微乎其微。

**场景二**：需要函数修改原变量的值 → **引用传递 `&`**。比如 swap、自增函数。"你要改原件，就得拿原件。"

**场景三**：参数是大对象（如长字符串、vector、结构体），但只读取不修改 → **const 引用 `const &`**。这是最高效且最安全的方案。

**场景四**：参数是数组 → 数组从不复制，它自动退化为指针传地址。这个后面专门讲。

来，咱们举一个综合例子：

```cpp
int square(int x) {                  // 小类型，不改 → 值传递
    return x * x;
}

void swap(int &a, int &b) {          // 要修改 → 引用传递
    int t = a; a = b; b = t;
}

void printReport(const string &name, // 大对象只读 → const引用
                 const int &score) {
    cout << name << " 的成绩：" << score << endl;
}
```

三个函数，三种传参方式，各得其所。

好，最后说一个常见的初学者困惑：**"const 引用到底防不防函数内部通过其他方式改？"** 答案是：const 只阻止通过**这个引用本身**去修改。如果函数内部还有另一个非 const 的指针指向同块内存，那照样能改。但正常使用中，const 引用已经足够安全了。

总结一下：`const &` = "我借来看一下，保证不弄坏，看完就还你。"

## 🖥️ 板书/屏幕要点
```
三种传参方式对比：
  值传递 (int x)          → 复制副本，改副本不影响原值
  引用传递 (int &x)       → 直接操作原变量
  const引用 (const int &x) → 不复制 + 只读保护 + 可接字面量

决策指南：
  不改 + 小类型 → 值传递
  要改 → 引用传递 (&)
  不改 + 大对象 → const引用

记忆：const & = "借来看，不弄坏，字面量也能接"
```

## 📌 小结
const 引用（`const T &`）兼具效率和安全：不复制数据（像引用），不允许修改（像 const），还能接受字面量。对于大对象只读场景是最佳选择。选择传参方式的口诀：小类型不改→值传递，要改→引用，大对象只读→const引用。

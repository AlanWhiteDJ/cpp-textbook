# 第28章 第3课时：nullptr 与 sizeof——空指针与指针大小

**时长**：15-20分钟 | **对应教材**：第28章 28.5-28.6节

## 🎯 本课时目标
- 理解 `nullptr` 的含义和使用场景——"暂时不指向任何东西"
- 养成解引用前检查空指针的安全习惯
- 理解 `sizeof(p)` 和 `sizeof(*p)` 的区别

## 📝 讲稿正文

上节课我们强调了"指针必须先指向有效变量才能解引用"。那如果一个指针暂时还没确定要指向谁呢？总不能让它装着垃圾地址吧。

C++ 给了一个标准答案：**`nullptr`**——空指针，表示"我目前不指向任何东西"。

来看用法（板书）：

```cpp
int* p = nullptr;   // p 明确表示：我现在不指向任何变量

// 使用前先检查
if (p != nullptr) {
    cout << *p << endl;    // 安全：确认了非空才解引用
} else {
    cout << "p 是空指针，不能解引用" << endl;
}
```

这个 `if` 检查太重要了——它是防止程序崩溃的"护身符"。如果你用 `nullptr` 解引用：

```cpp
int* p = nullptr;
cout << *p;     // ❌ 运行时崩溃！不能解引用空指针
```

程序直接崩——在竞赛里就是 Runtime Error，在应用里就是闪退。

**为什么要用 nullptr？三个原因：**

第一，**文档作用**——告诉读代码的人（包括自己）："这个指针目前还没指向有效数据，别急着用"。

第二，**安全检查**——你可以在使用前用 `if (p != nullptr)` 判断，从而避免崩溃。

第三，**调试利器**——很多调试器对空指针解引用有特殊处理，能快速定位问题。而垃圾地址解引用后可能产生随机行为，更难排查。

**好习惯：声明指针时要么指向变量，要么赋 nullptr。** 不初始化是给自己埋地雷。

```cpp
int* p;             // ❌ 坏习惯：垃圾地址
int* q = nullptr;   // ✅ 好习惯：明确表示"空"
int* r = &someVar;  // ✅ 好习惯：明确指向
```

---

**现在来说 sizeof——指针本身占多大？**

指针也是一个变量，自然也占内存。但奇怪的是——**不管指向什么类型，指针变量本身的大小是固定的**（板书）：

```cpp
cout << sizeof(int*) << endl;     // 8（64位系统）或 4（32位系统）
cout << sizeof(char*) << endl;    // 和 int* 一样大！
cout << sizeof(double*) << endl;  // 和 int* 一样大！
cout << sizeof(Student*) << endl; // 和 int* 一样大！
```

为什么？因为指针只是存地址的——不管你指向的是 int（4 字节）还是 Student（几十字节），指针只是记一个门牌号。门牌号纸条的大小，和你进了房间能看到多大的房间，是两回事。

`sizeof(p)` 和 `sizeof(*p)` 的含义完全不同（板书）：

```cpp
int x = 10;
int* p = &x;

cout << sizeof(p) << endl;   // 指针本身的大小：8 字节（64位系统）
cout << sizeof(*p) << endl;  // 指向内容的大小：4 字节（int）
```

画个类比：

- `sizeof(p)` = 通讯录里一行的宽度（写地址的地方，固定大小）
- `sizeof(*p)` = 小明家的实际面积（不同人可能不一样）
- `sizeof(int*)` = 所有通讯录行都一样宽
- `sizeof(int)` = 但不同人的家面积不同

所以：
- `p` 是地址 → `sizeof(p)` = 8（64位指针固定 8 字节）
- `*p` 是 int → `sizeof(*p)` = 4（int 是 4 字节）
- `int*` 是指针类型 → `sizeof(int*)` = 8
- `Student*` 也是指针类型 → `sizeof(Student*)` = 也是 8

记住这个对比，它能帮你区分"指针"和"指针指向的东西"——这是理解指针的关键。

好了，总结今天的两个知识点：**nullptr 让指针"合法地空着"**，解引用前先检查保平安；**sizeof 帮你看清指针（8 字节）和指向数据（大小各异）的区别。**

## 💬 互动提问
1. 如果不用 nullptr，一个未初始化的指针里是什么值？直接解引用会怎样？
2. 为什么所有类型的指针（int*、double*、struct*）大小都一样？
3. `int* p;` 中，`sizeof(p)`、`sizeof(*p)`、`sizeof(int*)` 分别是多少？

## 🖥️ 板书/屏幕要点
```
nullptr：空指针
  int* p = nullptr;     // 明确表示"不指向任何东西"
  if (p != nullptr) {   // 解引用前必须检查！
      *p = 42;
  }

sizeof：
  sizeof(p)  = 指针本身大小（64位=8字节）
  sizeof(*p) = 指向内容的大小（int=4, double=8...）

类比：
  p 是门牌号纸条 → 都一样大
  *p 是房间 → 大小各异
```

## ✏️ 随堂练习
1. 声明一个 `int* p = nullptr;`，用 if 判断后输出"指针为空"。然后让它指向一个变量，再次判断。
2. 分别输出 `sizeof(int*), sizeof(double*), sizeof(char*)`，比较结果。
3. 声明 `double x = 3.14; double* p = &x;`，输出 `sizeof(p)` 和 `sizeof(*p)`。

## 📌 小结
nullptr 是空指针，表示"暂时不指向任何变量"。解引用前用 `if (p != nullptr)` 检查，避免程序崩溃——这是好习惯。sizeof(p) 返回指针本身大小（所有类型指针一样大，64 位系统 8 字节），sizeof(*p) 返回指向对象的大小（取决于类型）。记住"门牌号纸条 vs 房间"的类比。

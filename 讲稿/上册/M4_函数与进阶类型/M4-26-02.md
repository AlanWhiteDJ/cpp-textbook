## 📝 讲稿正文

上节课我们抽象地讲了递归三要素。今天用最经典的例子——**阶乘**——把递归的执行过程扒开来，看得清清楚楚。

先回顾代码（板书）：

```cpp
int factorial(int n) {
    if (n == 1) {                // 边界条件
        return 1;
    }
    return n * factorial(n - 1); // 递推 + 递归调用
}

int main() {
    cout << factorial(5) << endl;  // 输出 120
    return 0;
}
```

看起来只有 4 行代码，但背后发生的事情并不简单。咱们来**手动模拟 `factorial(4)`**。

当你调用 `factorial(4)` 时，计算机做了什么？

第一步：进函数，n=4。不是 1，所以走到 `return 4 * factorial(3)`。

等等——`factorial(3)` 还没算出来呢！计算机只能先把 "4 * ???" 记在脑子里，然后跑去算 `factorial(3)`。

第二步：进函数，n=3。不是 1，走到 `return 3 * factorial(2)`。同样，`factorial(2)` 还没算——记下 "3 * ???"，跑去找 `factorial(2)`。

第三步：n=2，`return 2 * factorial(1)`。记下 "2 * ???"，跑去找 `factorial(1)`。

第四步：n=1！终于碰到边界条件了！**直接返回 1**。不再递归。

好，现在开始"回家"——从最深处一层层往外返回：

- `factorial(1)` 返回 1 给 `factorial(2)` → 2 × 1 = 2
- `factorial(2)` 返回 2 给 `factorial(3)` → 3 × 2 = 6
- `factorial(3)` 返回 6 给 `factorial(4)` → 4 × 6 = 24
- `factorial(4)` 返回 24 给 main

整个过程就像剥洋葱——剥进去 4 层，然后一层层拼回来。这就是"递"（进去）和"归"（回来）两个阶段。

画成图（板书）：

```
factorial(4)                          = 24
  └─ 4 * factorial(3)         = 4 × 6 = 24
          └─ 3 * factorial(2) = 3 × 2 = 6
                  └─ 2 * factorial(1) = 2 × 1 = 2
                          └─ 1        (边界！)
```

**关键细节**：注意最后一步！`factorial(1)` 返回 1 后，递推才真正开始执行。在那之前，所有的 `n *` 都在等待——记着"我要乘以谁"，但还不能算，因为乘数还没出来。这是递归最微妙的地方。

**容易犯的错误——边界条件写错了**：

有同学写 `if (n == 0) return 1;`——如果 n 从 1 开始，且每次减 1，那 n 会经过 1 到 0。如果边界是 `n == 1`，那 n=1 时返回 1，OK。但如果有人调用 `factorial(0)`，边界条件 `n==1` 碰不到——因为 0 直接跳到 -1、-2……无限递归！

所以写边界时要想清楚：**参数变化的方向最终能不能碰到边界？**

更好的写法是 `if (n <= 1) return 1;` —— 用 `<=` 而不是 `==`，更安全，涵盖了 n=0 甚至负数的情况。

现在咱们来对比递归和循环两种写法（板书）：

```cpp
// 递归版——自顶向下，优雅
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

// 循环版——自底向上，高效
int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

递归版更贴近数学定义——n! = n × (n-1)!。循环版从头开始算——1×2×3×...×n。两种都能算对，但思路相反。

好了，现在大家应该能独立写出阶乘递归了。关键不是背代码，而是理清"层进层出"的过程。建议你拿出纸笔，手工模拟 `factorial(4)`——如果能把每一步都画清楚，递归你就入门了。

## 🖥️ 板书/屏幕要点
```
factorial(4) 执行过程：
  递：4*f(3) → 3*f(2) → 2*f(1) → 1(边界！)
  归：1 → 2*1=2 → 3*2=6 → 4*6=24

关键：
  所有 n* 等待里面返回才真正执行乘法
  边界条件建议用 <= 而不是 == （更安全）

递归 vs 循环（阶乘）：
  递归：自顶向下 n! = n × (n-1)!
  循环：自底向上 1×2×3×...×n
```

## 📌 小结
阶乘是递归的 Hello World。递进去（不断调用自己 n→n-1→...→1），碰到边界就归回来（1→2→6→24...）。所有未完成的乘法在"归"阶段执行。边界条件建议用 `<=` 而非 `==`，防止意外输入导致无限递归。递归和循环都是阶乘的合法实现，思路互补。

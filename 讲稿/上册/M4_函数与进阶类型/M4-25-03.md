# 第25章 第3课时：static 局部变量——活过函数返回的"长寿变量"

**时长**：15-20分钟 | **对应教材**：第25章 25.4节

## 🎯 本课时目标
- 理解 `static` 局部变量的特殊行为：只初始化一次、函数返回后值保留
- 掌握 static 变量的内存位置（全局/静态区而非栈）
- 能写出利用 static 记录跨调用状态的实际代码

## 📝 讲稿正文

前面我们一直说：局部变量，函数调用时诞生，返回时死亡，下次调用重新生一个——像一个短命的临时工。

但如果你想让局部变量"长寿"——函数返回后它还活着，下次调用还能记住上次的值——怎么办？

加一个关键字：**`static`**。

来看例子（板书）：

```cpp
void counter() {
    static int count = 0;       // static：只初始化这一次
    count++;
    cout << "第 " << count << " 次调用" << endl;
}

int main() {
    counter();    // 输出：第 1 次调用
    counter();    // 输出：第 2 次调用  ← count 保留了上次的 1 ！
    counter();    // 输出：第 3 次调用  ← count 保留了上次的 2 ！
    return 0;
}
```

注意看！如果是普通局部变量，每次都会输出"第 1 次调用"——因为 `count = 0` 每次都会重新执行。但加了 `static`，`count` 只初始化一次，之后每次调用都在上一次的值上 +1。

**为什么？** static 变量不住在栈上——它住在**全局/静态区**。和全局变量住同一栋楼！所以它不会被函数的栈回收干掉。函数返回时栈帧销毁了，但 static 变量安安稳稳待在全局区里。

**做个对比实验**把 `static` 去掉（板书）：

```cpp
void counter() {
    int count = 0;              // 普通局部变量
    count++;
    cout << "第 " << count << " 次调用" << endl;
}
```

运行三次——每次都输出"第 1 次调用"。因为每次进入函数，`count` 都在栈上重新分配、重新初始化为 0。

好，现在来理解 static 的一个关键细节：**初始化只发生在第一次执行到这行代码时**。注意——不是程序启动时，而是运行时第一次走到这一行。而且只执行一次。

来看综合演示（板书）：

```cpp
int globalVar = 1;                    // 全局

void test() {
    int localVar = 2;                 // 普通局部
    static int staticVar = 3;         // static局部

    cout << "globalVar=" << globalVar
         << " localVar=" << localVar
         << " staticVar=" << staticVar << endl;

    globalVar++;   localVar++;   staticVar++;
}

int main() {
    test();  // globalVar=1 localVar=2 staticVar=3
    test();  // globalVar=2 localVar=2 staticVar=4
    test();  // globalVar=3 localVar=2 staticVar=5
    return 0;
}
```

观察三条曲线：

- `globalVar`：每次 +1（全局，一直被保留）
- `localVar`：每次都是 2（普通局部，每次重新创建）
- `staticVar`：每次 +1（static 局部，值被保留）

建议把这段代码敲一遍跑一遍，三条线的变化非常直观。

**static 在实战中怎么用？**

最常见的用途是**记录函数被调用的次数**。比如你写了一个 `bool isPrime(int n)` 函数，你想统计它在整个程序运行中被调用了多少次——在函数第一行加个 `static int callCount = 0; callCount++;` 就行。不需要额外的全局变量，干净利落。

另外一个用途——**保持跨调用的状态**。比如一个随机数生成器，需要记住上一次生成的"种子"。又比如一个简单的缓存机制——上次计算的结果先存着，如果下次参数相同直接返回。

**但也要注意**：static 变量的"长寿"也是一把双刃剑。它让函数有了"记忆"，但也让函数的输出不再只依赖于输入——同样的参数调用两次，结果可能不一样。这让函数变得"不那么纯粹"，调试时要多留意。

总结：`static` 让局部变量从"临时工"变"终身员工"——只上岗一次，一直干到程序结束。

## 🖥️ 板书/屏幕要点
```
static 局部变量：
  static int count = 0;  → 只初始化一次
  → 函数返回后值保留（不消失）
  → 下次调用时继续使用上次的值
  → 住在全局/静态区（不在栈上）

vs 普通局部变量：
  普通：每次调用 重新创建 → 重新初始化 → 返回就死
  static：首次调用创建+初始化 → 返回不死 → 下次保留

实战用途：
  记录函数调用次数
  保持跨调用状态
```

2. 写函数 `int nextID()`，每次返回一个递增的编号（1, 2, 3...）。用 static 实现。
3. 下面代码输出什么？为什么？
```cpp
void f() { static int x = 0; x++; cout << x << endl; }
void g() { static int x = 100; x++; cout << x << endl; }
int main() { f(); f(); g(); g(); f(); }
```

## 📌 小结
`static` 局部变量不随函数返回而销毁——它住在全局/静态区，只初始化一次。这让局部变量有了"记忆"，可以记录函数调用次数、保持跨调用状态。和普通局部变量的关键区别：普通局部每次重新创建，static 终身保留。

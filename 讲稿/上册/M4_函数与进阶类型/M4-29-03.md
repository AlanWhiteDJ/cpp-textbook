# 第29章 第3课时：ofstream——向文件写入数据

**时长**：15-20分钟 | **对应教材**：第29章 29.3-29.4节

## 🎯 本课时目标
- 掌握 `ofstream` 的完整使用流程：打开→检查→写入→关闭
- 理解覆盖模式与追加模式（`ios::app`）的区别
- 了解 `fstream` 的读写合一模式

## 📝 讲稿正文

上节课学了从文件**读**，今天学往文件**写**——让程序真正拥有"存档"功能。

写文件用 `ofstream`（output file stream）。它和 `cout` 语法几乎一样，只是输出目标从屏幕换成了文件。

来看完整流程（板书）：

```cpp
#include <fstream>
#include <iostream>
using namespace std;

int main() {
    ofstream fout("result.txt");  // ① 打开文件（默认覆盖模式）

    if (!fout.is_open()) {        // ② 检查是否创建/打开成功
        cout << "无法创建文件！" << endl;
        return 1;
    }

    fout << "Hello, file!" << endl;   // ③ 写入——和 cout 一模一样！
    fout << "这是第二行" << endl;

    for (int i = 1; i <= 10; i++) {
        fout << i << " ";              // 写入数字
    }
    fout << endl;

    fout.close();                       // ④ 关闭文件
    cout << "写入完成！" << endl;       // 这个输出到屏幕
    return 0;
}
```

注意看：`fout <<` 和 `cout <<` 一模一样的语法！只是前面不是 `cout` 而是 `fout`（你起的文件流变量名）。这非常方便——你不需要学一套新语法。

而且，屏幕输出和文件输出可以**同时进行**！`cout` 往屏幕打，`fout` 往文件写，互不干扰。这就是 ofstream 比 freopen 灵活的地方。

---

**重点：覆盖模式 vs 追加模式。**

当你用 `ofstream fout("result.txt")` 打开文件时，默认行为是——如果文件已存在，**清空旧内容**从头写。这有时候是你想要的（比如重新生成结果），有时候不是。

如果你想把新内容**追加到文件末尾**，保留旧内容，要用追加模式：

```cpp
ofstream fout("log.txt", ios::app);  // app = append（追加）
```

对比两种模式（板书）：

```cpp
// 覆盖模式（默认）——每次运行都是全新的文件
ofstream fout("data.txt");
fout << "第一行" << endl;
// 第一次运行后：data.txt 内容 = "第一行"
// 第二次运行后：data.txt 内容 = "第一行" （旧内容被清空，重新写）

// 追加模式——新内容接在旧内容后面
ofstream fout("log.txt", ios::app);
fout << "新的一行" << endl;
// 第一次运行后：log.txt 内容 = "新的一行"
// 第二次运行后：log.txt 内容 = "新的一行\n新的一行"
```

**什么时候用追加模式？** 日志文件——每次运行记录一条新日志，不覆盖旧日志。就像写日记，每天写新的一页，不撕掉旧的。

---

**顺便提一下 fstream——既能读又能写。**

`fstream` 是 `ifstream` 和 `ofstream` 的合体（板书）：

```cpp
#include <fstream>
fstream file("data.txt", ios::in | ios::out);  // 可读可写
```

实际中用得比较少——大多数场景下，要么只读（ifstream），要么只写（ofstream），读写同一个文件的情况不常见。了解它存在就行。

---

**一个重要提醒：写完文件要 close()。**

`fout.close()` 确保数据真正从缓冲区写入硬盘。虽然 C++ 在文件流对象析构时（比如离开作用域）会自动关闭，但显式关闭能让你的意图更清晰，也避免忘了关导致的数据丢失。

总结：ofstream 让写文件和 cout 一样简单。覆盖模式（默认）清空重写，追加模式（`ios::app`）保留旧内容。写入后别忘了 close()。

## 🖥️ 板书/屏幕要点
```
ofstream 四步流程：
  ① 打开：ofstream fout("文件名");
  ② 检查：if (!fout.is_open()) { 错误处理; }
  ③ 写入：fout << 内容; （和 cout 语法一样）
  ④ 关闭：fout.close();

打开模式：
  默认（覆盖）：ofstream fout("a.txt");
  追加模式：  ofstream fout("a.txt", ios::app);

fstream：可读可写（较少用）
  fstream file("a.txt", ios::in | ios::out);

⚠️ 写入后务必 close()！
```

## ✏️ 随堂练习
1. 用 ofstream 创建 `output.txt`，写入 1 到 100（每行一个数）。
2. 写一个"日志程序"，每次运行时用追加模式在 `log.txt` 末尾追加一行当前时间和一条消息。
3. 先用覆盖模式写一个文件，再用追加模式在同一文件末尾加内容。打开文件查看效果。

## 📌 小结
ofstream 用于向文件写入数据，语法与 cout 一样（`fout <<`）。四步流程：打开→检查→写入→关闭。默认覆盖模式会清空旧内容，用 `ios::app` 可切换为追加模式（保留旧内容，新内容接在末尾）。fstream 可读可写，但实际使用较少。写完文件后务必 close() 确保数据落盘。

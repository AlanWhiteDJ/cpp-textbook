# 第23章 第4课时：值传递与引用传递初探——复印件 vs 原件

**时长**：15-20分钟 | **对应教材**：第23章 23.4节

## 🎯 本课时目标
- 理解值传递的本质——函数拿到的是参数的副本
- 理解引用传递（`&`）的本质——函数直接操作原变量
- 能写出正确的 `swap` 交换函数并解释原理

## 📝 讲稿正文

好，今天这节课非常重要——它可能是整个函数模块里最容易搞混、也最常考的一个知识点。我们来讲**值传递和引用传递**。

先讲个生活中的例子。小明和小红是同桌，互相检查作业：

方案 A：小明把作业**复印**了一份给小红。小红在复印件上写写画画——原件一点没变。这是**值传递**。

方案 B：小明直接把**原件**递给小红。小红在上面乱画——原件面目全非。这是**引用传递**。

在 C++ 里，每当你调用函数传参数，你都在做类似的选择。

首先看**值传递**——C++ 的默认方式。来看代码（板书）：

```cpp
void addTen(int x) {
    x = x + 10;                  // 修改参数
    cout << "函数内：" << x << endl;  // 输出 30
}

int main() {
    int a = 20;
    addTen(a);
    cout << "函数外：" << a << endl;  // 输出 20 —— 没变！
    return 0;
}
```

啊？函数里明明把 x 改成了 30，为什么外面 a 还是 20？

因为 `addTen(a)` 这个调用发生时，计算机做了一件事——它把 `a` 的值 20 **复制了一份**，放到参数 `x` 的内存空间里。函数里面改的是 `x` 这个副本，原件 `a` 在另一个内存位置上，纹丝不动。

就像一个游戏存档。你复制了一份发给朋友，朋友在他的电脑上玩了三天三夜——你自己的存档还是老样子。

好，那么如果我就想让函数改掉外面的变量呢？**加一个 `&`**——这叫引用传递（板书）：

```cpp
void addTen(int &x) {            // 注意这个 &
    x = x + 10;
    cout << "函数内：" << x << endl;  // 30
}

int main() {
    int a = 20;
    addTen(a);
    cout << "函数外：" << a << endl;  // 30 —— 真的变了！
    return 0;
}
```

`&` 读作"引用"。它的意思是：参数 `x` 不是副本——它是 `a` 的**别名**。`x` 和 `a` 是同一个变量、同一块内存，只是穿了不同的马甲。你对 `x` 做任何操作，就等于对 `a` 做操作。

再来看一个经典例子——**交换两个变量的值**。假如你要写一个 `swap` 函数：

```cpp
// ❌ 错误写法——值传递，交换不了
void swapWrong(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}
// 调用 swapWrong(x, y) 后 → x 和 y 的值纹丝不动！为什么？因为 a、b 是副本。

// ✅ 正确写法——引用传递
void swapRight(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}
// 调用 swapRight(x, y) 后 → x 和 y 真正交换了！
```

`swap` 这个例子是全 C++ 教学史上最能说明值传递和引用传递区别的例子。很多同学第一次写 swap 都是值传递版本，然后发现怎么交换都没用，百思不得其解。现在你知道答案了——**要加 `&`**。

好，最后来个对比总结：

值传递 = "我给你复印件，你在复印件上随便画" → 安全，但不影响原件。

引用传递 = "我给你原件，你直接改原件" → 能真正修改外部变量，但要小心别改错了。

什么时候用引用？两个场景：一是你要修改传入的参数（像 swap）；二是传大对象时避免复制（这个后面详细讲）。

好了，现在大家动手试试，把这个 swap 写得滚瓜烂熟——因为后面学排序、学数据结构，这个 swap 会反复出现。记住：**如果写的 swap 不能交换——检查有没有 `&`！**

## 🖥️ 板书/屏幕要点
```
值传递（默认）：
  void f(int x)  →  x 是实参的副本
  → 修改 x 不影响原变量

引用传递（&）：
  void f(int &x)  →  x 是实参的别名（同一块内存）
  → 修改 x 就是修改原变量

经典案例——swap：
  ✅ void swap(int &a, int &b) { ... }
  ❌ void swap(int a, int b)   { ... }
```

```cpp
void swap(int a, int b) { int t=a; a=b; b=t; }
int main() { int x=3,y=5; swap(x,y); cout<<x<<y; }
```
3. 写一个 `void resetToZero(int &x)` 函数，把参数改成 0。测试效果。

## 📌 小结
值传递是 C++ 默认方式——函数拿到副本，修改不影响原变量。引用传递加 `&`——函数拿到原变量的别名，修改直接影响外部。swap 函数必须用引用传递。记住口诀："值传递是复印件，引用传递是原件。要改原变量，加个 `&` 就行。"

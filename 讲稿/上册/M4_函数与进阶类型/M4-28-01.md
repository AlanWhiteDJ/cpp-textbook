# 第28章 第1课时：内存地址与指针变量——数据住在哪里

**时长**：15-20分钟 | **对应教材**：第28章 28.1-28.2节

## 🎯 本课时目标
- 理解内存地址的概念——每个变量都有唯一的"门牌号"
- 学会用 `&` 获取变量地址
- 掌握指针变量的声明和初始化语法

## 📝 讲稿正文

同学们，今天我们要学 C++ 里最让人"谈虎色变"但学会后又觉得"不过如此"的知识——**指针**。

先别紧张。指针没那么可怕。它本质上就是一个简单的东西——**存地址的变量**。

想象你有一本通讯录，里面写着"小明——阳光小区 3 号楼 502 室"。现在你要去找小明，有两种办法：
- 办法 A：站在楼下喊"小明！"
- 办法 B：翻开通讯录，找到地址，敲门

办法 A 虽然简单，但有限制——你只能在"能喊到小明的地方"叫他。办法 B 灵活多了——只要有地址，不管小明在哪，你都能找到。

指针就是编程里的"通讯录"——它不存数据本身，它存的是**数据在内存里的地址**。

**首先，我们来看什么是内存地址。**

计算机的内存像一栋巨大的公寓楼，里面有几十亿个房间。每个房间能存 1 字节的数据，每个房间都有唯一的门牌号——这就是**地址**。

当你写 `int x = 42;` 时：
1. 计算机找到 4 个连续的空房间（int 占 4 字节）
2. 在这 4 个房间里放进 42
3. 把第一个房间的门牌号记为"x 的地址"

C++ 用 `&` 运算符可以查看这个地址（注意：这里的 `&` 和引用传递的 `&` 虽然长得一样，但用法不同——取地址的 `&` 放在变量前面，是单目运算符；引用的 `&` 放在类型后面）：

```cpp
int x = 42;
cout << "x 的值是：" << x << endl;
cout << "x 的地址是：" << &x << endl;  // 输出类似 0x7ffd4a3b6c2c
```

运行后你会看到一个十六进制数，比如 `0x7ffd4a3b6c2c`——这就是 x 在内存里的门牌号。每次运行程序，地址可能不一样（操作系统每次分配的内存位置不同），这很正常。

**好，既然地址也是一个值，那能不能用一个变量存它？**

当然可以。这种专门存地址的变量，就叫**指针变量**，简称**指针**（板书）：

```cpp
int x = 42;
int* p = &x;    // p 是一个指针，存着 x 的地址
```

拆开看：
- `int*`——"这是一个指针类型，它指向的东西是 int"
- `p`——指针变量的名字
- `= &x`——把 x 的地址存进 p

可以这样理解：
- `int` 类型的变量 → 存整数
- `double` 类型的变量 → 存小数
- `int*` 类型的变量 → 存"int 变量的地址"

声明指针有三种写法，本质上一样，选一种你喜欢的保持一致就行：

```cpp
int* p;    // 星号紧挨 int——强调"int* 是一种类型" ← 本书推荐
int *p;    // 星号紧挨变量名——强调"p 是指针"
int * p;   // 星号在中间
```

**重点：指针也要有类型。** `int*` 指向 int，`double*` 指向 double，`char*` 指向 char。类型不匹配会编译报错：

```cpp
double pi = 3.14;
int* p = &pi;   // ❌ 错误！int* 不能指向 double
```

为什么必须类型匹配？因为不同类型占的字节数不同——int 占 4 字节，double 占 8 字节。指针需要知道自己指向的东西占多大，才能正确地读写。

**还有个非常重要的安全规则——声明指针时尽量初始化：**

```cpp
int* p;          // ⚠️ 危险！p 里是一个垃圾地址，不知道指向哪
int* q = nullptr; // ✅ 安全！明确表示"暂时不指向任何东西"
```

未初始化的指针里是随机垃圾值——可能指向操作系统的核心区域。如果你不小心用了它，程序可能崩溃或产生诡异行为。所以养成习惯：**声明指针时，要么指向一个有效变量，要么初始化为 nullptr。**

好了，今天先到这里。记住核心：**指针 = 存地址的变量。`&` 取地址，`int*` 声明指针。** 下节课我们学怎么通过地址去访问数据——解引用 `*`。

## 🖥️ 板书/屏幕要点
```
地址：变量在内存中的"门牌号"
  &x  → 取 x 的地址（输出十六进制数）

指针：专门存地址的变量
  int* p = &x;    // p 存着 x 的地址
    
声明语法：
  int* p;   // 指向 int 的指针
  double* q; // 指向 double 的指针
  
规则：
  • 类型必须匹配（int* → int, double* → double）
  • 声明时最好初始化（给地址或 nullptr）
```

## ✏️ 随堂练习
1. 声明几个不同类型的变量（int、double、char），打印它们的值和地址。
2. 分别声明 `int*`、`double*`、`char*` 指针，让它们指向对应的变量，打印指针的值（存的就是地址）。
3. 连续声明几个 int 变量，打印地址，观察它们是否连续。

## 📌 小结
内存中每个变量都有唯一地址（门牌号）。`&` 运算符取地址。指针是存地址的变量，用 `类型*` 声明（如 `int* p = &x;`）。指针有类型要求，必须和指向的变量类型匹配。指针声明时务必初始化——要么指向有效变量，要么设为 `nullptr`。

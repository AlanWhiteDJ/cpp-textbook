# 第28章 第2课时：解引用——顺着地址找到数据

**时长**：15-20分钟 | **对应教材**：第28章 28.3-28.4节

## 🎯 本课时目标
- 掌握 `*` 解引用运算符——通过指针读写指向的数据
- 区分 `*` 在声明时（类型标记）和使用时（解引用）的不同含义
- 能画出指针→变量的内存指向关系图

## 📝 讲稿正文

上节课我们学会了用 `int* p = &x;` 让指针存地址。但光存地址有什么用？我们还需要**通过地址去操作数据**——这才是指针的真本事。

顺着指针里存着的地址，找到那个变量，读写它的值——这个动作叫**解引用（Dereference）**，运算符是 `*`。

注意：这个 `*` 和声明指针时的 `*` 长得一样，但含义完全不同：

| 位置 | 含义 | 示例 |
|------|------|------|
| 声明时 | "这是一个指针类型" | `int* p = &x;` |
| 使用时 | "去 p 指向的地方看看" | `cout << *p;` |

声明时是"自我介绍"，使用时是"行动"。就像"班长"这个词——自我介绍时说"我是班长"（身份），发号施令时说"班长命令你"（行动）。

来看具体怎么用（板书）：

```cpp
int x = 42;
int* p = &x;      // p 存着 x 的地址

cout << *p << endl;  // 输出 42 —— 顺着地址找到了 x 的值

*p = 100;            // 顺着地址找到 x，把它的值改成 100
cout << x << endl;   // 输出 100 —— x 真的被改了！
```

看到了吗？`*p` 就是 **"p 指向的那个变量"**。对 `*p` 做任何操作，等价于对 `x` 做操作——因为它们就是同一个东西。

画成图非常直观（板书）：

```
内存示意图：

变量 x（地址 0x1000）： ┌──────────┐
                        │   42     │
                        └──────────┘
                             ↑
指针 p（地址 0x2000）： ┌──────────┐
                        │ 0x1000   │──────┘
                        └──────────┘

p 存着 0x1000（x 的地址）
*p 就是 42（x 的值）
```

这张图是理解指针的万能钥匙。以后你遇到任何指针问题，**第一反应就是画这张图**。指针指向谁？地址是多少？解引用后是什么？——图上全讲清楚了。

**现在来看一段最经典的指针操作序列：**

```cpp
int a = 10, b = 20;
int* p = &a;       // ① p 指向 a

cout << *p;        // ② 通过 p 读 a → 输出 10
*p = 30;           // ③ 通过 p 改 a → a 变成 30

p = &b;            // ④ p 改指向 b（p 可以"变心"！）
cout << *p;        // ⑤ 通过 p 读 b → 输出 20

p = nullptr;       // ⑥ p 谁也不指向了
```

关键点：
- `*p` 读写的是 p 当前指向的变量——p 指向谁，`*p` 就是谁
- p 可以随时改变指向（④），就像通讯录里可以改地址
- 别解引用空指针（⑥之后再用 `*p` 会崩溃）

**解引用的重要安全规则——黄金法则：**

**指针必须先指向一个有效的变量，才能解引用。**

```cpp
int* p;            // ❌ 危险：p 没初始化，是垃圾地址
*p = 42;           // ❌ 往垃圾地址写数据——程序崩溃！

int* q = nullptr;
*q = 10;           // ❌ 往空地址写数据——立即崩溃！

int x;
int* r = &x;       // ✅ 先指向有效变量
*r = 10;           // ✅ 安全！
```

记住这条规则，你就避免了指针 90% 的运行时崩溃。

好，最后来一个实际应用——**用指针实现 swap**。这在第 24 章我们用了引用（`&`），现在用指针再做一遍：

```cpp
void swap(int* p, int* q) {
    int temp = *p;   // *p 就是第一个变量
    *p = *q;         // 交换值
    *q = temp;
}

int main() {
    int x = 10, y = 20;
    swap(&x, &y);    // 传地址
    cout << x << " " << y;  // 20 10
    return 0;
}
```

指针版 swap：调用时传 `&x, &y`（地址），函数里用 `*p, *q`（解引用）操作原变量。效果和引用版一模一样，只是语法不同。

## 🖥️ 板书/屏幕要点
```
解引用（*）：
  *p  → "p 指向的那个变量"
  cout << *p;  // 读
  *p = 100;    // 写

* 的双重身份：
  声明时 int* p  → "p 是指针类型"
  使用时 *p      → "去 p 指向的地方"

内存关系图（画指针→变量的箭头）：
  p ──→ [ x = 42 ]

黄金法则：指针必须先指向有效变量，才能解引用！
```

## 📌 小结
解引用 `*p` 就是顺着指针找到原变量并操作它。`*` 在声明时是类型标记（"我是指针"），在使用时是解引用（"去你指向的地方"）。画指针→变量的内存关系图是理解指针的万能方法。黄金法则：解引用前必须确保指针指向有效变量（不是 nullptr 也不是垃圾地址）。

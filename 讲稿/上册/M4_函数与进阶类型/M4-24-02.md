# 第24章 第2课时：引用传递深入——直接操作原变量的魔法

**时长**：15-20分钟 | **对应教材**：第24章 24.2节

## 🎯 本课时目标
- 从内存地址层面理解引用传递——形参和实参是同一块内存
- 掌握引用传递的语法和调用规则
- 能正确写出多个返回值、修改原数据的引用函数

## 📝 讲稿正文

上节课我们用地址证明了值传递的副本机制——两个地址不一样。今天我们来破解另一个谜题：加个 `&` 到底发生了什么？

同样的实验，这次加上 `&`（板书）：

```cpp
#include <iostream>
using namespace std;

void tryChange(int &x) {         // 注意这个 &
    cout << "函数内：x 的地址 = " << &x << endl;
    x = 999;
}

int main() {
    int a = 10;
    cout << "main里：a 的地址 = " << &a << endl;
    tryChange(a);
    cout << "调用后 a = " << a << endl;   // 999 —— 真的变了！
    return 0;
}
```

运行看看：

```
main里：a 的地址 = 0x7ffd1a3b4c1c
函数内：x 的地址 = 0x7ffd1a3b4c1c
调用后 a = 999
```

看到了吗？**两个地址一模一样！** `a` 和 `x` 住在同一个房间！

这就是引用传递的本质——`x`不是新创建的变量，它只是**`a` 的另一个名字**。就像你有个同学叫"李华"，同学都叫他"华仔"——两个名字，指的同一个人。你对"华仔"说的话，就是对着李华本人说的。

画成图（板书）：

```
调用时：  a / x ─── [10]  （同一个房间！名字不同而已）

x = 999:  a / x ─── [999] （直接改了这个房间里的东西）
```

就一行代码的区别——`int` 后面加不加 `&`——但效果天差地别。

现在我们来彻底搞定 swap（板书）：

```cpp
// ✅ 正确版 —— 引用传递
void swap(int &a, int &b) {
    int temp = a;     // temp 拿到的是 a 指向的那个真实变量
    a = b;            // 直接改写原变量
    b = temp;         // 直接改写原变量
}

int main() {
    int x = 10, y = 20;
    swap(x, y);
    cout << x << " " << y << endl;   // 20 10 ✓
    return 0;
}
```

建议你把值传递版和引用传递版各敲一遍，跑一遍，感受那个差异——**没错，就一个 `&` 的区别，一个行一个不行。**

**引用传递的三个重要语法点：**

第一，调用时不需要额外写 `&`。定义时写了 `int &x`，调用时直接 `func(a)`，别写成 `func(&a)`——`func(&a)` 那是传地址，是下一章指针的语法，别搞混。

第二，引用必须绑定到一个**实际存在的变量**。你不能 `func(5)`——数字 5 没有地址，没法引用。编译会报错。

第三，引用的效率很高。如果参数是一个很大的东西——比如一个长字符串、一个大结构体——值传递要复制整个东西，引用传递只传了一个"快捷方式"，几乎零成本。

**实战：多个返回值。**

函数只能 `return` 一个值——这是 C++ 的规矩。但有时候你就需要"返回"两个东西，怎么办？用引用！

```cpp
void minmax(int a, int b, int &mn, int &mx) {
    if (a < b) { mn = a; mx = b; }
    else      { mn = b; mx = a; }
}

int main() {
    int small, big;
    minmax(15, 8, small, big);
    cout << "min=" << small << ", max=" << big << endl;  // min=8, max=15
    return 0;
}
```

你看，`small` 和 `big` 在 main 里声明，通过引用传进 `minmax`，函数在里面给它们赋了值，出来就能用了——实现了"多个返回值"的效果。

好了，总结一下引用传递：**`&` 让形参变成实参的别名，共享同一块内存。修改形参就是修改实参。** 它是你实现 swap、"多返回值"、高效传大对象的利器。

## 🖥️ 板书/屏幕要点
```
引用传递（&）本质：
  形参不是副本，是实参的别名
  共享同一块内存 → 地址一样

语法：
  定义：void f(int &x)  ← & 在类型后面
  调用：f(a)            ← 不写 &，直接传变量名
  不能传字面量：f(5) ❌

经典：void swap(int &a, int &b) { ... }

多返回值技巧：
  用引用参数"带出"额外结果
```

## 📌 小结
引用传递让形参成为实参的别名——地址相同、直接操作原变量。swap 必须用引用传递。定义时在类型后加 `&`，调用时直接传变量名。引用传递还能实现"多返回值"——把想返回的额外结果通过引用参数带出去。注意引用不能绑定字面量（除非加 const）。

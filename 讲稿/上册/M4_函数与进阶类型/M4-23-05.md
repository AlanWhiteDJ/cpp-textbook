# 第23章 第5课时：局部变量、全局变量与作用域——谁的领地谁做主

## 📝 讲稿正文

前面几节课我们把函数的"外壳"搞清楚了——怎么定义、怎么传参。今天来看一个更细致的问题：变量到底"活"在哪里？

想象你在学校。每个班有一个班长——三班的班长只能管三班的事，跑去五班发号施令，没人理他。编程里的变量也一样：**定义在哪个 `{}` 里，就只能在那对 `{}` 里用。**

我们先区分两种变量（板书）：

**局部变量**：定义在函数内部或代码块内部，只在这个函数/代码块里有效。

**全局变量**：定义在所有函数外面（和 `main()` 平级），从定义处到文件末尾所有函数都能访问。

来看例子：

```cpp
#include <iostream>
using namespace std;

int gold = 100;           // ← 这是全局变量，全文件可见

void spend() {
    int price = 30;       // ← 这是局部变量，只属于 spend()
    gold -= price;        //  访问全局变量 OK
}

int main() {
    int wallet = 50;      // ← 局部变量，只属于 main()
    gold += wallet;       //  访问全局变量 OK
    spend();
    cout << gold << endl; // 输出 120
    // cout << price;     // ❌ 编译错误！price 只在 spend() 里活着
    return 0;
}
```

注意看：`gold` 定义在所有函数外面，所以 `main()` 和 `spend()` 都能访问它、修改它。`wallet` 定义在 `main()` 里面，只有 `main()` 能用。`price` 定义在 `spend()` 里面，只有 `spend()` 能用。

如果你在 `main()` 里试图用 `price`——编译器直接报错："我不认识 price"。因为出了 `spend()` 的大括号，`price` 就已经"死亡"了。

**现在我们深入理解作用域规则——`{}` 就是边界。**

不只是函数的大括号，`if`、`for`、`while`，甚至你手动加的裸 `{}`，都是作用域的"围墙"：

```cpp
int main() {
    int x = 10;                  // x 的领地：整个 main()

    if (x > 5) {
        int y = 20;              // y 的领地：这个 if 的花括号
        cout << x << " " << y;   // ✅ x 在外层，y 在内层，都能访问
    }                            // ← y 死在这里

    cout << x << endl;           // ✅ x 还活着
    // cout << y;                // ❌ y 已经死了
    return 0;
}
```

记住一句话：**内层可以访问外层的变量，但外层访问不到内层的变量。** 就像你可以从窗户看到外面的大街，但大街上看不到你房间里的细节。

再来看一个容易踩坑的地方——**同名变量屏蔽**：

```cpp
int x = 100;                     // 全局

int main() {
    cout << x << endl;           // 100（全局的）

    int x = 200;                 // 局部 x 屏蔽了全局 x
    cout << x << endl;           // 200（局部的）

    {
        int x = 300;             // 更内层又屏蔽了一层
        cout << x << endl;       // 300
    }                            // 300的x死亡

    cout << x << endl;           // 200（200的x重新露面）
    return 0;
}
```

这叫"近水楼台先得月"——内层同名变量会暂时遮住外层的。出了内层的 `}`，外层的又"重见天日"。

如果你想在被屏蔽的内层访问全局变量，用 `::`（两个冒号）：

```cpp
int value = 10;
int main() {
    int value = 20;
    cout << value << endl;     // 20（局部）
    cout << ::value << endl;   // 10（全局，::代表全局作用域）
    return 0;
}
```

虽然 `::` 能让全局变量"重见天日"，但同名屏蔽本身是一种代码坏味道——读代码的人（包括三个月后的你自己）很容易搞混。

**最后说说全局变量的问题。** 它看起来很方便——放那儿谁都能用。但有三个大风险：
1. **谁都能改**——出了 bug，你找不到是谁改坏的
2. **函数不独立**——依赖全局变量的函数不能单独复用
3. **命名冲突**——多个文件里不能定义同名全局变量

所以记住一个原则：**变量的活动范围越小越好。** 能放块里不放函数里，能放函数里不放全局。这是整个编程界的共识。

## 🖥️ 板书/屏幕要点
```
局部变量：函数/块内定义 → {} 内有效 → 出了{}就死
全局变量：函数外定义 → 全文件可见 → 慎用！

作用域规则：
  {} = 边界
  内层可访问外层  外层不可访问内层
  同名 → 内层屏蔽外层（:: 可突破）

最佳实践：变量的活动范围越小越好
```

## 📌 小结
局部变量定义在函数/块内，出了 `{}` 就死。全局变量在外面，谁都能访问——但方便带来风险。`{}` 就是作用域围墙，内层看得到外层，反过来不行。同名时内层屏蔽外层。养成良好的"局部优先"习惯——变量越"宅"越安全。

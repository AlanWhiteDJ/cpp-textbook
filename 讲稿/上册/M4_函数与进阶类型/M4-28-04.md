## 📝 讲稿正文

好，今天是我们指针章节的最后一课——把所有知识点串起来，形成完整的"指针操作流"，再把最常见的坑指给你看。

先来看**指针的六大核心操作**——这就是指针的全部"招式"（板书）：

```cpp
int a = 10, b = 20;
int* p = &a;       // ① 让 p 指向 a（存地址）

cout << *p;        // ② 通过 p 读取 a 的值 → 10
*p = 30;           // ③ 通过 p 修改 a 的值 → a 变成 30

p = &b;            // ④ 让 p 改指向 b（p 可以"变心"）
cout << *p;        // ⑤ 现在 *p 是 b 的值 → 20

p = nullptr;       // ⑥ p 清空，不指向任何东西
```

六大操作对应六句代码，记住这个"标准流程"你就掌握了指针的全部基本操作。核心就四个字：**存地址，访数据。**

---

现在说说**四大易错陷阱**——这些坑几乎每个初学者都踩过：

**陷阱一：未初始化就解引用。** 声明 `int* p;` 但没赋值，p 里是垃圾地址。然后直接 `*p = 42;`——程序崩溃或产生诡异行为。✅ 牢记：声明指针时要么指向变量，要么赋 nullptr。

**陷阱二：混淆声明 `*` 和使用 `*`。** 有同学以为 `int* p; *p = 10;` 就是给一个叫 `*p` 的变量赋值——错了！`*p` 是去 p 指向的地方。如果 p 没初始化，这就是往垃圾地址写数据。✅ 声明是声明，使用是使用，两个不同的 `*`。

**陷阱三：不同类型指针混用。** `double x = 3.14; int* p = &x;`——编译错误！`int*` 不能指向 `double`。因为 int 和 double 的二进制表示完全不同，用 int* 读 double 的数据会得到垃圾。✅ 类型必须匹配。

**陷阱四：解引用空指针。** `int* p = nullptr; cout << *p;`——运行时瞬间崩溃。✅ 解引用前先判断 `if (p != nullptr)`。

---

好，陷阱说完了，来两个实战练习巩固。

**实战一：用指针实现多返回值（板书）：**

```cpp
void calc(int a, int b, int* sum, int* diff, int* prod) {
    *sum = a + b;
    *diff = a - b;
    *prod = a * b;
}

int main() {
    int s, d, p;
    calc(10, 5, &s, &d, &p);
    cout << "和=" << s << ", 差=" << d << ", 积=" << p << endl;
    return 0;
}
```

函数只能 return 一个值——但通过指针参数，你可以"返回"任意多个结果。调用时传 `&s`（地址），函数里用 `*sum`（解引用）写值。这就是指针的实用价值。

**实战二：返回较大值的指针（板书）：**

```cpp
int* findMax(int* a, int* b) {
    if (*a > *b) return a;    // 返回指向较大值的指针
    else         return b;
}

int main() {
    int x = 10, y = 20;
    int* pMax = findMax(&x, &y);
    cout << "最大值：" << *pMax << endl;  // 20
    *pMax = 0;   // 把较大的那个改成 0
    cout << x << " " << y << endl;        // 10 0
    return 0;
}
```

这个函数返回的是指针——指向较大的那个变量。拿到指针后，你可以读它，也可以直接通过指针修改原变量的值。这就是指针的灵活之处。

最后，本章六字真言送给大家：**"存地址，访数据"** 。所有指针操作都在这六个字里。记住它，以后遇到指针就不慌了。

## 📌 小结
指针六大操作：指向、读取、修改、改指向、空指针、解引用检查。四大陷阱：未初始化、混淆*号、类型不匹配、解引用空指针。六字真言"存地址，访数据"就是指针的全部。指针可以实现多返回值、返回最大值的指针等高级技巧——本质上都是通过地址间接操作数据。

## 📝 讲稿正文

同学们，今天我们要学编程世界里最优雅、也最让人"又爱又恨"的思想——**递归**。

什么是递归？一句话：**函数自己调用自己。**

听起来不可思议——"左脚踩右脚能飞上天"？？但在数学和计算机的世界里，这不仅可能，而且是解决一大类问题的最优雅武器。

我们先建立直觉。

你见过俄罗斯套娃吗？一个大娃娃，打开里面有一个小一点但长得一模一样的娃娃，再打开又是一个更小的娃娃……一直开到最小的那个，它打不开了——那就是"终点"。

两面镜子面对面——镜子里有镜子，镜子里的镜子里还有镜子……一层套一层，越来越小，直到你看不见。

递归就是这样——**一个大问题里面藏着一个结构相同但规模更小的问题。** 你不断往里拆，拆到最小的问题可以直接解决（打不开了），然后一层层往回拼。

**举个例子——计算 5 的阶乘 5! = 5×4×3×2×1。**

你想：我要是知道 4! 就好了——5! = 5 × 4!。那 4! 呢？4! = 4 × 3!。3! = 3 × 2!。2! = 2 × 1!。最后 1! = 1——不用再拆了！然后往回算：1! = 1 → 2! = 2 → 3! = 6 → 4! = 24 → 5! = 120。

这里蕴涵了递归的三个要素。任何递归函数都必须包含这三样东西，缺一不可（板书）：

| 要素 | 说明 | 阶乘的例子 |
|------|------|-----------|
| **边界条件** | 什么时候停止，直接返回答案 | n == 1 时，返回 1 |
| **递推关系** | 大问题怎么用小问题的答案算 | n! = n × (n-1)! |
| **递归调用** | 调用自己，且参数在缩小 | factorial(n-1) |

**三要素缺一个会怎样？**

缺边界条件 → 无限递归，栈溢出，程序崩溃：

```cpp
int f(int n) { return n * f(n-1); }  // ❌ 没有 if 来终止！
```

缺递推关系 → 参数没缩小，永远到不了边界：

```cpp
int f(int n) { if (n <= 1) return 1; return f(n); }  // ❌ 参数没变！
```

缺递归调用 → 那就不是递归了，是普通函数。

**三个都齐了才是正确的递归**（板书）：

```cpp
int factorial(int n) {
    if (n == 1) {                // ① 边界条件
        return 1;
    }
    return n * factorial(n - 1); // ② 递推关系 + ③ 递归调用
}
```

我们来手动模拟一下 `factorial(3)` 的执行过程（这是理解递归的关键步骤，跟着我的板书走）：

```
factorial(3)
  → 3 * factorial(2)
       → 2 * factorial(1)
            → 1                          ← 碰到边界！开始返回
       → 2 * 1 = 2                       ← 返回 2
  → 3 * 2 = 6                            ← 返回 6
```

就像剥洋葱——一层层剥进去（调用），剥到最中心，再一层层拼回来（返回）。每一步都在等"里面那层"的结果。

注意一个关键点：**每次递归调用，参数必须在缩小**——`factorial(n-1)` 比 `factorial(n)` 的 n 小了 1。这样才能保证最终一定会碰到边界条件 `n == 1`。

如果参数不缩小——比如你写 `factorial(n)` 调的还是自己——那 n 永远是 3，永远碰不到 `n == 1`，永远停不下来。这就是无限递归。

好，现在让大家自己总结一下：**边界条件是什么？** ——就是让递归停下来的那个"最小问题"。就像俄罗斯套娃最里面那个打不开的小娃娃。"我已经到终点了，不需要再拆了。"

**递推关系是什么？** ——就是"大问题怎么用小问题表示"。5! = 5 × 4!，F(n) = F(n-1) + F(n-2)，等等。

**递归调用是什么？** ——就是"调用自己，参数变小"。factorial(n) 调 factorial(n-1)，从 5 到 4 到 3……一路往下。

三要素全了，递归就写对了。下节课我们用最经典的阶乘代码来深入剖析执行细节。

## 🖥️ 板书/屏幕要点
```
递归 = 函数自己调用自己

递归三要素（缺一不可）：
  ① 边界条件 → 什么时候停止
  ② 递推关系 → 大问题怎么拆成小问题
  ③ 递归调用 → 调用自己，参数必须缩小

阶乘例子：
  边界：n == 1 → return 1
  递推：n! = n × (n-1)!
  调用：factorial(n-1)

错误示范：
  ❌ int f(int n) { return n * f(n-1); }     // 缺边界
  ❌ int f(int n) { if(n<=1) return 1; return f(n); } // 参数不缩小
```

## 📌 小结
递归是函数调用自己的编程技巧，核心是把大问题拆成同类型的小问题。三要素缺一不可：边界条件（停止递归的底线）、递推关系（大问题用小问题表示）、递归调用（调用自己且参数缩小）。缺边界→无限递归崩溃，参数不缩小→永远到不了边界。

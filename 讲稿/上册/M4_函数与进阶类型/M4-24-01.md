# 第24章 第1课时：值传递深入——复印件的底层真相

**时长**：15-20分钟 | **对应教材**：第24章 24.1节

## 🎯 本课时目标
- 从内存地址层面理解值传递的本质——形参和实参是两个不同变量
- 能通过打印地址来验证值传递的行为
- 牢固掌握"值传递不影响原变量"的核心认知

## 📝 讲稿正文

上节课我们打了个比方——值传递就是"把作业复印件给别人，原件不变"。今天我们把比喻变成看得见的证据——**用地址来证明**。

计算机的内存就像一栋巨大的公寓楼，每间房有一个门牌号——地址。当你写 `int a = 10;`，计算机找个房间，把 10 放进去，房间号就是 `a` 的地址。

C++ 里用 `&`（取地址运算符）可以看这个地址。咱们来做个实验（板书）：

```cpp
#include <iostream>
using namespace std;

void tryChange(int x) {
    cout << "函数内：x 的地址 = " << &x << endl;   // 看x住哪儿
    x = 999;                                       // 在x的房间放999
}

int main() {
    int a = 10;
    cout << "main里：a 的地址 = " << &a << endl;    // 看a住哪儿
    tryChange(a);
    cout << "调用后 a = " << a << endl;             // 10 —— 没变！
    return 0;
}
```

运行一下，你会看到类似这样的输出：

```
main里：a 的地址 = 0x7ffd1a3b4c1c
函数内：x 的地址 = 0x7ffd1a3b4bec
调用后 a = 10
```

注意看！**两个地址完全不一样！** `a` 住在 `4c1c` 号房，`x` 住在 `4bec` 号房。`x = 999` 只是把 `4bec` 号房里的东西换成了 999，`4c1c` 号房里的 10 纹丝不动。

这就是值传递的底层真相——**函数调用时，计算机会为新参数 `x` 分配一块新内存，然后把实参 `a` 的值原样抄过去。** 从此 `a` 和 `x` 是两个独立的变量，各过各的。

画成图就是这样（板书画内存模型）：

```
调用前：  a ─── [10]  （住 0x4c1c 号房）

调用时：  a ─── [10]  （住 0x4c1c 号房）← 原变量，不改
          x ─── [10]  （住 0x4bec 号房）← 副本，改这里

x = 999:  a ─── [10]  （不变！）
          x ─── [999] （副本被改了，但原件没事）
```

这张图我建议你们每个人都自己画一遍。以后只要遇到"为什么函数里改了外面没变"的问题——画这张图，答案一目了然。

好，来看一个更刺激的例子——**swap 为什么不能用值传递**：

```cpp
void swapWrong(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    cout << "函数内：a=" << a << " b=" << b << endl;  // a=20, b=10 ✓
}

int main() {
    int x = 10, y = 20;
    swapWrong(x, y);
    cout << "main内：x=" << x << " y=" << y << endl;  // x=10, y=20 ✗
    return 0;
}
```

你看，函数里确实交换了！但交换的是 `a` 和 `b` 这两个**副本**。`x` 和 `y` 在它们自己的房间里好好的，没人动过。

那值传递什么时候用呢？答案是——**大部分时候都在用**。因为大部分函数我们只需要"借用"一下值来计算，不需要修改原变量。对于 `int`、`char`、`double` 这些小类型数据——复制一份的成本微乎其微，而且值传递最安全，函数里怎么折腾都不会影响外部。就像你去餐厅点菜——服务员拿菜单（副本）去厨房，你的原件（纸质菜单）还留在桌上。

总结一句话：**值传递 = 复制一份过去，两清。函数内部怎么改，原变量纹丝不动。**

下节课我们来看怎么打破这个规则——加上神秘的 `&`，让函数直接操作原变量。

## 🖥️ 板书/屏幕要点
```
值传递的本质：
  调用函数时 → 为形参分配新内存 → 复制实参的值
  → 形参和实参是两个独立变量，地址不同

验证方法：用 & 打印地址
  cout << &实参;   // 外面的地址
  cout << &形参;   // 里面的地址 → 不一样！

内存模型（值传递）：
  a ─── [10]   （原件，不动）
  x ─── [10] → [999]  （副本，可以乱改）
```

## 📌 小结
值传递时，形参是实参的全新副本，分配在不同内存地址。用 `&` 打印地址可以直观看到它们住不同房间。函数里对形参的任何修改都不会影响实参。对于不需要修改原变量的小型数据，值传递是默认且安全的选择。

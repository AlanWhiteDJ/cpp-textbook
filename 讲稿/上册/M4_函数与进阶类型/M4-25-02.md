# 第25章 第2课时：块作用域与同名屏蔽——大括号就是围墙

**时长**：15-20分钟 | **对应教材**：第25章 25.2-25.3节

## 🎯 本课时目标
- 牢固掌握"`{}` 就是作用域边界"的核心规则
- 理解内层可访问外层、外层不可访问内层的规则
- 掌握同名变量屏蔽规则和 `::` 突破屏蔽的方法

## 📝 讲稿正文

上节课我们说了全局和局部——那是"大格局"。今天把镜头拉近，看"小格局"——**块作用域**。

C++ 的作用域规则非常简单：**大括号 `{}` 就是一道墙。** 变量在定义它的那对 `{}` 内有效，出了 `}` 就消失。

不只是函数的花括号，`if`、`for`、`while`，甚至你随手加的裸 `{}`，都是一堵墙。来看（板书）：

```cpp
int main() {
    int x = 10;                  // x 的领地：整个 main 的 {}

    if (x > 5) {
        int y = 20;              // y 的领地：if 的 {}
        cout << x << " " << y;   // ✅ x 在外层，y 在内层，都能访问
    }                            // ← y 死亡

    cout << x << endl;           // ✅ x 活着
    // cout << y << endl;        // ❌ y 已死，编译错误
    return 0;
}
```

记住一条黄金规则：**内层可以访问外层的变量，但外层不能访问内层的。** 你可以从窗户看到外面的街道，但街上的人看不到你房间里的细节。

再来看 for 循环——这是块作用域最典型的应用（板书）：

```cpp
for (int i = 0; i < 10; i++) {   // i 的作用域：整个 for
    cout << i << " ";
}
// cout << i;                     // ❌ i 已死
```

你肯定在写代码时遇到过这种情况——想在循环外面用 `i`，编译器说"未声明"。不是编译器傻，是 `i` 真的已经死了。出了 for 的花括号，`i` 就不存在了。

这个设计其实很聪明——循环变量只在需要它的地方活着，循环结束自动释放。不会在后面的代码里被意外使用。

**好，现在说同名变量的屏蔽规则。**

如果外层和内层定义了同名的变量，会发生什么？——**内层屏蔽外层**。也叫"近水楼台先得月"（板书）：

```cpp
int x = 100;                     // 外层（全局）

int main() {
    cout << x << endl;           // 100 —— 全局的 x

    int x = 200;                 // 局部 x 屏蔽了全局 x
    cout << x << endl;           // 200 —— 局部的 x

    {
        int x = 300;             // 更内层又屏蔽了一层
        cout << x << endl;       // 300
    }                            // 300的x死亡

    cout << x << endl;           // 200 —— 200的x重新露面
    return 0;
}
```

就像俄罗斯套娃——你打开最外层娃娃，看到第二层娃娃。第二层娃娃把你视线挡住了，你看不到最外层的。等你合上第二层，最外层的又露出来了。

**那如果我就是要在内层访问全局变量呢？** 用 `::`——作用域解析运算符：

```cpp
int value = 10;

int main() {
    int value = 20;
    cout << value << endl;       // 20（局部）
    cout << ::value << endl;     // 10（全局！）
    return 0;
}
```

`::` 前面什么都不写，就代表"全局作用域"。`::value` 意思是"给我找全局的那个 value，不要被局部的挡住"。

虽然 `::` 能在内层"穿透屏蔽"，但说实话——如果你的代码需要频繁用 `::`，那说明你的命名方式有问题。同名变量会让人头晕：看到 `value`，到底是全局的还是局部的？每次都要翻到定义处才能确定——这不是找麻烦嘛。

**所以给你的建议是：尽量避免同名。** 全局变量和局部变量取不同的名字，代码可读性会好很多。如果真的必须同名（比如一个通用的 `count`），至少保持清醒——知道内层会屏蔽外层。

总结：作用域就是 `{}` 围墙。内可见外，外不见内。同名时"近的优先"，远的被挡住。`::` 可以突破屏蔽，但最好别依赖它。

## 🖥️ 板书/屏幕要点
```
块作用域规则：
  {} = 边界
  内层能访问外层变量
  外层不能访问内层变量
  for(int i...) → i 只在循环内有效

同名屏蔽：
  内层同名变量 → 暂时遮住外层
  出内层 → 外层重新露面
  ::变量名 → 强制访问全局版本

建议：避免同名！
```

## 📌 小结
C++ 作用域由 `{}` 决定——函数、if、for、while、裸花括号都是边界。内层可访问外层变量，反过来不行。同名变量内层屏蔽外层（近的优先）。`::` 可突破屏蔽访问全局，但频繁使用说明命名有问题——建议避免同名。

# 第26章 第4课时：递归调用栈——层层叠放的"待办事项"

**时长**：15-20分钟 | **对应教材**：第26章 26.4节

## 🎯 本课时目标
- 理解调用栈的概念——每次函数调用在栈上分配一帧
- 能画出递归调用栈的压栈和弹栈过程
- 理解栈溢出的原因，建立递归深度的安全意识

## 📝 讲稿正文

前面我们用"剥洋葱"、"套娃"来比喻递归。今天打开计算机的"天灵盖"，看看递归到底在内存里干了什么。

回忆一下第 25 章说的：局部变量住在**栈（Stack）**上。每次调用一个函数，计算机会在栈上分配一块空间——叫**栈帧（Stack Frame）**——用来存这个函数的参数、局部变量和"返回地址"（谁调的我，我算完了回哪去）。

函数返回时，这块空间就被回收——像搭帐篷，用完就拆。

**递归调用就是不断地在栈上叠栈帧**。我们以 factorial(3) 为例（板书）：

```
调用 factorial(3)：  栈顶 → [n=3, 返回地址=main]
调用 factorial(2)：  栈顶 → [n=2, 返回地址=factorial(3)]
调用 factorial(1)：  栈顶 → [n=1, 返回地址=factorial(2)]
                               ↑ 栈顶

碰到边界，开始返回 ↓

factorial(1)返回1：  栈顶弹出 [n=1]  → 回到 factorial(2)
factorial(2)返回2：  栈顶弹出 [n=2]  → 回到 factorial(3)
factorial(3)返回6：  栈顶弹出 [n=3]  → 回到 main
```

每进入一层递归，栈就高一截。每返回一层，栈就矮一截。栈顶永远是最内层、正在执行的那个函数。

如果递归层次太多，栈一直涨一直涨……栈空间是有限的（通常几 MB 到几十 MB）。涨到顶了还不停——**栈溢出（Stack Overflow）**，程序直接崩溃。

这就像你往桌上摞盘子——摞 10 个没问题，摞 100 个摇摇欲坠，摞 10000 个？桌子垮了。

**斐波那契的栈有多深？**

fib(5) 调用 fib(4) 再调用 fib(3)……最深的链是 fib(5)→fib(4)→fib(3)→fib(2)→fib(1)，深度大约是 n。也就是说 fib(10000) 最深的递归链有 10000 层栈帧——大概率栈溢出。虽然斐波那契的瓶颈主要是时间复杂度，但深度问题同样值得警惕。

**阶乘呢？** factorial(100000) —— 10 万层栈帧，妥妥栈溢出。尽管每次只调一个子函数（链形），但深度太大。

**那递归还能用吗？** 当然能！关键是**控制深度**：
- 如果题目 n ≤ 1000，递归安全
- 如果 n 可能到 10^6，考虑用循环（递推）
- 有些递归（如归并排序）深度是 O(log n)，即使 n=10^9 也只有约 30 层——稳得很

另外，C++ 有一个技巧叫**尾递归优化**。如果递归调用是函数的**最后一步**（return 后面直接就是递归调用，不掺杂其他运算），编译器可能把它优化成循环，不消耗栈空间。比如：

```cpp
int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);    // 尾递归：最后一步就是递归调用
}
```

但阶乘不是尾递归——`return n * factorial(n-1)` 里有个 `n *`，递归返回后还得做乘法。所以阶乘不能被优化成循环。

好了，理解调用栈是理解递归的"硬核"方式。你不用每次写递归都去想栈，但遇到"为什么递归卡死了"的问题时，脑子里要能浮现这张栈图——帧一层层叠上去，直到溢出崩溃。

## 🖥️ 板书/屏幕要点
```
调用栈（Call Stack）：
  每次函数调用 → 分配栈帧（参数+局部变量+返回地址）
  函数返回 → 释放栈帧

factorial(3) 的栈变化：
  压栈：f(3) → f(2) → f(1)  （栈越来越高）
  弹栈：f(1)→1  f(2)→2  f(3)→6  （栈越来越矮）

栈溢出（Stack Overflow）：
  递归太深 → 栈帧超限 → 崩溃
  通常深度 > 几千~几万就危险

尾递归：递归是函数最后一步 → 编译器可优化为循环
  例：gcd(a,b){ return gcd(b, a%b); } ✅ 尾递归
  例：factorial(n){ return n*f(n-1); } ❌ 不是尾递归
```

```cpp
int f(int n) {

}
```

## 📌 小结
调用栈是递归的底层机制——每层递归占用一个栈帧，层层叠放。深度太大时栈空间耗尽导致栈溢出崩溃。递归深度上限通常几千到几万层。尾递归（递归调用是最后一步）可被编译器优化为循环，不消耗栈空间。阶乘和斐波那契都不是尾递归。记住"递归深度"这个安全意识。

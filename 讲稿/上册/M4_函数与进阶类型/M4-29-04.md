**一、三种文件读写方式大比拼**

| 方式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| `freopen` | 竞赛标准 I/O | 两行代码，不改主体 | 不灵活，屏幕无输出 |
| `ifstream`/`ofstream` | 需要精细控制 | 灵活，可同时多文件 | 需要用 `fin >>` 替代 `cin >>` |
| `fstream` | 读写同一文件 | 一体多用 | 竞赛中用得少 |

**竞赛推荐方案——根据题目要求二选一：**

如果题目说"从 `xxx.in` 读入，输出到 `xxx.out`"→ 用 freopen，最省事（板书）：

```cpp
freopen("xxx.in", "r", stdin);
freopen("xxx.out", "w", stdout);
```

如果需要同时在屏幕上看调试信息，或者读写多个文件 → 用 ifstream/ofstream：

```cpp
ifstream fin("xxx.in");
ofstream fout("xxx.out");
// fin 替代 cin, fout 替代 cout
```

---

**二、文件打开失败检查——必须养成的习惯**

不管你用哪种方式，打开文件后一定要检查。两种写法等效（板书）：

```cpp
ifstream fin("data.txt");
if (!fin.is_open()) {   // 或 if (!fin)
    cerr << "无法打开文件 data.txt" << endl;
    return 1;
}
```

不要怕多写这几行——它能帮你省下大量调试时间。文件不存在、路径写错、权限不够……这些情况迟早会遇到，没检查的话你只能在"为什么程序没输出"里闷头找。

**三、综合示例——日记本程序**

把读和写串起来，做一个简单的"日记本"（板书）：

```cpp
#include <iostream>
#include <fstream>
#include <string>
using namespace std;

int main() {
    // 1. 先读上次的日记
    ifstream fin("diary.txt");
    if (fin.is_open()) {
        cout << "===== 上次的日记 =====" << endl;
        string line;
        while (getline(fin, line)) {   // 逐行读取（保留空格）
            cout << line << endl;
        }
        fin.close();
        cout << "======================" << endl;
    } else {
        cout << "还没有日记，开始写第一篇吧！" << endl;
    }

    // 2. 追加新日记
    ofstream fout("diary.txt", ios::app);  // 追加模式
    if (!fout.is_open()) {
        cout << "无法写入！" << endl;
        return 1;
    }

    string mood;
    cout << "今天心情怎么样？";
    cin >> mood;
    fout << "心情：" << mood << endl;
    fout.close();

    cout << "日记已保存！" << endl;
    return 0;
}
```

第一次运行：创建 diary.txt，写入心情。第二次运行：先显示上次的全部日记，再追加新的。完美的存档读档！

**四、M4 模块全景回顾**

同学们，从第 23 章到第 29 章，我们学完了 C++ 最核心的进阶内容：

- **函数（第23章）**：封装代码块，定义/调用/return/void/声明与定义分离
- **参数传递（第24章）**：值传递（副本）、引用传递（别名）、const引用（只读高效）
- **作用域（第25章）**：全局/局部/static、`{}` 边界、同名屏蔽、局部优先
- **递归（第26章）**：三要素、阶乘/斐波那契/调用栈、递归vs递推
- **结构体（第27章）**：自定义类型、`.` 和 `->`、结构体数组与排序
- **指针（第28章）**：取地址 `&`、解引用 `*`、nullptr、存地址访数据
- **文件操作（第29章）**：freopen、ifstream/ofstream、覆盖/追加模式

这七章构成了一个完整的"程序员工具箱"。从现在开始，你写的代码不再是"裸奔"的数据操作——而是有函数的封装、有结构体的打包、有指针的灵活访问、有文件的持久存储。

这些知识在后面的算法学习中会反复用到。打好基础，后面的路就宽了。恭喜大家完成 M4 模块！🎉

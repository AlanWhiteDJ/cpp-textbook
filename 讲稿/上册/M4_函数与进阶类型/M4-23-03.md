## 📝 讲稿正文

同学们，前两节课我们都是把函数写在 `main()` 上面——定义完就能调用。但今天我告诉你一个 C++ 的业界惯例：**声明和定义分开写**。

为什么要分开？我打个比方。

你去图书馆找一本书。图书馆门口贴了一张"书目索引"——告诉你每本书在几楼、几排、几号书架。你不需要先把整本书读一遍，只要扫一眼索引就知道"书在哪儿"。然后你带着索引的信息去书架取书。

C++ 的函数声明就是这张"书目索引"。它只告诉编译器：**"有这么个函数，它长这样"**——叫什么名字、要什么参数、返回什么类型。至于函数具体干了什么，那是"定义"的事，可以放在后面。

来看代码（板书）：

```cpp
#include <iostream>
using namespace std;

// 函数声明（也叫"函数原型"）——只写头部，加个分号
int max(int a, int b);          // 告诉编译器：有函数叫max，两个int参数，返回int
void printLine(int n);          // 有函数叫printLine，一个int参数，无返回值

int main() {
    cout << max(3, 5) << endl;  // ✅ 可以调用了！编译器已经知道max长啥样
    printLine(10);
    return 0;
}

// 函数定义——具体实现写在这里
int max(int a, int b) {
    return (a > b) ? a : b;
}

void printLine(int n) {
    for (int i = 0; i < n; i++) cout << "-";
    cout << endl;
}
```

注意看结构：先是两行声明（头部 + 分号），然后是 `main()`，最后是两个函数的定义。

**为什么要这样？** 编译器读代码是从上到下一行行读的。如果 `main()` 里调用了 `max`，但 `max` 的定义在 `main()` 下面——编译器读到调用的时候还不知道有 `max` 这个函数，就会报错。

声明就是解决这个问题的——你先"预告"一下：**"嘿编译器，有个函数叫 max，待会我会告诉你细节，先让我用着。"**

声明只写三样东西：返回类型、函数名、参数类型列表。参数名甚至可以省略：

```cpp
int max(int, int);  // 声明时参数名可省略（但不推荐，保留名字更清晰）
```

但分号绝对不能省！声明以分号结尾，定义以花括号开始——这是声明和定义最明显的语法区别。

**在实际项目中**，声明通常放在头文件（`.h`）里，定义放在源文件（`.cpp`）里。比如一个数学工具库：

```cpp
// math_utils.h —— 头文件（声明）
int gcd(int a, int b);
int lcm(int a, int b);
bool isPrime(int n);

// math_utils.cpp —— 源文件（定义）
#include "math_utils.h"
int gcd(int a, int b) { ... }
int lcm(int a, int b) { ... }
bool isPrime(int n) { ... }

// main.cpp —— 主程序
#include "math_utils.h"  // 引入声明就能用了！
int main() {
    cout << gcd(24, 36) << endl;
    return 0;
}
```

这就像是：头文件是"目录"，源文件是"正文"，主程序只需要看一眼目录就知道有哪些函数可用。

对于你们现阶段写的小程序，声明和定义分开可能显得多此一举。但随着代码量增长——几百行、几千行——你会感激这个习惯。至少记住一条：**如果你把函数定义写在 main() 下面，别忘了在上面加一行声明。**

最后来个易错点：声明和定义的头部必须一致——返回类型、函数名、参数类型都得对得上。如果不一致，编译报错说"声明和定义不匹配"。

## 🖥️ 板书/屏幕要点
```
函数声明（原型）：
  返回类型 函数名(参数类型列表);

函数定义：
  返回类型 函数名(参数列表) { 函数体 }

规则：
  声明在前（可调用），定义在后（可写在任意位置）
  声明末尾有分号，定义末尾是花括号
  参数名在声明中可省略，但保留更清晰

头文件(.h) = 声明合集  源文件(.cpp) = 定义合集
```

## 📌 小结
函数声明就是"预告"——告诉编译器函数的签名（返回类型+名字+参数类型）。声明要写在调用之前（加在 main() 上面）。定义写在哪都行。声明末尾加分号，定义用花括号。大型项目里声明放 .h 头文件，定义放 .cpp 源文件。从现在开始养成"先声明后定义"的好习惯。

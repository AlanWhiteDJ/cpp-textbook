# 第25章 第1课时：全局变量与局部变量——两种地盘，两种活法

**时长**：15-20分钟 | **对应教材**：第25章 25.1节

## 🎯 本课时目标
- 清晰区分全局变量和局部变量的定义位置、内存位置、生命周期
- 理解为什么全局变量方便但危险
- 能画出变量在内存中的分布图（全局区 vs 栈）

## 📝 讲稿正文

同学们，前面学了函数的参数怎么传。今天换个角度看变量——**变量住在哪里？活多久？**

想象你在办公大楼里工作。你的工位上有台私人电脑，只有你能用，下班就关机——这是**局部变量**。公司大厅有个公告栏，谁都可以贴条、谁都可以擦——这是**全局变量**。

在 C++ 里，变量的"居住地"取决于它**定义在什么位置**。

**局部变量**：定义在函数内部或代码块内部。只在这个函数/块里有效，出了大括号就不存在了。

**全局变量**：定义在所有函数外面（和 main 平级）。从定义处到文件末尾，任何函数都能访问它。

来看代码（板书）：

```cpp
#include <iostream>
using namespace std;

int gold = 100;            // ← 全局变量：定义在所有函数外面

void spend() {
    int price = 30;        // ← 局部变量：定义在 spend() 里面
    gold -= price;         //  访问全局变量 ← 可以
}

int main() {
    int wallet = 50;       // ← 局部变量：定义在 main() 里面
    gold += wallet;        //  访问全局变量 ← 可以
    spend();
    cout << gold << endl;  // 输出 120
    // cout << price;      // ❌ 编译错误！price 只在 spend() 里存在
    return 0;
}
```

看全局变量 `gold`：它定义在最外层，跟 main 是"平级"关系。`main` 能访问它，`spend` 也能访问它。它不是任何函数的"私人物品"，而是"公共资源"。

再看局部变量：`wallet` 是 main 的私人物品。`price` 是 spend 的私人物品。main 想用 price？编译器直接说："不认识"。price 出了 spend 的大括号，就已经"死了"。

**底层原理：它们住在不同的地方。**

计算机内存分成几个区域。两个最重要的：

**全局/静态区**：全局变量住这里。程序一启动就分配好，程序结束才释放。就像地基——打下去就一直在。

**栈（Stack）**：局部变量住这里。函数被调用时，在栈上临时占一块地方；函数返回时，栈自动收回这块地方。就像搭帐篷——搭起来用，用完拆掉。

画个图（板书）：

```
┌────────────────────┐
│   全局 / 静态区     │  ← gold 住这里，雷打不动
├────────────────────┤
│   栈 (Stack)       │  ← wallet、price 在这里，来去匆匆
│   函数调用时分配    │
│   返回时自动回收    │
├────────────────────┤
│   堆 (Heap)        │  ← 以后学 new/delete
└────────────────────┘
```

这个图你脑子里要有。以后学递归、学动态内存，都要回到这张图上来。

**全局变量既然谁都能用——为什么老师总说"慎用"？**

因为"谁都能用"的另一面是"谁都能改"。你写了 10 个函数，有 5 个改了全局变量 `gold`。某天你发现 `gold` 的值不对——到底是哪个函数改坏的？你要在 5 个函数里大海捞针。这就是"追责困难"。

还有"函数不独立"的问题。一个函数如果依赖全局变量，它就不能单独拿出来复用——你得把那个全局变量也带上。

所以编程界的共识是：**能用局部变量，绝不用全局变量。** 局部变量范围小，影响小，好控制。就像你家钥匙——放在家里安全，挂在大门口谁都能拿就危险了。

当然，信息学竞赛里有例外——大规模数组（`int a[100005]`）通常放全局，因为局部数组太大可能撑爆栈空间。但核心逻辑里的变量，永远遵循"局部优先"原则。

## 🖥️ 板书/屏幕要点
```
全局变量：
  定义在函数外面 → 全文件可见
  住在全局/静态区 → 程序启动分配，结束释放

局部变量：
  定义在函数/块内部 → 仅{}内可见
  住在栈上 → 函数调用时分配，返回时回收

内存布局：
  全局区：全局变量（雷打不动）
  栈：局部变量（来去匆匆）
  堆：动态分配（以后学）

原则：局部优先！变量范围越小越安全。
```

## ✏️ 随堂练习
1. 定义一个全局变量 `int counter = 0`，再写两个函数 `inc()` 和 `show()`。inc 让 counter+1，show 输出 counter。在 main 里交替调用，观察输出。
2. 定义一个全局变量 `int value = 100`，然后在 main 里定义同名局部变量 `int value = 200`，分别输出。观察屏蔽效果。
3. 改错：下面代码有什么问题？
```cpp
void func() { int secret = 42; }
int main() { func(); cout << secret; }
```

## 📌 小结
全局变量定义在函数外，住全局区，到处可见但风险大（谁都可能改）。局部变量定义在函数/块内，住栈上，范围小但安全。记住内存布局：全局区（稳如泰山）vs 栈（来去匆匆）。程序设计遵循"局部优先"原则。

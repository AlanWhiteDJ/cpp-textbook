# 第29章 第1课时：freopen 重定向——让 cin/cout 自动读写文件

**时长**：15-20分钟 | **对应教材**：第29章 29.1节

## 🎯 本课时目标
- 理解文件输入输出的意义——让数据跨越程序的生命周期
- 掌握 `freopen` 的用法，能将标准输入输出重定向到文件
- 了解 freopen 的优缺点和适用场景

## 📝 讲稿正文

同学们，到现在为止，所有的程序都是"用后即焚"的——输入靠键盘敲，输出看屏幕闪。程序一关，所有数据全部消失。下次运行又得重新输入。

这就像你打了一下午游戏，打到第 10 关——结果忘了存档，第二天打开，从第 1 关重新来。😱

能不能像游戏存档一样，把数据存到一个文件里，下次打开接着用？当然可以。这就是本章的主题——**文件操作**。

C++ 提供了几种读写文件的方式。今天先学最快最"暴力"的——**freopen 重定向**。

什么是重定向？就是把 `cin`（键盘输入）和 `cout`（屏幕输出）这两个"通道"拨到硬盘上的文件去。从此你用 `cin` 就是从文件读，用 `cout` 就是往文件写——代码完全不用改！

来看例子（板书）：

```cpp
#include <cstdio>    // freopen 需要这个头文件！
using namespace std;

int main() {
    freopen("input.txt", "r", stdin);    // stdin 重定向到 input.txt（只读）
    freopen("output.txt", "w", stdout);  // stdout 重定向到 output.txt（只写）

    // 下面的代码和平时完全一样！
    int a, b;
    cin >> a >> b;        // 实际上从 input.txt 读
    cout << a + b << endl; // 实际上写到 output.txt

    return 0;
}
```

就多了两行 `freopen`，代码主体零改动！你之前写的所有东西，加上这两行就变成"文件版"了。

解释一下三个参数：

| 参数 | 含义 | 示例 |
|------|------|------|
| 第一个 | 文件名（字符串） | `"input.txt"` |
| 第二个 | 打开模式 | `"r"` = 只读，`"w"` = 只写（覆盖） |
| 第三个 | 要重定向的流 | `stdin` = 标准输入，`stdout` = 标准输出 |

**使用步骤：**

1. 在程序同目录下创建一个 `input.txt`，用记事本写上测试数据，比如 `10 20`
2. 运行程序
3. 程序结束后，同目录下会多一个 `output.txt`，内容就是程序输出的结果

**freopen 的适用场景——信息学竞赛。**

很多竞赛题目要求"从 `xxx.in` 读入，输出到 `xxx.out`"。freopen 就是为此而生的。考场上你不需要去调文件路径、处理打开失败——两行代码，稳如泰山。

典型竞赛用法（板书）：

```cpp
int main() {
    freopen("sum.in", "r", stdin);
    freopen("sum.out", "w", stdout);
    // ... 正常写代码即可
    return 0;
}
```

**优点：** 代码零改动！所有 `cin >>` 和 `cout <<` 自动变成文件读写。

**缺点：** 不够灵活。一旦重定向，所有输入输出都走文件——屏幕上看不到任何东西。如果你想调试时在屏幕上看输出，或者同时读写多个文件，freopen 就满足不了了。

另外注意：freopen 之后 `cin` 和键盘彻底断了联系。如果你在 `freopen("in.txt", "r", stdin)` 之后还想从键盘读数据——做不到。所以需要交互的调试提示要在 freopen **之前**输出。

还有一个小细节：`"w"` 模式打开文件会**覆盖**已有内容。如果你想保留旧内容并在末尾追加——freopen 做不到。这种情况需要用后面要讲的 `ofstream` + `ios::app` 模式。

freopen 就像一个粗暴但高效的工具——简单场景下两行搞定，复杂场景下力不从心。但作为文件操作的入门，它是最好的开始。

## 🖥️ 板书/屏幕要点
```
freopen 重定向：
  freopen("输入文件", "r", stdin);   // cin → 文件读
  freopen("输出文件", "w", stdout);  // cout → 文件写

参数：
  ① 文件名（字符串）
  ② 模式："r"读写, "w"写（覆盖）
  ③ 流：stdin / stdout

注意：
  • 需要 #include <cstdio>
  • "w" 模式会覆盖已有文件
  • 重定向后屏幕无输出
```

## 📌 小结
freopen 将标准输入输出重定向到文件——两行代码，程序主体零改动。适合竞赛中"从 xxx.in 读到 xxx.out"的标准场景。`"r"` 只读，`"w"` 只写（覆盖）。局限：不灵活，不能同时操作多文件，不能屏幕输出调试。需要 `#include <cstdio>`。

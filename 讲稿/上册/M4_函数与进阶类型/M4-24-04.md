# 第24章 第4课时：数组作参数与常见陷阱——特殊公民与避坑指南

**时长**：15-20分钟 | **对应教材**：第24章 24.6-24.7节

## 🎯 本课时目标
- 理解数组作为函数参数时"退化成指针"的特性
- 掌握数组作参数的正确写法（传数组名 + 传长度）
- 识别并避开值传递与引用传递的七大常见陷阱

## 📝 讲稿正文

好，今天这节课我们讲一个"特殊公民"——**数组作为函数参数**，然后把整个值传递和引用传递的常见陷阱做个大盘点。

先说数组。数组在传参这件事上，和其他类型**完全不同**。

你回忆一下：`int` 传给函数，默认是值传递——复制一份。那数组呢？假如你写 `void func(int arr[100])`，调用时会把整个 100 个元素的数组复制一份吗？

**不会！** 数组从来不走值传递。编译器会悄悄把数组"退化"成一个指向首元素的指针。也就是说——函数里拿到的不是副本，而是直接指向原数组的指针！（板书）

```cpp
void fillArray(int arr[], int n) {   // arr 实际上是个指针！
    for (int i = 0; i < n; i++) {
        arr[i] = i * 10;             // 直接修改原数组
    }
}

int main() {
    int a[5] = {0};
    fillArray(a, 5);                 // 传数组名 = 传首地址
    for (int i = 0; i < 5; i++)
        cout << a[i] << " ";         // 0 10 20 30 40 —— 原数组被改了！
    return 0;
}
```

你看，函数里根本没有 `&`，但原数组还是被改了。这就是数组的"特权"——**默认就传地址**。

这带来一个重要推论：**数组作为参数时，函数不知道数组有多长。** 因为 `arr` 只是一个指针，它只知道"起始地址"，不知道后面有几个元素。所以你必须**额外传一个参数表示长度**：

```cpp
void process(int arr[], int n) { ... }  // n 就是数组长度
```

再注意一个细节：你在参数里写 `arr[5]` 或 `arr[100]`——这个数字**编译器会直接忽略**。`void f(int arr[5])` 和 `void f(int arr[100])` 和 `void f(int arr[])` 和 `void f(int *arr)`——这四个声明完全等价。所以干脆写 `arr[]` 不加数字，再加一个 `int n` 表示长度，最清晰。

好，数组讲完了。现在咱们来做个**全章避坑大盘点**。以下七大陷阱，逐一过一遍：

**陷阱一：swap 忘了 `&`。** 这是全 C++ 教学史上踩坑人数最多的错误。`void swap(int a, int b)` —— 少了 `&`，函数里交换的是副本，外面纹丝不动。✅ 改成 `void swap(int &a, int &b)`。

**陷阱二：给引用传字面量。** `void addTen(int &x)`，调用 `addTen(5)`——编译报错！5 没有地址，不能绑定到非 const 引用。✅ 如果确实要接字面量，加 `const`：`void addTen(const int &x)`。

**陷阱三：const 引用忘了用。** 函数只需要读一个长字符串，却写成了 `void f(string s)`——每次调用都复制整个字符串，白白浪费性能。✅ 改成 `void f(const string &s)`。

**陷阱四：值传递期望能改实参。** 写了 `void reset(int x) { x = 0; }` 然后纳闷为什么 a 没变——因为你改的是副本啊。✅ 加 `&`。

**陷阱五：引用传递后误改数据。** 明明是只读的参数，忘加 const，函数里不小心改了——调用者的数据被魔法般地篡改。✅ 只要不修改就加 const，让编译器帮你把关。

**陷阱六：数组参数里写了长度却不用额外参数。** `void print(int arr[5])`——那个 5 被编译器忽略了，函数还是不知道数组多长。✅ 写 `void print(int arr[], int n)`。

**陷阱七：试图用 `&` 阻止数组复制。** 有同学想："数组不是默认复制吗？我加个 `&` 避免复制"。错误——数组本来就不会复制，加不加 `&` 都一样是传地址。不需要多此一举。

好了，这些坑你迟早会踩——或者更准确地说，看一眼这个清单，让自己少踩几个。尤其是 swap 那个 `&`，记住了：**只要写 swap，先敲 &**。形成肌肉记忆。

## 🖥️ 板书/屏幕要点
```
数组作参数特性：
  • 数组永不复制 → 默认传首地址（退化为指针）
  • 必须额外传长度：void f(int arr[], int n)
  • 中括号里的数字被忽略：arr[5] = arr[100] = arr[]

七大避坑：
  1. swap忘&       → void swap(int &a, int &b)
  2. 引用接字面量    → 加const
  3. 大对象没用const& → const string &s
  4. 值传递想改实参   → 加&
  5. 引用误改数据     → 加const
  6. 数组参数少传长度 → void f(int arr[], int n)
  7. 数组加&防复制    → 不需要，数组本来就不复制
```

## 📌 小结
数组作为函数参数时默认传地址（不复制），直接操作原数组。必须额外传长度参数。七大陷阱中，swap 忘 `&` 排名第一——记住"写 swap 先敲 &"。const 引用是防误改的最佳武器。传参选择：能用 const 的别用普通引用，能用值传递的小类型别用引用。

# 第27章 第3课时：结构体指针与 `->` 运算符

**时长**：15-20分钟 | **对应教材**：第27章 27.4节

## 🎯 本课时目标
- 掌握结构体指针的声明和使用
- 熟练运用 `->` 运算符访问指针指向的结构体成员
- 理解 `.`（点）和 `->`（箭头）的区别和使用场景

## 📝 讲稿正文

上节课我们用结构体数组管理了一个班的学生。今天学一个新武器——**结构体指针**，让操作更高效。

既然 `Student` 是一种类型，那就自然可以有指向 `Student` 的指针——`Student*`。

先来看基本用法（板书）：

```cpp
Student s = {"小明", 13, 96.5};
Student* p = &s;     // p 是指向 s 的指针，存着 s 的地址
```

现在 `p` 指向 `s`。问题来了：怎么通过 `p` 访问 `s` 的成员？

**方法一：先解引用，再用 `.`** —— 有点啰嗦：

```cpp
cout << (*p).name << endl;   // 先 *p 得到 s，再 .name
```

`*p` 解引用得到 `s`，然后用 `.name`。注意括号不能省——`.` 的优先级比 `*` 高，不写括号变成 `*(p.name)`，p 不是结构体，没有 `.name`，编译报错。

**方法二：用 `->` 运算符** —— 简洁优雅（板书）：

```cpp
cout << p->name << endl;   // 等价于 (*p).name
cout << p->age << endl;    // 等价于 (*p).age
cout << p->score << endl;  // 等价于 (*p).score
```

`->` 是一个减号加一个大括号。读作"指向的"——`p->name` 就是"p 指向的那个结构体的 name"。

通过 `->` 也能修改成员（板书）：

```cpp
Student s = {"小明", 13, 96.5};
Student* p = &s;

p->score = 99.0;           // s.score 变成了 99.0
p->age = 14;               // s.age 变成了 14

cout << s.score << endl;   // 99.0 —— 真的改了！
```

**`. ` vs `->` 对比记忆口诀：**

- `. ` 是"的"—— `s.name` = s 的 name。用在结构体变量上。
- `->` 是"指向的"—— `p->name` = p 指向的那个结构体的 name。用在结构体**指针**上。

**常见错误**——用 `.` 操作指针：

```cpp
Student* p = &s;
p.name = "小明";   // ❌ 编译错误！p 是指针，不能用 .
```

指针必须用 `->` 或先解引用用 `*.`。这是新手最容易犯的错之一。

**结构体指针的实战用途——遍历数组**：

看一个用指针遍历结构体数组的例子（板书）：

```cpp
Student class1[3] = {
    {"小明", 13, 96.5},
    {"小红", 14, 92.0},
    {"小刚", 13, 88.5}
};

Student* p = class1;          // p 指向数组首元素
for (int i = 0; i < 3; i++) {
    cout << p->name << " " << p->score << endl;
    p++;                      // p 移向下一个 Student
}
```

`p++` 后，`p` 自动加 `sizeof(Student)` 个字节，跳到下一个学生。这比 `class1[i].name` 多了一种写法，在某些场景下更高效。

**结构体指针作为函数参数**——比传值高效：

```cpp
// 传值：复制整个结构体，数据量大时浪费
void printStudent(Student s) { cout << s.name; }

// 传指针：只传 8 字节地址，高效！
void printStudent(Student* p) { cout << p->name; }

// 调用
Student s = {"小明", 13, 96.5};
printStudent(&s);   // 传地址
```

如果 Student 里有长字符串，传值要复制所有字符串，而传指针只复制一个 8 字节的地址。这也是为什么很多函数用指针或引用参数。

好，总结一下：`.` 是变量的"的"，`->` 是指针的"指向的"。操作结构体成员，看清前面是变量还是指针——变量用 `.`，指针用 `->`。

## 🖥️ 板书/屏幕要点
```
结构体指针：
  Student* p = &s;

访问成员：
  方法一：(*p).name      （啰嗦，括号不能省）
  方法二：p->name         （简洁！推荐）

修改成员：
  p->score = 99.0;       // 等价于 (*p).score = 99.0;

口诀：
  .   是"的"     → 变量用 .   s.name
  ->  是"指向的"  → 指针用 ->  p->name
```

## 📌 小结
结构体指针用 `Student* p = &s;` 声明。访问成员用 `p->name`（等价于 `(*p).name`）。`.` 用于结构体变量，`->` 用于结构体指针——这是必须死记的语法规则。传结构体指针给函数比传值更高效（只传地址，不复制数据）。指针 `p++` 会自动跨越一个结构体的大小跳到下一个元素。

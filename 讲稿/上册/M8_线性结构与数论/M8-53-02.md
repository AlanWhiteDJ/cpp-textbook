# 第53章 第2课时：单链表的插入与删除操作
**时长**：15-20分钟 | **对应教材**：第53章

## 🎯 本课时目标
- 掌握头部插入、尾部插入的完整代码
- 掌握删除指定节点的操作
- 理解为何要使用 `Node*&` 引用传递头指针

## 📝 讲稿正文

上节课我们手动创建了几个节点，手工把它们串了起来。但真正的链表操作应该是函数化的——我要插入一个节点，调用一个函数就搞定。这节课我们就把头部插入、尾部插入和删除写成标准函数。

（敲黑板）

先看**头部插入**——在链表最前面加一个节点：

```cpp
void insertHead(Node*& head, int val) {
    Node* newNode = new Node;
    newNode->data = val;         // 填数据
    newNode->next = head;        // 新节点的 next 指向原来的头
    head = newNode;              // 头指针更新为新节点
}
```

注意参数里有一个奇怪的符号：`Node*& head`。这里的 `&` 是**引用**。为什么需要引用？因为我们要修改 head 本身——让它指向新节点。如果没有 `&`，函数里修改的只是 head 的副本，外部的 head 纹丝不动。

这是 C++ 里一个很重要的概念：**要想在函数里修改外部变量，传引用或传指针的地址。** 我们这里传的是 `Node*&`——对指针的引用。

（停顿）

接下来是**尾部插入**——在链表最后加一个节点：

```cpp
void insertTail(Node*& head, int val) {
    Node* newNode = new Node;
    newNode->data = val;
    newNode->next = nullptr;     // 新节点后面是空

    if (head == nullptr) {       // 空链表：新节点就是头
        head = newNode;
        return;
    }

    Node* cur = head;
    while (cur->next != nullptr) {  // 走到最后一个节点
        cur = cur->next;
    }
    cur->next = newNode;         // 挂上去
}
```

这里有个边界条件：如果链表是空的（head 为 nullptr），新节点直接当头。注意这里的遍历条件用的是 `cur->next != nullptr`——不是 `cur != nullptr`。因为我们要**停在最后一个节点**，而不是走到它的后面。

来对比一下：
- 遍历整个链表 → `while (cur != nullptr)`  
- 找最后一个节点 → `while (cur->next != nullptr)`

一字之差，含义完全不同。

（转身写第三段代码）

接下来是**删除**——删除链表中第一个值为 val 的节点：

```cpp
void deleteNode(Node*& head, int val) {
    if (head == nullptr) return;      // 空链表，什么也不做

    if (head->data == val) {          // 要删的是头节点
        Node* temp = head;
        head = head->next;            // 头指针后移
        delete temp;                  // 释放内存
        return;
    }

    Node* cur = head;
    while (cur->next != nullptr && cur->next->data != val) {
        cur = cur->next;              // 找目标节点的前一个
    }

    if (cur->next != nullptr) {       // 找到了
        Node* temp = cur->next;
        cur->next = cur->next->next;  // 跳过目标节点
        delete temp;                  // 释放内存
    }
}
```

删除操作有三个关键点：

第一，**先判空**。如果链表是空的，别瞎操作。

第二，**头节点单独处理**。删头节点和删中间节点的逻辑不同——删头节点没有"前驱"，直接让 head 指向下一个。

第三，**删除后要 `delete`**。用 `new` 创建的要记得用 `delete` 释放，不然内存会慢慢漏光。虽然在竞赛的小程序中可能无所谓，但养成良好的内存管理习惯很重要。

（停顿）

这三个操作中，头部插入是最简单的——O(1)。尾部插入需要遍历到末尾——O(n)。删除也需要先找到目标节点——O(n)。这里就体现了链表的一个尴尬之处：虽然"插入删除"这个动作本身是 O(1)（只改指针），但"找到插入/删除的位置"往往需要 O(n) 的遍历。

所以在实际开发中，链表并不是万能的。频繁随机访问用数组/vector，频繁在头部操作用链表，需要根据实际场景选择。

## 🖥️ 板书/屏幕要点

```
头部插入（O(1)）：
  newNode->next = head; head = newNode;

尾部插入（O(n)）：
  找到最后一个节点（cur->next == nullptr）
  cur->next = newNode;

删除（O(n)）：
  找到目标节点的前驱
  cur->next = cur->next->next;
  delete 被删节点;

遍历条件区别：
  遍历全部：while(cur != nullptr)
  找最后一个：while(cur->next != nullptr)
```

## 📌 小结

本课时系统学习了单链表的三大操作：头部插入（O(1)）、尾部插入（O(n)）、删除指定值（O(n)）。关键细节：传引用修改头指针、遍历条件的区别、删除后释放内存。操作链表时"先连后继再改前驱"的原则要刻在脑子里。下节课我们见识双向链表和循环链表——以及竞赛中更常用的数组模拟链表。

## 📝 讲稿正文

前面两节课，我们用辗转相除法求出了 GCD。但欧几里得算法还能做一件事——而且这件事在密码学、数论竞赛题中极其重要。

（在黑板上写）

问题：给你 a = 48, b = 18，已知 GCD = 6。请问——能不能找到整数 x 和 y，使得 **48x + 18y = 6**？

你可能会想：这有什么难的？无非是个方程嘛。试试看：x = -1, y = 3 → 48×(-1) + 18×3 = -48 + 54 = 6。找到了！

但如果是 a = 999999937, b = 1000000007 呢？随便试太不现实了。

好在这个方程**一定存在整数解**。这不是猜想，是定理——**裴蜀定理**（也叫贝祖定理）：

> 对于任意整数 a 和 b，存在整数 x 和 y，使得 **ax + by = gcd(a, b)**。而且，ax + by 能表示的所有正整数，恰好是 gcd(a, b) 的所有倍数。

更实用的是这个推论：

> 方程 **ax + by = c** 有整数解，**当且仅当 gcd(a, b) 整除 c**。

举例：3x + 5y = 1。gcd(3,5) = 1，1 整除 1 → 有解。（实际上 x=2, y=-1 就是一组解：3×2 + 5×(-1)=1）

3x + 6y = 4。gcd(3,6) = 3，3 不整除 4 → 无解。想想也对：左边一定是 3 的倍数，右边是 4，不可能相等。

（停顿）

那具体怎么求 x 和 y？在辗转相除法的过程中"顺便"算出来——这就是**扩展欧几里得算法**。代码如下：

```cpp
// 扩展欧几里得：返回 gcd，同时通过引用返回 x 和 y
int exgcd(int a, int b, int& x, int& y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;          // gcd(a, 0) = a, 此时 a*1 + 0*0 = a
    }
    int d = exgcd(b, a % b, y, x);   // 注意：参数位置交换了！
    y -= a / b * x;
    return d;
}
```

这段代码的推导需要一些数学，现阶段不要求大家完全理解每一步。但你能看懂它的结构：它和普通 GCD 的递归结构完全一样，只是在递归返回时多算了两步——更新 x 和 y。

调用方法：

```cpp
int x, y;
int g = exgcd(48, 18, x, y);
cout << "gcd = " << g << endl;       // 6
cout << x << " " << y << endl;       // -1 3
cout << 48*x + 18*y << endl;         // 6（验证）
```

（停顿，观察学生反应）

扩展欧几里得算法在竞赛中有三个重要用途：

第一，**解线性丢番图方程** ax + by = c。先判断 gcd(a,b) | c，如果有解，用 exgcd 求一组特解，然后推出通解。

第二，**求模逆元**。在模 m 意义下，如果 gcd(a, m) = 1，求 a 的逆元（即找一个 x 满足 a×x ≡ 1 mod m）。这等价于解 ax + my = 1——正好是 exgcd 能做的事。模逆元是 RSA 加密和组合数取模的基石。

第三，**扩展中国剩余定理**——处理同余方程组。这个我们下册再细说。

现阶段，大家先记住三件事：  
裴蜀定理告诉你"有没有解"；  
exgcd 能帮你"找到一组解"；  
这个工具在下册的密码学和数论进阶中会反复出现。

## 🖥️ 板书/屏幕要点

```
裴蜀定理：
  存在 x,y 使 ax+by = gcd(a,b)
  ax+by=c 有整数解 ⇔ gcd(a,b) | c

扩展欧几里得（exgcd）：
  exgcd(a,b,x,y) → 返回 gcd，x,y 是解
  在普通 GCD 递归中"顺便"算出

应用：
  ① 解 ax+by=c（先判是否有解）
  ② 求模逆元（密码学基础）
  ③ 中国剩余定理（下册）

现阶段：理解"能做什么"即可
```

## 📌 小结

裴蜀定理告诉我们 ax+by=c 有整数解的充要条件：gcd(a,b) | c。扩展欧几里得算法在辗转相除的同时求出方程的一组特解。这两者合在一起，是解决线性丢番图方程和求模逆元的标准工具。现阶段了解其存在性即可，下册会正式展开。下一章，我们进入排列组合的世界——算一算"有多少种可能"。

## 📝 讲稿正文

上节课我们用队列模拟了银行排队——先来的先服务，很简单。这节课我们玩点有挑战性的——**约瑟夫问题**。

（讲故事的语气）

约瑟夫是公元一世纪的一位犹太历史学家。传说他和 40 个士兵被罗马军队困在一个山洞里。他们决定宁死不降，于是 41 个人围成一圈，从第一个人开始报数，每数到第 3 个人就出列自杀，下一个人重新从 1 开始报。约瑟夫把自己安排在了第 31 号位置——最后他活了下来。

这个问题在算法竞赛中反复出现，核心就是：**N 个人围成一圈，从第一个开始报数，报到 M 的人出列，然后下一个人重新从 1 报数，直到所有人出列。求出列顺序。**

你可能会想：这题用数组怎么模拟？每次有人出列，数组里就空出一个位置，后面的人要"绕"回来……非常麻烦。

但用队列来做，优雅得让人惊叹。思路是这样的：

把 1 到 N 所有人依次入队。然后模拟报数：

每次把队首的人"拎"到队尾——front + pop + push——相当于这个人没被淘汰，重新排队。重复 M-1 次后，此时队首的人就是该出列的人。pop 掉他，继续下一轮。

来看代码：

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    queue<int> q;
    for (int i = 1; i <= n; i++) {
        q.push(i);          // 所有人先入队
    }

    while (!q.empty()) {
        // 报数：前 m-1 个人从队首移到队尾
        for (int i = 1; i < m; i++) {
            q.push(q.front());  // 队首的人排到队尾
            q.pop();            // 原来的队首离开
        }
        // 第 m 个人出列
        cout << q.front() << " ";
        q.pop();
    }
    cout << endl;
    return 0;
}
```

来，走一遍 n=5, m=3 的例子。

初始队列：1 2 3 4 5  
第一轮：把 1 移到队尾 → 2 3 4 5 1；把 2 移到队尾 → 3 4 5 1 2；第 3 个人是 3 → 输出 3，pop 掉。  
现在队列：4 5 1 2  
第二轮：4 移到队尾 → 5 1 2 4；5 移到队尾 → 1 2 4 5；出列的是 1 → 输出 1。  
现在队列：2 4 5  
……依此类推。最终输出：3 1 5 2 4。

你看，队列完美模拟了"围成一圈"的效果——队首到队尾，就是"绕一圈"。

（停顿）

为什么队列能这么好地解决约瑟夫问题？因为约瑟夫问题的核心是**轮流、循环、先来先处理**——这正是 FIFO 的天生优势。用栈来写这个题就会非常别扭，因为栈是 LIFO，处理不了"按顺序轮流"。

这个技巧——用 queue 实现循环——在竞赛中很常见。queue 本质上是"先进先出"的管道，而循环就是"把出口接到入口"。

说到循环，我还要提一下**循环队列**这个概念。STL 的 queue 底层其实是用 deque 实现的，已经帮你处理好了。但如果你想用数组手动实现一个队列，要注意：队首在数组左边，队尾在右边，当队尾走到数组尽头时，需要"绕回来"——这就是循环队列的思想。用两个指针 front 和 rear，下标到达数组末尾时回到 0。不过现阶段用 STL 的 queue 就完全够用了，先把用法学会，底层原理以后需要时再深挖。

## 🖥️ 板书/屏幕要点

```
约瑟夫问题（队列版）：
  1. 1~N 依次入队
  2. 重复直到队列空：
     a. 队首→队尾，做 M-1 次
     b. 输出并 pop 队首

核心技巧：front()+push()+pop()
  = 队首的人"绕到队尾重新排队"

循环队列思想：
  数组模拟时，rear 到头后回到 0
  用 (rear+1) % MAXSIZE 实现循环
```

## 📌 小结

约瑟夫问题是队列"循环"特性的经典应用。把队首移到队尾这个简单操作，完美模拟了"围成一圈轮流"的效果。队列天生适合处理"按顺序轮流"的场景，而栈适合"回溯撤销"——选对容器，代码能简洁十倍。下节课我们认识队列的加强版——deque 双端队列，两头都能操作，看看它有什么独特的用武之地。

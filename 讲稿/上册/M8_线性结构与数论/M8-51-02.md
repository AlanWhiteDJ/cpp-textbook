## 📝 讲稿正文

上节课我们认识了栈，学会了 push、pop、top 这些基本操作。有同学可能会想：这东西看着挺简单，实际有什么用？

今天我们就来见识栈的第一个经典应用——括号匹配。这个问题几乎每个学编程的人都必做，也是很多面试的标配题。

（在黑板上写）问题是这样的：给你一个括号串，比如 `"(()())"`，判断括号是否配对正确。配对的规则很简单——每个左括号必须有一个对应的右括号，而且不能交叉。

比如 `"(()())"` 是配对的，`"(()"` 不配对（少一个右括号），`")("` 也不配对（顺序错了）。

怎么用程序判断？你可能会想：数左括号和右括号的数量，相等就行了呗。这个思路有漏洞——`")("` 左右数量相等，但顺序不对。

栈来解决这个问题非常优雅。规则就两条：

**遇到左括号 `(` → 入栈。**  
**遇到右括号 `)` → 看栈顶有没有左括号。有就弹掉一个，没有就说明不匹配。**

遍历完整个字符串后，如果栈是空的，说明全部配对成功；如果栈里还有东西，说明左括号多了。

来看代码：

```cpp
bool isBalanced(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(') {
            st.push(c);                // 左括号入栈
        } else if (c == ')') {
            if (st.empty()) return false;  // 多了右括号
            st.pop();                      // 弹掉一个左括号
        }
    }
    return st.empty();  // 栈空 → 全部配对上
}
```

来，我们在这段代码上走一遍。假设输入是 `"(())"`：

第一步，遇到 `(` → push，栈里有 1 个。  
第二步，遇到 `(` → push，栈里有 2 个。  
第三步，遇到 `)` → 栈不空，pop，栈里剩 1 个。  
第四步，遇到 `)` → 栈不空，pop，栈空。  
遍历完，栈空 → 返回 true，配对！

再试 `")("`：  
第一步，遇到 `)` → 栈是空的 → 立刻返回 false！程序直接就能判断不配对。

（停顿，让学生跟上）

好，现在升级难度。如果不止小括号，还有中括号 `[]` 和大括号 `{}` 呢？规则还是一样，但多了一个要求：**右括号必须和栈顶的左括号是同一类型**。

比如 `"{ [ ( ) ] }"` 是对的，`"{ [ ( ] ) }"` 是错的——因为遇到 `]` 时栈顶是 `(`，类型不匹配。

代码稍作修改：

```cpp
bool isBalanced(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);  // 左括号一律入栈
        } else if (c == ')' || c == ']' || c == '}') {
            if (st.empty()) return false;
            char top = st.top();
            // 检查类型是否匹配
            if ((c == ')' && top != '(') ||
                (c == ']' && top != '[') ||
                (c == '}' && top != '{')) {
                return false;
            }
            st.pop();
        }
    }
    return st.empty();
}
```

这段代码的核心思想就是：栈里存的永远是"等待被配对的左括号"。每遇到一个右括号，它必须和最近一个还没配对的左括号（也就是栈顶）匹配。这就是 LIFO 的天然优势——最近放进去的左括号，最先等待配对。

理解了括号匹配，你就理解了栈的灵魂。这个"等待配对"的思想在编译原理、表达式求值等很多领域都会用到。

## 🖥️ 板书/屏幕要点

```
括号匹配算法：

1. 遇到左括号 → push
2. 遇到右括号 → 判空 + 判匹配 → pop
3. 遍历完 → 栈空则配对成功

多种括号：检查类型匹配
  ')' 配 '('   ']' 配 '['   '}' 配 '{'

核心思想：栈顶永远是最近一个"等配对"的左括号
```

## 📌 小结

括号匹配是栈的入门经典题，背后的原理简单而强大：用栈记录"等待配对的左括号"，每个右括号必须和栈顶的左括号配对。这个"记忆最近未匹配元素"的模式，也是栈在撤销操作、递归模拟、表达式求值等场景中的核心思路。下一课时，我们拓展到用数组模拟栈，以及用栈做进制转换——你会发现栈能做的事情远不止匹配括号。

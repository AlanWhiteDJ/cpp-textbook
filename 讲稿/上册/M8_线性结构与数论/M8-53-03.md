## 📝 讲稿正文

上两节课我们玩的是单链表——每个节点只有一个 `next` 指针，只能往一个方向走。这节课我们来看看链表的两个"变种"，以及一个竞赛中更常用的实现方式。

（在黑板上画双向链表）

先说**双向链表**。单链表有一个痛点：如果你在某个节点，想知道前一个节点是谁，对不起，你得从头再走一遍。双向链表给每个节点加了一个 `prev` 指针，指向前一个节点：

```cpp
struct DNode {
    int data;
    DNode* prev;  // 指向前一个
    DNode* next;  // 指向后一个
};
```

双向链表的好处非常直观：删除某个节点时，不需要先遍历找前驱——因为通过 `prev` 直接就能拿到前驱了。前后都能走，灵活度翻倍。代价是每个节点多存一个指针，内存开销多了一倍。在信息学竞赛中，双向链表用得不多——大多数场景单链表就够用了。

（停顿）

接下来是**循环链表**。普通链表的最后一个节点 `next` 指向 `nullptr`——走到头就是终点。循环链表把最后一个节点的 `next` 指向头节点，形成闭环——就像小朋友手拉手围成一个圈。

还记得约瑟夫问题吗？上一章我们用 queue 解的。用循环链表来解也很直观——N 个人围成一圈，每次数 M 步删一个：

```cpp
// 创建循环链表：1 → 2 → ... → n → 回到 1
Node* head = new Node{1, nullptr};
Node* cur = head;
for (int i = 2; i <= n; i++) {
    cur->next = new Node{i, nullptr};
    cur = cur->next;
}
cur->next = head;  // 尾连头，形成环

// 模拟出列
cur = head;  // 现在 cur 是最后一个节点（方便删头）
while (cur->next != cur) {  // 只剩一个时停止
    for (int i = 1; i < m; i++) cur = cur->next;  // 走 m-1 步
    Node* temp = cur->next;         // 要删除的节点
    cur->next = temp->next;         // 跳过它
    delete temp;
}
cout << cur->data << endl;  // 最后剩下的
```

如果看不懂没关系——实际竞赛中用循环链表做约瑟夫还不如用 queue 简洁。但理解"尾连头"这个思路本身就是收获——它是循环结构的入门。

好，现在进入今天最重要的一块内容——**数组模拟链表**。

如果你参加过信息学竞赛，你会发现真正用 `new`/`delete` 指针写链表的代码其实不多。为什么？第一，频繁的 `new` 和 `delete` 慢；第二，指针操作容易出 bug；第三，调试时指针地址你肉眼读不懂。

竞赛选手更喜欢用一个朴素但极其实用的技巧：**用数组下标代替指针**。

```cpp
const int MAXN = 100005;
int data[MAXN];    // 存节点的值
int nxt[MAXN];     // nxt[i] = 节点 i 的下一个节点的下标
int head = -1;     // 头节点的下标，-1 表示空链表
int tot = 0;       // 已经用了几个节点

// 头部插入
void insertHead(int val) {
    data[tot] = val;        // 新节点的数据
    nxt[tot] = head;        // 新节点的 next 指向原来的头
    head = tot;             // 头指针更新
    tot++;                  // 节点数 +1
}

// 遍历
void printList() {
    for (int i = head; i != -1; i = nxt[i]) {
        cout << data[i] << " ";
    }
    cout << endl;
}
```

你看，`nxt[tot] = head` 对应于指针版的 `newNode->next = head`。`head = tot` 对应于 `head = newNode`。`nxt[i]` 代替了 `node->next`，`data[i]` 代替了 `node->data`。

遍历时 `for (int i = head; i != -1; i = nxt[i])` 这行代码，相当于 `for (Node* cur = head; cur != nullptr; cur = cur->next)`——结构完全对应，只是把指针换成了数组下标。

数组模拟链表的三个好处：
1. **不需要 new/delete**，不会内存泄漏
2. **速度快**，数组操作比堆分配快得多
3. **调试友好**——想看所有节点？直接输出 data 和 nxt 数组就行

这个技巧有个名字叫"静态链表"——用静态的数组空间模拟动态的指针连接。学会这个技巧后，你会发现链表题目用数组写起来比指针版还顺手。

## 🖥️ 板书/屏幕要点

```
双向链表：多个 prev 指针，前后都能走
循环链表：尾->next 指向头，首尾相连

数组模拟链表（静态链表）：
  data[tot] / nxt[tot] / head / tot
  下标代替指针，-1 代替 nullptr
  
  头部插入：data[tot]=val; nxt[tot]=head; head=tot++;
  遍历：for(int i=head; i!=-1; i=nxt[i])

优势：无 new/delete，快，易调试
```

## 📌 小结

今天我们见识了链表的两个变种——双向链表前后都能走，循环链表首尾相连形成闭环。更重要的是学会了数组模拟链表——用 data[] 和 nxt[] 代替指针，在竞赛中比指针版更常用。到这儿，线性结构的最后一块拼图（链表）也齐了。下一章我们暂时离开"结构"，进入"数字本身"的世界——初等数论，从整除和质数开始。

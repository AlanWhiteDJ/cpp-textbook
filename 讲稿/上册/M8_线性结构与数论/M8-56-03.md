## 📝 讲稿正文

上节课我们算了 A(5,3) = 60——从 5 个人中选 3 个排成一排拍照。但如果我们只是**选 3 个人组成一个小组**，不排位置呢？ABC 和 ACB 就是同一个小组——没有顺序之分。这时候，排列数 60 就多算了。

（在黑板上写）

选出的 3 个人，他们内部可以排 3! = 6 种顺序，但这 6 种在"选小组"的场景下应该算**同一种**。所以，组合数 = 排列数 ÷ 内部排列数：

> **C(n, m) = A(n, m) / m! = n! / (m! × (n-m)!)**

C(5, 3) = 60 / 6 = 10 种。你可以验证一下：从 ABCDE 中选 3 个人，恰好 10 种组合。

C(n, m) 也经常写成二项式系数 $\binom{n}{m}$，读作"n 选 m"。

代码实现有两种方式。**方式一：直接公式法**。

```cpp
long long C(int n, int m) {
    if (m > n - m) m = n - m;   // 对称性：C(n,m) = C(n, n-m)
    long long result = 1;
    for (int i = 1; i <= m; i++) {
        result = result * (n - i + 1) / i;  // 先乘后除
    }
    return result;
}
```

注意 `if (m > n - m) m = n - m;` 这行——**组合数的对称性**：C(100, 98) 和 C(100, 2) 相等，但算 C(100, 2) 只要两次乘除，算 C(100, 98) 要 98 次。这就是为什么我们用对称性把 m 换成较小的那半边。

（停顿）

**方式二：杨辉三角递推法**。这个方法更强大——不需要阶乘，而且天然支持取模。

大家见过杨辉三角吗？

```
          1
        1   1
      1   2   1
    1   3   3   1
  1   4   6   4   1
1   5  10  10   5   1
```

注意到规律没有？**每个数等于它"肩上"两个数之和**。翻译成组合数就是：

> **C(n, m) = C(n-1, m-1) + C(n-1, m)**

为什么？从 n 个中选 m 个，分两种情况：
- 选了第 n 个元素 → 剩下 n-1 个中再选 m-1 个 → C(n-1, m-1)
- 没选第 n 个元素 → 剩下 n-1 个中选 m 个 → C(n-1, m)

两种情况互斥 → 加法原理 → 加起来。

这个递推公式是组合数的灵魂！用它可以建一张表：

```cpp
const int MAXN = 100;
long long comb[MAXN][MAXN];

void buildPascal(int n) {
    for (int i = 0; i <= n; i++) {
        comb[i][0] = comb[i][i] = 1;   // 边界：C(n,0)=C(n,n)=1
        for (int j = 1; j < i; j++) {
            comb[i][j] = comb[i-1][j-1] + comb[i-1][j];
        }
    }
}
// 建表后，查 C(5,2) 直接 comb[5][2] = 10
```

建表 O(n²)，查询 O(1)。当你要多次查询组合数时，建一次表反复用，效率极高。

（转身写板书的最后一块）

对比两种方式：公式法适合单次查询且 n 不大（约 60 以内不溢出），递推法适合多次查询或者需要取模的场景。如果 n 大到 10⁶——公式法和递推法都不行了，那就需要**阶乘 + 逆元**的组合数取模方法，属于下册进阶内容。

## 🖥️ 板书/屏幕要点

```
组合 C(n,m)：从 n 个中选 m 个，不考虑顺序
  公式：n! / (m!(n-m)!)
  关系：C(n,m) = A(n,m) / m!

对称性：C(n,m) = C(n, n-m)
  如 C(100,98) = C(100,2)

杨辉三角递推：
  C(n,m) = C(n-1,m-1) + C(n-1,m)
  comb[i][0]=comb[i][i]=1
  建表 O(n²)，查询 O(1)

实现选型：
  单次+小n → 公式法
  多次+取模 → 杨辉三角递推
  大n → 阶乘+逆元（下册）
```

## 📌 小结

组合数 C(n,m) 在排列数 A(n,m) 的基础上除去了顺序带来的 m! 种重复。对称性 C(n,m)=C(n,n-m) 和杨辉三角递推式是两个重要性质。杨辉三角用 O(n²) 建表、O(1) 查询，是竞赛中最常用的组合数计算方式——尤其配合取模。下节课我们进入组合数取模和鸽巢原理，看看怎么用杨辉三角自动取模，以及一个简单但威力巨大的组合定理。

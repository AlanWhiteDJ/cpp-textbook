## 📝 讲稿正文

前两节课我们学了栈和队列——一个只在一头操作，一个一头进一头出。今天来个更灵活的——**deque（双端队列）**。全名是 double-ended queue，**两头都能操作**。

（做手势）

想象一节地铁车厢，两头都有门——你可以从前面上车，也可以从后面上车；从前面下车也可以从后面下车。这就是 deque——前面能 push 和 pop，后面也能 push 和 pop。

在 C++ 中 deque 在 `<deque>` 头文件里：

```cpp
#include <deque>
using namespace std;

deque<int> dq;

dq.push_back(1);      // 从尾部插入
dq.push_back(2);      // 尾部再插
dq.push_front(0);     // 从头部插入
// 现在 deque：0, 1, 2

dq.pop_back();        // 尾部删除 → 删掉 2
dq.pop_front();       // 头部删除 → 删掉 0
// 现在 deque：1

cout << dq[0] << endl;  // 支持下标访问！和 vector 一样
```

deque 最特殊的地方是——**它可以像 vector 一样用下标访问**，同时又支持在头部快速插入删除。这意味着 deque 集合了 vector 和 queue 的部分优点。

不过实话实说，在日常竞赛题中，deque 用得没有 stack、queue、vector 那么多。它主要在一个特定场景大放异彩——**单调队列**。

（停顿，观察学生反应）

什么是单调队列？就是队列里的元素始终保持单调递增或递减。当新元素加入时，把队尾那些"不够好"的元素踢掉。这个技巧在求"滑动窗口最大值"这类题目中非常有用——但我们暂时不展开。今天先记住 deque"两头都能操作"这个特点，做个简单的操作练习就够了。

接下来，我们聊聊队列的一个重量级应用——**BFS（广度优先搜索）**。虽然 BFS 的详细内容在下册才会正式讲，但它的骨架非常简单，今天先混个脸熟。

BFS 的核心就是：**用一个队列，把"待处理"的元素依次放进去，按顺序一个一个处理。** 就像超市里排队结账的顾客——先排的先结，结完一个叫下一个。

```cpp
// BFS 骨架（伪代码）
queue<int> q;
q.push(start);           // 起点入队

while (!q.empty()) {
    int cur = q.front(); // 取出当前要处理的
    q.pop();

    // 处理 cur……
    // 把 cur 的所有"邻居"入队
    for (每个邻居 next) {
        q.push(next);
    }
}
```

这个骨架看着简单，但它是图论、搜索算法的基础。它的核心思想是**逐层扩展**——先处理距离起点为 1 的点，再处理距离为 2 的，再处理距离为 3 的……就像往水里扔一块石头，波纹一圈一圈扩散出去。

（起身，在黑板上画图）

比如你要从起点出发找终点，BFS 保证你找到的路径是最短的（步数最少）。为什么？因为它是按"层"搜索的——当你第一次遇到终点时，那一定是在距离最近的那一层。这个性质在竞赛中价值连城。

好，最后我们来做个总结对比——这三节课学的 stack、queue、deque 到底怎么选？

| 场景 | 容器 |
|------|------|
| 撤销操作、括号匹配、递归模拟 | stack |
| 排队模拟、BFS、轮流处理 | queue |
| 两头操作、下标访问、单调队列 | deque |

记住这句话：**后进先出用栈，先进先出用队列，两头都要用 deque，不确定大小用 vector。**

## 🖥️ 板书/屏幕要点

```
deque（双端队列）：
  push_back / push_front
  pop_back  / pop_front
  支持 dq[i] 下标访问
  
  #include <deque>

BFS 骨架：
  queue q; q.push(start);
  while(!q.empty()) {
    cur = front(); pop();
    处理 cur; 把邻居 push 入队;
  }

容器选择速记：
  LIFO → stack
  FIFO → queue
  两头 → deque
  动态数组 → vector
```

## 📌 小结

今天我们认识了 deque——两头都能操作的双端队列，既支持 push/pop 也支持下标访问。同时也初次见识了 BFS 骨架——用队列实现"逐层扩散"的搜索模式。到这儿，线性结构的"三大容器"（栈、队列、双端队列）我们全部学完了。下一章我们进入一个全新的结构——链表，看看没有下标、只能顺着指针走的数据结构能带来什么惊喜。

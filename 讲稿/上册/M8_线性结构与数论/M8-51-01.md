## 📝 讲稿正文

同学们好！今天我们要认识一种新的数据结构——栈。别被"数据结构"这个词吓到，其实你每天都在用，只是没意识到而已。

（停顿，做叠盘子的手势）

想象一下：中午在食堂，阿姨洗好了一摞盘子，一个一个往上叠。你来拿盘子的时候，拿的一定是最上面那个——也就是最后一个放上去的。你不可能从中间抽一个对吧？一是烫手，二是整摞会倒。

这就是栈的规则。它有个洋气的名字叫 LIFO——Last In, First Out。翻译成人话就是：**后进先出**。最后进去的那个，最先出来。

编程里的栈，就是一个"只有顶部开口"的容器。你只能操作最顶上的元素，不能跳过。C++ 给我们提供了一个现成的栈——`stack`，在 `<stack>` 头文件里。

来，我们看代码。（敲黑板）

```cpp
#include <iostream>
#include <stack>
using namespace std;

int main() {
    stack<int> s;        // 声明一个存整数的栈

    s.push(10);          // 压栈：把 10 放进去
    s.push(20);          // 再放 20，现在 20 在顶上
    s.push(30);          // 再放 30，30 是新的顶

    cout << s.top() << endl;   // top() 看栈顶 → 输出 30
    cout << s.size() << endl;  // size() 看有几个 → 输出 3

    s.pop();             // pop() 弹掉栈顶——30 被拿走了
    cout << s.top() << endl;   // 现在栈顶是 20

    cout << s.empty() << endl; // 空吗？不空 → 输出 0（false）

    return 0;
}
```

我们来复盘一下刚刚用到的五个操作。这五个你要像背乘法口诀一样记熟：

**push(x)**——把 x 压入栈顶，就像往盘子上再摞一个。  
**top()**——看一眼栈顶是什么，但不拿。  
**pop()**——把栈顶弹掉，东西就没了。  
**size()**——问一句"你有几个元素"。  
**empty()**——问"你空吗"，空就返回 true。

有一个坑，大家一定要记住：**pop() 没有返回值！** 很多同学想当然地写 `int x = s.pop();`，编译器会报错，因为 pop() 返回的是 void。你要先 `top()` 取值，再 `pop()` 弹掉。

为什么呢？这是 C++ 设计者出于安全考虑——如果 pop() 又弹又返回，万一返回过程中抛异常了，元素已经弹了但值没拿到，就丢了。分开操作，更安全。

（停顿，让学生消化）

现在我们动手验证一个有趣的现象。假设我们依次压入 1、2、3、4、5，然后一个一个弹出输出。输出顺序会是什么？

压入顺序：1 → 2 → 3 → 4 → 5（5 在顶上）  
弹出顺序：5 → 4 → 3 → 2 → 1

发现没有？正好是**反的**！入栈顺序是 1 2 3 4 5，出栈顺序变成了 5 4 3 2 1。这不是巧合——栈天生就能实现"逆序"。这个特性非常有用，等一下我们就会用到。

现在请大家自己在电脑上试试——创建一个 stack，压入 1 到 5，再依次弹出输出。感受一下 LIFO 的节奏。

## 🖥️ 板书/屏幕要点

```
栈 Stack：后进先出 LIFO

操作速记：
  push(x) → 压入栈顶（入栈）
  top()   → 查看栈顶（不删除）
  pop()   → 弹出栈顶（无返回值！）
  size()  → 元素个数
  empty() → 是否为空

包含头文件：#include <stack>
声明：stack<int> s;

⚠️ pop() 无返回值 → 先 top() 再 pop()
⚠️ 操作前先判空 → if (!s.empty())
```

## 📌 小结

今天我们认识了栈——一种"后进先出"的数据结构。它就像一个只有顶部开口的盒子，你只能操作最顶上的元素。五大核心操作 push、pop、top、empty、size 要烂熟于心，特别注意 pop() 没有返回值这个坑。下节课，我们将用栈来解决一个经典问题——括号匹配，看看栈在实际编程中的威力。

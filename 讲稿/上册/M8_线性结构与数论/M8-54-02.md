## 📝 讲稿正文

上节课我们聊了整除和约数。这节课我们聚焦一类特别的数——**质数**。

质数，也叫素数，定义很简单：**大于 1，且只能被 1 和它自身整除的正整数**。比如 2、3、5、7、11、13……

不是质数的、大于 1 的正整数叫**合数**。比如 4、6、8、9、10……

注意：**1 既不是质数也不是合数**。它是"单位元"——这是数学定义，考试会考，大家记住。

（停顿）

质数为什么重要？因为它们像是整数的"原子"——任何大于 1 的整数都可以唯一地分解成质数的乘积。比如 60 = 2×2×3×5。而且 RSA 加密——你们上网时地址栏里那个小锁头——安全性就建立在"把两个大质数乘起来容易，但把乘积分解回两个质数极难"这个事实上。

好，不扯远。回到编程：**怎么判断一个数 n 是不是质数？**

最笨的办法：从 2 到 n-1 每个数试一遍，看能不能整除 n。如果能，就不是质数。这个算法的复杂度是 O(n)——如果 n 是 10¹²，得试一万亿次，程序跑到天荒地老。

能不能优化？上节课刚说过：如果 n 是合数，它一定有一个不超过 √n 的因子。所以只需要试除到 √n：

```cpp
bool isPrime(int n) {
    if (n <= 1) return false;        // 1 不是质数
    for (int i = 2; i * i <= n; i++) {
        if (n % i == 0) return false;
    }
    return true;
}
```

`i * i <= n` 等价于 `i <= sqrt(n)`，但避免了浮点数运算。复杂度从 O(n) 降到 O(√n)——判断 10¹² 需要试约 10⁶ 次（一百万），肉眼可见地快多了。

（停顿，让学生消化）

还能再优化吗？能！既然 2 是唯一的偶质数，我们可以先把偶数（除了 2）全部排除：

```cpp
bool isPrime(int n) {
    if (n <= 1) return false;
    if (n == 2) return true;         // 2 是质数
    if (n % 2 == 0) return false;    // 其他偶数都不是质数
    for (int i = 3; i * i <= n; i += 2) {  // 只试奇数
        if (n % i == 0) return false;
    }
    return true;
}
```

从 3 开始，每次跳 2（i += 2），只试奇数。试除次数又少了一半。对于大多数竞赛题目，这个优化版的试除法已经足够用了。

（转身写板书）

注意一个细节：`i * i <= n` 在 n 很大（比如 10 亿以上）时，`i * i` 可能溢出 int。安全起见，可以用 `long long`：

```cpp
for (long long i = 3; i * i <= n; i += 2) { ... }
```

或者用 `i <= n / i` 来避免乘法溢出：

```cpp
for (int i = 3; i <= n / i; i += 2) { ... }
```

这两种写法效果一样，大家习惯用哪种都行。

（停顿）

好，现在我们有了高效的质数判定器。但如果要判断很多个数（比如 1 到 100 万）呢？每个数都试除一遍到 √n——还是有点慢。有没有更高效的办法？有——**筛法**。先做一张大表格，把合数全部划掉，剩下的就是质数。但筛法属于下册的内容，现阶段先把试除法练熟。

## 🖥️ 板书/屏幕要点

```
质数：大于 1，只被 1 和自身整除 → 2,3,5,7,11,...
合数：大于 1 且不是质数 → 4,6,8,9,10,...
1：既不是质数也不是合数

试除法 O(√n)：
  if(n<=1) return false;
  if(n==2) return true;
  if(n%2==0) return false;
  for(i=3; i*i<=n; i+=2)  // 只试奇数
    if(n%i==0) return false;
  return true;

防溢出：i <= n / i  或  long long
```

## 📌 小结

质数是只能被 1 和自身整除的大于 1 的整数。用试除法判断质数，只需试到 √n，先排除偶数再跳 2 试奇数——O(√n) 的效率处理大数也够用。注意 1 不是质数、2 是特殊偶质数这些边界情况。下节课我们继续深入——把合数拆成质因子的乘积，看看质因数分解怎么写。

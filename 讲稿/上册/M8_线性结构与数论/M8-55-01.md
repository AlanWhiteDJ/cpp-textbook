# 第55章 第1课时：辗转相除法——最大公约数
**时长**：15-20分钟 | **对应教材**：第55章

## 📝 讲稿正文

同学们，今天我们要学一个非常优雅的算法。它诞生于两千三百年前，由古希腊数学家欧几里得写在《几何原本》里。两千多年过去了，它依然是计算机求最大公约数（GCD）的标准算法。这就是——**辗转相除法**。

（在黑板上写）

先看问题本身：给你两个正整数，比如 48 和 18，求它们的**最大公约数**（GCD——Greatest Common Divisor）。也就是说，能同时整除 48 和 18 的最大的那个数。

48 的约数：1, 2, 3, 4, 6, 8, 12, 16, 24, 48  
18 的约数：1, 2, 3, 6, 9, 18  
公共的：1, 2, 3, 6 → 最大的是 6。所以 GCD(48, 18) = 6。

但你能一个一个枚举吗？如果 a 和 b 是一亿和两亿，你要试一亿次——太慢了。

欧几里得发现了一个极其巧妙的关系：

> **gcd(a, b) = gcd(b, a % b)**，当 b ≠ 0。

为什么成立？设 a = q × b + r（其中 0 ≤ r < b），r 就是 a % b。如果某个数 d 能整除 a 和 b，它也能整除 r = a - q × b。反过来，如果 d 能整除 b 和 r，它也能整除 a = q × b + r。所以 **a 和 b 的公约数集合 = b 和 r 的公约数集合**——完全一样！那最大公约数自然也相等。

这个关系把问题不断缩小：gcd(48, 18) → gcd(18, 12) → gcd(12, 6) → gcd(6, 0) = 6。每一步数字都在变小，直到 b 变成 0，a 就是答案。

（转身写代码）

代码简洁到令人惊叹：

```cpp
// 递归版——最优雅
int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);
}

// 迭代版——竞赛常用，避免递归开销
int gcd(int a, int b) {
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}
```

两行代码！但你细品，它背后是两千三百年的数学智慧。

（停顿，让学生跟上演算）

我们手动走一遍迭代版 gcd(48, 18)：

```
a=48, b=18 → b≠0, temp=18, b=48%18=12, a=18
a=18, b=12 → b≠0, temp=12, b=18%12=6,  a=12
a=12, b=6  → b≠0, temp=6,  b=12%6=0,   a=6
a=6,  b=0  → 返回 6
```

三步就算出来了——而我们枚举法要试到 48！

辗转相除法的复杂度是 **O(log min(a, b))** ——每次迭代数字至少减半（粗略估计）。即使 a 和 b 是 10¹⁸ 级别的天文数字，也只需要大约 60 次迭代。这就是算法的力量。

注意迭代版中 `temp = b` 这行不能省。如果你写成：

```cpp
b = a % b;
a = b;  // 此时 b 已经是新值了！错误！
```

顺序反了会出错。拆成三步最安全，或者用 C++ 的 `swap`：`a = b; b = a % b;` 也不太对，因为 `a` 先变了。所以老老实实用一个中间变量。

关于负数：如果传入负数，先取绝对值——`a = abs(a); b = abs(b);`。GCD 在数学上只对正数定义。

## 🖥️ 板书/屏幕要点

```
GCD（最大公约数）：能同时整除 a 和 b 的最大正整数

辗转相除法（欧几里得算法）：
  核心：gcd(a,b) = gcd(b, a%b)，b≠0
  终止：gcd(a,0) = a

迭代版：
  while(b != 0) {
    t = b; b = a % b; a = t;
  }
  return a;

复杂度：O(log min(a,b))
```

## 📌 小结

辗转相除法由欧几里得在公元前三百年提出，至今仍是 GCD 的标准算法。核心关系 gcd(a,b) = gcd(b, a%b) 将问题规模指数级缩小，O(log n) 的复杂度碾压枚举法。递归版和迭代版各有优势——竞赛推荐用迭代版避免递归开销。下节课我们学习最小公倍数（LCM）以及如何求多个数的 GCD/LCM。

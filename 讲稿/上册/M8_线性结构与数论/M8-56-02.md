## 📝 讲稿正文

上节课我们学了加法和乘法原理。今天用乘法原理来推导第一个重要公式——**排列数**。

（在黑板上写）

问题：班里有 5 个同学（A、B、C、D、E），要选 3 个人**排成一排**拍照。有多少种不同的排法？

注意关键词：**排成一排**——顺序很重要！ABC 和 ACB 是不同的排列，因为站的位置不一样。

怎么算？用上节课的乘法原理——分步选：

第 1 个位置：5 个人随便选 → 5 种选择  
第 2 个位置：剩下 4 个人随便选 → 4 种选择  
第 3 个位置：剩下 3 个人随便选 → 3 种选择

每一步都要做 → 乘法：**5 × 4 × 3 = 60 种**。

这个"从 n 个不同元素中取 m 个，按顺序排列"的计算结果，我们记作 **A(n, m)**。有些书上也写作 P(n, m)。

一般公式：

> **A(n, m) = n × (n-1) × (n-2) × … × (n-m+1)**

一共有 m 个因子相乘。也可以写成阶乘形式：

> **A(n, m) = n! / (n-m)!**

比如 A(5,3) = 5! / 2! = 120 / 2 = 60。

代码实现非常直接：

```cpp
// 计算 A(n, m)——注意用 long long 防止溢出
long long A(int n, int m) {
    long long result = 1;
    for (int i = 0; i < m; i++) {
        result *= (n - i);    // 依次乘 n, n-1, n-2, ...
    }
    return result;
}

// A(5, 3) = 5 × 4 × 3 = 60
// A(6, 2) = 6 × 5 = 30
// A(10, 4) = 10 × 9 × 8 × 7 = 5040
```

注意我用了 `long long`——因为排列数增长非常快。A(20, 10) 已经是一个 19 位数，int 根本装不下。所以实际竞赛中，排列组合题几乎都会要求"对某个大质数取模"——我们后面会学。

（停顿）

特别地，当 m = n 时，A(n, n) 就是从 n 个元素中选全部 n 个进行排列——这叫做**全排列**。

> **A(n, n) = n × (n-1) × … × 2 × 1 = n!**

`n!` 读作"n 的阶乘"。比如 3! = 3×2×1 = 6，5! = 5×4×3×2×1 = 120。

那么问题来了：**n! 增长有多快？**

1! = 1  
5! = 120  
10! = 3,628,800  
13! = 6,227,020,800 → 已经超过 int（约 21 亿）的范围  
20! ≈ 2.43×10¹⁸ → 快要超过 long long  

所以如果你在代码里写 `int f = 1; for(int i=2;i<=20;i++) f*=i;`——f 到 13 的时候就溢出了。**大阶乘一定要用 long long 或者取模。**

有个小优化技巧：如果要算 A(n,m) 且 m 比较小，不要先算 n! 再除以 (n-m)!——直接用循环乘 m 次，又简单又不容易溢出。

好，现在大家想一想：A(5,3) = 60 种排列。如果我把"顺序不重要"——ABC 和 ACB 算同一种——那 60 要除以多少？这个问题留到下节课揭晓，它就是**组合数** C(n,m) 的由来。

## 🖥️ 板书/屏幕要点

```
排列 A(n,m)：从 n 个中选 m 个，按顺序排列
  公式：n×(n-1)×...×(n-m+1)  [m 个因子]
  阶乘形式：n! / (n-m)!

  代码：long long res=1;
       for(i=0;i<m;i++) res *= (n-i);
       return res;

全排列：A(n,n) = n!
  3!=6, 5!=120, 10!=3628800
  13! 超 int，20! 近 long long 上限

⚠️ 大阶乘会溢出 → 用 long long 或取模
```

## 📌 小结

排列数 A(n,m) 是乘法原理的直接应用：依次从 n、n-1、n-2……中选人填位置，共乘 m 项。A(n,n) = n! 是全排列。注意大阶乘的增长速度——13! 就超过 int 了，排列组合代码必须注意溢出问题。下节课我们引入组合数 C(n,m)——在排列的基础上除去顺序带来的重复。

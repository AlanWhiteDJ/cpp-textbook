# 第53章 第1课时：火车车厢——链表的概念与遍历
**时长**：15-20分钟 | **对应教材**：第53章

## 🎯 本课时目标
- 理解链表在内存中的离散存储方式，与数组对比
- 掌握单链表节点的结构体定义
- 学会创建节点、链接节点和遍历链表

## 📝 讲稿正文

同学们，前几节课我们学的栈和队列，底层都依赖数组——数据在内存里是**连续排列**的，像连排别墅一样紧挨着。今天我们要学一种完全不同的数据组织方式——**链表**。

（在黑板上画一列火车）

想象一列火车，每节车厢独立存在，通过连接扣挂在一起。你想在中间加一节车厢——把连接扣解开，挂上新车厢，再把后面的扣上。只需要**改两个连接扣**，不需要移动后面所有的车厢。

数组如果要在中间插入一个元素呢？不好意思，后面所有元素都要往后挪一位——数据量大的时候非常慢。

这就是链表的优势：**插入删除不需要整体搬家**。

链表在内存里的样子：每个节点（就是一节"车厢"）可以在任意位置，通过一个"小箭头"——指针——指向下一个节点。你不需要把大家排成整齐的一排，只需要每个人都记住"下一个是谁"就行了。

（转身写代码）

C++ 中，我们用一个结构体来表示链表的一个节点：

```cpp
struct Node {
    int data;       // 数据域：这个节点存的值
    Node* next;     // 指针域：指向下一个节点的地址
};  // ← 别忘了分号！
```

注意这里 `Node* next` —— 结构体里包含了一个指向同类型结构体的指针。这在 C++ 里完全合法，而且是链表的精髓：**每个节点知道自己后面是谁**。

来，我们手动创建几个节点，把它们串起来：

```cpp
// 创建头节点
Node* head = new Node;
head->data = 10;
head->next = nullptr;    // 先让它指向"空"——链表暂时只有它一个

// 创建第二个节点
Node* second = new Node;
second->data = 20;
second->next = nullptr;

// 连起来！
head->next = second;     // 头节点指向第二个
// 现在的链：10 → 20

// 创建第三个节点，插入到中间
Node* mid = new Node;
mid->data = 15;
mid->next = head->next;  // 新节点先指向后面的（20）
head->next = mid;        // 前面的指向新节点
// 现在的链：10 → 15 → 20
```

这里有两件事要特别注意。第一，`new` 是在"堆"上分配内存，返回的是地址。出了大括号也不会自动消失——需要用 `delete` 手动释放。第二，插入时**指针修改的顺序**——一定要先让新节点指向后继，再让前驱指向新节点！顺序反了你就把后面的链给丢了。

（停顿，让学生消化）

好，链表建好了，怎么遍历它？

```cpp
void printList(Node* head) {
    Node* cur = head;           // cur 从头开始走
    while (cur != nullptr) {    // 没到末尾就继续
        cout << cur->data << " ";
        cur = cur->next;        // 走一步
    }
    cout << endl;
}
```

`cur->data` 等价于 `(*cur).data`——通过指针访问结构体成员。`cur = cur->next` 就是顺着"连接扣"走到下一个节点。

遍历条件是 `cur != nullptr`，不是 `cur->next != nullptr`。如果用后者，最后一个节点的值会被漏掉——这个错误非常常见，大家注意。

链表的核心体验就是"顺着走"——不能像数组那样 `a[5]` 直接跳转到第 5 个元素。你要从 head 出发，一步一步走到第 5 个。**随机访问 O(n)，这是链表最大的代价。**

## 🖥️ 板书/屏幕要点

```
链表节点：
  struct Node {
    int data;     // 数据
    Node* next;   // 指向下一个的指针
  };

创建节点：new Node
访问成员：cur->data, cur->next
遍历：while(cur != nullptr) { cur = cur->next; }

插入要点：
  ① newNode->next = head->next;  // 先连后继
  ② head->next = newNode;        // 再改前驱

数组 vs 链表：
  数组：连续内存 → 随机访问 O(1)，插入删除 O(n)
  链表：离散内存 → 随机访问 O(n)，插入删除 O(1)
```

## 📌 小结

链表是一种离散存储的线性结构，每个节点包含数据和指向下一个节点的指针。它的最大优势是插入删除 O(1)，代价是随机访问 O(n)。今天学会了节点定义和遍历，下节课我们系统学习链表的插入和删除操作——这是链表的看家本领。

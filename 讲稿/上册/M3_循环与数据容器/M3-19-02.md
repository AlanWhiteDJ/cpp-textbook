# 第19章 第2课时：cin与getline——两种读法，各有讲究
**时长**：15-20分钟 | **对应教材**：第19章 string入门

## 🎯 本课时目标
- 掌握`cin >>`和`getline`两种字符串输入方式
- 理解两者的核心区别：空格处理
- 学会解决混用时的"残留换行符"问题

## 📝 讲稿正文

同学们，上节课我们学会了声明和初始化string。现在我们来学怎么从键盘读入字符串——这是日常编程里最高频的操作之一。

C++读字符串有两种方式，它们的行为不一样，用错了会出Bug。

**第一种：cin >>。**

```cpp
string name;
cin >> name;
cout << "你输入的是：" << name << endl;
```

这个写法跟读int一模一样。但有一个关键特性——`cin >>`读到**空格、换行、制表符**就会停下来。

假如用户输入`Zhang Xiao Ming`，你猜name里存的是什么？是`Zhang`，不是`Zhang Xiao Ming`。因为cin一碰到空格就认为"这个词读完了"，后面的`Xiao Ming`还留在输入缓冲区里没读。

为什么呢？cin的设计理念是"读一个词"，而空格是词与词之间的分隔符。所以它读到分隔符就停。

那如果用户的名字真的带空格怎么办？比如"欧阳小小"——中文名还好没空格，但"John Smith"就有空格了。

这时候用**第二种方式：getline。**

```cpp
string line;
getline(cin, line);  // 读一整行，包括空格，直到换行符
cout << "你输入的是：" << line << endl;
```

`getline`会读入一整行，不管中间有多少空格，全部照单全收，直到碰到换行符（用户按了回车）才停。

所以规则很简单：**读一个词用cin >>，读一整行用getline。**

好，现在来个有点绕的——**混用问题**。

想象你写一个程序，先读年龄（int），再读名字（string，可能带空格）：

```cpp
int age;
string name;

cout << "请输入年龄：";
cin >> age;

cout << "请输入姓名：";
getline(cin, name);

cout << "年龄：" << age << "，姓名：" << name << endl;
```

你运行程序，输入：
```
12
张三
```

你期望输出"年龄：12，姓名：张三"。但实际运行的结果是——name是空的！程序直接跳过了姓名输入。

这是C++新手遇到最多的坑之一。为什么？因为`cin >> age`读取了`12`，但用户按的回车（换行符`\n`）还留在输入缓冲区里。接下来`getline`一上来就碰到了这个换行符，立刻认为"这行读完了，是个空行"，于是读了一个空字符串回去。

就好像你吃完饭碗里留了一粒米——下一个用碗的人看到的不是空碗，而是"碗里有东西"。

怎么解决？在`cin >>`和`getline`之间加一条`cin.ignore()`：

```cpp
cin >> age;
cin.ignore();           // 吞掉残留的换行符
getline(cin, name);     // 现在能正常读了
```

`cin.ignore()`的作用就是"忽略输入缓冲区里的下一个字符"——把那个讨厌的换行符吞掉。

如果你连续读了好几个数字才读字符串，可能需要`cin.ignore()`多次，或者用`cin.ignore(100, '\n')`——忽略直到换行符为止的所有内容。

好，现在总结一下今天的内容：读一个词用cin >>，读一整行用getline。两者混用时，在cin >>后面加cin.ignore()清缓冲。这个坑你早晚会踩到——但既然我已经告诉你了，你就比别人少踩一次。

## 🖥️ 板书/屏幕要点
- **cin >> s**：读一个词，遇空格/换行/制表符停止
- **getline(cin, s)**：读一整行，遇换行符停止
- 选择规则：带空格的文字 → getline；单个词 → cin >>
- **混合使用陷阱**：cin >> 残留换行符，getline读到空行
- **解决方案**：`cin >> x; cin.ignore(); getline(cin, s);`
- `cin.ignore()` = 吞掉缓冲区里下一个字符

## ✏️ 随堂练习
写一个程序，要求用户先输入年龄（int），再输入一句话（带空格，如"我喜欢C++"）。使用cin.ignore()解决换行符问题。确认两样都能正确读入并输出。然后去掉cin.ignore()，观察getline读到了什么。

## 📌 小结
字符串输入有两种方式：cin >> 读一个词，getline读一整行。两者混用时，cin >> 后面的getline会因为残留换行符而读到空行——这个坑要用cin.ignore()来填。这个知识点虽小，但如果不注意，你调试半天找不到问题出在哪。

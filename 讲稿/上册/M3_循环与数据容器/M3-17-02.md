## 📝 讲稿正文

同学们，上节课我们学了求和和最值。今天我们来学第三件统计工具——**计数**。

什么场景需要计数？"全班90分以上的有多少人？""这个字符串里有多少个数字？""投票结果——每个候选人各得几票？"这些问题本质都是数个数。

最简单的计数——数满足条件的元素有几个：

```cpp
int a[8] = {85, 92, 78, 95, 88, 90, 73, 91};
int cnt = 0;                         // 计数器
for (int i = 0; i < 8; i++) {
    if (a[i] >= 90) {
        cnt++;
    }
}
cout << "90分以上人数：" << cnt << endl; // 4
```

这个模式很直白——设一个计数器变量cnt初始为0，遍历数组，每遇到一个符合条件的就cnt++。遍历完了，cnt就是要的答案。

好，这个大家应该都能掌握。现在我们来看一个更有趣的——**桶计数**。

场景是这样的：一个班级投票选班长，有5个候选人，编号1到5。全班40个人投票，你拿到的是一串数字：`{3, 1, 2, 3, 5, 3, ...}`。现在要统计每个人得了多少票。

你可以用五个计数器cnt1、cnt2、cnt3、cnt4、cnt5。但如果候选人有100个呢？100个计数器显然不行。

这时候，"桶思想"就登场了。思路非常巧妙——**用数组的下标代表"哪个候选人"，用数组的值代表"得了多少票"**。

```cpp
int votes[10] = {3, 1, 2, 3, 5, 3, 1, 4, 5, 3};
int bucket[6] = {0};  // 下标1~5对应候选人1~5，bucket[0]空着不用

for (int i = 0; i < 10; i++) {
    int v = votes[i];   // 第i个人投给了v号候选人
    bucket[v]++;        // v号候选人的桶里+1
}

// bucket[1]=2, bucket[2]=1, bucket[3]=4, bucket[4]=1, bucket[5]=2
cout << "3号候选人得票：" << bucket[3] << endl;  // 4票
```

大家仔细品味这个设计——`bucket[3]`的值是什么？是3号候选人得的票数。数组下标天然就是一个整数编号，而我们要统计的东西刚好也有编号——天作之合！

这个思想的威力比看起来大得多。再举个例子——**统计每个分数各有几个人**：

```cpp
int scores[10] = {85, 92, 78, 95, 88, 90, 73, 91, 88, 79};
int bucket[101] = {0};               // 成绩0~100，共101个桶

for (int i = 0; i < 10; i++) {
    bucket[scores[i]]++;             // 在对应的分数桶里+1
}

cout << "考88分的有" << bucket[88] << "人" << endl;  // 2人
cout << "考95分的有" << bucket[95] << "人" << endl;  // 1人
```

看，`bucket[88]`的值就是考88分的人数。这个巧妙的映射关系——分数本身变成了数组下标——让统计变得极其简单。

桶思想的核心就一句话：**用数组下标表示"类别"，用数组的值表示"该类别出现的次数"**。这是竞赛里桶排序的基础，后面你还会学到完整的桶排序算法。

好，有两个容易出错的地方提醒大家。第一，桶数组必须初始化——`int bucket[101] = {0}`，不能忘了写`= {0}`，否则桶里都是垃圾值，计数全乱套。第二，桶数组的大小要够——如果分数范围是0到100，桶的大小至少101。如果只声明了100个元素，bucket[100]就访问不到了。

现在来做练习——模拟一个投票场景，用桶计数统计票数并找出票数最高的候选人。

## 🖥️ 板书/屏幕要点
- **条件计数**：`cnt=0; for(...) if(条件) cnt++;`
- **桶计数思想**：下标=类别，值=出现次数
- 关键写法：`bucket[要统计的值]++`
- 桶数组必须：①初始化`{0}` ②大小≥最大类别+1
- 应用场景：投票统计、成绩分布、字母频率统计

## ✏️ 随堂练习
模拟一个班级的投票——数组里存20个数字（1~5），每个数字代表投给一个候选人。用桶计数统计每个人得了多少票，并找出票数最高的候选人（如果有并列，输出编号最小的）。

## 📌 小结
计数统计有两种模式：简单条件计数（一个cnt变量），和桶计数（用数组下标做分类）。桶思想特别巧妙——利用数组下标天然有序的特点，把"类别编号"映射为下标，"出现次数"映射为值。记住两个要点：桶数组必须初始化为{0}，桶数组大小必须覆盖所有可能的类别编号。

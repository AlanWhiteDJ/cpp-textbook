## 📝 讲稿正文

同学们，今天是我们第17章的最后一节课。前面四节课我们学了求和、最值、桶计数、插入、删除、反转——每一样都是数组上的"独门武功"。今天我们要学最后一样——**去重**，然后来一场综合挑战。

什么叫去重？就是给你一堆可能有重复的数字，你要输出"每个值只出现一次"的结果。比如`{3, 1, 4, 1, 5, 9, 3, 4}`，去重后变成`{3, 1, 4, 5, 9}`——每个数字只保留第一次出现。

这个问题的朴素解法是"双重循环法"——用一个新数组存"还没出现过"的元素。遍历原数组，对每个元素检查它是否已经在新数组里存在了：没出现就加进去，出现了就跳过。

```cpp
int a[8] = {3, 1, 4, 1, 5, 9, 3, 4};
int n = 8;
int b[8];       // 存去重后的结果
int m = 0;      // b的元素个数

for (int i = 0; i < n; i++) {
    bool found = false;
    // 在b里找a[i]是否已经存在
    for (int j = 0; j < m; j++) {
        if (b[j] == a[i]) {
            found = true;
            break;      // 找到了，不用继续找
        }
    }
    if (!found) {
        b[m] = a[i];    // 没找到，加进b
        m++;
    }
}

// 输出去重结果：3 1 4 5 9
for (int i = 0; i < m; i++) {
    cout << b[i] << " ";
}
```

大家跟着我的思路走一遍。i=0时，a[0]=3。b是空的，found=false，所以b[0]=3，m=1。i=1时，a[1]=1。b里只有一个3，找不到1，所以b[1]=1，m=2。i=2时，a[2]=4，同样加进去。i=3时，a[3]=1——在b里找到了！found=true，跳过。就这样，3、1、4、5、9各出现一次。

这个方法的核心是两个嵌套的循环——外层遍历原数组，内层遍历新数组。时间复杂度是O(n²)，对于大数据量不太高效，但对于现阶段学习来说，它思路清晰、容易理解。等你学到排序，你会遇到更高效的去重方法——先排序，然后一遍扫描就能去重。但今天我们先从最朴素的方法开始，建立直觉。

好，去重学完了。现在我们来回顾一下，这个第17章你一共掌握了哪些技能？

1. 求和、平均值——累加器模式
2. 最大值、最小值——擂台法
3. 桶计数——下标=类别，值=次数
4. 插入——从后往前移
5. 删除——从前往后移
6. 反转——首尾交换，只换一半
7. 去重——双重循环检查

这七种技能，每一种都是对数组"遍历"的不同运用。同一个数组，换一种遍历方式就能回答一个完全不同的问题。就像同一把刀，切不同的菜需要不同的刀法。

现在我们来做一道综合题，把几项技能串起来。题目是这样的：输入n个学生的成绩（0~100），存入数组。然后输出以下内容：平均分、最高分、最低分、及格人数（≥60）、优秀人数（≥90）。

这道题需要同时用到求和、最值、计数——正好是今天复习的重点。

## 🖥️ 板书/屏幕要点
- **去重思路**：检查每个元素是否已在"不重复集合"中出现过
- 双重循环：外循环遍历原数组，内循环检查是否重复
- `bool found`标记 + `break`提前跳出
- 去重三步骤：检查 → 判断 → 加入或跳过
- 本章七技能一览：求和/最值/桶/插入/删除/反转/去重

## ✏️ 随堂练习
输入n（不超过100）和n个0~100的整数成绩，存入数组，输出以下五项统计：总分、平均分（保留1位小数）、最高分、最低分、优秀人数（≥90）。要求用一个for循环完成所有统计。

## 📌 小结
去重是数组处理中的常见需求。朴素的双重循环方法思路清晰但效率不高，适合初学者建立直觉。回顾整个第17章，我们学了七种数组操作技能——它们都是"遍历"这把刀的不同刀法。关键不在于记住多少种，而在于面对新问题时，能快速判断"我该用哪种遍历策略"。下一章，我们要给数组增加第二个维度——二维数组，编程世界的Excel表格。

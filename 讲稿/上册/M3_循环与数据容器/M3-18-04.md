## 🎯 本课时目标

## 📝 讲稿正文

同学们，前面三节课我们把二维数组的基本操作都过了一遍——声明、遍历、转置、对角线。今天我们来学两个进阶概念：二维数组怎么传给函数，以及三维数组是什么。

先说函数传参。假设你写了一个打印矩阵的函数：

```cpp
void printMatrix(int a[][4], int rows) {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < 4; j++) {
            cout << a[i][j] << " ";
        }
        cout << endl;
    }
}
```

注意看参数——`int a[][4]`。第一个方括号里是空的，但**第二个方括号必须写数字**！你不能写成`int a[][]`。

为什么？因为计算机要知道每行有多少列，才能算出`a[i][j]`在内存里的位置。

让我们来理解一下。二维数组在内存里实际上是一维排布的——第一行的所有元素紧挨着，然后是第二行、第三行……要找到a[2][3]，计算机需要计算：跳过前2整行（每行4个元素，共跳过8个），再在当前行里跳过3个。计算公式是：`地址 = 首地址 + (i × 列数 + j) × 每个元素的大小`。

看到了吗？列数必须已知！所以传参时必须指定列数。

常见的错误写法是：

```cpp
void func(int a[][], int n, int m) {  // ❌ 编译错误！
    ...
}
```

正确的做法有两种：

**做法一：列数写死。**

```cpp
void func(int a[][100], int n, int m) { ... }
```

**做法二：用全局常量。**

```cpp
const int MAXN = 100;
int a[MAXN][MAXN];

void process(int a[][MAXN], int n, int m) { ... }
```

竞赛中最常用的是全局常量法——在程序开头定义一个`const int MAXN = 1005`，然后所有函数参数都用这个常量。

好，二维数组的函数传参搞定了。现在我们来拓展一下视野——**三维数组**。

二维数组是"表格"，那三维数组就是"一摞表格"。声明方式：

```cpp
int cube[3][4][5];  // 3层 × 4行 × 5列
```

可以想象成3张4×5的表格叠在一起。访问方式加一层下标：

```cpp
cube[0][1][2] = 42;  // 第0层、第1行、第2列
```

遍历三维数组需要**三重循环**：

```cpp
for (int k = 0; k < 3; k++) {         // 层
    for (int i = 0; i < 4; i++) {     // 行
        for (int j = 0; j < 5; j++) { // 列
            cout << cube[k][i][j] << " ";
        }
        cout << endl;
    }
    cout << "---" << endl;  // 层之间隔开
}
```

三维数组的典型应用场景有哪些呢？比如："一个学校有3个年级，每个年级4个班，每个班5次考试"——正好是3×4×5的三维结构。再比如"3D游戏里的体素地图"、"RGB图片的三个颜色通道"。

对于信息学竞赛来说，三维偶尔会用到，四维及以上就很少见了。现阶段你只需要掌握二维数组，知道三维的概念就行。别被维度吓到——不管几维，本质上都是在内存里平铺的一维数据，只是下标多了一个。

## 🖥️ 板书/屏幕要点
- **二维数组传参必须指定列数**：`void f(int a[][列数], int 行数)`
- 两种方案：①写死 `int a[][100]` ②全局常量 `const int MAXN = 100; void f(int a[][MAXN],...)`
- 内存地址计算：`a[i][j]地址 = 首地址 + (i×列数+j)×元素大小`
- **三维数组**：`类型 数组名[层][行][列]`
- 三维遍历：三重循环（外层循环→行→列）
- 现阶段：主攻二维，了解三维即可

## ✏️ 随堂练习
写一个函数`int sumMatrix(int a[][100], int n, int m)`，计算并返回n×m矩阵中所有元素的总和。在main函数里创建一个3×4的矩阵并初始化，调用这个函数输出总和。注意：使用`const int MAXN = 100`全局常量来定义列数参数。

## 📌 小结
二维数组传参时，列数必须明确——要么写死一个常数，要么用全局常量。这个语法限制的底层原因是内存地址计算的数学需要。三维数组是"一摞表格"，遍历需要三重循环。虽然维度可以不断增加，但核心思想不变——每一维都是一个下标，在内存里最终都是一维排布。

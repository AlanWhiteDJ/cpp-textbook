# 第16章 第2课时：下标从0开始——数组的"门牌号"系统
**时长**：15-20分钟 | **对应教材**：第16章 一维数组

## 🎯 本课时目标
- 彻底理解数组下标从0开始的规则
- 牢记"n个元素，下标范围是0到n-1"
- 认识数组越界的严重危害，养成防越界的好习惯

## 📝 讲稿正文

上节课我们认识了数组——一排储物柜。现在我们来聊聊怎么找到每一个柜子。

你看，小区快递柜上面有什么？有编号。0号柜、1号柜、2号柜……一直到19号柜。在C++里，这个编号叫做**下标**，英文叫index。

好，第一个关键问题来了——C++的数组下标从几开始？

很多同学会说："从1开始呗，第一个柜子不就是1号吗？"

错！**C++的数组下标从0开始**。0号柜是第一个，1号柜是第二个，2号柜是第三个……以此类推。

我们来看代码：

```cpp
int arr[5];              // 5个柜子
arr[0] = 10;             // 第1个柜子放10
arr[1] = 20;             // 第2个柜子放20
arr[2] = 30;             // 第3个柜子放30
arr[3] = 40;             // 第4个柜子放40
arr[4] = 50;             // 第5个柜子——注意！这是最后一个！
```

大家仔细看——`arr[5]`声明了5个元素，最后一个有效下标是4，不是5！

这个规律用一个公式来记：**一个有n个元素的数组，下标范围是0到n-1。不存在下标为n的元素。**

为什么C++要从0开始？这跟内存地址计算有关。数组第一个元素放在某个内存地址上，数组名本身就指向这个地址。arr[0]就是地址+0个偏移量，arr[1]就是地址+1个偏移量……这样计算起来最高效。不过现阶段你只需要记住"从0开始"就行。

好，知道了下标规则，咱们来看一个最容易犯的错误——**越界**。

什么叫越界？就是你声明了`int arr[5]`，但你去访问`arr[5]`或者`arr[100]`。数组只有0到4号柜子，你去开5号柜子——这个柜子根本不存在！

越界会发生什么？答案是——说不准。C++不会帮你检查下标是否合法。它可能静悄悄地改掉别的变量的值，可能让程序直接崩溃，也可能看起来一切正常但埋下了一个定时炸弹。这三种结果里，最可怕的是第三种——你以为没问题，交上去OJ给你一个Wrong Answer，你找半天找不出bug在哪里。

怎么避免越界？教大家几个好习惯：

第一，定义数组时多给几个位置。题目说最多100个，我声明105个，留缓冲。

第二，写循环的时候特别小心。我们来看：

```cpp
int arr[5] = {10, 20, 30, 40, 50};

// ❌ 错误写法：
for (int i = 0; i <= 5; i++) {  // i会变成5，arr[5]越界！
    cout << arr[i] << " ";
}

// ✅ 正确写法：
for (int i = 0; i < 5; i++) {   // i最多到4，安全
    cout << arr[i] << " ";
}
```

大家注意看区别在哪里——小于等于和小于，就一个等号的差别，结果天差地别。

我们有一个口诀，大家跟我念一遍：**n个元素，i从0到n-1，循环条件是i < n。**

再念一遍：**n个元素，i从0到n-1，循环条件是i < n。** 这个口诀你要背下来，每次写循环的时候在心里默念一遍。

还有一个容易踩的坑——新手经常把下标当"第几个"。arr[1]是第一个元素吗？不对！arr[1]是第二个元素，arr[0]才是第一个。混淆了这一点，写出来的程序全是偏移一位的错误。

好，我们来动手试试。请大家写一段代码，故意访问arr[10]——前提是你只声明了10个元素。看看你的编译器会不会警告你，程序跑起来会发生什么。这是为了让你亲眼见证越界的危险。记住，正式写代码的时候绝对不能这么做，这只是"安全教育"。

## 🖥️ 板书/屏幕要点
- **下标从0开始，末元素是n-1**
- 口诀：n个元素，i从0到n-1，循环条件i < n
- 越界 = 访问arr[n]或更大 = 定时炸弹
- 正确循环：`for (int i = 0; i < n; i++)`
- 错误循环：`for (int i = 0; i <= n; i++)` ← 多等号，越界！
- 好习惯：声明时多留几个缓冲位置

## ✏️ 随堂练习
写一段代码，声明`int arr[6] = {10, 20, 30, 40, 50, 60}`，用两种循环条件分别输出：先写正确版本（`i < 6`），再写一个"故意错误"版本（`i <= 6`），观察两次输出的区别。记录错误版本在越界位置输出了什么奇怪的值。

## 📌 小结
今天最重要的一件事：记住下标从0开始。n个元素的数组，有效下标是0到n-1。循环条件永远写i < n，不写i <= n。数组越界是C++新手最危险也最常见的错误——编译器不帮你检查，后果不可预测。从现在开始，每次写循环，都在心里默念"i < n"。

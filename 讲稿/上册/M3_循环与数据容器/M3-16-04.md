# 第16章 第4课时：初始化与常见陷阱——别让垃圾值坑了你
**时长**：15-20分钟 | **对应教材**：第16章 一维数组

## 🎯 本课时目标
- 掌握数组的各种初始化方式及其区别
- 识别并避开数组使用的五大常见陷阱
- 养成"先初始化再使用"的编程习惯

## 📝 讲稿正文

同学们，前面三节课我们学会了数组的声明、下标和遍历。今天这节课特别重要——我们要聊聊数组的"坑"。这些坑都是我见过无数新手摔进去的地方。你提前知道了，就能绕过去。

第一个坑，也是最常见的——**数组没初始化就读**。

你声明了一个数组`int arr[10]`，然后马上`cout << arr[3]`。输出什么？不知道！可能是0，可能是32768，可能是上次程序残留的任何数字。我们管这种东西叫"垃圾值"。

为什么会这样？因为声明数组只是跟计算机说"给我留10个int的空间"，但空间里原来有什么东西，计算机不管。就像你去住酒店，房间是给你了，但上一位客人可能留了东西——你得先打扫。

所以，**要么声明时初始化，要么先赋值再读**。这是铁律。

怎么初始化呢？我们来看几种方式：

**方式一：全部指定。**

```cpp
int arr[5] = {10, 20, 30, 40, 50};
```

5个元素全给了值，清楚明白。

**方式二：指定前几个，其余自动补0。**

```cpp
int arr[5] = {10, 20, 30};
// arr[0]=10, arr[1]=20, arr[2]=30, arr[3]=0, arr[4]=0
```

这个特性很实用——你只需要给你关心的几个值，剩下的自动清零。

**方式三：全部清零——竞赛最常用的写法。**

```cpp
int arr[100] = {0};
```

100个元素全部变成0，干净利落。这是在OJ刷题时最常用的初始化方式，请一定要记住。

**方式四：让编译器数大小。**

```cpp
int arr[] = {15, 25, 35, 45};
```

不写大小，编译器自动数——哦，4个元素。方括号里不用填数字。

好，现在我们来说第二个坑——**`{1}`不等于全部初始化为1**。

很多同学写了`int arr[100] = {1}`，以为100个元素全变成1了。但实际上，只有arr[0]是1，后面99个全是0！`{1}`的意思是"第一个元素是1，其余默认补0"。

那怎么把100个元素全初始化为1呢？用循环：

```cpp
int arr[100];
for (int i = 0; i < 100; i++) {
    arr[i] = 1;
}
```

第三个坑——**声明数组时用变量做大小**。有些同学会这样写：

```cpp
int n;
cin >> n;
int arr[n];  // 不符合C++标准！
```

你可能会说"我在Dev-C++上能跑啊"。对，有些编译器允许，但这不是标准C++。换一个编译器、换一个评测平台，可能就编译不过了。正确的做法是用常量或者直接给一个足够大的范围。

第四个坑我们前面提过——**把下标当第几个**。arr[1]是第二个元素，不是第一个。这个错一次就记住了，但第一次总会有人犯。

第五个坑——**循环条件多写一个等号**。`i <= n`而不是`i < n`，导致访问arr[n]越界。这个Bug极其隐蔽，因为有时候程序不崩溃，只是输出了一个奇怪的值。

好，现在我们来做个练习，综合运用今天学的内容。请你声明一个长度为20的int数组，用`{0}`全部清零，然后用循环把前10个分别赋值为10、20、30……100，最后输出整个数组，观察后10个元素是否还是0。

## 🖥️ 板书/屏幕要点
- **五大陷阱**：
  1. 未初始化就读 → 垃圾值
  2. `{1}` ≠ 全部初始化为1 → 只有arr[0]=1
  3. `int arr[n]` → 不符合C++标准
  4. arr[1]是第二个 → 下标从0开始
  5. `i <= n` → 越界
- 正确初始化：`int arr[100] = {0};`（全部清零）
- 防护口诀：先初始化，再使用；多留缓冲，不越界

## ✏️ 随堂练习
写一段代码，声明`int arr[10]`但不初始化，输出每个元素的值，观察垃圾值长什么样。然后把代码改成`int arr[10] = {0}`，再输出一次。体会初始化和不初始化的区别。

## 📌 小结
今天这节课是"避坑指南"。数组使用中有五个经典陷阱：不初始化、误解{1}的含义、用变量做大小、混淆下标、循环多写等号。每一条都可能让你的程序悄无声息地出错。记住两个好习惯：数组声明就初始化为{0}，循环条件永远写i < n。这两条做到了，一半的数组Bug就不会找上你。

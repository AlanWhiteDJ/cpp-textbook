## 📝 讲稿正文

同学们，上一章我们把数据装进了数组这个"储物柜"里。但光存着不处理，就像把买回来的菜全塞冰箱里不做饭——冰箱再满也吃不到嘴里。

今天开始，我们要学"怎么做饭"——怎么在数组上做各种统计和变换。

我们来看一个场景。班长小李手里有全班40个人的数学成绩单。老师问他三个问题：平均分多少？最高分最低分是谁？90分以上有几个人？

小李做了一件很聪明的事——他把所有成绩抄在一张纸上，然后开始**唱票**。

第一遍唱票，他从第一个成绩划到最后一个，嘴里念着"85、加92、加78……"，最后除以40，平均分出炉。这个过程，就是**求和**。

在代码里怎么写呢？

```cpp
int a[5] = {89, 92, 78, 95, 88};
int sum = 0;                         // 累加器，从0开始
for (int i = 0; i < 5; i++) {
    sum += a[i];                     // 也可以写 sum = sum + a[i];
}
cout << "总分：" << sum << endl;      // 442
cout << "平均：" << sum / 5.0 << endl; // 88.4
```

求和的核心就是一个"累加器"——声明一个变量sum初始化为0，然后遍历数组，每碰到一个元素就加到sum里。遍历完了，sum就是总和。除以元素个数就是平均值。

这里有个小细节：`sum / 5.0`，5后面有个.0。这是为了做浮点数除法，保留小数。如果写`sum / 5`，整数除整数，小数部分就丢掉了。

好，第一遍唱票完成了。现在第二遍——**找最高分**。小李这次不看总分，而是盯"最大的那个"。"85？先记着。92？更大，换掉。78？不如92，跳过……"扫完，最高分出炉。

这种思路叫**"擂台法"**——先让第一个元素当"擂主"，然后遍历剩下的元素，谁比擂主大谁就当新擂主。

```cpp
int maxVal = a[0];                   // 先假设第一个是最大的
for (int i = 1; i < 5; i++) {        // 从第二个开始比
    if (a[i] > maxVal) {
        maxVal = a[i];               // 谁大就替换
    }
}
cout << "最高分：" << maxVal << endl;  // 95
```

注意，循环从i=1开始，不是i=0。因为maxVal已经存了a[0]的值，不需要自己跟自己比。

找最小值就是把大于号改成小于号，道理一样。

这里有一个非常容易踩的坑——**把最大值初始值设成0**。

```cpp
int maxVal = 0;                    // 危险！
int a[5] = {-10, -5, -20, -8, -15};
for (int i = 0; i < 5; i++) {
    if (a[i] > maxVal) maxVal = a[i];
}
cout << maxVal << endl;            // 输出0，但正确答案是-5！
```

为什么？因为数组里全是负数，没有一个比0大，所以"擂主"0从头赢到尾——可这个擂主根本不在数组里，是个"假冠军"！

正确的做法是什么？初始值设为`a[0]`——让数组自己的元素当擂主，这样无论数组里是正是负，都不会出错。

口诀：**初始值设a[0]，不是设0。**

好，今天我们学了数组统计三件套的前两件——求和与最值。下节课我们学第三件——计数统计和桶思想。

## 🖥️ 板书/屏幕要点
- **求和模式**：`sum = 0; for(i=0; i<n; i++) sum += a[i];`
- **平均值**：`(double)sum / n`（注意保留小数）
- **最大值模式**：`maxVal = a[0]; for(i=1; i<n; i++) if(a[i]>maxVal) maxVal=a[i];`
- ⚠️ 最值初始值必须设a[0]，不能设0！
- **累加器思想**：从0开始，逐个累加
- **擂台法思想**：从第一个开始，逐个比较替换

## ✏️ 随堂练习
定义一个int数组`{65, 34, 87, 23, 98, 56, 72, 41}`，分别求最大值、最小值、平均值。平均值保留2位小数。然后自己验证：如果把求最大值的初始值故意设成0，会不会出错？（这个数组不会，为什么？再试试全负数的数组。）

## 📌 小结
求和、平均值、最大值、最小值是数组统计的四大基本操作。求和用累加器（初始化为0），最值用擂台法（初始值必须设a[0]）。这四个操作共享同一个模式——初始化一个变量，遍历数组，逐个更新。遍历是同一把刀，切不同的菜靠的是不同的刀法。

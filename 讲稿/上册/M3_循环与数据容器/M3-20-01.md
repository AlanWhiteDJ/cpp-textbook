## 📝 讲稿正文

同学们，上一章我们学会了string的"基本功"——声明、输入、拼接、求长度。今天开始，我们要升级了——学string的"编辑功能"。第一件武器：**find**。

find是在一大段文字里找一小段文字。比如在"Hello World"里找"World"在第几个位置。C++的find返回找到的位置（下标），如果找不到就返回一个特殊值。

来看代码：

```cpp
string s = "I love programming, love C++";

int pos = s.find("love");          // 从开头找"love"
cout << pos << endl;               // 输出2——"love"从下标2开始

pos = s.find("love", 5);           // 从下标5开始找
cout << pos << endl;               // 输出18——第二个"love"在下标18

pos = s.find("Python");            // 找一个不存在的
if (pos == string::npos) {
    cout << "没找到" << endl;       // 输出这个
}
```

我们来仔细分析。`s.find("love")`从字符串开头（默认位置0）往后找，在位置2找到了"love"——因为"I "占了下标0和1（I和空格），"love"的"l"在位置2。

`find`可以带第二个参数——从哪个位置开始找。`s.find("love", 5)`就是从下标5开始找，跳过第一个"love"，找到第二个"love"在下标18。

没找到的时候，find不返回-1，而是返回一个特殊常量：**`string::npos`**。这个常量的全称是"no position"——没有这个位置。

很多同学会写成`if (s.find("abc") == -1)`，虽然有些编译器能通过，但这不规范。标准写法是`if (s.find("abc") == string::npos)`。从现在开始就用标准写法，养成好习惯。

好，掌握了find基本用法后，我们来做一件更有意思的事——**统计某个词出现了多少次**。

```cpp
string s = "apple banana apple orange apple";
string target = "apple";
int cnt = 0;
int pos = 0;

while ((pos = s.find(target, pos)) != string::npos) {
    cnt++;
    pos += target.size();  // 跳过已找到的，继续往后找
}
cout << "apple出现了" << cnt << "次" << endl;  // 3次
```

来推演一遍这个过程。

- 初始pos=0。`s.find("apple", 0)`在位置0找到，cnt=1，pos变成0+5=5。
- `s.find("apple", 5)`从位置5开始找，跳过第一个apple，在位置13找到，cnt=2，pos变成13+5=18。
- `s.find("apple", 18)`从位置18开始找，在位置25找到，cnt=3，pos变成25+5=30。
- `s.find("apple", 30)`找不到，返回string::npos，循环结束。

三次找到，结果正确。

这里有个细节——每次找到后，pos要加上target的长度才能跳过当前找到的词。如果不加，就会在原地无限循环找到同一个词。

好，find的用法就这些。现在总结一下：`s.find(要找的词, 起始位置)`返回位置下标，找不到返回`string::npos`。用while循环配合find可以统计出现次数。这几个知识点今天就要练熟。

## 🖥️ 板书/屏幕要点
- **find语法**：`s.find(子串, 起始位置=0)`
- **返回值**：找到→位置下标（0-based）；找不到→`string::npos`
- ✅ 正确判断：`if (s.find("x") == string::npos)`
- ❌ 不规范：`if (s.find("x") == -1)`
- **统计出现次数模式**：
  ```cpp
  while ((pos = s.find(t, pos)) != string::npos) {
      cnt++; pos += t.size();
  }
  ```

## 📌 小结
find是string最常用的搜索函数——在大段文字里找小段。它返回位置下标，找不到返回string::npos。配上while循环，可以轻松统计某个子串的出现次数。这是后面替换操作的基础——先找到位置，再替换。

## 📝 讲稿正文

同学们，今天这堂课我们要结合string和数组，来解决两个非常实用的综合题——**字符串统计**和**成绩排名**。

先看第一个——**统计英文段落中的字母数、数字数和单词数**。

如果给你一段英文："Hello World 123! C++ is fun."，你要统计多少个字母、多少个数字、多少个单词。字母和数字好办——遍历字符串，用isalpha和isdigit分别统计就行。

但单词数怎么统计？你不能数空格——因为可能有多个连续空格，可能有标点符号，可能开头就是空格。

正确的方法是用**inWord标记法**——用一个bool变量标记"当前是否在某个单词内部"。

```cpp
string line = "Hello World 123! C++ is fun.";
int letters = 0, digits = 0, words = 0;
bool inWord = false;

for (char ch : line) {
    if (isalpha(ch)) letters++;
    if (isdigit(ch)) digits++;

    if (isalpha(ch) && !inWord) {  // 进入一个新单词
        words++;
        inWord = true;
    }
    if (!isalpha(ch) && !isdigit(ch)) {  // 遇到分隔符
        inWord = false;
    }
}

cout << "字母：" << letters << endl;   // 17
cout << "数字：" << digits << endl;    // 3
cout << "单词：" << words << endl;     // 5
```

inWord标记法的精妙之处在于——只在从"不在单词内"切换到"在单词内"的那个瞬间才+1。所以无论一个单词有多长（"World"有5个字母），都只计一次。

这就是**状态标记**的思想——不是直接数，而是追踪状态变化，在变化边界上做统计。这个思想在很多场景下都有用。

好，字符串统计搞定了。现在来看第二个综合题——**成绩排名**。

题目：输入n个学生的姓名和三科成绩，按总分降序输出排名表。

这道题用平行数组：

```cpp
string name[100];      // 存姓名
int score[100][3];     // 存三科成绩（二维数组）
int total[100];        // 存总分

// 读入并计算总分
for (int i = 0; i < n; i++) {
    cin >> name[i];
    total[i] = 0;
    for (int j = 0; j < 3; j++) {
        cin >> score[i][j];
        total[i] += score[i][j];
    }
}

// 冒泡排序（按总分降序）
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - 1 - i; j++) {
        if (total[j] < total[j + 1]) {
            // 捆绑交换！一个都不能少！
            swap(total[j], total[j + 1]);
            swap(name[j], name[j + 1]);
            for (int k = 0; k < 3; k++)
                swap(score[j][k], score[j + 1][k]);
        }
    }
}

// 输出排名表
for (int i = 0; i < n; i++) {
    cout << i + 1 << "\t" << name[i];
    for (int j = 0; j < 3; j++)
        cout << "\t" << score[i][j];
    cout << "\t" << total[i] << endl;
}
```

来仔细看捆绑交换的部分。总分交换了，但**姓名和各科成绩也必须同步交换**。如果只swap总分，张三的总分就挂在了李四名下——数据全乱了。

这个错误非常隐蔽，因为程序不会崩溃，你只会得到"看起来差不多但仔细检查不对"的结果——这种Bug最难调试。所以从一开始就养成捆绑交换的习惯。

如果有多个排序条件——比如总分相同按语文成绩排，语文也相同按姓名字典序排——怎么办？只需要在if条件里加上额外的判断：

```cpp
if (total[j] < total[j+1] ||
    (total[j] == total[j+1] && score[j][0] < score[j+1][0]) ||
    (total[j] == total[j+1] && score[j][0] == score[j+1][0] && name[j] > name[j+1])) {
    // 捆绑交换
}
```

总分高的排前面；总分相同比语文（第一科）；语文也相同比姓名（字典序小的排前面）。

## 🖥️ 板书/屏幕要点
- **inWord标记法**：
  - `isalpha(ch) && !inWord` → words++（新单词开始）
  - `!isalpha(ch) && !isdigit(ch)` → inWord=false（离开单词）
- **成绩排名核心**：
  - 平行数组：name[] + score[][] + total[]
  - 捆绑交换：排序时所有数组同步swap
  - 多条件排序：if中用||连接额外条件

## ✏️ 随堂练习
输入n个学生的姓名和两科成绩（数学、语文），计算总分并按以下规则排名：①总分高的在前；②总分相同，数学成绩高的在前；③数学也相同，按姓名字典序升序。用冒泡排序和捆绑交换实现，输出排名表。

## 📌 小结
字符串统计用inWord标记法统计单词数——追踪状态变化比数空格准确得多。成绩排名用平行数组+捆绑交换——排序时每个属性都要同步更新。这两道题分别代表了string+字符函数、数组+排序的综合应用。inWord的"状态标记"思想和捆绑交换的"同步更新"原则，是综合题里最重要的编程思维。

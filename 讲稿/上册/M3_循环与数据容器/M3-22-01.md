# 第22章 第1课时：容器选择的艺术——数据形状决定容器
**时长**：15-20分钟 | **对应教材**：第22章 数据容器综合实战

## 🎯 本课时目标
- 建立"数据形状决定容器选择"的核心直觉
- 能根据问题描述快速选择合适的容器组合
- 理解平行数组和"捆绑交换"的必要性

## 📝 讲稿正文

同学们，M3模块走到了最后一站。回顾一下我们的旅程——第16章学了一维数组（一排柜子），第17章学了数组的统计变换（求和、最值、桶计数、插入删除），第18章学了二维数组（Excel表格），第19~20章学了string（文字的读写编辑），第21章学了ASCII（字符底层的数字密码）。

每一件武器都是单独练的。但真实世界里的问题是——**别人不会告诉你要用数组还是string**。一道题目摆在你面前："输入n个学生的姓名和语数英三科成绩，按总分从高到低排名。"这道题该用什么容器？

今天这节课，我们就来解决这个"选武器"的问题。

核心原则一句话：**数据的形状决定容器的形状。**

来看这个对照表：

| 数据长什么样 | 用什么容器 | 例子 |
|-------------|-----------|------|
| 一列数字 | `int a[N]` | 全班50个成绩 |
| 一行文字 | `string s` | 姓名、一句话 |
| 多行文字 | `string names[N]` | 全班姓名、单词列表 |
| 行×列的表格 | `int a[N][M]` | 成绩单（学生×科目）、棋盘 |
| 多个属性配对一个对象 | 平行数组 | 姓名+成绩+排名 |

关于最后一种"平行数组"，我想多说几句。当一个对象有多个属性——比如一个学生有姓名（string）、数学成绩（int）、语文成绩（int）、总分（int）——你需要用**多个数组**来存，每个数组存一个属性。

```
string name[100];      // 存姓名
int math[100];         // 存数学成绩
int chinese[100];      // 存语文成绩
int total[100];        // 存总分
```

这些数组是"平行"的——同一个下标i对应同一个人。name[i]是第i个人的姓名，math[i]是第i个人的数学成绩。

平行数组有一个关键规则：**排序时必须"捆绑交换"**。如果你按总分排序时交换了total[2]和total[3]，那你也必须同时交换name[2]和name[3]、math[2]和math[3]……否则张三的总分就挂在李四的名字下了。

这个原则用四个字记：**一条绳上的蚂蚱——动一个，全都要跟着动。**

好，现在我们用"容器选择直觉"来分析几道典型问题：

**问题一**："输入n个整数，输出去重后的升序排列。" → 数据是一列整数 → `int a[N]`。

**问题二**："输入一段英文，统计单词数。" → 数据是一段文字 → `string s`，配合字符判断函数。

**问题三**："输入一个N×N矩阵，顺时针旋转90度输出。" → 数据是行×列表格 → `int a[N][N]`，二维数组。

**问题四**："输入n个学生姓名和三科成绩，按总分排名。" → 姓名（string数组）+ 成绩（可二维可平行）+ 总分 → 平行数组，需捆绑交换。

这个对照表不是死记硬背的——是你看到题目中的"一排"、"一段"、"表格"、"排名"这些词，脑子里的"数据形状雷达"就应该自动指向对应的容器。

现在来动手——分析一道题：输入n个单词，统计每个单词出现的频率，输出出现最多的单词。该用什么容器？先自己想，再看答案。

## 🖥️ 板书/屏幕要点
- **容器选择总则**：数据的形状 → 容器的形状
- 一列数 → `int a[N]`
- 一段文字 → `string s`
- 多段文字 → `string arr[N]`
- 表格 → `int a[N][M]`
- 多属性对象 → 平行数组（需捆绑交换！）
- **捆绑交换口诀**：一条绳上的蚂蚱，动一个全都要动

## ✏️ 随堂练习

## 📌 小结
面对一道题，先别急着写代码——先问自己"数据长什么样"。一列数用数组，一段文字用string，表格用二维数组，多属性用平行数组。容器选对了，代码就成功了一半。平行数组排序时必须捆绑交换——这是综合题里最容易出错的点。

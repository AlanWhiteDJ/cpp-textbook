## 📝 讲稿正文

同学们，今天是第21章最后一节课。我们要学两个内容：**二进制入门**和**凯撒密码实战**。

先说二进制。

我们平时用的是十进制——逢十进一，有0到9十个数字。计算机用的是二进制——逢二进一，只有0和1两个数字。

为什么计算机用二进制？因为它的基本元件晶体管只有"开"和"关"两种状态，用0和1表示最自然。

我们先来感受一下二进制怎么数数：

| 十进制 | 二进制 | 解释 |
|--------|--------|------|
| 0 | 0 | — |
| 1 | 1 | — |
| 2 | 10 | 1+1=2，逢二进一 |
| 3 | 11 | 10+1 |
| 4 | 100 | 11+1=100 |
| 5 | 101 | — |
| 6 | 110 | — |
| 7 | 111 | — |
| 8 | 1000 | — |

跟十进制类比一下：十进制到9之后进位变成10，二进制到1之后进位变成10。道理一样，只是"满几进一"的数字不同。

好，怎么把十进制转成二进制？方法叫**除2取余，倒序排列**。

举个例子——把13转成二进制：

```
13 ÷ 2 = 6  … 余 1  ↑ （从下往上读）
 6 ÷ 2 = 3  … 余 0  ↑
 3 ÷ 2 = 1  … 余 1  ↑
 1 ÷ 2 = 0  … 余 1  ↑
```

所以：**13₁₀ = 1101₂**。从下往上读余数：1、1、0、1。

注意——一定要**倒序**（从下往上）读！从上往下读会得到1011，那是错误的。

那反过来，二进制怎么转回十进制？**按位展开求和**：

```
1101₂ = 1×2³ + 1×2² + 0×2¹ + 1×2⁰
      = 8 + 4 + 0 + 1
      = 13₁₀
```

从最右边开始（第0位），每位乘以2的"位数次方"，然后求和。2⁰=1, 2¹=2, 2²=4, 2³=8……2¹⁰=1024。

现在回头看ASCII：'A'=65=01000001₂。你每次敲键盘上的A，计算机里存的就是这8位0和1。

好，二进制入门就讲这些。接下来我们做一个有趣的实战——**凯撒密码**。

凯撒密码是历史上最古老的加密方法之一。传说古罗马的凯撒大帝用这种方法传递军事情报。怎么做呢？把每个字母按字母表往后移动固定位数。

比如偏移量k=3：'A'→'D'，'B'→'E'，'X'→'A'（循环回头），'Y'→'B'，'Z'→'C'。

加密"HELLO"（k=3）→ "KHOOR"。解密就是反过来往前移。

用ASCII知识来实现：

```cpp
string s = "HELLO";
int k = 3;
for (char &c : s) {
    c = 'A' + (c - 'A' + k) % 26;
}
cout << s << endl;  // "KHOOR"
```

来解析这个公式：`c - 'A'`先把字母转成0~25的编号（A=0, B=1, ...）。加上偏移量k，再对26取模——这样就实现了循环（X+3=23+3=26, 26%26=0→A）。最后加回'A'，转回字母。

这个公式是凯撒密码的核心。我们来手动验证一下：

- 'H'：H是第7个字母 → (7+3)%26=10 → 第10个字母是'K' ✓
- 'E'：(4+3)%26=7 → 'H' ✓
- 'L'：(11+3)%26=14 → 'O' ✓
- 'X'：(23+3)%26=0 → 'A' ✓（循环了！）

凯撒密码虽然简单，但是是很多加密算法的基础思想。理解了它，你就明白为什么字符能当数字算、为什么ASCII知识如此重要。

现在来回顾第21章你学会了什么：bit和Byte的关系、ASCII三大规律、char本质是整数、字符处理函数、二进制转换。这些知识让编程不再是"黑箱操作"——你知道`toupper('a')`底层在干嘛，知道为什么'5'-'0'=5，知道加密是怎么实现的。

## 🖥️ 板书/屏幕要点
- **二进制**：逢二进一，只有0和1
- **十进制→二进制**：除2取余，**倒序排列**
- **二进制→十进制**：按位展开求和（×2ⁿ）
- **凯撒密码**：`'A' + (c - 'A' + k) % 26`
  - `c - 'A'` → 转成0~25
  - `+ k` → 偏移
  - `% 26` → 循环
  - `+ 'A'` → 转回字母

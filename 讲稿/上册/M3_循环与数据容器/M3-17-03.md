# 第17章 第3课时：数组插入——在排队中加塞
**时长**：15-20分钟 | **对应教材**：第17章 数组的魔法

## 🎯 本课时目标
- 掌握在数组中插入一个元素的正确步骤
- 理解"从后往前移"的移动方向和原因
- 能写出有序数组的插入代码

## 📝 讲稿正文

同学们，前面我们学了怎么统计数组里的数据。今天我们来学怎么"改"数组——先从插入开始。

假设你有一个数组`{10, 20, 30, 40, 50}`，现在想在20和30之间插入一个25。怎么插？

你可能会想——数组是一排连续的柜子，中间没有缝隙啊。没错！数组不像链表那样能在中间"挤"进去一个位置。你要插入，就得先**把后面的元素整体往后挪一格，腾出空位**，然后把新元素放进去。

我们来看具体操作。想在位置2（也就是30那个位置）插入25：

```cpp
int a[10] = {10, 20, 30, 40, 50};    // 容量10，目前用了5个
int n = 5;                            // 当前元素个数

int pos = 2;  // 插入位置的下标
int val = 25; // 要插入的值

// 第一步：从后往前，把pos及之后的元素整体后移
for (int i = n; i > pos; i--) {
    a[i] = a[i - 1];
}

// 第二步：放入新元素
a[pos] = val;
n++;  // 元素个数+1

// 输出：10 20 25 30 40 50
```

好，我们来一步一步推演这个过程。初始状态是：

```
下标: 0   1   2   3   4   5   6   ...
值:  10  20  30  40  50   ?   ?   ...
```

现在我们要在下标2处插入25。第一步，把下标2及之后的元素往后挪。注意必须**从后往前挪**！

i从5开始递减：
- i=5：a[5] = a[4]，把50挪到位置5
- i=4：a[4] = a[3]，把40挪到位置4
- i=3：a[3] = a[2]，把30挪到位置3
- i=2：不满足i > pos，停止

现在数组变成了：`{10, 20, 30, 30, 40, 50, ...}`——位置2和位置3都是30。位置2被"空"出来了（其实是被旧值占着，但我们可以覆盖它）。

第二步：a[2] = 25，把25放进去。数组变成`{10, 20, 25, 30, 40, 50}`。完成！

这里有一个非常关键的细节——**为什么必须从后往前移？**

我们来试试从前往后移会怎样：

```cpp
// 错误写法！从前往后移：
for (int i = pos; i < n; i++) {
    a[i + 1] = a[i];
}
```

推演一下：
- i=2：a[3] = a[2]，30复制到位置3
- i=3：a[4] = a[3]，但此时a[3]已经是30了！40被覆盖了！
- i=4：a[5] = a[4]，但a[4]也是30了！50被覆盖了！

结果整个数组后面全是30——数据全丢了。

从前往后移就像排队插队时，你让前面的人先挪——他踩到了后面人的脚，后面的人又把再后面的人踩了。正确的做法是：**最后一个人先退一步，倒数第二个人再退一步……** 这就是从后往前移的道理。

好，掌握了插入之后，我们来做一个小综合——**有序插入**。假设有一个已经从小到大排好序的数组，现在要插入一个新数字，保持数组仍然有序。

思路分两步：第一步，找到新数字应该插入的位置；第二步，把那个位置及之后的元素后移，放入新数字。

这个留作今天的练习，大家试试看。

## 🖥️ 板书/屏幕要点
- **插入两步走**：①后移腾位 → ②放入新元素
- **移动方向**：从后往前（i从n递减到pos+1）
- 关键循环：`for (int i = n; i > pos; i--) a[i] = a[i-1];`
- ❌ 从前往后 → 数据被覆盖丢失
- ✅ 从后往前 → 安全移动
- 记忆技巧：排队插队，后面的人先退

## ✏️ 随堂练习
有一个已从小到大排好序的数组`{5, 12, 18, 25, 30, 40}`，当前有6个元素，容量为20。用户输入一个新数字x，把x插入到合适的位置，保持数组仍然有序。输出插入后的数组。测试用例：输入x=15，输出`5 12 15 18 25 30 40`。

## 📌 小结
数组插入的核心是"从后往前移"——先把最后一个元素往后挪，再挪倒数第二个，依次往前，腾出空位后放入新元素。从前往后移会导致数据覆盖丢失，这是新手最容易犯的错误。记住：插入完成要更新元素个数n。这个"移动-放入-更新n"的三步模式，也是删除操作的基础。

# 第20章 第4课时：replace——一步到位的替换术
**时长**：15-20分钟 | **对应教材**：第20章 字符串的十八般武艺

## 🎯 本课时目标
- 掌握replace的语法：指定位置、删除长度、新字符串
- 理解replace = erase + insert 的合体
- 能用find+replace实现批量替换

## 📝 讲稿正文

同学们，上节课我们学了erase和insert——一个删、一个插，组合起来就能做替换：先删掉旧内容，再插入新内容。

但每次都写两行太啰嗦了。C++提供了一个一步到位的函数——**replace**。

格式：`s.replace(起始位置, 要删的长度, 新字符串)`。它先删除指定的一段，再在同一个位置插入新字符串。

```cpp
string s = "I like cats";

s.replace(2, 4, "love");           // 从下标2开始，删4个字符，放入"love"
cout << s << endl;                 // "I love cats"

s.replace(7, 4, "dogs");           // 把"cats"换成"dogs"
cout << s << endl;                 // "I love dogs"
```

来推演第一步。初始`s = "I like cats"`。`s.replace(2, 4, "love")`——从下标2开始（"l"的位置），删除4个字符"like"，然后在同一位置放入"love"。新字符串"I love cats"——长度刚好不变，但内容变了。

第二步，`s.replace(7, 4, "dogs")`——从下标7（"c"的位置），删除4个字符"cats"，放入"dogs"。变成"I love dogs"。

好，replace的基本用法很简单。但更有意思的是——**用find+replace实现批量替换**。

场景：一篇文章里，把所有的"傻瓜"替换成"**"。这个需求在敏感词过滤、文本处理里很常见。

```cpp
string s = "你这个傻瓜，真是个大傻瓜！";
string from = "傻瓜";
string to = "**";

int pos = 0;
while ((pos = s.find(from, pos)) != string::npos) {
    s.replace(pos, from.size(), to);  // 用to替换from
    pos += to.size();                  // 跳过已替换的部分
}
cout << s << endl;  // "你这个**，真是个大**！"
```

来分析循环的每一步。

- pos=0，找到"傻瓜"在位置4。`s.replace(4, 4, "**")`——原来4个中文字符删掉，换成2个星号。字符串变短了。pos变成4+2=6。
- pos=6，继续找下一个"傻瓜"，在位置9找到（因为之前删掉了4个字符换了2个，位置偏移了）。替换，pos变成9+2=11。
- pos=11，再也找不到"傻瓜"，循环结束。

注意——每次替换后，pos要加上**新字符串**的长度（to.size()），而不是旧字符串的长度。因为替换后字符串内容变了，如果不跳过已替换的部分，可能会陷入死循环——比如把"aa"替换成"a"，如果pos不加，就会在同一位置反复替换。

另外，如果要替换的词比被替换的词长，那也很自然——replace会自动调整后面的内容。

现在我们来做一个实用练习——隐藏手机号的中间4位。比如"13812345678"变成"138****5678"。

用replace一步搞定：`s.replace(3, 4, "****")`——从下标3开始，删掉4位，放入4个星号。

## 🖥️ 板书/屏幕要点
- **replace语法**：`s.replace(起始位置, 删除长度, 新字符串)`
- replace = erase + insert 的合体
- **批量替换模式**：
  ```cpp
  while ((pos = s.find(from, pos)) != npos) {
      s.replace(pos, from.size(), to);
      pos += to.size();  // 注意：加新串长度！
  }
  ```
- 应用：敏感词过滤、手机号隐藏、文本批量修改

## ✏️ 随堂练习
用户输入一个11位手机号字符串，程序将中间4位替换为`****`后输出。例如输入`13812345678`，输出`138****5678`。要求：先用substr取头3位和尾4位拼接的方法做一遍，再用replace直接做一遍，体会两种方法的差异。

## 📌 小结
replace是string编辑功能的大招——指定位置、删除长度、新内容，一步完成"先删后插"。配合find循环使用，可以实现批量替换——敏感词过滤、格式整理、数据脱敏等场景全靠这个组合。记住每次替换后要跳过新插入的内容，否则可能死循环。

# 第17章 第4课时：删除与反转——数组元素的"乾坤大挪移"
**时长**：15-20分钟 | **对应教材**：第17章 数组的魔法

## 🎯 本课时目标
- 掌握在数组中删除元素的正确步骤
- 理解删除时"从前往后移"的方向及其与插入的对比
- 掌握数组反转的标准算法

## 📝 讲稿正文

同学们，上节课我们学了数组插入——把后面的元素往后挪，腾出空位。今天我们来学它的"逆向操作"——**删除**。

删除的思路刚好反过来：把要删的元素后面的所有元素整体**往前挪一格**，覆盖掉要删的那个。

来看代码。假设数组是`{10, 20, 30, 40, 50}`，要删除位置2的元素（也就是30）：

```cpp
int a[10] = {10, 20, 30, 40, 50};
int n = 5;

int pos = 2;  // 要删除的位置

// 从pos开始，每个元素被它后面的元素覆盖
for (int i = pos; i < n - 1; i++) {
    a[i] = a[i + 1];
}
n--;  // 元素个数-1

// 输出：10 20 40 50
```

推演一下过程。要删掉下标2的30：

- i=2：a[2] = a[3]，40覆盖30 → `{10, 20, 40, 40, 50}`
- i=3：a[3] = a[4]，50覆盖40 → `{10, 20, 40, 50, 50}`
- i=4：不满足i < n-1（4 < 4为假），停止

最后n--，我们就只关注前4个元素了。最后一个位置的50还在那里，但已经不算有效数据了。

大家注意——删除是从**前往后**移。这跟插入的方向刚好相反。为什么删除不用从后往前？因为删除是把后面的元素往前挪，如果从后往前挪，就会变成后面的值把前面的覆盖了，导致所有值都变成最后一个。你可以自己在纸上推演一下试试。

插入和删除的方向口诀：**插入从后往前，删除从前往后**。一个"后退"，一个"前进"。

好，接下来我们学一个跟删除插入不太一样，但非常经典的操作——**数组反转**。

反转就是把`{1, 2, 3, 4, 5}`变成`{5, 4, 3, 2, 1}`。思路是"首尾交换"。

```cpp
int a[5] = {1, 2, 3, 4, 5};
int n = 5;

for (int i = 0; i < n / 2; i++) {
    // 交换 a[i] 和 a[n - 1 - i]
    int temp = a[i];
    a[i] = a[n - 1 - i];
    a[n - 1 - i] = temp;
}

// 输出：5 4 3 2 1
```

我们来解读这个循环。n=5，n/2=2，所以循环只进行2次：

- i=0：交换a[0]和a[4]，1和5交换 → `{5, 2, 3, 4, 1}`
- i=1：交换a[1]和a[3]，2和4交换 → `{5, 4, 3, 2, 1}`
- 结束

中间的元素a[2]=3不需要动——它正好在中间，自己跟自己交换没意义。

这里有一个很容易犯的错误——**循环条件写成i < n**。如果你写成`for (int i = 0; i < n; i++)`，会发生什么？

- i=0：交换a[0]和a[4] ✓
- i=1：交换a[1]和a[3] ✓
- i=2：交换a[2]和a[2] ✓（自己换自己，白干但没坏处）
- i=3：交换a[3]和a[1]——把刚才换好的又换回去了！
- i=4：交换a[4]和a[0]——同样，前功尽弃！

所以循环条件必须是`i < n / 2`，只换一半。超过了中间点，就会把换好的又换回去。

今天学了两个重要操作——删除和反转。它们各自有一个坑：删除要注意移动方向（从前往后），反转要注意循环次数（只换一半）。现在我们来做练习。

## 💬 互动提问
1. 删除元素时为什么是从前往后移？和插入的方向为什么相反？
2. 数组反转时，循环条件为什么是`i < n/2`而不是`i < n`？
3. 删除操作完成后，为什么要执行n--？如果忘了n--会怎样？

## 🖥️ 板书/屏幕要点
- **删除**：`for (i=pos; i<n-1; i++) a[i]=a[i+1];` 然后`n--;`
- **插入方向**：从后往前 ← 删除方向：从前往后
- 口诀：**插入后退，删除前进**
- **反转**：`for (i=0; i<n/2; i++) swap(a[i], a[n-1-i]);`
- 反转循环次数 = n/2（不是n！）
- 交换三步法：temp = a; a = b; b = temp;

## ✏️ 随堂练习
声明数组`{15, 28, 34, 42, 56, 63, 71}`（7个元素）。完成两项操作：①删除位置3的元素（42），输出删除后的数组；②将删除后的数组反转，输出最终结果。注意每一步都要正确更新n的值。

## 📌 小结
插入和删除是一对"镜像操作"——插入从后往前移，删除从前往后移。它们的共同点是都要更新元素个数n。数组反转用的是"首尾交换"策略，循环n/2次，超过一半会前功尽弃。这三项操作——插入、删除、反转——构成了数组"修改操作"的基础工具箱。

## 📝 讲稿正文

同学们，上节课我们学会了"看数据形状选择容器"。今天开始我们进入实战——第一个综合题：**矩阵旋转**。

题目是这样的：给定一个n×n的矩阵，将其顺时针旋转90度后输出。

比如：

```
输入：        输出：
1  2  3      7  4  1
4  5  6      8  5  2
7  8  9      9  6  3
```

大家先自己观察一下——原来的1去哪了？去了右上角。原来的2去了哪？去了正上方中间。原来的7去了左上角……

我们来总结规律。原矩阵的第i行第j列，旋转后去了哪里？

拿几个元素验证：原(0,0)=1 → 新(0,2)。原(0,1)=2 → 新(1,2)。原(0,2)=3 → 新(2,2)。原(1,0)=4 → 新(0,1)……

规律是：**a[i][j] → b[j][n-1-i]**。原来的(i,j)变成了新矩阵的(j, n-1-i)位置。

来写代码：

```cpp
int n;
cin >> n;
int a[100][100], b[100][100];

// 读入原矩阵
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        cin >> a[i][j];

// 顺时针旋转90度
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        b[j][n - 1 - i] = a[i][j];  // 核心公式

// 输出旋转后的矩阵
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++)
        cout << b[i][j] << " ";
    cout << endl;
}
```

我们来推演n=3的情况。双重循环遍历12个元素：

- i=0, j=0: b[0][2] = a[0][0] = 1 → 原来左上角的1去了第一行最后
- i=0, j=1: b[1][2] = a[0][1] = 2 → 2去了第二行最后
- i=0, j=2: b[2][2] = a[0][2] = 3 → 3去了第三行最后
- i=1, j=0: b[0][1] = a[1][0] = 4 → 4去了第一行中间
- ……以此类推

全部填完，新矩阵就是正确的结果。

好，有一个容易犯的错误——**顺时针和逆时针搞反**。

顺时针：`b[j][n-1-i] = a[i][j]`
逆时针：`b[n-1-j][i] = a[i][j]`

怎么记？一个技巧——拿2×2的小矩阵在纸上画一遍。"顺时针"就是"向右转"，观察原(0,0)去了新矩阵的哪个位置，然后总结规律。平时多画几次就熟了。

另外，旋转180度 = 转两次90度。旋转270度 = 转三次90度（等价于逆时针90度）。你不需要记三个公式，掌握一个方向的变换规律就行。

还要提一个原地旋转的技巧——对于方阵，可以不新开数组b，直接在a上操作。但原地旋转涉及四角交换，逻辑比较绕。现阶段建议用辅助数组b，思路清晰不易错。等熟练了再挑战原地旋转。

现在来做练习——把顺时针旋转的代码自己敲一遍，测试n=3和n=4的情况。

## 🖥️ 板书/屏幕要点
- **顺时针90°**：`b[j][n-1-i] = a[i][j]`
- **逆时针90°**：`b[n-1-j][i] = a[i][j]`
- 180° = 两次90°
- 270° = 三次90° = 逆时针90°
- 建议用辅助数组b（清晰不易错）
- 验证方法：小矩阵（2×2）纸上画一遍

## ✏️ 随堂练习
输入一个n×n矩阵（n≤100），完成：①顺时针旋转90度并输出；②再对上一步的结果顺时针旋转90度（即旋转180度），验证是否等价于把原矩阵上下左右颠倒；③输出旋转180度后矩阵的主对角线元素。

## 📌 小结
矩阵旋转是二维数组的经典应用。核心是坐标变换公式——顺时针`b[j][n-1-i] = a[i][j]`。顺时针和逆时针公式不同，不要搞反。旋转180度和270度都可以通过多次90度实现。建议学习阶段用辅助数组，可读性优先。

# 第16章 第1课时：认识数组——编程世界的储物柜
**时长**：15-20分钟 | **对应教材**：第16章 一维数组

## 🎯 本课时目标
- 理解数组"同类型数据连续存储"的本质
- 掌握数组的声明语法和基本初始化方法
- 建立数组就像一个"编号从0开始的储物柜"的直觉

## 📝 讲稿正文

同学们，大家想一想——如果现在让你写一个程序，存全班50个同学的成绩，你会怎么做？

有人可能会说："声明50个变量呗，score1、score2、score3……"好，那你写给我看看。写到score10的时候手还行，写到score30的时候眼睛开始花，写到score50的时候——你可能已经不认识键盘了。

更崩溃的是什么呢？你要算平均分。你得写：`(score1 + score2 + score3 + ... + score50) / 50.0`。这一行代码比你的胳膊还长，而且万一隔壁班有55个人，所有代码都得重写。

这就是我们今天要解决的问题——有没有一种办法，把一列相同类型的数据"捆"在一起，用一个名字管，靠编号区分？

答案是——**数组**。

我们来看一个生活中的场景。你家小区楼下是不是有一排快递柜？一共20个，编号从0到19。你去取快递，短信说"您的快递在7号柜"。你走到第7个柜子前，打开，拿到包裹。你不会去开8号柜——那不是你的。

这个场景里，一排储物柜就是**数组**，柜子编号就是**下标**，柜子里的东西就是**数组元素**。数组等于一排编号从0开始的储物柜，每个柜子装同一种类型的东西。

好，那在C++里怎么声明一个数组呢？很简单，你要告诉计算机三件事：装什么类型、叫什么名字、有多少个柜子。

来看代码：

```cpp
int scores[50];         // 50个整数柜子，装成绩
double temps[7];        // 7个小数柜子，装一周温度
char grades[30];        // 30个字符柜子，装等级
```

方括号里的数字是柜子的个数。注意了——声明的时候，方括号里的数字必须是**常数**，编译时就能确定的值。不能写一个变量进去。

有同学可能要问：那我输入一个n，然后声明`int arr[n]`行不行？这是不符合C++标准的写法。有些编译器可能会让你通过，但这就像闯红灯——没被抓住不代表安全。

正确的做法是这样的：题目跟你说n不超过1000，你就声明`int arr[1005]`，多留5个缓冲位置。然后输入n，用前n个位置就够了。

好，声明了数组之后，怎么往里面放东西呢？我们可以在声明的同时"装货"，这叫**初始化**：

```cpp
int arr[5] = {10, 20, 30, 40, 50};  // 5个全装好
int evens[6] = {2, 4, 6};           // 前3个装好，后3个自动填0
int zeros[100] = {0};               // 全部初始化为0——这是竞赛里最常用的写法！
```

特别要记住`int zeros[100] = {0}`这种写法——100个柜子全部清零，一秒搞定。

那有同学可能要问：`int arr[100] = {1}`能不能把100个全部初始化为1？不行！这个写法只有第一个是1，后面99个全是0。这是一个很容易踩的坑，大家记牢了。

还有一件事要提醒大家——数组声明了但没初始化，里面的值是什么？是**垃圾值**。可能是0，可能是32768，可能是任何奇怪的数字。所以要么声明时初始化，要么先赋值再使用。不要以为空的柜子就是0，它可能是上一个房客留下的东西。

好，我们来动手试一试。现在请大家打开编译器，声明一个长度为10的int数组，全部初始化为0。然后自己试着输出第三个元素的值，看看是不是0。

## 🖥️ 板书/屏幕要点
- **数组 = 一排编号从0开始的同类型储物柜**
- 声明三要素：类型 + 名字 + [个数]
- `int arr[100];` ✓ —— `int arr[n];` ✗
- `int arr[100] = {0};` —— 全部清零（高频用法）
- `int arr[100] = {1};` —— 只有arr[0]=1，其余为0（坑！）
- 未初始化 = 垃圾值

## ✏️ 随堂练习
声明一个长度为10的int数组，全部初始化为0。然后用5行赋值语句，把前5个元素分别设成1、2、3、4、5，输出这5个元素。观察后5个元素的值是什么。

## 📌 小结
今天我们认识了数组——它是一排同类型的储物柜。声明数组要写类型、名字和大小。初始化可以用花括号，`{0}`能全部清零。记住声明时大小必须是常数，不要用变量。数组就像是给你一大排整齐的柜子，一个名字管到底，靠编号区分——这就是接下来所有高级数据结构的基础。

## 📝 讲稿正文

同学们，上节课我们学习了ASCII表。今天我们来做一件"颠覆认知"的事——把char当整数用。

很多人学C++的时候，会以为char和int是两种完全不搭边的类型。但实际上——**char本质上就是一个整数**，一个0到127之间的整数。它只是"穿了一件字符的外衣"。

我们来看证据：

```cpp
char ch = 65;            // 65是'A'的ASCII码
cout << ch << endl;      // 输出：A

char letter = 'B';
cout << (int)letter << endl;  // 输出：66 —— 把char转成int看
```

看到没？`char ch = 65`和`char ch = 'A'`是完全等价的！编译器看到65，就知道你指的是字母A。

正因为char本质是整数，所以它可以做算术运算：

```cpp
// 魔术①：A + 1 = B
char c = 'A';
c = c + 1;               // 65 + 1 = 66 → 'B'
cout << c << endl;       // 输出：B

// 魔术②：小写转大写（手动版）
char lower = 'g';
char upper = lower - 32; // 103 - 32 = 71 → 'G'
cout << upper << endl;   // 输出：G

// 魔术③：大写转小写（手动版）
char upper2 = 'M';
char lower2 = upper2 + 32; // 77 + 32 = 109 → 'm'
cout << lower2 << endl;     // 输出：m

// 魔术④：字母表偏移
char c2 = 'A' + 3;       // 65 + 3 = 68 → 'D'
cout << c2 << endl;      // 输出：D
```

好，现在来讲一个**超级重要的坑**——把字符'5'当数字5。

```cpp
char ch = '5';
int x = ch;          // 你以为x=5？大错特错！
cout << x << endl;   // 输出53！不是5！
```

为什么是53？因为'5'的ASCII码是53。当你把char赋值给int，你得到的是ASCII值，不是字符代表的数字。

那怎么把'5'变成真正的数字5？用减法！

```cpp
char ch = '5';
int x = ch - '0';    // 53 - 48 = 5 ✅
cout << x << endl;   // 输出：5
```

为什么`ch - '0'`能行？因为'0'到'9'连续排列——'0'的ASCII是48，'5'的ASCII是53。53减去48等于5，恰好就是字符'5'代表的数值。

这个技巧在处理输入时超级实用。比如你读入了一个字符串"123"，想把它变成整数123：

```cpp
string s = "123";
int num = 0;
for (int i = 0; i < s.size(); i++) {
    num = num * 10 + (s[i] - '0');
}
cout << num << endl;  // 123
```

每一步：num=0→0*10+1=1→1*10+2=12→12*10+3=123。

这就是`stoi`函数底层做的事情——遍历每个字符，减'0'得到数值，然后拼成整数。现在你知道stoi是怎么工作的了吧？

最后再告诉大家一个小技巧——`(int)ch`叫**强制类型转换**，意思是"把ch当成int来看"。反过来`(char)65`就是把65当成char来看。

## 🖥️ 板书/屏幕要点
- **char本质 = 0~127的整数**
- 整数赋值char：`char c = 65;` → 'A'
- char转整数：`(int)'A'` → 65
- **字符数字转真数字**：`ch - '0'`（如'5'-'0' = 5）
- 大小写手动转换：`ch - 32`（小→大）、`ch + 32`（大→小）
- 字母偏移：`'A' + n`（往后推n个字母）
- 强制类型转换：`(int)ch`、`(char)num`

## 📌 小结
char的本质是整数——它只不过"假装"自己是个字母。理解了这一点，你就明白了为什么`'A'+1='B'`、为什么`toupper`和`tolower`能工作、为什么`ch-'0'`能转数字。char的算术运算能力是把"字符魔术"变成"数学公式"的底层基础。

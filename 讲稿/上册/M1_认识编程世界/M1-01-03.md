# 第1章 第3课时：编程语言——人和计算机之间的"翻译官"

**时长**：15-20分钟 | **对应教材**：第1章 · 1.3-1.4节

## 🎯 本课时目标
- 学生能理解"为什么人不能直接跟计算机说人话"
- 学生能用自己的话解释"编程语言是干什么的"
- 学生知道 C++ 是信息学竞赛的主力语言，并对它的样子有初步印象

## 📝 讲稿正文

上两节课我们建立了一个重要认知：计算机是一个听话的机器，程序就是给它的指令清单。

那现在问题来了——我写"把两个数字加起来"这几个中文字，计算机能看懂吗？

（让学生回答）

**不能！** 这也是很多初学者的第一个困惑。我明明把指令写出来了，计算机为什么不执行？

原因很简单：计算机只认识一种语言，叫**机器语言**。机器语言长什么样呢？就是一堆 0 和 1，比如：

`10110000 01100001`

大家想象一下，让你看着这样一串 0 和 1，你能知道它是什么意思吗？别说你们了，就算是全世界最顶尖的程序员，一天也写不了 10 行这样的代码，还不一定对。

那我们怎么办？难道就不编程了？

这时候，聪明的人类发明了一个中间层——**编程语言**。编程语言是介于"人话"和"机器语言"之间的一种语言。它长得有点像英语，但比英语简单得多——只有几十个关键词，每条语句的意思都确定无疑，不会产生歧义。

我给大家展示一下，在 C++ 里面，如果要在屏幕上输出一句话，代码怎么写——

```cpp
cout << "Hello, World!";
```

就这一行。它的意思是："把 Hello, World! 这几个字送到屏幕上"。你看，是不是比 `10110000` 好理解多了？

世界上有很多种编程语言——C++、Python、Java、JavaScript……它们就像不同的方言。你能用中文点菜，也能用英语点菜，都能达到目的，但说法不一样。

那为什么我们信息学竞赛选 C++ 呢？

三个字：**快、强、稳。**

C++ 运行速度极快，功能极其强大，而且几乎所有信息学竞赛——不管是 NOI 还是国际比赛——都支持 C++。Python 虽然写起来更简单，但跑得太慢了，竞赛里很多题目 Python 根本跑不完。Java 也不错，但在竞赛圈子里，C++ 是当之无愧的王者。

所以，你未来几年要交的朋友，就是 C++。

好，现在我想让大家对 C++ 有一个更立体的认识。你们可能听说过"编译器"这个词。编译器是干什么的呢？

就是那个"翻译官"——把你写的 C++ 代码，一行一行翻译成计算机认识的那些 0 和 1。你写完代码，按一下"编译"按钮，翻译官就开始工作。如果翻译过程中发现你写错了——比如少了个分号——它就会停下来告诉你："喂，这里有问题，改一下。"

翻译完了，计算机就能执行了。这个过程，我们下一章会亲身体验。

好，回到今天的主题。为什么需要编程语言？一句话：**因为人不会说机器语言，机器听不懂人话，所以需要一个中间人来翻译。** 编程语言就是那个中间人。

大家想一想，生活中其实有很多类似的"中间人"——比如你和外国人交流需要翻译，发电报需要把文字转成摩尔斯电码。编程语言干的，本质上就是这件事。

## 🖥️ 板书/屏幕要点
```
计算机只认识：机器语言（0 和 1）
人最好写的：自然语言（中文、英文）

编程语言 = 中间人 = 翻译官

C++ 代码  ──编译──→  机器语言  ──执行──→  结果
（人看得懂）     （翻译官）    （计算机懂）

为什么选 C++？
  快 — 运行速度极快
  强 — 功能强大
  稳 — 全国/全球竞赛标配

Hello World 预览：
  cout << "Hello, World!";
  意思：把引号里的字输出到屏幕上
```

## ✏️ 随堂练习
在纸上写下：如果你要设计一种"跟计算机说话的语言"，你觉得它至少需要什么功能？列出 3 个你希望它能做的事情。（等学完 M1 再回头看看你的设计，会很有意思。）

## 📌 小结（30秒总结）
计算机只懂 0 和 1，人没法直接写 0 和 1。编程语言是人和计算机之间的翻译官——长得很像简化英语，但每条语句含义精确无歧义。C++ 是竞赛首选，因为它跑得快、功能强、哪里都认。你的 C++ 代码 → 编译器翻译 → 机器语言 → 计算机执行。这个流程，就是我们接下来要亲手体验的。

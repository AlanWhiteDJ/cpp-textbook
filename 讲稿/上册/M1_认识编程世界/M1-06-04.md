## 📝 讲稿正文

同学们，前面几节课我们学了不少运算符。今天我们学一组"偷懒神器"——**复合赋值运算符。**

什么叫复合赋值？就是"运算 + 赋值"一步完成。你们回忆一下，之前写"把 score 增加 5"需要这样：

```cpp
score = score + 5;
```

变量名出现了两次。能不能简单一点？可以：

```cpp
score += 5;   // 等价于 score = score + 5
```

这一行叫"复合赋值"——把加法和赋值"复合"在了一个运算符里。`+=` 的意思就是"把右边的值加到左边去"。

全部五种复合赋值运算符：

| 写法 | 等价于 | 意思 |
|------|--------|------|
| `a += 5` | `a = a + 5` | a 增加 5 |
| `a -= 5` | `a = a - 5` | a 减少 5 |
| `a *= 5` | `a = a * 5` | a 扩大 5 倍 |
| `a /= 5` | `a = a / 5` | a 除以 5（整数除法） |
| `a %= 5` | `a = a % 5` | a 对 5 取余 |

我们来看一个完整的例子：

```cpp
int score = 90;
score += 5;    // score = 95  （加了 5 分）
score *= 2;    // score = 190 （翻倍）
score -= 30;   // score = 160 （扣了 30 分）
score /= 10;   // score = 16  （除以 10）
score %= 7;    // score = 2   （除以 7 余 2）
```

每一步都像是给 score 这个变量做了一次"手术"——直接修改它本身的值。

复合赋值不光写起来更短，还有一个隐藏优势：**变量名只出现一次，不容易拼错。** 如果你写 `totalScore = totalScore + bonus;`，`totalScore` 这个长名字要打两遍，打到第二遍的时候可能就拼错了。而 `totalScore += bonus;` 只需要打一遍。

还有一点要注意——复合赋值符号是一个整体，中间不能有空格。`+ =` 是错的，必须写 `+=`。它的读法是"加等于"——"score 加等于 5"，意思就是 score 增加 5。

另外，复合赋值的优先级和普通赋值一样——都是最低的。所以：

```cpp
int a = 10;
a *= 3 + 2;   // a = a * (3+2) = a * 5 = 50
              // 不是 a = a * 3 + 2 = 32！
```

复合赋值的**右边一整块是一个整体**，先把右边的表达式算完，再用复合运算符作用到左边。`a *= 3 + 2` 等价于 `a = a * (3 + 2)`，不是 `a = a * 3 + 2`。

这个很容易搞错，请大家特别留意。如果拿不准右边有没有隐含的括号，就**自己加括号明确意图**——比如写成 `a *= (3 + 2);`，清晰又安全。

好了，今天的内容很简单但很实用。这五个"偷懒符号"你们以后每天都会用到——它们让你的代码变得干净利落。下节课是本阶段最后一课，我们讲运算符优先级和整数运算的安全范围。

## 🖥️ 板书/屏幕要点
```
复合赋值运算符：

  +=  a += 5   →  a = a + 5   "a 加等于 5"
  -=  a -= 5   →  a = a - 5   "a 减等于 5"
  *=  a *= 5   →  a = a * 5   "a 乘等于 5"
  /=  a /= 5   →  a = a / 5   "a 除以等于 5"
  %=  a %= 5   →  a = a % 5   "a 模等于 5"

优势：
  ① 简短 — 变量名只写一次
  ② 清晰 — 一眼看出在"原地修改"
  ③ 少拼写错误

注意：
  符号无空格：a += 5  ✓    a + = 5  ✗
  右边整体算：a *= 3 + 2  →  a = a * (3+2)
  拿不准就加括号：a *= (3 + 2);
```

## 📌 小结（30秒总结）
五种复合赋值 `+= -= *= /= %=`，每一种都是"运算+赋值"的简写——右边整体算完再作用到左边。优势是代码更短、变量名只写一次、不易拼错。注意两点：符号中间无空格，右边表达式隐含括号。下节课收官第六章——运算符优先级和整数运算的安全边界。

# 第4章 第2课时：溢出——小杯子装大水的恐怖故事

**时长**：15-20分钟 | **对应教材**：第4章 · 4.3-4.4节

## 🎯 本课时目标
- 学生能用自己的话解释什么是"溢出"
- 学生能判断一个运算会不会导致溢出
- 学生知道 `double` 是用来存小数的，并会正确使用

## 📝 讲稿正文

上节课我们提到了"溢出"这个词，但没有深入讲。今天我们就来彻底搞清楚：**溢出到底是什么，它有多可怕。**

先做一个类比。你有一个只能显示三位数的计数器——最多显示到 999。现在你把 999 再加 1，按理说应该是 1000。但你的计数器只有三位，怎么办？它只能显示 000。999 + 1 = 0？这显然不对。

在 C++ 里，类似的事情发生在 int 身上。int 的最大值是 2147483647（约 21 亿）。如果你把它加 1——

```cpp
int a = 2147483647;   // int 的最大值
a = a + 1;
cout << a;             // 输出：-2147483648（最小的负数！）
```

2147483647 + 1 = -2147483648？这在数学上是荒谬的，但在计算机里是真的——因为二进制位全部翻转了，符号位从 0 变成 1，正数变成了负数。

这就是**溢出**。当你的数字超过了类型能表示的范围，多出来的部分就"溢"成了错误的值。

更可怕的是——**溢出不会报错！** 程序会开开心心地编译通过、正常运行，然后给你一个完全错误的结果。你根本不知道算错了，拿这个错误结果去做后续计算，错上加错。

再来看一个更隐蔽的例子：

```cpp
int a = 100000;
int b = 100000;
int c = a * b;      // 100 亿！int 装不下！
cout << c;           // 输出：1410065408（一个奇怪的数字）
```

a 和 b 本身都在 int 范围里，所以你不会想到溢出。但 a 乘 b 的结果是 100 亿，远远超过了 21 亿。乘积溢出，c 就成了垃圾值。

怎么办？改用 long long：

```cpp
long long a = 100000;
long long b = 100000;
long long c = a * b;   // 安全：100 亿在 long long 范围内
cout << c;              // 输出：10000000000 ✓
```

好，溢出讲完了。现在我们来介绍第三种类型——`double`，专门用来装小数。

int 只能存整数。那 3.14 怎么办？考试分数 95.5 怎么办？用 `double`。

| 类型 | 占用空间 | 精度 | 适合场景 |
|------|----------|------|----------|
| `double` | 8 字节 | 约 15 位有效数字 | 所有需要小数的计算 |

```cpp
double pi = 3.14159265358979;
double score = 95.5;
double price = 12.8;
double avg = 91.25;
```

在信息学竞赛中，只要涉及小数——面积、平均分、百分比、概率——一律用 `double`。

注意一个细节：写小数的时候直接写 `3.14`，不用加什么后缀。C++ 会自动识别带小数点的数字是 double 类型。

还有一个小数除法的重点——这个我们第 6 章会详细展开，但先剧透一下：

```cpp
int a = 5 / 2;        // a = 2（整数除法，丢小数）
double b = 5.0 / 2;   // b = 2.5（至少一个操作数是小数就是小数除法）
```

看到区别了吗？`5 / 2` 两个都是整数，做整数除法得 2。`5.0 / 2` 至少有一个是小数，做小数除法得 2.5。这个一定要记住！

今天的核心信息量有点大——溢出和 double。大家消化一下，下节课我们继续探索另外三种类型。

## 🖥️ 板书/屏幕要点
```
溢出（Overflow）：
  数字 > 类型范围 → 结果错误，不报错！

  int a = 2147483647;
  a = a + 1;         // → -2147483648（溢出！）

  int c = 100000 * 100000;
  // 100 亿 > 21 亿 → 溢出成垃圾值！

  解决方案：换 long long！

---
double — 浮点数（小数）：
  8 字节，约 15 位有效数字

  double pi = 3.14159;
  double score = 95.5;

  整数除法 vs 小数除法：
    5 / 2   = 2    （整数除法）
    5.0 / 2 = 2.5  （小数除法，有一个是小数就行）
```

## ✏️ 随堂练习
在 IDE 里尝试：声明 `int x = 2000000000;`（20 亿），然后 `x = x + 500000000;`（再加 5 亿）。输出 x 的值——观察溢出后的结果。然后改用 `long long` 重做一遍，对比结果。把两个版本的结果记录在笔记上。

## 📌 小结（30秒总结）
溢出是 C++ 中最隐蔽的 bug——数字超过范围，程序不报错但结果全错。int 上限约 21 亿，超过就换 long long。double 是装小数的类型，用 5.0/2 这种写法做小数除法。记住：不确定会不会溢出，就用 long long；需要小数，就用 double。

# 第5章 第2课时：一次读多个值 + 综合输入练习

**时长**：15-20分钟 | **对应教材**：第5章 · 5.3-5.4节

## 🎯 本课时目标
- 学生能使用 `cin >> a >> b` 一次性读取多个数据
- 学生能写出一个跟用户多轮对话的互动程序
- 学生能理解输入缓冲区的概念

## 📝 讲稿正文

上节课我们学会了一个一个地读数据——问个问题，等用户输入，再问下个问题。今天我们来学更高效的写法——**一次读多个值**。

假设你要让用户输入两个数字，用上节课的方法你会这样写：

```cpp
int a, b;
cout << "请输入第一个数：";
cin >> a;
cout << "请输入第二个数：";
cin >> b;
```

可以是可以，但太啰嗦了。C++ 支持把多个 `cin >>` 连在一起写：

```cpp
int a, b;
cout << "请输入两个整数：";
cin >> a >> b;
cout << "它们的和是 " << a + b << endl;
```

运行效果：

```
请输入两个整数：10 20
它们的和是 30
```

你看，用户在一行里输入"10 20"（中间用空格隔开），cin 自动把 10 读给 a，把 20 读给 b。非常方便。

而且 cin 很聪明——用户不一定要在一行里输入。他可以先敲 10 按回车，再敲 20 按回车，效果一样。cin 会自动跳过空格和换行，只读取真正的数据。

现在我们把前面学的东西组合起来，写一个真正的互动对话程序。这个程序会问用户三个问题，然后输出一张"档案卡"：

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string name;
    int age;
    double score;

    cout << "你叫什么名字？";
    cin >> name;

    cout << "你今年几岁？";
    cin >> age;

    cout << "上次考试考了多少分？";
    cin >> score;

    cout << "———— 档案卡 ————" << endl;
    cout << "姓名：" << name << endl;
    cout << "年龄：" << age << " 岁" << endl;
    cout << "分数：" << score << " 分" << endl;

    return 0;
}
```

运行一遍试试——输入你的名字、年龄和分数，看看程序帮你生成的档案卡。

这个程序虽然只有二十几行，但它已经用到了变量（第三章）、多种类型（第四章）、输入输出（第五章）——把你的前三章知识全部串起来了。这就是"综合运用"的感觉。

---

好，最后我们来聊一个"底层"的东西——**输入缓冲区**。

你敲键盘的时候，字符不是直接送进程序的。它们先暂存在一个叫"输入缓冲区"的地方。你可以在缓冲区里随便打字、删改，程序完全不知道。等你按下**回车键**的那一刻，缓冲区里的所有内容才一次性交给程序处理。

这就是为什么你可以慢慢打字，程序不会中途打断你——它一直在等你的回车。也是为什么你可以一次输入很多个值——它们都在缓冲区里排队，cin 一个一个取。

理解缓冲区有个实际好处：如果你写了一个 `cin >> a >> b`，但用户只输入了一个数字就按了回车——程序不会报错，它会继续等着用户输入第二个数字。因为缓冲区里只有一个数字，不够，程序就继续等。

好了，今天的练习是写一个完整的对话程序。下节课我们学 cout 的高级输出控制——怎么让小数保留两位、怎么控制输出的格式。

## 🖥️ 板书/屏幕要点
```
一次读多个值：

  int a, b;
  cin >> a >> b;    // 用户输入 "10 20"
                    // a=10, b=20

  输入方式灵活：空格分隔或换行分隔都行

综合对话程序骨架：
  ① #include <iostream> + <string>
  ② 声明变量（string/int/double）
  ③ 用 cout 提问
  ④ 用 cin 接收答案
  ⑤ 用 cout 输出结果

输入缓冲区：
  键盘输入 → 缓冲区（暂存） → 按回车 → 交给程序
  好处：可以慢慢打字，可以一次输入多个值
```

## ✏️ 随堂练习
写一个"自我介绍生成器"：依次询问用户的姓名（string）、年龄（int）、身高（double，单位米）。然后输出一段完整的自我介绍——"我叫[名字]，今年[年龄]岁，身高[身高]米。" 编译运行，用你自己的信息测试。

## 📌 小结（30秒总结）
`cin >> a >> b` 可以一行读取多个数据，用空格或换行分隔都行。输入缓冲区的概念帮助理解"为什么要按回车"——数据先在缓冲区排队，回车键是"提交"信号。综合对话程序 = 变量声明 + cin 输入 + cout 输出，是你第一个真正能跟用户互动的完整程序。

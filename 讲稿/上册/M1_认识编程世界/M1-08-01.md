# 第8章 第1课时：从 IPO 骨架到防御性编程——计算器的"筋骨"

## 📝 讲稿正文

同学们，欢迎来到 M1 模块的收官之战——第八章！

回顾一下，你从第一章开始，认识了计算机是什么，学会了 Hello World，掌握了变量、六种类型、输入输出、运算符，还有数学工具箱。现在所有"零件"都准备好了，我们要把它们**组装成一个完整的机器**——一个能用的计算器。

先来听一个故事。小明学编程刚满一个月。有一天他妈妈问他："你学了这么久，能做个什么东西出来？"

小明愣了一下。他会写 `int a = 5;`，会用 `cin >> x;`，会算 `a + b`……但这些东西好像都是零件，从来没有组装成一台完整的机器。

他决定做一个计算器。妈妈输入两个数字和一个运算符，程序自动算出结果。一个小时后，小明把笔记本抱到妈妈面前："妈，试试我的计算器。"

妈妈输入 "15 / 0"。程序没有崩溃，而是优雅地提示："除数不能为 0！"

妈妈惊讶地看了小明一眼："你还知道考虑这个？"

小明笑了——这就是编程教会他的第一件事：**写程序不只是让功能跑起来，还要考虑它可能遇到的所有情况。**

---

好，故事讲完了。今天我们就来做小明做的这件事——从零设计一个计算器。

第一步，不是敲代码，是画骨架。还记得第一章学的 IPO 模型吗？

- **I（Input，输入）**：用户输入两个数字（操作数）和一个运算符（+、-、*、/）。
- **P（Process，处理）**：根据运算符，选择对应的计算方式。
- **O（Output，输出）**：把计算结果展示在屏幕上。

先把这个骨架画清楚，再往里填肉。这是高效编程的第一步——在动手之前，先想清楚。

好，骨架有了，我们来填充内容。

用户输入的运算符有四种可能——`+`、`-`、`*`、`/`。程序怎么知道用户输入的是哪一个、该做哪种计算？这里需要一个"选择结构"——根据条件走不同的分支。这个结构我们今天第一次正式用，但你一眼就能看懂：

```cpp
char op;   // 存运算符（单个字符用 char）
cin >> a >> op >> b;  // 一次读三个：数 运算符 数

if (op == '+') {
    result = a + b;
} else if (op == '-') {
    result = a - b;
} else if (op == '*') {
    result = a * b;
} else if (op == '/') {
    // 除法特殊处理
}
```

注意，运算符用 `char` 类型存储——一个字符变量，刚好装一个 `'+'` 或 `'*'`。

---

现在来说最关键的一点——**防御性编程。**

什么叫防御性编程？就是"假设用户会犯所有可能的错，提前把它们都挡住"。

在计算器里，最危险的操作是什么？**除以 0。** 15 ÷ 0 在数学上没有意义，在 C++ 里——如果是整数除法，程序直接崩溃；如果是浮点除法，输出一个奇怪的 `inf`（无穷大）。

所以，在做除法之前，我们必须加一堵"防火墙"：

```cpp
if (op == '/') {
    if (b == 0) {
        cout << "错误：除数不能为 0！" << endl;
        return 1;   // 异常退出
    }
    result = a / b;
}
```

多写这四行代码，你的计算器就从"脆弱的玩具"升级成"坚固的工具"。这种思维习惯，是普通程序员和优秀程序员之间的第一道分水岭。

今天我们只搭了骨架——IPO 分析、选择结构判断运算符、除零检查。下节课我们把所有零件拧紧，写出完整可运行的计算器。

2. "除了除数为 0，你还能想到计算器会遇到什么'意外情况'？"
3. "防御性编程的'防御'是在防御什么？为什么叫它防火墙？"

## 🖥️ 板书/屏幕要点
```
计算器 IPO 骨架：
  Input:  两个操作数（double）+ 一个运算符（char）
  Process: 根据运算符选择运算方式
  Output:  计算结果

运算符判断框架：
  char op;
  cin >> a >> op >> b;
  // 一行读三个值，自动跳过空格！

  if (op == '+')      → 加法
  else if (op == '-') → 减法
  else if (op == '*') → 乘法
  else if (op == '/') → 除法（需检查除数！）

防御性编程 — 除零检查：
  if (b == 0) {
      cout << "错误：除数不能为 0！" << endl;
      return 1;   // 异常退出
  }
```

## ✏️ 随堂练习
不写完整代码，只在纸上画出你的计算器程序结构：
1. 列出所有需要声明的变量（类型+名字+用途）
2. 画出判断运算符的流程图（用简单的箭头和分支）
3. 标出所有需要"防御"的位置（哪些运算有潜在风险？）

画完后跟同桌交换互评，看看有没有遗漏的防御点。

## 📌 小结（30秒总结）
写程序之前先画 IPO 骨架——输入什么、处理什么、输出什么——这是高效编程的第一原则。运算符用 char 存，if-else 做选择分支。最重要的是防御性编程——除法前必须检查除数是否为 0，一个 if 语句挡住所有崩溃。这个习惯从现在开始养成，你写的程序质量就能超过 80% 的初学者。

# 第2章 第1课时：准备工作——搭建你的编程环境

**时长**：15-20分钟 | **对应教材**：第2章 · 2.1节

## 🎯 本课时目标
- 学生能理解 IDE 是什么、为什么需要它
- 学生能在自己的电脑上安装一个 C++ IDE
- 学生能打开 IDE 并新建一个 C++ 源文件

## 📝 讲稿正文

同学们，恭喜你们顺利完成了第一章！从现在开始，我们不只动脑，还要动手了。

如果你学游泳，第一课一定不是教你蝶泳——第一课是**下水**。编程也一样。不管你以后要写出多厉害的程序，第一件事永远是：打开电脑，敲一行代码，看到它在屏幕上跑起来。

但在下水之前，我们需要先准备好"游泳池"——也就是编程的环境。

编程需要什么？不需要一台超级电脑。你们现在用的任何一台能开机的电脑都够了。你只需要两样东西：**第一样，一个 IDE；第二样，一双愿意敲键盘的手。**

好，先说 IDE。IDE 的全称是 Integrated Development Environment，集成开发环境。翻译成人话就是——**一个专门用来写代码的软件。**

你可能会问：我用记事本写不行吗？

技术上可以，但会非常痛苦。IDE 跟记事本的区别，就像手术刀跟菜刀的区别——都能切东西，但一个专门为做手术设计，一个不是。

IDE 至少帮我们做三件事：第一，用不同颜色标记代码的不同部分，让你一眼就能看清哪个是关键字、哪个是字符串——这叫**语法高亮**。第二，在你敲代码的时候检查有没有语法错误，像自动拼写检查一样——有的 IDE 还会在错误下面画红色波浪线。第三，最重要的一点——一键编译运行，你不用手动敲复杂的命令。

对于 C++ 初学者，我给大家推荐三个 IDE：

第一个是 **Dev-C++**。它的最大优点就是轻量——安装包小、启动快、装了就能用。没有花里胡哨的功能，但该有的都有。电脑配置不高的同学首选这个。

第二个是 **Code::Blocks**。功能比 Dev-C++ 更全，稳定性很好，是大部分信息学竞赛选手的选择。很多考场的电脑上装的就是它。

第三个是 **VS Code**。界面现代、插件丰富，是你以后可能一直用到大学毕业的工具。但它的配置稍微复杂一点——需要额外安装编译器和配置环境变量。如果你喜欢折腾、想自定义一切，选它。

我的建议是：大部分同学先用 Dev-C++ 或 Code::Blocks 入门，等以后熟悉了编程，再根据自己的喜好换。对今天这节课来说，不管你选哪个 IDE，请确保下课前完成了两件事：

第一，安装好了 IDE 并能正常打开。
第二，新建了一个 C++ 源文件——这个文件的后缀名是 `.cpp`。

如果你现在还没装好，举手找老师帮忙。装好的同学，试着在 IDE 里找到"编译并运行"的按钮——它一般在工具栏上，图标是 ▶ 或者一个齿轮。不同 IDE 位置不同，自己先找找看。

好，另一半准备工作——那双"愿意敲键盘的手"。编程是**手艺活**，不是听书活。看一百行代码不如自己敲一行。从今天开始，我写在屏幕上、写在书上的每一段代码，你们都要亲手敲一遍。**不要复制粘贴，不要复制粘贴，不要复制粘贴。** 重要的事情说三遍。编程是肌肉记忆，你的手指需要记住敲代码的感觉。

好，今天的环境搭好了，下节课我们就来敲人生第一行 C++ 代码。大家期待吗？

## 🖥️ 板书/屏幕要点
```
IDE = Integrated Development Environment
     = 集成开发环境
     = 专门写代码的软件

IDE 三大功能：
  ① 语法高亮 — 不同颜色标记代码
  ② 错误检查 — 红色提示
  ③ 一键编译运行 — 按 ▶ 就行

推荐 IDE（三选一）：
  Dev-C++     → 轻量、简单、配置低首选
  Code::Blocks → 功能全、稳定、竞赛选手爱用
  VS Code     → 现代、插件多、但配置稍复杂

编程第一原则：
  看一百行 ≠ 敲一行
  亲手敲 → 肌肉记忆
  不要复制粘贴！
```

## ✏️ 随堂练习
安装好你的 IDE，然后新建一个 C++ 源文件，命名为 `hello.cpp`。在文件里随便敲几个字母，保存。确认你的 IDE 能够正常创建和保存 `.cpp` 文件。如果不能，说出来让老师帮你解决。

## 📌 小结（30秒总结）
IDE 就是你写代码的游泳池——有语法高亮、错误检查、一键运行。选一个适合你的装上就行。但最重要的工具不是 IDE——是你自己的手。编程是手艺活，亲手敲每一行代码。今天搭好环境，明天我们就下水——写出人生第一个 C++ 程序！

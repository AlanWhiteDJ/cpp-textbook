# 第6章 第1课时：算术运算符 + 整数除法的"陷阱"

**时长**：15-20分钟 | **对应教材**：第6章 · 6.1-6.2节

## 🎯 本课时目标
- 学生能使用全部 5 种基本算术运算符
- 学生能准确区分"整数除法"和"浮点除法"的规则
- 学生理解"整数除法的结果不是四舍五入，而是直接丢弃小数部分"

## 📝 讲稿正文

同学们，前面五章我们学会了用变量存数据、用 cin 读数据、用 cout 看数据。但程序真正的本事是什么？是**计算**。今天开始，我们要让程序做算术题——从加减乘除开始，一路学到各种高级运算。

C++ 的基本算术运算符有五种：加、减、乘、除、取模。今天先讲前四种。

**加法 `+`**、**减法 `-`**、**乘法 `*`**——这三样跟数学课一模一样，没什么好说的。比如 `5 + 3 = 8`，`5 - 3 = 2`，`5 * 3 = 15`。

注意乘号是星号 `*`，不是字母 x——因为字母 x 很容易跟变量名 x 混淆。

好，重头戏来了——**除法 `/`**。

除法在 C++ 里有一个"陷阱"，每年都有无数初学者掉进去。这个陷阱就是——**整数除法和浮点除法的规则不同。**

规则只有一句话：**两个操作数都是整数 → 整数除法（丢弃小数部分）。至少有一个操作数是浮点数 → 浮点除法（保留小数）。**

来看几个例子：

```cpp
cout << 5 / 2 << endl;       // 输出 2  ← 整数除法，丢掉了 0.5
cout << 5.0 / 2 << endl;     // 输出 2.5 ← 浮点除法
cout << 5 / 2.0 << endl;     // 输出 2.5 ← 浮点除法
cout << 5.0 / 2.0 << endl;   // 输出 2.5 ← 浮点除法
```

看到区别了吗？只要参与运算的两个数中有一个是带小数点的，C++ 就自动切换到浮点除法。

再来看一个更具迷惑性的例子：

```cpp
int a = 5 / 2;        // a = 2
double b = 5 / 2;     // b = 2.0 —— 不是 2.5！
double c = 5.0 / 2;   // c = 2.5
```

第二行很反直觉——明明声明了 `double b`，为什么结果还是 2.0？因为 C++ 的执行顺序是：**先算等号右边，再把结果放进左边。** 右边 `5 / 2` 是两个整数相除，先做整数除法得到 2（整数），然后再把这个 2 转换成 2.0 放进 double b。到这一步的时候，0.5 早就在整数除法那一步被丢掉了。

这个规则极其重要，请大家务必记住。如果你想得到小数结果，就要确保除法中至少有一个数是浮点数。最简单的做法是**把其中一个数写成带小数点的形式**——比如 `5.0 / 2`。

另外，整数除法不是四舍五入——是**直接丢弃小数部分**。`7 / 3 = 2`，不是 `3`；`19 / 5 = 3`，不是 `4`。哪怕结果是 2.999，整数除法也只会得到 2。这一点跟数学课的直觉完全不同，需要适应。

好了，今天的关键词：**整数除法丢弃小数，浮点除法保留小数。** 这个认知，你们会在后面的编程里反复用到。下节课我们讲第五种算术运算符——取模 `%`，它在编程里的用途你绝对想不到。

## 🖥️ 板书/屏幕要点
```
算术运算符：
  +  加    5 + 3 = 8
  -  减    5 - 3 = 2
  *  乘    5 * 3 = 15（注意是*不是x）
  /  除    看下去↓
  %  取模（下节课讲）

除法的黄金规则：
  两个整数 ÷ → 整数除法（丢小数！）
  有浮点数 ÷ → 浮点除法（保留小数）

  5 / 2     = 2    （整数除法）
  5.0 / 2   = 2.5  （浮点除法）
  5 / 2.0   = 2.5  （浮点除法）

  double x = 5 / 2;     // x = 2.0 ❌
  double x = 5.0 / 2;  // x = 2.5 ✓

  整数除法 = 直接丢弃小数，不是四舍五入！
```

## 📌 小结（30秒总结）
加减乘跟数学课一样，乘号是星号不是字母 x。除法是关键：两个整数除 → 丢小数（不是四舍五入），有一个带小数点 → 保留小数。`double x = 5 / 2;` 结果为 2.0——因为先做整数除法再转换，0.5 已经丢了。要得小数，写成 `5.0 / 2`。

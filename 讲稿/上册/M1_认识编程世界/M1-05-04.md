## 📝 讲稿正文

同学们，今天是第五章最后一课。我们要做一个"大综合"——把所有输入输出的知识揉在一起，写出一个完整的互动程序，然后做全章复习。

先来做一个综合练习。题目是：**写一个"超市收银小程序"。**

要求：程序询问用户三个信息——商品名称、单价、购买数量。然后计算总价，显示一张漂亮的收据。总价保留两位小数。

我们来分析一下这个程序的 IPO：
- **Input（输入）**：商品名称（string）、单价（double）、数量（int）
- **Process（处理）**：总价 = 单价 × 数量
- **Output（输出）**：一张格式化的收据

把骨架画清楚之后，代码就水到渠成了：

```cpp
#include <iostream>
#include <string>
#include <iomanip>
using namespace std;

int main() {
    string productName;
    double price;
    int quantity;

    cout << "=== 超市收银系统 ===" << endl;
    
    cout << "商品名称：";
    cin >> productName;
    
    cout << "单价（元）：";
    cin >> price;
    
    cout << "数量：";
    cin >> quantity;
    
    double total = price * quantity;  // 计算总价
    
    cout << "\n===== 购物小票 =====" << endl;
    cout << "商品：" << productName << endl;
    cout << "单价：¥" << fixed << setprecision(2) << price << endl;
    cout << "数量：" << quantity << " 件" << endl;
    cout << "总价：¥" << fixed << setprecision(2) << total << endl;
    cout << "====================" << endl;
    
    return 0;
}
```

大家看，这个程序用到了 string、double、int 三种类型，每行输入用 cin，每行输出用 cout + 格式化。不到 30 行代码，就实现了一个有模有样的小程序——这已经非常接近真实的应用程序了。

注意代码中的 `\n`——单独一个空行是通过开头 `"\n====="` 里的 `\n` 实现的。这也是 `\n` 的一个实际用法。

---

好，现在我们来回顾第五章的全部内容。

第五章的标题叫"让程序跟你聊天"。我们学了什么？

第一条主线：**输入。** `cin >>` 是数据从键盘流向程序，箭头指向变量。能读几乎所有类型。多个 cin 可以串联：`cin >> a >> b;`。输入缓冲区的概念——数据在缓冲区里排队，回车键是"提交"信号。

第二条主线：**输出。** `cout <<` 是数据从程序流向屏幕。`<<` 串联各种内容。`endl` 或 `\n` 换行。`fixed` + `setprecision(N)` 控制小数位数，需要 `#include <iomanip>`。

第三条主线：**整合。** 把变量、类型、输入、输出组合在一起，写出能跟用户多轮互动的完整程序。从"请告诉我你的名字"到"这是你的购物小票"——程序不再是冷冰冰的计算器，而是能对话的助手。

记住一个总图：**程序 = 接收输入 → 处理数据 → 输出结果。** 输入输出就是这个 IPO 链条的首尾两端。中间的处理环节——计算、判断——我们下一章就来攻克。

## 🖥️ 板书/屏幕要点
```
综合示例 — 超市收银小程序：
  I: string name + double price + int quantity
  P: total = price * quantity
  O: 格式化收据（金额保留两位小数）

第五章知识地图：
  
  cin >> 变量         输入（键盘 → 程序）
  cin >> a >> b       多值串联输入
  输入缓冲区           暂存 → 回车提交
  
  cout << 内容         输出（程序 → 屏幕）
  << 串联              把各种内容粘成一行
  endl / \n            换行
  fixed + setprecision(N)  小数位数控制
  #include <iomanip>   格式化工具箱
```

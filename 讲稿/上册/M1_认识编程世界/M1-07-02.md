# 第7章 第2课时：幂运算与比较函数——pow、max、min

**时长**：15-20分钟 | **对应教材**：第7章 · 7.4-7.5节

## 🎯 本课时目标
- 学生能用 `pow(a, b)` 计算 a 的 b 次方
- 学生知道 `pow` 返回的是 double，存在浮点误差
- 学生能用 `max` 和 `min` 比较两个数的大小

## 📝 讲稿正文

上节课我们打开了 `<cmath>` 工具箱，学了绝对值 `abs`/`fabs` 和平方根 `sqrt`。今天继续翻箱底——`pow` 幂运算和 `max`/`min` 比较函数。

先说 `pow`。它的全称是 power，意思是"幂"。`pow(a, b)` 就是 a 的 b 次方——$a^b$。

用法很简单：

```cpp
cout << pow(2, 10) << endl;   // 2^10 = 1024
cout << pow(3, 4) << endl;    // 3^4 = 81
cout << pow(10, 6) << endl;   // 10^6 = 1000000
cout << pow(5, 2) << endl;    // 5^2 = 25
```

但是——这里有一个非常重要的提醒。**`pow` 返回的是 `double` 类型，存在微小的浮点误差。**

什么意思呢？理论上 `pow(5, 2)` 应该精确等于 25.0。但在计算机内部，它可能被算成 25.00000000000001 或者 24.99999999999999。这种误差极其微小（小数点后十几位），但你如果直接把它转成 `int`，可能会出问题：

```cpp
int result = pow(5, 2);   
// result 可能是 24！（如果内部值是 24.999...）
```

因为 `double` 转 `int` 的时候是直接截断小数部分——24.999999 截断成 24，不是四舍五入成 25。

那怎么办呢？有一个简单的补救方法——**加 0.5 再截断**：

```cpp
int result = (int)(pow(5, 2) + 0.5);  // 安全！
```

加 0.5 相当于手动做了四舍五入——24.999 + 0.5 = 25.499，截断后就是 25。

但对于小的整数幂（比如平方、立方），更推荐的做法是**直接写乘法**——`5 * 5` 肯定比 `pow(5, 2)` 精确，而且跑得更快。竞赛里需要幂运算的时候，能直接乘就直接乘，实在指数太大才用 `pow`。

---

好了，现在来学第二组工具——**比较大小：`max` 和 `min`。**

有时候你需要知道两个数谁大谁小。`max(a, b)` 返回较大的那个，`min(a, b)` 返回较小的那个。

```cpp
#include <algorithm>  // max 和 min 在这个头文件里
#include <iostream>
using namespace std;

int main() {
    int a = 15, b = 8;
    cout << max(a, b) << endl;   // 15
    cout << min(a, b) << endl;   // 8

    double x = 3.14, y = 2.71;
    cout << max(x, y) << endl;   // 3.14

    // 三个数中找最大：嵌套使用
    cout << max(max(10, 25), 7) << endl;  // 25
    
    return 0;
}
```

这里要特别注意——`max` 和 `min` 是定义在 `<algorithm>` 头文件里的，不是 `<cmath>`！所以你需要写 `#include <algorithm>`。

不过有些编译器会自动通过 `<iostream>` 间接引入 `<algorithm>`，所以你可能不写 `#include <algorithm>` 也能用 `max` 和 `min`。但这不标准，我建议你每次都写上——保平安。

另外，如果你不想引入 `<algorithm>`，还可以用一个叫做"三目运算符"的东西——`a > b ? a : b`，效果和 `max(a, b)` 一样。这个三目运算符我们后面模块会详细讲，现在知道有这么个东西就行。

好了，今天的两个工具：`pow` 做幂运算（注意浮点误差），`max`/`min` 做大小比较（需要 `<algorithm>`）。下节课我们学本章的"重头戏"——随机数。

```
pow(a, b) — a 的 b 次方
  pow(2, 10) = 1024
  pow(3, 4)  = 81

  ⚠ pow 返回 double，有浮点误差！
  int result = (int)(pow(5, 2) + 0.5);  // 安全整数做法
  // 小指数推荐直接写乘法：5*5 > pow(5,2)

max/min — 比较大小
  #include <algorithm>   ← 别忘！

  max(a, b)  → 返回较大的
  min(a, b)  → 返回较小的

  max(max(10, 25), 7)  → 三个数中最大 = 25

  三目运算符（预习）：a > b ? a : b
```

## 📌 小结（30秒总结）
`pow(a,b)` 计算 a 的 b 次方，但注意它是 double 类型有浮点误差——小指数直接乘，大指数用 `pow` 后加 0.5 再转整数。`max` 和 `min` 比较大小非常直观，在 `<algorithm>` 头文件里，可以嵌套处理多个数。下节课——随机数，让程序变得"不可预测"。

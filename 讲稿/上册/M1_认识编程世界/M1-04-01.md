# 第4章 第1课时：类型是什么——int 和它的"大哥" long long

**时长**：15-20分钟 | **对应教材**：第4章 · 4.1-4.3节

## 🎯 本课时目标
- 学生能理解"数据类型"的概念——告诉计算机分配多大空间
- 学生知道 int 占 4 字节，能存约 ±21 亿
- 学生知道什么时候该换 long long

## 📝 讲稿正文

同学们，前面三章我们一直在用一个类型——`int`。我们用 `int` 存年龄、存分数、存各种整数，一直没出过问题。

但今天我要告诉你们一个"恐怖"的事情——**int 不是万能的。** 如果你让它存一个太大的数字，程序不会报错，但会给你一个完全错误的结果。就像你拿一个小杯子去接一整桶水——多出来的水全洒了，你得到的只是一小杯。

我先讲一个真实的故事。小刚写了一个程序，算他玩的游戏里一共打了多少分。他写了 `int score = 8500000000;` —— 85 亿分。程序运行，结果显示：**5237120。**

他懵了。"我没写错啊？"

他没写错算法，他选错了装数字的容器。有些容器装不下大数字，硬塞就会出问题。

这就是今天我们要聊的主题——**数据类型。**

计算机的内存不是无限大的。当你声明一个变量，计算机需要在内存里给它分配空间。数据类型就是告诉计算机一句话："请给我准备多大的空间，我要装什么。"

就像你去奶茶店点单——"我要小杯 / 中杯 / 大杯"。不同的杯能装不同量的饮料。数据类型就是杯子的尺寸。

好，我们先来看整数家族的两个成员。你们已经认识了 `int`，今天介绍它的大哥——`long long`。

| 类型 | 占用空间 | 能存的范围 | 适合场景 |
|------|----------|-----------|---------|
| `int` | 4 字节 | 约 -21 亿 ~ 21 亿 | 人数、年龄、计数 |
| `long long` | 8 字节 | 约 ±900 亿亿 | 大乘积、国家 GDP |

4 字节和 8 字节是什么概念？一个字节可以存一个英文字母。4 个字节有 32 个二进制位，能组合出约 42 亿种不同的状态——一半分给负数，一半分给正数和零，所以范围是约 ±21 亿。

8 个字节有 64 个二进制位，能表示的范围就大得多了。

好，关键规则来了——**记住一句话：int 最多装到约 21 亿。超过这个数，立刻换 long long。**

什么场景会超过 21 亿？两个 5 万相乘是 25 亿，超了。一百万乘十万等于一千亿，超了。竞赛里经常要算总的方案数、总的分数——轻轻松松就超 21 亿。所以竞赛选手有一个习惯：**不确定会不会超的时候，一律用 long long。**

用 long long 怎么写呢？跟 int 一样简单：

```cpp
long long stars = 100000000000LL;  // 1000 亿，必须用 long long
long long total = 0;                // 累加器，从 0 开始
long long product = a * b;          // 两个大数相乘
```

注意那个 `LL` 后缀——它告诉编译器"这是一个 long long 类型的数字，别当 int 处理"。现阶段写不写 LL 大部分情况都能自动转换，但养成写的习惯没坏处。

好了，今天的关键就一句话：**int 是主力，long long 是后盾。数字超过 21 亿，别犹豫，换 long long。**

下节课我们要聊一个更刺激的话题——如果硬把大数字塞进 int，会发生什么？这就是"溢出"。

## 💬 互动提问
1. "你们学校有多少人？全国有多少中小学生？哪个能用 int 存，哪个得用 long long？"
2. "为什么 int 的范围是 ±21 亿而不是一个整数？你能联想到 2 的多少次方吗？"
3. "你觉得 C++ 为什么不把所有整数都默认用 long long？"

## 🖥️ 板书/屏幕要点
```
数据类型 = 告诉计算机：给我多大的空间

int         → 4 字节 → 约 -21亿 ~ 21亿  （主力）
long long   → 8 字节 → 约 ±9×10¹⁸       （后盾）

黄金规则：
  数字超过 21 亿 → 换 long long！

什么时候会超？
  两个 5 万相乘 = 25 亿 > 21 亿 ❌
  一百万 × 十万 = 一千亿 ❌
  竞赛选手习惯：不确定就用 long long

long long 写法：
  long long total = 0;
  long long product = 100000LL * 100000;
```

## ✏️ 随堂练习
在 IDE 里写程序：分别用 `int` 和 `long long` 计算 50000 × 50000（5 万乘 5 万）。输出两个结果，看看 int 的版本输出什么奇怪的数字。这个实验让你亲眼看到 int 的"天花板"。

## 📌 小结（30秒总结）
数据类型就是杯子的尺寸——不同大小能装不同范围的数据。int 是 4 字节的主力杯，装到约 21 亿。long long 是 8 字节的大杯，装到约 900 亿亿。超过 21 亿必须换 long long——程序不报错但结果会错，这比报错还可怕。养成"不确定就用 long long"的竞赛习惯。

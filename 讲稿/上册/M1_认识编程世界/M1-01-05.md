# 第1章 第5课时：第一章总复习——你建立的"计算机世界观"

**时长**：15-20分钟 | **对应教材**：第1章 · 全章复习

## 🎯 本课时目标
- 学生能完整复述计算机五部分、程序概念、编程语言作用和 IPO 模型
- 学生能纠正自己对计算机的常见误解
- 学生能自信地回答"计算机是什么"这个问题

## 📝 讲稿正文

同学们，今天是我们第一章的收官课。四节课下来，我们从"计算机是什么"这个问题出发，一路聊到了程序的骨架 IPO 模型。这节课我们不学新东西，而是把前面四节课串起来，看看第一章到底给了你什么武器。

首先，我要请大家回忆一个场景——第一节课我问你们，如果朋友从 1980 年穿越过来，指着电脑问"这到底是什么"，你怎么用三句话回答他？

现在学了四节课，你的答案应该是什么？

（让学生回答）

我帮大家梳理一下。第一句话：**计算机是一个能执行指令的机器，它自己不会思考。** 第二句话：**它由五个部分组成——输入设备、输出设备、运算器、控制器、存储器，CPU 就是运算器加控制器。** 第三句话：**你要让它干活，就得写程序——程序就是一份指令清单，用编程语言写好后，编译器帮你翻译成机器语言，计算机就能执行了。**

三句话，把你第一章的核心收获全说出来了。

好，现在我们来做一件重要的事——**纠正误解。** 很多人在学编程之前，对计算机有一些"想当然"的误解。今天我们要一个一个戳破。

**误解一："计算机很聪明，什么都会。"** 真相是什么？计算机是最笨的东西！它只会做你让它做的事，一条不多，一条不少。它不会自己决定"这道题太难了我不做了"——你让它除以零，它真的会去尝试除以零然后崩溃。它听话，但不聪明。

**误解二："编程是黑客戴着兜帽在黑暗里敲键盘，很神秘。"** 真相是什么？编程就是写指令清单，跟写菜谱没有本质区别。菜谱是"先放油，再放菜，炒三分钟"，程序是"先读一个数，再读一个数，加起来，输出"。就这么简单。

**误解三："内存就是 C 盘 D 盘。"** 这个我们已经反复强调过了——内存是工作台，关机消失；外存是仓库，永久保存。C 盘 D 盘是外存，不是内存。

好，最后我们做一个全章知识串联。请大家跟着我一起说——

计算机有五个部分……（等学生一起说）程序是……编程语言是……IPO 是……

很好！第一章给你们建立的是一个"计算机世界观"——你看计算机的眼光，从此不一样了。你不再是一个只会用电脑的人，而是一个开始理解电脑怎么工作的人。这个转变，比学会任何一行代码都重要。

下节课，我们就要真正动手了——打开 IDE，敲下人生第一行 C++ 代码。准备好了吗？

## 🖥️ 板书/屏幕要点
```
第一章知识地图：

计算机五部分 → 硬件/软件 → 程序 = 指令清单
                              ↓
                        编程语言 = 翻译官
                              ↓
                        C++ = 竞赛主力
                              ↓
                         IPO = 程序骨架

三大误解纠正：
  ✗ 计算机很聪明  → ✓ 计算机很听话
  ✗ 编程很神秘    → ✓ 编程 = 写指令清单
  ✗ 内存 = C盘    → ✓ 内存 = 工作台，外存 = 仓库
```

## ✏️ 随堂练习
拿出一张白纸，不看书，画出第一章的"知识地图"——用方框写下每一课时的核心概念，用箭头标出它们之间的关系。至少画出 6 个方框。和同桌交换，互相补充遗漏的知识点。

## 📌 小结（30秒总结）
第一章不是教你写代码，是教你"怎么看计算机"。你现在知道了计算机是听话的机器、程序是指令清单、编程语言是翻译官、IPO 是程序的骨架。这四个认知，是你今后几百章的基础。带着它们，下一章我们亲手让计算机说"Hello World"！

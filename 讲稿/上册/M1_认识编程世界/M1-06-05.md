## 📝 讲稿正文

同学们，今天是第六章最后一节课。我们做三件事：搞定运算符优先级，建立运算安全范围的意识，然后串一遍全章。

先说优先级。你们数学课上学过"先乘除，后加减"——C++ 里也是一样的。`3 + 5 * 2` 的结果是 13 不是 16，因为乘法优先级高于加法。

完整的优先级顺序，从高到低：

```
最高：  ( )    括号
  ↑    ++ --  自增自减（前置）
  ↑    * / %  乘除取模
  ↑    + -    加减
最低：  = += -= 等赋值
```

但是——**我不建议你们背这个表。** 我建议你们用一个更聪明的策略：**拿不准的，加括号。**

```cpp
// 不用猜，直接加括号明确意图：
int result = (a + b) * (c - d);
int answer = (x % 10) + (y / 10) * 3;
```

括号不仅让程序正确，也让读你代码的人——包括一个月后的你自己——一眼就能看懂你的意图。多敲两个括号，比猜优先级划算一万倍。

一个具体例子：`a * b + c` 这个怎么写更清晰？

不加括号也能跑——C++ 知道乘法优先。但如果你一个月后回来看，要花半秒在大脑里过一遍优先级。但如果你写成 `(a * b) + c`，连半秒都不用——意图一眼明了。

所以记住今天的黄金法则：**怀疑优先级 → 加括号。不确定优先级 → 加括号。想让代码更清晰 → 加括号。括号不要钱，但省时省命。**

---

好，第二件事——**整数运算的安全范围。**

这个我们在第四章讲过溢出，今天结合乘法再说一遍。任何两个 int 相乘，结果可能超过 int 的 21 亿上限。比如：

```cpp
int a = 50000;
int b = 50000;
int c = a * b;    // 25 亿 > 21 亿 → 溢出！c 是垃圾值！
```

怎么办？用 long long：

```cpp
long long a = 50000;
long long b = 50000;
long long c = a * b;   // 安全！
```

给你的安全口诀：**两个 10 万相乘 = 100 亿，必超 int。不确定会不会超，就用 long long。**

---

最后，我们来串第六章的全部内容。

第六章标题叫"做算术题"，我们学了六大块知识：

第一块，五种算术运算符：`+ - * / %`。其中 `/` 和 `%` 最需要留意——整数除法丢小数，取模只能用于整数。

第二块，自增自减：`++` 和 `--`。前置先加后用，后置先用后加。单独使用随便选，别混用。

第三块，复合赋值：`+= -= *= /= %=`。运算加赋值一步到位，变量名只写一次。

第四块，运算符优先级：记不住就加括号——永远正确的策略。

第五块，安全范围：乘法容易溢出，不确定就用 long long。

第六块，取模的三大神技：判奇偶、取数位、做循环。

第六章是你们运算能力的"弹药库"。这些运算符，你们接下来的每一章、每一个程序都会用到。下章我们进入 M1 的收官阶段——把数学工具箱和变量进阶技术装进你的武器库。

## 🖥️ 板书/屏幕要点
```
运算符优先级（高→低）：
  ( )              最优先
  ++ --（前置）
  * / %
  + -
  = += -= *= /= %=  最低

黄金法则：不确定就加括号！
  (a + b) * c   比   a + b * c   更清晰

安全运算范围：
  int × int 可能溢出（>21亿）
  → 用 long long 防止溢出
  → 口诀：两个10万相乘，必超int

第六章知识地图：
  算术：+ - * / %    (/丢小数，%只整数)
  自增：++ --        (前置vs后置)
  复合：+= -= *= /= %=
  优先级：加括号最安全
  安全：long long防溢出
```

## ✏️ 随堂练习
以下代码的输出是什么？先推算，再上机验证。

```cpp
int a = 10, b = 3;
cout << a / b << " " << a % b << endl;  // ?
a += b * 2;
cout << a << endl;  // ?
a *= a - 10;
cout << a << endl;  // ?
int c = a++;
int d = ++a;
cout << c << " " << d << " " << a << endl;  // ?
```

如果中间有任何一步跟你推算的不一样，把那一步的执行顺序用箭头画出来。

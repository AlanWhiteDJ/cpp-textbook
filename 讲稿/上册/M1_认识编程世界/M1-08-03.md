# 第8章 第3课时：M1 全模块总复习——你走过的 35 节课

**时长**：15-20分钟 | **对应教材**：第8章 · 全章复习 + M1 全模块回顾

## 🎯 本课时目标
- 学生能画出 M1 八章的知识地图
- 学生能独立完成一个综合编程题——从分析到编码
- 学生对 M2 模块建立清晰的期待

## 📝 讲稿正文

同学们，今天我们相聚在 M1 模块的最后一节课。35 节课，8 个章节——你从一个对编程一无所知的新手，变成了能独立写出计算器的程序员。这一路我们走了多远，今天来盘一盘。

我们先来做个"知识接力"——我开个头，你们接下去。

第一章，我们认识了计算机。计算机是什么？（等学生回答）——听话的机器。它有五个部分——输入、输出、运算器、控制器、存储器。CPU 就是运算器加控制器。内存是草稿纸，外存是笔记本。程序的骨架是——IPO：输入 → 处理 → 输出。

第二章，你写出了人生第一行代码。Hello World！六行代码你到现在应该烂熟于心了吧？`#include <iostream>` 拿工具，`using namespace std` 开场白，`int main()` 入口，`cout` 输出，`return 0` 结束。编译是把 C++ 翻成机器语言，运行是真正执行。`//` 单行注释，`/* */` 多行注释。

第三章，程序学会了"记东西"。变量 = 内存里贴了标签的格子。`int score = 100;` 声明并初始化。命名三条硬性规定——字母数字下划线、数字不开头、不用关键字。`=` 不是等于，是赋值——把右边算出来放进左边。`const` 上锁，谁也别想改。

第四章，你打开了六种"容器"。数字组：int（4 字节，±21 亿主力军）、long long（8 字节，大数后盾）、double（8 字节，小数专家）。文字组：char（单引号，一个字符）、string（双引号，别忘了 `#include <string>`）。判断组：bool（只有 true 和 false）。溢出——超过范围不报错但结果全错，这是最阴险的 bug。

第五章，程序开始跟人"聊天"。`cin >>` 往里收，`cout <<` 往外送。`cin >> a >> b` 一次读多个。`fixed << setprecision(N)` 控制小数位，别忘了 `#include <iomanip>`。输入缓冲区——数据排队，回车提交。

第六章，程序做起了算术题。`+ - * / %`——`/` 整数除法丢小数，`%` 只能整数。三大神技：`% 2` 判奇偶，`% 10` 取个位，`% N` 做循环。`++`、`--`——前置先加后用，后置先用后加。`+= -= *= /= %=`——复合赋值一步到位。优先级记不住？加括号！

第七章，打开了数学瑞士军刀。`<cmath>` 里有 `abs`/`sqrt`/`pow`/`floor`/`ceil`/`round`。`<algorithm>` 里有 `max`/`min`。`<cstdlib>` 和 `<ctime>` 联手给你随机数——`srand(time(0))` 设置种子，万能公式 `rand() % (b-a+1) + a` 生成任意范围。

第八章，所有零件组装成计算器。IPO 骨架 → double 类型 → else-if 选择 → 除零防火墙 → 好代码习惯。

---

同学们，35 节课你们学了这么多。现在你们每个人都可以回答开学第一课的问题了——"计算机到底是什么？程序是什么？你怎么让计算机听你的话？"

但更重要的是——你们已经有了一个程序员的思维方式。

你知道计算机不会自己思考，只会精确执行你的指令。
你知道写程序之前要先想 IPO——输入什么、处理什么、输出什么。
你知道写代码要考虑所有可能的情况——防御性编程。
你知道代码是写给人看的——好变量名、注释、缩进。

这些思维习惯，比任何一个具体语法都更值钱。

---

最后，我们展望一下 M2 模块。你现在的计算器还很"傻"——每次只能算一道题，算完就退出。如果想连续计算，得一遍遍重新运行程序。M2 模块要解决的就是这个问题——让程序能**做判断、做选择、做循环**。

`if` 和 `else` 让你的程序学会"如果……那么……否则……"——下雨就带伞，不下雨就不带。
`switch` 让你的程序像铁轨分岔器——根据不同的选项走不同的路。
循环让你的程序能重复做事——算完一道再算下一道，直到你让它停。

这才是"智能"的开始。M1 给了你建筑材料——砖、水泥、钢筋。M2 开始教你盖房子——怎么把这些材料组合成结构精巧的程序。

同学们，M1 结业快乐。M2 见！

## ✏️ 随堂练习
给你 10 分钟，完成下面这道综合编程题（开卷，可查书和笔记）：

**题目：成绩计算器**

要求：
1. 询问用户姓名（string）
2. 询问语文、数学、英语三科成绩（double）
3. 计算总分和平均分（保留 2 位小数）
4. 判断每科是否及格（≥60 为及格）——用 cout 输出每科的及格状态（提示：暂用三目运算符 `score >= 60 ? "及格" : "不及格"`）
5. 输出一张格式化的成绩单

这是 M1 所有知识的综合检验。做完的同学举手，老师来检查。没做完也没关系——这个题就是你 M2 之前可以反复练习的"毕业设计"。

## 📌 小结（30秒总结）
M1 的 35 节课，8 章内容，从"计算机是什么"到"写出完整计算器"。你不仅学会了 C++ 的基本语法，更建立了程序员的思维模式——IPO 规划、防御性编程、好代码习惯。带着这些武器，准备迎接 M2——让你的程序学会判断、选择和循环，从"机器"进化成"智能体"。M1 结业快乐，我们 M2 见！

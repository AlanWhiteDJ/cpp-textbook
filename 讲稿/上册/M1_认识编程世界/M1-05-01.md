# 第5章 第1课时：cin——让程序"听"你说话

**时长**：15-20分钟 | **对应教材**：第5章 · 5.1-5.2节

## 🎯 本课时目标
- 学生能正确使用 `cin >>` 读取用户输入
- 学生能区分 `cout <<`（输出）和 `cin >>`（输入）的箭头方向
- 学生能用 cin 读取 int、double 类型的数据

## 📝 讲稿正文

同学们，前四章我们的程序都是"自言自语"——所有内容都是我们事先写在代码里的，程序跑一遍就结束，每次输出都一模一样。

但你想想真正的应用程序——你在搜索框里打字，搜索结果就变了；你在游戏里按方向键，角色就移动；你点一个按钮，程序就做一件事。这些程序跟我们的程序最大的区别是什么？

**它们能接收你的输入，然后根据输入做出不同的反应。**

这就是本章的主题——输入输出。之前我们一直在用 `cout` 做输出，今天我们要认识它的兄弟——`cin`。

cin 的全称是 c + in = character input，字符输入。它等你在键盘上敲东西，然后把你敲的内容读进变量。符号是 `>>`——跟 cout 的 `<<` 正好相反。

注意看方向——我用一个比喻帮大家记住：
- `cout <<` ——数据从程序流出去，到屏幕。箭头指向外面。
- `cin >>` ——数据从键盘流进来，到程序。箭头指向变量。

来看第一个例子——读取一个整数：

```cpp
int age;
cout << "请输入你的年龄：";
cin >> age;
cout << "你今年 " << age << " 岁" << endl;
```

运行效果：

```
请输入你的年龄：12
你今年 12 岁
```

四行代码，实现了程序跟用户的第一次对话！用户输入的数字 12 被读进了变量 age，然后程序用 age 输出了一句完整的话。

这个箭头方向太容易错了——很多初学者把 `cin >>` 写成 `cin <<`，因为眼睛习惯了 cout 的箭头。**记住：cout 是往外送，用 << ；cin 是往里收，用 >> 。**

cin 不只是能读 int——之前学的类型，它基本上都能读：

```cpp
int age;            cin >> age;      // 读整数
long long big;      cin >> big;      // 读大整数
double score;       cin >> score;    // 读小数
char grade;         cin >> grade;    // 读单个字符
string name;        cin >> name;     // 读一个单词
bool flag;          cin >> flag;     // 读 1（true）或 0（false）
```

但是！这里有一个重要的注意事项——用 `cin >>` 读 string 的时候，**遇到空格就停了**。如果用户输入的是"张 三"（中间有空格），`cin >> name` 只能读到"张"，"三"会被留在输入缓冲区里。

对于这个问题的完整解决方案，后面章节会讲到 `getline`。现在你只要知道：**如果你要读的内容包含空格，先用简单的单词测试，别用带空格的中文名。**

好了，今天的内容很简单但很重要——你从此可以让程序不再自言自语，而是跟用户互动了。下节课我们学怎么一次输入多个值。

## 💬 互动提问
1. "cout 和 cin 的箭头方向正好相反——你能想一个永不出错的记忆方法吗？"
2. "如果用户输入的不是数字，比如你让输入年龄，他输入 'abc'，会发生什么？试试看。"
3. "为什么 cin 读到空格就停？你觉得这个设计是方便还是麻烦？"

## 🖥️ 板书/屏幕要点
```
cout vs cin：

  cout << 数据    数据往外流（程序 → 屏幕）
  cin >> 变量     数据往内流（键盘 → 程序）

第一个互动程序：

  int age;
  cout << "请输入你的年龄：";
  cin >> age;                  // 用户敲 12 回车
  cout << "你今年 " << age << " 岁" << endl;
  // 输出：你今年 12 岁

cin 能读的类型：
  int / long long / double / char / string / bool

⚠ cin >> string：遇到空格就停！
  输入"张 三"，name 只拿到"张"
```

## ✏️ 随堂练习
写一个程序，询问用户"你最喜欢什么数字？"，然后读入这个数字，输出"哇！[数字] 也是我最喜欢的数字！"。编译运行，用不同的数字测试三次，确认每次都正确输出。

## 📌 小结（30秒总结）
cin 是程序的耳朵——通过 `>>` 箭头把键盘输入的数据收进变量。cout 的箭头往外，cin 的箭头往内——方向绝不能写反。能读 int、double 等多种类型，但读 string 时遇到空格就停。从今天起，你的程序不再是自说自话——它能跟你聊天了。

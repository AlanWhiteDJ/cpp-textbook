# 第2章 第4课时：编译 vs 运行——代码变成程序的"两步走"

**时长**：15-20分钟 | **对应教材**：第2章 · 2.4-2.5节

## 🎯 本课时目标
- 学生能清晰区分"编译"和"运行"两个步骤
- 学生理解编译器的作用——翻译 C++ 代码为机器语言
- 学生学会写单行注释和多行注释

## 📝 讲稿正文

上节课我们把 Hello World 的六行代码拆开看了一遍。今天我们要聊一个经常被忽略但非常重要的问题——当你点击那个"运行"按钮的时候，到底发生了什么？

很多同学以为"运行"就是一步到位——点一下按钮，程序就跑起来了。其实，你点下按钮的瞬间，计算机偷偷做了**两件事**。

**第一步：编译。第二步：运行。**

我们一个一个来说。

编译是什么？编译就是把你的 C++ 代码翻译成计算机能看懂的机器语言——就是那些 0 和 1。做这件事的程序叫**编译器**。

我给大家一个非常贴切的比喻：**编译器就像一个严格的中文老师。** 你写了一篇作文交给他，他一个字一个字地检查。你的作文里有一个错别字——比如少了个分号——这个老师就把作文扔回给你，说："这里有个错误，改完再交。"他不让你通过，程序就不会运行。

所以，如果你的代码有语法错误——少分号、少括号、拼错关键字——编译器会报错，屏幕上出现红色的错误提示。这时候程序**还没有运行**，只是翻译到一半卡住了。

只有翻译完全通过——0 个错误——编译器才会生成一个**可执行文件**。在 Windows 上，这个文件通常叫 `hello.exe`。这个文件里面装的全是机器语言，是计算机可以直接拿来执行的。

好，翻译完成了，接下来就是**运行**。运行就是计算机开始一行一行执行那个可执行文件里的机器语言指令。这时候你才真正在屏幕上看到 "Hello, World!" 出现。

所以，**编译是检查你写得对不对，运行是真正干活。** 编译不通过，就没有运行这回事。

那编译错误常见的有哪些呢？

最常见的是**语法错误**——少分号、少括号、大小写写错、用了中文标点。编译器的错误提示虽然全是英文，但一般会告诉你在第几行出了问题。比如 `expected ';' before 'return'` 就是告诉你在 `return` 前面少了一个分号。养成自己读错误提示的习惯，比一遇到红色就举手问老师强一百倍。

---

好，聊完编译运行，我们学一个新工具——**注释**。

注释是什么？注释是你写在代码里的"笔记"，是给人看的，不是给计算机看的。计算机会跳过所有注释，当它们不存在。

C++ 有两种注释写法。

**单行注释**：用两个斜线 `//` 开头，从这两个斜线开始到这一行结束，全都是注释。

```cpp
// 这是一个注释，计算机不会理它
cout << "Hello";  // 这也是注释，写在代码后面
```

**多行注释**：用 `/*` 开头，`*/` 结尾，中间可以写很多行。

```cpp
/*
   这是一个多行注释。
   可以写很多行。
   计算机一律不理会。
*/
```

那我建议大家从今天开始养成一个好习惯——**每一个你写的程序，开头都用注释写上：程序是干什么的、谁写的、什么时候写的。** 就像这样：

```cpp
/*
 * 程序：Hello World
 * 用途：我的第一个 C++ 程序
 * 作者：[你的名字]
 * 日期：2026-06-04
 */
```

这个习惯会让你受益匪浅。一个月后你回头看自己写的代码，靠这些注释就能秒懂当时在干什么。而且竞赛中，清晰的文件头注释也是好习惯的体现。

好，今天的内容就这些。下节课我们专门来聊聊初学者最容易犯的错误，以及怎么排查它们。

## 🖥️ 板书/屏幕要点
```
运行按钮按下去，发生两件事：

① 编译（Compile）
   — 编译器检查语法
   — C++ 代码 → 机器语言（0 和 1）
   — 有错 → 红色提示，不生成程序
   — 没错 → 生成 .exe 可执行文件

② 运行（Run）
   — 执行 .exe 中的机器语言指令
   — 你才看到 Hello World

注释（给人看的笔记）：
  // 单行注释（双斜线）
  /* 多行注释（斜线星号 ... 星号斜线）*/

好习惯：
  每个文件开头写：程序名、用途、作者、日期
```

## ✏️ 随堂练习
打开你的 Hello World 程序，在文件开头加上多行注释，写清楚程序名称、用途、你的名字和今天的日期。然后在 `cout` 那行后面加上单行注释，解释这行代码的作用。编译运行，确认注释没有影响程序输出。

## 📌 小结（30秒总结）
点击运行按钮，计算机做了两件事：先编译（翻译+检查语法），再运行（真正执行）。编译器是严格的老师，有错就不让过。注释是写给人看的笔记——`//` 单行，`/* */` 多行，计算机全跳过。从今天起每个程序都写文件头注释，这是受益终身的好习惯。

## 🎯 本课时目标

## 📝 讲稿正文

同学们，今天是第二章的最后一节课。我们要做两件事：第一，把新手最容易犯的错误一个个摆出来，让你以后碰到红色提示的时候不慌；第二，把第二章学的所有东西串起来。

先说错误。我们来看看四大"坑王"。

**第一大坑：忘了分号。**

C++ 的每一条语句都必须以分号结尾。但偏偏分号这个符号又小又不起眼，特别容易漏掉。我见过太多同学写了十行代码，结果因为一个分号，编译器暴跳如雷。

如果你看到编译器提示 `expected ';' before ...`——它就是在告诉你：某某地方前面缺一个分号。找到那一行，在末尾加上分号就行了。

**第二大坑：引号没配对。**

你写 `cout << "Hello;`——注意到没有，左引号有，右引号没了。这时候编译器不会友好地说"你少了个引号"，而是会报一连串奇怪的错误，可能牵扯到下面好几行。

这是一个很重要的经验：**编译器的错误提示有时候不会指向真正的出错位置。** 如果你在第 5 行少了一个引号，错误提示可能显示在第 7 行、第 8 行甚至更后面。所以排查的时候，先从报错的那一行往上看，通常真正的错误就在上面一两行。

**第三大坑：大小写写错。**

C++ 是**严格区分大小写**的语言。`cout` 和 `Cout` 是两个完全不同的东西——第一个是"输出"，第二个是"未定义的变量"。`main` 不能写成 `Main`，`return` 不能写成 `Return`。

这个坑最大的问题是——你怎么看都觉得自己写的是对的。因为你的眼睛自动把大小写"纠正"了。我的建议是：以后碰到莫名其妙的"未定义"错误，先检查大小写。

**第四大坑：中英文标点混用。**

这在初学者里极其常见，也极其隐蔽。中文的逗号 `，` 和英文的逗号 `,` 看起来几乎一模一样；中文的分号 `；` 和英文的分号 `;` 也几乎长一样。但计算机眼里它们完全是两个不同的字符——中文标点在 C++ 里是非法字符。

解决方案就一条：**写代码时始终用英文输入法。** 需要打中文时切到中文，打完立刻切回来。养成肌肉习惯——左手小指随时准备按 Shift 切换输入法。

好，四大坑讲完了。我教大家一个排查错误的"黄金三步法"：

第一步，看错误提示的第一行——它会告诉你出了什么类型的错误。第二步，看行号——错误提示里会告诉你第几行有问题。第三步，往上查——真正的错误往往在提示行号的上面一两行。这三步走完，90% 的错误你都能自己解决。

---

现在我们来回顾第二章的全部内容。第二章的标题叫"你的第一个程序"，我们学了什么？

第一条主线：从零到 Hello World。你安装了 IDE → 新建了 .cpp 文件 → 敲了六行代码 → 编译运行 → 看到了 Hello World。

第二条主线：理解代码。你知道 `#include <iostream>` 是拿工具，`using namespace std` 是开场白，`int main()` 是入口，`cout` 是输出，`return 0` 是正常结束。

第三条主线：理解过程。编译是翻译+检查，运行是真正执行。注释是给人看的笔记，编译器不理会。

第四条主线：学会排查。分号、引号、大小写、中英文标点——这四个坑你认识了，以后碰到红色提示就不慌了。

好了，第二章结束。下一章我们要让程序学会"记住东西"——一个叫**变量**的东西。准备好你的记忆盒子了吗？

## ✏️ 随堂练习
老师给出以下"有 bug 的代码"，请学生找出所有错误并改正。一共 4 个错误：

```cpp
include <iostream>
using namespace std

int Main() {
    cout << "I love C++！"
    return 0;
}
```

（答案：缺少 #、缺少分号、Main 应小写、中文感叹号应改为英文标点、cout 行缺少分号。）

# 第3章 第2课时：声明与初始化——给变量"安家"

**时长**：15-20分钟 | **对应教材**：第3章 · 3.2节、3.5节

## 🎯 本课时目标
- 学生能正确使用三种方式声明并初始化变量
- 学生能理解"未初始化变量的危险"
- 学生养成"声明即初始化"的编程习惯

## 📝 讲稿正文

上节课我们知道了变量是什么——内存里贴了标签的格子。今天我们来深入聊聊"声明"和"初始化"这两个动作，以及一个非常重要的安全问题。

先回顾一下。声明一个整数变量很简单：

```cpp
int age;
```

这一行做了两件事：第一，在内存里找了一个空位置；第二，给这个位置贴上了标签 `age`，规定它只能放整数。

但是！这里有一个关键问题——刚刚声明的 `age`，里面装着什么？

大家想一想。你去超市存包，打开一个空柜子。柜子里有什么？可能什么都没有，但也可能上一个顾客忘了拿走一张小票。内存也一样，你声明了一个变量但没有给它赋值，它里面的值是**不确定**的——可能是 0，可能是 32767，可能是任何数字。

我们来看一个危险的例子：

```cpp
int score;
cout << score;   // 输出什么？可能是垃圾值！
```

这个垃圾值从哪里来的？你声明 `score` 的时候，操作系统只是"把这个位置划给你用"，但没有帮你把里面的旧数据清空。所以 `score` 里残留着之前某个程序留下的"垃圾数据"。你把它输出来，就是一个莫名其妙的数字。

这就是**未初始化变量**的危险。它是 C++ 中最常见的 bug 来源之一，而且它不会报错——程序开开心心地跑完了，但是结果是错的。你连自己错了都不知道。

那怎么避免？很简单——**声明的同时就初始化。** 如果你实在不知道初始值该设多少，就设 0：

```cpp
int sum = 0;        // 累加器，从 0 开始
int count = 0;      // 计数器，从 0 开始
int maxVal = -1;    // 最大值标记，初始设一个很小的数
```

大家记住一句话：**声明即初始化。** 这五个字的价值，你们以后会越来越体会到。

好，现在我们来看看 C++ 里声明并初始化一个变量有几种写法。以下四种写法效果完全一样——都是声明一个 int 变量 `score` 并设为 100：

```cpp
// 写法一：声明和赋值分开
int score;
score = 100;

// 写法二：声明同时初始化（最推荐）
int score = 100;

// 写法三：用括号初始化
int score(100);

// 写法四：用大括号初始化（C++11 风格）
int score{100};
```

四种写法都可以，但我建议大家统一用**写法二**——`int score = 100;`。它最直观，跟你数学课上学的一模一样。写法一多写了一行，浪费；写法三和四虽然也是对的，但你现在不需要纠结它们的区别。

好，最后我们来做一个实验。请大家在 IDE 里运行下面这段代码：

```cpp
int a = 10;
int b = a;       // b 得到 a 的值（10）
a = 20;          // 修改 a
cout << a << " " << b << endl;
```

输出是什么？a 变成 20 了，b 还是 10。为什么？

因为 `int b = a;` 这句话的意思是"把 a 当前的值（10）复制一份，放进 b 的柜子"。从此 a 和 b 是两个独立的柜子了——你改 a 的值，不影响 b。就像你把作业借给同桌抄——他抄完以后你在你的作业上改答案，他的不会跟着改。

好，今天的核心就两个词：**初始化**和**独立**。变量不初始化是定时炸弹，变量赋值是复制而不是引用。这两条记住，下节课我们学变量的命名规则。

## 🖥️ 板书/屏幕要点
```
⚠ 未初始化变量的危险：

  int score;       ← 柜子里有垃圾数据！
  cout << score;   ← 输出不可预知

  解决方案：声明即初始化！
    int sum = 0;
    int count = 0;

四种初始化写法（效果相同）：
  ① int score; score = 100;
  ② int score = 100;         ← 推荐！
  ③ int score(100);
  ④ int score{100};

变量赋值 = 复制值，不是关联：
  int a = 10;
  int b = a;   // b = 10（复制）
  a = 20;      // b 不变！a 和 b 是独立柜子
```

## ✏️ 随堂练习
写程序声明三个未初始化的 int 变量，分别输出它们的值。反复运行三次，观察每次的值是否一样。然后改为声明时初始化为 0，再运行。对比结果，在笔记上写下你的发现。

## 📌 小结（30秒总结）
未初始化的变量装的是垃圾数据——不报错但结果是错的，这是 C++ 里最隐蔽的 bug。养成"声明即初始化"的习惯，不确定就设 0。变量的赋值是复制而不是关联——a 改不带动 b 改。用 `int x = 值;` 这种写法最简单清晰。

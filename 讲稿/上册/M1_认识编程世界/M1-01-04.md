# 第1章 第4课时：程序是怎么跑起来的——IPO 模型解析

**时长**：15-20分钟 | **对应教材**：第1章 · 1.5节

## 🎯 本课时目标
- 学生能完整说出 IPO 三个字母分别代表什么
- 学生能用 IPO 模型分析任意一个程序的运行过程
- 学生能在脑海中建立"输入 → 处理 → 输出"的思维骨架

## 📝 讲稿正文

前面三节课，我们知道了计算机是什么、程序是什么、编程语言是干什么的。今天我们要回答一个最关键的问题——**从你写好程序到看到结果，中间到底发生了什么？**

这个问题我们用三个字母就能回答：**I、P、O。**

别笑，这真的就是全部。I 是 Input，输入；P 是 Process，处理；O 是 Output，输出。所有程序——不管多简单、不管多复杂——本质都是这三个步骤。

我们来看一个最熟悉的例子——你手机上的计算器 APP。

你打开计算器，按了"1""2""3""+""4""5""6""="。你的触摸屏接收了你按下的数字和符号，把它们送进了程序——这就是 **Input，输入**。

程序收到 123 和 456 以及那个加号，CPU 开始干活：把这两个数加起来——这就是 **Process，处理**。

算完了，结果 579 显示在屏幕上——这就是 **Output，输出**。

三个步骤，清清楚楚。

好，我们再换一个例子——你在搜索引擎里打字查东西。你敲了"今天天气"四个字然后按回车——Input。服务器收到你的查询，在海量数据库里翻找最匹配的结果——Process。搜索结果一排排显示在屏幕上——Output。

再换一个——你玩手机游戏，手指划过屏幕控制角色移动——你手指的滑动是 Input。游戏程序接收你的操作，计算角色应该往哪走、走多远、会不会撞墙——Process。角色在屏幕上移动了，画面更新了——Output。

大家发现没有？**I、P、O 这个模式是通用的。** 从最简单的计算器，到最复杂的 3D 游戏，骨架都是一样的。

那回到我们编程的世界。以后你写的每一个程序，设计的时候应该按什么顺序思考？

第一步，想清楚**输入**是什么——用户要输入什么？是一个数字？两个数字？还是一串文字？

第二步，想清楚**处理**是什么——拿到输入之后，你要做什么运算？做什么判断？

第三步，想清楚**输出**是什么——最后要把什么结果显示给用户？

这个思维模式，从现在开始就要建立。很多初学者一上来就埋头敲代码，敲了半天发现不知道该干什么。如果先画清楚 IPO，代码写起来就有方向了。

举个具体例子：如果让你写一个"算长方形面积"的程序，你先用 IPO 想一遍——

I：用户输入长和宽两个数字。P：长乘以宽。O：输出"面积是 XX"。想清楚这三步，代码还没写，你已经成功了一半。

好了，下节课是本章最后一节课，我们把这四节课的内容串起来，做一个总复习，然后看看自己在第一章学到了什么。

## 💬 互动提问
1. "你每天用手机的时候，能不能举出一个你用 IPO 模型分析的具体例子？"
2. "如果程序只有 Input 和 Output，没有 Process，会是什么样子？"
3. "你觉得写程序之前先想 IPO，和直接写代码，哪个更高效？为什么？"

## 🖥️ 板书/屏幕要点
```
IPO 模型：

  Input  →  Process  →  Output
  (输入)    (处理)      (输出)

例子1 — 计算器：
  I: 按 123 + 456 =
  P: CPU 做加法
  O: 显示 579

例子2 — 搜索引擎：
  I: 输入"今天天气"
  P: 服务器搜索匹配结果
  O: 显示天气信息

写程序的思考顺序：
  ① 输入是什么？
  ② 处理做什么？
  ③ 输出是什么？
```

## ✏️ 随堂练习
闭上眼睛，在心里走一遍"你用微信给朋友发一条消息"的完整过程。用手在纸上指出来每一步分别是 I、P、O 的哪一个？至少写出 3 个 Input 步骤。（提示：不只是打字，打开微信、点联系人这些都算。）

## 📌 小结（30秒总结）
IPO = Input → Process → Output。输入是信息进程序，处理是程序做运算和判断，输出是结果展示给用户。从计算器到搜索引擎到 3D 游戏，所有程序都遵循这个模型。写程序之前先画 IPO，你就成功了一半。从今天开始，用 IPO 的眼光看每一个你用的软件。

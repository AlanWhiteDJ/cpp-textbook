# 第8章 第2课时：完整计算器 + 好代码的三个习惯

**时长**：15-20分钟 | **对应教材**：第8章 · 8.4-8.5节

## 🎯 本课时目标
- 学生能独立写出完整的迷你计算器代码
- 学生能处理非法运算符和除零异常
- 学生养成"有意义变量名、适当注释、一致缩进"三个好习惯

## 📝 讲稿正文

上节课我们搭好了计算器的骨架——IPO 分析、选择结构、防御性编程。今天我们把所有零件拧紧，写出一段完整的、能用的、健壮的计算器代码。

在展示正确版本之前，我先给大家看一个"典型初学者版本"。你们来挑一挑毛病：

```cpp
// ❌ 错误版：隐患重重
#include <iostream>
using namespace std;

int main() {
    int a, b, result;
    char op;
    cin >> a >> op >> b;
    if (op == '+') result = a + b;
    if (op == '-') result = a - b;
    if (op == '*') result = a * b;
    if (op == '/') result = a / b;
    cout << result << endl;
    return 0;
}
```

大家来找茬——这个版本有什么问题？

（让学生找）

好，我来揭晓。至少有四个问题：第一，用 `int` 不支持小数——输入 `7/2` 只能得 3。第二，除法没检查除数为 0——输入 `8/0` 直接炸。第三，没有处理非法运算符——输入 `5^3` 不会给出任何提示。第四，用四个独立的 `if` 而不是 `else if`——效率低，而且即使前面匹配了也会继续检查后面的条件。

好，现在来看**正确版本**：

```cpp
#include <iostream>
using namespace std;

int main() {
    double num1, num2, result;   // 用 double 支持小数
    char op;                      // 运算符用 char

    cout << "=== 迷你计算器 ===" << endl;
    cout << "请输入算式（如 5 + 3）：" << endl;
    cin >> num1 >> op >> num2;    // 一次读三个值！

    if (op == '+') {
        result = num1 + num2;
    } else if (op == '-') {
        result = num1 - num2;
    } else if (op == '*') {
        result = num1 * num2;
    } else if (op == '/') {
        if (num2 == 0) {
            cout << "错误：除数不能为 0！" << endl;
            return 1;
        }
        result = num1 / num2;
    } else {
        cout << "错误：不支持的运算符 " << op << endl;
        return 1;
    }

    cout << num1 << " " << op << " " << num2 
         << " = " << result << endl;
    return 0;
}
```

来，我把关键的设计选择解释一遍：

第一，用了 `double` 而不是 `int`——支持小数运算，`7 / 2` 得到 3.5。

第二，用 `else if` 而不是四个独立 `if`——匹配到第一个条件后，后面的就不再检查，效率高。

第三，除法前检查 `num2 == 0`——防火墙，挡住崩溃。

第四，加了 `else` 分支处理非法运算符——用户输入 `5^3` 时友好提示"不支持的运算符"。

第五，`return 1` 而不是 `return 0`——当程序异常退出时，返回非 0 值告诉操作系统"出了问题"。

---

程序写对了，但我们不止于此——我们要写得**好**。从今天开始，养成三个习惯：

**习惯一：有意义的变量名。** `double a, b, c;` 三个月后自己都看不懂。`double num1, num2, result;` 一眼就知道是干嘛的。多打几个字母，省下半辈子的回忆时间。

**习惯二：适当写注释。** 不是每行都写——那反而干扰阅读。而是在关键位置写：比如除法前写一行"检查除数不为 0"，main 函数结尾写"程序正常结束"。让注释像路标，指路但不挡路。

**习惯三：一致的缩进。** 每一层大括号 `{}` 缩进 2 格或 4 格。大多数 IDE 按一下 Tab 键自动缩进——养成习惯：敲 `{` 就顺手缩进。缩进让代码像目录树一样层次分明，别人看得舒服，自己回看也舒服。

这三个习惯，从现在开始刻进你的肌肉记忆里。它们是区分"写代码的人"和"写好代码的人"的第一道坎。

```
完整计算器 —— 正确版要点：

  ① double 类型 —— 支持小数
  ② else if 链 —— 高效多分支选择
  ③ 除零检查 —— 防御性编程

好代码三个习惯：
  ✅ 有意义变量名：num1/num2/result > a/b/c
  ✅ 关键位置注释：像路标，指路不挡路
  ✅ 一致缩进：{} 内缩进2-4格，像目录树

变量名 | 注释 | 缩进 → 三件套
```

## ✏️ 随堂练习
把完整版计算器代码敲进你的 IDE，运行三次测试：
1. 输入 `10 + 3` → 应输出 13
2. 输入 `7 / 0` → 应提示"除数不能为 0"
3. 输入 `9 ^ 2` → 应提示"不支持的运算符"

每个测试都把输出截屏保存。然后给自己写的代码加上文件头注释（程序名、用途、作者、日期）。

## 📌 小结（30秒总结）
完整计算器 = IPO骨架 + double类型 + else-if选择 + 除零防火墙 + 非法运算符提示。三个好习惯从今天开始：有意义的变量名、关键位置写注释、一致缩进。你写的代码不只是给计算机看的，也是给人看的——包括一个月后的你自己。下节课，M1 全模块总复习！

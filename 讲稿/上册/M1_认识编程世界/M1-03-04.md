## 📝 讲稿正文

同学们，今天是第三章最后一节课。我们要学一个变量家族里的"特殊成员"——const 常量，然后把整章串起来。

大家想一想：在你的生活里，有些数字是"天经地义不变"的。一年有 12 个月，一个班最多 50 个学生，圆周率是 3.14159……这些数字在整个程序运行期间，都不应该被修改。

那问题来了——如果不小心改了怎么办？比如你写了一个程序计算月份，本来 `months = 12`，结果中间某行代码不小心写成了 `months = 10`。编译器不会报错，但你的程序结果全错了。

这时候，C++ 提供了一个工具——`const` 关键字。它的作用是：**把这个变量变成"只读"的，任何试图修改它的操作都会被编译器拒绝。**

写法很简单，在声明前面加一个 `const`：

```cpp
const int MONTHS = 12;       // 一年 12 个月，不会变
const int MAX_SCORE = 100;   // 满分 100，不会变
const double PI = 3.14159;   // 圆周率，不会变
```

注意一个命名习惯：**常量名通常用全大写字母**，多个单词用下划线连接。比如 `MAX_SIZE`、`CLASS_COUNT`。这样你一眼就能看出"这是一个常量，不能改"。

如果试图修改一个 const 常量——

```cpp
const int X = 10;
X = 20;   // ❌ 编译错误！
```

编译器会直接报错，告诉你"不能给只读变量赋值"。这就从根源上杜绝了"不小心改了一个不该改的值"这种 bug。

在信息学竞赛里，const 有几个经典用法：
- 数组最大大小：`const int MAXN = 100005;`
- 模数：`const int MOD = 1000000007;`

数组和模数你们现在可能还听不懂，没关系，以后会学到。现在只需要知道：**const 能让一个变量的值永久锁定，谁也改不了。**

---

好，常量的内容讲完了。现在我们来串第三章的全部内容。

第三章的标题叫"程序的记忆盒子——变量"。我们学了什么？

第一个核心概念：**变量是什么。** 内存里的一个贴了标签的格子。声明是找柜子贴标签，赋值是把东西放进去，使用是按标签取东西。

第二个核心概念：**命名规则。** 硬性规定三条——字母数字下划线、数字不开头、不用关键字。习惯三条——有意义、用英文、风格一致。

第三个核心概念：**等号的含义。** C++ 里的 `=` 是赋值号，不是等号。先算右边，再把结果放进左边。`x = x + 1` 在数学上是矛盾的，在 C++ 里是日常。

第四个核心概念：**初始化。** 未初始化的变量装的是垃圾数据——不报错但危险万分。养成"声明即初始化"的习惯。

第五个核心概念：**const 常量。** 不可修改的变量，加了 const 前缀，一般全大写命名。

大家看，就这么五个点，第三章就全了。从下一章开始，我们要学习更多类型的"柜子"——int 只是一个开始，还有装小数的、装文字的、装真假的。准备好打开类型的大门了吗？

## 🖥️ 板书/屏幕要点
```
const 常量 = 不可修改的变量

  const int MONTHS = 12;     // 一年 12 月
  const double PI = 3.14159;  // 圆周率
  MONTHS = 10;  // ❌ 编译错误，常量不能改

  命名习惯：全大写、下划线分隔  MAX_SIZE

第三章知识地图：

  变量概念      → 内存格子，声明/赋值/使用
  声明初始化    → int age = 13; 声明即初始化！
  命名规则      → 硬性规定 + 推荐习惯
  赋值 =        → 不是等于，是把右边放进左边
  const 常量    → 不可修改，全大写命名
```

## 📌 小结（30秒总结）
const 就是一个"上了锁的变量"——声明后谁也别想改，改了编译器就报错。第三章学完，你掌握了变量的全部基础：声明、初始化、命名、赋值、常量。这五块基石，是你以后写任何程序都离不开的。下一章，我们打开 C++ 的类型宝箱——六种容器帮你装下整个世界的数据。

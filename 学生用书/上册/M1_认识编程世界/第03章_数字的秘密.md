## 📖 故事引入

小刚写了一个程序，算全校有多少学生。他把人数存在一个变量里，数字是 1280——完全正常。

然后他又写了一个程序，算他玩的游戏里一共打了多少分。分数是 8500000000（85 亿）。

程序运行……显示的结果是 **5237120**。完全不对。

小刚懵了。"我没写错啊？"

他没写错算法。他选错了"数字的容器"——有些容器只能装小数字，硬塞大数字进去就会**溢出**，像一个小杯子接一整桶水，多的全洒了。

今天我们就来搞清楚：C++ 里有哪些"数字容器"，以及什么时候该用哪一种。

## 🧱 知识讲解

### 3.1  数据类型是什么

计算机的内存不是无限大的。当你写下一个数字，计算机需要在内存里给它分配空间。**数据类型就是告诉计算机："请给我准备多大的空间来存这个数"**。

就像你去超市买饮料，你得告诉店员："我要小杯 / 中杯 / 大杯"。不同的杯能装不同量的饮料。数据类型就是杯子的尺寸。

### 3.2  四种整数类型

C++ 里有四种主要的整数类型，从小到大排：

| 类型 | 关键字 | 占用空间 | 能表示的范围 | 适合场景 |
|------|--------|----------|-------------|----------|
| 短整型 | `short` | 2 字节 | -32768 ~ 32767 | 几乎不用 |
| 整型 | `int` | 4 字节 | 约 -21 亿 ~ 21 亿 | **最常用** |
| 长整型 | `long long` | 8 字节 | 约 ±9×10¹⁸ | 大数字 |
| 超长整型 | `__int128` | 16 字节 | 约 ±1.7×10³⁸ | 极少用 |

> 竞赛中绝大多数情况用 `int` 和 `long long`。记住：**`int` 能装到约 21 亿，超过这个数立刻换 `long long`。**

### 3.3  什么是溢出

如果给 `int` 存一个超过 21 亿的数，会发生什么？

```cpp
int a = 3000000000;  // 30 亿，超过 int 上限！
cout << a;            // 输出：-1294967296（一个奇怪的数字）
```

这叫做**溢出**——数字太大，容器装不下，多出来的部分"溢"成了负数或者乱七八糟的数字。

> 溢出不会报错！程序会开开心心地给你一个错误结果。这是溢出的可怕之处——你完全不知道算错了。

**记住**：如果你的题目涉及的数字可能超过 20 亿，直接声明为 `long long`。

### 3.4  无符号整数

有时候你知道一个数字绝对不可能是负数——比如人数、分数、年龄。这时可以用 `unsigned`（无符号）：

| 类型 | 范围 |
|------|------|
| `unsigned int` | 0 ~ 约 42 亿 |

但**竞赛中慎用无符号类型**。因为如果无符号数和有符号数混在一起计算，容易产生诡异的 bug。初学阶段，保持用 `int` 和 `long long` 就好。

### 3.5  浮点数：带小数点的数

整数用 `int`，那 3.14 怎么办？答案是用**浮点数**（小数点可以"浮来浮去"）：

| 类型 | 关键字 | 占用空间 | 精度 | 适合场景 |
|------|--------|----------|------|----------|
| 单精度 | `float` | 4 字节 | 约 7 位有效数字 | 几乎不用 |
| 双精度 | `double` | 8 字节 | 约 15 位有效数字 | **默认选择** |

> 竞赛中需要小数时，**一律用 `double`**。`float` 精度太低，容易出错。

### 3.6  sizeof：看看类型有多大

你可以用 `sizeof` 关键字来查一个类型占几个字节：

```cpp
cout << sizeof(int) << endl;       // 输出 4
cout << sizeof(long long) << endl; // 输出 8
cout << sizeof(double) << endl;    // 输出 8
```

### 3.7  类型选择速查表

| 场景 | 用什么 |
|------|--------|
| 计数、人数、年龄、小坐标 | `int` |
| 可能很大的数字（乘积、总分、大数据） | `long long` |
| 带小数点的计算（面积、平均分） | `double` |
| 字符（单个字母/符号） | `char`（下章讲） |

## ⚠️ 容易犯的错

### 错 1：忘记溢出，所有数字都用 int

❌ `int total = 100000 * 100000;` // 结果 100 亿，int 装不下

✅ `long long total = 100000LL * 100000;` // 用 long long，加 LL 后缀

### 错 2：整数除法当小数用

❌ `int a = 10 / 3;` // a = 3，不是 3.333...

✅ `double a = 10.0 / 3;` // a = 3.333...

> 两个整数相除，C++ 默认做整数除法，小数部分直接丢掉！

### 错 3：float 和 double 混用

❌ `float a = 3.14159265358979;` // 精度不够，后面的数字丢了

✅ `double a = 3.14159265358979;` // 精度足够

